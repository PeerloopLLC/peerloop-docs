# Conv Learnings - 2026-07-05

## 1. Off-grid Tailwind spacing silently renders 4× via the @theme bridge
**Topics:** tailwind, matt-design, spacing, astro

**Context:** `CourseSessionsActions` pills coded `px-14 py-6 gap-6` rendered as `padding: 24px 56px`, 66px tall — 4× the intended size.

**Learning:** `tokens-tailwind-bridge.css`'s `@theme` remaps ONLY Matt's 4px-grid spacing values `{4,8,12,16,20,24,32,40,48,64}` to literal px (`--spacing-16: var(--space-16)` = 16px). Any spacing utility whose number is NOT in that set (6, 10, 14, 2, …) falls through to Tailwind v4's default multiplicative scale = **N × 4px** — so `px-14`→56px, `py-6`→24px, `gap-6`→24px, `py-10`→40px. Authoring a compact component with off-grid values silently 4×'s it. 119 `src/` files use off-grid 6/10/14; most are benign (e.g. the stepper's `gap-6`=24px looked fine), so this needs a targeted audit of clearly-compact components, not a blind sweep. Snap true-bug sites to the nearest grid value (ties round up, per `project_spacing_snap_over_matt_exception`). Filed as [SPACING-OFFGRID].

**Pattern:** When a Matt-token component renders far larger/tighter than its class names imply, check whether a spacing utility uses an off-grid number; the bridge only covers the 10 grid values.

## 2. Peerloop "continue learning" means the session funnel, not the Modules curriculum
**Topics:** course-ux, cta, product-model

**Context:** Both the hero and sticky course CTAs pointed enrolled students at `/modules` (the static `ModulesTab` curriculum).

**Learning:** Peerloop's learning happens in **1:1 sessions**, not in module content — `ModulesTab` is read-only "what this course teaches." So a primary "continue" CTA that lands on Modules is passive and off-model. The right primary action is state-aware and session-funnel-centric: book next session → go to scheduled session → review completed sessions. The journey model (`CourseJourneyState`) already exposes everything needed (`nextSession`, `bookableCount`, `isComplete`, `completedCount/totalModules`); the CTA just ignored it.

**Pattern:** `buildCoursePrimaryCta(slug, journey)` — one shared helper feeding both the hero (CourseHeader `primaryCta` prop) and the sticky tab-strip CTA so they never diverge.

## 3. Reproduce mobile reflow by constraining a breakpoint-free element's width
**Topics:** dom-verification, plato, responsive

**Context:** The automation Chrome window resisted `resize_window` for most of the conv (stuck at ~1130px innerWidth), blocking true-mobile DOM checks.

**Learning:** For a component with NO internal breakpoints (e.g. `CourseSessionsActions`'s `flex flex-wrap`), temporarily setting `el.style.width='358px'` reproduces the mobile wrap/reflow exactly, since only the container width — not a media query — drives it. This let pill widths/rows be measured at desktop viewport. (Breakpoint-gated classes like `md:size-[32px]` still need the real viewport or a bare-class probe.) Separately: `resize_window` DID eventually take effect after several navigations (viewport dropped to 375×667 mid-conv), enabling direct real-mobile verification later.

**Pattern:** Constrain-width probe for breakpoint-free layout; bare-class injection probe (`document.createElement` + the class) to prove a breakpoint-gated utility resolves.
