# Conv Learnings - 2026-07-12

## 1. "Course completion" was procedurally conflated with "teaching readiness"
**Topics:** certificates, domain-model, flywheel

**Context:** The user flagged that the app used to have one "certificate" concept and now has two.

**Learning:** Pre-Conv-389, the ONLY way to obtain any `certificates` row was the teacher-recommends → admin-approves flow, and `recommend.ts` required the recommender to hold a `teacher_certifications` row for the course. So even a "you finished the course" credential had to be vouched for by a certified teacher — procedurally identical to "this person can teach." Only the `'teaching'` type had behavior (upserts `teacher_certifications`); `'mastery'` was display-only (labels/tints, nothing keyed off it); `'completion'` was the 2-day-old outlier. That made removing completion/mastery from the certificate concept clean.

**Pattern:** When a "type" enum has one member with real behavior and the rest are decorative, the decorative members are usually a modeling smell — collapse or rename them.

---

## 2. A completion credential needs no table — the enrollment already IS the record
**Topics:** schema, diploma, denormalization

**Context:** Deciding where a "Diploma" lives after removing it from `certificates`.

**Learning:** Course completion was already fully represented by `enrollments.status='completed'` + `completed_at` + the `user_stats.courses_completed` counter. The Diploma is a *rendering* of that (a page at `/diploma/[enrollmentId]`), not a new row. Added only one column — `enrollments.diploma_awarded_at` — as an idempotency/provenance marker (never cleared, unlike `completed_at` which is nulled on un-complete), so the award side-effects fire at most once even across un-complete → re-complete.

## 3. The same enrollment completes via TWO different criteria — centralize the side-effects
**Topics:** booking, completion, idempotency

**Context:** Tracing why the completion cert missed the session path.

**Learning:** An enrollment reaches `completed` via (a) `progress.ts` — student self-marks all `module_progress` rows complete; (b) `booking.ts checkEnrollmentCompletion` — a *completed session per module* (counts distinct `sessions.module_id` where status='completed'); (c) admin `force-complete`. Each guards on `status != 'completed'`, so exactly one fires per enrollment. Before the fix, the cert fired on (a)+(c) while the `courses_completed` stat bumped only on (b) — so credential and stat never co-occurred. Fix: one `onEnrollmentCompleted(db, enrollmentId, opts?)` hook does ALL completion side-effects (stat bumps + notifications + Diploma email), called from all three paths.

## 4. A JSDoc `*/` hidden inside prose silently truncates the whole block comment
**Topics:** typescript, tooling

**Context:** First `completion.ts` (then `certificates.ts`) failed vitest transform with `Unexpected "*"`.

**Learning:** The comment text `The *mastery*/*teaching* tiers` contains the literal `*/` sequence (from `mastery*/`), which closed the `/* … */` block comment early and turned the rest into invalid code. esbuild's error pointed at the wrong spot. Avoid `*/` inside JSDoc prose (write "mastery and teaching", not `*mastery*/*teaching*`).

---

## 5. Tightening a CHECK constraint breaks every test fixture that used the old values
**Topics:** testing, schema, migration

**Context:** After `certificates.type CHECK ('teaching')`, ~9 test files failed — most at `beforeAll` INSERT (CHECK violation), plus `recommend.test.ts` on type-validation ordering.

**Learning:** Test fixtures used `'completion'`/`'mastery'` generically to exercise the cert apparatus. Converting them to `'teaching'` risked `UNIQUE(user_id,course_id,type)` collisions where a single setup inserted two now-same-type certs for one (user,course) — resolved by giving colliding certs distinct courses/users. Delegated this mechanical conversion to a subagent with a verify-to-green requirement. Also: the local dev D1 and PLATO seed scenarios had to be cleaned of completion-cert inserts (they'd violate the new CHECK at seed time).
