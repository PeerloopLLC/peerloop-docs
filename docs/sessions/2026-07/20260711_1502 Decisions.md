# Conv Decisions - 2026-07-11

## 1. Testing approach — walk live, then codify
**Type:** Strategic
**Topics:** timezone, testing

**Trigger:** "How can we test cross-tz booking/emails/day-of?"

**Options Considered:**
1. Automated suite only
2. Walk live, then codify ← Chosen
3. Live walk only
4. Confirm the suspected bug first

**Decision:** Do a live LA-teacher/Tokyo-student browser walk to observe the real SSR/hydration/emails and confirm-or-clear the suspected day-boundary bug, then codify what's observed into a durable automated suite.

**Rationale:** The live walk exercises the parts automation can't (real SSR render, actual per-recipient output) and de-risks the test spec; codifying afterward makes it regression-proof.

**Consequences:** Found + fixed the hero bug live; produced a 5-file test suite.

## 2. Hero fix direction — align to stored tz, server-side
**Type:** Implementation
**Topics:** timezone, astro

**Trigger:** Hero used browser tz while everything else uses stored `users.timezone`; the `[TZ-AUDIT]` comment (and the Conv-242 decision) claimed browser-tz was intentional.

**Options Considered:**
1. Align the hero to stored `users.timezone` (server-side) ← Chosen
2. Keep browser-tz as intentional per the comment

**Decision:** Render the hero next-session label server-side in the stored `users.timezone` via a new shared helper, consistent with every other surface; delete the browser-tz client script. This supersedes the Conv-242 "format client-side in the browser's TZ via `<time datetime>` + `astro:page-load`" decision — the last browser-tz session-time surface is now aligned to Model A.

**Rationale:** A mixed state (hero disagreeing with the list beneath it) is the worst outcome; the app is invested in stored-tz everywhere (Model A, Conv 371–374). The "server is UTC, must format client-side" premise was false — the shared helpers pass an explicit `timeZone` to `Intl`.

**Consequences:** `formatSessionRelativeWhen` added; 3 hero pages + `CourseHeader` changed; browser-tz script removed; regression tests added. Routed to `docs/decisions/05-ui-ux-components.md` (replaces the Conv-242 entry).

## 3. `formatSessionRelativeWhen` lives in shared `timezone.ts`, pure with an explicit `nowIso`
**Type:** Implementation
**Topics:** timezone

**Trigger:** Need a relative "Today/Tomorrow" label in the stored tz, testably.

**Decision:** Put the helper in `src/lib/timezone.ts` (keeps the module dependency-free) taking `nowIso` explicitly rather than importing the clock; callers pass `getNow().toISOString()`.

**Rationale:** Keeps `timezone.ts` pure and unit-testable with a fixed `now`; matches the existing helper style.

**Consequences:** Deterministic unit tests for the relative-day logic.
