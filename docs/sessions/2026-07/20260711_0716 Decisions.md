# Conv Decisions - 2026-07-11

## 1. Handle the orphaned homework-submission intent as an API bridge
**Type:** Implementation
**Topics:** plato, testing, homework

**Trigger:** The planned pure-UI "submit homework" intent had no reachable UI (orphaned `HomeworkTab`).

**Options Considered:**
1. API-bridge the submission now + file the orphaned UI as a product task ← Chosen
2. Fix the Matt wiring first (add Homework/Learn tab), then browser-submit
3. Skip the submission (capture differs by 1 row)

**Decision:** POST the submission via `/api/homework/[id]/submit` (the exact endpoint the API scenario uses), same class of move as the CUT-3 bridge; file `[HW-SUBMIT-UI]` for a proper fix.

**Rationale:** Preserves the row-identity goal and finishes the walk this conv; the un-browsable step becomes a bridge like any external cut. Fixing the wiring is a real but separate, novel UI change.

**Consequences:** `activities-walk` captured row-identical; `[HW-SUBMIT-UI]` in the backlog (Opus-tagged). The walk validates less of the homework UI (there is none) — expected given the finding.
