# Conv Learnings - 2026-07-06

## 1. Playwright test-count grep must exclude describe/hooks and be title-verified
**Topics:** testing, e2e, tooling

**Context:** Reconciling E2E counts (E2E-DOCS). A first pass with a loose `(^|[^.])test(\.(skip|only))?\(` regex counted `feed-badges.spec.ts` at 4; the file actually has 2 tests.

**Learning:** A loose `test(` matcher over-counts — it catches `test.describe`/`test.beforeEach`. Count with a matcher that excludes those, and cross-check by extracting the `test('…')` titles; reading the titles is the authoritative count. Disk truth was 28 files / 125 tests, not the 127 the loose grep implied.

**Pattern:**
```
grep -nE "^\s*test\(" file.spec.ts   # strict anchored count
# then extract titles to verify — the titles ARE the count
```

## 2. Two drift-check docs can disagree — reconcile against disk, not each other
**Topics:** docs-infra, testing

**Context:** `TEST-COVERAGE.md`'s detailed E2E table already listed all 28 spec files with correct per-file counts, while its own Summary row said 30 files and the sibling `TEST-E2E.md` was frozen at a 25-file / 105-test Session-390 snapshot.

**Learning:** When two drift-check docs disagree, neither is authoritative — recount from the source of truth (the spec files on disk) and bring both docs to that number. Don't propagate one doc's figure into the other; a doc can even be internally inconsistent (its detailed table right, its summary stale).

## 3. Hand-editing an outlined-vector wordmark: swap the glyph + group-translate to re-kern
**Topics:** astro, ui

**Context:** BRAND-CASE "fix logo too" — the Large-variant `peerloop-wordmark.svg` draws letters as outlined vector paths (not text); path 5 was a capital "L" (full-cap stem + baseline foot), so the Large logo genuinely read "PeerLoop".

**Learning:** To change one glyph in an outlined wordmark without redrawing the rest: replace that path with a minimal stem (`M x 0 H x2 V baseline H x Z` for a lowercase "l"), wrap the following glyphs in `<g transform="translate(-Δ 0)">` to close the kerning gap (match Δ to an adjacent ink gap — here 7.683, matched to the "oo" ink gap ~2.05), then tighten **both** the `viewBox` width and the container pixel width proportionally so `preserveAspectRatio="none"` doesn't stretch (viewBox 108.245→100.562, `w-[108px]`→`w-[100px]`).

**Pattern:**
```
# file:// is blocked by the Chrome extension — serve over HTTP to verify visually
python3 -m http.server
# render at real size + a 4× zoom + an Inter reference; xmllint for well-formedness
```

## 4. Icon usage-weight is wildly uneven — icon-paths.ts is near-legacy
**Topics:** ui, architecture

**Context:** ICN-NS audit of the 4 coexisting icon systems.

**Learning:** The Astro `icon-paths.ts` registry has only **6** `<Icon>` call sites, vs `icons.tsx`'s **175** importers. That asymmetry is the key to a safe reconciliation: retiring `icon-paths.ts` (cheap) erases the sharpest collision categories without touching the dominant React system. Also: suspected intra-system dupes are often distinct glyphs (`MoreIcon` horizontal ⋯ ≠ `DotsVerticalIcon` vertical ⋮) — verify path data before claiming a dedup.
