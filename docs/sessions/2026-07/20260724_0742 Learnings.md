# Conv Learnings - 2026-07-24

## 1. Undefined spacing tokens silently fall through to stock Tailwind (N×4px)
**Topics:** astro, tailwind, spacing

**Context:** M1 hero "compression" via `py-24 sm:py-28` barely shrank the hero (360→350px). A live DOM measure showed `padding-top: 112px`, not the expected 28px.

**Learning:** The project defines `--spacing-N = N px` only for a specific set `{4,8,12,16,20,24,32,40,48,64}`. A value OUTSIDE that set (`py-28`, and `px-40`/`px-48` when unscanned) does NOT error — Tailwind v4's dynamic scale resolves it to `N × 0.25rem = N×4px` (stock), so `py-28` = 112px, not 28px. This is invisible to tsc/lint/build; only a runtime `getComputedStyle` reveals it. **Only use spacing values from the defined set.** Fixing to defined tokens (`py-16 sm:py-20`) dropped the hero to 166px (~194px reclaimed, hitting the plan target). Captured in `memory/project_spacing_snap_over_matt_exception.md`.

**Pattern:**
After any spacing edit on a visual component, DOM-measure the element (`getComputedStyle(el).paddingTop`) rather than trusting the class string — a green build ≠ correct spacing.

## 2. A provenance-marker flip cascades into prov:sweep registry errors
**Topics:** provenance, prim-registry

**Context:** Flipping `CourseHeader` `@matt-source`→`@matt-inspired` (honest, since the compression departs from the 1:1 frame) added 2 `prov:sweep` errors — the registry still lists it as `matt-sourced 517:8934`, so the sweep demands the stamp match.

**Learning:** For a *cosmetic* change to a `@matt-source` component, keep the `@matt-source` stamp and document the departure as a **Strict-B drift** (the codebase's established escape hatch). A true `@matt-inspired` reclassify needs the prov-registry update tracked under `[PROV-SWEEP-DEBT2]` — out of scope for a cosmetic pass. Don't flip provenance as a side effect of a visual change.

## 3. The full-suite run is worth it — it surfaces pre-existing residue
**Topics:** testing

**Context:** Running the full suite for the relabel change also failed `EarningsDetail.test.tsx` (expected `/settings/payments`, component correctly points at `/profile/payments`).

**Learning:** That failure pre-existed this conv (Conv-408 `/settings/*`→`/profile/*` flattening updated the component but missed the test).
