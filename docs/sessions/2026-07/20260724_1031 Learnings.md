# Conv Learnings - 2026-07-24

## 1. The project's custom spacing scale also governs `size-*`, not only padding/margin
**Topics:** astro, ui, tailwind

**Context:** Building the receipt page + M10 affiliation logo; wary of the `[[project_spacing_snap_over_matt_exception]]` trap (undefined `py-28` silently → stock 112px).

**Learning:** The custom `N=px` scale applies to `size-*` too — `size-24` resolves to 24px (the diploma uses it that way), not stock Tailwind's 96px. Defined values `{4,8,12,16,20,24,32,40,48,64}` are safe; anything off-scale silently falls through to stock 0.25rem/unit. `CourseHeader.tsx`'s local idiom is the arbitrary `size-[20px]` form, which is trap-proof regardless of scale config — matched it for the logo.

**Pattern:** For any width/height/size utility, use a defined-scale value or the explicit `[Npx]` bracket; never an off-scale bare number. DOM-measure after spacing/size edits.

## 2. Private record pages need an owner-scoped loader + discriminated result, unlike the public Diploma
**Topics:** astro, auth, ssr

**Context:** The receipt carries financial data (`transactions`), so it can't follow the public `/diploma/[id]` pattern (no auth, enrollment-scoped only).

**Learning:** Enforce ownership in the SQL (`WHERE e.student_id = ?`) so a non-owner gets `null` — rendered as an indistinguishable "Not Found" (no enumeration leak). Redirect logged-out viewers to `/login?redirect=` in the page frontmatter before calling the loader. A discriminated result (`ok` / `no-payment` / `not-found`) lets the page distinguish "owned but free enrollment (no transaction)" from "not yours / doesn't exist" using a single `LEFT JOIN` row.

**Pattern:**
```sql
LEFT JOIN transactions t ON t.enrollment_id = e.id
WHERE e.id = ? AND e.student_id = ?
-- null row = not-found; row with null transaction_id = no-payment; else ok
```

## 3. `communities` carried two image fields already; a brand logo is a genuine third
**Topics:** d1, schema

**Context:** M10 disposition said author `logo_url`; discovered `communities` already has `cover_image_url` (wide banner) + `icon` (emoji/name).

**Learning:** Reusing either would be wrong — a cover banner and an emoji are not a brand logo. Added `logo_url` as a distinct nullable column and documented the distinction inline so a future reader doesn't read it as redundant. Existing INSERTs are unaffected (new column not in their column lists → defaults NULL); only the dev seed opts in.
