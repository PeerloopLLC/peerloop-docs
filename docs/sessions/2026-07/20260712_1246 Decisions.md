# Conv Decisions - 2026-07-12

## 1. Course completion is a "Diploma", not a certificate; certificates are teach-readiness only
**Type:** Strategic
**Topics:** certificates, domain-model, diploma

**Trigger:** User: "remove the certificate label for completion and use a different name ‚Ä¶ Diploma for completion. Drop Mastery."

**Options Considered:**
1. Keep completion as an auto-issued `certificates` row (the CERT-AUTO first cut)
2. Completion = a separate non-certificate "Diploma" concept; `certificates` = teaching only ‚Üê Chosen
3. Leave the conflation, just document it

**Decision:** `certificates.type` CHECK tightened to `('teaching')` (retired `completion`/`mastery`). Course completion becomes a **Diploma** ‚Äî derived from the completed enrollment (no table), rendered at `/diploma/[enrollmentId]`, counted in `user_stats.courses_completed`.

**Rationale:** The original single credential was teach-readiness ("mastery" = able to teach). A completion event is factual and shouldn't overload the "certificate" word, which internally meant "can teach." Only `'teaching'` had behavior; `'mastery'` was decorative.

**Consequences:** Deleted `src/lib/certificates.ts`; new `src/lib/completion.ts`, `DiplomaEmail.tsx`, `/diploma/[id].astro` + loader, `/api/me/diplomas`, `DiplomasSection`. Reworded the teaching cert email/page. This is a deliberate partial walk-back of the same-conv [CERT-AUTO].

## 2. Unify all completion side-effects into one idempotent hook
**Type:** Implementation
**Topics:** booking, completion, stats

**Trigger:** The completion cert fired on only 2 of 3 completion paths; the `courses_completed` stat only on the 3rd.

**Options Considered:**
1. Unify ‚Äî one `onEnrollmentCompleted()` for all 3 paths ‚Üê Chosen
2. Cert-parity only ‚Äî add issuance to the session path, leave the stat divergence
3. Leave as-is / document

**Decision:** `onEnrollmentCompleted(db, enrollmentId, opts?)` (idempotent via `diploma_awarded_at`) does the full set ‚Äî `courses_completed` + `students_taught` bumps, `notifyEnrollmentCompleted` (student + teacher), `DiplomaEmail` ‚Äî called from `progress.ts`, `force-complete.ts`, and `booking.ts` (replacing its inline block).

**Rationale:** Credential + stats must co-occur on every completion path.

**Consequences:** Session path no longer sends the Diploma **email** (no Resend key in scope there ‚Äî documented limitation); in-app notification + stat + diploma marker still fire.

## 3. Artifact = rendered HTML page, not a generated PDF
**Type:** Implementation
**Topics:** diploma, certificates, cloudflare

**Trigger:** "generating the certificate" ‚Äî what is the downloadable artifact?

**Options Considered:**
1. Server-rendered HTML page (printable / Save-as-PDF) ‚Üê Chosen
2. Generated PDF (pdf-lib on Workers) stored in R2

**Decision:** Render the credential as an SSR page (`/diploma/[id]`, `/certificates/[id]`), reusing the `/verify` loader pattern; "download" = browser Print-to-PDF.

**Rationale:** Cloudflare-Workers-native, completable in-conv, and it fixed the broken `/certificates/[id]` link in one move. PDF would add a dependency + pipeline and push to multi-conv.

**Consequences:** No R2 artifact; `certificate_url` set to the relative page path.
