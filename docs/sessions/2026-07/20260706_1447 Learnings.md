# Conv Learnings - 2026-07-06

## 1. The off-grid-spacing 4× bug only bites Matt-convention components
**Topics:** tailwind, spacing, matt

**Context:** [SPACING-OFFGRID] — 329 off-grid `value-6` sites looked alarming, but most were harmless.

**Learning:** The `tokens-tailwind-bridge.css` `@theme` remaps only the 4px-grid set to literal px, so off-grid `p-6`=24px = standard Tailwind. A **legacy** component (`text-sm`/`rounded-lg`/`secondary-*`) author thinks in standard Tailwind, where off-grid `px-6`=24px is exactly what they intended → renders correctly (accidentally-correct). The bug ONLY fires when a **Matt-convention** author (`text-body-*`/`rounded-8/12`/`border-border-default`) adopts the grid's "N px literal" mental model and writes an off-grid value expecting N px. That's the triage discriminator; it's why a "blind sweep" is wrong.

## 2. Faithful multi-width responsive testing = exact-width same-origin iframe
**Topics:** testing, browser, responsive

**Context:** [MINWIDTH] — needed true 320/375px renders; `resize_window` was laggy and the user's DevTools device panel was invisible to automation.

**Learning:** `resize_window` is laggy/unreliable (`innerWidth` read 341 then 1087 for the same request). MCP automation measures/screenshots the **real page viewport, not a DevTools device-mode panel** (`innerWidth` stayed 1087 with the panel open). Container-forcing lies (media queries key off the real viewport). An **exact-width same-origin iframe** is a genuine separate viewport — Tailwind `min-width`/`sm:`/`lg:` evaluate against the iframe width. `minFitWidth = document.documentElement.scrollWidth`. Offender detection must exclude `position:fixed` + `overflow-x`-clipped ancestors (off-canvas drawers pollute otherwise). Saved as memory `reference_responsive_iframe_harness`.

## 3. `flex justify-between` for a top/bottom split fails under tight height; `mt-auto` + overflow is the fix
**Topics:** css, flexbox, sidebar

**Context:** [SIDEBAR-MOBILE].

**Learning:** `flex flex-col justify-between` with two children goes to negative free space on short viewports — engines differ (Safari overlaps top-at-top/bottom-at-bottom; Chrome falls back to flex-start and relies on scroll). `mt-auto` on the bottom child is deterministic across engines: pinned to the bottom when there's room, flows into scroll when tight — no overlap. Separately, a fixed-height sticky sidebar (`lg:h-[calc(100vh-32px)]`) needs `overflow-y-auto` or its overflowing tail is unreachable (stranded 184px off-screen here).

## 4. A CC-launched dev server is owned by the CLI process, not the conversation
**Topics:** claude-code, dev-workflow

**Context:** Dev-server investigation.

**Learning:** `npm run dev` started via a Claude background shell is a child of the long-lived `claude` CLI process (PID 48120), not the conversation. `/clear` wipes context but does NOT restart the CLI or kill/untrack its background shells — so the server survives across cleared convs (ran 22h here) and makes `/quit` warn about a running background shell. Implication: if CC launches a dev server, it must tear it down (the new ephemeral-situational workflow) or it orphans.

## 5. Route inline gaps through a CSS var for VT-safe responsive spacing
**Topics:** css, view-transitions, tailwind

**Context:** [SIDEBAR-MOBILE] seam-uniformity tweak.

**Learning:** The sidebar's gaps are inline styles specifically because island-unique arbitrary utilities drop on `transition:persist` VT swaps (Conv 250). To make those inline gaps responsive without reintroducing the drop OR needing `!important`, change `style={{gap:'18px'}}` → `style={{gap:'var(--sb-item-gap, 18px)'}}` and override the variable in a plain (non-arbitrary-variant) `@media` block. The inline style stays (VT-safe); one media-query var-override cascades to every consumer.

**Pattern:**
```jsx
// inline style stays VT-safe; media query overrides the var
style={{ gap: 'var(--sb-item-gap, 18px)' }}
```
```css
@media (max-height: 500px) {
  [data-prov-name="Sidebar"] { --sb-item-gap: 12px; }
}
```

## 6. grep can match doc-comment prose — verify the hit is live code
**Topics:** tooling, audit

**Context:** [SPACING-OFFGRID] — `Button.tsx` flagged `p-10`/`px-10` at lines 14/16/17.

**Learning:** Those were inside the file's `/* … */` doc-comment describing Matt's spec, not live classNames (real code used grid `p-12`/`px-12 py-8`). An audit grep over source must confirm a hit is executable code before treating it as a finding — filtering `* `-prefixed comment lines, or reading the surrounding context.
