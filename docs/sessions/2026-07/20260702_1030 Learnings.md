# Conv Learnings - 2026-07-02

## 1. Calibrate a classification heuristic against the ground-truth case BEFORE building the classifier
**Topics:** cc-workflow, memory

**Context:** Phase 2 of MEM-CAP-ARCH was to build an auto-re-tier enforcer for the HOT/COLD memory grammar.

**Learning:** Running the candidate heuristics against the hand-tiered file (the intended-correct state) exposed that content-keyword re-tiering misfires — 2/2 false demotions on ground-truth, because "NEVER"/"always" appear in situational project facts too and a CLAUDE.md cross-ref is non-discriminating. Only `**Load when**` was a high-precision demotion signal (0 false positives). The calibration killed a whole planned feature before any code was written; the honest signal was "this classifier can't be trusted," which reframed the entire effort. Mirrors `feedback_heuristic_calibration` (run against the canonical case first).

## 2. A strong contrarian framing can be half-wrong — audit before cutting
**Topics:** cc-workflow, docs-infra

**Context:** The opening contrarian claim was "the HOT tier duplicates CLAUDE.md," used to motivate collapsing MEMORY.md.

**Learning:** The per-entry audit found **0 pure duplicates**. The real structure was a 3-way conflation: 8 redundant echoes (rule in CLAUDE.md, HOT pointer redundant), 10 unique always-on rules absent from CLAUDE.md (incl. security/safety guards), 4 mis-filed situational entries. Acting on the un-audited claim would have deleted 10 genuine always-on rules. The sharpened finding was more useful than the original hypothesis and only emerged from the audit; owning the overstatement openly built trust.

## 3. Two always-loaded files make rule-relocation context-neutral — the decision is about framing, not cost
**Topics:** docs-infra, memory

**Context:** Weighing whether to move always-on rules from MEMORY.md's HOT tier into CLAUDE.md.

**Learning:** Both CLAUDE.md and the first 25KB of MEMORY.md auto-load every session, so moving a rule between them doesn't change what's in context — it changes *which file's cap it counts against* and *how it's framed* (instruction vs background). This dissolved the apparent tradeoff: consolidating rules into CLAUDE.md is a pure clarity win (one home), paid only in CLAUDE.md's soft size budget (handled by `/r-prune-claude`).
