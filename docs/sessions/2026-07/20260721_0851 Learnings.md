# Conv Learnings - 2026-07-21

## 1. Blanket `npm audit fix` can be both churny AND ineffective — prefer targeted in-range moves
**Topics:** dependencies, security, tooling

**Context:** NPMVULN triage of 21 audit advisories.

**Learning:** `npm audit fix` (no `--force`) dry-ran as **+205 / −2 / ~52 packages and still reported 21 vulnerabilities remaining** — heavy lockfile churn that doesn't even resolve the advisories (the in-range fixes it makes aren't the ones that clear them without major bumps). The durable path was a **lockfile-only, in-range `npm update <pkg>`** of just the two direct deps whose ranges already permitted the fixed transitives — net −23 lines, no `package.json` edit, no major bumps. Always dry-run and read the trailing vuln count before trusting `audit fix`.

**Pattern:** `npm update resend devalue` moved resend 6.10.0→6.17.2 (cascading svix + uuid ≥11.1.1) and devalue 5.7.1→5.8.2 within existing ranges — zero `package.json` change.

## 2. A transitive vuln is often fixable in-range without touching the parent major
**Topics:** dependencies, astro

**Context:** devalue (high) enters via astro's `devalue ^5.6.3`.

**Learning:** The patched devalue 5.8.2 is in-range of astro-6's own `^5.6.3` constraint, so `npm update devalue` cleared the high **without** needing the Astro 7 migration. Don't assume a transitive fix requires bumping the parent — check the parent's declared range against the patched version first.

## 3. Astro 7 migration is far more contained than a "4-major + compiler swap" sounds
**Topics:** astro, migration, dependencies

**Context:** [ASTRO7] read-only assessment + empirical worktree de-risk.

**Learning:** Despite being a coordinated 4-major bump (astro + adapter 14 + react-integration 6 + vite 8) plus a Go→Rust compiler swap, the empirical de-risk was **green with zero code changes**: `astro build` 0 errors (Rust compiler accepted all 90 `.astro`), tsc 0, astro-check 0, 6540 tests pass, no peer conflicts, adapter-14 built clean. 6 of 9 documented breaking changes had zero exposure for us. The migration reduces to the dependency bump + one optional config line (`compressHTML: true`).

---

## 4. Throwaway git worktree converts a theoretical build risk into a measured fact at zero cost
**Topics:** tooling, migration, cc-workflow

**Context:** The biggest ASTRO7 unknown was whether the stricter Rust compiler would reject any of 90 `.astro` files.

**Learning:** A `git worktree add -b <spike>` + full dependency install + `astro build` in an isolated tree answered "does it compile?" definitively without risking `jfg-dev-14`. The install was 35s; the whole de-risk (build + tsc + check + 6540 tests) a few minutes. This is the [DEPEXP]-endorsed pattern for any dependency experiment — and the spike is torn down after because it's fully reproducible from the documented command.

**Pattern:** `git -C <repo> worktree add -b jfg-<spike> <path> <base>` → `cp .dev.vars` → `npm install <set>` → build/check/test → `git worktree remove --force` + `git branch -D`.

## 5. Astro 7 also clears most of the deferred NPMVULN dev-tail
**Topics:** astro, security, dependencies

**Context:** `npm audit` on the v7 worktree.

**Learning:** On Astro 7 the audit dropped 17→7 — the astro roll-up, vite, esbuild, and postcss advisories are all subsumed by the v7 upgrade (vite 8, refreshed transitive tree). So the NPMVULN dev-tail and the ASTRO7 migration are complementary: doing v7 later will retire most of what NPMVULN deferred, leaving only undici/ws (miniflare), brace-expansion (eslint), fast-uri/js-yaml (@astrojs/check).
