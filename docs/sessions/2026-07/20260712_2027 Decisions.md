## 1. Extract a shared loadCourseDetail loader so PUT returns the full enriched shape
**Type:** Implementation
**Topics:** api, ssr

**Trigger:** [CERT-ROWSHAPE-FOLLOWUP] — PUT `/api/me/courses/[id]` returned a bare row+tags while GET returned the fully-enriched `CourseDetail` (Conv 391 flagged the residual).

**Options Considered:**
1. Duplicate the enrichment query in the PUT handler
2. Extract a shared `loadCourseDetail(db, courseId, creatorId)` used by both GET and PUT ← Chosen
3. Leave PUT thin, keep the client's follow-up GET

**Decision:** Extract `loadCourseDetail()` from GET; PUT calls it and returns the full `CourseDetail` (6 joined arrays + boolean coercion). The CourseEditor nested detail-editor drops its now-redundant follow-up GET.

**Rationale:** One loader = one enriched shape for read and mutate, no duplication; drops a client round-trip (2→1); also fixed a latent number-vs-boolean merge on `is_active`/`lifetime_access`. +1 regression test.

**Consequences:** PUT and GET can't diverge. 5 gates green (suite 6893). Committed `ce6fcd81`.

## 3. Admin Diploma visibility lives on the Enrollments admin, not a separate view
**Type:** Implementation (Architecture)
**Topics:** admin, d1, ui

**Trigger:** [ADMIN-DIPLOMA] — completed courses were visible in admin (Enrollments status filter) but a Diploma appeared NOWHERE in admin; the Certificates page is teaching-only.

**Options Considered:**
1. A separate "Diplomas" admin view/list
2. Surface the Diploma on the existing Enrollments admin (row action + detail field) ← Chosen

**Decision:** List + detail admin APIs return `diploma_awarded_at`; completed Enrollments rows get a "View Diploma" row action; the detail panel's Status section shows a Diploma field (View-Diploma link + awarded date).

**Rationale:** A Diploma has no table — it's derived from the enrollment (Conv-389 DIPLOMA model). A derived credential surfaces on its source entity's admin, so a separate Diplomas view would be a view over a non-existent table. Enrollments already IS the completion record.

**Consequences:** Admin can now see completed-course Diplomas. Verified live. Committed `086e293c`/`27775c6`. Routed to `docs/decisions/10-admin.md`.
