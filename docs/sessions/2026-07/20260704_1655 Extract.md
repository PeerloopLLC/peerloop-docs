Root cause: `nav_layout` was added to the base `migrations/0001_schema.sql` (line 63) at Conv 356, but `0001` is edited in place (not shipped as an incremental migration), so an un-reset staging DB never received it. Proposed Fix A (data) + Fix B (code guard).

Recommended `db:setup:staging:booking` + Fix B; gated on explicit go-ahead (destructive staging write).
