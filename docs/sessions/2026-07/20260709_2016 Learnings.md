# Conv Learnings - 2026-07-09

## 1. Vitest verbose-reporter line-counts over-count; JSON `assertionResults` is authoritative
**Topics:** testing, docs-infra

**Context:** Reconciling per-file test-case counts for TEST-PAGES.md / TEST-COVERAGE.md. A `grep -oE 'tests/pages/…' | uniq -c` over `--reporter=verbose` output gave inflated counts (e.g. TeacherProfile **80** vs true **58**, LoginForm 22 vs 21) because verbose repeats the file path on describe/summary lines, not just per-test lines.

**Learning:** For authoritative per-file case counts, use `--reporter=json --outputFile=…` and read `testResults[].assertionResults.length` per file. Grep-counting a human reporter's lines is unreliable for exact totals.

**Pattern:**
```
npx vitest run <dir> --reporter=json --outputFile=out.json
node -e 'const r=require("out.json"); for(const t of r.testResults) console.log(t.assertionResults.length, t.name)'
```
