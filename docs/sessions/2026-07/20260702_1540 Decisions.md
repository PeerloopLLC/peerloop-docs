# Conv Decisions - 2026-07-02

## 1. CTA mechanism = merge-into-tab-strip (then refined to reveal-on-stuck)
**Type:** Implementation
**Topics:** astro, cc-workflow

**Trigger:** STICKY-P2 needed a home for the detail-page primary CTA that stays reachable when the tall header scrolls off.

**Options Considered:**
1. Merge into the sticky tab strip (pure CSS, always visible) ← Chosen
2. Reveal-on-scroll dedicated bar (JS island)
3. Dedicated always-present slim bar (pure CSS, most redundancy)

**Decision:** Merge the CTA into the already-sticky `SubNav` tab strip via an opt-in `action` prop. Later refined (Decision 4) to hide-at-rest + reveal-on-stuck to remove at-rest duplication — converging toward option 2's behavior but via a small inline script rather than a React island.

**Rationale:** Zero new chrome, extends the Conv-359 sticky primitive, consistent with Phase-1's no-island ethos. The at-rest redundancy the merge accepted turned out to be worse than estimated (triple Enroll for not-enrolled), motivating the reveal refinement.

**Consequences:** New `SubNav` `action` prop + `SubNavAction` type; `CourseRail` forwarding; reveal-on-stuck script.

## 3. Side-rail CTA = top-bar-only action + Join in the header
**Type:** Implementation
**Topics:** astro, cc-workflow

**Trigger:** In side-rail, the community `action` orphaned at the bottom of the tab rail (community lacks a rail stepper to host it, unlike course).

**Options Considered:**
1. Top-bar-only action + move non-member Join to the header ← Chosen
2. Leave as-is (Join/Manage pill at the bottom of the rail)

**Decision:** Render the merged `action` only in top-bar layout (side-rail already self-pins); put the non-member Join in the community **header** so it's naturally placed and present in EVERY layout.

**Rationale:** Matches course's behavior (action is a top-bar merge feature), un-clutters the side-rail, fixes the gap layout-independently, and collapses to a single wired Join instance (no dual-wiring).

**Consequences:** SubNav `action` gained a `navOnTop` guard; community strip action reduced to creator→Manage; Join moved from strip to header block.

## 4. Triple-Enroll (top-bar, at rest) = dial back via reveal-on-stuck
**Type:** Implementation
**Topics:** astro

**Trigger:** A not-enrolled course showed three Enroll affordances at rest (hero + journey stepper + strip).

**Options Considered:**
1. Dial it back (reveal the strip CTA only once the bar sticks) ← Chosen
2. Leave it (only the strip one pins on scroll anyway)

**Decision:** Hide the strip CTA at rest; reveal it once the bar is stuck (hero scrolled off), via a small inline script + scoped `[data-reveal]/[data-stuck]` style, degrading gracefully to always-visible without JS.

**Rationale:** Removes the at-rest duplication for every course state (and any future community strip CTA) while keeping reachability-when-scrolled.

**Consequences:** SubNav gained the reveal-on-stuck script + scoped style.
