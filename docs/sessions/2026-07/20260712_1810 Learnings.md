# Conv Learnings - 2026-07-12

## 1. db/types.ts is raw-rows-only — joined/response types belong elsewhere
**Topics:** typescript, api, code-organization

**Context:** [CERT-ROWSHAPE] needed a home for a shared "certificate row + joined names" type.

**Learning:** `src/lib/db/types.ts` (876 lines) contains ONLY raw table-row interfaces + status/enum unions — zero joined/derived/`Pick`/intersection types. Adding a joined shape there breaks its cohesion. Per-domain `types.ts` modules exist (promotion/video/discovery-rails/ssr) but there's no shared "API response types" module. `Pick<>` is the established derivation convention across `src/pages/api/**`. Chosen resolution: co-locate a shared joined type in one route file, import into its sibling.

## 2. PUT endpoints returning a thinner shape than GET is a latent client-staleness bug class
**Topics:** api, react, ssr

**Context:** [PUB-CHECKLIST-STALE] — `PUT /api/me/courses/[id]` returned `SELECT * FROM courses` (no joined arrays) while GET returned the full enriched `CourseDetail`; the client cast the PUT response `as CourseDetail` (a lie) and merged it, silently keeping stale joined data (`course.tags`).

**Learning:** When a client merges a PUT response into state that a GET populated with joined data, the PUT MUST return the same joined fields the client relies on — or the merge silently preserves stale values. Watch for `as FullType` casts on responses that don't actually carry the full type.

## 3. driftCheck doc snapshots that duplicate a maintained manifest re-drift every conv
**Topics:** docs-infra, plato

**Context:** [PLATO-DOCTREE] — the plato.md "Current File Structure" ASCII tree duplicated both `tests/plato/PLATO-REGISTRY.md` and TEST-COVERAGE.md, and had drifted every conv a scenario was added (379/380/382).

**Learning:** A hand-maintained file-tree/inventory snapshot inside a design doc that duplicates an actively-maintained manifest is a permanent drift liability. Trimming it to a pointer stub is strictly more durable than reconciling (which just resets the drift clock). Reserve the doc for what only it holds (design rationale).

---

## 4. Universal auto-derived features make per-item creator toggles vestigial
**Topics:** product, schema, ui

**Context:** [DIPLOMA-UI-GAPS] — the creator's per-course "Award certificate on completion" toggle (`has_certificate`) couldn't gate anything, because Conv-389 made the Diploma universal + automatic (`onEnrollmentCompleted` awards regardless).

**Learning:** When a feature becomes universal/automatic, any pre-existing per-item opt-in control for it is now a lie (toggling it off doesn't prevent the thing). The clean fix is to remove the control and surface the feature as a static, always-true perk — not to relabel the dead toggle. DB columns can stay unsurfaced (harmless).
