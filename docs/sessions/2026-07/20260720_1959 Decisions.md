## 1. Adopt react-hooks `recommended-latest` at warn (RHOOKS)
**Type:** Implementation
**Topics:** testing, react

**Trigger:** Conv-397 RDOC audit found 15 of 17 recommended-latest rules shipped-but-unrun.

**Options Considered:**
1. Enable at `error` — turns the lint baseline gate RED immediately (105 findings, 105 error-severity)
2. Enable at `warn`, triage incrementally ← Chosen
3. Leave unrun

**Decision:** Spread the full recommended-latest set at warn (`asWarn`), keep `rules-of-hooks` at error. No dependency change / no overrides pin.

**Rationale:** Same warn-first pattern as [LE-TRIAGE]/[A11Y]; keeps the gate green; no new dep since react-hooks@7.1.1 already ships the eslint ^10 peer.

**Consequences:** 105 warnings surfaced; gate stays exit-0. Confirmed react-hooks does not fire on `.astro` (scoped to `**/*.{ts,tsx}`). Recorded in `docs/decisions/06-testing-ci.md` § RHOOKS.

## 2. Accept `set-state-in-effect` as a baseline, don't chase it
**Type:** Strategic
**Topics:** testing, react, astro

**Trigger:** 95 set-state-in-effect findings dominated the react-hooks backlog.

**Options Considered:**
1. Accept as baseline at warn; triage only new/egregious ← Chosen
2. Turn the rule off (poor SSR fit) — loses signal on genuinely-new cascading-render bugs
3. Grind all 95 — introduces SSR/hydration bugs

**Decision:** Keep at warn; the existing ~93 are an accepted baseline (~90 idiomatic fetch-on-mount or SSR-hydration-safety). Do the one genuine contained fix (current-user → useSyncExternalStore).

**Rationale:** The rule can't see SSR constraints; "fixing" the bulk is actively harmful. Mirrors exhaustive-deps treatment.

**Consequences:** Recorded in `docs/decisions/06-testing-ci.md` § "set-state-in-effect Is an Accepted Baseline".

## 3. Migrate useCurrentUser + useAuthStatus to useSyncExternalStore
**Type:** Implementation
**Topics:** react, auth, astro

**Trigger:** The one genuine set-state-in-effect fix worth doing (core auth hooks).

**Decision:** Replace useState+hydrate-useEffect with `useSyncExternalStore` (subscribe / getSnapshot / getServerSnapshot). The reusable pattern for future client-store hydration hooks.

**Rationale:** React-blessed primitive; store already had the pieces; getSnapshot stability verified; getServerSnapshot preserves hydration.

**Consequences:** react-hooks −2; 151 targeted tests pass. Recorded in the same decision-doc entry as #2 (`docs/decisions/06-testing-ci.md`).
