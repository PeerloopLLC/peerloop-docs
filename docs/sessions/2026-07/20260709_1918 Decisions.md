# Conv Decisions - 2026-07-09

## 1. [TZ-BROWSER-AUTO] — automated tz-display regression = a jsdom component-render suite, NOT Playwright
**Type:** Strategic
**Topics:** testing, timezone

**Trigger:** The Conv-375 `[TZ-BROWSER-AUTO]` open decision (whether to add an automated browser-level tz-DISPLAY regression test), picked up as the conv's work item.

**Options Considered:**
1. A — jsdom component-render suite, no Playwright ← Chosen
2. B — one targeted Playwright spec (closes the SSR/hydration residual but reverses the Conv-347 [BROWSER-SMOKE-2B] Playwright freeze)
3. C — decline, keep the manual `TZ-MANUAL-VERIFICATION.md` checklist only

**Decision:** Option A. Generalized the Conv-376 SessionRoom render-test pattern to 6 more display islands (12 flip-verified tests) inside the already-green, hostile-TZ-gated `npm test`. Playwright declined; the SSR `.astro` + hydration-flicker residual stays owned by the manual checklist, and automated browser tz coverage is deferred post-launch with [BROWSER-SMOKE-2B].

**Rationale:** The viewer-tz model formats to the user's stored zone, so a hostile *host* TZ (already in CI) — not a browser `timezoneId` — is what exposes the "dropped explicit timeZone" bug class. A jsdom render test covers that class deterministically without reversing the Playwright freeze or taking on E2E flakiness for one spec.

**Consequences:** 6 test files (2 new) + 12 tests; full suite 6824✓. **Already routed this conv** to `docs/decisions/06-testing-ci.md` (new entry) and `TZ-MANUAL-VERIFICATION.md` (corrected the stale "no automated test" claim). No live bug found — all islands already used the viewer-tz helpers.

## 2. CI e2e job made non-gating (`continue-on-error: true`)
**Type:** Implementation
**Topics:** testing, deployment

**Trigger:** A surfaced FYI that the CI `e2e` job runs the 28 frozen/expected-red Playwright specs with no `continue-on-error`, contradicting the Conv-347 "e2e is not a gate" decision.

**Options Considered:**
1. A — add `continue-on-error: true` (runs, uploads report, never blocks) ← Chosen
2. B — disable the job until specs are un-frozen
3. C — document only, defer to [CBG]/CI-hygiene
4. Verify the last main-run's color first (blocked — `gh` unauthenticated)

**Decision:** Option A — minimal, faithful to "still runs but is not a gate."

**Rationale:** Aligns the CI config with the documented decision regardless of the last run's color; keeps the Playwright HTML report artifact; one-line change. The job only runs on `main` push/PR, not dev branches.

**Consequences:** `.github/workflows/ci.yml` e2e job gains `continue-on-error: true`; **already noted this conv** in the Conv-347 entry of `docs/decisions/06-testing-ci.md`.
