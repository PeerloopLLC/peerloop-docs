# Conv Decisions - 2026-07-05

## 1. Course stepper gets the course-green tint (bg-course-background)
**Type:** Implementation
**Topics:** course-ux, matt-design

**Trigger:** User: stepper dissolves into surrounding white; wants subtle uniqueness.

**Options Considered:**
1. Course green #E8F4DF (`bg-course-background`, Matt's course Entity-Background) ‚Üê Chosen
2. Neutral grey #F8F8F8 (`bg-neutral-50`)
3. Primary blue #F1F9FF (`bg-primary-light`)

**Decision:** `bg-white`‚Üí`bg-course-background` on all 4 stepper variant roots; inner step-circles kept white.

**Rationale:** On-brand (ties to the green step-circles/progress already in the stepper), fully within the design system, white circles pop against it.

**Consequences:** Stepper reads as a distinct unit; part of addressing the "too much whiteness" concern.

## 2. Sessions card ‚Äî keep the SESSIONS caption, drop the current-page pill

**Trigger:** User called the SESSIONS label "out of place"; CC initially proposed removing it.

**Options Considered:**

**Decision:** Keep the SESSIONS caption; drop the pill for the route you're already on (`actions.filter(!isActive)`).

## 4. State-aware course primary CTA (retire "Continue ‚Üí /modules")
**Type:** Strategic
**Topics:** course-ux, cta, product-model

**Trigger:** User: "Is this a good CTA?" ‚Äî the enrolled CTA was binary and pointed at the static Modules tab.

**Options Considered:**
1. State-aware redesign (session-funnel-centric) ‚Üê Chosen
2. Minimal: fix completers only
3. Leave as-is (evaluating only)

**Decision:** New `buildCoursePrimaryCta(slug, journey)` shared by hero + sticky: nextSession‚ÜíGo to Session N, bookableCount‚ÜíBook first/next session, isComplete‚ÜíReview your sessions, else‚ÜíView my sessions, not-enrolled‚ÜíEnroll. Never points at /modules.

**Rationale:** Peerloop learning = sessions; the CTA should drive the funnel. All needed state already on `CourseJourneyState`. Shared helper keeps hero + sticky in agreement.

**Consequences:** 5 files. Certificate dead-end for completers remains (no student cert route ‚Äî `CERT-APPROVAL`); CTA now stops sending them backward but the real fix is that separate PLAN item.
