# Conv Decisions - 2026-07-02

## 1. Sticky convention: pin re-used controls, release orientation
**Type:** Strategic
**Topics:** layout, ux

**Trigger:** User asked whether breadcrumbs/title should stay sticky on detail pages.

**Options Considered:**
1. Pin the header/title too
2. Pin only the re-used controls (search/sort/tabs); let breadcrumb + title + carousels scroll off ← Chosen
3. Collapsing-header (compress title into a persistent top bar)

**Decision:** Pin only the interactive/re-used controls; breadcrumb, page title, and recommendation carousels scroll off.

**Rationale:** Standard listing/feed convention; the sidebar already answers "where am I"; there's no desktop top-bar to compress a title into; pinned vertical space is costly (worst on mobile).

**Consequences:** Applied uniformly across listing + detail pages. Course journey stepper follows the same rule (see Decision 5).

## 2. Extract a shared StickyListingToolbar primitive rather than copy-paste
**Type:** Implementation
**Topics:** astro, layout

**Trigger:** Rolling the sticky toolbar to /courses + /members after building it inline on /communities.

**Options Considered:**
1. Copy the sticky wrapper className into each page
2. Extract one `StickyListingToolbar.astro` primitive and use it everywhere ← Chosen

**Decision:** New `src/components/layout/StickyListingToolbar.astro` (sibling to ListingShell); all three listing pages use it.

**Rationale:** The className encodes two hard-won gotchas (the Tailwind-v4 offset trap + the riser); copy-paste invites drift. One documented home means a future tweak is a single edit.

**Consequences:** Three pages stay identical; the traps are documented once in the primitive's docstring.

## 3. Detail-page sticky is an OPT-IN SubNav prop, not a global change
**Type:** Strategic
**Topics:** astro, layout

**Trigger:** Making the detail-page tab bars sticky, where the tab bar is the shared `SubNav.astro`.

**Options Considered:**
1. Make SubNav's top strip sticky globally
2. Add an opt-in `sticky` prop, applied only to chosen pages ← Chosen

**Decision:** `SubNav` gets a `sticky` prop (default false); `CourseRail` forwards it; only `/course`, `/community`, `/profile` opt in. CTA-bar pinning deferred to Phase 2; `/@handle` left unpinned (no tab bar).

**Rationale:** SubNav is used by ~20 pages; a global change would sticky-pin all of them. Opt-in contains the blast radius (verified `/learning` stayed static).

**Consequences:** Phase 1 (tab bars on 3 detail pages) shipped; Phase 2 (condensed Enroll/Join action bars) tracked as deferred.

---

## 4. Fix the sticky/z bleed-throughs at their root (mutually-exclusive variants; riser; isolate)
**Type:** Implementation
**Topics:** css, tailwind, layout

**Trigger:** Three distinct visual bugs surfaced while building the toolbars.

**Options Considered (per bug):** patch-at-the-bar vs root-cause fix.

**Decision:** (a) offsets via mutually-exclusive `max-lg`/`lg` variants; (b) `before:` riser sized to the at-rest gap; (c) `isolate` on the offending card to scope its z-10 — rather than bumping the bar's z-index.

**Rationale:** Each root-cause fix is order-/context-independent and keeps the z-scale (10=sticky) clean.

**Consequences:** Encoded in StickyListingToolbar + SubNav docstrings and a CourseCatalogCard comment.

---

## 5. Course journey stepper stays non-sticky (scrolls off)
**Type:** Implementation
**Topics:** layout, ux

**Trigger:** For enrolled users the course page stacks the journey stepper (Enroll/Payment/Sessions/Certificate) above the now-sticky Explore tabs.

**Options Considered:**
1. Keep the stepper scrolling off; only Explore tabs pin ← Chosen
2. Pin stepper + tabs together (~275px of chrome)
3. Drop sticky on /course

**Decision:** Keep the stepper non-sticky; verified it occludes cleanly under the pinned tabs.

**Rationale:** The stepper is a glance-once status band (Decision 1 convention); pinning both eats too much space. Persistent enrollment context belongs in a future condensed bar (Phase 2), not a pinned 154px stepper.

**Consequences:** Added a "do not make this sticky" comment on the stepper placement so it isn't mistaken for an oversight.
