# Conv Learnings - 2026-07-06

## 1. `scrollHeight` clamps to `clientHeight` — collision "release" can't use scrollHeight slack
**Topics:** astro, ui-ux, testing

**Context:** Building the SIDEBAR-COLLIDE ResizeObserver, the first-cut hysteresis released the merge when `slack = clientHeight - scrollHeight` exceeded a threshold.

**Learning:** `Element.scrollHeight` is defined as `≥ clientHeight` (it clamps to the client box when content fits), so "slack" maxes at 0 and can never report *room to spare*. Enter-on-overflow (`scrollHeight > clientHeight`) is reliable; a hysteresis **release** must be driven off the **un-clamped** `clientHeight` compared to a separately-recorded content height, not off a scrollHeight-based slack. The bug was invisible to tsc/lint/build and to jsdom tests (0 heights) — only a real-browser height sweep exposed it.

**Pattern:** For overflow-gated toggles with hysteresis: record `unmergedContentH = scrollHeight` while un-collided (read the live DOM attribute for current state to avoid React-state lag), enter when `scrollHeight > clientHeight`, release only when `clientHeight > unmergedContentH + HYSTERESIS`.

## 2. HMR / React Fast Refresh contaminates iframe-harness DOM tests
**Topics:** testing, cc-workflow

**Context:** A mid-sweep `vite hot updated: Sidebar.tsx` (from my own edits settling) left the observer in a state where it never reacted — looked like a code bug but wasn't.

**Learning:** When DOM-verifying a `client:load` island via the iframe harness against a live dev server, Fast Refresh re-running mid-measurement disrupts the effect/observer lifecycle. Let edits settle, then **recreate the iframe fresh** (`f.src='/'`) and measure — don't trust a sweep that overlaps an HMR event. Check the iframe console for `hot updated` timestamps when results look impossible.

## 3. The exact-width iframe harness generalizes to min-HEIGHT (landscape) testing
**Topics:** testing, ui-ux

**Context:** `reference_responsive_iframe_harness` was written for min-*width*; SIDEBAR-COLLIDE needed min-*height*.

**Learning:** An exact-*height* same-origin iframe drives both `@media (max-height:…)` blocks and `calc(100vh-…)` box heights inside it, so a ResizeObserver reading `clientHeight` sees the simulated viewport correctly. Resizing `iframe.style.height` across a sequence (down then up) is the clean way to exercise a collision + hysteresis observer and confirm no oscillation.

## 4. `transition:persist` HELPS a JS-observer pattern (vs. hurting CSS classes)
**Topics:** astro, ui-ux

**Context:** Conv 367 avoided JS partly out of VT-persist caution; the concern didn't apply here.

**Learning:** The Conv-250 lesson was that island-unique **classes** drop on a `transition:persist` swap. A JS observer that owns a **data-attribute** on the persisted node is the opposite: the node + attribute + the still-attached observer all survive the swap. An `astro:after-swap` re-measure covers a re-parent. So "it's a persisted island" is not a reason to avoid JS measurement.
