# Conv Decisions - 2026-07-12

## 1. Co-locate the shared admin Certificate type (cross-route import)
**Type:** Implementation
**Topics:** typescript, api

**Trigger:** [CERT-ROWSHAPE] needed a home for `CertificateAdminRow` (Certificate + joins); db/types.ts is raw-rows-only.

**Options Considered:**
1. New `src/lib/certificates/types.ts` domain module
2. Co-locate in admin `index.ts`, import into sibling `[id].ts` ← Chosen
3. Derive inline in each endpoint (no share)

**Decision:** Export `CertificateAdminRow` from `admin/certificates/index.ts`, import into `[id].ts`.

**Rationale:** Lowest surface for a low-value refactor; fixes the bug + de-dups the byte-identical pair without spawning a new module. User-selected.

**Consequences:** Cross-route type import (mildly unconventional but valid). Sets the local precedent; a broader "API response types" module remains a future option.

## 4. Remove the vestigial has_certificate toggle
**Type:** Implementation
**Topics:** product, ui, schema

**Trigger:** [DIPLOMA-UI-GAPS] — the creator's "Award certificate on completion" toggle can't gate the now-universal Diploma.

**Options Considered:**
1. Remove the toggle; static "Diploma on completion" perk ← Chosen
2. Relabel to "Award a Diploma on completion", keep toggle
3. Wording-only swap

**Decision:** Strip the toggle + custom-title field from CourseEditor; CourseHero shows "Diploma on completion" unconditionally.

**Rationale:** Most accurate to the universal-Diploma model; a toggle for a universal thing is misleading. DB columns left in place (unsurfaced).

**Consequences:** 6-spot removal in CourseEditor; CourseHero perk now unconditional; a CourseDetail test updated to assert the new perk.

## 5. Reuse the enrollment_completed type for the teacher notification
**Type:** Implementation
**Topics:** notifications

**Trigger:** [TEACHER-NOTIF] — needed a teacher-facing "your student completed" notification.

**Options Considered:**
1. Reuse `enrollment_completed` type with a new function ← Chosen
2. Add a new `student_completed` NotificationType (+ icon-map changes)
3. Remove the teacher notification entirely

**Decision:** New `notifyTeacherStudentCompleted` function reusing `type: 'enrollment_completed'`.

**Rationale:** Both are completion-category events; reusing the type avoids touching NotificationCenter/NotificationsList icon maps. Keeps the valuable teacher signal (cue to recommend) with correct wording.

**Consequences:** 3 full-replacement notification mocks (booking, cleanup, plato-scenarios) needed the new function registered.
