# Conv Learnings - 2026-07-07

## 1. Rename-collision from a local wrapper named after an alias target
**Topics:** refactor, icons, rename-lessons

**Context:** During the Phase-3 `icons.tsx` rename, `tsc` flagged `BecomeATeacherPage.tsx` + `ForStudentsSection.tsx` with "Import declaration conflicts with local declaration."

**Learning:** These files use an `import { XIcon }` + `const Wrapper = () => <XIcon .../>` pattern where the *wrapper was named after the alias's target* (e.g. imported `XIcon`, wrapper named `CloseIcon`). A word-boundary rename that maps the import onto the wrapper's name collides the two bindings. Fix = alias the import (`import { CloseIcon as CloseIconShared }`) and point the wrapper at the alias — matching each file's existing `…Shared` convention. `tsc` catches this deterministically, so a rename pass is safe as long as you re-run typecheck before the suite.

**Pattern:**
After a bulk identifier rename, run `tsc` FIRST (cheapest, catches import/redeclaration collisions), then lint/astro, then the full suite.

## 2. Test mocks reference module exports by name — renames must update mock keys
**Topics:** testing, refactor, icons

**Context:** `tests/` was out of the `src/`-scoped perl pass; `ContextActionsPanel.test.tsx` mocks `@components/ui/icons` with an object keyed by export name.

**Learning:** `vi.mock('@components/ui/icons', () => ({ EditIcon: ..., MoneyIcon: ... }))` provides mock impls keyed by the real export names. Renaming the real export (EditIcon→WriteIcon) silently breaks the mock (the component imports WriteIcon, the mock only supplies EditIcon) unless the mock key is renamed too. Grep `tests/`+`e2e/` for old identifiers before running the ~3-min suite.

## 3. Audit-doc counts can be wrong — verify against source before acting
**Topics:** icons, verification

**Context:** icon-system.md (a Conv-369 audit) stated `icon-paths.ts` had "6 `<Icon>` call sites."

**Learning:** The real count was **4**. The audit conflated 2 doc-comment `<Icon>` examples inside `Icon.astro` and possibly the `<Icon className=…>` React-local-variable matches. Always re-derive the number from a fresh grep of the source before scoping work off an audit figure — the audit is a hypothesis.

## 4. Glyph comparison resolves an apparent "concept-merge" into a clean rename
**Topics:** icons, design

**Context:** §7.3 flagged `TeamIcon`+`UsersIcon`→`group` as a fraught merge of two distinct glyphs.

**Learning:** Reading the actual SVG paths settled it deterministically: `TeamIcon` = Heroicons 3-person user-group, `UsersIcon` = 2-person, MattIcon `group` = 2-person Material. So `UsersIcon` is the clean 1:1 match (→`GroupIcon`) and `TeamIcon` is a distinct no-twin glyph to keep. The "merge" framing dissolved once the glyphs were compared — cheaper than a design debate.

## 5. A "let it ride the migration" recommendation must verify the migration exists
**Topics:** matt-migration, icons, process

**Context:** I recommended deferring §3.3 unification to "the per-page Matt retrofit," then the user asked which migration.

**Learning:** Verifying the claim falsified it: RTMIG-4 (the route sweep) **closed Conv 340** and only ever conformed colour/type/spacing tokens — icon-glyph unification was never in its scope. So the stroke/filled split is a **steady state**, not a transient the migration will resolve. A recommendation that leans on an ongoing process must confirm that process is (a) still open and (b) actually scoped to do the thing.

## 6. Icon licensing splits three ways by source
**Topics:** licensing, icons, commercial, mvp-golive

**Context:** "Are the icons usable in a commercial app?"

**Learning:** `icons.tsx` = Heroicons (**MIT**, Tailwind Labs; no package dep, inlined paths). `MattIcon` = Matt's commissioned catalogue (owned, per designer agreement) + Material Symbols (**Apache 2.0**, Google). `brand-icons.tsx` = third-party **trademarks** — governed by brand guidelines, NOT a license. Permissive licenses ≠ zero obligation: MIT + Apache 2.0 both require the notice retained, and there was **no** `LICENSE`/`NOTICE`/attribution file. The brand logos (esp. Google Sign-In, Stripe, and the `currentColor` recoloring) are the real pre-launch caution. Tracked as `[ICON-LIC]` under MVP-GOLIVE. (Not legal advice.)
