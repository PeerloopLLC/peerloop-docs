### Root-causing the Task-subsystem outage

**User:** "Do some research and see if the CC harness itself has stopped using or altered the usage of the Task subsystem"

**Claude:** Ran a `claude-code-guide` research agent (docs + GitHub issues) in parallel with a local binary decompile of the installed CC 2.1.214/215/216. Found the decisive evidence locally: an **undocumented second gate** in the bundle. `TaskCreate`/`TaskList`/`TaskUpdate`/`TaskGet` are `isEnabled(){ return uH() && !uZ() }` and `TodoWrite` is `!uH() && !uZ()`. `uH()` is the documented `CLAUDE_CODE_ENABLE_TASKS` env gate. `uZ()` reads remote dynamic config `tengu_vellum_ash` and returns true if the current model ID (`Di()`) contains any listed string. Because `uZ()` is on both branches, a true `uZ()` suppresses `Task*` AND `TodoWrite` simultaneously — a combination no local setting can produce. `uH()` is provably true (var verified absent), so `uZ()` is deduced true. Binary identical across 214/215/216; `~/.claude/tasks/` last written 07-21 07:21 (after the 2.1.216 install) then stopped → server-side, not a version bump, not our config. Grep/Glob confirmed a separate, older, still-open upstream registry bug (#52121, #63525).

**User:** "try going back to v 2.1.200 and just examine the CC changeogs"

**Claude:** Read the local cached changelog `~/.claude/cache/changelog.md` (2.1.0→2.1.216) across the 2.1.200→2.1.216 range. **No deprecation/removal/gating of the task-list tools anywhere** — the only Task-ish hit (2.1.212 "Deprecated the Task tool's `mode` parameter") is the *subagent* Task tool, which the earlier research agent had conflated. Task lists + Grep/Glob got active bugfixes through 2.1.208/2.1.210 (positive evidence they're alive). `vellum` = 0 hits in the whole changelog → the `uZ()` gate is wholly unannounced. Did not downgrade (the gate is runtime remote config; a rollback drops ~16 versions of permission/security fixes). Found `CLAUDE_CODE_DISABLE_1M_CONTEXT`, proposed it as a cheap test, the user picked it — then traced `mi()`/`Tb()`/`kD()` and **self-falsified the proposal**: that var short-circuits `Tb()` so the `[1m]` suffix is never stripped, so `Di()` returns `claude-opus-4-8[1m]` either way. Void. Proposed the valid substitute: select the non-1M Opus 4.8 via `/model` so the model string carries no `[1m]` suffix at all.
