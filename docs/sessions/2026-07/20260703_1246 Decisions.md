# Conv Decisions - 2026-07-03

## 1. Disable the AFK auto-proceed nudge in BOTH settings scopes
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** User called the 60s auto-proceed "potentially disastrous" and asked to remove it.

**Options Considered:**
1. Global `~/.claude/settings.json` only (all projects, this machine; no cross-machine sync)
2. Project `.claude/settings.json` only (git-synced to M4; Peerloop-only)
3. Both ← Chosen
4. Shell-alias export only

**Decision:** Set `CLAUDE_AFK_TIMEOUT_MS=2147483647` in both global and project settings.

**Rationale:** The risk is universal (all projects on this machine → global) AND the user works cross-machine (git-tracked project settings auto-sync to M4). `0` rejected as a trap (fires immediately).

**Consequences:** Unanswered `AskUserQuestion` dialogs now stay open indefinitely; effective next launch.

## 2. Mobile/tablet nav = Arrangement A (door up top)
**Type:** Strategic
**Topics:** navigation, matt-design, ux

**Trigger:** "wide-open discussion" on how mobile/tablet reaches the full sidebar.

**Options Considered (from an interactive 4-arrangement mockup):**
1. A · door up top (hamburger top-left; 4 bare bottom shortcuts, no More) ← Chosen
2. B · door at thumb (avatar top-left; bottom "More")
3. C · quiet top / rich bottom (5-item bottom bar incl. More)
4. D · hamburger only, no bottom bar

**Decision:** Arrangement A — top pill `≡`/logo/`🔔`; bottom pill 4 bare shortcuts (Home·Courses·Communities·Messages); hamburger opens a left slide-out of the full sidebar.

**Rationale:** A few well-understood icons read fine bare; the discoverable top-left hamburger owns "everything else"; keeps content space and one-tap access.

**Consequences:** Built as [MOBNAV]; retired the emoji/dead-route stub.
