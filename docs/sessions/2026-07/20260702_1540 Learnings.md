# Conv Learnings - 2026-07-02

## 1. Astro exports a TS type from .astro frontmatter, importable cross-.astro
**Topics:** astro, cc-workflow

**Context:** Needed a shared `SubNavAction` shape for both `SubNav.astro` (owner) and `CourseRail.astro` (forwarder).

**Learning:** `export interface SubNavAction {…}` in an `.astro` component's frontmatter is importable elsewhere via `import type { SubNavAction } from '../SubNav.astro'`, and `astro check` type-validates the flow across `.astro` boundaries (0 errors/0 warnings/0 hints). Plain `tsc --noEmit` does NOT check `.astro` files — the cross-`.astro` type wiring is only validated by the `npm run check` (astro) gate, reinforcing why that gate is mandatory (not just `tsc`).

**Pattern:** Define a shared prop-shape type once in the primitive that owns it; `export interface` from its frontmatter; `import type` into forwarders/callers.

## 2. Detect "position:sticky is stuck" with a rAF-throttled scroll check
**Topics:** astro, cc-workflow

**Context:** Reveal-on-scroll needed to show the merged CTA only once the tab bar pins (entity header scrolled off).

**Learning:** Compare `nav.getBoundingClientRect().top` against the element's own resolved sticky offset (`parseFloat(getComputedStyle(nav).top)`, recomputed each tick to survive breakpoint changes 16↔88); `stuck = rect.top <= offset + 1`. Toggle a `data-stuck` attribute the CSS keys off. rAF-throttled, `passive:true`. No sentinel element / IntersectionObserver restructuring of the DOM needed.

**Pattern:**
```js
const update = () => { const off = parseFloat(getComputedStyle(nav).top)||0;
  nav.toggleAttribute('data-stuck', nav.getBoundingClientRect().top <= off+1); };
window.addEventListener('scroll', () => { if(!raf) raf = requestAnimationFrame(update); }, {passive:true});
```

## 3. Graceful-degradation reveal: default visible, JS opts into hidden-until-stuck
**Topics:** astro, cc-workflow

**Context:** Reveal-on-scroll adds a JS enhancement to an otherwise pure-CSS merge; must not break no-JS.

**Learning:** Author the hidden state behind a JS-added attribute (`nav[data-reveal] [data-sticky-action]{display:none}`), NOT as the default. Without JS, `data-reveal` is never added → the CTA stays visible (the plain always-on merge behavior). The script adds `data-reveal` on init, then toggles `data-stuck`. Progressive enhancement with a sane fallback.

## 4. Non-destructive click-handler verification via a never-resolving fetch stub
**Topics:** testing, cc-workflow

**Context:** Wanted to prove the community Join button fires the right POST without actually joining (a real state mutation on the user's behalf) and without triggering the handler's `alert()` (which blocks the claude-in-chrome harness).

**Learning:** Stub `window.fetch` to capture `{url, method, credentials}` and return `new Promise(()=>{})` (never resolves) → the handler runs, sets `disabled=true`, and awaits forever, so neither the success `reload()` nor the error `alert()` fires. Read the captured request, restore `fetch`. Verifies wiring end-to-end with zero side effects.
