**Topics:** dependencies, security

**Trigger:** 21 audit advisories; user asked whether to fix as planned given a pending Astro 7 assessment.

**Options Considered:**
1. Runtime chains + document rest ← **Chosen**
2. Runtime + override-sweep all dev-tooling

**Decision:** Fix the two production-runtime chains in-range now (resend/uuid + devalue), defer the ~17 dev-only/tooling advisories (vite/esbuild/etc. to be subsumed by the v7 migration; undici/ws/brace-expansion/fast-uri/js-yaml as low-priority dev-only).

## 3. compressHTML — pin `true` at promotion time (recommended, not yet applied)
**Type:** Implementation
**Topics:** astro, ui

**Trigger:** Astro 7 flips `compressHTML` default `true`→`'jsx'` (whitespace between inline elements).

**Decision:** At promotion, pin `compressHTML: true` in `astro.config.mjs` to preserve exact current rendering.

**Rationale:** Not build-detectable; the project's "never lose appearance" rule makes the zero-visual-risk conservative default correct. Accepting `'jsx'` would require a full visual regression pass.

**Consequences:** Deferred to the promotion conv (no config change this conv).
