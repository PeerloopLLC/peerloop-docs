## 2. /profile layout control stays a segmented toggle (not a checkbox)
**Type:** Implementation
**Topics:** ui

**Trigger:** [LAYOUT-TOGGLE-AFF] — Conv-357's "check a box in /profile" phrasing raised whether `nav_layout` should be a checkbox rather than `LayoutToggle.tsx`'s segmented "Top bar / Side rail" radiogroup.

**Options Considered:**
1. Keep the segmented toggle — both options named + symmetric, no implied default ← Chosen
2. Swap to a checkbox "Use side rail on desktop" (unchecked=top default / checked=rail) — matches the "check a box" phrasing

**Decision:** Keep the segmented two-option toggle.

**Rationale:** The explicit named-options control is clearer for a symmetric either/or; the checkbox would frame `top` as an implied default. No functional gap in the current control.

**Consequences:** No code change; `LayoutToggle.tsx` unchanged. [LAYOUT-TOGGLE-AFF] closed.
