# Session Learnings - 2025-12-29

## Technical

### Schema Audit Reveals Data Sync Gaps
- Comparing DB-SCHEMA.md vs D1 migrations vs mock-data.ts exposed 15+ missing tables
- Many page specs reference entities that don't exist in D1 yet
- Categories: D1 has 15, mock-data has only 3 - reconciliation needed
- Created Block 1.8 to address marketing content gaps before Block 2

### Testimonials vs Success Stories - Different Data Models
- `course_testimonials`: Short quotes for homepage (already in D1)
- `success_stories`: Full case studies with stats, video, narrative (needed new table)
- Homepage testimonials ≠ Stories page - they serve different purposes
- API needed two modes: `?featured=true` (homepage) vs `?featured=false` (browse)

### Tailwind v4 Gradient Classes
- `bg-gradient-to-*` renamed to `bg-linear-to-*` in Tailwind v4
- The `check:tailwind` script catches these during code quality checks
- Always run `/q-codecheck --smart` before committing to catch class renames

## Architecture

### API Design Pattern for Dual-Mode Endpoints
```typescript
// Single endpoint serving two response formats
if (featured && !filters) {
  // Homepage mode: simple array
  return { testimonials: items };
}
// Browse mode: paginated response
return createPaginatedResult(items, total, page, limit);
```
