# Session Learnings - 2025-12-16

## Technical

### Astro + Cloudflare Stack
- Astro with React and Tailwind is the confirmed stack for Peer Loop
- Cloudflare Pages + D1 + R2 + Queues provides a complete serverless platform
- Wrangler CLI is essential for local development with Cloudflare bindings

### Testing Strategy
- Vitest (CLI mode only) chosen over Jest for speed and native TypeScript support
- Playwright for E2E tests - good Astro support, simpler than Cypress
- User had bad experience with Vitest GUI - confirmed CLI-only approach
- tsx + node:test identified as fallback if Vitest fails

### CI/CD
- GitHub Actions for automated testing on PR/push
- Cloudflare Pages handles deployment automatically on merge

### SEO and Security
- SEO foundations (meta tags, Open Graph, structured data) should be built into base layouts from start
- Security headers and rate limiting utilities are easier to build in than bolt on later
