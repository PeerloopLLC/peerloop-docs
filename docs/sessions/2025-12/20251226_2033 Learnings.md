# Session Learnings - 2025-12-26

## Technical

### Marketing Page Architecture Pattern
When creating new public-facing pages for a marketing site, the established pattern is:
1. **JSON data file** (`src/data/pages/*.json`) - Contains all page content, connections, and sample data
2. **Astro page file** (`src/pages/*.astro`) - Minimal file that imports JSON and renders through PageSpecView
3. **Update ROUTES.md** - Add new page codes to the routing documentation
4. **Update LandingLayout** - Add navigation links to header and footer as appropriate

This separation keeps content editable without touching page structure.

### Page Code Conventions
- 4-letter uppercase codes (e.g., ABOU, HOWI, PRIC, FCRE)
- Codes should be mnemonic for the page purpose
- Codes are used in connections to reference navigation targets

### JSON Page Specification Structure
The sample content section in JSON files should include realistic placeholder data that:
- Demonstrates the data structure for implementers
- Can be used for development/staging environments
- Shows the full range of fields each section needs

## Documentation

### DB Schema Additions for Marketing
Marketing pages require their own set of database tables:
- `team_members` - About page team section
- `platform_stats` - Aggregated stats for display across pages
- `success_stories` - Case studies with typed categories (student/teacher/creator)
- `faq_entries` - Categorized Q&A content
- `contact_submissions` - Form submissions with status tracking
- `blog_categories` + `blog_posts` - Content marketing infrastructure
- `job_listings` - Careers page openings

These are content tables (CMS-like) vs transactional tables (user activity).

### Footer Column Organization
A 5-column footer works well for marketing sites:
1. Brand + tagline
2. Platform (product features)
3. Get Involved (conversion paths)
4. Company (about, blog, careers)
5. Support (FAQ, legal)

## Workflow

### Marketing Site Completeness Checklist
When auditing a marketing/landing page site, check for:
- **Trust builders**: About, Team, Success Stories
- **Conversion pages**: How It Works, Pricing, For [Role]
- **Support**: FAQ, Contact, Help Center
- **Legal requirements**: Privacy Policy, Terms of Service
- **Growth**: Blog, Careers

Missing pages are often in the footer but don't exist yet.
