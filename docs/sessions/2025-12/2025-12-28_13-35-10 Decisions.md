# Session Decisions - 2025-12-28

## Decision 1: Move Tech Docs from research/ to docs/

**Type**: Strategic
**Driver**: User-driven
**Working On**: Documentation reorganization
**Trigger**: Tech documents evolved beyond research/reference into active development documentation

**Options Considered**:
1. `docs/tech/` - Parallel to `docs/reference/`, dedicated folder <- **Chosen**
2. `docs/reference/tech/` - Nested under reference as a subcategory
3. `docs/decisions/` - Emphasizes they capture technology decisions

**Decision**: Move all tech documents from `research/tech/` to `docs/tech/`

**Rationale**:
- Keeps it at the same level as `docs/reference/`
- Clear separation: `reference/` = how to use, `tech/` = why we chose
- Simple path for linking
- Tech docs have become active development documentation, not just research notes

**Consequences**:
- Moved 16 files (tech-*.md, comp-*.md) to new location
- Updated 11+ documentation files with new paths:
  - CLAUDE.md, PLAN.md, q-local-docs.md, DEVELOPMENT-GUIDE.md
  - research/REMOTE-API.md, SERVER.md, CLAUDE.md, STRUCTURE.md
  - run-001 assets (AMENDMENT-PLAN.md, stream-usage.md, video-platform-decisions.md)
  - client-docs-index.md
- Removed empty `research/tech/` folder
- Session files (historical) left unchanged - accurately reflect state at time of writing
