# Session Learnings - 2025-12-30

## Technical

### D1 Migration Architecture
- **Lesson**: Avoid seed data conflicts by keeping migrations clean and sequential
- **Before**: 5 migrations with duplicate category INSERTs across 0001 and 0002, causing UNIQUE constraint failures
- **After**: 2 clean migrations - `0001_schema.sql` (tables only) + `0002_seed.sql` (data only)
- **Pattern**: Schema migrations should never contain seed data; seed data should use a single source of truth (mock-data.ts)

### CI E2E Database Setup
- **Issue**: GitHub Actions e2e tests failed with "no such table: users"
- **Root Cause**: CI workflow didn't run D1 migrations before starting the dev server
- **Fix**: Added `npm run db:migrate:local` step before `npm run test:e2e` in `.github/workflows/ci.yml`
- **Key Insight**: Local D1 in CI creates SQLite database that needs schema/seed before tests can run

### Playwright Selector Ambiguity
- **Issue**: Tests failing with "strict mode violation" - multiple elements matching selectors
- **Root Cause**: Generic selectors like `getByText('Peerloop')` matched multiple elements (nav, footer, page content)
- **Fix**: Scope selectors to specific containers: `page.getByRole('navigation').getByRole('link', { name: 'Courses' })`
- **Pattern**: Always use navigation-scoped selectors for header/footer elements to avoid ambiguity

### React Hydration Timing in E2E Tests
- **Issue**: "Sign In" link not found in navigation
- **Root Cause**: Header component fetches auth state async; Sign In only renders after `isLoading` becomes false
- **Fix**: Add explicit timeout for async elements: `toBeVisible({ timeout: 10000 })`
- **Pattern**: For React components with async state, tests need sufficient timeout for hydration + data fetch

## Workflow

### Build-Time Variable Injection
- **Approach**: Use Vite's `define` option in astro.config.mjs to inject constants at build time
- **Implementation**: `__APP_VERSION__` and `__COMMIT_HASH__` available globally
- **TypeScript**: Declare in `src/env.d.ts` for type safety
- **Access**: Check `typeof __VAR__ !== 'undefined'` for fallback in SSR contexts

### Environment Detection Strategy
- **DEV**: localhost or no `CF_PAGES` env var
- **STG**: Cloudflare Pages + branch !== 'main' (preview deploys)
- **PROD**: Cloudflare Pages + branch === 'main' or production domain
- **Implementation**: Server-side uses `process.env.CF_PAGES_BRANCH`, client-side uses `window.location.hostname`

## Documentation

### Single Source of Truth for Seed Data
- **Problem**: Seed data scattered across multiple migrations led to conflicts and data drift
- **Solution**: `src/lib/mock-data.ts` is the canonical source; SQL migrations generated from it
- **Benefit**: TypeScript types, easier editing, one place to update, no migration conflicts

## Architecture

### Footer Version Display Pattern
- **Version format**: `v1.0.0` (from package.json) with commit hash in tooltip
- **Environment badge**: Colored badge (DEV=blue, STG=orange) hidden in PROD
- **UX principle**: Developers always know their environment; production users see clean interface
