# Session Learnings - 2025-12-28

## Technical

### Static Data Pattern for Early Development
- Using static data in React components (matching seed data) allows rapid UI development without API dependencies
- Components like `CourseBrowse`, `CourseDetail`, `CreatorBrowse`, `CreatorProfile` all use this pattern
- Data is structured to match DB schema so API swap later is straightforward
- Trade-off: Data duplication between component and seed file, but enables parallel development

### Component Architecture for Browse/Detail Pages
- Browse pages (CourseBrowse, CreatorBrowse): client-side filtering with `useMemo`, debounced search
- Detail pages (CourseDetail, CreatorProfile): slug-based lookup from static data object
- Both patterns use `client:load` directive for interactivity
- Astro page extracts params and passes to React component

### Curriculum Grouping Pattern
- Course curriculum stored flat in DB with `session_number` field
- Component groups by session using `reduce()` for accordion display
- This matches the 1-on-1 session teaching model

## Architecture

### Page Structure Pattern
Established consistent pattern for Block 1 pages:
```
src/pages/[route].astro          # Astro wrapper with layout
  ├── PageSpecView               # Dev mode spec display
  ├── Sidepanel navigation       # Links to related pages
  └── Fragment slot="content"    # Real component with client:load
src/components/[domain]/[Component].tsx  # React component with static data
```

### Creator/Course Data Relationship
- Courses reference creator by ID
- CourseCard accepts `creator: { name, avatar_url }` inline
- CourseDetail embeds full creator object
- CreatorProfile embeds array of creator's courses
- This denormalization in static data mirrors what API would return with JOINs
