# Session Decisions - 2025-12-27

## Decision 1: CSS-Based Image Fallbacks Instead of Placeholder Service

**Type**: Implementation
**Driver**: User-driven
**Working On**: Block 1.1 Homepage - CourseCard component
**Trigger**: placehold.co auto-sizes text based on length, causing inconsistent font sizes across course cards

**Options Considered**:
1. Use placehold.co with course titles (auto-sized text)
2. Use placehold.co with generic "Course" text
3. **Use CSS overlay with Tailwind classes** <- **Chosen**
4. Use local placeholder images

**Decision**: Use CSS-based fallback rendering with a div and Tailwind typography classes instead of external placeholder service.

**Rationale**:
- Full control over font size, weight, and color
- Consistent appearance regardless of title length
- No external service dependency
- Easier to adjust styling without changing URLs

**Consequences**:
- CourseCard renders a `<div>` with `text-2xl font-semibold` when no thumbnail_url
- placehold.co code removed; gradient/initials fallback code commented out for future use
- All course cards now have identical typography in placeholder state

## Decision 2: Toggleable Spec Panel for Development

**Type**: Strategic
**Driver**: User-driven
**Working On**: Block 1.1 Homepage layout
**Trigger**: User wanted to keep existing PageSpecView documentation accessible while building real content

**Options Considered**:
1. Replace spec view entirely with real content
2. **Keep both in toggleable panel** <- **Chosen**
3. Move spec to separate dev-only route

**Decision**: Implement a toggle button that switches between "Spec View" (PageSpecView documentation) and "Content View" (real page content).

**Rationale**:
- Easy reference to spec requirements during development
- No loss of existing documentation work
- Developers can validate implementation against spec quickly

**Consequences**:
- Created `SpecPanelToggle.tsx` component
- Added CSS classes `.spec-content` and `.real-content` to global.css
- Modified `LandingLayout.astro` to support both content slots
- State persisted in localStorage across navigation

## Decision 4: NULL Image URLs in Seed Data

**Type**: Implementation
**Driver**: Claude-driven (following user's fallback preference)
**Working On**: Seed data migration
**Trigger**: No actual images exist yet; need to test fallback behavior

**Options Considered**:
1. Use placeholder URLs pointing to non-existent files
2. **Use NULL values to trigger component fallbacks** <- **Chosen**
3. Upload placeholder images to public/

**Decision**: Set all `thumbnail_url` and `avatar_url` values to NULL in seed data migration.

**Rationale**:
- Tests the fallback rendering path in components
- No 404 errors from missing image files
- Clear indication that real images are needed later
- Comments document the intended future paths

**Consequences**:
- All courses display CSS fallback (light blue bg with title)
- All creators display CSS fallback (circle with initial)
- Easy to update when real images are added
