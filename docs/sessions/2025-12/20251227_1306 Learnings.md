# Session Learnings - 2025-12-27

## Technical

### Local D1 Database Setup
- Local D1 works without Cloudflare authentication - wrangler bundles its own SQLite
- Migration command: `npm run db:migrate:local`
- Local database stored at `.wrangler/state/v3/d1/`
- Can query directly: `npx wrangler d1 execute peerloop-db --local --command "SELECT..."`

### React Components in Astro
- Use `client:load` directive for interactive React components in Astro pages
- Path aliases (`@components/*`) work across both .astro and .tsx files
- TypeScript strict mode requires explicit typing for `response.json()` results

### Astro + ESLint
- HTML comments (`<!-- -->`) inside JSX expressions cause ESLint parsing errors
- Remove comments from JSX ternary expressions or use JSX comment syntax
- Astro files are parsed by eslint-plugin-astro which has stricter JSX rules

### Auth Session Pattern
- Fetch `/api/auth/session` client-side to get auth state
- Return `{ authenticated: boolean, user: User | null }` for consistent handling
- On AppLayout pages, redirect to login if not authenticated

## Architecture

### Navigation Component Structure
```
src/components/navigation/
├── Header.tsx       # Public pages - fetches session, shows login OR user menu
├── AppHeader.tsx    # Auth pages - assumes auth, shows sidebar toggle + user menu
└── Footer.astro     # Shared - accepts variant prop ('full' | 'compact')
```

### Auth-Aware Header Pattern
1. Fetch session on mount with `useEffect`
2. Show loading skeleton while fetching
3. Render login buttons OR user menu based on result
4. Role-based dashboard links: check `is_admin`, `is_creator`, `is_student_teacher`

### Mobile Navigation Pattern
- Use React state for menu toggle (not CSS-only)
- Overlay backdrop for closing mobile menu
- Slide-out animation with CSS transform
- Fixed positioning for mobile sidebar
