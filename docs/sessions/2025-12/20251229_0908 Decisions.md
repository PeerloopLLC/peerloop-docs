## Decision 2: Use react-dom/server.edge for React 19 SSR

**Type**: Implementation
**Driver**: Claude-driven (via web research)
**Working On**: Block 0.8 - Fixing CF Pages deployment error
**Trigger**: `MessageChannel is not defined` error during Worker deployment

**Options Considered**:
1. Update compatibility_date to newer date
2. Add MessageChannel polyfill
3. Use Vite alias to swap react-dom/server.edge <- **Chosen**
4. Disable SSR for React components (client:only)

**Decision**: Add Vite resolve alias in astro.config.mjs to use `react-dom/server.edge` instead of `react-dom/server.browser` in production builds.

**Rationale**:
- Root cause: React 19's browser SSR module requires MessageChannel which Cloudflare Workers don't have
- The `.edge` version is specifically designed for edge environments (Cloudflare, Deno) that support Web Streams
- This is the official workaround documented in GitHub issues
- Doesn't require disabling SSR or adding polyfills

**Consequences**:
- React SSR works correctly on Cloudflare Workers
- Fix applies only to production builds (dev uses normal react-dom/server)
- Added comment with GitHub issue references for future maintainers
- May need to revisit when React/Astro fix this upstream
