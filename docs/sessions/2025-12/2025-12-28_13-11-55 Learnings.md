# Session Learnings - 2025-12-28

## Documentation

### CLI Documentation System
- Created a hierarchical documentation system: CLI-QUICKREF.md as index → detail files (CLI-REFERENCE, CLI-TESTING, CLI-MISC, API-REFERENCE)
- Each file has consistent structure: title, intro with cross-references, sections with `---` separators, "Related Documentation" footer
- Templates stored in `.claude/templates/` ensure consistency across projects

### Portable vs Project-Specific Commands
- `/q-docs` handles standard docs (CLI-*, API-REFERENCE, TEST-COVERAGE, DEVELOPMENT-GUIDE, PLAN.md) - portable across projects
- `/q-local-docs` handles project-specific docs (research/tech/*.md, page specs, DB schema) - customized per project
- This separation keeps q-* commands reusable while allowing project customization

### Machine Name Standardization
- Standardized names (`MacMini`, `MBA-2017`) enable grep scanning of session files
- When session files mention machine names → triggers review of `research/tech/tech-013-devcomputers.md`
- Convention documented in both CLAUDE.md and DEVELOPMENT-GUIDE.md for consistency
