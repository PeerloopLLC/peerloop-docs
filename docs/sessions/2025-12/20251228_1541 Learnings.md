# Session Learnings - 2025-12-28_15-41-59

## Workflow

### Skill Circular References
- **Issue**: `/q-save-state` called `/q-end-session`, which in turn suggested running `/q-save-state` when pending work was detected
- **Solution**: Merge the two skills into one with a `SAVE_STATE` flag
- **Pattern**: When skill A calls skill B, and skill B has logic that redirects to skill A, merge them or add a "called-from" parameter to skip redundant checks

### Startup Checks via CLAUDE.md
- **Insight**: Rather than building checks into every skill or creating hooks, adding a "Session Startup Check" section to CLAUDE.md ensures Claude automatically performs the check at session start
- **Benefit**: Single point of truth, no maintenance burden on individual skills, works automatically
- **Use case**: Detecting stale `RESUME-STATE.md` files that were never consumed after `/compact`

## Technical

### Skill Caching
- **Observation**: The skill system may cache skill content - after editing a skill file, the old version may still be loaded
- **Workaround**: The updated logic should still be followed since CLAUDE.md provides the instructions

## Documentation

### CLAUDE.md as Behavior Configuration
- **Pattern**: CLAUDE.md can define runtime behaviors (like startup checks), not just static project information
- **Location matters**: Placing critical checks at the top of CLAUDE.md ensures they're seen first
