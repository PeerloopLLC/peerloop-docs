# Session Decisions - 2025-12-28

## Decision 1: Static Data vs API for Block 1 Pages

**Type**: Implementation
**Driver**: User-driven
**Working On**: Block 1.1-1.5 (Course Display pages)
**Trigger**: Discussion about whether to use real seed data from database

**Options Considered**:
1. Build API endpoints first (Block 1.6), then build UI components that fetch from DB
2. **Static data in components matching seed data, API later** <- Chosen
3. Hybrid: Build minimal API now, expand later

**Decision**: Use static data in React components that matches the seed data structure. Defer API endpoint creation to Block 1.6.

**Rationale**: Faster UI development without API dependencies. Data structure in components mirrors DB schema, so swapping to API later is straightforward.

**Consequences**:
- Data duplicated in components and seed file
- All Block 1 UI pages built rapidly (1.1-1.5 completed in one session)
- Block 1.6 (API Endpoints) remains pending
