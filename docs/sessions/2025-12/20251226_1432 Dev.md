### Technology Decisions Investigation

13. **User:** "/q-resume"
    - **Action:** Analyzed PLAN.md. Identified 5 pending decisions. Recommended starting with Authentication.

14. **User:** Confirmed ready to start.

#### Authentication Decision

15. **Action:** Researched Custom JWT vs Supabase Auth vs Clerk.
    - Found Clerk has known issues with Astro + Cloudflare Pages (node:async_hooks error).
    - Supabase Auth adds external dependency when not using Supabase DB.
    - Custom JWT using jose + bcryptjs + arctic is edge-compatible, no vendor lock-in.

16. **User:** "Custom JWT"
    - **Decision:** Custom JWT with jose, bcryptjs, arctic. Updated PLAN.md.

#### Calendar System Decision

17. **Action:** Researched Cal.com vs react-big-calendar vs custom.
    - Cal.com = full booking platform (overkill for our needs).
    - react-big-calendar = UI component only, build booking logic ourselves.

18. **User:** "react-big-calendar + custom booking logic"
    - **Decision:** react-big-calendar for UI, custom D1 logic for bookings. Updated PLAN.md.

#### Analytics Vendor Decision

19. **Action:** Researched PostHog vs Mixpanel vs Plausible.
    - PostHog: All-in-one (analytics + session replays + feature flags + A/B testing).
    - 1M free events/month. Also resolves Feature Flags decision.

20. **User:** "PostHog"
    - **Decision:** PostHog for analytics and session replays. Updated both Analytics and Feature Flags rows.

#### Error Tracking Decision

21. **Action:** Researched Sentry vs LogRocket vs Highlight.io.
    - Sentry: Industry standard, Cloudflare uses it, no overlap with PostHog.
    - LogRocket: Would duplicate PostHog's session replay.

22. **User:** "Sentry"
    - **Decision:** Sentry for error tracking. All 5 decisions now resolved.

#### Feature Flags Clarification

23. **User:** "You overrode the custom feature flags in favour of PostHog. Why did you do that?"
    - **Explanation:** Made incorrect assumption that PostHog analytics choice meant PostHog for feature flags too.

24. **User:** Asked if both D1 and PostHog feature flags can coexist.
    - **Answer:** Yes - two-tier approach makes sense:
      - Custom D1 for core features (must always work, no external dependency)
      - PostHog for A/B experiments (can gracefully fail)
