# Session Decisions - 2025-12-28

## Decision 1: Merge /q-save-state into /q-end-session

**Type**: Strategic
**Driver**: User-driven
**Working On**: Skill workflow optimization
**Trigger**: User noticed /q-save-state calls /q-end-session, creating potential circular reference

**Options Considered**:
1. Keep skills separate, fix circular reference with parameter <- Not chosen
2. Merge into single /q-end-session with SAVE_STATE flag <- **Chosen**
3. Have /q-save-state inline the steps it needs instead of calling /q-end-session <- Not chosen

**Decision**: Merge /q-save-state functionality into /q-end-session. When pending work is detected, offer "Save state for resume" as option 1, which sets SAVE_STATE=true and creates RESUME-STATE.md before proceeding with session documentation.

**Rationale**:
- Eliminates circular reference problem
- Single skill to maintain instead of two
- Clearer user mental model (one end-of-session command)
- The skills had overlapping purposes anyway

**Consequences**:
- Deleted /q-save-state.md
- Updated /q-end-session.md with Step 1 for resume state creation
- Users now use only /q-end-session for all end-of-session workflows

## Decision 2: Use CLAUDE.md for startup check instead of hooks or skill modifications

**Type**: Strategic
**Driver**: Claude-recommended, User-approved
**Working On**: Preventing stale RESUME-STATE.md files from being forgotten
**Trigger**: User concerned they might forget to run /q-resume after /compact

**Options Considered**:
1. Create a startup hook that runs on session start <- Not chosen (more complex)
2. Build check into every skill <- Not chosen (maintenance burden)
3. Add "Session Startup Check" section to CLAUDE.md <- **Chosen**

**Decision**: Add a "Session Startup Check" section at the top of CLAUDE.md instructing Claude to check for RESUME-STATE.md at session start and prompt the user with options.

**Rationale**:
- Simplest implementation (just add text to CLAUDE.md)
- Single point of truth
- No maintenance burden on individual skills
- Works automatically at every session start
- No hooks or additional infrastructure needed

**Consequences**:
- Added new section at top of CLAUDE.md
- Claude will now automatically detect stale resume states
- User gets prompted with 4 options: resume, view, delete, or ignore
