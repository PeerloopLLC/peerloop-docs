# Session Learnings - 2025-12-27

## Technical

### Tailwind CSS v4 Upgrade

1. **Official upgrade tool works well**: `npx @tailwindcss/upgrade` handles most of the migration automatically:
   - Migrates `tailwind.config.mjs` → CSS-based `@theme`
   - Updates class names in templates (shadow-sm→xs, shrink-0, etc.)
   - Converts `@tailwind` directives to `@import "tailwindcss"`
   - Converts plugins from `require()` to `@plugin`

2. **Astro requires manual integration change**: The upgrade tool doesn't handle the Astro-specific change from `@astrojs/tailwind` to `@tailwindcss/vite`. Must manually:
   - Uninstall `@astrojs/tailwind` first (conflicts with v4)
   - Install `@tailwindcss/vite`
   - Update `astro.config.mjs` to use Vite plugin instead of integration

3. **Git must be clean**: The upgrade tool requires a clean git directory or `--force` flag. Stashing uncommitted changes before running is recommended.

4. **Node.js 20+ required**: v4 upgrade tool won't run on older Node versions.

### Class Name Changes (v3 → v4)

| v3 | v4 |
|----|----|
| `shadow-sm` | `shadow-xs` |
| `backdrop-blur-sm` | `backdrop-blur-xs` |
| `flex-shrink-0` | `shrink-0` |
| `focus:outline-none` | `focus:outline-hidden` |

### Configuration Changes

- **JS → CSS**: Configuration moved from `tailwind.config.mjs` to `@theme` in CSS
- **Plugins**: Use `@plugin '@tailwindcss/forms'` instead of `require()`
- **Dark mode**: Use `@custom-variant dark (&:is(.dark *))` instead of `darkMode: 'class'`
- **Custom utilities**: Use `@utility` directive instead of `@layer components`

## Documentation

### Tech Document Updates

When upgrading technologies, remember to update the corresponding tech doc in `research/tech/`:
- Add version number and update date
- Document migration steps and caveats
- Include "what's new" comparison tables
- Update code examples to new syntax

### /q-docs Enhancement

Added triggers for tech doc updates to `/q-docs` Change Detection Matrix:
- Package versions, additions, or deletions
- Upgraded or reconfigured a technology
- Discovered a caveat or gotcha

This ensures tech docs stay current as the project evolves.
