# Session Learnings - 2025-12-27

## Technical

### Vitest + Astro Integration
- Use `getViteConfig` from `astro/config` to integrate Vitest properly with Astro
- The `vitest.setup.ts` file imports `@testing-library/jest-dom/vitest` for matcher support
- TypeScript requires explicit types in `tsconfig.json`: `"types": ["vitest/globals", "@testing-library/jest-dom"]`

### TypeScript + Jest-DOM Matchers
- Adding `vitest/globals` and `@testing-library/jest-dom` to `compilerOptions.types` resolves "toBeInTheDocument" and "toHaveClass" type errors
- Must also include the setup file in `tsconfig.json` `include` array

### Playwright Configuration
- Astro dev server runs on port 4321 by default
- Use `webServer.reuseExistingServer: !process.env.CI` to avoid starting duplicate servers locally
- Installing just `chromium` browser is sufficient for initial setup (`npx playwright install chromium`)

### Wrangler CLI for Cloudflare
- D1 migrations use `wrangler d1 migrations apply <db-name> --local/--remote`
- Pages dev server: `wrangler pages dev ./dist`
- `.dev.vars` file holds local environment variables (gitignored)

## Workflow

### Sidepanel Pattern for Page Specs
- LandingLayout has `showSidepanel` prop (default false)
- Pages must pass `showSidepanel={true}` AND provide `<Fragment slot="sidepanel">` content
- Navigation buttons are generated from `pageData.connections.outgoing`

### Testing Infrastructure Setup Order
1. Install Vitest + testing-library + jsdom
2. Create vitest.config.ts with Astro integration
3. Create vitest.setup.ts for jest-dom
4. Update tsconfig.json with types
5. Install Playwright separately
6. Create playwright.config.ts with webServer config
7. Add npm scripts

## Documentation

### Environment Variables Template
- Create `.dev.vars.example` with all required env vars documented
- Actual `.dev.vars` is gitignored
- Group by service: JWT, OAuth, Stripe, Stream.io, Resend, etc.

## Architecture

### NPM Script Organization
- Group scripts by purpose: dev, build, test, cloudflare, database
- Use prefixes: `test:*`, `cf:*`, `db:*` for discoverability
- Keep utility scripts (parse-page, generate-pages) at the end
