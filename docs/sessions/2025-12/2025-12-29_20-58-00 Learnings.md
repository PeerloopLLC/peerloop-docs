# Session Learnings - 2025-12-29

## Technical

### Admin API Authentication Pattern
- No separate middleware file needed for admin routes
- Use existing `requireRole(cookies, jwtSecret, ['admin'])` directly in API handlers
- The auth system already has role-based access control built in (`src/lib/auth/session.ts`)
- Pattern: Call `requireRole` at top of handler, it throws Response on failure

### Soft Delete Implementation
- Added `deleted_at`, `suspended_at`, `suspended_reason` fields to User type
- SQLite uses TEXT for timestamps (ISO strings)
- Pattern: Check `deleted_at IS NULL` in queries to exclude soft-deleted records
- Block deletion if user has active enrollments (referential integrity via business logic)

### TypeScript Generic Constraints
- When using generics with `extends Record<string, unknown>`, interface types without index signatures fail
- Solution: Use `<T>` without constraints when the generic is only used for type inference
- API response typing: Use `as Type` assertion on `response.json()` result

## Architecture

### Shared Admin Components
Created reusable component library at `src/components/admin/`:

| Component | Purpose | Key Props |
|-----------|---------|-----------|
| AdminDataTable | Sortable, paginated table | columns, data, keyField, onSort |
| AdminFilterBar | Search + filter controls | searchValue, filters, actions |
| AdminPagination | Page navigation | page, totalPages, onPageChange |
| AdminDetailPanel | Slide-in detail/edit | isOpen, onClose, title, footer |
| AdminActionMenu | Row actions dropdown | actions array |

Helper exports from AdminDetailPanel:
- `PanelSection` - Section grouping with title
- `PanelField` - Key-value display
- `StatusBadge` - Colored status indicator
- `RoleBadge` - User role badge

### API Response Format
Consistent paginated response shape:
```typescript
{
  items: T[];
  total: number;
  page: number;
  limit: number;
  totalPages: number;
}
```

## Workflow

### Tailwind v4 Class Updates
When creating new components, watch for v3→v4 class renames:
- `outline-none` → `outline-hidden`
- `bg-opacity-X` → `bg-color/X` (e.g., `bg-gray-500/75`)
- `ring-opacity-X` → `ring-color/X`
- `flex-shrink-0` → `shrink-0`

Run `npm run check:tailwind` before committing new components.

### Admin Page Pattern
1. JSON spec exists (`src/data/pages/admin/*.json`) with API requirements
2. Create API endpoints under `src/pages/api/admin/`
3. Create React component in `src/components/admin/`
4. Update `.astro` page to use component with `client:load`
