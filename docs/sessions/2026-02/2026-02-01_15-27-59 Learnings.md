# Session Learnings - 2026-02-01

## 1. Centralized Icon Library Eliminates Duplication
**Topics:** astro, workflow

**Context:** AppNavbar.tsx had 9 inline SVG icon components, and similar icons were duplicated across marketing components (CheckIcon appeared 4 times in different files).

**Learning:** Creating a centralized `icons.tsx` file with parameterized icons (`className` prop with defaults) provides:
- Single source of truth for each icon
- Consistent sizing and styling via defaults
- Tree-shaking capability (unused icons excluded)
- Discoverability (developers can browse available icons)

**Pattern:**
```tsx
// Icons accept optional className with sensible default
export const CheckIcon = ({ className = 'h-5 w-5' }: IconProps) => (
  <svg className={className} fill="none" stroke="currentColor" viewBox="0 0 24 24">
    <path ... />
  </svg>
);

// Usage - default size
<CheckIcon />

// Usage - custom size
<CheckIcon className="h-4 w-4 text-green-600" />
```

## 2. Z-Index Layering for "Slide From Beneath" Effect
**Topics:** astro, workflow

**Context:** Wanted slide-out panels to appear to emerge from behind the navbar rather than overlay it.

**Learning:** To create a "slide from beneath" visual effect:
1. Position panel at navbar's right edge (`left-64` for 256px navbar)
2. Give panel lower z-index than navbar (`z-40` vs `z-50`)
3. Use transform for animation (`-translate-x-full` when closed)
4. Panel starts hidden behind navbar, slides right to reveal

The layering order matters:
- `z-30` - invisible click-catcher overlay
- `z-40` - slide panel
- `z-50` - navbar (always on top)

## 3. useEffect Must Follow Variable Declarations
**Topics:** astro, workflow

**Context:** Role-based home behavior useEffect referenced variables (`isAdmin`, `canTeachCourses`, etc.) that were declared later in the component.

**Learning:** In React functional components, `useEffect` hooks that reference variables must be placed AFTER those variable declarations. TypeScript will catch this with "Block-scoped variable used before its declaration" error.

**Pattern:**
```tsx
// Wrong - useEffect before declarations
useEffect(() => {
  if (isAdmin) { ... }  // Error: isAdmin used before declaration
}, [isAdmin]);

const isAdmin = user?.isAdmin ?? false;

// Correct - declarations before useEffect
const isAdmin = user?.isAdmin ?? false;

useEffect(() => {
  if (isAdmin) { ... }  // Works
}, [isAdmin]);
```

## 4. Git Restore Recovers Uncommitted Deletions
**Topics:** workflow

**Context:** Accidentally deleted `_src/pages/` folder with `rm -rf` after files had been moved there with `git mv`.

**Learning:** When files are tracked by git (moved via `git mv`) but deletion wasn't committed (`rm -rf` without `git rm`), they show as unstaged deletions. Running `git restore <path>` recovers all files from the git index.

**Pattern:**
```bash
# Check for deleted files
git status --short | grep "^D"

# Restore deleted folder
git restore _src/pages/
```

This works because `git mv` maintains tracking - the deletion was an "unstaged change" that `git restore` undoes.

## 5. Trigger-Once Pattern for Home Navigation
**Topics:** astro, workflow

**Context:** Role-based home behavior needed to auto-open panels or navigate based on user type, but only once per page load.

**Learning:** Use a state flag to prevent re-triggering navigation/panel actions on re-renders:

**Pattern:**
```tsx
const [hasTriggeredHomeAction, setHasTriggeredHomeAction] = useState(false);

useEffect(() => {
  if (currentPath !== '/') return;
  if (hasTriggeredHomeAction) return;  // Only run once
  if (authStatus === 'loading') return;

  setHasTriggeredHomeAction(true);

  // Now safe to trigger navigation or panel opens
  if (isAdmin) {
    window.location.href = '/messages';
    return;
  }
  // ... other role checks
}, [currentPath, authStatus, hasTriggeredHomeAction, ...]);
```

This prevents:
- Re-opening panels after user closes them
- Multiple navigation redirects
- Actions during auth loading state
