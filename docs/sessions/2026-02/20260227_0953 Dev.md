### Discovering the __peerloop Race Condition Bug

**Claude:** All 19 tests failed. Homepage title test passed but navigation tests failed. Debug test revealed the login modal never rendered — `Dialog: 0, EmailByLabel: 0`. Console error: `Cannot read properties of undefined (reading 'authError')`.

**Claude:** Traced the bug: `auth-modal.ts`'s `getPeerloopGlobal()` creates `window.__peerloop = {}` (without `networkState`). When this runs before `current-user.ts`'s `ensureGlobal()`, the latter skips initialization because `window.__peerloop` already exists as a truthy object. `getNetworkState()` then returns `undefined`, crashing AppNavbar hydration.

### Fixing the Bug

**Claude:** Added a defensive check to `ensureGlobal()` in `current-user.ts`: after confirming `window.__peerloop` exists, also checks and initializes `networkState` if missing. Debug test confirmed fix: `Dialog: 1, EmailByLabel: 1`.
