# Session Decisions - 2026-02-25

## 1. Community and Course Restructuring
**Type:** Strategic
**Topics:** d1

**Trigger:** Client direction to reorganize communities and add Gabriel's Q-System courses. The old structure had creator mismatches (Gabriel owned Vibe Coding community but Guy's courses were the actual content).

**Options Considered:**
1. Keep existing community structure, just add Q-System courses ← Not chosen (leaves mismatches)
2. Restructure communities to match actual creator ownership ← Chosen
3. Merge all into fewer communities ← Not chosen (loses topic focus)

**Decision:** Restructured to 3 communities with clean creator ownership:
- `comm-ai-for-you` (Guy) → AI Tools Overview, Intro to Claude Code, Vibe Coding 101
- `comm-automation-majors` (Guy) → Intro to n8n
- `comm-q-system` (Gabriel) → Intro to the Q-System, Intermediate Q-System

**Rationale:** Each community now has a single creator who owns all courses within it. The "AI for You" umbrella naturally groups Guy's AI/coding courses, while Gabriel gets a dedicated Q-System community.

**Consequences:** All community IDs changed (old: `comm-ai-tools`, `comm-n8n-automation`, `comm-vibe-coding`; new: `comm-ai-for-you`, `comm-automation-majors`, `comm-q-system`). Progression IDs updated accordingly. Mock-data.ts does not track communities so no change needed there.

## 2. Single Seed File for All Dev Data
**Type:** Implementation
**Topics:** d1, deployment

**Trigger:** Seed file grew from ~320 lines (18 tables) to ~780 lines (all tables). Question of whether to split into multiple files.

**Decision:** Keep as a single `migrations-dev/0001_seed_dev.sql` file with clearly labeled parts (Part 0-14).

**Rationale:** D1 handles it without issue, the `db:seed:local` npm script expects one file, and a single file is easier to reason about for FK ordering. Splitting would require either multiple `wrangler d1 execute` calls or a wrapper script.

**Consequences:** File is organized into 14+ labeled sections with clear headers for navigation.
