# Session Log - 2026-02-17

## Development Transcript

### Community Count Bug Investigation

**User:** Reported that `/discover/communities` shows "4 available to join" but only displays 3 community cards. Provided a screenshot.

**Claude:** Traced the data flow from the API endpoint through the page component. Found the root cause: the SQL count query included system communities (The Commons) in its total, but the page filtered them out of the rendered list with `!c.isSystem` while using the raw `data.total` for the label. Fixed at both layers — added `AND c.is_system = 0` to the API's discover filter SQL, and changed the page to derive the count from `communities.length` instead of `data.total`.

### Breadcrumb System Design Discussion

**User:** Asked about implementing breadcrumbs in the Main Panel. Raised two design questions: (1) should breadcrumbs be inside the MainPanel or above it, and (2) how to handle breadcrumbs when users arrive from external sources or navigate via links from other pages.

**Claude:** Explored the codebase to understand the current layout architecture (AppLayout, AppNavbar, existing hand-coded breadcrumbs on 2 pages, LegacyAppLayout's breadcrumb prop). Presented three approaches for navigation context: route-based (static), history-based (sessionStorage), and hybrid (route-based + `?via=` query param). Recommended hybrid approach. Discussed breadcrumb placement above vs below the title.

### Breadcrumb Design Decisions
