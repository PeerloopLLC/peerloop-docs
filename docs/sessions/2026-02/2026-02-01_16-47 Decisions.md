# Session Decisions - 2026-02-01

## 1. Window Focus Refresh for CurrentUser Updates
**Type:** Implementation
**Topics:** astro, auth

**Trigger:** Discussion of how external changes (admin/creator actions) propagate to a user's `currentUser` global. The "push" side is handled by calling `refreshCurrentUser()` after mutations. The question was how to "pull" these updates on the client.

**Options Considered:**
1. Page navigation only (already implemented)
2. Window focus refresh ← Chosen
3. Periodic polling (every N minutes)
4. Lightweight version check endpoint
5. SSE/WebSocket push

**Decision:** Implement window focus refresh in AppNavbar. When the user returns to the tab, `refreshCurrentUser()` is called to fetch fresh data.

**Rationale:**
- Simple to implement (5 lines of code)
- No server changes needed
- Covers the main scenario: user is away, admin makes change, user returns
- Page navigation already refreshes on route change
- Polling/push are overkill for MVP where capability changes are rare

**Consequences:**
- Added `refreshCurrentUser` import to AppNavbar.tsx
- Added `useEffect` with window focus listener
- Updated `docs/tech/tech-017-state-management.md` with new "Data Freshness & External Changes" section
- Documented as implemented pattern in tech doc
