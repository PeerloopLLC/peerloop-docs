# Session Decisions - 2026-02-01

## 1. Layout Naming Convention for Parallel Development
**Type:** Implementation
**Topics:** astro, workflow

**Trigger:** Need to reorganize routing structure while keeping existing pages functional. Layout names conflicted with intended new structure.

**Options Considered:**
1. Rename old layouts with underscore prefix (`_AppLayout`) - rejected due to ESLint parsing issues
2. Rename old layouts with `Legacy` prefix (`LegacyAppLayout`) ← Chosen
3. Rename old layouts with `Old` prefix (`OldAppLayout`)
4. Use temporary names for new layouts, rename later

**Decision:** Use `Legacy` prefix for old layouts being phased out.

**Rationale:**
- `Legacy` is clearer and more descriptive than `Old`
- Works with ESLint/Astro parser (unlike underscore prefix)
- Signals intent: these layouts will eventually be removed
- Allows the "clean" name (`AppLayout`) to be used for the new implementation

**Consequences:**
- Renamed `AppLayout.astro` → `LegacyAppLayout.astro` (30 routes)
- Renamed `DashLayout.astro` → `AppLayout.astro` (7 routes)
- All imports updated across affected pages

**Important:** Added to `DECISIONS.md` Section 11 (New Routing)
