# Session Decisions - 2026-02-19

## 1. Informational Hook Scripts Should Always Exit 0
**Type:** Implementation
**Topics:** workflow

**Trigger:** `scripts/check-env.sh` was causing "SessionStart:startup hook error" by exiting with code 1 when missing tools were detected.

**Options Considered:**
1. Remove the hook from `.claude/settings.json` entirely
2. Change exit code to 0 so errors are reported but don't block startup <- Chosen
3. Wrap the hook in a shell that swallows the exit code

**Decision:** Changed `check-env.sh` to always exit 0, even when errors are found. The script still reports all problems in its output (which Claude Code displays), but no longer causes a hook error.

**Rationale:** The env check is informational — it tells the developer what needs fixing. Blocking Claude Code startup for a missing tool is counterproductive since Claude Code itself can help fix the issue. Non-zero exits should be reserved for hooks that genuinely need to prevent a session from starting.

**Consequences:** Modified `scripts/check-env.sh` lines 14-16 (added nvm sourcing) and line 172 (exit 0 instead of exit 1).
