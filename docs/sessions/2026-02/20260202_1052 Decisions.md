# Session Decisions - 2026-02-02

## 1. Use Astro View Transitions for SPA-like Navigation
**Type:** Strategic
**Topics:** astro

**Trigger:** Need for smooth, flicker-free navigation without losing SSR benefits.

**Options Considered:**
1. Custom SPA router (more control, but complex)
2. Astro View Transitions ← Chosen
3. Full SPA with client-side routing (loses SEO)

**Decision:** Use Astro's built-in View Transitions (ClientRouter) for SPA-like navigation.

**Rationale:**
- No flicker (animation masks page load)
- Full SEO retained (SSR always)
- Deep linking works natively
- Simpler than custom router
- AppNavbar can persist across navigations with `transition:persist`

**Consequences:**
- Added `<ClientRouter />` to AppLayout and LandingLayout
- AppNavbar marked with `transition:persist="app-navbar"`
- Main content area uses `transition:animate="fade"`
- AppNavbar listens for `astro:page-load` to sync path

## 2. Accept Initial Load Delay, Optimize Subsequent Navigations
**Type:** Strategic
**Topics:** astro, auth

**Trigger:** Initial page load shows skeleton for 3+ seconds on slower hardware while JS loads.

**Options Considered:**
1. Use `client:only` to skip SSR (loses SEO, no improvement)
2. Server-side sessions with cookies (significant refactor)
3. Accept initial delay, make subsequent navigations instant ← Chosen

**Decision:** Accept that first page load requires JS to load (skeleton shows), but use View Transitions to make subsequent navigations instant.

**Rationale:**
- First load delay is JS download/parse time, not React inefficiency
- No React optimization can fix this - JS must load first
- Server can't access localStorage for auth state
- View Transitions solve the "repeated reload" problem
- Production build is much faster than dev mode

**Consequences:**
- Users see skeleton on first visit (unavoidable without server sessions)
- Navigations within app are instant (AppNavbar persists)
- Dev mode testing gives misleading slow performance
- Should test with `npm run preview` for accurate timing
