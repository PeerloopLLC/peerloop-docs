# Session Decisions - 2026-02-04

## 1. Remove `/course/[slug]/discuss` Route
**Type:** Implementation
**Topics:** astro, workflow

**Trigger:** User asked why both `/discuss` and `/feed` routes existed for courses.

**Options Considered:**
1. Keep both routes with distinct purposes (forum vs social feed)
2. Remove `/discuss` as superseded by `/feed` ← Chosen
3. Consolidate into a single route with query param switching

**Decision:** Remove `/course/[slug]/discuss` from routing documentation and codebase. The `/feed` route with tag filtering (`?tag=help`, `?tag=general`) serves the same purpose.

**Rationale:**
- `/discuss` was original design (CDIS - forum-style Q&A with threads)
- `/feed` was implemented in Sessions 181-183 using Stream.io
- Session 183 "Bookmarkable Feed Tabs" decision only lists About/Feed/Curriculum tabs - no Discuss tab
- Maintaining two routes for the same conceptual feature adds confusion

**Consequences:**
- Updated `tech-021-url-routing.md` to remove `/discuss` references
- Deleted `src/pages/course/[slug]/discuss.astro`
- `/feed` is now the sole course discussion route

## 2. No Redirects for Pre-Launch Route Changes
**Type:** Strategic
**Topics:** astro, workflow

**Trigger:** The deleted `discuss.astro` file contained a 301 redirect to `/feed`.

**Options Considered:**
1. Keep redirect file for backwards compatibility
2. Delete redirect file entirely ← Chosen

**Decision:** Delete redirect files for route changes made before production launch.

**Rationale:**
- Pre-launch, no real users have bookmarked URLs
- Redirect files add maintenance overhead
- Post-launch, redirects become valuable for SEO and user bookmarks
- Routing documentation serves as record of what routes exist

**Consequences:**
- Simpler codebase with fewer files to maintain
- Will need to add redirects for any route changes post-launch

## 3. Single Tab Component for All Course Subroutes
**Type:** Implementation
**Topics:** astro

**Trigger:** Discovered `index.astro` used `CourseDetail` while `feed.astro` used `CourseTabs`, causing inconsistent UX.

**Options Considered:**
1. Keep separate components for different views
2. Use `CourseTabs` for all course subroutes ← Chosen
3. Create a wrapper component that switches based on route

**Decision:** All course subroutes (`/course/[slug]`, `/course/[slug]/feed`, etc.) use `CourseTabs` with different `initialTab` values.

**Rationale:**
- Consistent UI across all course pages
- Tab navigation works correctly (clicking About tab shows About content)
- Easier maintenance (single component to update)
- Matches the Session 183 architecture decision

**Consequences:**
- Updated `index.astro` to use `CourseTabs` with `initialTab="about"`
- `CourseDetail.tsx` is now orphaned (may be deleted)
- Both pages now fetch same data and pass to same component

## 4. CourseHero Above Tabs (Not Inside About Tab)
**Type:** Implementation
**Topics:** astro

**Trigger:** Merging `CourseDetail` content into `CourseTabs` required deciding where `CourseHero` should render.

**Options Considered:**
1. Render CourseHero inside About tab (disappears on other tabs)
2. Render CourseHero above tabs (always visible) ← Chosen
3. Create a condensed hero for tabs, full hero for About

**Decision:** `CourseHero` renders above `CourseTabs` in the Astro page, always visible regardless of active tab.

**Rationale:**
- Price and CTA should always be visible for conversion
- Course identity provides context on Feed/Curriculum tabs
- Matches YouTube pattern (video player above tabs)
- Better for mobile (no need to scroll back to see price)

**Consequences:**
- Astro pages render `CourseHero` then `CourseTabs`
- Added `fullWidth` prop to AppLayout for hero to span full width
- Removed redundant "Course Info Bar" from About tab (hero has that info)

## 5. Merge CourseDetail Content Into CourseTabs
**Type:** Implementation
**Topics:** astro

**Trigger:** After switching to `CourseTabs`, content from `CourseDetail` was missing from the About tab.

**Options Considered:**
1. Keep both components, use CourseDetail for non-tabbed view
2. Merge all CourseDetail content into CourseTabs About tab ← Chosen
3. Create shared subcomponents used by both

**Decision:** Migrate relevant sections from `CourseDetail` into `CourseTabs`:
- Prerequisites section
- What's Included section
- Who Is This For? section
- Tags display
- Related Courses section

**Rationale:**
- Single source of truth for course page content
- Avoids maintaining two similar components
- CourseTabs already had most sections (objectives, curriculum, testimonials, teachers, creator)

**Consequences:**
- `CourseTabs` now accepts additional props: `prerequisites`, `includes`, `targetAudience`, `tags`, `relatedCourses`
- Astro pages fetch additional data and pass to component
- `CourseDetail.tsx` is fully orphaned (candidate for deletion)
- Omitted: "The PeerLoop Difference" marketing section (promotional filler)
