# Session Decisions - 2026-02-24

## 1. Reschedule via Booking Page Navigation
**Type:** Implementation
**Topics:** video, astro

**Trigger:** Needed to add a "Reschedule" button to the session room UI. Two options: build an inline reschedule modal within SessionRoom, or navigate to the existing booking page with a reschedule parameter.

**Options Considered:**
1. Inline reschedule modal in SessionRoom (date/time picker, availability check, PATCH call)
2. Navigate to `/course/:slug/book?reschedule=:id` ← Chosen

**Decision:** Navigate to the booking page with `?reschedule=sessionId` query parameter rather than building inline rescheduling UI.

**Rationale:** The booking page (`SessionBooking` component) already has the availability calendar, S-T selection, and time slot picker. Reusing it avoids duplicating complex UI. The `reschedule` query param can pre-fill the existing session's details and call PATCH instead of POST on submit.

**Consequences:** Added `handleReschedule()` to SessionRoom that does `window.location.href = /course/${slug}/book?reschedule=${id}`. Reschedule button appears on both early and joinable screens. The booking page will need a small update in a future session to detect the `reschedule` param and use PATCH instead of POST.
