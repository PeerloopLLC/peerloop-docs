# Session Learnings - 2026-02-19

## 1. Claude Code Hooks Run in a Minimal Shell Environment
**Topics:** workflow, deployment

**Context:** The `scripts/check-env.sh` SessionStart hook was reporting "startup hook error" because it couldn't find `node`, `npm`, `wrangler`, or `stripe`.

**Learning:** Claude Code hooks execute in a minimal shell that does NOT source `~/.zshrc` or `~/.bash_profile`. This means tools managed by nvm, Homebrew (`/opt/homebrew/bin`), or other shell profile–dependent managers are not on the PATH. Hook scripts must explicitly source their dependencies or add paths.

**Pattern:**
```bash
# Add to top of hook scripts that need nvm
export NVM_DIR="${NVM_DIR:-$HOME/.nvm}"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
```

## 2. Hook Exit Codes Determine Success/Error Display
**Topics:** workflow

**Context:** `check-env.sh` exited with code 1 when it found missing tools, which Claude Code displayed as "SessionStart:startup hook error".

**Learning:** Claude Code treats any non-zero exit code from a SessionStart hook as an error. Informational/diagnostic scripts that report problems but shouldn't block startup must always exit 0. Reserve non-zero exits for hooks that should genuinely block the session.

## 3. nvm Sourcing Fails in zsh but Works in bash -c
**Topics:** workflow

**Context:** When trying to install Node via the Bash tool, `source "$NVM_DIR/nvm.sh"` failed silently in the tool's default zsh context.

**Learning:** The Bash tool runs in zsh by default. nvm sourcing can behave differently in zsh vs bash. Wrapping nvm commands in `bash -c 'export NVM_DIR=...; source .../nvm.sh; <command>'` is a reliable workaround when the Bash tool's zsh doesn't cooperate.

---

## 4. Homebrew Install Requires Interactive Terminal
**Topics:** workflow

**Context:** Attempted to install Homebrew via the Bash tool but it failed with "Need sudo access on macOS".

**Learning:** The Homebrew installer requires interactive sudo (password prompt) and TTY access. It cannot be installed through Claude Code's Bash tool. Must be done directly in the user's terminal. Same applies to any command requiring interactive authentication.

---

## 5. Stripe CLI Login Uses Terminal Pairing Code
**Topics:** stripe, workflow

**Context:** User ran `stripe login` but wasn't sure where to find the verification words.

**Learning:** The Stripe CLI login flow displays a pairing code (series of words) in the terminal output. The browser page then asks the user to confirm they see those same words. The words appear a few lines above the "waiting" message in the terminal — easy to miss if you switch to the browser quickly.
