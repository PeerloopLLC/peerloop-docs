### Flaky ModeratorQueue Test on CF Pages CI

**User:** Reported that the latest merge to main failed a unit test — ModeratorQueue "hides action buttons for actioned flags" threw `Unable to find an accessible element with the role "row"`.

**Claude:** Investigated the test and component. ### Flaky R2 Health Check Test

**User:** Shared a second CI failure from the same build — R2 health check test "performs write, read, and delete operations" failed because `mockR2.delete` was never called.

**Claude:** Found that the mock's `get().text()` generated its own `new Date().toISOString()` independently from the endpoint's write. On slow CI the timestamps differed, causing a content mismatch error that short-circuited before `delete`. Fixed by capturing written content and returning it on read (matching the pattern in the passing first test).
