# Session Decisions - 2026-02-01

## 1. Use SyntheticEvent for Form Submit Handlers
**Type:** Implementation
**Topics:** astro, workflow

**Trigger:** React's `FormEvent` type is deprecated in `@types/react`, causing TypeScript warnings on all form handler functions.

**Options Considered:**
1. Keep `React.FormEvent<HTMLFormElement>` and accept warnings
2. Use `React.SyntheticEvent<HTMLFormElement>` ← Chosen
3. Use native DOM `Event` with type assertions
4. Suppress warnings with `@ts-ignore`

**Decision:** Use `React.SyntheticEvent<HTMLFormElement>` for all form submission handlers.

**Rationale:**
- `SyntheticEvent` is the base type that `FormEvent` extends
- Recommended by the deprecation notice itself
- Provides the same `preventDefault()` and target typing we need
- No functionality loss

**Consequences:**
- Updated 9 source files with form handlers
- No behavioral changes, just type annotation updates

## 2. Replace Recharts Cell with Data-Driven Colors
**Type:** Implementation
**Topics:** astro, workflow

**Trigger:** The `Cell` component from recharts triggers deprecation warnings in TypeScript, adding noise to build output.

**Options Considered:**
1. Keep `Cell` and suppress warnings with `@ts-expect-error` - rejected (directive flagged as unused)
2. Keep `Cell` and accept warnings
3. Add `fill` property to data entries ← Chosen
4. Switch to a different charting library

**Decision:** Pre-process chart data to include `fill` property, which recharts uses automatically for coloring.

**Rationale:**
- Cleaner code (no Cell children mapping)
- No deprecation warnings
- Same visual result
- Easier to maintain

**Consequences:**
- Updated `BarChart.tsx` and `PieChart.tsx`
- Removed `Cell` import
- Added `coloredData` transformation before rendering

## 3. Default Slot for Production Content in AppLayout
**Type:** Implementation
**Topics:** astro

**Trigger:** AppLayout had spec view content in the unnamed slot and real content in a named slot, which felt backwards.

**Options Considered:**
1. Keep spec view as default (unnamed slot)
2. Make real content the default (unnamed slot) ← Chosen

**Decision:** Swap slots so the unnamed/default slot receives production content, and `slot="spec"` receives the PageSpecView component.

**Rationale:**
- Production content is the "real" use case - should be the default
- Spec view is opt-in for development - should require explicit named slot
- More intuitive: default behavior = production behavior

**Consequences:**
- Updated AppLayout.astro slot structure
- Updated all 7 pages using AppLayout to use new slot pattern
