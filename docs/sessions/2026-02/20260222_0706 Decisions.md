# Session Decisions - 2026-02-22

## 1. Remove Deprecated Auth-Modal Aliases
**Type:** Implementation
**Topics:** auth, testing

**Trigger:** `astro check` flagged 4 deprecation hints for `closeLoginModal` and `handleLoginSuccess` in the test file. Investigation showed these deprecated exports in `src/lib/auth-modal.ts` had zero production consumers — only the backwards-compatibility test used them.

**Options Considered:**
1. Suppress warnings in the test file (e.g., `// @ts-expect-error`) — preserves aliases
2. Keep aliases, remove only the backwards-compat tests — silences hints but leaves dead code
3. Remove deprecated exports from source + remove backwards-compat tests ← Chosen

**Decision:** Remove `closeLoginModal` and `handleLoginSuccess` deprecated aliases from `src/lib/auth-modal.ts` and delete the "Backwards Compatibility" test describe block (2 tests).

**Rationale:** No production code imports these aliases anymore. The canonical names (`closeAuthModal`, `handleAuthSuccess`) are used everywhere. Keeping dead aliases and their tests adds maintenance burden and noise in tooling output. Clean removal is safe since grep confirmed zero non-test consumers.

**Consequences:** `auth-modal.test.ts` dropped from 28 to 26 tests. `astro check` went from 7 hints to 0.
