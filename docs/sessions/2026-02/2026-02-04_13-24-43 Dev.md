**User:** Cost implications of separate feeds?

**Claude:** Researched Stream pricing:

- Genesis cohort easily within free tier
- Fan-out is the real cost driver

**User:** Can pagination limit what Stream returns?

**Claude:** Confirmed pagination already implemented (`limit`, `offset`, `id_lt`). Current TownHallFeed fetches 25 activities per request with infinite scroll.

**User:** How to get all posts on Topic X?

2. Client-side filtering (doesn't scale)
3. D1 index hybrid
4. Wait for Stream v3

**User:** Client-side filtering won't work for mobile with thousands of posts

**Claude:** Agreed - client-side filtering is a non-starter. Only viable options are separate feeds or D1 hybrid.

**User:** Separate feeds require knowing topics upfront - limitations for business

**Claude:** Acknowledged Stream v2's limitations for ad-hoc search. Presented alternatives: Algolia, Meilisearch, D1 FTS5, or hybrid approaches.

### Algorithmic Feeds Research

**User:** Client wants Stream's "algorithmic feeds" - can we control the parameters?

**Claude:** Researched ranked feeds extensively:
- Custom scoring formulas supported
- Variables: time decay, custom fields, reactions, external parameters
- Per-user personalization via `ranking_vars` at query time
- **Critical limitation:** Cannot combine ranked feeds with date filtering (`id_lt`/`id_gt`)

**User:** Can ranked feeds combine with date ranges?

**Claude:** Confirmed NO - you can only use `limit` and `offset` with ranked feeds. Workarounds: aggressive time decay, two-phase query, or D1 hybrid.
