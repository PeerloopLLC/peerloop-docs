# Session Decisions - 2026-02-05

## 1. Adopt Subroute Pattern for Community Tabs
**Type:** Architecture
**Topics:** astro, architecture

**Trigger:** Community tabs used client-side state (`useState`), meaning tab selection wasn't reflected in URL and couldn't be bookmarked or shared.

**Options Considered:**
1. Keep client-side only with `useState` - Simple but no bookmarkable URLs
2. Use query params (`?tab=members`) - Bookmarkable but feels like second-class routing
3. Subroute folder pattern (`/community/[slug]/members`) ← Chosen
4. Hash-based routing (`#members`) - SEO unfriendly, feels outdated

**Decision:** Convert `/community/[slug].astro` to folder pattern `/community/[slug]/` with separate .astro files for each tab (index, courses, resources, members).

**Rationale:**
- Matches the established course pattern (`/course/[slug]/feed`)
- Provides bookmarkable, shareable URLs
- Server renders correct tab immediately (no flash)
- Browser back/forward works naturally with `popstate` listener

**Consequences:**
- Created 4 new files in `src/pages/community/[slug]/`
- Deleted single `[slug].astro` file
- Added `initialTab` prop to `CommunityTabs.tsx`
- Updated tech-021 routing documentation
