# Session Decisions - 2026-02-03

## 1. Use EXISTS Subqueries for Derived State Checks
**Type:** Implementation
**Topics:** d1, architecture

**Trigger:** Found 5 files with SQL queries referencing non-existent `is_creator` and `is_student_teacher` columns on the users table.

**Options Considered:**
1. Add the columns back to the schema
2. Use JOINs to derive state
3. Use EXISTS subqueries to derive state ← Chosen

**Decision:** Replace deprecated column references with `EXISTS (SELECT 1 FROM ...)` subqueries that derive state from relationship tables.

**Rationale:**
- Aligns with the Permission vs State architecture established in Session 163
- EXISTS is semantically clearer for existence checks
- Avoids duplicate rows that JOINs could produce
- SQLite optimizer handles EXISTS efficiently
- Maintains single source of truth in relationship tables

**Consequences:**
- Updated 3 SSR loaders (`home.ts`, `creators.ts`, `teachers.ts`)
- Queries now derive `is_creator` from `courses` table presence
- Queries now derive `is_student_teacher` from `student_teachers` table presence
- All 4750 tests pass

## 2. Remove Redundant User Flag Updates from Certificate Endpoints
**Type:** Implementation
**Topics:** d1, architecture, auth

**Trigger:** Found `approve.ts` and `revoke.ts` endpoints trying to UPDATE `users.is_student_teacher` - a column that no longer exists.

**Options Considered:**
1. Re-add the `is_student_teacher` column and keep the updates
2. Remove the UPDATE statements entirely ← Chosen
3. Add the column but make it a computed/generated column

**Decision:** Remove the `UPDATE users SET is_student_teacher = ...` statements from both endpoints.

**Rationale:**
- State is now derived from the `student_teachers` table at query time
- The act of inserting/deactivating ST records IS the state change
- Eliminates potential for flag/table desynchronization bugs
- Simpler code with fewer database operations

**Consequences:**
- `approve.ts`: Removed UPDATE statement after creating ST record
- `revoke.ts`: Removed check for other active STs and conditional UPDATE
- Both endpoints now rely entirely on relationship table state
- Certificate tests (48) all pass
