# Session Learnings - 2026-02-06

## 1. chart.js Requires Canvas + ResizeObserver Mocks in jsdom
**Topics:** testing, astro

**Context:** After replacing Recharts (SVG-based) with chart.js (canvas-based), the AdminAnalytics and CreatorAnalytics tests started failing with `HTMLCanvasElement.getContext() not implemented` and `ResizeObserver is not defined` errors.

**Learning:** jsdom doesn't implement `<canvas>` getContext or `ResizeObserver`. Recharts never needed these because it renders SVG DOM elements. chart.js renders to `<canvas>` and uses `ResizeObserver` for responsive sizing. Both must be mocked globally in `vitest.setup.ts` for any test that renders chart.js components without mocking them.

**Pattern:**
```typescript
// vitest.setup.ts — canvas mock
HTMLCanvasElement.prototype.getContext = function (this: HTMLCanvasElement) {
  return { canvas: this, fillRect: () => {}, clearRect: () => {}, /* ... */ } as unknown as RenderingContext;
};

// ResizeObserver mock
globalThis.ResizeObserver = class ResizeObserver {
  observe() {} unobserve() {} disconnect() {}
} as unknown as typeof globalThis.ResizeObserver;
```

## 2. chart.js v4 Removed `drawBorder` from Grid Options
**Topics:** astro

**Context:** TypeScript reported `drawBorder does not exist in type GridLineOptions` after adding `ChartOptions<'bar'>` type annotations.

**Learning:** In chart.js v4, the `grid.drawBorder` option was moved to `border.display` as a separate config object. Code migrated from examples or v3 docs will hit this. The fix is to use `border: { display: true/false, color: '...' }` alongside the `grid` config.

## 3. Vite Dependency Cache Stale After Library Swap
**Topics:** astro, deployment

**Context:** After removing `recharts` and adding `chart.js` + `react-chartjs-2`, the dev server returned a 505 error for `react-chartjs-2.js` from the `.vite/deps/` cache.

**Learning:** Vite's pre-bundled dependency cache (`node_modules/.vite/`) can become stale when dependencies are swapped. The fix is `rm -rf node_modules/.vite` and restart the dev server. This forces Vite to re-discover and re-bundle all dependencies. This only affects `npm run dev` — production builds (`npm run build`) don't use this cache.

## 4. chart.js Type Safety via `ChartOptions<T>` Annotation
**Topics:** astro, testing

**Context:** Initially wrote chart options as plain object literals with manually typed callback parameters. TypeScript couldn't match the callback signatures.

**Learning:** Annotating the options object with `ChartOptions<'bar'>` (or `'line'`, `'doughnut'`) lets TypeScript automatically infer all callback parameter types. This eliminates the need to manually type `context` parameters in tooltip/legend callbacks — just use `(context) => ...` and let the type flow from the annotation.

## 5. Clean Abstraction Layer Enabled Zero-Consumer-Change Library Swap
**Topics:** astro

**Context:** The original chart components were designed as an "abstraction layer for future migration to Tremor/Nivo" — all 9 consumer components imported from `@components/ui/charts`, never from `recharts` directly.

**Learning:** This abstraction paid off exactly as designed. Swapping Recharts for chart.js required changes to only 3 wrapper files + 1 new setup file. Zero consumer components needed modification. The interfaces (`TimeSeriesData`, `CategoryData`, `ChartConfig`) remained identical. This validates the pattern of wrapping third-party rendering libraries behind stable interfaces.
