# Session Learnings - 2026-02-01

## 1. /compact Is Context Compression, Not Session Restart
**Topics:** workflow

**Context:** After running `/compact` to reduce context usage, noticed that startup hooks (machine detection, resume state check, dev environment check) did not re-run.

**Learning:** The `/compact` command compresses conversation history within the same session - it's a context management operation, not a session restart. Startup hooks are tied to session initialization, so they only run when Claude Code is freshly invoked. The compacted summary preserves information from pre-compact (machine, session number, etc.) but the hooks themselves don't re-execute.

## 2. Documentation Skills Have Different Overlap Risks Post-Compact
**Topics:** workflow

**Context:** Concerned about running documentation skills (like `/q-learn-decide`) after `/compact`, since the summary includes information about learnings/decisions already documented.

**Learning:** Skills fall into two categories based on their data source:

| Risk Level | Skills | Why |
|------------|--------|-----|
| 🔴 High | `/q-learn-decide`, `/q-dump` | Extract from conversation - summary includes prior work |
| 🟢 Low | `/q-commit`, `/q-timecard`, `/q-git-history` | Based on git state, not conversation |

**Pattern:** After `/compact`, avoid re-running conversation-based documentation skills for work already documented. Git-based skills are safe to run anytime.

## 3. _src/pages/ Pattern Disables Routes in Astro
**Topics:** astro

**Context:** Wanted to start fresh with routing, keeping only `/` as an active route while preserving existing pages for reference.

**Learning:** Astro uses file-based routing where anything in `src/pages/` becomes a route. Moving files to `_src/pages/` (outside the standard directory) effectively disables all those routes. The underscore prefix is a convention for "archived/reference, not active" that Astro ignores entirely.

**Pattern:**
```bash
# Archive all pages except index
git mv src/pages/[file-or-dir] _src/pages/

# Result: Only src/pages/index.astro is a route (/)
# _src/pages/* preserved but not routed
```

## 4. API Routes Are Part of src/pages/api/
**Topics:** astro

**Context:** After moving pages to `_src/pages/`, TypeScript reported ~174 errors from test files.

**Learning:** In Astro, API routes live in `src/pages/api/`, so they're affected when moving the entire pages directory. When restoring a page feature, you may also need to restore its corresponding API routes. Test failures will indicate which API endpoints a page depends on.
