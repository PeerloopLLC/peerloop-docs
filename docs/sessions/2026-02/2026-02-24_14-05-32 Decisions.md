# Session Decisions - 2026-02-24

## 1. Add 'no_show' to Sessions Table CHECK Constraint
**Type:** Implementation
**Topics:** d1, video

**Trigger:** Discovered that the `no_show` status added to st-sessions.ts in session 278 was silently failing because the sessions table CHECK constraint didn't include it.

**Options Considered:**
1. Add `'no_show'` to the sessions CHECK constraint ← Chosen
2. Remove the no-show auto-detection feature from st-sessions.ts

**Decision:** Extended the CHECK constraint from `('scheduled', 'in_progress', 'completed', 'cancelled')` to include `'no_show'`.

**Rationale:** The no-show feature is valuable for S-T session history and flagging unreliable participants. Removing the feature would lose functionality that was deliberately added. The schema should match the code's intent.

**Consequences:** `migrations/0001_schema.sql` line 648 updated. All 5267 tests pass with the change.

**Type:** Strategic
**Topics:** docs-infra, video
