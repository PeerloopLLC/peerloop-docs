# Session Decisions - 2026-02-06

## 1. Defer Stripe Production Secrets Until MVP Go-Live
**Type:** Strategic
**Topics:** stripe, deployment

**Trigger:** While reviewing which secrets to set in the Cloudflare Production environment, considered whether to add live Stripe keys now or wait.

**Options Considered:**
1. Add all production secrets now, including Stripe live keys
2. Defer Stripe production secrets until MVP go-live ← Chosen
3. Use test keys in production temporarily

**Decision:** Do NOT add `STRIPE_SECRET_KEY` or `STRIPE_WEBHOOK_SECRET` to the Cloudflare Production environment until MVP go-live. Live values are stored in `.secrets.cloudflare` for reference but should not be entered into the CF Dashboard yet.

**Rationale:** With no Stripe secrets in production, even if someone hits a payment endpoint on the production URL, it will fail gracefully (missing key) rather than charge a real card. This creates a clear launch gate — adding live Stripe secrets becomes an explicit go-live checklist item rather than something that's been sitting there untested.

**Consequences:**
- Added DECISION comment block in `.secrets.cloudflare` (production Stripe section)
- Updated `docs/tech/tech-026-env-vars-secrets.md` deployment table: Stripe production values show "Deferred until go-live" with a callout block explaining the rationale
- Adding live Stripe secrets is now a formal go-live checklist item
