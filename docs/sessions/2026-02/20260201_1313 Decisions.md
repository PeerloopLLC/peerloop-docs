## 2. Archive Pages to _src/pages/ Instead of Deleting
**Type:** Implementation
**Topics:** astro, workflow

**Trigger:** Wanted to reset routing to a single "/" route while preserving existing pages for reference during cherry-picking.

**Options Considered:**
1. Delete all pages except index.astro ← Destructive, loses reference
2. Move to _src/pages/ for archival ← Chosen
3. Create separate branch with old pages ← More complex git workflow

**Decision:** Move all pages (except `index.astro`) to `_src/pages/` using `git mv`.

**Rationale:**
- Preserves all existing code for reference during cherry-picking
- `git mv` maintains file history for traceability
- Underscore prefix convention clearly signals "archived, not active"
- Astro ignores anything outside `src/pages/`

**Consequences:**
- Only `/` route is active now
- ~174 TypeScript errors from tests importing `@/pages/api/*`
- Can selectively restore pages by moving back from `_src/pages/`
- API routes also archived (must restore with their pages)
