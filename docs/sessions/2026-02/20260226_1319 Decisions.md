# Session Decisions - 2026-02-26

## 1. Inline Auth/DB Setup Rather Than Helper Abstraction
**Type:** Implementation
**Topics:** astro

**Trigger:** Three endpoint handlers (POST, GET, PATCH) in expectations.ts shared identical auth+DB boilerplate. Attempted to extract a shared helper.

**Options Considered:**
1. Extract `getAuthAndDB()` helper with discriminated union return type
2. Inline the boilerplate in each handler ← Chosen
3. Use middleware-style wrapper function

**Decision:** Inline the standard auth+DB setup directly in each handler, matching the pattern used by review.ts and course-review.ts.

**Rationale:** The helper caused TypeScript inference errors with Astro's APIRoute type. Inlining is consistent with every other endpoint in the codebase and only adds ~8 lines per handler. The duplication is acceptable boilerplate, not logic worth abstracting.

**Consequences:** Each handler has identical setup blocks. If auth patterns change, each handler updates independently (same as all other endpoints).
