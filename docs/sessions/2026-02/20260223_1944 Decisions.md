# Session Decisions - 2026-02-23

## 1. Members Endpoint as File-Level Route, Not Directory
**Type:** Implementation
**Topics:** astro

**Trigger:** Need to create the members list endpoint under `/api/me/communities/[slug]/`. The progressions endpoint uses a directory pattern (`progressions/index.ts`, `progressions/[id].ts`, `progressions/reorder.ts`) because it has GET, POST, PATCH, DELETE, and reorder operations.

**Options Considered:**
1. Create `members.ts` as a single file with GET handler ← Chosen
2. Create `members/index.ts` directory pattern (matching progressions)

**Decision:** Members endpoint is a single `members.ts` file, not a directory.

**Rationale:** The members endpoint only needs GET (list members for the creator view). No POST/PATCH/DELETE — member creation happens via the existing `/api/communities/[slug]/join` public endpoint, and role management isn't in scope for this block. A directory structure would be premature; it can be promoted to a directory later if member management endpoints are added.

**Consequences:** Clean single-file endpoint. If member role editing is needed later, the file becomes `members/index.ts` and a `members/[id].ts` is added.

## 2. Members Section Placed Between Settings and Progressions
**Type:** Implementation
**Topics:** astro

**Trigger:** CommunityManagement.tsx has three sections: Header, Settings (collapsible), and Progressions. Where should the new Members section go?

**Options Considered:**
1. Between Settings and Progressions ← Chosen
2. After Progressions (at bottom)
3. Collapsible inside the header area

**Decision:** Members section sits between the collapsible Settings panel and the Progressions section.

**Rationale:** Groups community metadata together (settings = configuration, members = who's in it) above content management (progressions = what's taught). Members is a read-only informational section that creators glance at, while progressions is the primary workspace — keeping progressions last gives it the most visual weight and avoids pushing it down the page.

**Consequences:** Page layout flows: Header → Settings (toggle) → Members → Progressions → Create Modal.
