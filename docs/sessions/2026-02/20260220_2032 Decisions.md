# Session Decisions - 2026-02-20

## 1. Full Prune of MBA-2017 Test Artifacts
**Type:** Implementation
**Topics:** testing, dev-environment

**Trigger:** During docs-primary migration testing, reviewed test helpers for MBA-2017 remnants and discovered massive dead code in the barrel file.

**Options Considered:**
1. Trim barrel only — remove dead exports from index.ts but keep source functions intact
2. Full prune — remove dead exports AND delete unused functions from source files ← Chosen
3. Full prune + delete mock-db.ts — everything above plus delete the entire mock-db.ts file ← Also Chosen

**Decision:** Delete `mock-db.ts` entirely, remove all deprecated MBA-2017 skip helpers from `skip.ts`, remove deprecated functions from `machine.ts`, and slim the barrel file from 75 to 16 exports.

**Rationale:** All MBA-2017 artifacts had zero consumers. The mock-db module was a machine-capability workaround that became invisible dead code when both machines gained full D1 support. Keeping deprecated-but-present code creates confusion about what's actively used. Since both dev machines now have identical capabilities, the entire skip/mock infrastructure for the retired machine serves no purpose.

**Consequences:** Removed ~240 lines of dead code. Barrel file slimmed from 75 to 16 exports. `vitest.setup.ts` required a fix (`getMachineCapabilities` → `getMachineName`). All 4891 tests pass.
