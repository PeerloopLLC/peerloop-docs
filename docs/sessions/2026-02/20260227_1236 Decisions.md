# Session Decisions - 2026-02-27

## 1. Simplify Signup E2E Tests to Form Rendering Only
**Type:** Implementation
**Topics:** testing

**Trigger:** Signup form validation tests failed because `role="alert"` is only used for API errors, not field-level validation. The `form` locator also didn't work reliably inside the dialog modal.

**Options Considered:**
1. Debug and fix validation E2E tests with correct selectors
2. Simplify to form rendering + navigation tests only ← Chosen
3. Skip signup tests entirely

**Decision:** Signup E2E tests verify form rendering (fields visible, button present, Terms/Privacy links) and sidebar-triggered modal opening. Validation logic is left to unit tests.

**Rationale:** E2E tests should focus on "does the page load and show the right things" not "does client-side validation work correctly." Validation is a unit test concern. The form rendering tests provide regression protection without fragile DOM assertions.

**Consequences:** 4 focused signup tests instead of 3-4 fragile validation tests. More maintainable and faster.

## 2. Scope E2E Selectors to `main` for AppLayout Pages
**Type:** Implementation
**Topics:** testing, astro

**Trigger:** Multiple test failures caused by sidebar panels ("My Feeds", "Discover") containing duplicate headings, links, and text that matched before the main content elements.

**Options Considered:**
1. Use more specific text/role matchers per test ← Chosen (combined with scoping)
2. Always scope every selector to `page.locator('main')` ← Chosen (for discovery pages)
3. Disable sidebar panels in test mode

**Decision:** For pages using AppLayout where sidebar panels may duplicate elements (especially discovery pages), scope selectors to `page.locator('main')`. For other pages, use `exact: true`, `.first()`, or navigation-scoped selectors as needed.

**Rationale:** The sidebar panels are part of the real UI and shouldn't be disabled. Scoping to `main` is the cleanest way to avoid ambiguity without changing application code.

**Consequences:** Established a pattern for all future E2E tests to follow. Discovery tests use `main` scoping consistently.
