# Session Learnings - 2026-02-20

## 1. Claude Code Hooks Run in a Minimal Shell Environment
**Topics:** workflow

**Context:** The `scripts/check-env.sh` SessionStart hook reported "startup hook error" because tools managed by nvm and Homebrew weren't on PATH.

**Learning:** Claude Code hooks execute in a minimal shell that does NOT source `~/.zshrc` or `~/.bash_profile`. Tools managed by nvm, Homebrew (`/opt/homebrew/bin`), or other profile-dependent managers are invisible. Hook scripts must explicitly source dependencies.

**Pattern:**
```bash
export NVM_DIR="${NVM_DIR:-$HOME/.nvm}"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
```

## 2. Hook Exit Codes Control Startup Error Display
**Topics:** workflow

**Context:** `check-env.sh` exited with code 1 when tools were missing, causing Claude Code to show "SessionStart:startup hook error."

**Learning:** Any non-zero exit from a SessionStart hook is treated as an error by Claude Code. Informational/diagnostic scripts should always exit 0. Reserve non-zero for hooks that should genuinely block the session.

## 3. nvm Sourcing Fails in Bash Tool's zsh Context
**Topics:** workflow

**Context:** Attempted to install Node via the Bash tool, but `source "$NVM_DIR/nvm.sh"` failed silently in zsh.

**Learning:** The Bash tool runs in zsh by default. nvm sourcing can fail silently in this context. Wrapping commands in `bash -c 'export NVM_DIR=...; source .../nvm.sh; <command>'` is a reliable workaround.

---

## 4. Homebrew and Stripe CLI Require Interactive Terminal
**Topics:** workflow

**Context:** Tried to install Homebrew via Claude Code's Bash tool — failed because it requires interactive sudo with TTY access. Same for `stripe login`.

**Learning:** Any installer requiring sudo password prompts or browser-based auth flows (Homebrew install, `stripe login`) cannot be done through Claude Code's Bash tool. Must be run directly in the user's terminal.

---

## 5. Stripe CLI Login Pairing Code Appears in Terminal
**Topics:** stripe, workflow

**Context:** User ran `stripe login` but couldn't find the verification words to match against the browser.

**Learning:** The Stripe CLI login flow displays a pairing code (series of words) in the terminal output, a few lines above the "waiting" message. The browser asks to confirm these same words. Easy to miss if you switch to the browser quickly.

## 6. Cursor Intercepts File Drag-and-Drop — Shift+Drag Workaround
**Topics:** workflow

**Context:** Dragging an image file into Cursor's terminal opened it in an editor tab instead of pasting the file path.

**Learning:** Cursor's default behavior opens dragged files as editor tabs. Holding **Shift while dragging** forces the file path to be pasted as text in the terminal instead. The setting `terminal.integrated.enableDragAndDrop` may also control this behavior.

---

## 7. macOS /tmp/ Lives at /private/tmp/ (Not Under Home Directory)
**Topics:** workflow

**Context:** User needed to find a file written to `/tmp/routes.tsv` using Finder.

**Learning:** On macOS, `/tmp` is a symlink to `/private/tmp/`. It is NOT under the home directory. In Finder, use **Go > Go to Folder** (`Cmd+Shift+G`) and type `/tmp/` to navigate there. For easier access, copy files to `~/Desktop/` instead.

---

## 8. Astro File-Based Routing Gives Complete Route Inventory
**Topics:** astro

**Context:** Needed to export all routes for a spreadsheet. Scanned `src/pages/` directory.

**Learning:** In Astro.js, the `src/pages/` directory IS the route map — every file maps to a URL path. This makes extracting a complete route inventory straightforward: glob for all `.astro` and `.ts` files under `src/pages/`, convert paths to routes using Astro's naming conventions (`index.astro` → `/`, `[slug].astro` → `/[slug]`). Peerloop has 295 routes (54 pages, 241 API endpoints).
