# Session 169 Decisions - 2026-02-03

## D1: URL Routing Scheme (tech-021)

**Decision:** Adopt "Bare = My" routing convention

| Pattern | Meaning | Example |
|---------|---------|---------|
| `/thing` | My personal content | `/courses` = my enrolled courses |
| `/discover/thing` | Site-wide browse | `/discover/courses` = catalog |
| `/@handle` | Universal user profile | `/@jane` |
| `/settings/*` | Account configuration | `/settings/profile` |

**Rationale:** Users spend most time with their own content. Default should be personal; discovery is explicit.

**Documentation:** `docs/tech/tech-021-url-routing.md`

## D2: Content Hierarchy (CD-036)

**Decision:** Community → Progression → Course hierarchy

- Every course must belong to a progression
- Every progression must belong to a community
- Progressions are the primary discovery unit
- Course can only be in ONE progression (copy to reuse)
- Progression order is advisory, not enforced

**Display:**
- 1 course = "Stand-alone" badge
- 2+ courses = "Learning Path" badge

**Documentation:** `RFC/CD-036/CD-036.md`

## D3: Feed Scoping

**Decision:** Four feed types with distinct posting permissions

| Feed | Scope | Who Can Post |
|------|-------|--------------|
| The Commons | Platform-wide | Anyone |
| Community | Per community | Members only |
| Course | Per course | Enrolled + instructors |
| Personal | Per user | That user |

Community membership is derived from enrollments (no separate table).
