### OAuth Credentials Explanation

**User:** Asked why Google Client ID and Secret are needed — whose credentials are they?

**Claude:** Explained that OAuth Client ID/Secret are app-level credentials that identify Peerloop as an application to Google. Walked through the full OAuth flow: user clicks "Sign in with Google" → redirected to Google → logs in with their account → Google redirects back with a code → Peerloop exchanges code + Client Secret for user profile info. The same pattern applies to GitHub.

**User:** Asked for more detail — are these apps in Google that authorize other people using their Google OAuth?

**Claude:** Provided a detailed step-by-step explanation of the OAuth 2.0 flow, clarifying that the Client ID is semi-public (visible in redirect URLs) while the Client Secret is truly secret (server-side only). Each provider requires its own app registration (Google Cloud Console, GitHub Developer Settings).
