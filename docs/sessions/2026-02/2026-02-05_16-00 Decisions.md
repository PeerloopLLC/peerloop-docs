## 1. Create Dedicated OAuth Provider Tech Doc (tech-025)
**Type:** Implementation
**Topics:** auth, workflow

**Trigger:** Secrets audit revealed Google/GitHub OAuth credentials missing from all environments, and existing auth doc (tech-010) only covered the library choices, not the provider registration process.

**Options Considered:**
1. Add OAuth setup instructions to existing `tech-010-auth-libraries.md`
2. Create a new `tech-025-google-oauth.md` dedicated to provider setup ← Chosen
3. Document only in PLAN.md as a checklist

**Decision:** Created `docs/tech/tech-025-google-oauth.md` covering both Google and GitHub OAuth app registration, with step-by-step setup instructions written for non-developers (the client).

**Rationale:** tech-010 is about *library selection rationale* (jose, bcryptjs, arctic). OAuth provider setup is a separate concern — it's an operational task about registering the app with Google/GitHub, not a coding decision. A dedicated doc keeps concerns separated and makes it easy for the client to follow without wading through code examples.

**Consequences:** New tech doc created. GitHub OAuth included as a companion section in the same doc rather than a separate tech-026, since the setup pattern is nearly identical.
