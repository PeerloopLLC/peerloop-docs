## 1. PAGES-MAP.md Obsolete After Twitter UI Pivot
**Topics:** astro, workflow

**Context:** Attempted to update PAGES-MAP.md with new community pages from CD-036.

**Learning:** PAGES-MAP.md and its JSON page specs (`src/data/pages/**/*.json`) represent pre-Twitter-UI architecture. After the client's pivot to Twitter-like UX, `docs/tech/tech-021-url-routing.md` became the source of truth for routes. The old system was archived as `ORIG-PAGES-MAP.md` rather than updated.

**Pattern:** When architecture pivots make documentation obsolete, archive with clear naming (`ORIG-*`) rather than deleting—preserves historical context.

## 2. Mock Data in UI Components Requires API Wiring
**Topics:** astro, workflow

**Context:** FeedSlidePanel showed hardcoded mock communities (web-dev, python, design) instead of real seeded data.

**Learning:** During development, it's common to build UI components with mock data first, then wire to APIs later. The FeedSlidePanel had `const myCommunities = [...]` hardcoded arrays that needed replacement with `fetch('/api/communities')` calls.

**Pattern:** When testing UI against seed data, check whether components fetch from APIs or use inline mocks:
```typescript
// Mock pattern (pre-wiring)
const data = [{ id: 'mock-1', ... }];

// API pattern (production-ready)
const response = await fetch('/api/endpoint');
const data = await response.json();
```

## 3. Tab-as-Route Pattern for Bookmarkable Tabs
**Topics:** astro, workflow

**Context:** Discussed whether course/community feed tabs should be bookmarkable.

**Learning:** The GitHub/Notion pattern solves this elegantly: tabs ARE routes. Sub-routes render the same page with different active tabs. This provides both discoverability (tabs) and bookmarkability (URLs).

**Pattern:**
```
/course/[slug]           → About tab (default)
/course/[slug]/feed      → Feed tab active
/course/[slug]/curriculum → Curriculum tab active
```

URL changes via `pushState`, tab clicks update URL, back/forward navigation works. Same page component, different `initialTab` prop.

## 4. Communities Are Flexible Containers
**Topics:** d1, astro

**Context:** Discussing whether creator feeds should exist separately from communities.

**Learning:** Communities in PeerLoop are flexible containers that may or may not have courses. A creator can have:
- Course-based communities (n8n Automation, Boating Life)
- Announcement-only community ("Main Hall" with no courses, just feed + resources)

This eliminates need for separate "creator feeds"—creators just create a community for their announcements.

## 5. Discover Page Shows Unjoinable Communities
**Topics:** astro

**Context:** User confused why Discover showed different communities than My Feeds.

**Learning:** The `filter=discover` API parameter returns communities the user has NOT joined. This is intentional—Discover is for finding new communities to join, not viewing existing memberships. Clear labeling helps: changed header to "Communities to Join" with subtitle "Public communities you haven't joined yet."
