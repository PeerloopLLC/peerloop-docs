# Session Decisions - 2026-02-03

## 1. Use `/discover/*` Routes for New App Shell
**Type:** Strategic
**Topics:** astro, workflow

**Trigger:** DiscoverSlidePanel had links to `/discover/*` routes but PAGES-MAP.md showed bare routes (`/teachers`, `/creators`).

**Options Considered:**
1. Change DiscoverSlidePanel to use bare routes from PAGES-MAP.md
2. Keep `/discover/*` routes as part of new routing architecture ← Chosen

**Decision:** Keep the `/discover/*` route pattern for Discover panel destinations. PAGES-MAP.md is a historical reference, not the source of truth for the new architecture.

**Rationale:** The new app shell uses AppLayout with persistent sidebar. The `/discover/*` namespace provides clear organization for public browsing pages within the app context.

**Consequences:** Created pages at `/discover/creators`, `/discover/teachers`, `/discover/students`, `/discover/leaderboard`, `/discover/courses`.

## 2. Create `/L-convert-page` Skill for Page Migration
**Type:** Implementation
**Topics:** workflow, astro

**Trigger:** Need to convert multiple pages from `_src/pages` (LandingLayout) to new `src/pages` (AppLayout) locations.

**Options Considered:**
1. Manually convert each page ad-hoc
2. Create a reusable skill to systematize the process ← Chosen

**Decision:** Created `/L-convert-page <pageCode> <route>` skill that automates the conversion process.

**Rationale:** Multiple pages need conversion with a consistent pattern. A skill ensures consistency and documents the process.

**Consequences:**
- Skill created at `.claude/commands/L-convert-page.md`
- Successfully used to convert CRLS, STDR, LEAD, CBRO pages
- Can be reused for future page migrations

## 3. Leave `_src` Files Historically Accurate
**Type:** Strategic
**Topics:** workflow

**Trigger:** Found bug in `_src/pages/creators/index.astro` (used `is_creator` instead of `can_create_courses`).

**Options Considered:**
1. Fix bugs in `_src` files when discovered
2. Leave `_src` files untouched as historical reference ← Chosen

**Decision:** Do not modify files in `_src/pages/`. They serve as historical reference of the original implementation, including any bugs.

**Rationale:** The `_src` folder preserves the state of the codebase before the new routing architecture. Modifying it would lose that historical context and potentially mask bugs that need fixing in the new implementations.

**Consequences:** Fixed the bug only in the new `src/pages/discover/creators/index.astro`. Original remains unchanged.

## 4. Add Students to Discover Panel
**Type:** Implementation
**Topics:** astro

**Trigger:** DiscoverSlidePanel had Creators and Teachers but no Students entry.

**Options Considered:**
1. Don't add Students (rely on Leaderboard for peer visibility)
2. Add Students as a new Discover option ← Chosen

**Decision:** Added "Students" menu item to DiscoverSlidePanel linking to `/discover/students`.

**Rationale:** Provides symmetry with other user listing pages. Allows learners to discover fellow students and peers in the community.

**Consequences:**
- Added Students item to DiscoverSlidePanel (between Teachers and Creators)
- Created `/discover/students` page with StudentDirectory component
- Used `LearningIcon` for visual consistency
