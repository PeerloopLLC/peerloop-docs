# Session Learnings - 2026-02-05

## 1. Stream Timeline Aggregation via Follow Relationships
**Topics:** stream, architecture

**Context:** Implementing an aggregated `/feed` page that shows posts from all user's communities and courses.

**Learning:** Stream.io's `timeline:{userId}` feed doesn't automatically show content. It only displays posts from feeds the user explicitly **follows**. The aggregation is built on follow relationships that must be established at specific events:
- User enrolls in course → `timeline:userId` follows `course:courseId`
- User joins community → `timeline:userId` follows `community:slug`
- User registers → `timeline:userId` follows `community:the-commons`

**Pattern:**
```typescript
// In stream.ts - follow method
async follow(targetFeedGroup: string, targetFeedId: string, activityCopyLimit = 0): Promise<void> {
  const endpoint = `/feed/${this.feedGroup}/${this.feedId}/follows/`;
  await streamFetch(this.config, endpoint, {
    method: 'POST',
    body: JSON.stringify({
      target: `${targetFeedGroup}:${targetFeedId}`,
      activity_copy_limit: activityCopyLimit,
    }),
  });
}
```

## 2. Routing Symmetry vs User Need - Recognizing Over-Engineering
**Topics:** astro, architecture

**Context:** Reviewing the planned `/teachers` and `/certificates` routes which were marked as TODO in the routing doc.

**Learning:** Just because a route fits a pattern (e.g., "bare = my" convention) doesn't mean it's needed. Both routes were over-engineering:
- `/teachers` - Users navigate course-first (`/course/[slug]/teachers`), not teacher-first
- `/certificates` - Already displayed on `/learning` dashboard

The key question to ask: "Where does the user naturally go to find this?" If the answer is "through another page they're already visiting," a dedicated route adds navigation surface without value.

**Pattern:** Before adding a route for routing symmetry, ask:
1. Is this a primary navigation target or secondary?
2. Is this data already accessible through the user's natural flow?
3. Would users bookmark/share this URL specifically?

## 3. Stream REST API Follow Endpoint Format
**Topics:** stream

**Context:** Implementing follow/unfollow methods in the edge-compatible Stream client.

**Learning:** Stream's REST API for follows uses a specific endpoint structure:
- **Follow:** `POST /feed/{group}/{id}/follows/` with body `{ target: "targetGroup:targetId" }`
- **Unfollow:** `DELETE /feed/{group}/{id}/follows/{targetGroup}:{targetId}/`
- **List following:** `GET /feed/{group}/{id}/follows/`

The target format uses colon separator: `community:the-commons`, not slash.
