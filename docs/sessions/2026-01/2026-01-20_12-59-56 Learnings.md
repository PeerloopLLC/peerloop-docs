# Session Learnings - 2026-01-20

## 1. Stream REST API Requires Path-Based Endpoints for Reaction Filtering
**Topics:** stream

**Context:** When implementing comments for the community feed, the comments API was returning all comments from all activities instead of filtering by the specific activity ID.

**Learning:** Stream.io's REST API does NOT support filtering reactions via query parameters like `/reaction/?activity_id=xxx`. Instead, it requires path-based endpoints:
- By activity: `/reaction/activity_id/{activity_id}/{kind}/`
- By reaction (children): `/reaction/reaction_id/{reaction_id}/{kind}/`
- By user: `/reaction/user_id/{user_id}/{kind}/`

The query parameter approach silently returns all reactions without filtering.

**Pattern:**
```typescript
// WRONG - returns ALL reactions, ignores activity_id filter
const endpoint = `/reaction/?activity_id=${activityId}&kind=comment`;

// CORRECT - properly filters by activity
const endpoint = `/reaction/activity_id/${activityId}/comment/`;
```

## 2. Stream Enriched Feed Endpoint Required for Reaction Counts
**Topics:** stream

**Context:** The community feed wasn't showing comment counts on the activity cards. The `reaction_counts` field was undefined in the feed response.

**Learning:** Stream.io has two feed endpoint variants:
- `/feed/{group}/{id}/` - Basic feed without enrichment
- `/enrich/feed/{group}/{id}/` - Enriched feed with reaction_counts, own_reactions, etc.

Additionally, the parameter format for requesting counts is `with_reaction_counts=true` (snake_case with `with_` prefix), NOT `reactions[counts]=true`.

**Pattern:**
```typescript
// Use /enrich/feed/ endpoint with snake_case params
const endpoint = `/enrich/feed/${feedGroup}/${feedId}/?with_reaction_counts=true&with_own_reactions=true`;
```

## 3. Stream reaction_counts Only Counts Top-Level Reactions
**Topics:** stream

**Context:** After getting reaction counts working, the count showed "1" even though there were multiple nested replies.

**Learning:** Stream's `reaction_counts.comment` only counts top-level reactions on an activity, not child reactions (replies). Nested replies have their own `children_counts` on each reaction. This is intentional and matches patterns on platforms like YouTube and Facebook where the shown count is top-level comments only.
