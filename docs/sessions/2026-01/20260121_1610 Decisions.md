# Session Decisions - 2026-01-21

## 1. Remove Orphaned process.env Polyfill
**Type:** Implementation
**Topics:** astro, cloudflare

**Trigger:** Previous session identified that `'process.env': '{}'` in `astro.config.mjs` was breaking MBA-2017 D1 REST fallback. Needed to investigate if it was still required.

**Options Considered:**
1. Keep the polyfill and find workarounds for MBA-2017 ← Previous approach (reverted)
2. Remove the polyfill and test if Stream/Resend still work ← Chosen
3. Create conditional polyfill based on environment

**Decision:** Remove `'process.env': '{}'` from Vite define config entirely.

**Rationale:**
- The polyfill was added for the `getstream` npm SDK
- The SDK was removed the same day (Jan 19) and replaced with a custom REST client
- The polyfill was never cleaned up - orphaned code
- Testing confirmed Stream and Resend both work without it

**Consequences:**
- MBA-2017 D1 REST fallback now works (process.env accessible at runtime)
- Removed 1 line from astro.config.mjs
- `global: 'globalThis'` polyfill retained (may still be needed)

## 2. Update Stream Config to Use getEnv Helper
**Type:** Implementation
**Topics:** stream, cloudflare, d1

**Trigger:** After removing the process.env polyfill, Stream credentials still weren't loading on MBA-2017 because `getStreamConfig()` directly accessed `locals.runtime.env`.

**Options Considered:**
1. Add explicit MBA-2017 check in getStreamConfig ← Rejected (duplicate logic)
2. Use existing getEnv helper which already has fallback logic ← Chosen
3. Create Stream-specific env loader

**Decision:** Update `getStreamConfig()` to use `getEnv()` from `src/lib/env.ts`.

**Rationale:**
- `getEnv` already has the proper fallback logic
- Only falls back to `process.env` when `ENABLE_D1_REST_FALLBACK=true` AND not production
- Maintains consistent pattern across codebase
- No behavior change for MacMini/Production (they use `locals.runtime.env` first)

**Consequences:**
- Stream now works on MBA-2017 with Node adapter
- Consistent env access pattern across codebase
- Added import for `getEnv` in `src/lib/stream.ts`
