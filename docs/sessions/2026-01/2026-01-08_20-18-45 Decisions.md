# Session Decisions - 2026-01-08

## D1: Use Accent Colors for BTAT Page

**Context:** BTAT (Become a Teacher) needed visual differentiation from other marketing pages.

**Decision:** Use `accent-*` color scheme (teal/green) instead of `primary-*` (blue).

**Rationale:**
- Creates visual distinction between creator-focused (FCRE) and teacher-focused (BTAT) pages
- Aligns with the different target audiences
- Accent colors reserved for teacher/S-T related content

## D2: Static Data for Marketing Pages

**Context:** JSON specs specified API calls for testimonials and platform stats.

**Decision:** Use hardcoded static data instead of API calls for MVP.

**Rationale:**
- Marketing pages don't need real-time data
- Reduces complexity for initial launch
- Stats can be updated periodically via code changes
- Dynamic APIs can be added later when needed

**Documented in:** `_discrepancies.implementedDifferently` in JSON specs

## D3: Fix Page Code Inconsistency

**Context:** BTAT page had mismatched codes - JSON spec had `BCMT`, PAGES-MAP had `BTAT`.

**Decision:** Use `BTAT` as canonical code.

**Rationale:**
- PAGES-MAP.md is the single source of truth for page codes
- `BTAT` is more intuitive (Become A Teacher)
- Updated JSON spec to match

## D4: Simplified State Variations

**Context:** BTAT spec planned for 3 state variations (Visitor, Enrolled Student, Course Completed).

**Decision:** Implement single static view for all visitors.

**Rationale:**
- Personalization requires auth integration
- Static page serves MVP needs
- Can add personalization later
- Documented as `_discrepancies.implementedDifferently`
