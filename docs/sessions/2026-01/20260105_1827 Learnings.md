# Session Learnings - 2026-01-05

## Technical

### Block 3.5 Homework System Implementation

1. **API Route Parameter Naming Conflicts**
   - Issue: Variable named `params` in query building code shadowed the `params` argument from Astro's APIRoute
   - Fix: Rename local variable to `queryParams` to avoid collision
   - Prevention: Be mindful of Astro's destructured route parameters when naming local variables

2. **TypeScript Strict Mode with fetch() Responses**
   - Issue: `await response.json()` returns `unknown` type, causing errors when accessing properties
   - Fix: Define response interfaces and type-cast: `const data: MyResponse = await response.json()`
   - Pattern: Create specific interfaces for each API response shape

3. **React State with undefined Values**
   - Issue: `Record<string, T | null>` doesn't allow `undefined` values when clearing cache
   - Fix: Change type to `Record<string, T | null | undefined>` to allow cache invalidation via `undefined`

### R2 Storage Integration

1. **Wrangler R2 CLI Limitations**
   - `wrangler r2 object put` requires `--file` flag with actual file path
   - Stdin redirection (`<<<`) doesn't work for `--file=-`
   - Workaround: Write to temp file first, then upload, then cleanup

2. **R2 Health Check Pattern**
   - Created `/api/health/r2` endpoint that tests write/read/delete cycle
   - Good practice: Health endpoints should test full lifecycle, not just connectivity
   - Cleanup: Always delete test files after verification

## Architecture

### Tab Navigation Pattern in React

1. **CourseLearning Tab Structure**
   - Added `activeTab` state with type union: `'modules' | 'homework' | 'resources'`
   - Tab buttons in header area, content rendered conditionally below
   - Each tab component is self-contained with its own data fetching

2. **Homework Submission Workflow**
   - Statuses: `submitted` → `reviewed` or `resubmit_requested`
   - One submission per student per assignment (UNIQUE constraint)
   - Resubmission updates existing record rather than creating new

### API Endpoint Organization

```
/api/courses/[id]/homework     - List assignments for course
/api/homework/[id]             - Single assignment details
/api/homework/[id]/submit      - Submit assignment (POST)
/api/homework/[id]/submissions - List all submissions (ST/Creator)
/api/homework/[id]/submissions/me - My submission
/api/homework/[id]/submissions/[subId] - Grade submission (PATCH)
/api/submissions/[id]          - Update my submission (PUT)
```

Nesting pattern: Resource-centric with sub-resources for related operations.
