# Session Learnings - 2026-01-09

## Technical

### Timestamp Comparison for Dependency Sync Checks

**Problem:** The `check-env.sh` script was comparing `package-lock.json` timestamp against `node_modules` to detect out-of-sync dependencies. This caused false positives because `npm install` updates `package-lock.json` at the same moment (or after) `node_modules`.

**Insight:** `package-lock.json` is auto-updated by npm during install, making it a poor signal for "needs install." `package.json` only changes when a human modifies dependencies.

**Solution:** Compare `package.json` instead of `package-lock.json`:
```bash
if [ "$PROJECT_ROOT/package.json" -nt "$PROJECT_ROOT/node_modules" ]; then
```

**Why this works:**
- `package.json` → changes only when human adds/removes/updates dependencies
- `package-lock.json` → auto-updated by npm, causes false positives
- Warning now triggers only when there's a real dependency change requiring `npm install`

## Workflow

### Adding Scripts to Claude Code SessionStart Hooks

Scripts can be added to `.claude/settings.json` under `hooks.SessionStart` to run automatically at session start:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "$CLAUDE_PROJECT_DIR/scripts/check-env.sh"
          }
        ]
      }
    ]
  }
}
```

The `$CLAUDE_PROJECT_DIR` variable resolves to the project root.
