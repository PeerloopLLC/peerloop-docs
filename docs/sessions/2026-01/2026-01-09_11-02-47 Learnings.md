## Development Environment Safeguards

### Package Version Drift Prevention
When working across multiple development machines with different capabilities:

1. **`.nvmrc` + `engines` in package.json** - Pin Node.js version exactly (not semver range) to ensure consistency
2. **`package-lock.json`** - Always commit this file; `npm install` respects exact versions from lock file
3. **`npm install` vs `npm update`**:
   - `npm install` - Installs exact versions from lock file
   - `npm update` - Updates within semver range AND rewrites lock file

### Handling Machine-Specific Tool Versions
When a tool (like wrangler) can't run the same version on all machines due to OS constraints:

- **Remove from package.json** - Don't include in devDependencies
- **Use global installs** - Each machine installs its own compatible version
- **No lock file conflicts** - Avoids package-lock.json drift between machines

### Bash Arithmetic with `set -e`
The pattern `((VARIABLE++))` returns exit code 1 when VARIABLE is 0 (before incrementing), which triggers `set -e` to exit the script. Use `VARIABLE=$((VARIABLE + 1))` instead for safe arithmetic in scripts with `set -e`.

### Environment File Patterns
- **`.dev.vars`** - Wrangler/Cloudflare convention for local environment variables
- **`.env`** - dotenv package convention
- **Symlink solution** - `ln -s .dev.vars .env` allows both tools to use same file

### Detecting Stale node_modules
Compare timestamps: if `package-lock.json` is newer than `node_modules`, dependencies need syncing. This catches the scenario after `git pull` when lock file was updated by another machine.

## macOS/Finder Notes

Symlinks in Finder show a small curved arrow badge (similar to aliases). More reliable to use `ls -la` in terminal to identify symlinks.
