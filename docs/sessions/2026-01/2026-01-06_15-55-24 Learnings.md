## Git Branch Management

### Cherry-Pick with Conflicts
When cherry-picking commits between branches that have diverged:
- Expect conflicts in files modified by both branches
- Resolve by keeping content from both sides where appropriate
- Use `git stash` to preserve uncommitted work before branch operations

### Stash with Untracked Files
- Use `git stash push -u` to include untracked files in the stash
- This is essential when moving new files between branches

## Documentation Consolidation

### Single Source of Truth Pattern
When multiple files track similar information:
1. Identify the superset file (contains all info from others)
2. Consolidate into one authoritative file
3. Slim down references in other files to point to the source
4. Delete redundant files
