# Session Learnings - 2026-01-05_20-09-05

## Technical

### Block 10: Page Feature Backlog Pattern
- Created a new Block type that serves as a **living backlog** rather than a scheduled implementation block
- Structure: Pages grouped by category with per-page feature checklists
- Status indicators: ✅ (implemented) vs 📋 (spec-only with PageSpecView)
- Each spec-only page includes note: *Consult JSON for this page as it is not yet rendered with app content.*

### CEAR vs SPAY: Two Distinct Pages
- **CEAR** (`/dashboard/teaching/earnings`) - Full earnings dashboard with charts, transaction history, payout requests
- **SPAY** (`/settings/payments`) - Stripe Connect setup/management page
- Initially confused these as one page; they serve different purposes and belong in different sections
- SPAY is implemented with real `StripeConnectSettings` component; CEAR is spec-only

### Route-to-Page Mapping
- 57 route files in `src/pages/` = 57 pages in Block 10
- Discrepancies revealed duplicate routes (two paths to CEAR concept)
- Resolution: Keep routes consistent with their patterns (`/settings/*` vs `/dashboard/teaching/*`)

## Workflow

### Page Analysis Approach
- When comparing implementation to spec, read BOTH the route file AND the JSON spec
- Don't assume feature completeness from route file alone - check what JSON specifies
- For implemented pages: list what's done [x] AND what's pending [ ]
- For spec-only pages: add JSON consult note + placeholder feature list

### Block 10 Feature List Sources
- Implemented pages (16): Features derived from actual code analysis
- Spec-only pages (42): Features are educated guesses, NOT from JSON specs
- Added explicit note to remind future readers to consult JSON for authoritative spec

## Architecture

### Dual-View Pattern for Pages
- Pages with real implementation should show BOTH:
  1. `PageSpecView` rendering the JSON spec
  2. Real content in `slot="content"`
- SPAY was missing PageSpecView - added it to maintain consistency
- Pattern allows comparing spec vs implementation during development
