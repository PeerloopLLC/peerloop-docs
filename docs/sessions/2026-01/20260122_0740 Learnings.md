## 1. Creator Course APIs Follow /api/me/ Pattern
**Topics:** astro, api

**Context:** Building the Creator Studio required APIs for course management that are creator-scoped (only see your own courses).

**Learning:** The `/api/me/` prefix convention works well for user-scoped resources. It clearly distinguishes personal resource APIs from admin APIs (`/api/admin/`) and public APIs (`/api/courses/`). Session validation happens once at the top of each endpoint with `getSession()`.

**Pattern:**
```typescript
// User-scoped API pattern
const session = await getSession(cookies, jwtSecret);
if (!session) return Response.json({ error: 'Authentication required' }, { status: 401 });

// Verify ownership in query
const course = await queryFirst(db,
  'SELECT * FROM courses WHERE id = ? AND creator_id = ? AND deleted_at IS NULL',
  [courseId, session.userId]
);
```

## 2. React useEffect Required for Modal Form Population
**Topics:** astro

**Context:** The CurriculumEditor modal needed to populate form fields when editing an existing module. Initial attempt used `useState()` with a function, which doesn't work as intended.

**Learning:** For populating form state when props change (like when a modal opens with different data), use `useEffect` with the relevant props in the dependency array. The `useState` hook's initializer only runs once on mount - it won't re-run when props change.

**Pattern:**
```typescript
// Correct - useEffect for prop-driven updates
useEffect(() => {
  if (isOpen && module) {
    setTitle(module.title);
    setDescription(module.description);
    // ... other fields
  }
}, [isOpen, module]);

// Wrong - useState initializer only runs once
useState(() => {
  // This won't re-run when module changes!
  setTitle(module.title);
});
```

## 3. Tailwind v4 Class Renames in /q-codecheck
**Topics:** workflow, astro

**Context:** The code quality check caught `flex-shrink-0` classes that should be `shrink-0` in Tailwind v4.

**Learning:** The `check:tailwind` script catches v3→v4 class renames. Common renames include:
- `flex-shrink-*` → `shrink-*`
- `flex-grow-*` → `grow-*`
- `bg-gradient-to-*` → `bg-linear-to-*`
- `shadow` → `shadow-sm`, `shadow-sm` → `shadow-xs`

The script may report false positives for classes in comments, so review each match.

## 4. Publishing Checklist Pattern for Course Validation
**Topics:** api, workflow

**Context:** Courses need all required fields populated before publishing. Rather than a simple boolean check, a checklist pattern provides better user feedback.

**Learning:** When validating complex objects before state transitions, return a checklist object showing which requirements pass/fail. This enables the UI to show progress and guide users to fix specific issues.

**Pattern:**
```typescript
// API returns checklist + missing items
const checklist = {
  title: !!course.title,
  description: course.description.length >= 50,
  thumbnail: !!course.thumbnail_url,
  modules: moduleCount >= 1,
};
const missing = Object.entries(checklist)
  .filter(([_, v]) => !v)
  .map(([k]) => k);

if (missing.length > 0) {
  return Response.json({ success: false, checklist, missing }, { status: 400 });
}
```

## 5. Move Up/Down Pattern from CategoriesAdmin
**Topics:** astro, api

**Context:** The curriculum editor needed module reordering without drag-and-drop (keeping MVP simple).

**Learning:** The CategoriesAdmin pattern for move up/down works well: swap items locally for optimistic UI, send array of IDs to reorder endpoint, endpoint updates `display_order` for each item. On error, refetch to revert. This avoids complex drag-drop libraries while providing good UX.

**Pattern:**
```typescript
// Frontend
const handleMove = async (id: string, direction: 'up' | 'down') => {
  const newOrder = [...items];
  const idx = newOrder.findIndex(i => i.id === id);
  const swapIdx = direction === 'up' ? idx - 1 : idx + 1;
  [newOrder[idx], newOrder[swapIdx]] = [newOrder[swapIdx], newOrder[idx]];
  setItems(newOrder); // Optimistic

  const res = await fetch('/api/.../reorder', {
    method: 'PUT',
    body: JSON.stringify({ order: newOrder.map(i => i.id) })
  });
  if (!res.ok) fetchItems(); // Revert on error
};
```
