# Session Learnings - 2026-01-08

## Marketing Page Pattern

Marketing pages follow a consistent implementation pattern:

1. **Component Location:** `src/components/marketing/{PageName}Page.tsx`
2. **Export via Index:** Add export to `src/components/marketing/index.ts`
3. **Astro Page Structure:**
   - Import from `@components/marketing`
   - Use `LandingLayout` with `fullWidth={true}` and `pageCode`
   - Include PageSpecView in `container-main py-8` div
   - Use `client:load` directive for React component
4. **Color Schemes:**
   - Primary pages (FCRE): Use `primary-*` colors (blue)
   - Teacher pages (BTAT): Use `accent-*` colors (teal/green) for differentiation

## Interactive Earnings Calculators

Both FCRE and BTAT pages include interactive earnings calculators:
- Use React `useState` for slider values
- Range inputs with `accent-{color}-600` for consistent styling
- Real-time calculation updates
- Display weekly/monthly/yearly projections

## Page Code Consistency

Found inconsistency: JSON spec had `BCMT` while PAGES-MAP.md used `BTAT`. Always verify page codes match across:
- `PAGES-MAP.md`
- JSON spec `metadata.code`
- Astro page comment header

## Verification Workflow

The `/q-verify-page` → `/q-consolidate-page` workflow:
1. Reads component code for TODOs and interactive elements
2. Analyzes screenshot for visual verification
3. Creates/updates `docs/pages/{CODE}/{CODE}.md`
4. Updates JSON spec with `features`, `interactiveElements`, `verificationNotes`, `_discrepancies`
5. Moves screenshots to `docs/pages/{CODE}/`

## Static vs Dynamic Pages

Marketing pages (FCRE, BTAT) use static data for:
- Testimonials
- Platform stats
- Success stories

This is acceptable for MVP - dynamic APIs can be added later. Document in `_discrepancies.implementedDifferently`.
