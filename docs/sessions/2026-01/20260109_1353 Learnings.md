# Session Learnings - 2026-01-09

## Vitest `it.runIf()` Timing

**Context:** Tests in `d1-rest-fallback.test.ts` were failing on CI but passing locally.

**Learning:** Vitest's `it.runIf(condition)` evaluates the condition at **test definition time** (when the file is parsed), not at test execution time.

**Problem Pattern:**
```typescript
describe('Test Suite', () => {
  let machine: MachineType;

  beforeAll(() => {
    machine = detectMachine();  // Set at runtime
  });

  // WRONG: machine is undefined when this line executes
  it.runIf(!machine?.supportsFeature)('conditional test', () => {
    // ...
  });
});
```

**Solution Pattern:**
```typescript
// Detect at module load time (top level)
const machine = detectMachine();
const shouldRunTest = !machine.supportsFeature;

describe('Test Suite', () => {
  // CORRECT: shouldRunTest is defined when this executes
  it.runIf(shouldRunTest)('conditional test', () => {
    // ...
  });
});
```

**Key Insight:** Any variables used in `it.runIf()`, `it.skipIf()`, or `describe.runIf()` conditions must be defined at module scope, not inside `beforeAll()` or `beforeEach()`.

## CI Environment Detection

**Context:** Tests checking for `.dev.vars` file failed on GitHub CI because that file contains secrets and is gitignored.

**Learning:** Tests that verify local development environment configuration should be skipped on CI environments.

**Pattern:**
```typescript
const isLocalDev = machine.name !== 'Unknown';  // CI detected as 'Unknown'

it.runIf(isLocalDev)('should have .dev.vars file', () => {
  // Only runs on known local machines
});
```

## Wrangler as Project Dependency

**Context:** E2E tests failed on CI with `wrangler: not found`.

**Learning:** Tools used in npm scripts must be project dependencies, not just globally installed on dev machines.

**Before:** Wrangler installed globally on dev machines, npm scripts used `wrangler` directly.

**Fix:** Added `wrangler` to `devDependencies` in package.json.

**CI Insight:** GitHub Actions runners have a fresh environment - only dependencies in package.json are available after `npm ci`.
