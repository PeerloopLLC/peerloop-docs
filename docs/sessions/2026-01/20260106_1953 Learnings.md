# Session Learnings - 2026-01-06

## Documentation

### Page Documentation Refactoring Pattern
- **PAGES-MAP.md** works best as a slim lookup table (~140 lines) with individual page details in `docs/pages/CODE.md`
- Each page file should follow a consistent structure: metadata table, Purpose, User Stories, Connections, Data Requirements, Features, Sections, API Endpoints, States, Error Handling, Implementation Notes
- User story tags `[US-XXXX]` on feature items create traceability back to requirements

### Preserving Original Content
- When refactoring documentation, always save the original to `.claude/` before making changes (e.g., `.claude/PAGES-MAP-original.md`)
- This allows recovery and serves as source for incremental migration

## Workflow

### Batching Large Documentation Tasks
- When merging 40+ page spec files, batch by logical groups (Admin, Creator, Teaching, etc.)
- Pause after each batch to check token usage before continuing
- Use TodoWrite to track progress through batches explicitly

### Resume State for Mid-Task Pauses
- For large multi-file tasks, RESUME-STATE.md is essential to preserve context across `/compact`
- Include: current batch, completed batches, next action, todo list state, and important context (templates, patterns)
- The todo list is session-scoped; don't restore pending items blindly - ask if continuing same work

## Technical

### Page Spec File Format
The optimal merged page file format includes:
1. Metadata table with Route, Access, Priority, Status, Block, JSON Spec, Astro Page
2. Purpose (1-2 sentences)
3. User Stories table (ID, Story, Priority, Status)
4. Connections (Incoming/Outgoing navigation)
5. Data Requirements (Entity, Fields, Purpose)
6. Features grouped by category with story tags
7. Sections from Plan (UI details, tables)
8. API Endpoints table with request/response examples
9. States & Variations
10. Error Handling
11. Implementation Notes

### Source File Locations
- Research specs: `research/run-001/pages/page-CODE-name.md`
- Target docs: `docs/pages/CODE.md`
- Templates: ACRS.md and ACAT.md (implemented pages with full specs)
