# Session Decisions - 2026-01-08

## Technical Decisions

### 1. Database Schema Status Pattern - Deferred to Technical Debt

**Context:** Discovered that status-like fields use inconsistent patterns across tables:
- Boolean flags (`is_active`, `suspended_at`) on courses/users
- Enum status fields on enrollments/transactions
- Badge field on courses for 'featured' status

**Decision:** Document as Technical Debt in PLAN.md rather than fix immediately.

**Rationale:**
- Fixing requires schema migration affecting multiple tables
- Current queries work correctly with proper column names
- Not blocking any immediate work
- Can be addressed post-MVP if warranted

**Location:** PLAN.md > Technical Debt / Future Investigation

### 2. Admin Dashboard Metrics - Server-Side Aggregation

**Context:** Dashboard needs to display multiple platform metrics.

**Decision:** Aggregate all metrics server-side in a single API call.

**Rationale:**
- Single network request for all data
- Reduces client-side complexity
- D1 queries are efficient for aggregations
- REST API fallback works on MBA-2017

### 3. Analytics Events - Deferred

**Context:** PageSpec defines `page_view` and `admin_refresh` analytics events.

**Decision:** Document as "planned but not implemented" in discrepancies rather than implement now.

**Rationale:**
- Analytics infrastructure not yet set up
- Dashboard functionality complete without them
- Can be added when analytics system is implemented
- Low priority for MVP
