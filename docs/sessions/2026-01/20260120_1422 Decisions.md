# Session Decisions - 2026-01-20

## 1. Create /q-end-with-plan Skill
**Type:** Strategic
**Topics:** workflow

**Trigger:** Realized that `/q-end-session` doesn't detect planned work discussed in conversation - only TodoWrite items trigger save state.

**Options Considered:**
1. Manually run `/q-save-state` before `/q-end-session`
2. Manually add todos before `/q-end-session`
3. Create a combined skill that captures plan → adds todos → runs end-session ← Chosen

**Decision:** Create `/q-end-with-plan` skill that takes a plan description, generates TodoWrite items, then runs `/q-end-session`.

**Rationale:** Options 1 and 2 require user to remember multiple steps. A combined skill provides a single command that bridges the gap between "we discussed what's next" and "state is saved for resume."

**Consequences:**
- New skill at `~/.claude/commands/q-end-with-plan.md`
- Usage: `/q-end-with-plan <description>` or `/q-end-with-plan --recent`
- Enables clean handoff when pausing mid-plan

## 2. Add --recent Argument to /q-end-with-plan
**Type:** Implementation
**Topics:** workflow

**Trigger:** User noted it would be tedious to re-type a plan that was just discussed in conversation.

**Options Considered:**
1. Always require explicit description
2. Add `--recent` flag to scan conversation for discussed plans ← Chosen
3. Add `--context` or `-c` shorthand

**Decision:** Add `--recent` argument that scans recent conversation for requirements, "we need to...", and next step discussions.

**Rationale:** When you've just been discussing requirements (like CDIS access model), you shouldn't have to re-type it. The conversation context has all the information needed.

**Consequences:**
- `/q-end-with-plan --recent` scans conversation and extracts plan
- Still shows generated todos for confirmation before proceeding
- Reduces friction for the common case of "we just talked about this"
