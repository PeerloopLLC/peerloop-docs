# Session Decisions - 2026-01-25

## 1. Admin User Creation Requires Password
**Type:** Implementation
**Topics:** auth, admin

**Trigger:** Schema has `password_hash TEXT NOT NULL` but the admin user creation endpoint allowed null password for an "email invite flow" that wasn't implemented.

**Options Considered:**
1. Allow NULL `password_hash` in schema for email-invite flow (user sets password via welcome email)
2. Require password when creating users via admin ← Chosen
3. Auto-generate random password and return in API response

**Decision:** Admin-created users must have a password. The endpoint returns 400 if password is not provided.

**Rationale:** For MVP/Genesis cohort (60-80 students with direct contact):
- No dependency on email system (Resend domain verification not yet complete)
- User can log in immediately after admin creates account
- Admin can share credentials during onboarding calls/meetings
- No NULL handling complexity in login logic

**Consequences:**
- Updated `POST /api/admin/users` to require password field
- Updated tests to always include password
- Documented in `DECISIONS.md` for future reference
- Post-MVP may revisit email-invite flow

## 2. Add ROLES Block to PLAN.md
**Type:** Strategic
**Topics:** workflow, admin

**Trigger:** Analysis revealed that admins cannot promote existing users to Admin or Moderator roles via the UI, despite the API supporting it.

**Options Considered:**
1. Add to existing ADMIN block as enhancement
2. Add to POLISH block since it's refinement
3. Create dedicated ROLES block ← Chosen

**Decision:** Create new ROLES block in PLAN.md (Priority 2, after TESTING) to address admin role management gap.

**Rationale:**
- Clear scope: Edit UI, Create UI, optional Audit
- Distinct from original ADMIN block (which was page roughing)
- More urgent than SEEDDATA (admin needs this for onboarding)
- API already exists, just needs UI

**Consequences:**
- Added `## Pending: ROLES` section to PLAN.md
- Defined ROLES.EDIT_UI, ROLES.CREATE_UI, ROLES.AUDIT subsections
- Shifted SEEDDATA to Priority 3, POLISH to Priority 4
