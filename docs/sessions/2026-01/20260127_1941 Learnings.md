# Session Learnings - 2026-01-27 (Session 130)

## 1. ESM __dirname Not Available in tsx Scripts
**Topics:** astro, workflow

**Context:** The `populate-page-metadata.ts` script used `__dirname` which fails in ESM mode (Node 22 with tsx).

**Learning:** Always use `fileURLToPath(import.meta.url)` + `dirname()` instead of `__dirname` in scripts run via `npx tsx`, since the project is ESM.

**Pattern:**
```ts
import { fileURLToPath } from 'url';
import { dirname } from 'path';
const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);
```

## 2. Barrel Imports Resolve to index.ts Not index.tsx
**Topics:** astro, workflow

**Context:** When parsing `.astro` frontmatter imports like `import { AboutPage } from '@components/marketing'`, the script needed to resolve the actual file path. Initially only checked for `index.tsx`.

**Learning:** Barrel exports in this project use `index.ts` (not `index.tsx`). The resolution order should be: `.tsx` file → `index.tsx` → `index.ts` → `.jsx` → fallback `.tsx`.

## 3. Type Imports Must Be Filtered When Parsing Astro Frontmatter
**Topics:** astro

**Context:** Parsing `.astro` imports for component discovery was picking up `import type { STListItem } from '@components/teachers/STDirectory'` as a component reference.

**Learning:** Filter out lines starting with `import type ` when scanning for runtime component imports. Only value imports represent actual components loaded by the page.

## 4. Node -e with !== Fails Due to Shell Escaping
**Topics:** workflow

**Context:** Running `node -e "..."` with strict inequality (`!==`) caused shell escape issues on zsh — the `!` is interpreted as history expansion.

**Learning:** Use heredoc (`node --input-type=commonjs <<'SCRIPT'`) for inline Node scripts containing special characters, or use `==` with loose equality instead.

## 5. PageSpec Schema Had Stale Type Export (ApiCall vs PlannedApiCall)
**Topics:** workflow

**Context:** `src/lib/types/page-spec.ts` re-exported `ApiCall` but the schema defines `PlannedApiCall`. This was silently broken.

**Learning:** When renaming schema types, search re-export files too. The types file is a thin re-export layer that can fall out of sync with the schema without TypeScript catching it (if nobody imports the stale name).

---

## 6. JSON Validation Failures Were Pre-Existing and Unrelated to Changes
**Topics:** testing, workflow

**Context:** After updating 66 JSON files, validation found 8 failures. All were pre-existing issues (parenthetical notes in enum values, missing required fields, wrong types).

**Learning:** Running full validation after bulk updates is essential — it surfaces pre-existing debt that was previously invisible. The bulk update was a good forcing function to clean up data quality across all 66 files.
