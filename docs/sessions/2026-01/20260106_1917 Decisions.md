## Decision 2: Page Documentation Merge Strategy

**Type**: Strategic
**Driver**: User-driven
**Working On**: Documentation consolidation
**Trigger**: User noted overlap between `/research/run-001/pages/page-*.md` (design specs) and `/docs/pages/*.md` (implementation tracking)

**Options Considered**:
1. **Merge into `docs/pages/`** - Single source of truth per page <- **Chosen**
2. Keep separate with clear roles (research = frozen spec, docs = living implementation)
3. Merge into `research/run-001/pages/` (research becomes implementation tracker)

**Decision**: Merge research page specs into docs/pages/, combining design specs with implementation status. Delete research files after merge.

**Rationale**:
- Research specs already "contaminated" with updates during implementation
- Single source of truth easier to maintain
- docs/pages/ is the natural home for active development documentation

**Consequences**:
- docs/pages/ACRS.md and ACAT.md now contain full spec details + implementation status
- Added "Plan vs Implementation" sections to track gaps
- Added Status columns (✅/⏳) throughout
- Deleted research/run-001/pages/page-ACRS-admin-courses.md and page-ACAT-admin-categories.md
- Pattern established for merging remaining ~40 page docs

## Decision 3: Preserving Plan vs Implementation Differences

**Type**: Implementation
**Driver**: User-driven
**Working On**: Page documentation merge
**Trigger**: User specified that planned-but-not-implemented features must not be lost

**Options Considered**:
1. Simply delete unimplemented planned items
2. **Mark unimplemented items with notes** <- **Chosen**
3. Create separate "future features" section

**Decision**: Keep all planned features in merged docs, mark unimplemented ones with "(planned, not implemented)" or "(planned for future)" notes. Add "Plan vs Implementation" summary table at end of each page doc.

**Rationale**: Preserves the planning work while clearly indicating current implementation state. Future developers can see what was planned but deferred.

**Consequences**:
- Pending Features sections show planned items with status notes
- Plan vs Implementation tables summarize gaps at a glance
- No planning information lost during merge
