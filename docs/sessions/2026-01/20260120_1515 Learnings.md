# Session Learnings - 2026-01-20

## 1. Stream Feed Groups Must Be Pre-Configured
**Topics:** stream, cloudflare

**Context:** Attempted to create course discussion feeds using `FEED_GROUP = 'course'` but got a 500 error from Stream API.

**Learning:** Stream.io requires feed groups to be pre-configured in the Stream dashboard before use. You cannot dynamically create new feed groups via the API. The existing `townhall` feed group works because it was already set up.

**Pattern:**
```typescript
// ❌ Will fail if 'course' feed group doesn't exist in Stream dashboard
const feed = client.feed('course', courseId);

// ✅ Use existing feed group with namespaced ID
const feed = client.feed('townhall', `course-${courseId}`);
```

## 2. Instructor Feed Uses Client-Side Filtering (Not Separate Feeds)
**Topics:** stream, architecture

**Context:** Investigated how the existing InstructorFeed implementation works, expecting separate feeds per instructor.

**Learning:** The instructor feed doesn't create a separate Stream feed per instructor. Instead, it queries the single global `townhall:main` feed, fetches 3x the requested limit, and filters client-side by `courseId` matching the instructor's courses. This pattern is simple but has scalability limits (pagination breaks if early pages have no matches).

**Pattern:**
```typescript
// Current pattern: filter from global feed
const feed = client.feed('townhall', 'main');
const response = await feed.get({ limit: limit * 3 }); // Overfetch
const filtered = response.results.filter(a => courseIds.has(a.courseId));
```

## 3. Feed Flags Require Explicit Migration
**Topics:** d1, stream

**Context:** Adding `discussion_feed_enabled` and `instructor_feed_enabled` columns to enable/disable feeds.

**Learning:** SQLite/D1 `ALTER TABLE ADD COLUMN` works well for adding nullable columns with defaults. The pattern of `enabled` + `created_at` provides both current state and audit trail of when a feed was first created in Stream.

**Pattern:**
```sql
ALTER TABLE courses ADD COLUMN discussion_feed_enabled INTEGER DEFAULT 0;
ALTER TABLE courses ADD COLUMN discussion_feed_created_at TEXT;
```
