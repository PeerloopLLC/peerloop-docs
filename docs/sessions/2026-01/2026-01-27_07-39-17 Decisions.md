# Session Decisions - 2026-01-27

## 1. Enhanced /q-commit Format to 15-20 Lines with Grouped Sections
**Type:** Strategic
**Topics:** workflow

**Trigger:** User requested more detailed commit messages while maintaining current focus.

**Options Considered:**
1. More bullet points only
2. Section grouping only (Changes, Fixes, Tests, Scripts)
3. Stats/metrics only (test counts, file counts)
4. All of the above — more bullets, grouped by category, with stats ← Chosen

**Decision:** Update `/q-commit` to require 15-20 line commit messages with grouped sections (`Changes:`, `Fixes:`, `Tests:`, `Scripts:`), specific file/component names, test counts, and a `Stats:` summary line. Only relevant sections are included (skip empty categories).

**Rationale:** The previous format was too terse for sessions that touch many files and tests. The grouped format makes commit history scannable by category while the stats line gives a quick scope summary.

**Consequences:** Updated `~/.claude/commands/q-commit.md` (global) and `.claude/commands/q-commit-local.md` (Peerloop example). All future commits across all projects will use the longer format.
