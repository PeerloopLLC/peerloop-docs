# Session Decisions - 2026-01-21

## 1. Availability Page Location
**Type:** Implementation
**Topics:** astro, workflow

**Trigger:** Where to put the full availability editor - Settings page (SETT) or dedicated teaching page?

**Options Considered:**
1. Settings page at `/settings` - matches page spec which says "Edit Availability" → SETT
2. Dedicated page at `/dashboard/teaching/availability` ← Chosen

**Decision:** Create dedicated availability page at `/dashboard/teaching/availability` rather than adding to Settings.

**Rationale:**
- Settings page (SETT) is still a spec placeholder, would require implementing the whole page
- Teaching-related functionality should be in the teaching dashboard area
- Consistent with earnings and students being under `/dashboard/teaching/`
- Quick action in dashboard can link directly to the right context

**Consequences:**
- Created `/dashboard/teaching/availability.astro`
- Updated TeacherDashboard quick action link from `/settings` to `/dashboard/teaching/availability`
- Future Settings page implementation can link to this page rather than duplicate the component

## 2. Payout Request as Pending (Not Auto-Process)
**Type:** Implementation
**Topics:** stripe, workflow

**Trigger:** Should payout requests automatically trigger Stripe transfers or require admin approval?

**Options Considered:**
1. Auto-process via Stripe Transfer immediately on request
2. Create pending record for admin to process ← Chosen
3. Scheduled batch processing (e.g., weekly)

**Decision:** Create payout record with 'pending' status; admin processes via admin panel.

**Rationale:**
- MVP approach - simpler to implement
- Gives admin oversight on initial payouts
- Allows manual review before money leaves platform
- Stripe webhook integration for auto-processing can be added later

**Consequences:**
- `POST /api/me/payouts/request` creates pending payout record
- Admin uses existing `/admin/payouts` to process pending payouts
- Payout status flow: pending → processing → completed/failed
