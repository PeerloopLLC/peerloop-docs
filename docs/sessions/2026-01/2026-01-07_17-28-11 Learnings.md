## Technical

### Astro + Cloudflare Environment Variables

**Issue:** `.dev.vars` is wrangler-specific and not loaded by `astro dev`.

**Solution:** Create a symlink: `ln -s .dev.vars .env`

**Why:** Astro's dev server reads `.env` files; wrangler reads `.dev.vars`. The symlink lets both tools use the same file without duplication.

**Documentation:** Added to `docs/tech/tech-013-devcomputers.md` for MBA-2017 setup.

### D1 REST Fallback Pattern

**Issue:** `learn.astro` returned 503 because it checked for native D1 binding directly instead of using `getDB()`.

**Wrong:**
```typescript
const db = Astro.locals.runtime?.env?.DB;
if (!db) return new Response('Service unavailable', { status: 503 });
```

**Correct:**
```typescript
import { getDB } from '@lib/db';
let db: D1Database;
try {
  db = getDB(Astro.locals);
} catch {
  return new Response('Service unavailable', { status: 503 });
}
```

**Why:** `getDB()` handles the REST API fallback for MBA-2017 (macOS 12) which can't run local D1 emulation.

### Stripe Connect Account Management

**Learning:** Connected accounts in Stripe can only be deleted via API, not dashboard UI.

```bash
curl -X DELETE https://api.stripe.com/v1/accounts/acct_XXXXX \
  -u sk_test_YOUR_SECRET_KEY:
```

Dashboard "Reject" option changes status but doesn't remove from list. API delete removes completely (after cache refresh).

### Stripe Webhook Testing Locally

**Setup for local webhook testing:**
1. Install Stripe CLI: `brew install stripe/stripe-cli/stripe`
2. Login: `stripe login`
3. Forward webhooks: `stripe listen --forward-to localhost:4321/api/webhooks/stripe`
4. Add webhook secret to `.dev.vars`: `STRIPE_WEBHOOK_SECRET=whsec_...`

**Key insight:** Webhooks need both terminals running - dev server AND stripe listen.

## Workflow

### PageSpecView Pattern for New Pages

All pages using `AppLayout` or `LandingLayout` should include PageSpecView for dev/spec toggle:

1. Import: `PageSpecView` and `pageData` JSON
2. Add `pageCode={pageData.metadata.code}` to layout
3. Add `<PageSpecView pageData={pageData} />` in main content
4. Wrap real content in `<Fragment slot="content">`
5. Add sidepanel navigation in `<Fragment slot="sidepanel">`

---

### Page Code Consistency

**Learning:** JSON spec `code` field must match docs folder name.

- Docs folder: `docs/pages/CSUC/`
- JSON spec: `"code": "CSUC"` (not "SUCC")

Inconsistency causes confusion when toggling between spec and content views.

## Architecture

### Stripe Checkout Flow (End-to-End)

1. **CDET** → Student clicks "Enroll Now"
2. **EnrollButton** → POST `/api/checkout/create-session`
3. **Stripe** → Redirect to Checkout
4. **Stripe** → Webhook POST to `/api/webhooks/stripe`
5. **Webhook** → Creates enrollment in D1
6. **Stripe** → Redirect to **CSUC** (`/courses/[slug]/success`)
7. **CSUC** → Student clicks "Start Learning"
8. **CCNT** → Course content loads (enrollment check passes)

Critical: Enrollment is created by webhook, not by success page load.
