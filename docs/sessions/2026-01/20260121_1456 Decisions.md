# Session Decisions - 2026-01-21

## 1. Revert MBA-2017 Fixes, Use MacMini for Development
**Type:** Strategic
**Topics:** Development Environment, MBA-2017, Architecture

**Trigger:** MBA-2017 D1 REST fallback fix worked but was hacky (explicit env var injection in astro.config.mjs) and broke JWT_SECRET access.

**Options Considered:**
1. Keep hacky fix, inject all needed env vars explicitly
2. Remove `'process.env': '{}'` entirely and find other solution for getstream ← Deferred investigation
3. Revert changes, use MacMini for development ← Chosen

**Decision:** Revert all MBA-2017 fix attempts and use MacMini for development.

**Rationale:**
- The fix was unsustainable - every new env var would need explicit injection
- Broke JWT_SECRET which affected login
- Risk to stable MacMini/Cloudflare Preview/Production environments
- MacMini provides clean development experience without workarounds

**Consequences:**
- MBA-2017 cannot be used for database-dependent development
- Need to investigate if `'process.env': '{}'` is still required for getstream
- All changes reverted to restore stable state

## 2. Use Separate Resend API Keys for Dev vs Production
**Type:** Implementation
**Topics:** Resend, Email, Security

**Trigger:** Setting up Resend for moderator invite emails.

**Options Considered:**
1. Single API key for all environments
2. Separate dev and production API keys ← Chosen

**Decision:** Create separate Resend API keys for development and production.

**Rationale:**
- Standard practice for API key management
- Dev key in `.dev.vars`, production via `wrangler secret put`
- Allows revoking dev key without affecting production
- Clear audit trail for email sends

**Consequences:**
- `.dev.vars.example` updated with documentation
- Production key set in Cloudflare dashboard
