# Session Decisions — 2026-01-27

## Session 129: Strengthen CANA Tests (Items 11-17)

### D1: Child component tests go in same directory as orchestrator test
**Decision:** Place child component tests alongside the orchestrator test in `tests/components/analytics/`.
**Rationale:** Follows existing convention (AdminAnalytics.test.tsx is in same directory). Makes `test-CANA.sh` script straightforward — CHILD_TESTS lists explicit file paths.
**Alternatives considered:** Separate `tests/components/analytics/children/` subdirectory — rejected, too many nesting levels for no benefit.

### D2: Test script uses explicit CHILD_TESTS variable (not glob)
**Decision:** List all 7 child test files explicitly in `test-CANA.sh` as a CHILD_TESTS variable rather than using a glob pattern.
**Rationale:** Explicit is better — if a new analytics component is added, it should be deliberately added to the test script, not auto-included. Also, other analytics tests in the same directory (AdminAnalytics.test.tsx) should not be included in CANA's test run.

### D3: Re-audit overwrites AUDIT-<CODE>.md (not append)
**Decision:** Running `/L-audit-tests` overwrites the existing audit file with a fresh assessment.
**Rationale:** The audit is a point-in-time snapshot. Old audits are only useful for comparison (user backed up manually before re-running). The current state matters most.
