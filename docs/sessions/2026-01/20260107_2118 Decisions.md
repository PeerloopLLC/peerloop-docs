## Decision 3: Contact Form Stores to D1, Email Deferred

**Type**: Implementation
**Driver**: Claude-driven
**Working On**: POST /api/contact implementation
**Trigger**: Spec mentions both D1 storage and Resend email notification

**Options Considered**:
1. Implement both D1 storage and Resend email in this session
2. Implement D1 storage now, defer email to Block 9 <- **Chosen**
3. Skip D1, only send email

**Decision**: Store contact submissions to D1 contact_submissions table. Defer Resend email notification to Block 9 (Notifications).

**Rationale**:
- Block 9 is designated for notifications (email + in-app)
- D1 storage provides immediate value (submissions tracked, status managed)
- Adding email later is additive, doesn't require refactoring
- Keeps this session focused on marketing pages

**Consequences**:
- Contact form fully functional for submission
- Admin can view submissions in D1 (future admin page)
- Email notification documented as "Planned: Block 9" in page spec
