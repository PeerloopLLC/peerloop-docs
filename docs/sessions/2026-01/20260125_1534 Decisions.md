# Session Decisions - 2026-01-25

## 1. Fix Both Schema and Endpoint Code for Analytics Mismatches
**Type:** Implementation
**Topics:** d1, testing

**Trigger:** Analytics endpoints had 7+ column mismatches with the actual schema. User was asked how to proceed.

**Options Considered:**
1. Fix the endpoint code to match current schema
2. Update the schema to add missing columns ← Chosen
3. Skip analytics tests for now
4. Partial test coverage only

**Decision:** Update the schema AND fix endpoint code where it's just a column name mismatch.

**Rationale:** The missing columns (`last_login`, `progress_percent`, `recommended_as_st`, `duration_minutes`) are genuinely useful for analytics. The column name mismatches (`certified_at` vs `certified_date`, `scheduled_at` vs `scheduled_start`) were just bugs in the endpoint code that should be fixed to use the correct schema column names.

**Consequences:**
- Added 4 new columns to schema: `users.last_login`, `enrollments.progress_percent`, `enrollments.recommended_as_st`, `sessions.duration_minutes`
- Fixed 5 analytics endpoint files to use correct column names
- All 56 analytics tests now pass
