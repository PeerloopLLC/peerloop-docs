# Session Decisions - 2026-01-26

## 1. Document Astro SSR Integration Testing Gap in PLAN.md
**Type:** Strategic
**Topics:** testing, astro, workflow

**Trigger:** User questioned why SSR pages were marked as having "fewer testing concerns" when they still need error handling for database failures.

**Options Considered:**
1. Acknowledge gap and continue with current approach (defer to future) ← Chosen
2. Immediately create integration tests for Astro SSR error scenarios
3. Modify all SSR pages to handle errors at component level

**Decision:** Document the SSR integration testing gap in PLAN.md's TESTING.INTEGRATION section, creating actionable items for future work rather than addressing immediately.

**Rationale:**
- Current component tests verify correct behavior *given data*
- Current API tests verify endpoints work *given requests*
- The Astro SSR layer is the untested middle ground
- This is a legitimate gap but not blocking current test verification work
- Documenting it ensures it won't be forgotten

**Consequences:**
- Added to PLAN.md with table showing what's tested vs missing
- Created 3 pending items: Astro SSR error handling tests, error boundary verification, graceful degradation decisions
- Future sessions can address this systematically
