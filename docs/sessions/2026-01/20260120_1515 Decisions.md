# Session Decisions - 2026-01-20

## 1. Explicit Feed Creation Model
**Type:** Strategic
**Topics:** stream, architecture

**Trigger:** Designing how course discussion and instructor feeds should be created and managed.

**Options Considered:**
1. Auto-create feeds when courses/instructors are created
2. Explicit creator action to enable feeds ← Chosen
3. Lazy creation on first access

**Decision:** Feeds require explicit creator action to enable. Creators toggle "Enable Discussion" in Course Studio or "Enable Instructor Feed" in their settings.

**Rationale:**
- Resource control: Not every course needs a discussion feed
- Creator intent: Opt-in signals commitment to moderate/engage
- Stream costs: Only create feeds that will be used
- Progressive disclosure: Feature appears when ready, not by default

**Consequences:**
- Added `discussion_feed_enabled` column to `courses` table
- Added `instructor_feed_enabled` column to `users` table
- Created API endpoints for enable/disable actions
- Feed pages show "not available" state when disabled

## 2. Course Discussion Feeds Are Separate From TownHall
**Type:** Strategic
**Topics:** stream, architecture

**Trigger:** Discussion of whether course feeds should be filtered views of TownHall or separate feeds.

**Options Considered:**
1. Single feed + client-side filtering (like current instructor feed)
2. Separate dedicated feeds per course ← Chosen
3. Fan-out on write (post to multiple feeds)

**Decision:** Course discussion feeds are separate Stream feeds (`townhall:course-{id}`), not filtered views of the global TownHall feed.

**Rationale:**
- Scalability: Client-side filtering doesn't scale
- Pagination: Separate feeds have reliable pagination
- Isolation: Course discussions stay within course context

**Consequences:**
- Course feeds use `townhall:course-{courseId}` as feed ID
- Need explicit feed creation when enabled
- Fan-out question deferred (important for later)

## 3. Fan-Out Mechanism Deferred
**Type:** Strategic
**Topics:** stream, architecture

**Trigger:** Question raised: "When posting to TownHall with course tag, should it also appear in course feed?"

**Options Considered:**
1. Implement fan-out now
2. Defer decision ← Chosen
3. No cross-posting

**Decision:** Defer the fan-out decision. Currently, TownHall posts and course discussion posts are completely separate.

**Rationale:** Fan-out adds complexity and needs clearer UX requirements. Better to launch basic functionality first and add cross-posting if users request it.

**Consequences:**
- Marked as important pending architectural decision in PLAN.md
- Current behavior: Posts in course feed only appear there
- Future work may add fan-out capability
