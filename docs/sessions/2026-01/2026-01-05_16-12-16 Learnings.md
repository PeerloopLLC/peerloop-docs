# Session Learnings - 2026-01-05

## Technical

### R2 Storage Integration
- R2 bindings in Cloudflare Workers are accessed via `locals.runtime.env.STORAGE`
- No presigned URLs needed - direct access through binding
- For protected downloads, create an API endpoint that checks auth then streams from R2
- File download pattern: check enrollment → get R2 object → stream with proper headers

### Video Embed Detection
- YouTube URLs have multiple formats: youtube.com/watch?v=, youtu.be/, youtube.com/embed/
- Vimeo is simpler: vimeo.com/{id} or player.vimeo.com/video/{id}
- Regex patterns should handle all common formats
- Fallback to external link when URL format unknown

### Progress Tracking Architecture
- Store completion at module level (module_progress table)
- Calculate percentage at query time via subqueries
- Auto-update enrollment status when all modules complete
- Consider: "in_progress" status when first module marked (not just "enrolled")

### API Response Typing
- When using `await response.json()`, TypeScript infers `unknown`
- Define interface and type the result: `const data: MyResponse = await response.json()`
- Error responses should also be typed: `{ error?: string }`

## Architecture

### Learning Component Structure
- Main orchestrator (CourseLearning) manages state and API calls
- Sidebar (ModuleSidebar) handles navigation with completion indicators
- Content (ModuleContent) displays current module with all sections
- Utility components (VideoEmbed, ResourcesList) are reusable
- Props flow down, callbacks flow up (React pattern)

### Course Content Page Data Flow
```
Page (SSR) → Auth check → Enrollment check → Pass props to React
                ↓
React (CSR) → Fetch curriculum, progress, resources → Render
                ↓
User action → POST progress → Update local state → Auto-advance
```

### API Endpoint Patterns Established
- `/api/courses/[id]/curriculum` - Public, returns modules
- `/api/courses/[id]/resources` - Auth required, returns resources with signed URLs
- `/api/enrollments/[id]/progress` - GET/POST, returns/updates module completion
- `/api/resources/[id]/download` - Auth required, streams file from R2
