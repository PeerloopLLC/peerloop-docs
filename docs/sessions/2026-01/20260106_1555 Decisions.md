## Decision 3: Categories Admin Uses Modal Pattern

**Context:** Could use slide-in detail panel (like Users Admin) or modal for create/edit.

**Decision:** Use modal for Categories Admin.

**Rationale:**
- Categories have only 3 fields (name, slug, display_order)
- Modal is simpler for few fields
- No need for full detail panel
- Validates that shared components work for simpler use cases

## Decision 4: Move Up/Down for Category Reordering

**Context:** Could implement drag-and-drop or simpler move buttons.

**Decision:** Use Move Up/Move Down action buttons.

**Rationale:**
- Simpler to implement
- Works well for small lists
- Drag-and-drop can be added later if needed
- Reorder API already supports arbitrary ordering
