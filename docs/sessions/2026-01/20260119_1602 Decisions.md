# Session Decisions - 2026-01-19

## 1. Always Stage All Changes in /q-commit
**Type:** Strategic
**Topics:** workflow

**Trigger:** During commit, selective staging was done based on perceived relevance to the current task, leaving some files uncommitted.

**Options Considered:**
1. Selective staging - commit only files related to current task
2. Always stage all changes with `git add .` ← Chosen

**Decision:** Updated `/q-commit` skill to always stage all changes without exception.

**Rationale:** Selective staging based on task context leads to orphaned changes that may be forgotten. At session end, all changes in the working directory should be committed together. If changes truly shouldn't be committed, they should be in `.gitignore` or explicitly stashed.

**Consequences:** Modified `~/.claude/commands/q-commit.md` to remove selective staging guidance and add explicit rule "Always commit ALL changes".
