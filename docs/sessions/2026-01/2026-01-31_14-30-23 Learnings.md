# Session Learnings - 2026-01-31

## 1. TSV Format for Schema + Data Documentation
**Topics:** d1, workflow

**Context:** User wanted to export database schema and seed data in a portable format for review/analysis.

**Learning:** A TSV file with schema metadata as columns (column_name, data_type, nullable, default_value, primary_key, unique, check_constraint, foreign_key, notes) plus data record columns provides a comprehensive view that:
- Shows column definitions alongside actual values
- Remains human-readable in any spreadsheet
- Self-documents the schema without separate files
- Makes it easy to compare values across records (read down a column)

**Pattern:**
```
column_name	data_type	nullable	...	record-1	record-2	record-3
id	TEXT	NO	...	usr-001	usr-002	usr-003
email	TEXT	NO	...	a@x.com	b@x.com	c@x.com
```

## 2. Schema-Only TSV Files for Empty Tables
**Topics:** d1, workflow

**Context:** 20 tables in the schema have no seed data (messaging, moderation, notifications, etc.).

**Learning:** Creating schema-only TSV files (header row + schema metadata row, no data columns) provides value even for empty tables:
- Documents all columns and constraints
- Shows relationships via foreign_key column
- Identifies which tables need seed data for testing
- Maintains consistency with seeded table format

## 3. Extracting Tables from SQL Files
**Topics:** d1, workflow

**Context:** Needed to identify all tables and compare schema vs seed data.

**Learning:** Simple bash commands can parse SQL migration files effectively:
```bash
# Get table names from schema
grep -E "^CREATE TABLE" 0001_schema.sql | sed 's/CREATE TABLE IF NOT EXISTS //' | sed 's/ (.*//'

# Get seeded tables
grep -E "^INSERT INTO" 0002_seed.sql | sed 's/INSERT INTO //' | sed 's/ (.*//' | sort -u

# Find tables without seed data
comm -23 schema_tables.txt seed_tables.txt
```

## 5. Encapsulating Patterns as Skills
**Topics:** workflow

**Context:** After manually creating 48 schema TSV files, user requested a reusable skill for future exports.

**Learning:** When a multi-step process proves useful, encapsulate it as a skill (`/L-*` for project-specific):
- Document arguments and defaults clearly
- Include error handling for missing/invalid inputs
- Limit output size for readability (e.g., max 6 records)
- Use dated folders for versioning

**Pattern:** `/L-schema-dump TABLE=users DATA=true` creates `data/schema/YYYY-MM-DD/users_schema.tsv`
