## 1. IntersectionObserver for Infinite Scroll
**Topics:** astro, ui

**Context:** Implementing infinite scroll pagination for the community feed.

**Learning:** IntersectionObserver with `rootMargin: '200px'` preloads content before the user reaches the bottom, creating a smoother experience. The sentinel element should be a minimal div (1px height) at the end of the list.

**Pattern:**
```typescript
const observer = new IntersectionObserver(
  (entries) => {
    if (entries[0].isIntersecting) {
      loadMore();
    }
  },
  { rootMargin: '200px' } // Start loading 200px before reaching bottom
);
observer.observe(sentinelRef.current);
```

## 2. Stream.io Custom Field Filtering is Client-Side Only
**Topics:** stream

**Context:** Needed to filter feed activities by course or instructor.

**Learning:** Stream Activity Feeds don't support server-side filtering by custom fields. To filter by `courseId` or `instructorId`, you must either:
1. Use separate feed groups (`course:{id}`, `instructor:{id}`) - proper architecture
2. Fetch more than needed and filter client-side - simpler MVP approach

For MVP, we chose client-side filtering of the townhall feed.

## 3. Feed Access Patterns Vary by Use Case
**Topics:** stream, auth

**Context:** Implementing three different feed types with different access models.

**Learning:** Each feed type has distinct access requirements:

| Feed | View Access | Post Access |
|------|-------------|-------------|
| TownHall (FEED) | Authenticated | Authenticated |
| Instructor (IFED) | Enrolled in any instructor course | Same |
| Course (CDIS) | Public | Enrolled/completed students |

The enrollment check query joins enrollments with courses to verify access:
```sql
SELECT 1 FROM enrollments e
JOIN courses c ON e.course_id = c.id
WHERE c.creator_id = :instructor_id
  AND e.student_id = :user_id
  AND e.status IN ('enrolled', 'in_progress', 'completed')
```
