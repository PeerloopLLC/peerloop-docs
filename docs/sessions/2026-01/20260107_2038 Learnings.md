# Session Learnings - 2026-01-07

## Stripe Refund Integration

### transfer_group is not searchable
- `stripe.charges.search({ query: 'transfer_group:"..."' })` fails
- `transfer_group` is not a supported search field for charges or payment_intents
- **Solution:** Store `stripe_payment_intent_id` and `stripe_charge_id` in database when payment succeeds, then look up directly

### Proper payment tracking pattern
1. When checkout completes (webhook), store payment data in `transactions` table:
   - `stripe_payment_intent_id`
   - `stripe_charge_id`
   - `transfer_group`
   - `amount_cents`
2. When refunding, look up from database - no Stripe API search needed
3. Webhook `charge.refunded` updates both `transactions` and `enrollments` tables

### Webhook must be running for full flow
- `stripe listen --forward-to localhost:4321/api/webhooks/stripe`
- Without this, refund API succeeds but database isn't updated
- The webhook handles: enrollment cancellation, transaction status update, transfer reversal
