**Type:** Strategic
**Topics:** testing, workflow

## 2. Fix API Code to Match Schema (Not Vice Versa)
**Type:** Implementation
**Topics:** d1, testing

**Trigger:** Tests revealed `video_sessions` table doesn't exist - API endpoints used wrong table name.

**Options Considered:**
1. Fix API endpoints to use correct `sessions` table and column names ← Chosen
2. Create `video_sessions` view as alias
3. Rename `sessions` table to `video_sessions`

**Decision:** Update API endpoints (`sessions.ts`, `st-performance.ts`) to use the existing `sessions` table with its actual column names.

**Rationale:**
- The `sessions` table already has all needed fields (including BBB video fields)
- Creating aliases adds unnecessary complexity
- Renaming table would break other code
- Schema is the source of truth during development

**Consequences:**
- Fixed column mappings: `scheduled_at` → `scheduled_start`, `actual_duration_minutes` → `duration_minutes`
- Fixed table name: `video_sessions` → `sessions`
- Fixed FK references: `st_id` → `student_teacher_id`, join on `user_id`
- All CANA API tests now pass (40/40)
