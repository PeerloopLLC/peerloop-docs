# Session Learnings - 2026-01-08

## Technical Learnings

### 1. SQLite Column Name Mismatch Debugging
**Context:** Teacher Dashboard "My Students" section showed empty despite correct data in database.

**Problem:** Query used `mp.status = 'completed'` but actual column is `is_complete` (INTEGER 0/1). The `.catch(() => [])` silently swallowed the SQL error.

**Solution:** Changed to `AVG(mp.is_complete) * 100` which correctly calculates progress percentage.

**Lesson:** When queries silently return empty arrays, check:
1. Column names match actual schema (use `PRAGMA table_info(tablename)`)
2. Remove `.catch()` temporarily to see actual errors during debugging

### 2. D1 Remote-Only Development (MBA-2017)
**Context:** macOS 12 doesn't support local D1/R2 emulation (requires macOS 13.5+).

**Pattern:** Always use `--remote` flag for wrangler D1 commands:
```bash
npx wrangler d1 execute peerloop-db --remote --file=migrations/0009_teacher_dashboard.sql
```

### 3. Missing Table Graceful Handling
**Context:** API referenced tables that didn't exist yet (`user_availability`, `intro_sessions`, `certificates`).

**Pattern:** Add `.catch()` handlers to queries that reference optional/future tables:
```typescript
queryFirst<Row>(db, sql, [params]).catch(() => ({ default_value: 0 }))
```

Then create proper migration when ready to implement the feature.

## Process Learnings

### 1. Verification Before Marking Complete
Running `/q-verify-page` after implementation catches:
- Data display bugs (like My Students empty)
- Missing API integrations (buttons that are visual-only)
- Broken links (like `/teachers/me` which doesn't exist)

### 2. Seed Data Timing
When screenshots show unexpected empty states, check if:
1. Migration was run after the API was called
2. Data exists in DB but query has bugs
3. JOIN conditions are failing silently
