## 1. Centralized Email Service Pattern for Preference Checking
**Topics:** astro, workflow

**Context:** Implementing email notifications that respect user opt-out preferences required checking database preferences before sending each email.

**Learning:** Wrapping Resend in a centralized email service (`src/lib/email.ts`) that accepts user ID and preference type allows automatic preference checking before any email is sent. The service maps notification types to database columns and short-circuits if the user has opted out.

**Pattern:**
```typescript
// Map notification types to user preference columns
const EMAIL_PREFERENCE_COLUMNS: Record<EmailPreferenceType, string> = {
  session_reminder: 'email_session_reminder',
  session_booked: 'email_session_booked',
  // ...
};

async function sendEmailIfAllowed(db, apiKey, userId, preferenceType, params) {
  const isAllowed = await checkEmailPreference(db, userId, preferenceType);
  if (!isAllowed) return { success: false, error: 'User opted out' };
  return sendEmail(apiKey, params);
}
```

## 2. Non-Blocking Notification Triggers in API Endpoints
**Topics:** astro, workflow

**Context:** Adding notification triggers to existing API endpoints (session booking, certificate approval, payout processing) without affecting response latency.

**Learning:** Wrapping notification sends in `Promise.all([...]).catch(err => console.error(...))` allows them to run in parallel and fail silently without blocking the main response. The user gets immediate feedback while notifications happen asynchronously.

**Pattern:**
```typescript
// After successful operation, trigger notifications non-blocking
Promise.all([
  notifySessionBooked(db, teacherId, ...),
  resendApiKey ? sendEmailToUser(db, resendApiKey, teacherId, 'session_booked', {...}) : Promise.resolve(),
]).catch((err) => console.error('Failed to notify:', err));

return Response.json({ success: true }); // Returns immediately
```

## 3. Shared Email Styles for React Email Templates
**Topics:** workflow

**Context:** Creating 6+ email templates with consistent Peerloop branding.

**Learning:** Extracting shared styles into a separate `styles.ts` file allows all email templates to import the same base styles (colors, typography, buttons, spacing) while remaining individually customizable. React Email uses inline styles for maximum email client compatibility.

**Pattern:**
```typescript
// src/emails/styles.ts
export const button = {
  backgroundColor: '#4F46E5',
  borderRadius: '8px',
  color: '#fff',
  // ... inline styles for email compatibility
};

// Template usage
import { button, paragraph } from './styles';
<Button style={button}>Click Here</Button>
```

## 4. Type-Specific Notification Icons with Color Mapping
**Topics:** astro

**Context:** Building a notification list component that visually distinguishes different notification types.

**Learning:** Creating a constant object that maps notification types to both SVG path data and Tailwind color classes allows the component to render type-appropriate icons without conditional logic chains. The fallback to a "system" type handles unknown types gracefully.

**Pattern:**
```typescript
const TYPE_ICONS: Record<string, { icon: string; color: string }> = {
  session_booked: {
    icon: 'M8 7V3m8 4V3m...', // calendar icon path
    color: 'text-blue-500 bg-blue-50',
  },
  certificate_earned: {
    icon: 'M9 12l2 2 4-4...', // badge icon path
    color: 'text-green-500 bg-green-50',
  },
  // ...
};

const typeStyle = TYPE_ICONS[notification.type] || TYPE_ICONS.system;
```

## 5. Email Preference Columns as Database Flags
**Topics:** d1

**Context:** Users need granular control over which email types they receive.

**Learning:** Adding boolean columns directly to the users table (e.g., `email_session_reminder`, `email_marketing`) is simpler than a separate preferences table for MVP. Defaults can be set in the migration (most default to 1/true, marketing defaults to 0/false), and checking is a simple column read.

**Pattern:**
```sql
ALTER TABLE users ADD COLUMN email_session_reminder INTEGER NOT NULL DEFAULT 1;
ALTER TABLE users ADD COLUMN email_marketing INTEGER NOT NULL DEFAULT 0;
```
