# Session Learnings - 2026-01-19

## 1. getstream Node.js SDK Incompatible with Cloudflare Workers Runtime
**Topics:** stream, cloudflare, deployment

**Context:** Community feed feature worked locally on both MBA-2017 (Node adapter) and MacMini (Cloudflare adapter), but failed with 503 error when deployed to Cloudflare Pages.

**Learning:** The `getstream` Node.js SDK uses `http.Agent` which is not available in Cloudflare Workers/Pages runtime. Even with `nodejs_compat_v2` and `enable_nodejs_http_modules` compatibility flags enabled, Cloudflare's `http.Agent` is a stub implementation that doesn't satisfy the SDK's constructor requirements. The SDK is fundamentally incompatible with edge runtimes.

**Pattern:** When a Node.js library fails on Cloudflare with "X is not a constructor" errors, check if it uses Node-specific APIs like `http.Agent`, `net.Socket`, or `fs`. The solution is to replace the SDK with direct REST API calls using `fetch()`.

## 2. Debug Production Errors by Returning Diagnostic Info in API Response
**Topics:** cloudflare, deployment, workflow

**Context:** Stream.io integration returned 503 error on Cloudflare Pages. Initial assumption was missing environment variables, but after verifying they were set, needed to identify the actual error.

**Learning:** When debugging production-only errors, temporarily modify the API to return diagnostic information in the error response. This approach:
1. First returned `envKeys` array to verify env vars existed
2. Then returned `actualError` and `errorStack` to reveal the true `http.Agent` constructor error

**Pattern:**
```typescript
return Response.json({
  error: 'Service not configured',
  debug: {
    envKeys: Object.keys(locals.runtime?.env || {}),
    hasApiKey: !!locals.runtime?.env?.API_KEY,
    actualError: error instanceof Error ? error.message : String(error),
    errorStack: error instanceof Error ? error.stack : undefined,
  }
}, { status: 503 });
```

## 3. Cloudflare Pages Compatibility Flags Must Be Set in Dashboard
**Topics:** cloudflare, deployment

**Context:** Added `enable_nodejs_http_modules` to `wrangler.toml` but the flag wasn't applied to Cloudflare Pages deployments via GitHub integration.

**Learning:** For Cloudflare Pages deployed via git integration, compatibility flags in `wrangler.toml` are applied to local development (workerd) but may need to be separately configured in the Cloudflare Dashboard under Workers & Pages → Project → Settings → Functions → Runtime. Preview and Production environments have separate settings.

## 4. Local Cloudflare Adapter vs Production Runtime Differ
**Topics:** cloudflare, deployment, astro

**Context:** Code worked on MacMini with Cloudflare adapter in dev mode but failed on actual Cloudflare Pages deployment.

**Learning:** Local development with `wrangler dev` / workerd provides a more complete Node.js compatibility layer than the actual Cloudflare Pages production runtime. Code that works locally with the Cloudflare adapter may still fail in production if it uses Node-specific APIs. Always test Node.js library compatibility by deploying to a preview environment, not just local dev.

## 5. Stream.io REST API as Edge-Compatible Alternative
**Topics:** stream, cloudflare

**Context:** Needed to make Stream.io Activity Feeds work on Cloudflare Pages after SDK proved incompatible.

**Learning:** Stream.io's REST API can be called directly using `fetch()` with JWT authentication. The `jose` library (already installed for other JWT needs) works in edge environments and can sign Stream API tokens. The REST API supports all needed operations: feed get/add activities, reactions add/delete/filter.

**Pattern:**
```typescript
// Generate server token with jose
const token = await new jose.SignJWT({
  resource: '*',
  action: '*',
  feed_id: '*',
})
  .setProtectedHeader({ alg: 'HS256', typ: 'JWT' })
  .setIssuedAt()
  .sign(new TextEncoder().encode(apiSecret));

// Call Stream REST API
const response = await fetch(`${STREAM_BASE_URL}${endpoint}?api_key=${apiKey}`, {
  headers: {
    'Content-Type': 'application/json',
    'Stream-Auth-Type': 'jwt',
    'Authorization': token,
  },
});
```

## 6. wrangler pages secret put for Environment Variables
**Topics:** cloudflare, deployment

**Context:** Needed to add Stream.io API credentials to Cloudflare Pages production environment.

**Learning:** Use `wrangler pages secret put VARIABLE_NAME` to add secrets to Cloudflare Pages. Run interactively to paste values when prompted. Use `wrangler pages secret list` to verify configured secrets. Secrets are stored encrypted and require a new deployment to take effect.
