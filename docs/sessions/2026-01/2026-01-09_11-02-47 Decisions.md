# Session Decisions - 2026-01-09_11-02-47

## Decision 1: Pin Node.js to Exact Version

**Context:** Working across two machines (MacMini and MBA-2017) with potential for version drift.

**Decision:** Pin Node.js to exact version 22.19.0 in both `.nvmrc` and `package.json` engines field.

**Rationale:**
- Semver ranges (^, ~) allow drift between machines
- Exact version ensures identical runtime on all development machines
- `engines` field warns developers if wrong version is used

---

## Decision 2: Remove Wrangler from package.json

**Context:** MBA-2017 cannot run wrangler 4.58.0 due to macOS 12 limitations. Having wrangler in devDependencies causes package-lock.json conflicts between machines.

**Decision:** Remove wrangler from devDependencies entirely; rely on global installs.

**Rationale:**
- Wrangler is a CLI tool, never imported by application code
- Each machine can install whatever version is compatible with its OS
- Eliminates package-lock.json drift for wrangler versions
- npm scripts using wrangler still work (finds global install)

**Trade-offs:**
- Must ensure wrangler is globally installed on each machine
- Version differences between machines (acceptable since it's dev tooling only)

---

## Decision 3: Symlink .env to .dev.vars

**Context:** Cloudflare/Wrangler uses `.dev.vars` for local environment variables. The dotenv package looks for `.env`.

**Decision:** Create `.env` as a symlink to `.dev.vars` on all development machines.

**Rationale:**
- Single source of truth for environment variables
- Both Wrangler and dotenv package work with same file
- Changes to either file automatically reflected in both
