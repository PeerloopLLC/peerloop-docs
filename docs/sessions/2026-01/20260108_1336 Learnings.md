# Session Learnings - 2026-01-08

## Technical Learnings

### 1. Database Schema Status Field Inconsistency

**Discovery:** When implementing Admin Dashboard (ADMN), SQL queries failed with "no such column: status" because different tables use different patterns for status-like fields.

**Pattern Variations:**
| Pattern | Tables | Example |
|---------|--------|---------|
| Boolean flags | `courses`, `users` | `is_active`, `suspended_at`, `is_retired` |
| Enum status | `enrollments`, `transactions` | `status IN ('enrolled', 'completed')` |
| Badge field | `courses` | `badge IN ('featured', 'popular')` |

**Impact:** Boolean flags imply multiple statuses can coexist, while enum status implies mutual exclusivity. This caused bugs assuming consistent `status` columns.

**Resolution:** Fixed queries to use actual column names. Added to PLAN.md Technical Debt section for future standardization review.

### 2. requireRole() Throws Response Objects, Not Errors

**Discovery:** Admin dashboard API returned 500 errors instead of proper 403/302 responses.

**Root Cause:** The `requireRole()` function from `@lib/auth` throws a `Response` object (for redirects and 403), but the catch block was treating it as an Error and returning 500.

**Fix Pattern:**
```typescript
} catch (error) {
  // Re-throw auth redirects/errors (Response objects from requireRole)
  if (error instanceof Response) {
    return error;
  }
  // Handle actual errors
  console.error('Error:', error);
  return Response.json({ error: 'Failed' }, { status: 500 });
}
```

**Applied To:** All admin API endpoints should use this pattern.

### 3. Transactions Table Uses amount_cents, Not amount

**Discovery:** Revenue stats query failed because transactions table uses `amount_cents` column, not `amount`.

**Key Fields:**
- `amount_cents` (INTEGER) - Amount in cents
- `status` IN ('pending', 'completed', 'refunded', 'partially_refunded', 'failed')
- No `type` column - all rows are payment records

### 4. Enrollments Status Values

**Discovery:** Enrollments use specific status values, not 'active'.

**Actual Values:** `'enrolled'`, `'in_progress'`, `'completed'`, `'cancelled'`

**Active Query:** `status IN ('enrolled', 'in_progress')` not `status = 'active'`

## Process Learnings

### 1. JSON Spec Schema v1.1 Requirements

When creating PageSpec JSON files, the schema requires specific structures:

- `interactiveElements` must be an object with `buttons`, `links`, `targetPages`, `summary`
- `verificationNotes` must be an object with `verifiedDate`, `consolidatedDate`, `components`, `screenshots`, `notes`
- `_discrepancies` requires `plannedButNotImplemented`, `implementedDifferently`, `removedFromSpec` arrays
