# Session Learnings - 2026-01-06

## Technical

### Admin Courses API Pattern
- Created comprehensive admin API at `src/pages/api/admin/courses/`
- Pattern: main `index.ts` for list/create, `[id].ts` for detail/update/delete
- Action-specific endpoints: `[id]/feature.ts`, `[id]/suspend.ts`, `[id]/badge.ts`
- All endpoints use `requireRole(cookies, jwtSecret, ['admin'])` for auth

### CoursesAdmin Component
- Reused shared components: AdminDataTable, AdminFilterBar, AdminDetailPanel
- Added stats cards pattern for quick metrics display
- Badge selector allows quick promotional badge changes without modal

### Soft Delete Protection
- Courses with enrollments cannot be deleted
- Check enrollment count before allowing delete action
- Return helpful error message with count

## Documentation

### PAGES-MAP Refactoring Pattern
- **Problem**: PAGES-MAP.md grew to 820+ lines, becoming unwieldy
- **Solution**: Split into individual `docs/pages/CODE.md` files
- **PAGES-MAP.md** becomes slim lookup table (~140 lines):
  - Quick stats
  - Lookup table with Details column linking to individual files
  - Implemented pages by block
  - User journeys
  - File locations
- **Individual files** contain:
  - Metadata table (route, component, status, etc.)
  - Features grouped by category with `[US-XXXX]` story tags
  - User Stories Covered table
  - Gap Analysis (features without stories)
  - API Endpoints table
  - Implementation notes

### User Story ID Sources
- **SCOPE.md** has page-specific story IDs (US-A008, US-A009, etc.)
- **stories-A-admin.md** has different IDs with different content
- Use SCOPE.md IDs when mapping features to pages - they're designed for this purpose

### Pre-Refactor Preservation
- Before major doc refactoring, save original to `.claude/` folder
- Command: `git show HEAD:FILENAME > .claude/FILENAME-original.md`
- This survives commits and enables recovery of detailed content

## Workflow

### Feature-to-Story Traceability
- Tag each feature with user story IDs: `- [x] Feature name [US-XXXX]`
- Include "Gap Analysis" section for features lacking stories
- Suggests new stories that should be added to backlog

### Document-as-Built vs Snapshot
- **Document-as-built** is more accurate for feature tracking
- During implementation, developer knows intent and edge cases
- Reading code later requires inferring this from structure
- Best practice: Add features to page docs as you implement them
