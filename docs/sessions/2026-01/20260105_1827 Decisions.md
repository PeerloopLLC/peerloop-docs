## Decision 3: Use File URL for Homework Attachments

**Type**: Implementation
**Driver**: Claude-driven
**Working On**: SubmissionForm component design
**Trigger**: R2 bucket wasn't available at implementation start

**Options Considered**:
1. Implement full R2 file upload with drag-drop UI
2. Use file URL field (links to Google Drive, Dropbox, etc.) <- **Chosen**
3. Text-only submissions (no file support)

**Decision**: Use a URL input field for file attachments rather than direct R2 upload.

**Rationale**:
- R2 wasn't enabled when implementation started
- URL approach works immediately without additional infrastructure
- Students commonly share work via Google Drive/Dropbox anyway
- R2 upload UI can be added as enhancement later

**Consequences**:
- `file_url` field stores external URLs, not R2 keys
- Simpler implementation with no file validation needed
- R2 still available for future direct upload enhancement
- Schema supports both approaches (field is just TEXT)
