## Decision 1: Fix All API Endpoints for D1 REST Fallback

**Type**: Implementation
**Driver**: Claude-driven (with user approval)
**Working On**: Investigating "Database not available" error on CDET page

## Decision 2: Add process.env Fallbacks for Environment Variables

**Type**: Implementation
**Driver**: Claude-driven
**Working On**: D1 REST API fallback fix
**Trigger**: Node adapter doesn't populate `locals.runtime?.env`

**Options Considered**:
1. Only use `locals.runtime?.env` (original pattern)
2. **Add `|| process.env.X` fallback** <- **Chosen**
3. Load all env vars via dotenv at app startup

**Decision**: Add `process.env` fallback for all environment variables accessed in API endpoints.

**Rationale**:
- Simple one-line change per variable
- Works with both Cloudflare adapter (production/Mac Mini) and Node adapter (MBA-2017)
- `.dev.vars` is already loaded by dotenv in the d1-rest module
- No architectural changes needed

**Consequences**:
- Pattern: `locals.runtime?.env?.JWT_SECRET || process.env.JWT_SECRET`
- Applied to JWT_SECRET, STRIPE_WEBHOOK_SECRET, OAuth credentials
- Development on MBA-2017 now fully functional
