# Session Learnings - 2026-01-22

## 1. Recharts Chart Wrapper Pattern for Future Migration
**Topics:** astro, workflow

**Context:** Implementing charts for Creator Analytics (CANA) with potential future migration to Tremor or Nivo.

**Learning:** Creating thin wrapper components around Recharts allows swapping implementations later without changing consumer code. Each wrapper (AreaChart, BarChart, PieChart, FunnelChart) takes standardized props and handles library-specific implementation internally.

**Pattern:**
```typescript
// Wrapper accepts standard props
interface AreaChartProps {
  data: TimeSeriesData[];
  dataKeys: { key: string; label: string; color?: string }[];
  config?: ChartConfig;
}

// Library-specific code stays in wrapper
export function AreaChart({ data, dataKeys, config }: AreaChartProps) {
  return (
    <ResponsiveContainer>
      <RechartsArea data={data}>
        {/* Recharts internals */}
      </RechartsArea>
    </ResponsiveContainer>
  );
}
```

## 2. TypeScript Type Narrowing After Error Check Still Requires Property Check
**Topics:** workflow

**Context:** Fetching data from multiple API endpoints with union types `{ data: T } | { error: string }`.

**Learning:** Even after checking `'error' in data`, TypeScript doesn't narrow the type sufficiently to access `data.propertyName`. Need additional property check: `if ('metrics' in data) setMetrics(data.metrics)`.

**Pattern:**
```typescript
// Not enough:
if ('error' in data) throw new Error(data.error);
setSummary(data.metrics); // TS error: Property 'metrics' does not exist

// Need:
if ('error' in data) throw new Error(data.error);
if ('metrics' in data) setSummary(data.metrics); // Works
```

## 3. Recharts Formatter Callbacks Have Permissive Types
**Topics:** workflow

**Context:** TypeScript errors when adding formatters to Recharts Tooltip and Legend components.

**Learning:** Recharts formatter callbacks receive `value` and `name` as `ValueType` which can be various types. Cast or check types inline rather than annotating parameters.

**Pattern:**
```typescript
// Instead of: formatter={(value: number, name: string) => ...}
formatter={(value, name) => {
  const numValue = typeof value === 'number' ? value : 0;
  const strName = String(name);
  return [numValue, strName];
}}
```

## 4. Analytics Page Hierarchy: Admin > Creator > ST
**Topics:** workflow

**Context:** Planning analytics pages for different user roles.

**Learning:** Analytics pages have natural hierarchy by scope:
- **AANA (Admin)**: Platform-wide metrics, all users/courses/revenue
- **CANA (Creator)**: Creator's courses, their S-Ts, their revenue share
- **TANA (S-T)**: Personal teaching metrics only

Components and patterns flow downward - CANA patterns inform TANA implementation.

## 5. FunnelChart Not Native to Recharts
**Topics:** workflow

**Context:** Needed funnel visualization for conversion metrics.

**Learning:** Recharts doesn't have native funnel chart support. Implemented custom horizontal bar funnel using Tailwind CSS with percentage-based widths. Simple and works well for the use case.

**Pattern:**
```tsx
// Custom funnel as horizontal bars
{data.map((step) => (
  <div className="flex items-center gap-3">
    <span className="w-28">{step.name}</span>
    <div className="flex-1 bg-secondary-100 rounded-full">
      <div
        className="h-full rounded-full"
        style={{ width: `${step.widthPercent}%` }}
      />
    </div>
    <span>{step.value}</span>
  </div>
))}
```
