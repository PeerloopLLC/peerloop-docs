# Session Decisions - 2026-01-08_19-14-36

## Decision 1: Implement Comparison Section Beyond Spec

**Context:** The original pricing.json spec had "Consider showing comparison with traditional tutoring costs" in the notes section as a suggestion, not a requirement.

**Decision:** Implemented the comparison section as a full dedicated section with:
- Professional Tutor: $50-150/hour
- Tutoring Center: $30-80/hour
- Peerloop: $20-50/hour (highlighted with "Best Value" badge)

**Rationale:** This section adds significant value to the pricing page by showing Peerloop's price advantage. It was a suggestion that made sense to implement.

**Documented:** Added to `_discrepancies.implementedDifferently` in pricing.json

## Decision 2: Omit PayPal from Payment Methods

**Context:** Original spec listed "PayPal (coming soon)" in the Payment Methods section.

**Decision:** Did not show PayPal at all on the implemented page (no "coming soon" mention).

**Rationale:** Showing "coming soon" for features that aren't ready can create customer expectations. Better to add PayPal when it's actually ready.

**Documented:** Added to `_discrepancies.plannedButNotImplemented` with priority "Low - marked as coming soon"

## Decision 3: Keep Analytics Events in Discrepancies

**Context:** Original spec defined 3 analytics events (page_view, calculator_used, cta_click) but analytics infrastructure isn't implemented yet.

**Decision:** Moved analytics events from `analyticsEvents[]` to `_discrepancies.plannedButNotImplemented`.

**Rationale:** The JSON spec should reflect current reality. Analytics events are planned but not implemented, so they belong in discrepancies rather than the main spec.

## Decision 4: Remove sampleContent from JSON Spec

**Context:** Original spec had extensive `sampleContent` section with copy text for each section.

**Decision:** Removed sampleContent during consolidation.

**Rationale:** This was planning content used when building the component. Now that the component is built with hardcoded content, the sampleContent is redundant and clutters the spec.
