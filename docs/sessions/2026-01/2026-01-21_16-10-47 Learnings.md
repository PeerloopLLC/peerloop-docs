# Session Learnings - 2026-01-21

## 1. Orphaned Vite Polyfills Can Break Unrelated Features
**Topics:** astro, cloudflare, d1

**Context:** MBA-2017 D1 REST fallback was failing because `process.env` access returned `undefined`. The `'process.env': '{}'` Vite define in `astro.config.mjs` was replacing all `process.env` references with an empty object at bundle time.

**Learning:** When removing a package that required polyfills, always check if the polyfills are still needed. The `getstream` SDK was removed and replaced with a custom REST client, but the `'process.env': '{}'` polyfill that was added for it was never cleaned up. This orphaned code broke the MBA-2017 D1 REST fallback months later.

**Pattern:**
```javascript
// Vite define replaces at bundle time - affects ALL code
define: {
  'process.env': '{}',  // Breaks runtime process.env access everywhere
}
```

## 2. Use getEnv Helper Consistently for MBA-2017 Compatibility
**Topics:** cloudflare, d1, astro

**Context:** The `getStreamConfig()` function directly accessed `locals.runtime?.env?.STREAM_API_KEY` which only works with Cloudflare Workers runtime (MacMini/Production), not on MBA-2017 with Node adapter.

**Learning:** Always use the `getEnv(locals, key)` helper from `src/lib/env.ts` when accessing environment variables in server-side code. This helper:
1. First tries `locals.runtime.env` (Cloudflare Workers)
2. Falls back to `process.env` only when `ENABLE_D1_REST_FALLBACK=true` AND not production

**Pattern:**
```typescript
// Bad - only works on Cloudflare Workers runtime
const apiKey = locals.runtime?.env?.STREAM_API_KEY;

// Good - works on both MacMini/Production and MBA-2017
import { getEnv } from './env';
const apiKey = getEnv(locals, 'STREAM_API_KEY');
```

## 3. Git Log with -S Flag Finds Code Addition/Removal
**Topics:** workflow

**Context:** Needed to find when and why `'process.env': '{}'` was added to the codebase.

**Learning:** Use `git log -S "string"` to find commits that added or removed a specific string. Combined with `--all` to search all branches. This is more precise than `--grep` which only searches commit messages.

**Pattern:**
```bash
# Find commits that added/removed a specific string
git log --all --oneline -S "'process.env': '{}'"

# Find commits mentioning something in the message
git log --all --oneline --grep="getstream"
```

## 4. Resend Requires Domain Verification for Custom From Addresses
**Topics:** deployment

**Context:** Moderator invite API succeeded but email failed with "Invalid `from` field" error when using `Peerloop <noreply@peerloop.co>`.

**Learning:** Resend requires domain verification before sending from custom domains. For development/testing, use `onboarding@resend.dev` as the from address. For production, verify the domain in Resend dashboard first.
