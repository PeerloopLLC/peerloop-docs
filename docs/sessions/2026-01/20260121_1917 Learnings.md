## 1. API Response Type Safety Pattern for Fetch Error Handling
**Topics:** astro, workflow

**Context:** When creating React components that fetch from API endpoints, TypeScript complained about `result` being `unknown` type when calling `response.json()`.

**Learning:** Always define explicit interface types for API responses and type the result of `response.json()` calls. Create separate interfaces for error responses (`ApiErrorResponse`) and success responses.

**Pattern:**
```typescript
interface ApiErrorResponse {
  error?: string;
}

interface SuccessResponse {
  data: SomeType;
  message?: string;
}

// In fetch handler:
if (!response.ok) {
  const errorResult: ApiErrorResponse = await response.json();
  throw new Error(errorResult.error || 'Default error message');
}

const result: SuccessResponse = await response.json();
```

## 3. SQL Query Pattern for Progress Percentage Calculation
**Topics:** d1, astro

**Context:** Calculating student progress from module_progress table required counting completed modules vs total modules per enrollment.

**Learning:** Use subqueries in SELECT clause for related counts, and handle division-by-zero with COALESCE. Calculate percentage in application code for flexibility.

**Pattern:**
```sql
SELECT
  COALESCE((SELECT COUNT(*) FROM module_progress mp
            WHERE mp.enrollment_id = e.id AND mp.is_complete = 1), 0) as modules_completed,
  COALESCE((SELECT COUNT(*) FROM course_curriculum cc
            WHERE cc.course_id = c.id), 0) as modules_total
FROM enrollments e
JOIN courses c ON e.course_id = c.id
```

Then in TypeScript:
```typescript
const modulesTotal = s.modules_total || 1; // Avoid division by zero
const progressPercent = Math.round((s.modules_completed / modulesTotal) * 100);
```

## 4. Period-Based Date Range Filtering Pattern
**Topics:** d1, astro

**Context:** The earnings API needed to filter by different time periods (this month, last month, this year, all time).

**Learning:** Create a helper function that returns start/end dates based on period string, then use these in SQL queries with `>=` and `<=` operators on ISO date strings.

**Pattern:**
```typescript
function getPeriodDates(period: string): { start: string; end: string } {
  const now = new Date();
  const year = now.getFullYear();
  const month = now.getMonth();

  switch (period) {
    case 'last_month': {
      const lastMonth = month === 0 ? 11 : month - 1;
      const lastMonthYear = month === 0 ? year - 1 : year;
      const start = new Date(lastMonthYear, lastMonth, 1);
      const end = new Date(year, month, 0, 23, 59, 59); // Last day of last month
      return { start: start.toISOString(), end: end.toISOString() };
    }
    // ... other cases
  }
}
```

## 5. Sortable Table Pattern with URL State
**Topics:** astro, workflow

**Context:** The MyStudents component needed sortable columns that maintain sort state across re-renders.

**Learning:** Use React state for sort field and order, pass to API as query params, and create a SortButton component that shows the current sort direction and handles click to toggle.

**Pattern:**
```typescript
function SortButton({ field, currentSort, currentOrder, onClick }) {
  const isActive = currentSort === field;
  return (
    <button onClick={() => onClick(field)}>
      {label}
      {isActive && <ChevronIcon rotated={currentOrder === 'desc'} />}
    </button>
  );
}

function handleSort(field: SortField) {
  if (sort === field) {
    setOrder(order === 'asc' ? 'desc' : 'asc');
  } else {
    setSort(field);
    setOrder('desc');
  }
  setPage(1); // Reset to first page
}
```
