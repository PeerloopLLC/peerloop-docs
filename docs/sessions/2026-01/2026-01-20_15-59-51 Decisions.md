# Session Decisions - 2026-01-20

## 1. Dedicated Stream Feed Groups for Course and Instructor
**Type:** Implementation
**Topics:** stream

**Trigger:** Reviewing Stream Dashboard configuration and realizing all feeds were in the `townhall` group with ID prefixes.

**Options Considered:**
1. Keep using `townhall` group with ID prefixes (`townhall:course-{id}`, `townhall:instructor-{id}`)
2. Create dedicated `course` and `instructor` feed groups ← Chosen

**Decision:** Create dedicated `course` and `instructor` feed groups in Stream Dashboard, each configured as flat type with realtime notifications enabled.

**Rationale:**
- Cleaner feed IDs (`course:abc123` instead of `townhall:course-abc123`)
- Each group can be configured independently in the future
- Eliminates need for client-side filtering in instructor feed reader
- Better separation of concerns

**Consequences:**
- Added `course` and `instructor` groups in Stream Dashboard (flat type, realtime enabled)
- Updated 4 API files to use new groups
- Updated `tech-002-stream.md` documentation
- Feed patterns: `course:{courseId}`, `instructor:{userId}`

## 3. Use Flat Feed Type for Course and Instructor Feeds
**Type:** Implementation
**Topics:** stream

**Trigger:** When creating feed groups in Stream Dashboard, needed to choose between flat, aggregated, and notification types.

**Options Considered:**
1. **Flat** - Chronological list of activities ← Chosen
2. **Aggregated** - Groups similar activities ("3 new posts today")
3. **Notification** - Special type with read/unread tracking

**Decision:** Use flat type for both `course` and `instructor` feed groups.

**Rationale:**
- Discussion forums work best as chronological threads
- Creator updates should be seen individually (like Twitter)
- Aggregated adds complexity in display logic
- Genesis Cohort scale (60-80 students) doesn't need aggregation
- Can change later if high-volume posting becomes an issue

**Consequences:**
- Both groups configured as flat in Stream Dashboard
- Simple chronological display in feed components
