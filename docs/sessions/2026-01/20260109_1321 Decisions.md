# Session Decisions - 2026-01-09

## Decision 1: Direct D1 Access Pattern

**Context:** Pages were using self-fetch pattern (`fetch(\`\${origin}/api/...\`)`) to get data, which worked but was architecturally messy.

**Decision:** Standardize all D1-accessing pages to use direct `getDB()` calls instead of self-fetch.

**Rationale:**
- Eliminates unnecessary HTTP round-trip
- Avoids nested fetch on MBA-2017 (fetch → API → getDB → REST)
- Single consistent pattern across all environments
- Cleaner error handling with try-catch at the page level

**Implementation:**
- All 10 D1-accessing pages now use direct pattern
- API endpoints remain for client-side fetching (React components)

## Decision 2: Defensive Module-Level Code

**Context:** Module-level `dotenv.config()` in `d1-rest.ts` crashed on Cloudflare Workers.

**Decision:** Wrap all module-level environment/package loading in try-catch.

**Rationale:**
- Module code runs in all environments that import it
- Can't assume packages (like dotenv) are available everywhere
- Silent failure is acceptable when the fallback isn't needed anyway

## Decision 3: Test on Actual Target Machine

**Context:** Attempted to emulate MBA-2017 behavior on MacMini by forcing Node adapter.

**Decision:** Test environment-specific features on the actual target machine rather than emulating.

**Rationale:**
- Emulation introduces additional complexity (ESM/CJS differences)
- Direct testing is more reliable and catches real issues
- User confirmed MBA-2017 testing was straightforward
