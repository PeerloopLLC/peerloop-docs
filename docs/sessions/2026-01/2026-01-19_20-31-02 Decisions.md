# Session Decisions - 2026-01-19

## 1. Replace getstream SDK with Custom REST API Client
**Type:** Implementation
**Topics:** stream, cloudflare

**Trigger:** getstream Node.js SDK failed on Cloudflare Pages with `http.Agent is not a constructor` error.

**Options Considered:**
1. Add more Cloudflare compatibility flags ← Tried, didn't work
2. Use a different Stream.io SDK (browser/edge version)
3. Replace SDK with direct REST API calls using fetch() ← Chosen
4. Move Stream functionality to an external service

**Decision:** Replace the getstream SDK with a custom REST API client that uses `fetch()` and `jose` for JWT authentication.

**Rationale:**
- The getstream SDK is fundamentally incompatible with edge runtimes (uses Node.js http.Agent)
- Stream.io's REST API is well-documented and supports all needed operations
- `jose` library is already installed and edge-compatible
- Custom client maintains same interface for minimal code changes

**Consequences:**
- Removed 38 packages (getstream and dependencies)
- Created new `src/lib/stream.ts` with StreamClient, StreamFeed, and StreamReactions classes
- Code changes limited to stream.ts and token.ts - API endpoints unchanged
- Community feed now works on Cloudflare Pages production
