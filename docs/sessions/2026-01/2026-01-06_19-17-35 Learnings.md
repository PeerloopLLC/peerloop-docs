# Session Learnings - 2026-01-06

## Technical

### User Story ID Collision Between SCOPE.md and Canonical Stories
- **Issue:** `research/run-001/SCOPE.md` created placeholder admin story IDs (US-A001-A024) that collided with canonical stories in `research/stories/stories-A-admin.md` which had different content
- **Example:** SCOPE.md had "US-A008 = View all courses" but canonical had "US-A008 = Allow third party organizations to pay for students"
- **Resolution:** Added 21 new canonical stories (US-A040 through US-A060) to cover admin page requirements, then updated SCOPE.md to reference these new IDs
- **Prevention:** When creating page-derived stories, always check canonical files first; use next available ID, not arbitrary numbers

### USER-STORIES-MAP.md as Master Index
- Created `USER-STORIES-MAP.md` in project root as a consolidated index of all 391 user stories
- Organized by role with links back to source files in `/research/stories/`
- Serves as quick reference for page-to-story assignment work
- Total stories increased from 370 to 391 after adding admin page stories

## Documentation

### Page Documentation Merge Pattern
- **Before:** Two overlapping doc sets - `research/run-001/pages/page-*.md` (design specs) and `docs/pages/*.md` (implementation tracking)
- **After:** Merged into single `docs/pages/*.md` files containing both spec details AND implementation status
- **Key pattern:** Added "Plan vs Implementation" section to track planned features not yet implemented
- **Key pattern:** Added Status columns (✅/⏳) to tables to show what's done vs pending
- **Action:** Delete research page files after merge to avoid confusion

### Story Assignment Format in Page Docs
- Features tagged with story IDs: `- [x] Feature description \`[US-A047]\``
- User Stories section with full table: ID, Story, Priority, Status
- Gap Analysis section for features lacking formal story coverage

## Workflow

### Canonical Story Management
- When adding stories from a new source (like SCOPE.md page requirements), add them to the role-specific file (e.g., `stories-A-admin.md`)
- Update `USER-STORIES.md` index with new counts and next available number
- Update `USER-STORIES-MAP.md` for the master consolidated view
- Add source to the Source Document Index table
