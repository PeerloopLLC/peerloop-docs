# Session Learnings — 2026-01-27

## Session 129: Strengthen CANA Tests (Items 11-17)

### L1: Chart component mocking pattern for child component tests
When testing components that render chart libraries (AreaChart, BarChart, PieChart, FunnelChart), mock the individual chart module with a div that exposes `data-testid` and `data-count` attributes. This avoids canvas/SVG rendering issues while still verifying data transformation logic. Pattern from STAnalytics.test.tsx, replicated across all 7 CANA child tests.

```tsx
vi.mock('@components/charts/BarChart', () => ({
  BarChart: ({ data }: { data: unknown[] }) => (
    <div data-testid="bar-chart" data-count={data.length}>Mock</div>
  ),
}));
```

### L2: `alt=""` images don't match `getByRole('img')`
In testing-library, images with empty `alt=""` (decorative images per ARIA spec) are not findable via `getByRole('img')`. Use `container.querySelector('img[src="..."]')` instead. Hit this with CoursePerformanceTable thumbnail test.

### L3: Standalone child tests catch formatting bugs orchestrator tests miss
The orchestrator test checks one text node per child (e.g., "$1,250" for MetricsRow). Standalone child tests verify all 5 metric cards, null-rating N/A fallback, positive/negative change indicators, and loading skeleton count. These would catch: currency formatting bugs, missing % suffixes, wrong card count in loading state.

### L4: Sort callback toggle logic needs explicit testing
CoursePerformanceTable's `handleSort` has non-obvious behavior: same column → flip direction, different column → default desc. Testing both paths in the standalone test caught this clearly, whereas the orchestrator only verifies one sort change.
