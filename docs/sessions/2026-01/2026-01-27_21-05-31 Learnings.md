# Session Learnings - 2026-01-27

## 1. Mermaid Diagrams Need Outbound-Only Edge Filtering for Focused Views
**Topics:** workflow, astro

**Context:** Generated a full site interconnection Mermaid diagram with 60 pages and 202 edges. The diagram was too dense to be useful. Added focus areas (public, student, teacher, creator, admin) but initial implementation pulled in all edges touching a focus page — including inbound edges from every other page that links to the focus area.

**Learning:** For focused subgraph views, only include edges where the source is a primary (focus) page. This shows "where does this area navigate to?" without pulling in the entire graph as boundary nodes. The student focus dropped from 26 boundary nodes to 6 with this change.

**Pattern:** `allEdges.filter(e => primaryCodes.has(e.from))` instead of `allEdges.filter(e => primaryCodes.has(e.from) || primaryCodes.has(e.to))`

## 2. Mermaid Special Characters Must Be Escaped in Node Labels and Edge Labels
**Topics:** workflow

**Context:** Page titles and trigger descriptions from JSON contain characters that break Mermaid syntax: `"`, `|`, `[`, `]`, `(`, `)`, `#`, `&`, `;`.

**Learning:** Mermaid edge labels are delimited by `|`, so any pipe in the label breaks the syntax. Node labels in `["..."]` break on brackets/quotes. Built an `escapeMermaid()` function that strips or replaces these characters. Also truncate edge labels to ~25 chars to keep diagrams readable.

## 3. import.meta.glob vs fs.readFileSync for Page JSON Access
**Topics:** astro, cloudflare

**Context:** Considered two approaches for a potential sitemap.xml endpoint reading page JSONs: `import.meta.glob` (Vite build-time) vs `fs.readFileSync` (Node runtime). The scripts use `fs.readFileSync` since they run via `npx tsx`.

**Learning:** For scripts run via `npx tsx`, `fs.readFileSync` + `globSync` is the right choice — simple, synchronous, no build step. For Astro SSR endpoints deployed to Cloudflare Workers, `import.meta.glob` would be required since Workers have no filesystem. The two approaches serve different contexts and shouldn't be mixed.

## 4. Page JSON "title" Fields Had Data Quality Issues
**Topics:** workflow

**Context:** Two admin page JSONs (`certificates.json`, `sessions.json`) had `"title": "Unknown"` — a pre-existing data issue that surfaced when the auto-generated PAGES-MAP.md displayed them.

**Learning:** Auto-generating documentation from data files surfaces data quality issues that manual curation hides. The act of generating PAGES-MAP.md and SITE-MAP.md from the JSONs served as a validation pass, catching 2 bad titles that had been invisible in the hand-maintained file.
