# Session Decisions - 2026-01-09

## Decision: Add Wrangler as Dev Dependency

**Context:** Wrangler was installed globally on dev machines but not in package.json. CI failed because wrangler wasn't available.

**Options Considered:**
1. Add wrangler to devDependencies
2. Install wrangler globally in CI workflow
3. Skip E2E tests on CI

**Decision:** Add wrangler to devDependencies

**Rationale:**
- Best practice: all tools used in npm scripts should be project dependencies
- Ensures consistent versions across all environments
- CI workflow doesn't need special setup steps
- Other developers can run the full test suite after `npm install`

## Decision: Machine Detection Strategy for Tests

**Context:** Tests needed to run conditionally based on machine type, but CI was being detected incorrectly.

**Decision:** Use "Unknown" as the CI/non-macOS machine identifier, and skip local-dev-specific tests when machine is "Unknown"

**Rationale:**
- Simple heuristic: if `sw_vers` command fails, it's not macOS
- Non-macOS environments (Linux CI) get `supportsWorkerd: true` assumption
- Local dev environment tests (like .dev.vars existence) are skipped on CI
- No need for explicit CI detection environment variables
