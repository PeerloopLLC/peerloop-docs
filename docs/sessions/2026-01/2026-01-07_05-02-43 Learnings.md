# Session Learnings - 2026-01-07

## Technical

### Page Spec Verification Methodology
- **Insight:** Comparing page specs to actual implementation requires both code analysis AND visual inspection
- **Process:**
  1. Read Astro page to find imported components
  2. Read each component file looking for TODO/NYI/Stub markers
  3. Visually inspect rendered page via screenshot
  4. Cross-reference spec features against visible UI elements
- **Value:** Catches discrepancies like "spec says carousel, implementation is grid"

### Image Reading in Claude Code
- **Learning:** Claude Code can read image files directly using the Read tool
- **Use case:** Analyzing screenshots placed in a folder (`incoming/`) without user needing to paste in chat
- **Workflow:** Glob for images → Read each image → Analyze → Move to destination

### Tailwind v4 Class Verification
- **Finding:** HeroSection and other components use gradient backgrounds with `bg-linear-to-br` (v4 canonical form)
- **Pattern:** Decorative elements use `blur-3xl` for soft background effects

## Workflow

### Screenshot Management for Page Verification
- **Established pattern:**
  1. User saves screenshot(s) to `incoming/` folder at project root
  2. Skill reads and analyzes images
  3. Skill moves images to `docs/pages/{CODE}/` with timestamp naming
- **Naming convention:** `{CODE}-{YYYY-MM-DD-HH-MM-SS}.png`

### Page Spec Folder Structure
- **Migration:** Changed from `docs/pages/{CODE}.md` to `docs/pages/{CODE}/{CODE}.md`
- **Benefit:** Each page gets its own folder for spec + screenshots + future assets
- **Scale:** 57 page folders created
