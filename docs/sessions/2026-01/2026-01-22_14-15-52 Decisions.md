# Session Decisions - 2026-01-22

## 1. PATCH Instead of PUT for Settings/Profile APIs
**Type:** Implementation
**Topics:** astro, workflow

**Trigger:** Designing the notification and profile settings APIs.

**Options Considered:**
1. PUT - Replace entire resource ← Not Chosen
2. PATCH - Partial updates ← Chosen

**Decision:** Use PATCH for both `/api/me/settings` and `/api/me/profile` endpoints.

**Rationale:** PATCH allows partial updates - user can toggle one notification preference without resending all 7. More efficient and prevents race conditions with multiple toggles.

**Consequences:** API validates only fields present in request body, builds dynamic UPDATE query.

## 2. Allow Hyphens in Handle Validation
**Type:** Implementation
**Topics:** validation, astro

**Trigger:** User tried handle "guy-rymbergX" which failed validation.

**Options Considered:**
1. Alphanumeric + underscore only (like Twitter) ← Not Chosen
2. Alphanumeric + underscore + hyphen (like GitHub) ← Chosen

**Decision:** Allow hyphens in handles, but still require start/end with alphanumeric.

**Rationale:** Hyphens are common in names and usernames. GitHub pattern is more permissive and user-friendly than Twitter's.

**Consequences:** Updated regex to `/^[a-zA-Z0-9][a-zA-Z0-9_-]{1,28}[a-zA-Z0-9]$/`.

## 3. On-Blur Validation Only for Handle Field
**Type:** Implementation
**Topics:** astro, workflow

**Trigger:** User asked about validating fields when focus leaves them.

**Options Considered:**
1. On-blur for all fields ← Not Chosen
2. On-blur only for handle (uniqueness check) ← Chosen
3. On-submit only ← Not Chosen

**Decision:** Implement on-blur validation only for the handle field, which has both format and uniqueness checks.

**Rationale:** Handle is the only field requiring server-side validation (uniqueness). Other fields have simple client-side rules that are already enforced by maxLength and shown on submit. Full on-blur would require duplicating all validation rules client-side.

**Consequences:** Created `/api/users/check-handle` endpoint. Handle shows spinner during uniqueness check.
