# Session Learnings - 2026-01-09

## Technical Learnings

### 1. Cloudflare Pages 500 Errors from Module-Level Code

**Context:** Homepage was returning 500 errors on Cloudflare Pages deployments while working fine locally.

**Root Cause:** The `d1-rest.ts` module had top-level imports and execution that ran at module import time:
```javascript
import dotenv from 'dotenv';
dotenv.config({ path: resolve(process.cwd(), '.dev.vars') });
```

This crashed on Cloudflare Workers because:
- `dotenv` package isn't available in the Workers runtime
- The code executed before any try-catch could handle it
- Any module importing from `@lib/db` would trigger this crash

**Solution:** Wrap environment loading in a try-catch that silently fails on Cloudflare:
```javascript
function loadDevVars() {
  try {
    if (typeof process !== 'undefined' && process.env?.NODE_ENV !== 'production') {
      const dotenv = require('dotenv');
      // ...
    }
  } catch {
    // Expected to fail on Cloudflare Workers - silently ignore
  }
}
loadDevVars();
```

**Key Insight:** Module-level code in libraries must be defensive - it runs in all environments that import the module, not just where it's actually used.

### 2. Self-Fetch Pattern Issues in SSR

**Context:** Pages using `fetch(\`\${origin}/api/...\`)` for data fetching.

**Issues Identified:**
- Creates unnecessary HTTP round-trip even when running on same server
- On MBA-2017 with REST fallback: fetch → API → getDB() → REST fetch (nested fetches)
- While it works on Cloudflare Pages, it's architecturally messy

**Better Pattern:** Direct D1 access via `getDB(Astro.locals)`:
```javascript
const db = getDB(Astro.locals);
const data = await queryAll(db, 'SELECT ...');
```

**Benefits:**
- Single code path works on MacMini (local D1), MBA-2017 (REST), and Cloudflare (production)
- No network hop overhead
- Cleaner error handling

### 3. ESM vs CommonJS in Astro

**Context:** Attempted to use `require()` for dynamic imports in Astro.

**Learning:** Astro runs in ESM mode where `require` is not defined. For dynamic imports:
- Use `await import('module')` instead of `require('module')`
- Or ensure the code path is never reached in ESM environments

### 4. D1 Database Binding Availability

**Environment Differences:**
| Environment | `locals.runtime.env.DB` | REST Fallback |
|-------------|-------------------------|---------------|
| MacMini (Cloudflare adapter) | ✅ Available | Not needed |
| MBA-2017 (Node adapter) | ❌ Not available | Required |
| Cloudflare Pages | ✅ Available (if configured in dashboard) | Not needed |

**Key Point:** The binding's presence depends on the adapter, not just configuration.

## Process Learnings

### 1. Debugging Strategy for Environment-Specific Bugs

When a bug only appears in one environment:
1. First identify which code paths differ between environments
2. Check module-level code that runs at import time
3. Look for packages that might not be available in all runtimes
4. Add defensive checks rather than assuming package availability

### 2. Testing REST Fallback

Attempting to emulate MBA-2017 on MacMini by forcing Node adapter revealed additional complexity (dotenv loading issues in ESM). Direct testing on the actual target machine is more reliable for environment-specific features.
