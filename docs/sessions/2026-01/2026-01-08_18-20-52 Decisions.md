# Session Decisions - 2026-01-08

## Technical Decisions

### 1. Created Missing Tables via Migration
**Decision:** Create `certificates`, `user_availability`, and `intro_sessions` tables in a new migration rather than modifying existing migrations.

**Rationale:**
- Tables are needed for TDSH functionality
- New migration (0009) keeps migration history clean
- Includes seed data for testing

**Alternative considered:** Adding tables to 0001_schema.sql would require re-running all migrations.

### 2. Graceful Error Handling for Missing Tables
**Decision:** Add `.catch(() => defaultValue)` handlers to queries that reference tables that might not exist.

**Rationale:**
- Allows API to work even if some tables aren't created yet
- Prevents silent failures from breaking entire dashboard
- Can be tightened later when all tables are guaranteed to exist

### 3. Progress Calculation Using AVG(is_complete)
**Decision:** Calculate student progress as `ROUND(AVG(mp.is_complete) * 100)` where `is_complete` is 0 or 1.

**Rationale:**
- Simple and efficient single-query calculation
- Works correctly: [1,0,1,1,0] → AVG=0.6 → 60%
- Returns 0 when no module_progress rows exist (COALESCE)

### 4. Page Docs in Subfolders
**Decision:** Keep page documentation in `docs/pages/{CODE}/{CODE}.md` format with screenshots in same folder.

**Rationale:**
- Consistent with existing pages (CDSH, ADMN, etc.)
- Screenshots stay with their related documentation
- Easy to find all assets for a page
