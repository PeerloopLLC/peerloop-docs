## Key Learnings

### 1. Component-First Dashboard Architecture

When building role-specific dashboards that may later be unified into a multi-role view:

- Build each dashboard as a **standalone React component** (not just a page)
- Components should be self-contained with their own data fetching
- Later, a unified `/dashboard` page can compose these components based on user role flags
- This approach avoids Block 9.6 (Role-Based Page Access Model) dependency

**Pattern:**
```
/dashboard/learning  → StudentDashboard component
/dashboard/teaching  → STDashboard component (future)
/dashboard/creator   → CreatorDashboard component
/dashboard           → Composes all based on user.is_* flags (future)
```

### 2. Earnings Data from payment_splits

Creator earnings come from the `payment_splits` table with:
- `recipient_type IN ('creator', 'creator_royalty')` for filtering
- `status = 'pending'` for unpaid balance
- `status = 'paid'` for total earned
- Monthly earnings via `created_at >= date('now', 'start of month')`

### 3. PageSpec JSON Schema v1.1 Requirements

The PageSpec schema has specific enum values that must be used:
- Button/link status: `'active' | 'inactive'` (not 'broken')
- Component status: `'clean' | 'hasTodos'` (not 'hasPlaceholders')
- Summary requires `buttons`, `links`, `analytics` keys with `total`, `active`, `inactive`

### 4. Verification → Consolidation Workflow

The `/q-verify-page` → `/q-consolidate-page` workflow effectively:
1. Verifies implementation against spec
2. Updates the .md doc with reality
3. Syncs JSON spec to match the .md (source of truth)
4. Documents discrepancies in `_discrepancies` section
5. Validates JSON against Zod schema

## Technical Notes

### API Pattern for Dashboard Data

Dashboard APIs should aggregate multiple queries in parallel:
```typescript
const [courses, earnings, stCount, totalStudents, monthlyEarnings] = await Promise.all([
  queryAll<CourseRow>(db, coursesSQL, [userId]),
  queryAll<EarningsRow>(db, earningsSQL, [userId]),
  queryFirst<STCountRow>(db, stCountSQL, [userId]),
  queryFirst<TotalStudentsRow>(db, totalStudentsSQL, [userId]),
  queryFirst<MonthlyEarningsRow>(db, monthlySQL, [userId]),
]);
```
