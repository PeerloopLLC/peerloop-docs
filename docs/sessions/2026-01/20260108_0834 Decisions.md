# Session Decisions - 2026-01-08

## Decision 1: TypeScript Schema with JSDoc over Markdown Documentation

**Context**: Needed to define the canonical schema for page specification JSON files.

**Options Considered**:
1. Markdown documentation file (`docs/reference/PAGE-SPEC-SCHEMA.md`)
2. TypeScript interface file with JSDoc comments

**Decision**: TypeScript with JSDoc (`src/lib/types/page-spec.ts`)

**Rationale**:
- Type checking when editing JSON specs
- IDE autocomplete support
- Single source of truth that's executable
- JSDoc provides inline documentation
- Can be imported by PageSpecView.astro

---

## Decision 2: .md File as Source of Truth After Verification

**Context**: JSON spec and verified .md file can drift apart.

**Options Considered**:
1. JSON spec as source of truth (update .md to match)
2. .md file as source of truth (update JSON to match)

**Decision**: .md file is source of truth after `/q-verify-page`

**Rationale**:
- .md file reflects actual verified implementation
- JSON spec was the design intent; .md reflects reality
- Discrepancies documented in JSON `_discrepancies` section
- `/q-consolidate-page` syncs JSON from .md

---

## Decision 3: _discrepancies Section in JSON Spec

**Context**: Need to track what was planned vs what was built.

**Decision**: Add `_discrepancies` object to JSON spec with:
- `asOf` - consolidation date
- `plannedButNotImplemented` - features in spec but not built
- `implementedDifferently` - features built differently than spec
- `removedFromSpec` - features removed entirely

**Rationale**:
- Preserves design intent while reflecting reality
- Visible in PageSpecView for developer reference
- Actionable list for future enhancement work
- Underscore prefix indicates metadata, not spec content

---

## Decision 4: Schema v1.1 Optional Sections

**Context**: Verified .md files have richer content than original JSON schema.

**Decision**: Add optional sections to PageSpec:
- `features` - checkbox lists with implementation status
- `interactiveElements` - buttons, links, target pages
- `verificationNotes` - verification metadata and screenshots
- Expand `metadata` with block, component, layout

**Rationale**:
- Optional (with `?`) so existing JSON specs remain valid
- Captures rich verification data from .md files
- Enables PageSpecView to show more useful information
- Progressive enhancement - add sections as pages are verified
