# Conv Learnings - 2026-04-19

## 1. CC Command Cache Persists Deleted Skills
**Topics:** cc-workflow, dual-repo

**Context:** After deleting r-end2, r-commit2, r-timecard-day2 skill directories, user saw `/r-end2` still appearing in autocomplete.

**Learning:** Claude Code caches the skill/command list across sessions. Deleting a skill directory does not immediately remove it from the autocomplete list — a CC restart is required to clear the cache.

---

## 2. v2 Skill Promotion Path: Contents-In-Place
**Topics:** cc-workflow, dual-repo

**Context:** Upgrading three parallel v1/v2 skill pairs to consolidate on v2.

**Learning:** The cleanest upgrade path is to overwrite the v1 `SKILL.md` in-place (preserving the directory name and supporting files like `scripts/` and `refs/`) and delete the v2 directory. This keeps all internal cross-references (skill refs files, CLAUDE.md table, r-start prose) intact without needing a rename sweep.

**Pattern:**
```
cp skills/r-commit2/SKILL.md skills/r-commit/SKILL.md
rm -rf skills/r-commit2/
```
