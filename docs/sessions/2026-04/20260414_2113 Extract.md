### 4. Deeper concern → role-drift audit

**User:** "I'm afraid we have a deeper issue if isAdmin depends in any way on Moderators. currentUser global resulves the capabilities of the current user, their roles, their course (please investigate properly). currentUser is populate from an API call so on the server we have access to the same information. It looks like we have some lingering, very old role code from when teachers and moderators were not course-aware constructs and it worries me that this may be widespread"

**Claude:** Full audit performed:
- Read `src/lib/current-user.ts` 1-406 — confirmed `CurrentUser` model:
  - Platform flags: `canCreateCourses`, `canTakeCourses`, `canTeachCourses`, `canModerateCourses`, `isAdmin`
  - Course methods: `isStudentFor(courseId)`, `isTeacherFor(courseId)`, `isCreatorFor(courseId)`, `canModerateFor(courseId)`
  - Community methods: `isMemberOf(communityId)`, `isCommunityModeratorFor(communityId)`
- `canModerateFor()` correctly combines: admin → creator → `can_moderate_courses` → `community_moderators` row
- `CurrentUser.isAdmin` does NOT depend on moderator state — it's a pure mirror of `users.is_admin`
- Server-side: `src/pages/api/me/full.ts` (635 lines) populates this exact shape, so SSR can call the loader directly

Dead-code discovery (the smoking gun):
- Grepped all INSERTs to `community_members`: **3 sites, ALL hardcode `'member'` or `'creator'`**
  - `src/pages/api/communities/[slug]/join.ts:101` → 'member'
  - `src/pages/api/me/communities/index.ts:205` → 'creator' (community creation)
  - `src/lib/onboarding.ts:61` → 'member'
- No application code anywhere sets `member_role = 'teacher'`
- But dev seed (`migrations-dev/0001_seed_dev.sql:387-409`) manually inserts 7+ rows with 'teacher'
- 20+ UI/type/API sites consume `.role === 'teacher'` as if it's real:
  - 6 Astro `canUploadResources` gates
  - `CommunityTeachingTab.tsx` entire tab (filters `memberRole === 'teacher'`)
  - `CommunityRolePillFilters.tsx:47` Teaching filter pill
  - `ExploreCommunities.tsx:92` `teachingCount` badge
  - `CommunityManagement.tsx:352-356` member list role badge
  - TS types `lib/db/types.ts:156`, `lib/current-user.ts:150`
- Conclusion: in prod (no dev seed), NO row will ever have `member_role='teacher'`. Every feature gated on it is empty-by-default.

Presented three recommendation chunks: immediate #26 fix, COMMUNITY-TEACHER-CONCEPT medium, ROLE-AUDIT broad.
