## Uncategorized

### form_input vs keyboard input for React forms
The Chrome MCP `form_input` tool sets DOM values directly without triggering React's synthetic onChange events. This meant signup form submissions silently failed (button appeared to do nothing). Switching to `click` + `type` (keyboard) fixed it. This is a recurring pattern — all React-controlled forms in browser walks should use keyboard input, not form_input. Native `<select>` elements work fine with form_input since they use native DOM events.

### Self-healing enrollment verified without webhook
The enrollment self-healing path (success page detects Stripe session_id, retrieves checkout session, creates enrollment) worked correctly on MacMiniM4 where Stripe CLI webhook forwarding is not running. This is the first live verification of this code path — previously only tested in unit tests.
