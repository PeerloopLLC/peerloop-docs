# Conv Learnings - 2026-04-18

## 1. `reset-d1.js` already handles orphaned indexes automatically
**Topics:** db, deployment, cloudflare

**Context:** CLAUDE.md had a "Known issue" warning that the remote D1 reset could leave orphaned indexes from failed batch drops. Checking the actual script revealed it was already fixed.

**Learning:** Before writing a CLAUDE.md warning about a script limitation, read the current script — it may have been fixed without updating the docs. The script comment even referenced CLAUDE.md §Known issue, creating a circular documentation trail.

**Pattern:** When a known-issue doc entry exists, grep the referenced script for its status before treating the limitation as current.

## 2. Local dev DB needs `db:setup:local:dev` to populate privacy_public=1 users
**Topics:** db, testing, development-workflow

**Context:** `/discover/members` showed 0 members after CSS fix. Root-cause: UI-created accounts default to `privacy_public = 0`. The `db:setup:local` command only applies schema + core seed; it doesn't apply the dev seed with realistic `privacy_public = 1` users.

**Learning:** If the members page (or any page gated by `privacy_public = 1`) shows 0 results locally, the dev seed likely hasn't been applied. Run `npm run db:setup:local:dev` to get realistic user data.

**Pattern:** After any local DB reset or fresh machine setup, always run `db:setup:local:dev` not just `db:setup:local` if you need realistic member/course/enrollment data.

## 3. `AppNavbar`'s mobile spacer was in the wrong place for the flex layout
**Topics:** css, layout, mobile

**Context:** `<div className="h-14 lg:hidden" />` was inside the AppNavbar React component, which renders as a flex column in the `div.flex.flex-1` row layout. This spacer added height to the nav column rather than padding to the content column, causing bottom-row clipping on `/discover/members`.

**Learning:** Mobile header spacers must live in the content container, not in the nav component. The nav component's job is the nav; pushing content down is the layout's job.

**Pattern:** Use `pt-14 lg:pt-0` on the main content wrapper, not a spacer element inside the nav component.
