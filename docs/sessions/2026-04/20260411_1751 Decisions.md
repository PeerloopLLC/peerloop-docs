**Type:** Strategic
**Topics:** testing, cc-workflow

**Trigger:** /w-codecheck surfaced 10 `.astro` errors invisible to tsc. Conv 102's "clean baselines" claim was retroactively incomplete.

**Options Considered:**
1. Keep the 3-gate status quo (tsc + test + build) and accept periodic drift

**Consequences:**

## 5. CourseTabs `initialTab` type widened to honestly reflect runtime
**Type:** Implementation
**Topics:** astro, testing

**Trigger:** While fixing [AC], the TabId narrowing fix surfaced a second error: `DetailTabId` not assignable to `TabId` when passing role-specific tab IDs.

**Options Considered:**
1. Keep `initialTab?: TabId` and cast at every caller (lie preservation)
2. Widen `initialTab?: TabId | (string & {})` to match runtime's `useState<string>` ← Chosen
3. Introduce two props `initialTab` and `initialExtraTab`

**Decision:** Option 2.

**Rationale:** The type was a lie — runtime already accepts any string via `extraTabs`. The `(string & {})` idiom is the standard TypeScript pattern for "literal union OR any string" while keeping autocomplete for the literal part.

**Consequences:** Documented the prop with a comment explaining it accepts extraTabs IDs too. No runtime change. Type now matches behavior.
