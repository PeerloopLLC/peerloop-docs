## 2. /r-end Auto-Advances Drift Baseline After Each Conv
**Type:** Implementation
**Topics:** r-end, drift-detection, doc-sync

**Trigger:** Question about whether /r-end needed updating given the new baseline mechanism.

**Options Considered:**
1. Manual advance only (user runs `advance-drift-baseline.sh` when needed)
2. Auto-advance in /r-end after code repo commit, before docs commit ← Chosen
3. Auto-advance after push (would require a separate commit or dirty state at next r-start)

**Decision:** /r-end advances the baseline between code commit and docs commit, so `.drift-baseline-sha` is included in the docs commit.

**Rationale:** Without auto-advance, the same drift flags would reappear at the next SessionStart even though the docs agent had already reviewed them. Auto-advance closes the loop: docs agent reviews flags, baseline advances, next session starts clean.

**Consequences:** `/r-end` SKILL.md Step 6 updated. `tech-doc-drift.sh` hook footer updated to note auto-advance.
