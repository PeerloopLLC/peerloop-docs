# Conv Decisions - 2026-04-06

## 1. Mnemonic Collision Rule
**Type:** Implementation
**Topics:** cc-workflow, work-tracking

**Trigger:** User observed duplicate mnemonic short codes appearing across TodoWrite tasks, making "do GE" ambiguous.

**Options Considered:**
1. Require globally unique codes across all conversations — too strict, codes are ephemeral
2. Append sequential numbers on collision within current task list ← Chosen
3. Use longer codes to reduce collision probability — doesn't eliminate the problem

**Decision:** When a mnemonic code collides with an existing task in the current list, append a sequential number (e.g., `[GE]` → `[GE2]` → `[GE3]`).

**Rationale:** Simple, deterministic, preserves the mnemonic readability while guaranteeing uniqueness within a conversation.

**Consequences:** Updated global CLAUDE.md rule. Saved as feedback memory.
