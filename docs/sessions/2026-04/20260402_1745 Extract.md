### Toast Debugging Saga

**Claude:** Spent significant time debugging why the success toast wasn't appearing:
- React state-based toast: `setSuccessMessage` confirmed called via console.log, but element never in DOM
- Root cause: `setCourse()` state update likely triggers component remount (Astro island re-render), resetting React state before toast renders
- Solution: DOM-based `showToast()` that appends directly to `document.body` — survives React re-renders
- Additional dev-mode issue: Vite HMR can reload the page and strip DOM elements. 5-second default duration (up from 3) mitigates this. Not an issue in production.

### STUMBLE Fix-and-Verify Workflow Documentation

**User:** "We need to formalize the process we would normally follow in a failed browser-run"

**User described the workflow:**
1. Run PLATO instance in API mode, expecting clean pass
2. Find failure
3. Split instance into Pre-segment (steps before failure) and Post-segment (failure onward)
4. Run Pre-segment in API mode with `snapshot: true` → produces DB state right before the fix point
5. Restore snapshot, walk Post-segment in browser to verify fix

**User clarification:** "You split the Instance into all the steps before the issue into a Pre segment, and all the remaining steps into a Post segments and then you ran the Pre-segment to create the snapshot at the end."
