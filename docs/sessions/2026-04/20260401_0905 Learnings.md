# Conv Learnings - 2026-04-01

## 1. PLATO Taxonomy: Step / Scenario / Persona Set / Instance
**Topics:** plato, testing, architecture

**Context:** User confusion about "run" meaning both atomic actions and executed sequences

**Learning:** Four-concept taxonomy resolves ambiguity: Steps are atomic action templates (data-independent). Scenarios compose steps into sequences with verification. Persona Sets provide the data. Instances bind scenarios to persona sets with an execution mode (test/seed/walkthrough/repro). Today instances are implicit (personaSet hardcoded in scenario files). Future: extract into separate files.

## 2. Handle Validation: Social Platform Standard
**Topics:** auth, validation, ux

**Context:** 3 different handle validators with conflicting rules (hyphens, start chars, max length)

**Learning:** For social platforms with @mentions, hyphens in handles create parsing ambiguity (`@john-smith` = `@john` minus `smith`?). Standard: letters, numbers, underscores only; must start with letter; 3-20 chars. Single source of truth in `auth/index.ts`, with `isValidHandleFormat()` boolean export for call sites that don't need detailed errors.

## 3. Chrome Extension Requires --chrome Launch Flag
**Topics:** tooling, cc-workflow

**Context:** Tried to enable Chrome extension mid-session via `/chrome` — shows "Disabled" status

**Learning:** The `/chrome` command manages settings but doesn't establish the connection. The native messaging host bridge requires launching with `claude --chrome` flag. Mid-session `/chrome` cannot enable it. Update shell alias to include `--chrome` if browser testing becomes routine.

---

## 4. ChainStep to ChainEntry: Avoiding Name Collisions in Renames
**Topics:** typescript, refactoring

**Context:** Renaming "run" to "step" created collision with `ChainStep` type (union of RunRef | SqlTopUpRef)

**Learning:** When renaming creates a collision with an existing type that uses the new name, rename the colliding type too. `ChainStep` → `ChainEntry` avoids confusion between "a step in the chain" (entry) and "a PLATO step" (atomic action template). Always check for this class of collision before starting bulk renames.
