# Conv Learnings - 2026-04-10

## 1. Explore agent's porting assessment can reverse direction mid-report
**Topics:** cc-workflow, skills, w-sync-skills

**Context:** During the full-scan `/w-sync-skills` and the deep r-end dive, the delegated Explore agent produced thorough diffs but ended its synthesis with backward assessments — asking "should SPT adopt this from Peerloop?" instead of the requested "should Peerloop adopt this from SPT?". Required manual re-filtering to surface the one real finding (fmt-learn-decide.md's "Doc reorganization" criterion).

**Learning:** When delegating skill-sync diffs, the agent prompt should include an explicit "DIRECTION" reminder at the **end** of the prompt — right before the output format section — not just in the introduction. Alternatively, frame each finding as "source has X / local lacks X" rather than "assessment: port?" so direction is implicit in the phrasing.

**Pattern:** In w-sync-skills Step 3, require findings to be phrased as `Source has: ... Local has: ...` (subject-first, source always first) to eliminate directional ambiguity.

## 2. Structured-tag ports need both sides (generation + extraction)
**Topics:** cc-workflow, skills, r-commit, r-end, r-timecard-day

**Context:** Porting the `Doc Changes` extraction to r-timecard-day alone was insufficient — the timecard section would have stayed empty forever because r-commit and r-end weren't generating `Doc:` lines in commit bodies. Completing the loop required coordinated edits across all three skills.

**Learning:** Structured commit tags are a bidirectional contract: the writer skills (r-commit, r-end) must emit the tag, and the reader skills (r-timecard-day, r-timecard, w-git-history) must extract it. A port to one side is dead code until the other side is updated. When porting any structured tag, grep the full skills/ tree for the tag name before claiming the work is done.

## 3. Option-prompts for doc reorganization should be explicit, not hedged
**Topics:** cc-workflow, collaboration

**Context:** When presenting the `Doc:` vs `Infra:` split for "docs reorganization", offering Option A (keep in Infra) / Option B (move to Doc) / Option C (both) was effective — user picked B cleanly. The explicit A/B/C labeling made the question answerable in one character.

**Learning:** For adaptation questions where the answer is genuinely a judgment call (not a fact lookup), present 2-3 labeled options with brief trade-offs and let the user answer with a single character. Avoid leading with a recommendation when the user's preference is the deciding factor.
