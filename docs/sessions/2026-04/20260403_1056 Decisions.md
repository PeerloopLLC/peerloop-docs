# Conv Decisions - 2026-04-03

## 1. Add Cancel Session Button to Session Room
**Type:** Implementation
**Topics:** booking, ux, session-room

**Trigger:** STUMBLE-AUDIT walkthrough discovered no UI path to cancel a session without rebooking

**Options Considered:**
1. Cancel button on session room page alongside Reschedule <- Chosen
2. Cancel option only on sessions list page
3. Cancel only via Reschedule flow (status quo)

**Decision:** Added "Cancel Session" button to both early state (SessionRoom) and joinable state (SessionJoinableView), with ConfirmModal confirmation.

**Rationale:** Users who want to cancel without rebooking had no UI path. The Reschedule flow forces a new booking. A standalone cancel with confirmation modal is the standard UX pattern.

**Consequences:** SessionRoom.tsx and SessionJoinableView.tsx both updated. ConfirmModal added to SessionRoom. Timer guard updated to preserve cancelled state.
