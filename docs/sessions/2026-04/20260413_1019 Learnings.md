# Conv Learnings - 2026-04-13

## 1. React 19 useRef Requires Explicit Initial Value
**Topics:** react, typescript, astro6

**Context:** MemberDirectory.tsx used `useRef<ReturnType<typeof setTimeout>>()` without args

**Learning:** React 19 changed useRef to require explicit initial value. `useRef<T>()` (no args) is now a TS error. Fix: `useRef<T>(undefined)`.

**Pattern:** Always pass initial value to useRef in React 19+

---

## 2. Client-Side Search Creates Invisible Data Problem
**Topics:** ux, api-design, discover

**Context:** /discover/creators and /discover/teachers both had search bars that only filtered 50-100 pre-loaded records

**Learning:** Client-side-only filtering with a capped dataset creates a deceptive UX — the search bar suggests full search capability but silently fails for records beyond the LIMIT. Server-side search eliminates this. The creators page had an API endpoint that already supported search but was never called.

---

## 3. last_login Column Is Dead (Never Written To)
**Topics:** schema, users, activity-tracking

**Context:** Investigated sort-by-activity options for member directory

**Learning:** The `users.last_login` column exists in schema and is queried by admin analytics, but is never written to by any application code — it's always NULL. `updated_at` is the only available activity proxy. This is a schema debt item.
