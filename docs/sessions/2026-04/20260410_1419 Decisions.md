# Conv Decisions - 2026-04-10

## 1. Separate `Doc:` from `Infra:` in structured commit tags (Option B)
**Type:** Implementation
**Topics:** cc-workflow, docs-infra, skills, r-commit, r-end

**Trigger:** While porting the `Doc:` tag from spt-docs to r-commit, local's existing `Infra:` tag description included "docs reorganization". Needed to decide where doc-reorganization commits should be tagged going forward.

**Options Considered:**
1. Keep "docs reorganization" under `Infra:` (structural/workflow nature). Doc: would cover only content changes.
2. ← **Chosen.** Remove "docs reorganization" from Infra, move it to Doc:, making Doc: the single home for all doc-related commits (content + structural).
3. Mention reorganization under both tags with a "either is acceptable" note.

**Decision:** Doc: is now the primary and unambiguous home for all documentation commits, including structural reorganization (moves, renames, splits, consolidation). Infra: is reserved for tooling, scripts, hooks, skills, build config, and dev workflow — not docs of any kind.

**Rationale:** A clean split is easier to follow than a fuzzy boundary. The user's phrasing ("clear;y separate Docs from Infra") indicated strong preference for unambiguous categorization. Both r-end and r-timecard-day are now consistent with this boundary.

**Consequences:**
- Future docs-reorganization commits should carry `Doc:` not `Infra:`.
- Daily timecards will aggregate structural doc work into `#### Doc Changes` instead of `#### Infra Changes`.
- The r-end commit-template rewrite makes Doc: the primary tag suggestion for r-end (docs-only) commits.

## 2. Port "Doc reorganization" decision criterion from spt-docs
**Type:** Implementation
**Topics:** cc-workflow, docs-infra, skills, r-end

**Trigger:** Deep-dive diff of r-end's `refs/fmt-learn-decide.md` found source has a 6th "Important Decision Criteria" row that local was missing.

**Options Considered:**
1. ← **Chosen.** Port the criterion verbatim — it's domain-agnostic.
2. Skip — not useful enough.

**Decision:** Added "Doc reorganization | New doc categories, naming conventions, cross-reference patterns, per-section docs, doc consolidation" as a 6th criterion in the Important Decision Criteria table.

**Rationale:** Peerloop does a lot of doc reorganization work that isn't architecture, code style, or technology selection but is durable and worth recording in DECISIONS.md / DOC-DECISIONS.md. Before this port, such decisions had no obvious home in the "important" criteria list.

**Consequences:** r-end's learn-decide agent now has explicit grounds to promote doc-reorganization decisions to DECISIONS.md or DOC-DECISIONS.md. Relates to Decision 1 — both moves formalize "docs reorganization" as its own first-class concept across the skills.
