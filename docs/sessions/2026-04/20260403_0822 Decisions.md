# Conv Decisions - 2026-04-03

## 1. Shared Utilities Over Inline Duplication for Toast and Confirm Modal
**Type:** Implementation
**Topics:** react, architecture, components

**Trigger:** Need to replace ~73 alert()/confirm() calls across 18 files

**Options Considered:**
1. Inline showToast + modal in each file (matches existing CourseEditor pattern)
2. Shared `src/lib/toast.ts` + `src/components/ui/ConfirmModal.tsx` (extracted, reusable) ← Chosen

**Decision:** Created shared utilities — one toast function, one ConfirmModal component

**Rationale:** 18 files with identical inline code would be ~500 lines of pure duplication. Shared utilities: single source of truth, consistent UX, testable in isolation.

**Consequences:** Created new `src/components/ui/` directory. All 18 files import from shared locations. CourseEditor's inline showToast removed.

**Type:** Strategic
**Topics:** plato, testing, admin

**Trigger:** User asked how PLATO emulates admin actions like granting creator role
