# Conv Learnings - 2026-04-02

## 1. Onboarding Nudge-to-Settings Dead-End Loop
**Topics:** onboarding, ux, navigation

**Context:** STUMBLE walkthrough revealed that OnboardingNudgeBanner linked to /settings/interests, which only saves tags and does NOT set onboarding_completed_at. Users following the nudge remained "not onboarded."

**Learning:** When a nudge directs users to complete an action, the destination must actually complete the action. Settings/interests is a subset editor, not the full onboarding flow.

---

## 2. Checkbox vs Radio for Independent Goals
**Topics:** onboarding, ux, schema-design

**Context:** Goal selector (Learn/Teach/Both) was implemented as radio-style cards. User corrected: Learn and Teach are independent flags, not mutually exclusive. "Both" was an artifact of the old single-column design.

**Learning:** When options can be independently selected, use checkboxes (boolean columns) not radio buttons (enum column). This also scales to future additions without requiring combo values.

---

## 3. Fixed-Width Dropdowns Prevent Layout Overflow
**Topics:** css, ui, topic-picker

**Context:** Level dropdown (Beginner/Intermediate/Advanced) expanded to fit content width, causing "Intermediate" to visually overlap the topic card chevron.

**Learning:** Select elements containing variable-length options need a fixed width (e.g., `w-[120px]`) to prevent layout instability.
