### users.last_login column is dead
The `users.last_login` column exists in schema and is queried by admin analytics (`/api/admin/analytics`) but is never written to by any application code. It's always NULL. This means admin analytics reporting on "active in last 30/60 days" is returning 0 for all users. This is a silent data bug in the analytics dashboard.
