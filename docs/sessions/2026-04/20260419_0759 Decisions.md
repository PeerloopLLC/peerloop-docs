# Conv Decisions - 2026-04-19

## 1. Script location: consolidate to `.claude/scripts/`
**Type:** Strategic
**Topics:** cc-workflow, dual-repo, docs-infra

**Trigger:** Phase 2 migration required touching both the `.sh` runtime and the JSON reader — a natural inflection point to resolve the existing duplication.

**Options Considered:**
1. **Quick:** Update both r-end/scripts/ and r-end2/scripts/ copies in place. No structural churn; latent drift risk remains.
2. **Middle:** Keep `.sh` scripts per-skill, hoist only the new `docs-registry.mjs` helper to `.claude/scripts/`. Partial DRY.
3. **Durable:** Consolidate all 4 files (sync-gaps.sh, tech-doc-sweep.sh, route-mapping.txt, docs-registry.mjs) under `.claude/scripts/`. Update both skills' SKILL.md + fmt-docs.md to reference the new paths. ← Chosen

**Decision:** Option 3. Single canonical location under `.claude/scripts/`; both `/r-end` and `/r-end2` reference the same files.

**Rationale:** The duplication was silent drift waiting to happen. Consolidating during a migration is cheaper than consolidating later — the callers are already being touched. Directory already existed with 10 helper scripts (`timecard-day.js`, `conv-*.sh`), so the new location was a natural fit rather than a novel convention.

**Consequences:** 6 files deleted from `r-end/scripts/` + `r-end2/scripts/`; 5 caller files updated; 4 stale `DOC-DECISIONS.md` refs corrected; 2 live paths (`.claude/config.json` routeMapping + strategy doc) fixed. `detect-changes.sh` + `dev-env-scan.sh` still duplicated — out of Phase 2 scope, flagged for future cleanup.

## 2. Correct the `codePattern` truncation bug rather than preserve it
**Type:** Implementation
**Topics:** cc-workflow, docs-infra

**Trigger:** Empirical test of `${rule%%|*}` on a representative rule showed the old bash runtime truncated every multi-alternation rule at the first `|`. The strategy doc §4 said "no behavioral change." These are in direct tension.

**Options Considered:**
1. Emit the truncated pattern from `docs-registry.mjs` to reproduce the bug byte-for-byte.
2. Emit the full `codePattern` and accept that flag volume will jump on first post-migration run. ← Chosen
3. Emit truncated + file a follow-up task to fix the bug later.

**Decision:** Option 2.

**Rationale:** The JSON in `docsRegistry` was authored with full-alternation intent from Phase 1 (Conv 132). Preserving the bug means preserving a reader-runtime mismatch — the JSON would be actively misleading about what the tool checks. Correcting the runtime to match the spec is more durable. The one-time 9-flag bump is an auditable event, not a silent regression.

**Consequences:** 9 new flags surfaced on HEAD~5. Triage: 2 REAL_DRIFT, 2 PARTIAL, 5 FALSE_POSITIVE. The 2 REAL items were fixed this conv (resend.md, availability-calendar.md). The 5 FALSE_POSITIVEs are the "known-noise" pattern analogous to the documented auth-doc case — worth a future DOC-DECISIONS entry if they recur.
