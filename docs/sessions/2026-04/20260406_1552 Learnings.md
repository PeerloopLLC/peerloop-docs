# Conv Learnings - 2026-04-06

## 1. openssl HMAC matches Web Crypto HMAC
**Topics:** testing, infrastructure

**Context:** Needed to generate BBB webhook HMAC tokens in shell scripts without Node.js dependency

**Learning:** `echo -n "$data" | openssl dgst -sha256 -hmac "$key" | sed 's/.*= //'` produces identical hex output to Web Crypto's `crypto.subtle.sign('HMAC', key, data)`. Verified with test vector — both produce `51825373bdd06272b7861a11bded98d02c786641b2042103c1257899b31fbc7c`.

**Pattern:**
```bash
# Shell HMAC-SHA256 (matches Node.js Web Crypto)
echo -n "$data" | openssl dgst -sha256 -hmac "$key" | sed 's/.*= //'
```

## 2. Claude Code PreToolUse hooks can gate commits
**Topics:** cc-workflow, infrastructure

**Context:** Needed to enforce lint-timezone.sh before code repo commits

**Learning:** PreToolUse hooks with Bash matcher can intercept commands by regex, run checks, and return `{"hookSpecificOutput": {"permissionDecision": "deny", "permissionDecisionReason": "..."}}` to block tool execution. This is Claude-only enforcement — human git commits bypass it.

## 3. Stripe trigger --override for metadata
**Topics:** stripe, testing

**Context:** Stripe CLI `stripe trigger` sends synthetic events with fake IDs that don't match DB records

**Learning:** `--override "checkout_session:metadata[key]=value"` injects metadata into triggered events, allowing seed-data alignment for checkout flows. Refund/dispute triggers can't be aligned because their charge IDs are Stripe-generated.
