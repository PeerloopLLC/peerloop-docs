# Conv Learnings - 2026-04-02

## 1. DOM-Based Toast Survives React Remounts
**Topics:** react, astro, ui-feedback

**Context:** React state-based toast in CourseEditor never appeared despite `setSuccessMessage` being confirmed called. Component remount after `setCourse()` resets React state before render.

**Learning:** When a React component inside an Astro island may remount during state updates, use DOM manipulation (`document.body.appendChild`) for transient feedback instead of React state. DOM-appended elements survive React re-renders and Astro island remounts.

**Pattern:** `showToast()` function at module scope: creates a div, styles it inline, appends to body, removes after timeout.

---

## 2. Tag Resolution Should Accept Names Not Just IDs
**Topics:** api, database, ux

**Context:** Course editor Tags field sent tag names ("AI Strategy"), but the API inserted them as tag IDs, causing FK constraint violation.

**Learning:** API endpoints that accept arrays of reference values should resolve both IDs and names/slugs, using `COLLATE NOCASE` for case-insensitive matching. Use `INSERT OR IGNORE` to handle duplicates gracefully, and silently skip unknown values rather than failing the entire save.

---

## 3. Sidebar Nav Items Must Exist for Every Capability
**Topics:** navigation, ux, rbac

**Context:** "Become a Creator" link correctly hides when user has `canCreateCourses`, but no "Creating" link appeared as replacement. Creator couldn't navigate to their hub.

**Learning:** When a capability-gated nav item hides via `excludeCapabilities`, there MUST be a corresponding item that shows via `capabilities` for the same capability. Audit all `excludeCapabilities` entries to ensure the replacement exists.

---

## 4. Vite HMR Can Strip DOM Elements During Dev
**Topics:** vite, dev-tools, debugging

**Context:** DOM-based toast appeared to not work because Vite's hot module replacement reloaded the page within seconds of the save completing, removing the appended element.

**Learning:** During dev-mode debugging of DOM manipulation, Vite HMR can silently reload pages when source files change. Use longer timeouts (5-10s) when testing transient UI elements in dev. This is a non-issue in production builds.
