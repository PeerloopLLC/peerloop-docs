# Conv Learnings - 2026-04-14

## 1. `community_members.member_role='teacher'` is dead code
**Topics:** schema, roles, drift

**Context:** Investigating a simple `canUploadResources` UI gate mismatch uncovered widespread drift.

**Learning:** A schema CHECK constraint can permit a value that no application code ever writes. Dev seeds can manually populate such a value, creating the illusion of a working feature. In prod (no dev seed), that value never occurs, and all features gated on it are silently empty. The symptom shows up only in production, because dev always has the hand-seeded rows.

**Pattern:** When a schema enum/CHECK allows a value, audit every INSERT/UPDATE site to confirm the value is actually reachable from application code. If seed is the only writer, the value is effectively a test fixture, not a real enum member.

## 2. CurrentUser is the single role-state source; server can call the same loader
**Topics:** currentuser, roles, ssr

**Context:** Confusion about where role state "lives" — client global vs SSR loader.

**Learning:** `/api/me/full` is the API that returns `MeFullResponse`. On the client, `initializeCurrentUser()` fetches it and constructs a `CurrentUser` instance. On the server, the same loader logic lives in `src/pages/api/me/full.ts` — Astro pages can call its query helpers directly without an HTTP round-trip. So "role state" has one shape and two consumers, not two separate stacks.

**Pattern:** For any capability check, prefer: SSR page loads minimal data via loader function → client hydrates with full `CurrentUser` via global. Avoid inlining role SQL in page files.

## 3. Glossary is prescriptive — code drift accumulates without it
**Topics:** terminology, glossary, governance

**Context:** `GLOSSARY.md` explicitly states Teacher is course-scoped (via `teacher_certifications`). Schema still carries a non-course-scoped `member_role='teacher'` value.

**Learning:** Living terminology documents are only useful if they're audited against the schema and code periodically. The TERMINOLOGY block (Sessions 346-356) did ~5000 replacements but didn't scrub this particular drift because it was semantically (not textually) wrong — the word "teacher" was correct, but its *referent* in `community_members` contradicted the glossary definition.

**Pattern:** Include semantic audits alongside textual ones. Ask not just "does this identifier match the glossary term?" but "does this identifier mean what the glossary says it should mean?"

## 4. Minimal test seeding paid off
**Topics:** testing, fixtures

**Context:** The 3 new Phase 7 test files seeded only `'creator'` and `'member'` in `community_members`, not `'teacher'`.

**Learning:** When a schema offers multiple enum values, seed only the ones the test actually needs. This paid off immediately — tests survive the incoming COMMUNITY-TEACHER-KILL without modification.

**Pattern:** Test fixtures should seed the minimum enum surface. Avoid seeding "all possible values for coverage" — that creates drag on every future schema change.
