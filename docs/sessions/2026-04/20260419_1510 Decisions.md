# Conv Decisions - 2026-04-19

## 1. Enrolled Users See Disabled "Following (enrolled)" Indicator (Not Hidden Button)
**Type:** Implementation
**Topics:** course-follows, ux

**Trigger:** Enrolled users already have Stream timeline follow via enrollment. Should the Follow button be hidden or show a different state?

**Options Considered:**
1. Hide button entirely when enrolled
2. Show disabled "Following (enrolled)" indicator ← Chosen

**Decision:** Option 2 — show disabled indicator.

**Rationale:** User choice (1B). Makes the follow relationship visible and explains why the button is disabled, rather than silently absent.

**Consequences:** `CourseFollowButton` renders a small indicator row (star + text) for enrolled users instead of hiding.

## 2. Student Profile Followed Courses List Deferred
**Type:** Strategic
**Topics:** course-follows, student-profile

**Trigger:** Client docs mention showing followed courses on student profile.

**Options Considered:**
1. Include in COURSE-FOLLOWS block
2. Defer — just build the follow/unfollow mechanism and API ← Chosen

**Decision:** Option 2 — deferred.

**Rationale:** User choice (2A). Core feature is the follow/unfollow button + API; profile display is additive and can be done in a separate block.

**Consequences:** `GET /api/me/course-follows` is implemented and ready for the profile page to consume when that work is scoped.
