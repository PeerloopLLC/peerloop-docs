- **Three failed attempts at [VH] signature verification before success.** First attempt used `sed`-based function extraction from the helper script; failed because the sed range pattern matched too eagerly on `}` tokens. Second attempt inlined the openssl command directly from JS; failed because JS template-string interpolation of the JSON payload broke bash's double-quote re-parsing. Third attempt used a self-contained wrapper shell script invoked via `spawnSync` with `input:`; passed. Worth noting for future harness/signing-verification work — the problem wasn't signing, it was the test harness-for-the-harness. Consider adding a note to `docs/as-designed/webhook-miss-resilience.md` about the wrapper-script pattern for any future HMAC-over-JSON verification work.
