# Conv Learnings - 2026-04-03

## 1. Timer Effect Guard Must Cover All Client-Side State Transitions
**Topics:** react, session-room, state-management

**Context:** Added Cancel button to SessionRoom that sets viewState to 'cancelled' client-side. The 1-second countdown timer interval immediately overwrote it back to 'early'.

**Learning:** The timer's guard list (`joining`, `in_session`, `completed`, `rating`) was designed for states that could only be set from the server (via prop) or user-initiated join actions. When adding new client-side state transitions (like cancellation via API), the guard list must be updated to include the new state. The original code never hit this bug because cancellation could only happen server-side (the `session.status` prop would already be `'cancelled'`).

**Pattern:** In any component with a polling/timer effect that recomputes state from props, ensure the "don't override" guard covers ALL states that can be set from client-side actions, not just the original set.

## 2. Session Booking Wizard Skips Teacher Selection When Pre-Assigned
**Topics:** booking, ux

**Context:** Walking the booking wizard as Alex (student with pre-assigned teacher from enrollment).

**Learning:** The wizard intelligently skips Step 1 when `assignedTeacherId` is set, showing "Selected at enrollment" on the teacher card. This is the common case for 1-teacher courses. The multi-teacher selection path is available when no teacher is pre-assigned.

## 3. Positional Module Assignment Handles Cancellation Correctly
**Topics:** booking, modules

**Context:** After completing Module 1 and cancelling a Module 2 session, rebooking correctly offered Module 2 again.

**Learning:** Cancelled sessions have `module_id = null` (never frozen). The positional assignment system only counts non-cancelled completed sessions to determine the next module. This means cancelled sessions don't consume module slots -- the expected behavior for a learning platform.
