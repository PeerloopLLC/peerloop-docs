# Conv Learnings - 2026-04-15

## 1. Differential reproduction beats config spelunking
**Topics:** testing, astro, cloudflare

**Context:** Investigating the `dev:staging` React SSR bug, instinct was to read adapter config and changelog first.

**Learning:** Running `dev:staging` and plain `dev` back-to-back with identical `curl` probes took 30 seconds and immediately localized the bug to the `remoteBindings` branch. Reading @astrojs/cloudflare source would have taken an order of magnitude longer. **When a regression is env-flag-gated, build a differential repro before reading source.**

**Pattern:** (a) identify the smallest env delta between working and broken configurations, (b) run both with identical probes, (c) diff the output — only read source if differential output doesn't converge on a hypothesis.

## 2. `ssr.noExternal` and `resolve.dedupe` are not always honored by adapters
**Topics:** astro, cloudflare, testing

**Context:** Tried two standard React-dup fixes for the dev:staging bug — both failed, leaving the `deps_ssr/react-dom_server.js` artifact unchanged.

**Learning:** When an Astro adapter has its own SSR externalization logic (as `@astrojs/cloudflare` 13 does with its `remoteBindings` branch), user-space vite config can be overridden. Evidence: `react-dom_server.js` stayed in `deps_ssr/` both times, and `useState` remained null. **Config-level dedupe assumes vite has final say over SSR resolution — adapters can invalidate this assumption.**

**Pattern:** If two standard fixes fail back-to-back against a regression in an adapter-controlled branch, escalate to reading adapter source before attempting a third config-level fix.

## 3. `node -c` works for ESM syntax check only when run against the absolute path
**Topics:** testing, infra

**Context:** Tried `node -c scripts/reset-d1.js` from the docs-repo cwd — succeeded misleadingly (wrong file). Re-ran with absolute path to confirm.

**Learning:** Dual-repo shell drift causes relative-path syntax checks to silently pass on the wrong file (or fail with a misleading error). Always use absolute paths for standalone node/python/bash syntax checks in dual-repo work. Concrete instance of the `git -C` rule applied to non-git commands.

**Pattern:** `node -c /abs/path/to/file.js` instead of `node -c scripts/file.js`.
