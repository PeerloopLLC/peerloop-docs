# Conv Decisions - 2026-04-06

## 1. Claude Code hook over git pre-commit hook for lint-timezone enforcement
**Type:** Implementation
**Topics:** infrastructure, cc-workflow

**Trigger:** User's low confidence in timezone correctness due to recurring incomplete sweeps

**Options Considered:**
1. Git pre-commit hook (via husky) — gates all committers
2. Claude Code PreToolUse hook — gates Claude commits only ← Chosen
3. Both

**Decision:** Claude Code PreToolUse hook only, with documented fragility analysis

**Rationale:** Claude is currently the sole committer. Adding husky introduces a build dependency for a single-dev project. The fragility is documented with a clear mitigation path (add husky + CI when a second developer joins).

**Consequences:** Human commits are ungated. Created `docs/as-designed/lint-timezone.md` documenting this.

## 2. Always use /r-end for commits, never /r-commit directly
**Type:** Strategic
**Topics:** cc-workflow, dual-repo

**Trigger:** User correction — direct /r-commit causes overlapped session docs

**Options Considered:**
1. /r-commit for mid-session saves
2. /r-end always ← Chosen

**Decision:** Always use /r-end. /r-commit should only be used if user explicitly requests mid-session save.

**Rationale:** /r-end runs session documentation agents that prevent overlapped docs. Saved as feedback memory.

**Consequences:** Feedback memory saved. Future conversations will use /r-end by default.
