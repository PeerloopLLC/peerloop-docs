# Conv Learnings - 2026-04-07

## 1. Astro Cloudflare Adapter Always Auto-Configures KV Sessions
**Topics:** astro, cloudflare

**Context:** Investigating KV removal from codebase — KV bindings were provisioned but unused.

**Learning:** The `@astrojs/cloudflare` adapter (line 55 of dist/index.js) always sets `session.driver = 'cloudflare-kv-binding'` if no driver is configured. There's no "disable sessions" option. To suppress info messages, explicitly set the session config even if the KV binding doesn't exist. As long as no code calls `Astro.session`, the missing binding causes no runtime errors.

## 2. PLAN.md Blocks Accumulate Faster Than They Complete
**Topics:** project-mgmt

**Context:** Full PLAN.md review revealed 44 deferred entries, many overlapping.

**Learning:** Blocks created during discovery (terminology review, capabilities review, stumble audit) tend to be granular and single-purpose. Over time these accumulate into a long list where related items are scattered. Periodic amalgamation reviews are essential to keep the plan navigable. The 46% reduction (2278 to 1231 lines) came mostly from merging related blocks, not dropping work.
