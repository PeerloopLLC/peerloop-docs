# Conv Learnings - 2026-04-05

## 1. VS Code Claude Code Extension Diff Preview vs Terminal Mode
**Topics:** vscode, claude-code, developer-tooling

**Context:** User migrated from Cursor to VS Code and noticed file edit diffs no longer appeared in side-by-side editor panels.

**Learning:** Claude Code in a VS Code terminal only shows inline terminal diffs. The VS Code Claude Code extension (sidebar/panel mode) intercepts file edits and displays them in VS Code's native diff editor for approval — matching the Cursor experience. The distinction is launch method: terminal vs extension panel.

## 2. Workspace File Case Sensitivity Causes Vitest Extension Crash
**Topics:** vscode, vitest, workspace-config

**Context:** Vitest extension threw `Fatal Error: Attempted to get parent of root folder "/"` with recursive `getOrCreateFolderTestItem` stack trace.

**Learning:** The workspace file had `"path": "../peerloop"` (lowercase) while the actual directory is `../Peerloop`. macOS resolves this (case-insensitive FS), but the Vitest extension's path-matching logic walks up the directory tree comparing against workspace roots. The case mismatch prevents a match, so it recurses all the way to `/` and crashes. Always use exact case in `.code-workspace` folder paths.

## 3. Material Theme No Longer Available in VS Code Marketplace
**Topics:** vscode, themes

**Context:** User migrated from Cursor which had Material Theme installed.

**Learning:** Material Theme has been removed from the VS Code marketplace. The suggested successor is Vira Theme (paid). Free alternatives closest to Material Theme's aesthetic: One Monokai, Ayu (especially Mirage variant), Night Owl, Nocterial Palenight (specifically bridges Material Palenight), Noctis (8 dark variants).
