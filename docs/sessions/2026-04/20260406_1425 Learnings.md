# Conv Learnings - 2026-04-06

## 1. Midnight-Crossing Sessions Fail Availability Windows
**Topics:** testing, timezone, availability

**Context:** `futureISO(300)` computed independently for start and end could generate sessions spanning midnight UTC (e.g., 23:15-00:15), which failed `isSlotWithinAvailability()` even with all-day availability.

**Learning:** Session time generation in tests must produce paired start/end times that stay within a single UTC day. Independent computation of start and end can diverge across midnight boundaries.

**Pattern:** `futureSession(minutesFromNow, duration)` returns `{ start, end }` pair with midnight-avoidance logic.

## 2. getNow() Adoption Was Only 4 Files Despite Existing Infrastructure
**Topics:** clock, testing, timezone

**Context:** `src/lib/clock.ts` with `getNow()` was created in Conv 003 and documented in DECISIONS.md, but only 4 of 22+ eligible files actually used it.

**Learning:** Infrastructure without enforcement drifts. The clock abstraction existed but was consistently bypassed by new code. Adding a lint rule (`lint-timezone.sh` source scan) closes the feedback loop.

## 3. Cross-Project Directive Porting Works Best as Selective Merge
**Topics:** cc-workflow, dual-repo

**Context:** User asked to port directives from spt-docs CLAUDE.md. Both projects share the same skill/harness architecture.

**Learning:** Compare, identify gaps, present differences — don't blindly copy. SPT's `SKILL-INTERPLAY.md` was a separate file but fit naturally as a section in Peerloop's existing `skills-system.md`.
