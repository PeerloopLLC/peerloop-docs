# Conv Decisions - 2026-04-06

## 1. TIMELINE.md Monthly Grouping for Backfill, Daily for New Entries
**Type:** Implementation
**Topics:** docs-infra

**Trigger:** Backfilling 35 entries — daily H2 sections would create 20 mostly-single-row tables

**Options Considered:**
1. Daily H2 sections (matching /r-end format) — consistent but verbose for backfill
2. Monthly tables with Date column ← Chosen

**Decision:** Use monthly H2 sections for the backfill, with Date as first column. Future /r-end entries use daily `## YYYY-Mmm-DD` sections per fmt-learn-decide.md format rules. Both styles coexist.

**Rationale:** Monthly grouping is 7 tables vs 20 for the same data. Future daily entries append naturally at the bottom.

## 2. TIMELINE.md Ref Column Convention (S### / ### / —)
**Type:** Implementation
**Topics:** docs-infra

**Trigger:** Pre-Conv entries have Session numbers, Conv entries have Conv numbers, Dec 2025 entries have neither

**Options Considered:**
1. Single column with mixed formats
2. Separate columns for Session and Conv numbers
3. Ref column with prefix convention (S### / ### / —) ← Chosen

**Decision:** Ref column uses `S###` for Sessions, `###` for Convs, `—` for pre-tracking entries. Header note explains the convention.

**Rationale:** Preserves traceability to both numbering systems without forcing a single convention.
