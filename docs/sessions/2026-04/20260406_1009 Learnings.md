# Conv Learnings - 2026-04-06

## 1. TIMELINE.md Fills a Gap Between DECISIONS.md and Session Logs
**Topics:** docs-infra, dual-repo

**Context:** Porting TIMELINE.md from spt-docs to peerloop-docs

**Learning:** DECISIONS.md captures detailed rationale ("why"), session logs capture what happened ("what"), but neither provides a scannable "inflection point history" — when the project changed direction. TIMELINE.md answers that with one-line events per milestone, filtered by 6 significance criteria to prevent bloat.

---

## 2. Backfill Reveals Missing Pre-Session History
**Topics:** docs-infra

**Context:** First backfill draft had no entries before March 2026

**Learning:** DECISIONS.md has rich entries dating back to Dec 2025 (tech stack, auth, data layer, layouts) that predate session tracking (Session 0 starts Jan 16, 2026). These founding decisions are some of the most timeline-worthy events. Always check the full date range of DECISIONS.md, not just the recent entries.

---

## 3. /w-sync-skills Single-Skill Mode Uses Namespace Disambiguation
**Topics:** cc-workflow

**Context:** Adding single-skill argument to /w-sync-skills

**Learning:** Source names (repo identifiers like `spt-docs`) and skill names (`r-end`, `w-commit`) are drawn from non-overlapping namespaces — source names are repo identifiers while skill names use the `r-`/`w-`/`q-` prefix convention. Simple token matching works without ambiguity.
