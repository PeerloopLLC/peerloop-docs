# Conv Decisions - 2026-04-02

## 1. Onboarding Goals: Boolean Columns over Enum
**Type:** Implementation
**Topics:** schema, onboarding, member-profiles

**Trigger:** User feedback that Learn/Teach are independent selections, not either/or

**Options Considered:**
1. Keep single `primary_goal` TEXT column with "learn", "teach", "learn,teach" values
2. Two boolean columns `goal_learn INTEGER` and `goal_teach INTEGER` ← Chosen

**Decision:** Replace `primary_goal` TEXT with independent `goal_learn`/`goal_teach` boolean columns

**Rationale:** More reasons may be added in the future; independent booleans scale without combo value explosion. No existing users means schema can change freely.

**Consequences:** Schema, API, UI, seed data, tests, PLATO all updated (16 files). "Both" option removed from UI — selecting both checkboxes implicitly means "both."

> **Insight:** When modeling user preferences that may grow over time, independent boolean flags are more durable than enum columns with combo values. Each new option is an additive column rather than an exponential expansion of valid enum strings.

## 2. Nudge Banner Links to /onboarding (Not /settings/interests)
**Type:** Implementation
**Topics:** onboarding, nudge-banner, navigation

**Trigger:** STUMBLE walkthrough discovered dead-end loop

**Options Considered:**
1. Make /settings/interests also set onboarding_completed_at
2. Change nudge link to /onboarding ← Chosen
3. Add separate "Mark complete" action in settings

**Decision:** Nudge banner links to /onboarding which has the full form and sets onboarding_completed_at on submit

**Rationale:** The full onboarding page is the correct destination — it has goal selection, tags, and about-you. Settings/interests is a post-onboarding subset editor.

**Consequences:** Both compact and full nudge variants updated. Tests updated.

## 3. Onboarding Skip UX: Explicit "Skip for now" Link
**Type:** Strategic
**Topics:** onboarding, ux

**Trigger:** First-time users couldn't submit empty form (disabled button) and had no clear way to skip

**Options Considered:**
1. Allow empty submit (mark complete with no data)
2. Add explicit "Skip for now" link ← Chosen
3. Leave as-is (navigate away)

**Decision:** Add "Skip for now" link that navigates to homepage without setting onboarding_completed_at

**Rationale:** Explicit skip affordance is better UX than relying on users figuring out they can navigate away. Not marking complete means nudge banners will remind them.

**Consequences:** Link appears only for first-time users when no changes exist. Disappears once user makes any selection.
