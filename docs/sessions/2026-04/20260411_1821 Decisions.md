# Conv Decisions - 2026-04-11

## 1. [HW] — delete dead half-wired state instead of building inline banners
**Type:** Implementation
**Topics:** react, testing

**Trigger:** Conv 104 left [HW] as "fix-or-defer before [P6]". On investigation the features weren't half-wired — they were superseded by FormModal's internal loading + showToast.

**Options Considered:**
1. A. Delete dead state (+ replace 4 silent-failure `setError` sites with `showToast`) ← **Chosen**
2. B. Replace `showToast` with inline banners site-wide, wire up `actionLoading` around FormModal open/submit cycles
3. C. Redundant — keep toast AND render `_error`/`_successMessage` as inline banners

**Decision:** A. Minimum viable durable fix. Deletes 3 unused state pairs, 8 dead setter-resets, and 8 no-op `disabled` guards. Replaces 4 genuine silent-failure sites with `showToast` — the shipped UX pattern everywhere else in these files.

**Rationale:** The shipped feedback UX is toast. Inline banners would be churn and would regress the UX. The `disabled={actionLoading}` guards were protecting against a double-submit that FormModal already prevents. Minimum code change, maximum alignment with the shipped pattern.

**Consequences:**
- 3 files changed, 11 insertions / 46 deletions
- 4 pre-existing silent failures in TeachersTab + PeerLoopFeaturesTab now surface via toast (net UX improvement)
- Five-gate baseline still green on jfg-dev-10up
- [P6] PR is now a pure package-upgrade PR + this [HW] cleanup — no half-wired debt rides along
