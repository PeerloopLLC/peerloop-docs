## 1. PLATO Four-Concept Taxonomy
**Type:** Strategic
**Topics:** plato, testing, architecture

**Trigger:** User confusion during STUMBLE-AUDIT about what "run" means

**Options Considered:**
1. Two-tier rename: run→step, scenario stays — Partial
2. Three-tier: step→run→scenario — Doesn't match code structure
3. Four-concept: step/scenario/persona set/instance ← Chosen

**Decision:** Adopt four-concept taxonomy. Rename run→step now. Keep scenario. Instance is future work.

**Rationale:** Steps are templates, scenarios compose them, persona sets provide data, instances bind them. This maps to user's three use cases (seed DB, test flywheel, ad-hoc repro) and eliminates the ambiguity where "run" was both a noun and a verb.

**Consequences:** 30+ files renamed, ~150 replacements. plato.md updated with taxonomy section. Future work: extract personaSet from scenarios into instance files.

## 2. Handle Validation: Option D (Social Platform Standard)
**Type:** Implementation
**Topics:** auth, validation

**Trigger:** STUMBLE-AUDIT finding: 3 validators with conflicting rules

**Options Considered:**
1. Auth rules (no hyphens, allow `_` start, max 30)
2. Profile rules (hyphens allowed, alphanumeric start/end, max 30)
3. Most restrictive merge (no hyphens, alphanumeric start, max 30)
4. Social platform standard (no hyphens, must start with letter, max 20) ← Chosen

**Decision:** `^[a-zA-Z][a-zA-Z0-9_]{2,19}$` — must start with letter, letters/numbers/underscores only, 3-20 chars.

**Rationale:** Matches Twitter/Instagram conventions. Avoids @mention ambiguity with hyphens. Max 20 aligns with auto-gen cap. Single source of truth in `auth/index.ts`.

**Consequences:** Updated `auth/index.ts`, `api/me/profile.ts`, `ProfileSettings.tsx`, `ProfileSettings.test.tsx`. 99 tests passing. Existing auto-generated handles (lowercase alphanumeric) are compliant.
