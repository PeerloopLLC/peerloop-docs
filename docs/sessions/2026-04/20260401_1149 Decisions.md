# Conv Decisions - 2026-04-01

## 1. PLATO Instance Architecture: Inline Scenarios + When Guards + Multi-Instance Accumulation
**Type:** Strategic
**Topics:** plato, testing

**Trigger:** User wanted to run same scenario with different parameters (skip vs complete onboarding)

**Options Considered:**
1. Write separate scenarios for each variant (copy-paste)
2. Build formal instance system with conditional steps ← Chosen
3. Skip PLATO, test only in browser

**Decision:** Built PlatoInstance/PlatoInstanceFile types with `when` predicate guards on StepRef, multi-instance execution against same DB (accumulation), inline scenario support, and WalkthroughCheckpoint type for STUMBLE pairing.

**Rationale:** Instances solve the general parameterization problem — any future scenario variant uses the same infrastructure. The when guard is the minimal mechanism for conditional steps. Accumulation model proves coexistence.

**Consequences:** 4th PLATO concept now operational. Instance files pair API proof (PLATO) with UX proof (STUMBLE walkthrough checkpoints). New step: complete-onboarding (20th step). New category: 'walkthrough'.

## 2. STUMBLE-AUDIT Formalization: Lightweight Pairing with PLATO Instances
**Type:** Strategic
**Topics:** stumble-audit, plato, testing

**Trigger:** User observed natural pairing between PLATO instances and STUMBLE walkthroughs

**Options Considered:**
1. Keep informal (ad-hoc browser testing)
2. Lightweight pairing: walkthrough checkpoints in instance files + TodoWrite for issues ← Chosen
3. Full formalization with structured result artifacts

**Decision:** Added WalkthroughCheckpoint type to instance files. Execution uses accumulate-with-checkpoints model (walk sub-flow, batch findings, pause for triage). Issues captured as TodoWrite tasks with severity (broken/confusion/cosmetic) and source tag.

**Rationale:** Adds structure without over-engineering. Instance file is the natural home for "what to check in browser." TodoWrite captures issues persistently.

**Consequences:** Each PLATO instance file can now include a walkthrough script. STUMBLE findings flow through TodoWrite → RESUME-STATE.md for cross-conv persistence.
