# Conv Learnings - 2026-04-02

## 1. plato:restore Corrupts D1 When Dev Server Is Running
**Topics:** plato, d1, sqlite, dev-workflow

**Context:** Ran `plato:restore` while dev server was on port 4321. The restored SQLite file was immediately corrupted (SQLITE_CORRUPT).

**Learning:** Workerd holds the D1 SQLite file open. Copying over it while the process has a handle corrupts the file. The `plato-restore-snapshot.js` script's comment said "stop dev server first" but didn't enforce it. Added a port-check guard using `lsof -ti:4321`.

**Pattern:** Always check for running processes before replacing files they hold open.

## 2. Dynamic PLATO Instance Runner via PLATO_INSTANCE Env Var
**Topics:** plato, testing, vitest

**Context:** Split instances registered in instances/index.ts but had no test blocks in plato-scenarios.api.test.ts. `--testNamePattern` only matches describe/it text.

**Learning:** Added a conditional `describe` block that only creates when `PLATO_INSTANCE` env var is set. `plato-restore.js` passes the env var through. Normal `npm test` is unaffected (env var not set).

**Pattern:** `if (process.env.PLATO_INSTANCE) { describe(...) }` for dynamic test registration.

## 3. form_input Tool Doesn't Trigger React onChange
**Topics:** browser-automation, react, chrome-mcp

**Context:** Used `form_input` to fill signup form fields. Button click did nothing — React state didn't know fields were filled.

**Learning:** `form_input` sets DOM values directly without dispatching React synthetic events. Use `click` + `type` (keyboard input) instead, which properly triggers React's onChange handlers. `form_input` works fine for native `<select>` elements.

## 4. PLATO params Field Does Double Duty
**Topics:** plato, api-runner

**Context:** submit-expectations step needed to POST to `/api/enrollments/:id/expectations` with a dynamic enrollment ID.

**Learning:** The `params` field on PageAction serves two purposes: (1) passes route params to the Astro handler, and (2) the runner uses it to reverse-map the resolved URL back to the filesystem path (replacing `enr-abc123` → `[id]`). No explicit `handler` override needed.

## 5. Split Point Must Include All Prerequisites
**Topics:** plato, stumble-workflow

**Context:** Initially split flywheel at step 7 (register-student). The course page showed "Enrollment Unavailable" because self-certify-creator (step 8) hadn't run.

**Learning:** When choosing a split point for browser walks, the Pre-segment must include all steps that establish the preconditions for what you're about to walk. For enrollment, the teacher must be certified first. Think about the DB state the browser walk needs, not just the step you want to start from.
