# Conv Learnings - 2026-04-18

## 1. `docRootExclude` controls `validDocs` membership — affects all Doc-mention routing downstream
**Topics:** timecard-tooling, docs-infra

**Context:** PLAN.md and DOC-DECISIONS.md bullets were routing to Testing (word match on "plan") or Work Effort instead of Doc Changes. Root cause: both files were in `docRootExclude`, which prevented them from entering `validDocs`. The `extractDocs()` function gates all bare filename matches on `validDocs.has(candidate)`, so excluded filenames are never recognized as doc mentions.

**Learning:** `docRootExclude` is a double-edged config field — it excludes files from *both* the "counted as doc" detection AND the "recognized as doc mention" detection. If you want a file to appear in Doc Changes when mentioned, it must not be in `docRootExclude`. The `routineStrip.countFiles` list handles the separate concern of stripping "N files synced" logging bullets.

## 2. Tier ordering in classifyItem() must account for cross-signal contamination
**Topics:** timecard-tooling

**Context:** When adding T3b (API detection), placing it before T3a (infraPrefixWords) would cause `db:migrate:prod /api/something` style bullets to be routed to API Changes instead of Infra > Scripts. Placing T2.5 (DB signals) before T3a ensures explicit DB tags beat the db:* prefix detection.

**Learning:** When adding new classification tiers, trace through existing bullet examples — especially multi-signal bullets that contain substrings matching multiple tiers. The ordering `T2.5 (DB) → T3a (infraPrefixWords) → T3b (API) → T3c (infraPathSubstring) → T3g (codePrefixRe)` within T3 is load-bearing for correctness. Document the tier ordering rationale in the skill spec.

## 3. `!isTestRelated` guard is required on any path-substring detection in workEffort
**Topics:** timecard-tooling

**Context:** After adding `apiPathRe = /api/[a-zA-Z0-9_]/`, test file paths like `tests/api/me/enrollments.test.ts` contained "/api/" and were caught at T3b, wrongly routing to API Changes instead of Testing.

**Learning:** Any heuristic that matches on path substrings must be guarded by `!isTestRelated()` when applied to workEffort items. Test paths frequently overlap with non-test path patterns (api/, src/, etc.). The guard prevents false-positive routing from path contamination.

## 4. ALL-CAPS doc stems appear in bullets without `.md` extension — whitelist is the right solution
**Topics:** timecard-tooling, docs-infra

**Context:** Commit bullets reference docs by their stem (e.g., "updated API-ENROLLMENTS to add…" not "updated API-ENROLLMENTS.md"). The `extractDocs()` regex only matched filenames with `.md` suffix, so these references were invisible to doc detection and couldn't route to Doc Changes.

**Learning:** A `docNameWhitelist` config field with explicit ALL-CAPS stems is cleaner than trying to auto-detect ALL-CAPS patterns. The guarded approach (`validDocs.has(stem + '.md')` check) prevents false positives from arbitrary ALL-CAPS words in bullets. The combined regex `(?:\.md)?` handles both extension variants in one pass.
