# Conv Learnings - 2026-04-19

## 1. Measure-before-choose catches inverted assumptions in written plans
**Topics:** cc-workflow, docs-infra

**Context:** PLAN.md Phase 3 said "fast subset: sync-gaps only, skip tech-doc-sweep." Timing the two scripts revealed the opposite: sync-gaps is 4.5s, tech-doc-sweep is 1.3s. If the evaluation had begun from the PLAN framing, we'd have proposed the slower script for the hook.

**Learning:** When a plan written in a prior conv specifies a technical trade-off by name ("subset X is fast"), verify the empirical claim at start of the follow-up conv before designing around it. Written plans decay silently as scripts change.

**Pattern:** `time bash path/to/script 2>&1 | tail -N` on both candidates before choosing.

## 2. Cross-repo architecture constrains hook placement more than hook-vs-CI does
**Topics:** cc-workflow, docs-infra, dual-repo

**Context:** Evaluating Phase 3 felt like "pre-commit vs CI" but really had 4 axes (where, when, what, block-vs-warn). The `peerloop-docs`/`Peerloop` split meant any hook installed in the code repo had to reach sideways into the docs repo for scripts.

**Learning:** In a dual-repo Claude Code setup, "where does the hook live" dominates "what triggers it". The CC SessionStart hook in the docs repo wins on friction because all dev work enters through the `peerloop` alias — the CC boundary is already the natural point-of-change moment, and the docs repo already has the scripts at hand.

**Pattern:** Before designing a drift/validation hook, list the entry points a human actually uses to start work. If one of them is already a CC session, SessionStart hooks beat git-level hooks on friction.

## 3. Silent-on-clean is the default for recurring informational hooks
**Topics:** cc-workflow, docs-infra

**Context:** SessionStart hooks run at every conv start. A drift hook that prints `No drift detected.` every time becomes noise the eye filters out — defeating its own purpose.

**Learning:** Recurring informational hooks should produce zero output when everything is fine. Presence-of-output becomes the signal. Same idea as Unix tools: `grep` prints nothing when nothing matches, not `"No matches."`

**Pattern:** In a wrapper hook, detect the "all clear" sentinel strings from the wrapped script and `exit 0` silently. Only emit a header block when there's something to act on.
