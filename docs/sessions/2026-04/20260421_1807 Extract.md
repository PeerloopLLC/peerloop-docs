### New Patterns
- **Stripe mode isolation enforcement rule** — every `.dev.vars*` file and every `wrangler secret put --env <x>` invocation is single-mode. Local=Test, staging=Sandbox, prod=Live.
- **`wrangler tail --env <env>` first, speculation second** — when a deployed Worker returns an opaque 4xx/5xx, open the tail terminal before hypothesizing. It streams `console.error` output in real time.
- **Secret-rotation sequencing rule** — rotate with immediate-expire on the old secret BEFORE re-enabling a webhook endpoint. Avoids re-activating a leaked-prefix secret and in-between rejected events.
- **`ctx.waitUntil()` for forensics/telemetry writes** — fire-and-forget `.then().catch()` patterns race Worker termination in short-path handlers. Tracked in [VW] for retrofit across Stripe + BBB webhooks.
