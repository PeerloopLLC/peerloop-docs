# Conv Decisions - 2026-04-13

## 1. SQL Top-Up vs Step Action for privacy_public
**Type:** Implementation
**Topics:** plato, testing

**Trigger:** browse-members step needs privacy_public=1 but registered users default to 0

**Options Considered:**
1. Add profile-update action inside browse-members step (realistic but not purely read-only)
2. SQL top-up in each scenario before browse-members (clean separation) <- Chosen
3. New atomic step `set-profile-public` as separate chain entry

**Decision:** Use SQL top-up in each scenario chain before browse-members

**Rationale:** Keeps browse-members purely read-only. The privacy setting is a data precondition, not a user action being tested.

**Consequences:** Each scenario using browse-members needs its own `topup-make-profiles-public` entry.

## 2. AbortController Pattern for MemberDirectory Fetch
**Type:** Implementation
**Topics:** react, data-fetching

**Trigger:** Creator filter empty on initial page load due to hydration race

**Options Considered:**
1. Add stale guard boolean in useEffect closure
2. AbortController with rolesKey serialization <- Chosen
3. Debounce the entire fetch pipeline

**Decision:** Replace useCallback+useEffect fetch pattern with inline useEffect + AbortController

**Rationale:** AbortController is the React-recommended cleanup pattern. Serializing Set to string eliminates reference-identity dependency issues. Removing setMembers([]) from handlers prevents intermediate empty states.

**Consequences:** MemberDirectory.tsx now has separate useEffect (initial/filter fetch) and useCallback (Load More only).

## 3. recordLogin() in Auth Endpoints
**Type:** Implementation
**Topics:** auth, analytics

**Trigger:** users.last_login column never written, admin MAU always 0

**Options Considered:**
1. Update last_login inside setAuthCookies (requires DB param change)
2. Standalone recordLogin() called at each auth site <- Chosen
3. Middleware/hook on all authenticated requests

**Decision:** Add recordLogin(db, userId) to session.ts, call from all 4 auth endpoints

**Rationale:** Minimal change, no function signature changes to setAuthCookies. Each auth endpoint already has `db` in scope.

**Consequences:** Admin analytics MAU will now be non-zero for active users. Uses strftime ISO format per project convention.
