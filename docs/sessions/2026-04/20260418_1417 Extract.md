### New Patterns
- **Named H5/H6 strategies in config with implementations in script.** Strategy names (e.g. `"infraGroupLabels"`, `"docFilename"`) live in `h4Sections[].h5Strategy`; strategy bodies live in `H5_STRATEGIES` / `H6_STRATEGIES` lookup tables in `timecard-day.js`. Adding a new strategy is a script edit; swapping or configuring an existing one is config-only.
- **Config-field references via dotted strings.** Predicate values like `"reroute.apiMethodRe"` or `"routineStrip.multiDocMinCount"` resolve through `resolveConfigRef(ref, rt)` to the referenced config value at eval time. Keeps the pattern library separate from the predicates that consume it.
- **2-pass predicate engine with recursive fallthrough detection.** Pass 1: every H4 predicate evaluates over every bullet (independent, multi-match allowed). Pass 2: unmatched bullets routed to the H4 whose `include` contains `fallthrough: true` (detected by `predicateHasFallthrough` walking `anyOf`/`allOf` trees).
