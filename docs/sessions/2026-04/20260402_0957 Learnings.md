# Conv Learnings - 2026-04-02

## 1. Two parallel completion paths must sync module_progress
**Topics:** d1, booking, dashboard

**Context:** BBB webhook path completed sessions but never wrote to `module_progress` table. Dashboard reads `module_progress` for stats.

**Learning:** When multiple code paths update the same entity (enrollment completion), they must write to ALL downstream data sources, not just the primary table. The webhook path updated `sessions` and `enrollments.progress_percent` but missed `module_progress`.

**Pattern:** Use `ON CONFLICT ... DO UPDATE` (upsert) so both paths are idempotent and can't create duplicates.

---

## 2. Route-map scanner architecture: BFS from nav components
**Topics:** plato, testing, infrastructure

**Context:** Built a scanner to map API endpoints to UI pages and compute navigation paths.

**Learning:** Treating navigation components (AppNavbar, AdminNavbar, DiscoverSlidePanel) as BFS source nodes and page-to-page links as edges creates a complete navigation graph. Tab sub-routes need explicit edges since dynamic hrefs aren't traceable by regex.

**Pattern:** Scanner outputs both TypeScript (for code consumption) and Markdown (for human reference) from the same data. Self-maintaining via `/r-end` docs agent hook.

---

## 3. Structured navigation + prose page actions is the right split
**Topics:** plato, testing, architecture

**Context:** Designing BrowserIntent to replace WalkthroughCheckpoint for PLATO browser-runs.

**Learning:** Navigation TO a page is mechanical and deterministic (click element → verify destination). Actions ON a page are contextual and varied (form fills, async loads, modals). Structuring navigation enables fail-fast testing. Keeping page actions as prose avoids building a Playwright DSL.

**Pattern:** `navigate: { via, clicks: NavClick[] }` for structured nav; `pageAction: string` for prose instructions.

---

## 4. Diagnostic segments are ephemeral
**Topics:** plato, testing

**Context:** `flywheel-to-enrollment` was created in Conv 073 to isolate booking→certification bugs.

**Learning:** Segmented instances that snapshot at a midpoint for debugging are diagnostic tools, not permanent test artifacts. Once bugs are fixed, the segment's purpose is served. If the journey is independently useful, promote it to a named scenario.

---

## 5. Conditional nav items need fail-fast handling
**Topics:** plato, testing, navigation

**Context:** `/onboarding` link disappears from AppNavbar after first tag selection (`hideWhenOnboarded`).

**Learning:** Static route-map nav paths assume links always exist. State-dependent nav items (capability-gated, onboarding-gated) may not be present at runtime. Browser-runs should fail immediately when an expected nav element is missing, then use the route-map as a diagnostic tool to suggest alternatives.
