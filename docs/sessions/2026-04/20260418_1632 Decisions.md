# Conv Decisions - 2026-04-18

## 1. Booking Availability Window: 28 Days on All Surfaces
**Type:** Implementation
**Topics:** booking, availability, frontend

**Trigger:** SessionBooking (28 days) and CourseAvailabilityPreview (30 days default) were misaligned.

**Options Considered:**
1. Keep 28 in booking, fix preview default to 28 ← Chosen
2. Expand both to a wider horizon (8–12 weeks) with on-demand fetch
3. No cap, show empty-month message

**Decision:** Align both surfaces on 28 days. Cap SessionBooking forward nav at today+28. Fix `availability-summary.ts` default from 30 → 28.

**Rationale:** 28 days matches current teacher scheduling horizon. Consistency across surfaces is more important than the exact number right now.

**Consequences:** `SessionBooking` next-month button disabled when current month contains day 28+. `CourseAvailabilityPreview` now shows "28 days" when `platform_stats` key is absent.

## 2. Disputed Enrollment Download Access: Allowed
**Type:** Strategic
**Topics:** enrollment, auth, download, policy

**Trigger:** `download.ts` gate needed a policy decision on whether `disputed` status should block resource access.

**Options Considered:**
1. Block — treat disputed same as cancelled (conservative, protects creator revenue)
2. Allow — student keeps resource access during dispute ← Chosen
3. Allow completed-disputed only

**Decision:** Disputed enrollments retain download access.

**Rationale:** Revoking access during an open dispute would be perceived as bad faith and could escalate the dispute. Access can be revoked after dispute resolution if needed.

**Consequences:** Only `cancelled` + `deleted_at IS NOT NULL` enrollments are blocked.
