**Type:** Implementation
**Topics:** d1, testing, infra

**Trigger:** #12 [RS] asked for orphan-index bug fix. CLAUDE.md §Known issue documents this bug with a manual recovery procedure — strong signal it recurs.

**Options Considered:**
1. Quick — append a post-table-drop index sweep (~15 lines)
2. Durable ← Chosen — restructure into 6 idempotent steps with index pre-drop, post-sweep, and final verification (~50 lines)

**Decision:** Durable approach.

**Rationale:** Manual recovery already documented in CLAUDE.md means this bug has bitten before (Session 359) and likely will again. **Consequences:** `scripts/reset-d1.js` gained ~110 lines. Local path unchanged. Remote path now prints drop-order, sweep report, and verification summary on every run. If `queryWrangler` fails (old wrangler version), `listUserObjects` returns `[]` and sweeps are no-ops — preserves pre-change behavior as fallback.
