### New Patterns
- **Centralized env access directive:** Application code MUST access Cloudflare bindings and secrets through helpers. Direct `locals.runtime?.env?.X` reads are limited to helper implementations (`env.ts`, `db/index.ts`, `r2.ts`). Grep verification after any refactor: `grep -rn "locals\.runtime\?\.env" src/` should return only helper files.
- **Preparatory refactor pattern:** When a version bump requires a pattern change across many call sites, centralize the pattern behind helpers on the OLD version first. The centralization commit is independently valuable. The version bump commit then touches only helpers.
- **Test mock update rule:** When refactoring a file that's mocked in tests, grep for `vi.mock('@/lib/<mocked>')` and update mock surfaces to match new imports. Partial mocks break silently.
