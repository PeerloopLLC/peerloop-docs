# Conv Learnings - 2026-04-15

## 1. Discriminant-vs-descriptor distinction for split-backend tables
**Topics:** schema, storage, r2, community_resources

**Context:** `community_resources` stores files (R2) and external links in one table. `type` (document/image/audio/video_link/other) is descriptive metadata; `r2_key` (set ⇒ file, NULL ⇒ link) is the real backend discriminant. Conflating them leads to UI bugs because user-facing taxonomy doesn't have to match the storage split.

**Learning:** When a single table holds records backed by different storage mechanisms, pick ONE non-user-facing column as the discriminant (usually a nullable backend-specific key like `r2_key`) and keep user-facing metadata columns free of branching semantics. Document the discriminant explicitly in the schema doc.

## 2. SSR pre-compute as a backend-decoupling boundary
**Topics:** ssr, react, astro, community_resources, storage

**Context:** The SSR loader `fetchCommunityDetailData` emits `downloadUrl: string` per resource — either the download API (files) or `external_url` (links). The React tree uses a single `<a href={downloadUrl}>` without branching.

**Learning:** Pushing a storage-backend decision through SSR into one pre-computed client-facing URL is a clean abstraction boundary. Alternatives (client-side branching, polymorphic URLs, typed discriminated DTO unions) all leak backend knowledge into React. Formalized in `docs/as-designed/r2-storage.md`.

## 3. Alias symmetry in vi.mock type arguments
**Topics:** testing, vitest, typescript, path-aliases

**Context:** `vi.mock('@/lib/auth', importOriginal => ...)` pairs best with `importOriginal<typeof import('@/lib/auth')>()` — not a relative path. Vitest resolves by alias at runtime; tsc resolves type-only imports by alias. A relative path works at runtime (vitest aliases hide off-by-one `..` counts) but tsc's stricter resolution catches errors. `tests/api/community-resources/[id]/download.test.ts:29` had 5 `..`s (off by one) and was only caught after alias switch.

**Learning:** When writing `vi.mock` with `importOriginal`, always use the same alias in the type argument as the mock target. Symmetry eliminates path-depth coupling and makes the test self-contained.

## 4. `grep -F basename` is a false-negative generator for shared-name test files
**Topics:** sync-gaps, documentation, testing

**Context:** `sync-gaps.sh` test-file check used basename substring matching. Multiple test files share `download.test.ts` / `index.test.ts` etc.; any single mention in any TEST-*.md doc masked all collisions as "documented".

**Learning:** When gap-detection scripts check "is X documented?", prefer full-path match first; allow basename fallback only for names that cannot collide. A small blocklist of known shared basenames (download/index/stats/delete/update/create.test.ts) is sufficient.

## 5. Drift cleanup is recursive
**Topics:** cc-workflow, docs-drift, todowrite

**Context:** A drain pass of 15 pending items surfaced 6 new related issues (#35, #36, #37, #38, plus sibling task updates). Each stale-doc fix exposed adjacent staleness.

**Learning:** "Drain" is not monotonic shrinkage. Budget for 30-50% task growth during drain phases. End state = `pending − closed + spawned`. Plan an extra commit checkpoint at the midpoint.

## 6. Horizontal-flex top-spacer antipattern
**Topics:** css, layout, astro

**Context:** `AppNavbar.tsx:593` places `<div className="h-14 lg:hidden" />` as a sibling of `<main>` inside a `flex flex-1` (horizontal) container, intending to push mobile content down by 3.5rem. Because it's a horizontal-flex child, it becomes a 0×3.5rem column, not a top spacer. The fixed mobile header then occludes the top of `<main>` → perceived bottom clipping.

**Learning:** Top-spacer divs only work when the parent is a **vertical** flex or block layout. In horizontal flex, they are zero-width columns. The right place for a viewport-top spacer is inside `<main>` or its content div, with `pt-<N>` in the column direction.

## 7. Visual alert style patterns render as bulleted markdown in skill outputs
**Topics:** cc-workflow, visual-alerts

**Context:** The 🟠🟠🟠 / 🔴🔴🔴 alert prefixes this conv surfaced two drift discoveries (§Sub-Communities aspirational endpoints, `AppLayout` horizontal-flex spacer bug) that would otherwise have been buried.

**Learning:** Keep the pattern consistent — three emoji + bold short description + `→` indented action. Deviating (e.g., single 🟠, or omitting the `→` action line) degrades the visual scan value.
