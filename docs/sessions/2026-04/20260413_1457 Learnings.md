# Conv Learnings - 2026-04-13

## 1. React Set Dependency Causes Hydration Race
**Topics:** react, astro, hydration

**Context:** MemberDirectory.tsx used `useCallback` with `selectedRoles` (a `Set`) in the dependency array. On Astro `client:load` hydration, the Set gets a new reference each render even when contents are identical, causing the callback to recreate and the `useEffect` to re-fire.

**Learning:** Never use mutable reference types (Set, Map, Object) directly in React dependency arrays. Serialize to a stable primitive (sorted comma string) for dependency tracking.

**Pattern:** `const rolesKey = [...selectedRoles].sort().join(',');` then use `rolesKey` in deps instead of `selectedRoles`.

## 2. AbortController for Effect-Based Fetching
**Topics:** react, data-fetching

**Context:** The old pattern used `useCallback` + `useEffect([callback])` which is fragile â€” any dependency change recreates the callback, triggering the effect. The `setMembers([])` calls in filter handlers created intermediate empty states.

**Learning:** For filter/sort/search -> fetch patterns, inline the fetch in `useEffect` with an `AbortController` cleanup. Remove `setMembers([])` from handlers â€” let the effect exclusively own the data. Handlers just change dependency values.

## 3. SQL Top-Up for Test Data Preconditions
**Topics:** plato, testing

**Context:** PLATO's `browse-members` step is read-only but requires `privacy_public = 1` to see results. Adding a profile-update action would make the step not purely read-only.

**Learning:** Use `SqlTopUpRef` entries in scenario chains for data preconditions that don't map to a user action. This keeps steps pure (one goal per step) while scenarios handle environment setup.

---

## 4. Refresh Token as Silent Auth Fallback
**Topics:** auth, security

**Context:** `getSession()` falls back to refresh token when access token expires. Comment says "could refresh access token here, for now just return the payload." This creates a 7-day window where the API treats expired-access-token users as fully authenticated.

**Learning:** Refresh tokens should only be used to issue new access tokens, not as direct auth credentials. The fallback pattern silently defeats the security purpose of short-lived access tokens. The UI showed "session expired" but API returned admin-level data.

---

## 5. Route-API-Map Script Has Hardcoded Discover Links
**Topics:** tooling, maintenance

**Context:** `scripts/route-api-map.mjs` has a `DISCOVER_LINKS` constant that must be manually updated when DiscoverSlidePanel routes change.

**Learning:** When consolidating routes (like Conv 111's 3->1 member pages), also update scripts that reference those routes by name. The route-api-map script is not auto-detected by any CI gate.
