# Conv Decisions - 2026-04-06

## 1. Export parseBlindsideCaptureUrl for Direct Testing
**Type:** Implementation
**Topics:** testing, r2

**Trigger:** Need to test Blindside URL pattern detection — function was private

**Options Considered:**
1. Test only through replicateRecordingToR2 public API
2. Export parseBlindsideCaptureUrl for direct unit testing ← Chosen

**Decision:** Export the pure function for direct testing

**Rationale:** Pure function with no side effects, interesting edge cases (trailing slash, .m4v URL, different regions) that deserve direct coverage. Testing through the public API would require mocking fetch for each URL variant.

**Consequences:** Added `export` keyword to function declaration in r2.ts
