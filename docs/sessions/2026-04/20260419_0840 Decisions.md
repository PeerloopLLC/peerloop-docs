# Conv Decisions - 2026-04-19

## 1. Implement Phase 3 as Option D (CC SessionStart hook), defer Option A (CI check)
**Type:** Strategic
**Topics:** cc-workflow, docs-infra, dual-repo

**Trigger:** PLAN.md Phase 3 required choosing between pre-commit hook, CI check, or both; user asked for an evaluation with trade-offs.

**Options Considered:**
1. A. CI-only drift-check job in `Peerloop/.github/workflows/ci.yml` (needs cross-repo checkout; ~10s in CI; blocks merge)
2. B. Git pre-commit hook in `Peerloop/.githooks/pre-commit` running tech-doc-sweep (~1.3s; warn-only; per-clone install via `core.hooksPath`)
3. C. Both A and B (belt-and-suspenders)
4. D. CC SessionStart hook in `peerloop-docs/.claude/settings.json` running tech-doc-sweep ← Chosen; zero install friction; fits the `peerloop` alias entry point

**Decision:** Implement D now. Defer A with explicit reactivation triggers. Reject B and C.

**Rationale:**
- D fits the actual workflow — `peerloop` alias is the canonical entry, SessionStart is already a trusted surface.
- B's per-clone install friction doesn't pay for a 2-machine (MacMiniM4 + MacMiniM4-Pro) setup; warn-only hooks get ignored.
- C is overkill given D's coverage of the CC loop.
- A becomes valuable once a non-CC commit path emerges OR if SessionStart misses drift over 10+ convs; deferring it avoids premature complexity without losing the option.

**Consequences:**
- Every `/r-start` on either dev machine will now run tech-doc-sweep and surface flagged docs if any (silent otherwise). No install step needed — settings.json is committed.
- Drift introduced by commits made outside CC (rare under current workflow) will not be caught until manually running `/w-sync-docs` or `/r-end2`. Accepted risk.
- If a future conv introduces a non-CC commit path (e.g., automated bot commits, a second human contributor not using CC), Option A moves from deferred to needed.

## 2. Silent-on-clean over always-print for the drift hook
**Type:** Implementation
**Topics:** cc-workflow, docs-infra

**Trigger:** Designing the output format for `.claude/hooks/tech-doc-drift.sh`.

**Options Considered:**
1. Always print `=== TECH-DOC DRIFT ===` block with either "No drift" or the flagged list.
2. Print only when drift is flagged; silent on clean state. ← Chosen

**Decision:** Silent on clean; verbose block when drift exists.

**Rationale:** SessionStart hooks run every conv. A block that says "nothing to see" every time teaches the eye to skip the region where real drift would appear. Presence-as-signal (a pattern borrowed from Unix tools) preserves the hook's value over time.

**Consequences:** Hook output is self-calibrating. No user-facing "silent success" indicator, which is fine — the other three SessionStart hooks still print their headers, so the session-start region isn't empty.
