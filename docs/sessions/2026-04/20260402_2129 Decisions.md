# Conv Decisions - 2026-04-02

## 1. Inline Scenarios for Ephemeral Split Files
**Type:** Implementation
**Topics:** plato, split-tooling

**Trigger:** Designing plato:split output format

**Options Considered:**
1. Separate .scenario.ts files for each split side
2. Inline scenarios in .instance.ts files ← Chosen
3. Both (separate scenarios + instances)

**Decision:** Split files use inline scenarios. If promoted via cleanup, the inline scenario is extracted to its own file.

**Rationale:** Split files are ephemeral by default — creating separate scenario files for throwaway segments adds file bloat. Promotion happens only when the user explicitly chooses to keep a side.

**Consequences:** plato:split creates 2 files (not 4). Cleanup's promote path extracts the inline scenario and rewrites the instance to reference the named scenario.

## 2. Per-Side Independent Keep/Delete for Split Cleanup
**Type:** Implementation
**Topics:** plato, split-tooling

**Trigger:** User feedback that both sides should be independently asked

**Options Considered:**
1. All-or-nothing (keep both or delete both)
2. Per-side independent choice ← Chosen

**Decision:** Each side (pre/post) is independently asked: keep as named scenario, delete, or skip.

**Rationale:** Most commonly both are discarded, but occasionally one side (usually Pre) is valuable as a reusable checkpoint. flywheel-pre-9 was kept because it's a useful "enrollment-ready" snapshot point.

**Consequences:** Cleanup script supports interactive prompts (terminal) and `--keep`/`--delete` flags (non-interactive/CC sessions).

## 3. Enrollment-Aware Breadcrumb Fallback
**Type:** Implementation
**Topics:** ui, navigation, courses

**Trigger:** STUMBLE walkthrough showed "My Courses" breadcrumb for unenrolled student

**Options Considered:**
1. Always show "Courses" (simple but wrong for enrolled students)
2. Check enrollment status for fallback ← Chosen
3. Always show "My Courses" (original behavior, wrong for non-enrolled)

**Decision:** Fallback breadcrumb checks `isEnrolled`: enrolled → "My Courses", not enrolled → "Courses". Also added `via=recommendations` and `via=discover` cases.

**Rationale:** The same page is reachable from multiple entry points. The `via` param covers explicit navigation; the enrollment-aware fallback handles direct links.

**Consequences:** Only `index.astro` needed the fix — sub-pages (learn, book, sessions, etc.) require enrollment anyway.
