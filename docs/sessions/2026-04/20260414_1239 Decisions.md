# Conv Decisions - 2026-04-14

## 1. Auto-npm-install → prompt-gated `/r-start` step, not a hook
**Type:** Implementation
**Topics:** cc-workflow, dev-env

**Trigger:** `check-env.sh` surfaced a `node_modules out of sync` warning at SessionStart; user asked how to automate the follow-up.

**Options Considered:**
1. Auto-run `npm install` silently inside `check-env.sh` when drift detected
2. New SessionStart hook `auto-npm-install.sh` that runs after `check-env.sh` with visible output
3. New prompt-gated Step 5.5 inside `/r-start` that asks before running ← Chosen

**Decision:** Add a pre-computed drift check and a new Step 5.5 that prompts the user before running `npm install`.

**Rationale:** CLAUDE.md §Skills: Preserve `!` Backtick Determinism and §Critical Rule: Ask Before Deciding both discourage silent side effects. A `git pull` bringing in new deps is a moment the user benefits from seeing. Options 1 and 2 add invisible or near-invisible automation that could mask lockfile changes or silently fail on a broken lockfile.

**Consequences:** One extra prompt per conv when drift is present (rare); no prompt when in sync (common). File changes from install are attributable to the current conv because Step 5.5 runs after the counter is pushed.
