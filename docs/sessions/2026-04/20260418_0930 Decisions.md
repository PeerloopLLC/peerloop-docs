# Conv Decisions - 2026-04-18

## 1. `docNameWhitelist` as explicit config тАФ not auto-detection of ALL-CAPS patterns
**Type:** Implementation
**Topics:** timecard-tooling, docs-infra

**Trigger:** Commit bullets reference docs by ALL-CAPS stems without `.md` extension (e.g., "updated API-ENROLLMENTS")

**Options Considered:**
1. Auto-detect any ALL-CAPS hyphenated token as a potential doc name
2. Explicit whitelist in config.json with known doc stems тЖР Chosen
3. Require `.md` extension in commit message conventions

**Decision:** Explicit `docNameWhitelist` in `config.json тЖТ rTimecardDay`, guarded by `validDocs.has(stem + '.md')`

**Rationale:** Auto-detection would match non-doc ALL-CAPS tokens (acronyms, constants, error codes). Explicit whitelist is maintainable тАФ the doc set is stable and the config lives next to the other routing config. Guarding by validDocs prevents the whitelist from running ahead of the actual doc inventory.

**Consequences:** When a new reference doc is added, its stem must be added to `docNameWhitelist` to be recognized in commits without `.md`.

## 2. T3b API detection scoped to workEffort only, guarded by !isTestRelated
**Type:** Implementation
**Topics:** timecard-tooling

**Trigger:** API path detection caught `tests/api/...` paths, routing them to API Changes

**Options Considered:**
1. Apply API detection to all item sources
2. Apply only to workEffort items, guarded by !isTestRelated тЖР Chosen
3. Use more specific regex that excludes test file paths

**Decision:** T3b applies only when `item.src === 'workEffort' && !isTestRelated({ text }, rt)`

**Rationale:** Test items should always flow through the T4 isTestRelated heuristic or be caught by T3 structural signals targeting tests/. Scoping T3b to workEffort prevents test bullets from being intercepted mid-tier. The `!isTestRelated` guard adds a second layer for workEffort bullets that happen to mention test-related words.

**Consequences:** API detection only fires for work-effort items (tagged commits' structured bullets). A plain commit bullet mentioning an API path without workEffort src won't be caught тАФ which is the desired behavior since those fall to T5 defaults.
