# Conv Learnings - 2026-04-19

## 1. Substring Match Bug in grep-based Keyword Routing
**Topics:** drift-detection, docsRegistry, bash

**Context:** The SessionStart drift hook was flagging `ratings-feedback.md` whenever community/feed code changed. Root cause: the stream rule's topicKeyword `feed` matched `ratings-feedback` as a filename via `grep -qi "feed"` — "feedback" contains "feed" as a substring.

**Learning:** When using `grep -qi "$keyword"` for filename matching, any keyword that is a prefix or substring of another word will false-match. Change keywords to be the full word form (e.g. `feed` → `feeds`) so the substring doesn't appear in unrelated filenames. Alternatively, use `-w` for word-boundary matching, but note `-w` won't match plural forms.

## 2. Stored Baseline Pattern for Drift Detection
**Topics:** drift-detection, bash, CI

**Context:** `tech-doc-sweep.sh` used `HEAD~5` as a proxy for "recent changes," which caused two problems: (1) drift from old resolved changes kept appearing; (2) the window was arbitrary, not tied to the last time docs were actually reviewed.

**Learning:** A stored baseline SHA file (`.drift-baseline-sha`) solves both problems: records the exact code commit where docs were last reviewed, advances explicitly after each /r-end, gives CI a precise diff of what changed in a PR. The `CODE_CHANGES_OVERRIDE` env var (already in the script for testing) doubles as the CI injection point — no new code path needed.

## 3. The New Drift System is Additive, Not a Replacement
**Topics:** drift-detection, doc-sync, architecture

**Context:** User asked whether the new tech-doc-sweep pipeline replaces /w-sync-docs or the /r-end docs agent.

**Learning:** The three tools serve distinct roles and are complementary: `tech-doc-sweep` gives topical signals ("booking code changed — check these docs"), `/w-sync-docs` verifies structural correctness ("are the file counts and paths in TEST-COVERAGE.md accurate?"), and the `/r-end` docs agent does the actual prose updating work. None can replace the others. The new system adds zero-effort ambient awareness and a CI gate — both previously absent.

## 4. react-big-calendar.md Was a Research Doc with No Codebase Adoption
**Topics:** drift-detection, docsRegistry

**Context:** The `react-big-calendar.md` doc was being falsely triggered by any booking-related code change via the `calendar` topicKeyword. Investigating revealed the library has zero occurrences in `src/` and is not in `package.json`.

**Learning:** When a vendor doc covers a library that was researched but not adopted, its topicKeyword should require the library's import string (e.g. `react-big-calendar|BigCalendar`) rather than a generic keyword. This makes the doc self-silencing until the library is actually adopted — the precise behavior you want for unadopted-but-documented libraries.
