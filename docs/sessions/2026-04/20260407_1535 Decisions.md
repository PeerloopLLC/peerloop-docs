## 1. Remove Cloudflare KV from Codebase
**Type:** Implementation
**Topics:** cloudflare, infrastructure

**Trigger:** PLAN.md review — KV-CONSISTENCY block assessment

**Options Considered:**
1. Keep KV bindings (status quo)
2. Remove KV bindings + switch session driver
3. Remove KV bindings, keep session config as-is with updated comment <- Chosen

**Decision:** Remove all KV namespace bindings from wrangler.toml, delete health check endpoint, keep session config in astro.config.mjs with note about re-adding KV post-launch for feature flags.

**Rationale:** KV is completely unused — no application code references it. The adapter's session config stays to suppress warnings. Feature flags are noted as Post-MVP #19.

**Consequences:** wrangler.toml simplified (3 KV blocks removed), health endpoint deleted, KV namespaces still exist in Cloudflare dashboard but are unbound.
