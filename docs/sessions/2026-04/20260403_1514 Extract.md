## Uncategorized

### PLATO Stripe Account Status
The PLATO seed creates Stripe accounts with `status = 'pending'` and `payouts_enabled = 0` (mock accounts from the `self-certify-creator` step). The SQL seed has `status = 'active'` and `payouts_enabled = 1`. This means UI elements gated on active Stripe status may render differently between the two seed paths. Not a bug — but testers may see "Connect Stripe" prompts on creator/teacher pages that wouldn't appear with the SQL seed.

### Two Parallel Seed Paths Not Yet Unified
The SQL seed chain (`db:setup:*`) and PLATO seed (`plato:seed*`) are parallel, incompatible paths. PLATO generates UUIDs; SQL overlays reference hardcoded IDs. No plan exists to unify them. This is manageable now (3 testers, one staging env) but will become confusing as more developers onboard.
