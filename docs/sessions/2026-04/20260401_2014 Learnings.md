# Conv Learnings - 2026-04-01

## 1. PLATO/STUMBLE Terminology Is Operator Protocol
**Topics:** plato, stumble, workflow

**Context:** User clarified how they use the terms in conversation

**Learning:** "PLATO run [instance]" = automated API execution. "STUMBLE through it" = browser-automate the same instance's WalkthroughCheckpoints. The instance is always the common unit.

---

## 2. Backward-Compatible API Design Protects Test Infrastructure
**Topics:** api, testing, schema

**Context:** Adding `level` column to `user_tags` and new `tags` format to onboarding API

**Learning:** Accepting both `tags: [{tagId, level}]` (new) and `tagIds: ["id"]` (legacy, defaults level to 'beginner') meant zero changes to PLATO steps, scenarios, or instances. The entire test suite passed without modification.

---

## 3. Direct DB Changes Bypass CurrentUser Cache
**Topics:** currentuser, cache, stumble

**Context:** Granting creator role via DB during STUMBLE — sidebar didn't update

**Learning:** `bumpUserDataVersion()` must be called for CurrentUser cache to refresh. Direct DB changes (common during STUMBLE setup) require either bumping `data_version` on the users table, or clearing cookies and re-logging in.

---

## 4. STUMBLE Service Boundaries Are Predictable
**Topics:** stumble, services, testing

**Context:** Flywheel walkthrough hit Stripe checkout (worked) then would need BBB (unavailable)

**Learning:** The browser-walkable boundary aligns with external service dependencies: registration/course creation/enrollment are all local, while session booking/video/completion need external services. Composable segments should align with these boundaries.

---

## 5. Stripe Self-Healing Creates Enrollment But Skips Side Effects
**Topics:** stripe, enrollment, self-healing

**Context:** Real Stripe payment without webhook forwarding

**Learning:** The success page self-healing correctly creates enrollment + transaction + expectations, but doesn't assign teacher or increment student_count. These are webhook handler side effects. The self-healing is a safety net, not a full replacement.

---

## 6. Publish Gate Checklist Has Silent Failure
**Topics:** ux, creator, publishing

**Context:** Clicking "Publish Course" with incomplete checklist items

**Learning:** The publish button gives no feedback (no toast, no highlight) when requirements aren't met. User must manually scan the checklist to find what's missing. Also, "Topic selected" showed unchecked despite a topic being set — possible evaluation bug.
