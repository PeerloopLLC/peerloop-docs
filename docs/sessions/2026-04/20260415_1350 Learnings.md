# Conv Learnings - 2026-04-15

## 1. Audit reports can overcount if not drilled into per-site before scoping
**Topics:** docs, role-audit, methodology

**Context:** The ROLE-AUDIT report listed "10 inline `SELECT is_admin` sites." When implementing [RA-ADM], a site-by-site re-read revealed only 5 matched the current-user admin pattern, 2 were multi-field (`is_admin, can_moderate_courses`), 3 were target-user fetches with existence+deletion filtering, 2 were set queries for admin-list fan-out. The report's category-C count was technically correct as "sites with any `is_admin` mention" but misleading as "helper candidates."

**Learning:** When producing an audit report that seeds follow-up tasks, classify sites by *query shape* not just *keyword match*. A single helper covers one shape; sites with different shapes need separate helpers or stay inline. Include an explicit "sites that look similar but need different treatment" callout.

**Pattern:** For future audit reports, structure §C (ad-hoc patterns) as a table with columns `File | Query shape | Helper candidate? | Why/why-not`. Prevents downstream scope confusion.

## 2. "Proper cleanup" sometimes requires widening beyond the stated task
**Topics:** role-audit, refactoring, scope

**Context:** [RA-RO] was estimated at 15 min as a mechanical helper extract. Inspection revealed the helper encoded a dead `'teacher'` branch post-Conv-120 (COMMUNITY-TEACHER-KILL); extracting mechanically would preserve stale types in `CommunityTabs.tsx`. The user's "proper cleanup" directive meant the narrow task expanded to narrow 4 SSR loader types + `CommunityTabs.tsx` types + simplify `RoleBadge` + 18 Astro edits.

**Learning:** When a "pure DRY extract" reveals the thing-being-extracted encodes a historical bug, the narrow task becomes a widening one. Surface the discovery with red emoji prefix, present the scope options, let user decide — don't silently either preserve dead code or auto-widen.

**Pattern:** The RoleBadge collapse (3-key config → single conditional) is a useful template for "config-lookup components where only one key is meaningful" — when types narrow, the lookup disappears.

## 3. Helper coverage decisions should turn on query shape, not caller intent
**Topics:** refactoring, auth, helpers

**Context:** Initial [RA-ADM] scoping separated "is current user admin?" (5 sites) from "is this target user admin?" (3 sites, moderator endpoints) — treating them as different. Under user challenge, I realized the *query* is identical (`SELECT is_admin FROM users WHERE id = ?`); only the caller's downstream reasoning differs. A helper doesn't care why you're asking. But the moderator sites' queries ALSO select `id, name, handle, deleted_at` — they're doing user-validation in one query, not just admin-check. Those genuinely stay inline.

**Learning:** "Same SQL shape → same helper" is the right discriminator. "Same caller intent" is a red herring. And when a caller's query is a *superset* of the helper's query (extra fields, extra filters), a helper would force splitting one query into two round-trips — often worse ergonomics than leaving it inline.

**Pattern:** `isUserAdmin(db, userId)` covers both self-checks and target-checks. `getUserPermissionFlags(db, userId)` for multi-flag. `getAllAdminUserIds(db)` for set fan-out. Three shapes, three helpers, no forcing.

## 4. Generated files inside source dirs break find-based scripts
**Topics:** infra, skills, scripts

**Context:** Astro 5's content-collection types generate `src/pages/api/**/.astro/content.d.ts` shims under API route directories. `find src/pages/api -name "*.ts"` picks them up, sync-gaps.sh then tries to find them in API docs, flags them as undocumented routes.

**Learning:** `find` patterns in source-scan scripts should preemptively exclude `.astro/` (and any other tool-generated artifact dirs) even if only one directory currently causes false positives. The Astro content-types location can move per-version.

**Pattern:** `find ... -not -path '*/.astro/*'` is the narrow exclusion. For defense in depth, scripts that walk `src/` could also exclude `**/dist/**`, `**/.next/**`, `**/.astro/**`, `**/node_modules/**` as a standard preamble.
