# Conv Learnings - 2026-04-12

## 1. futureUTC(0, hour) Is Not Future-Safe
**Topics:** testing, time-dependent

**Context:** Late cancellation tests used `futureUTC(0, 14)` which sets today at 14:00 UTC — past tense after that hour.

**Learning:** For time-sensitive tests that need a future timestamp within 24h, use `Date.now() + N hours` instead of fixed UTC hours. The fixed-hour pattern silently fails when tests run after that hour.

**Pattern:** `const soonStart = new Date(Date.now() + 4 * 60 * 60 * 1000);`

---

## 2. PLATO Snapshot Pipeline Requires File-Level Flag
**Topics:** plato, testing

**Context:** Put `snapshot: true` on the `PlatoInstance` object but it must be on the `PlatoInstanceFile` level.

**Learning:** The `snapshot` property in PLATO is a file-level concern, not instance-level. It controls whether the SQLite DB is copied to `tests/plato/snapshots/` after the API run.

---

## 3. Miniflare metadata.sqlite Coexists with App DB
**Topics:** cloudflare, d1, testing

**Context:** `plato-restore-snapshot.js` found 2 .sqlite files in the D1 directory and failed.

**Learning:** Miniflare stores its own `metadata.sqlite` alongside the app's D1 database. Any script scanning for the app DB file must filter it out.

---

## 4. Authenticated Users Should Not Reach /login
**Topics:** auth, ux

**Context:** Login page uses AppLayout with sidebar — authenticated users see login modal over their authenticated sidebar.

**Learning:** Auth pages (/login, /signup) need server-side `getSession()` checks to redirect authenticated users to /dashboard. Without this, the modal-over-sidebar creates a confusing UX.

---

## 5. Denormalized Counters Must Be Updated in Seed SQL
**Topics:** database, migrations

**Context:** The Commons showed "0 members" because seed SQL inserted community_members rows but never updated the `member_count` field.

**Learning:** When seed data inserts into child tables, always add a corresponding UPDATE to the parent table's denormalized counter. The application's runtime code paths (join endpoint, auto-join on signup) handle this, but seed data bypasses those paths.

---

## 6. Codecheck Grep Patterns Need All Render Variants
**Topics:** tooling, codecheck

**Context:** Error-captured-never-rendered check missed `error ||` pattern, producing 4 false positives.

**Learning:** React components render error state via multiple patterns: `error &&`, `error ?`, `error !==`, `{error}`, `error.message`, AND `error ||`. The codecheck grep must include all variants to avoid false positives.
