# Conv Learnings - 2026-04-01

## 1. Silent Field Name Mismatch in API POST Body
**Topics:** onboarding, api, debugging

**Context:** Onboarding tag selections were never persisting — the POST body sent `topicInterests` but the API destructured `tagIds`.

**Learning:** When a form save silently succeeds but data doesn't persist, check that the POST body field names match what the API destructures. The API returned 200 OK even when `tagIds` was undefined — it just skipped the insert. No error surfaced client-side.

**Pattern:** Always verify field name alignment between frontend `JSON.stringify()` and backend destructuring, especially after renames.

## 2. Bulk String Replacement with perl on macOS
**Topics:** tooling, bulk-edit

**Context:** Needed to strip ` | PeerLoop` from 77 page files. macOS `sed` has limitations.

**Learning:** `grep -rl 'pattern' dir/ | xargs perl -pi -e 's/pattern//g'` is the reliable bulk-replace pattern on macOS. This was validated during the TERMINOLOGY block (Sessions 346-356) and reconfirmed here.

## 3. SignupForm Reads URL Params at Runtime (No Prop Threading)
**Topics:** astro, react, architecture

**Context:** The email pre-fill for ModeratorInvite works without the Astro page passing `email` as a prop.

**Learning:** In Astro's multi-island architecture, React components can read URL params directly via `window.location.search` at hydration time, bypassing the Astro server layer. This is less discoverable (not in the props interface) but avoids threading values through Astro > Layout > Modal > Form.
