# Conv Decisions - 2026-04-05

## 1. Editor Migration: Cursor to VS Code
**Type:** Strategic
**Topics:** developer-tooling, cc-workflow

**Trigger:** User decided to move from Cursor to VS Code as the editor housing Claude Code terminal.

**Options Considered:**
1. Stay with Cursor
2. Move to VS Code <- Chosen

**Decision:** Migrate all editor references from `cursor` to `code` across skills, hooks, config, templates, and docs.

**Rationale:** User decision — VS Code chosen as replacement editor.

**Consequences:** 9 files updated across skills (4), global config (3), template (1), and docs (1). Session logs left as historical records. `config.json` already had `"editor": "code"`.

> **Insight:** Editor references are scattered across skills, hooks, config templates, permission entries, and docs. When migrating editors, a full-repo search for the old editor command name is necessary — `config.json` alone is not sufficient.
