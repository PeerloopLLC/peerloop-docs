# Conv Learnings - 2026-04-06

## 1. SkillSync Replacement Rules Don't Cover File Paths
**Topics:** skills, dual-repo, tooling

**Context:** `/r-timecard-day` wrote to `/tmp/timecard.md` in Peerloop but `.timecard.md` in spt-docs — drift undetected because skillSync replacements only handle repo names and terminology, not arbitrary file paths.

**Learning:** When porting skills between repos, check for hardcoded paths that fall outside the replacement rules. File paths, temp dirs, and output locations are common drift vectors.

---

## 2. Block Metadata Requires Diff-Based Determination
**Topics:** commits, metadata, accuracy

**Context:** Conv 084 was tagged `Block: STUMBLE-AUDIT` despite being purely editor migration work, because the skill defaulted to the PLAN.md Focus block.

**Learning:** The `Block:` field in commits must be determined from the actual diff, not inherited from whatever PLAN.md currently says. This is a downstream data quality issue — inaccurate Block metadata propagates into timecards.

---

## 3. Structured Tags Enable Categorical Timecard Sections
**Topics:** commits, timecards, workflow

**Context:** API, Page, Role, and Infra changes were buried in generic "Changes:" bullets with no way to automatically categorize them.

**Learning:** Prefixed tag lines (`API:`, `Page:`, `Role:`, `Infra:`) in commit bodies are greppable, optional, and backward-compatible. They follow the existing `User-facing:` / `Admin-facing:` pattern. The key design constraint: tags in commits, extraction in timecards — the commit is the single source of truth.
