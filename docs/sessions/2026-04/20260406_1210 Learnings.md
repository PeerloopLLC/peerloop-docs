# Conv Learnings - 2026-04-06

## 1. SQLite datetime() Format Causes JavaScript Parsing Bug in Tests
**Topics:** testing, d1, sqlite

**Context:** Session complete tests failed because `datetime('now', '-1 hour')` stores space-separated timestamps that JavaScript's `new Date()` parses as local time, shifting by timezone offset.

**Learning:** Always use `strftime('%Y-%m-%dT%H:%M:%fZ', ...)` in test data INSERTs when the endpoint code will parse the timestamp with JavaScript `new Date()`. This is the same rule as the SQLite Datetime Rule in CLAUDE.md but applies to test setup SQL, not just production code.

**Pattern:** `strftime('%Y-%m-%dT%H:%M:%fZ', 'now', '-1 hour')` instead of `datetime('now', '-1 hour')`

## 2. Audit Agents Can Report False Positives — Verify Before Acting
**Topics:** testing, workflow

**Context:** 5 audit agents reported 3 "critical gaps" (zero test coverage). 2 of the 3 were false positives — test files existed but agents missed them.

**Learning:** When agents report missing files, verify with Glob/Read before writing new tests. Agent file searches can miss existing test coverage, especially when file paths contain brackets or special characters.

## 3. Partial Unique Index Blocks Test Data INSERT OR IGNORE
**Topics:** testing, d1

**Context:** Booking limit test failed with FK constraint because `INSERT OR IGNORE INTO enrollments` was silently skipped by partial unique index `idx_enrollments_one_active ON enrollments(student_id, course_id) WHERE status IN ('enrolled', 'in_progress')`.

**Learning:** When creating test enrollments for the same course, use a different student_id to avoid the partial unique index. `INSERT OR IGNORE` silently skips without error, then subsequent session INSERTs fail with FK constraint.

## 4. reset-password Must Not Use parseBody — Security Exception
**Topics:** auth, security

**Context:** After adding `parseBody()` to all endpoints, reset-password tests failed because the endpoint intentionally returns 200 on ALL inputs (including malformed JSON) to prevent email enumeration.

**Learning:** Security-sensitive endpoints that intentionally suppress errors (reset-password, login timing) should NOT use `parseBody()`. They need their own error handling that returns the same response regardless of input quality.

---

## 5. Request Body Stream Can Only Be Read Once
**Topics:** api, webhooks

**Context:** BBB webhook handler tried `request.json()` first, then `request.text()` on failure — but the stream was already consumed by the first read.

**Learning:** When handling unknown content types, read the body as text first (`request.text()`), then try `JSON.parse()` on the string. Never call `request.json()` followed by `request.text()` in a catch — the stream is consumed.
