# Conv Learnings - 2026-04-13

## 1. Astro 6 + @astrojs/cloudflare@13 Targets Workers, Not Pages
**Topics:** astro, cloudflare, deployment

**Context:** CF Pages build failed with 3 wrangler.json validation errors after merging Astro 6 code to staging

**Learning:** Starting with @astrojs/cloudflare@13, the adapter generates Workers-format output. Cloudflare Pages is no longer supported. The Astro maintainer explicitly confirmed this. Cloudflare's own Pages Astro docs are stale and don't mention this.

**Pattern:** The generated `dist/server/wrangler.json` contains Workers-specific fields (`triggers`, `assets.binding = "ASSETS"`) that Pages validation rejects.

---

## 2. Adapter Config Path Affects Build Output Structure
**Topics:** astro, cloudflare

**Context:** Postbuild script worked locally but failed on CF because `dist/server/wrangler.json` didn't exist

**Learning:** The `platformProxy` (old) vs `remoteBindings` (new) adapter config option causes different code paths in the build. With `platformProxy`, the adapter may not generate a wrangler.json at all, or generates it in a different location. Always search for generated files rather than hardcoding paths.

---

## 3. gh CLI Installation on MacMiniM4
**Topics:** dev-environment, tooling

**Context:** Attempted to create PR via `gh` but it wasn't installed

**Learning:** `gh` (GitHub CLI) was not installed on MacMiniM4. Installed via `brew install gh` (v2.89.0). Requires interactive `gh auth login` — user must run this themselves via `!` prefix in Claude Code.
