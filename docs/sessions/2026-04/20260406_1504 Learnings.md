# Conv Learnings - 2026-04-06

## 1. getNow() Sweep Incompleteness Pattern
**Topics:** timezone, testing, code-quality

**Context:** Conv 089 did a "comprehensive" getNow() sweep across 22 files, but Conv 090 found 25+ more files still using bare `new Date()` for business logic.

**Learning:** Post-hoc sweeps are inherently incomplete when the codebase is large and the unsafe pattern is the language default. Each sweep catches what was visible at the time, but copy-paste, new endpoints, and overlooked files mean the problem recurs. A preventive mechanism (pre-commit hook, CI gate) is needed to make this durable.

**Pattern:** The lint-timezone.sh script exists as a check, but it's not enforced in CI or pre-commit. Making it a hard gate would prevent recurrence.

## 2. Task Scope Discovery During Execution
**Topics:** workflow, planning

**Context:** The GE task was scoped as "add exempt comments to ~20 uses" but investigation revealed ~30 needed conversion and only ~13 needed exemption.

**Learning:** Always re-assess scope at execution time rather than trusting the estimate from when the task was created. The RESUME-STATE.md description was based on the original lint run during Conv 089, which may have had different results or was interpreted differently.
