## 2. Broader docs audit over targeted version grep
**Type:** Implementation
**Topics:** docs-infra, package-updates

**Trigger:** Phase 6 checklist item "update any remaining docs referencing old versions."

**Options Considered:**
1. Targeted grep for specific old versions that were bumped — fast, narrow
2. Broader audit of reference docs for stale version mentions ← **Chosen**

**Decision:** Run a broader audit across `docs/**` for every bumped package's pre-bump version string, filtering out `docs/sessions/**` as frozen history.

**Rationale:** A targeted grep on *only* bumped versions can miss adjacent stale context — e.g., "Astro 5.x officially supports X" in a decision record where the rationale is tied to a specific major. A broader sweep caught 3 stale Node-24 references that the narrow pattern would have missed.

**Consequences:** Larger scan surface (sessions archive noise), but higher catch rate. Confirmed sessions archive is truly frozen — zero need to touch historical session files.
