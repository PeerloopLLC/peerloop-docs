# Conv Decisions - 2026-03-26

## 1. Remove role-specific dashboard links from AppNavbar
**Type:** Strategic
**Topics:** navigation, ux, dashboard

**Trigger:** Conv 032 created unified /dashboard; Conv 033 reviewed CurrentUser patterns and found nav crowding

**Options Considered:**
1. Keep all 4 nav items (Dashboard + Learning + Teaching + Creating)
2. Remove 3 role-specific items, keep Dashboard as single entry point ← Chosen
3. Add nav badges to role-specific items (Option D from discussion)

**Decision:** Remove Learning, Teaching, Creating from AppNavbar. Dashboard is the single nav entry point. Role-specific pages remain accessible via direct URL and DashboardLinks button row.

**Rationale:** Eliminates nav crowding. DashboardLinks component already provides drill-down. Badge requirements are deferred to a separate block.

**Consequences:** 3 menu items removed, 2 icon imports cleaned up. become-creator CTA retained for non-creators.

## 2. Standardize dashboard data flow: CurrentUser for identity, API for transactional
**Type:** Implementation
**Topics:** architecture, currentUser, api-design

**Trigger:** Review found /learning, /teaching, /creating used 3 different patterns for user data

**Options Considered:**
1. Option A: Put everything in CurrentUser (rejected — cache staleness for earnings)
2. Option B: Tier 2 summaries in CurrentUser (rejected — complexity for marginal gain)
3. Option C: Unified API endpoint (deferred — would consolidate 3 endpoints)
4. Option D: Pending counts in CurrentUser for nav badges (unnecessary after nav item removal)
5. Refactor dashboards to use CurrentUser for identity, keep APIs for transactional data ← Chosen

**Decision:** All dashboards use useCurrentUser() for identity (name, handle, capabilities). APIs return only transactional data (earnings, stats, pending counts, rosters). is_available stays in teacher API (operational state, not identity).

**Rationale:** Consistent pattern across all 4 dashboards. Eliminates identity re-fetching. Lighter API payloads. No CurrentUser expansion needed.

**Consequences:** TeacherDashboard and CreatorDashboard refactored. API responses trimmed. 6 test files updated (159 tests). Dashboard badges deferred to separate block.
