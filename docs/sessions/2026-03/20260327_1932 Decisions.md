# Conv Decisions - 2026-03-27

## 1. Decompose CourseTabs into Per-Tab Sub-Components
**Type:** Implementation
**Topics:** component-architecture, refactoring

**Trigger:** Role tabs buried below the fold due to Conv 039 wrapping approach; user wanted them in the same tab bar

**Options Considered:**
1. Extend CourseTabs with `extraTabs` prop while keeping monolith — Rejected (1392 lines grows to 2000+)
2. Decompose into per-tab sub-components + extraTabs prop — Chosen
3. Fork CourseTabs entirely for explore page — Rejected (code duplication)

**Decision:** Break CourseTabs (1392 lines) into 5 sub-component files + thin shell (~195 lines). Add `extraTabs` and `basePath` props for role tab injection.

**Rationale:** Each render function was already self-contained. Decomposition solves both the monolith problem and the role tab UX problem simultaneously. The shell stays role-agnostic; the explore page passes role tabs via props.

**Consequences:** CourseTabs.tsx: 1392 to 195 lines. 5 new files in course-tabs/. ExploreCourseTabs.tsx: 179 to 95 lines. Both routes work. 6092 tests pass.

## 2. Use Pick Types Instead of Full DB Types for Component Props
**Type:** Implementation
**Topics:** typescript, component-api

**Trigger:** TypeScript errors when test data didn't include all DB fields (course_id, display_order on CourseObjective)

**Options Considered:**
1. Add missing fields to all test data — Rejected (unnecessary test verbosity)
2. Use Pick<DBType, fields> for component types — Chosen
3. Keep local simplified interfaces (pre-refactor approach) — Rejected (no connection to DB types)

**Decision:** Component-facing types use `Pick<DBType, 'field1' | 'field2'>` to select only the fields the component renders.

**Rationale:** Components should accept DB objects but not require fields they don't use. Tests should only provide what the component cares about.

**Consequences:** Zero type errors. Tests pass without modification. Full DB objects from SSR are accepted via structural subtyping.
