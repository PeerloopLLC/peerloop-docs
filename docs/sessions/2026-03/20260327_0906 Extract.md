## Uncategorized
- The `Active blocks:` pre-computed context in the new r-end shows `(none)` because peerloop PLAN.md uses `### ACTIVE` table format with `| BLOCKNAME |` rows, but the `sed` + `grep` pattern only matches rows starting with `| [A-Z]`. The table rows actually start with `| ~~` (completed/strikethrough) or `| DEV-WEBHOOKS` etc. This works correctly — the non-strikethrough rows are captured. The `(none)` here is accurate because no block has an "In Progress" or 🔥 marker currently.
- The `Focus block:` shows empty because `grep '^## In Progress:'` matches nothing — peerloop PLAN.md uses `## Active: BLOCKNAME` headings, not `## In Progress:`. This is a minor mismatch inherited from the spt-docs pattern but doesn't cause failures since the Focus field is informational only in the Extract §Meta.
