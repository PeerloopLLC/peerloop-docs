### DISCOVER-FEEDS Realignment

**User:** "There has been a little drift. The slideout panel for discover AppNavBar item says "My Feeds" It should just be feeds. BUt your destination link does match you MyFeeds format The problem is that discover/feeds should follow the same role-aware workings as /discover/courses and /discover/communities. It should have a view for vistors and new members, It should have a carousel that says "Based on your Interests""

**Claude:** Audited the full discover feeds ecosystem. Found significant drift: `/discover/feeds` was auth-required, had no SSR data, no recommendation carousel, and "My Feeds" branding — unlike courses/communities which are visitor-accessible.

**User:** "You make an excellent point. WHat good is discovering a feed if you can't directly join it. You have to join the community first. SO mayne the premise of discover feeds is erroneous from the user's side. Let's discuss"

**Claude:** Analyzed the fundamental problem: feeds are derivative artifacts (you get them by joining communities/enrolling in courses), not first-class entities. Presented 3 options: A) kill `/discover/feeds`, B) reframe as inventory, C) transform into "Conversations Worth Joining."

**User:** "Option C is intriguing. The goal would be to surface feed content that matches their interest. That could be usefull if they get the chance to 1) join the community of the feed (or go to the /discover/community page) or 2) visit the /discover/course/{course name} page/tab for the course feed. We would need to assess Smart Feeds to see how much overlap is there"

**Claude:** Audited Smart Feed system. Found `getDiscoveryCandidates()` already surfaces content from non-member feeds matched by topic/category with vitality ranking — ~70% overlap. Presented C1 (feed-level, reuse existing) vs C2 (thread-level, new schema).

**User:** "For C2 vs C1, how would what a user see be different?"

**Claude:** Provided concrete visual mockups of both options showing the UX difference: C1 shows "this feed is active" with one post preview; C2 shows specific conversation threads with multiple participants.

**User:** "C1 for now, as long as it is obvios what the user can do about each entry (go to community, go to course, as applicable)"

**Claude:** Built DISCOVER-FEEDS realignment: new `GET /api/feeds/discover` endpoint (visitor + auth paths), `DiscoverFeedsGrid` component with clear CTAs, rewrote `/discover/feeds` page (removed auth wall, added discovery section + "Your Feeds" for auth users), fixed DiscoverSlidePanel "My Feeds" → "Feeds", updated discover index description, updated Smart Feed CTA URLs to `/discover/` paths. 7 new endpoint tests. 350 files, 6182 tests passing.
