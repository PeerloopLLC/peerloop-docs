# Conv Decisions - 2026-03-29

## 1. Keep Old Test Directory Names (Cosmetic)
**Type:** Implementation
**Topics:** testing, tag-taxonomy

**Trigger:** `tests/api/admin/categories/` directory still exists with old name after TAG-TAXONOMY rename

**Options Considered:**
1. Rename directory to `tests/api/admin/topics/` — risk of breaking import resolution
2. Keep old directory name, update imports to point to new API paths <- Chosen

**Decision:** Keep `tests/api/admin/categories/` directory name as-is. Tests inside import from the new `/api/admin/topics` endpoints. Marked with † in TEST-COVERAGE.md.

**Rationale:** Vitest resolves by import path not file path. Renaming the directory risks breaking test discovery. Cosmetic cleanup can happen later.

**Consequences:** TEST-COVERAGE.md has † footnote explaining the naming mismatch.
