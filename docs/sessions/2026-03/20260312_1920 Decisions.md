# Session Decisions - 2026-03-12

## 1. Normalize Hardcoded Slugs in Route-Matrix Scanner
**Type:** Implementation
**Topics:** docs-infra, astro

**Trigger:** `/community/the-commons` flagged as broken target — false positive since it's a valid concrete instance of `/community/[slug]`.

**Options Considered:**
1. Add to an exclusion list (skip entirely) — simple but hides the link from the matrix
2. Add a `LITERAL_SLUG_ROUTES` regex normalizer in `addLink()` ← Chosen
3. Hardcode it in `normalizeDynamic()` — less extensible

**Decision:** Added regex-based `LITERAL_SLUG_ROUTES` array that normalizes hardcoded literal paths to their dynamic route equivalents before adding to the link map.

**Rationale:** This preserves the link in the matrix (mapped to the correct dynamic route) while being extensible for future hardcoded slugs. Better than an exclusion list which would hide the connection entirely.

**Consequences:** The scanner now correctly maps `/community/the-commons` → `/community/[slug]` in the adjacency matrix.

## 2. Remove Dead Certificate Link Rather Than Create Stub Page
**Type:** Implementation
**Topics:** astro

**Trigger:** `/course/[slug]/certificate` referenced by 6 course tab pages but no page exists.

**Options Considered:**
1. Create a stub `/course/[slug]/certificate.astro` page — premature, feature not built yet
2. Link to `/verify/[id]` — requires certificate ID which LearnTab doesn't have
3. Remove the link with a TODO comment ← Chosen
4. Link to `/learning` — not the right UX

**Decision:** Removed the "View Certificate" `<a>` tag from LearnTab's completion card and replaced with a TODO comment for when the certificate page is built.

**Rationale:** Can't link to `/verify/[id]` without fetching the certificate ID first (adds unnecessary API call for a feature that isn't built). Creating a stub page adds dead code. The TODO preserves intent.

**Consequences:** The completion celebration card in LearnTab now only shows the "Write a Review" button. Certificate viewing will be added when that feature is built.

## 3. Route "Write Review" to Learn Tab Instead of Non-Existent Review Page
**Type:** Implementation
**Topics:** astro

**Trigger:** MyCourses linked to `/course/[slug]/review` which doesn't exist. Review functionality is already a modal in LearnTab.

**Options Considered:**
1. Create `/course/[slug]/review.astro` page — duplicates existing modal functionality
2. Change link to `/course/[slug]/learn` where the review modal lives ← Chosen
3. Convert the `<a>` to a button with inline modal — MyCourses doesn't have the review modal imported

**Decision:** Changed the "Write Review" href in MyCourses from `/course/${course.slug}/review` to `/course/${course.slug}/learn`, which is where the `CourseReviewModal` already exists.

**Rationale:** The learn tab already shows the review button in its completion card. Sending the user there avoids duplicating modal infrastructure in MyCourses. The learn tab also provides better context (seeing the completed modules before reviewing).

**Consequences:** Users clicking "Write Review" from My Courses now land on the learn tab where they can write their review via the existing modal.
