# Session Decisions - 2026-03-12

## 1. Delete site-map.md Rather Than Maintain It
**Type:** Strategic
**Topics:** docs-infra

**Trigger:** site-map.md was a static artifact (last generated 2026-01-28) from the deleted PageSpec system. The generation script was removed in Session 307. The file had 30+ references to CCNT (Course Content), a page that no longer exists.

**Options Considered:**
1. Keep as static historical reference with stale-notes ← Initially done
2. Delete it — url-routing.md and route-stories.md are the maintained sources of truth ← Chosen
3. Rebuild it with a new script generating Mermaid diagrams from .astro pages

**Decision:** Delete site-map.md. Current page architecture is maintained in url-routing.md (routes), route-stories.md (story mapping), and page-connections.md (auto-generated interconnections).

**Rationale:** The file couldn't be regenerated (source data deleted), was deeply stale, and duplicated information available in maintained docs. Adding stale-notes to 30+ Mermaid diagram references would be worse than removing it.

**Consequences:** 732 lines removed. One reference in DOCS-REORG-MAP.md left as historical record.

## 2. Consolidate Grid TSV Generation Into route-matrix.mjs
**Type:** Implementation
**Topics:** docs-infra, dual-repo

**Trigger:** ROUTE-GRID-MAP.tsv and ROUTE-GRID-REFERENCE.tsv were created by a one-off `/tmp/build-route-grid.js` (Session 367) that no longer existed. The TSVs were stale and couldn't be regenerated.

**Options Considered:**
1. Recreate the one-off script as a permanent separate script
2. Fold grid generation into the existing route-matrix.mjs ← Chosen
3. Delete the grid TSVs and rely only on the adjacency matrix

**Decision:** Extend route-matrix.mjs to generate all 4 outputs: page-connections.md, ROUTE-ADJACENCY-MATRIX.tsv, ROUTE-GRID-REFERENCE.tsv, and ROUTE-GRID-MAP.tsv. Added `npm run route-matrix` entry.

**Rationale:** One command, one script, four outputs. No separate scripts to maintain. The grid TSVs provide a visual spatial layout and per-type connection breakdown that the raw adjacency matrix doesn't — they have distinct value worth preserving.

**Consequences:** `npm run route-matrix` regenerates everything. Old one-off script pattern eliminated.

## 3. Create DOC-SYNC-STRATEGY Block for Next Session
**Type:** Strategic
**Topics:** docs-infra, workflow

**Trigger:** Session 383's docs sweep revealed systemic drift: 5 stale architecture docs, 258 undocumented test files, incorrect output paths in generators, missing API docs. The pattern repeats because there's no automated way to detect when docs fall out of sync with code.

**Options Considered:**
1. Accept manual maintenance as-is (status quo)
2. Expand /w-docs to cover more files
3. Design a comprehensive doc-sync strategy examining generated vs hand-written docs, drift detection, and automation ← Chosen

**Decision:** Added DOC-SYNC-STRATEGY block to PLAN.md for examination next session. Will evaluate which docs should be generated from code, which need automated staleness checks, and whether hooks or CI can flag drift.

**Rationale:** Fixing individual stale docs is treating symptoms. The root cause is that manually-written docs describing generated artifacts have no enforcement mechanism. This needs a strategic solution, not more manual sweeps.

**Consequences:** Block added to PLAN.md with problem statement, root causes, and questions to answer.
