# Session Decisions - 2026-03-16

## 1. Standardize Test Doc Paths to Repo-Root-Relative
**Type:** Implementation
**Topics:** docs-infra, testing

**Trigger:** Automated diffing of TEST-COVERAGE.md against disk files failed because API paths used section-relative format (`admin/dashboard.test.ts`) while other categories used `tests/lib/...`. Required an awk normalization script to compare.

**Options Considered:**
1. Section-relative paths (shorter, matches section header context)
2. Repo-root-relative paths (longer, unambiguous, copy-paste-friendly) <- Chosen
3. Full absolute paths (too verbose, machine-dependent)

**Decision:** All test file paths in backticks across all 5 TEST docs use repo-root-relative format: `tests/api/...`, `tests/lib/...`, `tests/pages/...`, `e2e/...`.

**Rationale:** Repo-root-relative paths work as direct `npx vitest run` arguments, are grepable without section context, and diff cleanly against `find` output. The slight verbosity is worth the tooling compatibility.

**Consequence:** Applied via perl one-liner to TEST-COVERAGE.md. Other 4 docs already used this format.
