# Conv Decisions - 2026-03-29

## 1. useAuthStatus Hook Design — Separate Hook, Not Modified Return Type
**Type:** Implementation
**Topics:** current-user, react-hooks, api-design

**Trigger:** Need to distinguish "loading" from "visitor" in components showing skeleton loaders

**Options Considered:**
1. Change `useCurrentUser()` return type to `{ user, authStatus }` — breaks all 14 consumers
2. Add separate `useAuthStatus()` hook — zero breaking changes, opt-in ← Chosen
3. Add `isVisitor()` check in each component — targeted but no reactive updates

**Decision:** Added `useAuthStatus()` as a separate reactive hook alongside the unchanged `useCurrentUser()`.

**Rationale:** Zero breaking changes. Mirrors existing `subscribeToUserChange` pattern. Consumers opt in when they need the distinction. The `AuthStatus` type was already exported.

**Consequences:** 5 files modified (1 core + 4 consumers). All existing code continues to work unchanged.

## 2. Vitality as Ranking Signal, Not Inclusion Gate
**Type:** Strategic
**Topics:** smart-feed, feed-discovery, platform-launch

**Trigger:** Fresh platform with no feed_activities makes discover page appear empty for all users

**Options Considered:**
1. Remove vitality filter entirely — feeds ranked by vitality but all shown ← Chosen
2. Fallback query when results empty — more complex, same effect
3. Configurable threshold via platform_settings — over-engineering for now
4. Keep gate, add seed data — doesn't fix the real cold-start problem

**Decision:** Removed `vitality > 0` filter from all 5 locations (discover endpoint + smart feed candidates). Kept `ORDER BY vitality DESC` for ranking.

**Rationale:** Active feeds still surface first via ranking. Quiet feeds appear but rank lower. Critical for genesis cohort when most feeds have zero activity.

**Consequences:** New users see tag-matched feeds even with no posts. "0 posts in 2 weeks" shown transparently.

> **Insight:** Cold-start filtering is a common trap in recommendation systems — any hard threshold that excludes items with no engagement data creates a bootstrapping paradox where new content can never gain engagement because it's never shown. Using the metric for ranking rather than gating solves this while still rewarding active content.
