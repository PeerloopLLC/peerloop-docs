# Session Learnings - 2026-03-07

## 1. DB-SCHEMA.md Was Chronically Out of Sync With the Actual Schema
**Topics:** d1, docs-infra

**Context:** During a capabilities review, we compared `research/DB-SCHEMA.md` against `migrations/0001_schema.sql` and found massive drift — the TERMINOLOGY rename block (Sessions 346-356) was never applied to DB-SCHEMA.md, 15+ columns were undocumented, and 23 tables were documented but never built.

**Learning:** Documentation files that duplicate information available in code (like a markdown schema doc mirroring a SQL file) will always drift. The SQL file is the source of truth and is what developers actually reference. The only documentation worth maintaining alongside the SQL is *rationale* — why the schema is designed the way it is, not what the columns are.

**Pattern:** Replace duplicate-of-code docs with slim "guide" docs that explain the *why*. Point to the code for the *what*. This eliminates an entire class of sync failures.

## 2. Capabilities Review Reveals Coverage Gaps That Code Exploration Alone Misses
**Topics:** workflow, docs-infra

**Context:** A consultant's capabilities list from early in the project was compared against the current codebase. Of 102 capabilities listed, 81 were fully built, 10 were partial, and 11 needed runtime verification. Zero were completely missing.

**Learning:** An external capabilities checklist (written by someone outside the development process) is a valuable validation tool. It surfaces gaps in areas the development team takes for granted — like "does a specific user account exist in seed data?" or "are .ics calendar attachments sent?" — that wouldn't surface during feature-focused development.

## 3. Audit Logging Has Multiple Viable Approaches for Cloudflare Workers
**Topics:** cloudflare, workflow

**Context:** Researched npm packages and commercial services for audit logging on CF Workers. Options ranged from custom D1, Pangea (tamper-proof blockchain), WorkOS (enterprise B2B), CF Workers Logs (built-in), to Better Stack (edge-compatible log aggregator).

**Learning:** For MVP-scale apps on Cloudflare, custom D1 audit logging is the simplest approach — zero external dependencies, no edge-runtime compatibility concerns, and the volume is trivial for D1. Commercial audit services (Pangea, WorkOS) add compliance features (tamper-proofing, SIEM streaming) that matter at enterprise scale but are overkill for early-stage. CF Workers Logs (GA April 2025) is a good complement for infrastructure observability but doesn't provide per-user action isolation needed for audit trails.
