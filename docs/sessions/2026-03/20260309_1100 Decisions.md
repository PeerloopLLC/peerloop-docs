# Session Decisions - 2026-03-09

## 1. Route Grid Map: Group-Letter + 2-Digit Code System
**Type:** Strategic
**Topics:** docs-infra

**Trigger:** Need for compact, unique route identifiers that fit in small grid cells and are self-documenting.

**Options Considered:**
1. Reuse old 4-letter codes (HOME, CDET, SBOK) from ORIG-PAGES-MAP.md
2. Fresh 3-letter acronyms (HME, CDT, SBK)
3. Group letter + 2-digit number (M01, C01, T01) ‚Üê Chosen

**Decision:** Use group letter + 2-digit number. The first character identifies the category (M=Marketing, A=Auth, D=Discovery, P=Personal, Y=Community, C=Course, R=Resource, T=Teaching, K=Creating, S=Settings, X=Admin, N=Chrome).

**Rationale:** Self-documenting (group is visible in the code), guaranteed unique (sequential within group), compact (3 chars), sortable. Old 4-letter codes were from a different route structure and weren't always intuitive.

**Consequences:** 12 groups, 92 items (86 routes + 6 pseudo-pages). All existing docs (ORIG-PAGES-MAP.md, ROUTE-STORIES.md) retain their own coding systems ‚Äî this system is specific to the grid map.

## 2. Split Footer Into Two Pseudo-Pages (Full vs Compact)
**Type:** Implementation
**Topics:** docs-infra, astro

**Trigger:** User observed that marketing pages and app pages have visually different footers with different link sets.

**Options Considered:**
1. Single Footer pseudo-page with all links ‚Üê Original approach
2. Two pseudo-pages: Footer Full (N03) + Footer Compact (N06) ‚Üê Chosen

**Decision:** Split into N03 (Footer Full, 19 links, LandingLayout) and N06 (Footer Compact, 3 links: Help/Privacy/Terms, AppLayout+AdminLayout).

**Rationale:** They serve different pages with different link sets. A single pseudo-page would incorrectly imply all app pages link to all marketing routes via footer. The compact footer only provides Help, Privacy, Terms.

**Consequences:** Chrome group grew from 5 to 7 items (3√ó3 grid). Added N07 (PublicHeader) at the same time since the same layout split applies to headers.

## 3. Chrome Stripping Strategy for Grid Cells
**Type:** Strategic
**Topics:** docs-infra

**Trigger:** Raw adjacency matrix shows ~60 connections per page, mostly from shared chrome (navbar, footer, panels). Grid cells would be unreadable.

**Options Considered:**
1. Show all connections in each cell
2. Strip chrome connections, show only page-specific exits ‚Üê Chosen
3. Strip chrome and also remove breadcrumbs

**Decision:** Strip all chrome-provided connections (foot, nav, nav:disc, nav:feed, nav:user, nav:back, logo, and modal‚Üílogin/signup). Only flag exceptions: "no footer" / "no navbar" when a page lacks expected chrome. Keep breadcrumbs (crm) as they're page-specific navigation.

**Rationale:** Chrome is documented once in pseudo-page cells (N01-N07). Each page cell then shows only its unique exits ‚Äî the connections that come from the page's own content. This makes the grid useful for testing: you see exactly what buttons, links, cards, and tabs each page should have.

**Consequences:** Average page dropped from ~60 to 3-5 visible connections. 308 non-chrome connections total across 78 pages.

## 4. Grid Spatial Layout: 4-Tier Island Arrangement
**Type:** Strategic
**Topics:** docs-infra

**Trigger:** Need to arrange 12 sub-grids on a single sheet (max 15 columns) with navigational affinity determining adjacency.

**Options Considered:**
1. Single row of 7 groups (user's original concern ‚Äî connection lines overlap)
2. Squarish sub-grids as islands with spacer columns/rows ‚Üê Chosen
3. Separate tabs per category

**Decision:** 4-tier layout, 3 sub-grids per tier, separated by spacer rows and columns. Groups placed by navigational affinity: Marketing‚ÜîAuth‚ÜîChrome (top), Discovery‚ÜîPersonal‚ÜîSettings (mid-top), Community‚ÜîCourse‚ÜîResource (mid-bottom), Teaching‚ÜîCreating‚ÜîAdmin (bottom).

**Rationale:** Related groups are adjacent for short inter-group connection lines. Squarish sub-grids (3√ó3, 4√ó4, 2√ó2) keep intra-group connections compact. 15 columns √ó 23 rows fits on a single printable page or Google Sheet.

**Consequences:** Grid fits in 15√ó23 cells. Each tier groups pages by functional affinity. Manual line-drawing between islands will be clear because of spacer gaps.
