# Conv Learnings - 2026-03-28

## 1. HAVING Clause Incompatibility Between D1 and better-sqlite3
**Topics:** testing, d1, sqlite

**Context:** Feed discovery endpoint used `HAVING vitality > 0` without GROUP BY to filter computed column

**Learning:** D1 (Cloudflare's SQLite) is more permissive with HAVING on non-aggregated queries, but better-sqlite3 (test DB) enforces the SQL standard strictly. Filter in app code after the query instead.

**Pattern:** `const filtered = results.filter(r => r.vitality > 0)` after the query, instead of `HAVING vitality > 0` in SQL.

## 2. Feeds Are Derivative, Not Discoverable
**Topics:** ux, architecture, feeds

**Context:** Attempting to build `/discover/feeds` as a parallel to `/discover/courses` and `/discover/communities`

**Learning:** Feeds are byproducts of joining communities or enrolling in courses — you can't "join a feed" directly. Discovery should target the subscription level (course/community), not the artifact level (feed). The correct framing for feed discovery is "here are active conversations in entities you could join" with CTAs to the parent entity, not "here are feeds you could subscribe to."

## 3. Smart Feed Discovery Pipeline Already Solves ~70% of Feed Discovery
**Topics:** smart-feed, architecture, reuse

**Context:** Building a new feed discovery page

**Learning:** `getDiscoveryCandidates()` in `src/lib/smart-feed/candidates.ts` already finds public feeds matching user interests, ranked by vitality, with dismissed-feed tracking. Extracting this into a standalone endpoint (without Stream.io enrichment) was a 2-hour build vs building from scratch.

## 4. extraTabs Pattern Enables Component Extension Without Modification
**Topics:** architecture, react, components

**Context:** Adding role-specific tabs to CommunityTabs

**Learning:** The `extraTabs` + `basePath` pattern (proven in CourseTabs) allows any wrapper component to inject role-specific tabs without modifying the base component. CommunityTabs has no knowledge of role tabs — `ExploreCommunityTabs` injects them from outside. This is Open/Closed Principle in practice.
