# Session Learnings - 2026-03-12

## 1. Documentation Drift is Systemic, Not Incidental
**Topics:** docs-infra, workflow

**Context:** Session 383 swept all docs/ subfolders for stale content after completing the TEST-COVERAGE.md rebuild. Found 5 architecture/reference docs still describing the old 5-tab course page structure, 2+ sessions after the COURSE-PAGE-MERGE changed it to 7 tabs.

**Learning:** Manual documentation about generated artifacts (routes, components, test files) drifts silently because nothing enforces sync. The problem isn't forgetfulness — it's that `/w-eos` and `/w-docs` cover reference docs but not architecture docs, DECISIONS.md, or research/COMPONENTS.md. Changes that touch many docs (like a page merge) need a sweep step, not just updating the files the developer remembers.

## 2. Gap Audits Overestimate — Always Verify Before Fixing
**Topics:** docs-infra, workflow

**Context:** DOCS-GAPS-381.md reported 11 genuinely missing API endpoint docs. Investigation found only 4 were actually missing — 7 were documented under different names or in different files (e.g., onboarding-profile was in API-USERS.md, not API-ENROLLMENTS.md). Of those 4, 2 were dev-only debug endpoints.

**Learning:** Automated gap detection that matches filenames against doc content produces many false positives. A smarter approach would match route paths (not filenames) and check all API doc files (not just the "expected" one). Always verify programmatic gap reports before acting on them.

## 3. One-Off Scripts Should Be Folded Into Permanent Tooling Immediately
**Topics:** docs-infra, dual-repo

**Context:** Session 367 created a one-off `/tmp/build-route-grid.js` to transform the adjacency matrix into spatial grid TSVs. By Session 383, the TSVs were stale and the one-off script was gone. Had to reverse-engineer the output format from the TSV files to fold the logic into `route-matrix.mjs`.

**Learning:** Temporary scripts that produce artifacts people rely on should be consolidated into permanent tooling in the same session they're created. The cost of folding it in immediately is lower than reverse-engineering it later. The key signal: if the output gets committed to the repo, the script that creates it should too.

## 4. Generated Docs Need a Single Regeneration Command
**Topics:** docs-infra, workflow

**Context:** route-matrix.mjs generated 2 of 4 page connection files. The other 2 required a separate one-off script. There was no npm script entry, so you had to know the exact `node scripts/route-matrix.mjs` invocation.

**Learning:** Every generated doc should be reproducible with a single, discoverable command (`npm run X`). When extending a generator to produce additional outputs, add them to the same command rather than creating separate scripts. The test: can someone who's never seen the project regenerate all docs by reading package.json?

---

## 5. Pseudo-Page Definitions in Route Scanner Need Manual Maintenance
**Topics:** astro, docs-infra

**Context:** `route-matrix.mjs` hardcodes shared navigation component links (AppNavbar, Footer, etc.) because regex-parsing React state logic is fragile. The AdminNavbar pseudo-page still referenced `/admin/student-teachers` (renamed to `/admin/teachers` months ago).

**Learning:** Hardcoded pseudo-page definitions are a maintenance liability — they're correct when written but drift when the actual components change. The script's comment explicitly acknowledges this trade-off ("hardcoded from reading the actual component source to avoid fragile regex parsing"). Future improvement: extract link targets from the component files at scan time using AST parsing, or at minimum add a staleness check that compares pseudo-page targets against the known route list.

## 6. PAGE-CONNECTIONS.md Output Path Was Wrong Since Creation
**Topics:** docs-infra, dual-repo

**Context:** `route-matrix.mjs` wrote PAGE-CONNECTIONS.md to the docs repo root. Session 362 reorganized docs and created a DOCS-REORG-MAP.md showing it should be at `docs/architecture/page-connections.md`. But the script was never updated — it kept writing to the root, creating a duplicate.

**Learning:** When reorganizing file locations, update all generators/scripts that output to the old path. The DOCS-REORG-MAP.md captured the intended move but nobody updated the script. A grep for the old path across all scripts would have caught this.
