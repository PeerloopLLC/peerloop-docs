### maybeUpdateActorSession Design Flaw
The `maybeUpdateActorSession` method in `api-runner.ts` auto-detects user creation by checking for `provided.userId`/`studentId`/`creatorId`/`teacherId` keys. This is fragile — any discovery action that extracts someone else's ID under those key names corrupts the current actor's session. Worked around with a rename (`assignedTeacherUserId`), but the root cause (pattern-matching on key names) remains. A future fix could scope the auto-detection to registration actions only (where `source: 'register'`).
