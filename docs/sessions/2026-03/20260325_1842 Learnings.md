# Conv Learnings - 2026-03-25

## 1. Skills Are Prompt Injections, Not Functions
**Topics:** cc-workflow

**Context:** Debugging recurring r-end failures — stopping after step 2, missing TodoWrite items from nested skill calls.

**Learning:** Skills don't create stack frames — they inject prompts into the same context window. Nesting 3 levels deep (r-end → r-eos → 4 terminal skills) asks Claude to mentally track a return path across multiple prompt injections, which it does unreliably. The deeper the nesting and the more context each inner skill consumes, the more likely the outer instructions are lost. This is the root cause of "forgets to return to r-end step 3 after r-eos completes."

**Pattern:** Treat skills as flat entry points, not composable functions. If a skill needs to orchestrate multiple stages, keep orchestration inline or use agents (which get isolated context) rather than nested skill calls.

## 2. Collector Pattern Eliminates Redundant Conversation Scans
**Topics:** cc-workflow

**Context:** Designing how agents would get conversation data without each copying the full history.

**Learning:** Instead of N agents each re-scanning the full conversation (expensive, redundant), scan once in the main context and write a structured extract file. Agents then read only their relevant sections. This is a materialized view pattern — pre-compute once, read many times. The extract file also doubles as a debugging artifact that the nested-skill approach completely lacked.

**Pattern:** Main context → scan conversation → write extract to disk → dispatch agents that read extract sections. Each agent's context stays small and focused.

## 3. Two-Level Prefix Matching in sync-gaps.sh
**Topics:** docs-infra

**Context:** Fixing false-positive "undocumented route" flags for `webhooks/bbb-analytics` — the script matched the generic `webhooks` entry before reaching specific `webhooks/*` entries.

**Learning:** When a shell script matches route prefixes via first-match grep, generic prefixes (e.g., `webhooks`) shadow specific ones (e.g., `webhooks/bbb-analytics`). The fix is to check multi-level prefixes first — same pattern already existed for `me/*` routes, just needed extending to `webhooks/*`. Any future prefix namespace with sub-prefixes needs to be added to the same condition on line 82.
