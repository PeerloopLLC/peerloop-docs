# Conv Decisions - 2026-03-28

## 1. Promote /explore/* to /discover/* (Not Coexist)
**Type:** Strategic
**Topics:** routing, architecture

**Trigger:** User confirmed the explore experiment was a success

**Options Considered:**
1. Keep both `/explore/*` and `/discover/*` routes ← rejected (confusion, maintenance)
2. Promote `/explore/*` to `/discover/*`, replacing old browse page ← Chosen
3. Merge into `/course/*` routes ← not considered (different purpose)

**Decision:** Replace `/discover/courses` with the role-aware explore version, create `/discover/course/[slug]` as new detail route. Delete `/explore/` entirely. Keep `/course/[slug]` as-is (client to confirm its future).

**Rationale:** Single authoritative route per resource type. The role-aware version is strictly superior — visitors see the same content, members get role-specific tabs.

**Consequences:** `/explore/` routes no longer exist. Old non-role-aware `/discover/courses` replaced. Breadcrumbs use existing `/discover` index page.

## 2. Catch-All Over Per-File Sub-Routes
**Type:** Implementation
**Topics:** astro, routing

**Trigger:** Need for bookmarkable tab URLs on discover detail page

**Options Considered:**
1. Per-file sub-routes (8+ .astro files like `/course/[slug]/`) ← rejected (duplication)
2. Single `[...tab].astro` catch-all with validation ← Chosen
3. Astro middleware rewrite ← rejected (unnecessary complexity)

**Decision:** Use `[...tab].astro` catch-all with `VALID_TABS` Set validation and redirect for invalid segments.

**Rationale:** 1 file vs 8+. Adding new tabs only requires adding to the Set. SSR code is still duplicated between index and catch-all, but that's 2 files not 9.

**Consequences:** New pattern established for discover detail page. Could be retrofitted to `/course/[slug]/` in a future cleanup.
