# Conv Decisions - 2026-03-20

## 1. Embed TodoWrite-Gap Reminder in Skill Chain, Not Just Memory
**Type:** Strategic
**Topics:** cc-workflow

**Trigger:** Recurring failure across multiple convs — documentation gaps found during `/r-docs` were mentioned in commentary as "pre-existing" but never TodoWritten, despite two existing feedback memory entries addressing this exact behavior.

**Options Considered:**
1. Add another memory entry (stronger wording) — already have two, hasn't worked
2. Embed rule in `/r-docs` SKILL.md only — covers the executor but not the orchestrator
3. Embed rule in both `/r-eos` and `/r-docs` SKILL.md ← Chosen

**Decision:** Added a `⚠️ CRITICAL` callout block at the top of `/r-docs` Action Plan section with trigger-word self-monitoring, and a lighter rule in `/r-eos` Rules section as defense-in-depth.

**Rationale:** Memory entries load at conversation start but can be evicted by context compression during long `/r-end` chains. Skill SKILL.md files are freshly injected each time the Skill tool invokes them, guaranteeing the instruction is present at the moment gaps are discovered. Two levels (orchestrator + executor) provide redundancy.

**Consequences:** Both `/r-eos/SKILL.md` and `/r-docs/SKILL.md` now contain explicit TodoWrite-gap rules. The memory entries remain as additional reinforcement for conversations that don't run these skills.
