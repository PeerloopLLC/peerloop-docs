# Conv Learnings - 2026-03-25

## 1. BBB Learning Analytics Uses Callback, Not Pull API
**Topics:** video, d1

**Context:** Researching how to capture per-attendee engagement metrics (talk time, chat count, attendance duration) from BigBlueButton sessions.

**Learning:** BBB has no API endpoint for retrieving analytics data. The only programmatic mechanism is `meta_analytics-callback-url` — a parameter set on room creation that causes BBB to POST a JWT-signed JSON payload after the meeting ends. The data powering the Learning Dashboard is stored on the BBB server filesystem and deleted ~2 minutes after meeting end. `getRecordings()` returns only recording metadata (no engagement data). `getMeetingInfo()` only works while the meeting is running.

**Pattern:** Set `meta_analytics-callback-url` alongside `meta_endCallbackUrl` on room creation. Receive POST with `Authorization: Bearer <JWT>` (HS512 signed with BBB shared secret). Payload contains `data.attendees[]` with per-user `duration`, `engagement.talk_time`, `engagement.chats`, `engagement.raisehand`, `engagement.poll_votes`.

## 2. SQLite datetime() Format Mismatch Causes Silent Comparison Bugs
**Topics:** d1, testing

**Context:** `detectStaleInProgress()` used `datetime(s.scheduled_end, '+1 hour') < ?` where `?` was an ISO timestamp. A test for the 1-hour grace period was failing unexpectedly.

**Learning:** SQLite's `datetime()` returns space-separated format (`2026-03-25 11:00:00`) while Peerloop stores timestamps in ISO format with `T` separator (`2026-03-25T11:00:00.000Z`). In SQLite string comparisons, space (ASCII 32) < T (ASCII 84), so `datetime()` results always compare "less than" ISO strings with the same date prefix. This made the grace period ineffective — sessions were caught immediately instead of after 1 hour.

**Pattern:** Always use `strftime('%Y-%m-%dT%H:%M:%fZ', expr, modifier)` instead of `datetime()` for any SQL expression that will be compared against stored ISO timestamps. Added CLAUDE.md rule + `/w-codecheck` lint check to prevent recurrence.

## 3. BBB Analytics Data Is Not Recoverable After Meeting End
**Topics:** video

**Context:** Designing self-healing for missed BBB webhooks. Reconciliation can catch missed `meeting-ended` (via `getRoomInfo`) and missed `recording_ready` (via `getRecordings`), but what about missed analytics callbacks?

**Learning:** Analytics callbacks are fire-and-forget. BBB deletes the Learning Dashboard data from its server shortly after meeting end (`learningDashboardCleanupDelayInMinutes`, default 2 min). There is no API endpoint to retrieve analytics after the fact. The only defense is ensuring the callback endpoint is available when the meeting ends.

## 4. vi.mock Partial Mocks Can Cause Silent Failures in Vitest
**Topics:** testing

**Context:** Cleanup tests mocked `@/lib/notifications` but only exported `notifySessionNoShow`. When `completeSession()` called `notifySessionCompleted()` (from the same mock), Vitest threw "No export defined on mock" — but this was caught by the fire-and-forget `.catch()` and silently swallowed.

**Learning:** When partially mocking a module in Vitest, any unmocked export that gets called will throw. If the calling code uses `.catch()` or `Promise.allSettled()`, the error is swallowed silently. Always mock ALL exports that will be called in the test path, even if they're called indirectly via fire-and-forget chains.

## 5. TodoWrite Must Track Everything — "Pre-existing" Is Not an Excuse
**Topics:** workflow

**Context:** During SESSION-FIX.CLEANUP, I found 3 TODOs in resolve.ts and noted them as "pre-existing (not from SESSION-FIX work)" without adding to TodoWrite. User correctly flagged this as violating the work tracking rule.

**Learning:** The CLAUDE.md instruction is explicit: "Any time you encounter a problem, issue, opportunity, or question that you don't immediately resolve — for any reason — add it to TodoWrite. 'Pre-existing' or 'out of scope' is not a reason to stay silent." Labeling something as "pre-existing" feels like acknowledging it, but without TodoWrite tracking it's effectively forgotten.
