# Conv Learnings - 2026-03-18

## 1. Seed Data assigned_teacher_id Gap
**Topics:** d1, testing

**Context:** Guy Rymberg's `/teaching` dashboard showed "No students assigned yet" despite having 4 completed student enrollments in the seed data.

**Learning:** The early enrollments (Sarah, Marcus, Amanda on Guy's courses) had `assigned_teacher_id = NULL` because the seed data didn't reflect that Guy-as-Creator was also the teacher before any Student-Teachers were certified. The dashboard query `WHERE assigned_teacher_id = ?` correctly returned nothing.

**Pattern:** When seeding enrollments, always set `assigned_teacher_id` — even for Creator-as-Teacher scenarios. Also set `teacher_certification_id` to match. Verify FK references exist before inserting (e.g., `st-claude-code-sarah` didn't exist — caught during edit).

## 2. SQLite datetime('now') vs ISO 8601 Format Mismatch
**Topics:** d1, astro

**Context:** Past student completion dates displayed one day early (e.g., seed data `2024-09-10T00:00:00Z` showed as "9/9/2024").

**Learning:** Three-layer interaction causes the bug:
1. **Seed data** uses ISO 8601 with Z: `2024-09-10T00:00:00Z`
2. **SQLite's `datetime('now')`** produces `2026-03-18 23:41:25` (no Z, no T)
3. **JavaScript's `new Date()`** parses Z-suffix strings as UTC but no-timezone strings as local time
4. **`toLocaleDateString()`** then converts to browser timezone, shifting UTC midnight backward in west-of-UTC zones

The same rendering code produces different results depending on whether data came from seed or real usage. This was the trigger for the DATE-FORMAT decision.

## 3. Shared Component Title Props for Role-Specific Dashboards
**Topics:** astro

**Context:** `EarningsOverview` was shared between Teacher and Creator dashboards but showed the generic "Earnings Overview" label on both.

**Learning:** When a component is shared across role-specific dashboards that query different `recipient_type` values (teacher vs creator_as_*), the label must distinguish which revenue stream is shown. A simple `title` prop with a sensible default preserves backward compatibility while enabling role-specific labels.

## 4. /r-resume False Positive on Fresh Conv Start
**Topics:** cc-workflow

**Context:** `/r-resume` warned about stale/missing RESUME-STATE.md immediately after `/r-start` created `.conv-current`.

**Learning:** The stale-context check (`if .conv-current exists AND RESUME-STATE.md is missing`) doesn't distinguish between a fresh start (no resume state expected) and a lost state (raw `/clear` without `/r-pre-clear`). Fixed by checking if the most recent git commit matches "Conv NNN start" for the current conv number — if so, suppress the warning.

## 5. Global CLAUDE.md for Cross-Project Directives
**Topics:** cc-workflow

**Context:** User wanted question-formatting directive (bold + 3 pointing emojis) to persist across all projects and sessions.

**Learning:** `~/.claude/CLAUDE.md` is loaded for every Claude Code session regardless of project or directory. It's the most durable place for cross-project user preferences. It survives `/quit`, `/clear`, project switches. Must be created manually on each machine (not git-synced).

---

## 6. Creator "My Teachers" Excludes Self
**Topics:** astro, d1

**Context:** Building the Creator dashboard "My Teachers" section with Active/Past tabs.

**Learning:** Creators who also teach their own courses (like Guy) must be excluded from the "My Teachers" list (`st.user_id != ?`) because they already appear in the `CreatorTeachingSummary` ("My Teaching") section. Without exclusion, the same person appears in two sections with overlapping but different data.
