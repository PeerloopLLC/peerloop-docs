# Session Decisions - 2026-03-07

## 1. Replace DB-SCHEMA.md With Slim DB-GUIDE.md
**Type:** Strategic
**Topics:** docs-infra, d1

**Trigger:** DB-SCHEMA.md was found to be massively out of sync with the actual SQL schema after the TERMINOLOGY rename block. 23 tables were documented but never built, 15+ columns were undocumented, and the core permission model used stale column names.

**Options Considered:**
1. Fix DB-SCHEMA.md — full rewrite to match current SQL
2. Drop DB-SCHEMA.md entirely — just use the SQL file
3. Replace with slim DB-GUIDE.md that covers only rationale ← Chosen

**Decision:** Created `research/DB-GUIDE.md` as a slim design rationale document. DB-SCHEMA.md marked as DEPRECATED with banner. All live references updated to point to DB-GUIDE.md.

**Rationale:** The SQL file is the source of truth for column names, types, and constraints. A duplicate markdown doc will always drift. The only value DB-SCHEMA.md provided beyond the SQL was design rationale (why the Community > Progression > Course chain exists, why capabilities not roles, how the two rating systems work). DB-GUIDE.md captures that and nothing else.

**Consequences:** Updated references in CLAUDE.md (4), PLAYBOOK.md (1), DECISIONS.md (1), q-docs-local.md (3). Session logs and historical docs left as-is. DB-SCHEMA.md kept for history with deprecation banner.

## 2. Custom D1 for Audit Logging (MVP Recommendation)
**Type:** Implementation
**Topics:** d1, cloudflare

**Trigger:** Capabilities review identified that no application logging infrastructure exists — API endpoints use bare `console.error` which is ephemeral on CF Workers.

**Options Considered:**
1. Custom D1-backed audit log ← Recommended
2. Pangea Secure Audit Log (tamper-proof, commercial)
3. WorkOS Audit Logs (enterprise B2B, commercial)
4. CF Workers Logs (built-in, infrastructure-level)
5. Better Stack / Logtail (edge-compatible log aggregator)

**Decision:** Recommended custom D1 for MVP with CF Workers Logs as infrastructure complement. Full AUDIT-LOG block added to PLAN.md with schema, action catalog, admin UI spec, retention policy, and package evaluation section. Final decision deferred to implementation time.

**Rationale:** D1 handles the volume, zero external dependencies, full control over schema and retention. Commercial services add compliance features not needed at Genesis scale. CF Workers Logs is complementary (infrastructure observability) but doesn't provide per-user action isolation.

**Consequences:** AUDIT-LOG block added as deferred priority 24 in PLAN.md with detailed sub-sections (SCHEMA, ACTIONS, LIB, ADMIN_UI, RETENTION, PACKAGES, TASKS, DEPENDENCIES).
