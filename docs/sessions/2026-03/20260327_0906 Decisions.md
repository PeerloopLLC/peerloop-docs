# Conv Decisions - 2026-03-27

## 1. Skill Consolidation: spt-docs Back-Port
**Type:** Strategic
**Topics:** cc-workflow, dual-repo, docs-infra

**Trigger:** User simplified skills in spt-docs project (derived from peerloop-docs) and wanted to bring improvements back.

**Options Considered:**
1. Keep current 22-skill architecture — no risk, but maintenance burden
2. Port spt-docs skills with full adaptation ← Chosen
3. Start fresh with a new skill design — too risky, untested

**Decision:** Port the simplified 6-skill architecture from spt-docs, reducing peerloop-docs from 22 to 14 skills.

**Rationale:** Fewer skills = less maintenance, fewer skill-to-skill call failures, faster end-of-conv execution. The spt-docs version proved stable. Rollback tag provides safety net.

**Consequences:** 11 skill directories deleted, r-end now owns all end-of-conv processing, r-start inlines resume, CLAUDE.md and DOC-DECISIONS.md updated.

> **Insight:** Porting battle-tested simplifications from a derived project back to the original is a powerful pattern — the derived project serves as a low-risk proving ground for structural changes that would be riskier to prototype in the production repo.

## 3. w-timecard preserved separately from r-timecard

**Topics:** cc-workflow

1. Merge both into r-timecard — user rejected
2. r-timecard replaces only w-timecard-dual, w-timecard preserved ← Chosen
