# Session Decisions - 2026-03-11

## 1. Rename Project Skills from q-* to w-* Prefix
**Type:** Strategic
**Topics:** cc-workflow, docs-infra

**Trigger:** The Claude Code UI showed both `/q-eos (project)` and `/q-eos (user)` when typing a skill name, making it easy to accidentally invoke the wrong one.

**Options Considered:**
1. Live with the ambiguity and be careful selecting ← Not viable (repeated mistakes)
2. Rename project skills to use a different prefix (`w-*`) ← Chosen
3. Delete the global commands entirely — would break other projects

**Decision:** All 11 project skills in `.claude/skills/` renamed from `q-*` to `w-*`. Global commands in `~/.claude/commands/` remain `q-*`. The `w-` prefix is project-local; `q-` is global.

**Rationale:** Different prefixes make skill origin immediately obvious in the autocomplete UI. No ambiguity, no mis-selection. The rename is confined to one repo and doesn't affect other projects using the global commands.

**Consequences:** Updated SKILL.md frontmatter (11 files), cross-references within skills, CLAUDE.md (3 refs), PLAYBOOK.md (~15 refs), skills-system.md (~15 refs), DEVELOPMENT-GUIDE.md (2 refs), detect-changes.sh (marker path + comments). Session logs left unchanged (historical records). Local `.claude/commands/q-*-local` files were not renamed (they extend global commands).

## 2. Delete Orphaned *-local Command Files
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** Three local command files (`q-end-session-local.md`, `q-force-end-local.md`, `q-resume-session-local.md`) remained in `.claude/commands/` from the pre-Skills 2 era. They extended global commands that were no longer the primary workflow entry points.

**Options Considered:**
1. Keep them for backward compatibility
2. Delete them as orphaned ← Chosen

**Decision:** Deleted all three. The global commands they extend (`/q-end-session`, `/q-force-end`, `/q-resume-session`) are no longer the primary workflow — replaced by project skills (`/w-eos`, etc.).

**Rationale:** Dead code creates confusion. The global commands still say "run *-local if it exists" but nobody invokes those globals in this project anymore.

**Consequences:** `.claude/commands/` reduced from 5 files to 2 (the `L-*` files, subsequently also migrated).

## 3. Migrate L-* Commands to Skills 2 as w-* Skills
**Type:** Implementation
**Topics:** cc-workflow, docs-infra

**Trigger:** `L-add-client-note.md` and `L-schema-dump.md` were the last remaining old-format commands in `.claude/commands/`. Both had stale references (PAGES-MAP.md, DB-SCHEMA.md, 0002_seed.sql).

**Options Considered:**
1. Leave as old-format commands
2. Migrate to Skills 2 and modernize references ← Chosen

**Decision:** Migrated both to `.claude/skills/` as `w-add-client-note` and `w-schema-dump`. Modernized all file references to current paths. `.claude/commands/` directory is now empty.

**Rationale:** Consistent format across all project skills. The migration opportunity allowed fixing stale references (PAGES-MAP.md → url-routing.md, DB-SCHEMA.md → 0001_schema.sql + DB-GUIDE.md, 0002_seed.sql → 0002_seed_core.sql).

**Consequences:** 13 project skills now in `.claude/skills/`, all using `w-*` prefix. Zero old-format commands remain. Updated RFC/INDEX.md command reference.
