# Conv Learnings - 2026-03-28

## 1. Orphaned Infrastructure: ContextActionsPanel
**Topics:** architecture, admin, context-actions

**Context:** During ROLE-AWARE-PAGE-FEATURES audit, discovered a complete FAB system built in Session 28 but never re-wired after Session 154 routing reset.

**Learning:** Infrastructure can survive code maintenance (renames, migrations) while losing its consumers. The ContextActionsPanel was updated through TERMINOLOGY renames and getCurrentUser() migration but no page rendered it for ~200 sessions. Git blame shows the original integration was in now-deleted route files (`/courses/[slug].astro`, `/creators/[handle].astro`).

**Pattern:** When doing major route restructures, grep for component imports to identify orphaned consumers.

## 2. Admin Role Not Represented in Page UIs
**Topics:** admin, ux, architecture

**Context:** Comprehensive audit of all 99 pages revealed admin has 13 dedicated pages + 61 API endpoints but zero presence in regular page UIs.

**Learning:** An admin browsing the platform sees the exact same thing as a visitor on every non-admin page. The existing admin system is comprehensive on its own (`/admin/*`) but doesn't surface admin capabilities inline. The admin-as-tab pattern (adding an "Admin" tab alongside existing role tabs like Student/Teaching/Created) is more discoverable than a floating button and scales naturally — an admin who is also teacher and creator sees all role tabs plus Admin.

## 3. getCurrentUser() as Lightweight Role Resolver
**Topics:** current-user, pattern, react

**Context:** Needed to detect viewer's relationship to content in CourseHero, CreatorProfileHeader, and TeacherProfileHeader components.

**Learning:** `getCurrentUser()` is synchronous (reads from `window.__peerloop`, pre-populated by AppNavbar before hydration). The pattern `useState(false) → useEffect(() => { const user = getCurrentUser(); if (user?.someCheck()) setState(true); }, [dep])` is near-instant and causes no visible flash for the majority case (non-owner viewers see default UI immediately). This matches the existing PublicProfile.tsx pattern.
