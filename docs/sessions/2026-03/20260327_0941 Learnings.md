# Conv Learnings - 2026-03-27

## 1. No-/clear Continuation Pattern Works Cleanly
**Topics:** cc-workflow

**Context:** Tested running /r-start without /clear after /r-end to continue with full conversation context.

**Learning:** The pattern works because: (1) /r-end commits and pushes, leaving repos clean; (2) /r-start pulls, increments counter, pushes — which also pushes any post-/r-end commits that piggybacked; (3) /r-start overwrites .conv-current with the new conv number. The only friction is if RESUME-STATE.md was modified post-/r-end (requires a commit before /r-start can pull).

## 2. RESUME-STATE.md Tasks Don't Duplicate Existing TodoWrite Items
**Topics:** cc-workflow

**Context:** Concerned that /r-start would create duplicate TaskCreate entries for items already in TodoWrite.

**Learning:** It doesn't — the agent manually checks for existing tasks and skips duplicates. In this test, #10 already existed in TodoWrite from the post-/r-end creation, and only #11 and #12 were created from RESUME-STATE.md.
