### Route-Matrix Fixes

**User:** "Let's fix the two route-matrix tasks"

**Claude:** Explored the route-matrix.mjs script (1039 lines), traced import chains for CourseHero, CreatorProfileHeader, TeacherProfileHeader to understand both issues. Ran the script to capture current broken targets (3: `/@[id]`, `/course/[slug]/certificate`, `/teaching/courses/[id]`).

Diagnosed both root causes:
1. **Ternary links missed:** `extractLinks` used simple regex patterns (`href={"/path"}`, `` href={`/path`} ``) that couldn't handle `href={cond ? '/path1' : `/path2`}`. CourseHero's ternary and Leaderboard's ternary both invisible to the scanner.
2. **Param name mismatch:** `normalizeDynamic` maps `${courseIdParam}` → `[id]` (matches substring "id") and `${teacher.id}` → `[id]`, but actual routes are `/teaching/courses/[courseId]` and `/@[handle]`.

Also discovered `MergedPeople.tsx:60` uses `teacher.id` (UUID) instead of `teacher.handle` for profile URLs — a real code bug.

**Fix 1 — Balanced expression extractor:** Added `extractBalancedExpr()` function that properly parses `href={...}` blocks with brace balancing, handling nested `${}` in template literals, string escaping. Replaces the two simple JSX patterns. Now extracts ALL string and template literal values from any `href={...}` expression.

**Fix 2 — Structural param resolver:** Added `resolveTargetParams()` that matches extracted targets against known routes by comparing segment structure (replacing `[*]` wildcards). Applied as post-processing pass after route collection. `/teaching/courses/[id]` → matches `/teaching/courses/[courseId]` → resolved.

**Fix 3 — MergedPeople bug:** Added `handle` field to teacher map, changed `href` to use `teacher.handle` with fallback to `teacher.id`. The `teacherHandle` field was already available on `UserEnrollment` interface but unused.

Results: broken targets 3 → 1 (only genuinely missing `/course/[slug]/certificate`). New links detected: `/creating` on course pages, `/@[handle]` + `/teacher/[handle]` on leaderboard.
