# Conv Learnings - 2026-03-20

## 1. git --grep Matches Anywhere in Commit Message Body
**Topics:** cc-workflow

**Context:** Using `git log --grep="Conv 015"` to select commits by conv number for the new `conv=NNN` timecard parameter.

**Learning:** `--grep` matches the pattern anywhere in the full commit message (subject + body), not just in a specific metadata field. A Conv 016 commit that mentions "Conv 015-016" in its body text gets returned as a false match. Manual filtering by checking the subject line or `Conv:` metadata line is needed to exclude these.

**Pattern:** For now, visually verify each commit's subject line starts with the target conv. A future improvement could use `git log --format` with grep on the `Conv: NNN` metadata line only.

---

## 2. Multiple --grep Flags Use OR Logic by Default
**Topics:** cc-workflow

**Context:** Needed to select commits from multiple convs (e.g., `conv=017,018`) in a single git log command.

**Learning:** `git log` with multiple `--grep` flags matches commits containing ANY of the patterns (OR logic). The `--all-match` flag would switch to AND logic. For conv-based selection where each commit belongs to exactly one conv, OR is the correct behavior.

---

## 3. Heartbeat Commits Always Present in Conv-Based Selection
**Topics:** cc-workflow, dual-repo

**Context:** Every `/r-start` creates a `Conv NNN start — MachineName` commit in the docs repo. When using `conv=NNN` grep, this commit is always captured.

**Learning:** Conv-based selection inherently includes heartbeat commits from the docs repo (at least 1 per conv requested). The existing heartbeat suppression rules in the timecard skill handle this correctly — heartbeats contribute to timing and count but are excluded from Git History output. No special handling needed beyond what already exists.
