# Conv Decisions - 2026-03-20

## 1. Conv-Based Timecard Selection via git --grep
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** User needed to generate timecards for specific convs retroactively, but count-based selection (`cNdN`) requires knowing how many commits each repo has per conv — tedious for multi-conv timecards.

**Options Considered:**
1. Count-based only — user counts commits manually per repo ← Existing
2. Conv-based selection using `git log --grep="Conv NNN"` ← Chosen (added as alternative)
3. Date-range based selection — too coarse, doesn't respect conv boundaries

**Decision:** Added `conv=NNN[,NNN,...]` as a second mode in `/w-timecard-dual`, mutually exclusive with the existing `cNdN` mode. Uses `--grep` with OR logic to find all commits matching the requested conv numbers across both repos.

**Rationale:** Conv numbers are the natural unit of work in this project. Grep-based selection automatically finds the right commits in both repos without manual counting. The `--grep` false-match caveat (body text mentioning other convs) is manageable with manual verification and rare in practice.

**Consequences:** Skill now has two argument parsing paths (count mode vs conv mode) but the commit processing pipeline (Steps 3-6) is unchanged.
