### AppNavbar LocalStorage Cache Bug Investigation

**User:** Reported that after deploying to staging, the AppNavbar contents disappeared for users with existing localStorage data. Navbar flashed briefly then vanished. Traced to two localStorage entries — deleting them restored the screen. Asked how the AppNavbar was hidden.

**Claude:** Investigated `AppNavbar.tsx` and `current-user.ts`. Identified the stale-while-revalidate pattern: `getCurrentUser()` loads cached `MeFullResponse` from localStorage instantly, renders with stale data (the flash), then the background API refresh reveals a structural mismatch, causing the navbar to re-render empty. The two keys are `peerloop_user_cache` (full user data) and `peerloop_was_logged_in` (auth flag).

### Cache Protection Strategy Discussion

**User:** Asked if build version key was the only solution.

**Claude:** Presented three approaches: (1) build version key, (2) schema validation with graceful fallback, (3) hybrid. Discussed pros/cons of each.

**User:** Asked about "malformed" — does it mean a different JSON shape?

**Claude:** Clarified: "malformed" means JSON parses successfully but the object shape doesn't match what code expects. `user.displayName` returns `undefined` from old cached `user.name`, React renders nothing.

**User:** Noted that even same-shape data could be semantically stale.

**Claude:** Agreed — shape validation only catches structural staleness. Semantic staleness (correct shape, wrong values) is already handled by the API refresh. Recommended build version key tied to deploy as the more robust approach.

**User:** Asked about guaranteeing every build gets a new ID. No versioning system exists yet.

**Claude:** Reconsidered from first principles. Since the API refresh always runs and always corrects the cache, the real problem is only "don't render garbage during that brief window." Recommended structural guard over build version key — it's more surgical, requires no build infrastructure, and only triggers when there's an actual mismatch. Mentioned `Date.now().toString(36)` in astro.config.mjs as a future option.

**User:** Approved structural guard approach.

### Structural Guard Implementation

**Claude:** Added `isValidCachedData()` type guard function to `current-user.ts:681-696`. Checks 6 critical fields: `user.id`, `user.name`, `user.handle` (strings), `enrollments`, `teacherCertifications`, `createdCourses` (arrays). If validation fails, cache is silently discarded and `null` returned. TypeScript type predicate provides downstream type safety. Type check passed, no existing tests to break.
