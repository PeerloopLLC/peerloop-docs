# Conv Learnings - 2026-03-27

## 1. Wrapping a Monolith Hides UX Problems
**Topics:** component-architecture, ux

**Context:** Conv 039 wrapped CourseTabs as a black box to avoid modifying 1392 lines, placing role tabs in a separate section below all tab content.

**Learning:** Wrapping a large component to avoid modifying it can create worse UX than the engineering cost it saves. The role tabs were technically functional but invisible to users because they were buried below the fold. The user correctly identified that "only one version survives" — making the wrapping workaround unnecessary friction.

---

## 2. Pick<DBType, fields> for Component-Facing Types
**Topics:** typescript, testing

**Context:** Importing full DB types (CourseObjective with course_id, display_order) into component props caused test failures because test data only provided the fields the component renders (id, objective).

**Learning:** Use `Pick<DBType, 'field1' | 'field2'>` for component-facing types. This lets the component accept full DB objects from SSR while only requiring the fields it renders — keeping test data simple and component contracts minimal.

**Pattern:** `export type CourseObjective = Pick<DBCourseObjective, 'id' | 'objective'>;`

## 3. Section Labels Above Tab Groups Save Horizontal Space
**Topics:** ui-layout, tabs

**Context:** Inline section labels ("TEACHING", "YOUR COURSE") in the tab bar consumed ~150px each, pushing tabs off-screen.

**Learning:** Moving section labels above their tab group (stacked layout) eliminates horizontal space waste while preserving visual grouping. Combined with icon size reduction (w-5 to w-4) and gap tightening (gap-6 to gap-3), all tabs fit on screen.

---

## 4. Price Card Height Drives Hero Height in 2-Column Grid
**Topics:** css-layout, responsive

**Context:** After reducing hero padding, the hero was still tall because the right-column price card (thumbnail + price + button + checklist) was taller than the left-column course info.

**Learning:** In a 2-column CSS grid, the taller column determines the section height. Reducing padding on the outer container doesn't help if the inner content (price card) is the height driver. Must shrink the card's contents: fixed thumbnail height instead of aspect-ratio, smaller text sizes, tighter spacing.
