## 1. Skill Bang-Backtick Environment Variables
**Topics:** cc-workflow, dual-repo

**Context:** Ran `/w-test-env` on MacMiniM4 to verify which environment variables are available inside skill `!`backtick`` expressions.

**Learning:** `$CLAUDE_PROJECT_DIR` and `$PWD` are available and resolve correctly. `$CLAUDE_SKILL_DIR` and `$CLAUDE_SESSION_ID` are NOT available — they are either hook-only or not yet implemented. All `r-*` and `w-*` skills that use `$CLAUDE_PROJECT_DIR` for portable paths work on both dev machines.

## 2. Self-Referential FK vs Circular Dependency
**Topics:** d1, testing

**Context:** The `reset-d1.js` script's topological sort was logging `⚠️ Circular dependency detected involving courses` during staging resets. Investigated whether this indicated a schema design problem.

**Learning:** The "circular dependency" was actually a self-referential FK (`courses.source_course_id → courses.id`) — a standard pattern for tracking copied/derived records. Self-references are harmless for DROP operations since the table and its FK constraint disappear together. The real danger is cross-table cycles (A → B → A) where neither table can be dropped first. The script's `visiting` set detection couldn't distinguish the two cases because it only knew the target table, not the relationship direction.

**Pattern:** To distinguish self-references from true cycles in a topological sort, move the detection into the ref iteration loop where both `table` (current) and `ref` (target) are known:
```javascript
if (ref === table) {
  // Self-reference — safe
} else if (visiting.has(ref)) {
  // True cross-table cycle — problem
}
```

## 3. sync-gaps.sh False Positives from Split Test Docs
**Topics:** docs-infra, cc-workflow

**Context:** The `/r-docs` sync-gaps report claimed 75 of 326 test files were undocumented. This seemed wrong — Session 383 had rebuilt TEST-COVERAGE.md from scratch covering all 404 files. Investigation revealed the rebuild split test docs into 3 files (`TEST-COVERAGE.md`, `TEST-COMPONENTS.md`, `TEST-UNIT.md`), but `sync-gaps.sh` only checked the first.

**Learning:** When a parent doc is split into sub-docs, any tooling that checked the parent must be updated to check all children. Hardcoding filenames creates a maintenance trap — each new sub-doc requires a script change. The fix was to dynamically discover all `TEST-*.md` files via glob (`ls "$REF_DOCS"/TEST-*.md`), making the script future-proof against further splits.

**Pattern:** Use dynamic discovery over hardcoded lists when the set of files can grow:
```bash
# Instead of: grep -qF "$name" TEST-COVERAGE.md && grep -qF "$name" TEST-COMPONENTS.md && ...
# Do:
test_docs=$(ls "$REF_DOCS"/TEST-*.md 2>/dev/null)
while IFS= read -r doc; do
  grep -qF "$name" "$doc" && found=true && break
done <<< "$test_docs"
```
