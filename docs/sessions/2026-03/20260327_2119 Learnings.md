# Conv Learnings - 2026-03-27

## 1. Extending Data Interfaces Requires Immediate Test Mock Updates
**Topics:** testing, typescript

**Context:** Adding `courseRating`, `courseRatingCount`, `teacherCount`, `nextSessionAt` to `UserTeacherCertification` broke 3 test files.

**Learning:** When extending a TypeScript interface with required fields, immediately grep all test files for mock usages of that interface and add the new fields. This is where interface changes most often break builds.

**Pattern:** `grep -r "InterfaceName" tests/` -- fix all mock factories before running tests.

---

## 2. Shared Components Need Feature Flags for Context-Specific Behavior
**Topics:** react, component-design

**Context:** `CreatorCourseCard` is shared between Dashboard (needs Discussion toggle) and Explore page (doesn't need it).

**Learning:** When a shared component needs different behavior in different contexts, add opt-out boolean props with backward-compatible defaults (e.g., `showDiscussionToggle = true`) rather than forking the component.

---

## 3. SSR Subtitle + Client-Side Role Data via DOM Update
**Topics:** astro, react, hydration

**Context:** Subtitle needs role-awareness from `CurrentUser` (client-side) but lives in the Astro page (SSR).

**Learning:** For SSR elements that need client-side data updates, use `document.getElementById` from the React component to update the DOM after hydration. Avoids layout shift since the element already exists with a visitor-friendly default.
