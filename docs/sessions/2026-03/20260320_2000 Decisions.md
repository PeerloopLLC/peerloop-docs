# Conv Decisions - 2026-03-20

## 1. Public Certificate Verification Endpoint → API-PLATFORM.md
**Type:** Implementation
**Topics:** docs-infra

**Trigger:** `/api/certificates/[id]/verify` is a public, unauthenticated endpoint but the `certificates` route prefix was initially mapped to API-ADMIN.md (where the admin certificate CRUD lives).

**Options Considered:**
1. Keep in API-ADMIN.md alongside admin cert endpoints
2. Map to API-PLATFORM.md where other public endpoints live ← Chosen
3. Create a new API-CERTIFICATES.md

**Decision:** Map the `certificates` prefix to API-PLATFORM.md, where it joins `/api/faq`, `/api/stories`, `/api/contact`, and other public endpoints.

**Rationale:** API-ADMIN.md is for admin-only endpoints. The verify endpoint has no auth requirement — it's designed for external verification (e.g., embedding in resumes, sharing links). API-PLATFORM.md already groups public utility endpoints.

**Consequences:** The admin certificate routes (`/api/admin/certificates/*`) are correctly handled by the `admin` prefix → API-ADMIN.md mapping and are unaffected.

## 2. Two-Level Sub-Route Mapping for `/me/*` Routes
**Type:** Implementation
**Topics:** docs-infra, cc-workflow

**Trigger:** All `/me/*` routes mapped to API-ENROLLMENTS.md via the single `me` prefix, but these routes span 6 different doc files.

**Options Considered:**
1. Keep single mapping, accept false positives ← Rejected
2. Add two-level `me/{sub}` prefix matching ← Chosen
3. Move all `/me/*` docs into a single API-ME.md file

**Decision:** Extended sync-gaps.sh to check `me/{sub}` mappings first (15 entries), falling back to `me` → API-ENROLLMENTS.md for unmapped sub-routes.

**Rationale:** The existing doc structure (API-USERS.md for profile, API-MESSAGES.md for messages, etc.) organizes by domain, not by URL prefix. Moving everything to API-ME.md would duplicate content and break the domain-based organization. Two-level mapping keeps the script accurate without restructuring docs.

**Consequences:** route-mapping.txt grew from 25 to 49 entries. Future `/me/*` routes need a mapping entry or they'll report as undocumented against API-ENROLLMENTS.md.
