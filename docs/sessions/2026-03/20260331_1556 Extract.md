### CTE Enrollment Lookup Failure

**Claude:** Discovered that CTE-based enrollment lookups (`_e` CTE referencing `_u` and `_c`) return NULL when used inside INSERT statements in the D1/better-sqlite3 test environment. User/course CTEs (`_u`, `_c`) work fine. Root cause unclear — possibly a SQLite CTE scoping limitation with cross-CTE scalar subqueries in INSERT context. Workaround: replaced enrollment lookups with explicit JOIN pattern (`FROM enrollments e JOIN users u ON ... JOIN courses c ON ... WHERE u.email = X AND c.title = Y`). Split enrollment-dependent inserts from UNION ALL into individual steps for reliability.

### flattenCourseData Bug Fix

**Claude:** Discovered Amanda was enrolling in AI Tools Overview (Guy's course 0) instead of Intro to the Q-System (Gabriel's course 0). Fixed by checking the `'creator'` actor slot first (which correctly points to the bound creator persona). Also fixed `courseTitle` propagation to prefer the `'creator'` slot. This was a pre-existing bug affecting all multi-creator scenarios where a non-Guy creator's course was selected via courseIndex.
