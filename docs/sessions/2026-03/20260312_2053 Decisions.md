# Session Decisions - 2026-03-12

## 1. TodoWrite for All Unresolved Items
**Type:** Strategic
**Topics:** cc-workflow

**Trigger:** During `/w-codecheck fix`, I silently categorized 28 pre-existing TypeScript errors as "not my problem" and only fixed the 3 errors from the current session's code. The user called this out.

**Options Considered:**
1. Only fix regressions from current session (what I did)
2. Fix all issues when explicitly running `/w-codecheck fix` ← Chosen
3. TodoWrite pre-existing issues but don't fix them

**Decision:** `/w-codecheck fix` means fix ALL issues. Any issue not immediately resolved — for any reason — must be added to TodoWrite. This includes pre-existing errors, ideas, opportunities, and questions for the user.

**Rationale:** TodoWrite items carry forward via `/w-save-state` to RESUME-STATE.md. Silently skipping issues means they're lost. The user should decide priority, not the assistant.

**Consequences:** Added to `~/.claude/CLAUDE.md` (global), project feedback memory, and updated `w-save-state/SKILL.md` to require TaskList pruning.

**Type:** Strategic
**Topics:** cc-workflow

**Options Considered:**

**Rationale:** The user's best ideas often come as asides during other work. Without capture, they're lost in conversation noise.

**Consequences:** Added to `~/.claude/CLAUDE.md` and project feedback memory.

## 4. w-save-state Must Prune TaskList Before Writing
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** Question about whether TodoWrite items would be captured in RESUME-STATE.md when running `/w-save-state`.

**Options Considered:**
1. Hope the conversation scan picks them up (unreliable)
2. Explicitly call TaskList and carry forward all items
3. Call TaskList, prune completed, carry forward remaining ← Chosen

**Decision:** `/w-save-state` must call TaskList, mark completed tasks done (don't carry forward finished work), then include all remaining pending/in-progress tasks in the Remaining section.

**Rationale:** Without explicit TaskList integration, items captured via TodoWrite during the session could be lost. Pruning prevents stale completed items from cluttering the next session.

**Consequences:** Updated `w-save-state/SKILL.md` with explicit instructions.
