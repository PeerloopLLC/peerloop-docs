# Conv Learnings - 2026-03-31

## 1. findBy Dot-Path Parser Requires Paren-Aware Splitting
**Topics:** plato, testing

**Context:** Adding findBy(field,$persona.courseTitle) to extractPath caused failures because the standard split('.') broke $persona.courseTitle across the paren boundary.

**Learning:** When adding syntax with nested dot-paths (like findBy), you can't naively split on dots — need a stateful parser that tracks paren depth.

**Pattern:** parseDotPath() method walks characters, only splits on dots outside parens.

## 2. Multi-Course Scenarios Need Stripe Connect Separation
**Topics:** plato, testing

**Context:** self-certify-creator combines Stripe Connect + teacher certification. Second course invocation fails because Stripe is already connected.

**Learning:** When a run combines a one-time setup with a per-entity operation, split into separate runs for reusable composition. Created add-teacher-cert as the per-course atomic run.

---

## 3. Scenario-Level DB Verification Is The Real Assertion
**Topics:** plato, testing

**Context:** User emphasized that PLATO mutates the DB and "the test that matters" is whether DB state is correct after all runs.

**Learning:** Per-run verify blocks are sanity gates. Scenario-level verify is what proves the intended situation was reproduced. For seed scenarios, this means comprehensive state checks (counts, statuses, relationships) not just existence checks.
