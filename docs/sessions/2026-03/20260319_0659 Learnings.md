# Conv Learnings - 2026-03-19

## 1. RESUME-STATE.md as Transparent TodoWrite Persistence
**Topics:** cc-workflow

**Context:** User wanted a single interface for outstanding work — TodoWrite during conversations, without having to think about RESUME-STATE.md as a separate artifact.

**Learning:** RESUME-STATE.md works best as an invisible serialization layer, not a user-facing file. The pattern mirrors localStorage-backed React state: the user interacts with TodoWrite, and `/r-end` serializes pending tasks to disk, while `/r-start` deserializes them back into TodoWrite and deletes the file. The user never needs to reference RESUME-STATE.md directly.

**Pattern:** Deserialize on start (read → TaskCreate → delete file), serialize on end (TaskList → write file). Git history preserves all states.

## 2. Stale TODO Estimates Drift as Background Work Progresses
**Topics:** docs-infra

**Context:** RESUME-STATE.md carried "~63 undocumented API routes" from a sync-gaps audit. Investigation revealed only 9 remained — most had been documented in intervening sessions.

**Learning:** Long-lived TODO items with quantified estimates become stale as work progresses around them. The RESUME-STATE → TodoWrite transfer helps by surfacing items for re-evaluation each conv, but the estimates themselves should be verified before starting work rather than trusted at face value.

## 3. Response.json() Returns Unknown in Strict TypeScript
**Topics:** astro, testing

**Context:** EnrollButton.tsx had a TS error on `res.json()` — `'data' is of type 'unknown'`.

**Learning:** `Response.json()` returns `Promise<any>` in standard DOM typings, but strict TS configs (or type overlays) can tighten this to `unknown`. Fix with a type assertion at the call site: `const data = (await res.json()) as { fieldName: type }`. Safe when the response shape is known from the endpoint contract and guarded by `res.ok`.
