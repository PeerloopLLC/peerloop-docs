# Session Decisions - 2026-03-04

## 1. Skip st_certification_id on Sessions Table
**Type:** Implementation
**Topics:** d1

**Trigger:** CERT-AUDIT plan considered adding `st_certification_id` to both enrollments and sessions tables.

**Options Considered:**
1. Add to both enrollments and sessions ← More denormalized, direct lookup
2. Add to enrollments only ← Chosen — sessions already link via `enrollment_id`

**Decision:** Only add `st_certification_id` to the enrollments table. Sessions already reference enrollments via `enrollment_id`, so the ST certification is one JOIN away.

**Rationale:** Adding it to sessions would be redundant denormalization with no query benefit. The audit question "which ST certification was active when this session happened?" is answered by joining through the enrollment.

**Consequences:** Simpler schema change (one table instead of two). Queries needing certification info from a session do `sessions → enrollments → st_certification_id`.
