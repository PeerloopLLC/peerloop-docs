### Timezone Bug Discovery

**User:** "The client tested this. He arrived 8 minutes late and he couldn't join. I haven't confirmed this"

**Claude:** Read the join endpoint (`join.ts`) and `SessionRoom.tsx` to trace the join window logic. The API allows joins from 15 min before start through `scheduled_end` — 8 min late should work. Identified the root cause: bare datetime strings (no Z suffix) stored in D1. `SessionBooking.tsx` line 163 constructs `${date}T${time}:00` without Z. Workers parse this as UTC, browsers as local time. A US Eastern user booking 2:00 PM creates `14:00` on the Worker (UTC), but at 2:00 PM local (18:00 UTC) the Worker thinks the session ended at 15:00 UTC.
