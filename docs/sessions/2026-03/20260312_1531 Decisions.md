# Session Decisions - 2026-03-12

## 1. Merge /learn Into Course Detail Page as Learn Tab
**Type:** Strategic
**Topics:** astro

**Trigger:** Implementation of COURSE-PAGE-MERGE plan (decided Session 377, planned Session 378). Students had two overlapping pages per course: marketing detail (`/course/{slug}`) and learning workspace (`/course/{slug}/learn`).

**Options Considered:**
1. Keep both pages, improve cross-links
2. Merge /learn into course detail page as a tab ← Chosen
3. Replace detail page with learning page for enrolled students

**Decision:** Merge the learning workspace into the course detail page as an enrolled-only "Learn" tab with accordion modules, replacing both the standalone /learn page and the Curriculum tab.

**Rationale:** Eliminates navigation confusion. Enrolled students land directly on the Learn tab; visitors see About. The accordion design (Option C from Session 377) shows module titles in collapsed state, giving post-purchase reassurance.

**Consequences:** CourseLearning.tsx (579 lines) and ModuleSidebar.tsx (110 lines) retired. learn.astro converted from standalone page to tab page. Curriculum tab removed entirely. All internal /learn links remain functional.

## 2. Session Action Buttons as Peer Buttons in Module Actions Row
**Type:** Implementation
**Topics:** astro

**Trigger:** User feedback that "Book a Session" and "Session scheduled" links were too small as `text-xs` hints above the Mark Complete button.

**Options Considered:**
1. Keep as hint text above Mark Complete button
2. Move to proper buttons alongside Mark Complete in a flex-wrap row ← Chosen

**Decision:** Refactor the ModuleContent actions section into a horizontal `flex-wrap` button row where Book a Session / Session Scheduled sit as styled buttons alongside Mark Complete.

**Rationale:** Buttons are more discoverable and tappable than tiny text links. The flex-wrap row is extensible for future actions without layout changes.

**Consequences:** ModuleContent actions section rewritten. Book a Session gets `CalendarIcon` + primary-outline style. Session Scheduled gets `CalendarIcon` + green-outline style. Mark Complete unchanged visually.

## 3. Teachers Tab: Assigned Teacher Gating for Enrolled Students
**Type:** Implementation
**Topics:** astro, d1

**Trigger:** When a student is enrolled and assigned to a specific teacher, the Book Session button should only be active for their assigned teacher, and disabled when all modules are already booked.

**Options Considered:**
1. Show Book Session for all teachers equally (current behavior)
2. Only show Book Session for assigned teacher, disable for others ← Chosen
3. Hide non-assigned teachers entirely

**Decision:** Show all teachers but gate the Book Session button: active only for the assigned teacher and only if unbooked modules remain. Non-assigned teachers get a disabled "Book Session" button with tooltip. Fully-booked assigned teacher gets "All Booked" disabled button. Added "Your Teacher" badge on the assigned teacher's card.

**Rationale:** Students should see all course teachers (social proof, awareness) but shouldn't be confused by multiple Book Session buttons when they have an assigned teacher. The badge makes the assignment immediately visible.

**Consequences:** Added `assignedTeacherId` prop to CourseTabsProps, propagated through all 6 Astro tab pages. Teachers tab triggers `fetchCourseSessions()` to compute booked module count. Enrollment query now also selects `e.assigned_teacher_id`.
