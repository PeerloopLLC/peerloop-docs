## 1. Standardize Test File Naming to Path-Mirroring Convention
**Type:** Implementation
**Topics:** testing, docs-infra

**Trigger:** Automated test coverage audit (`comm -23`) produced 24 false positives because 4 test files used flattened names instead of mirroring the source path structure.

**Options Considered:**
1. Keep flattened names, accept manual audit overhead
2. Rename flattened tests to match source paths ← Chosen
3. Write a custom audit script that handles both conventions

**Decision:** All API test files must mirror the source path structure. Renamed 4 files:
- `slug.test.ts` → `[slug]/index.test.ts`
- `slug-members.test.ts` → `[slug]/members.test.ts`
- `slug-progressions.test.ts` → `[slug]/progressions.test.ts`
- `toggle.test.ts` → `[courseId]/toggle.test.ts`

**Rationale:** Path-mirroring is already the convention for 95%+ of test files. Standardizing eliminates false positives in automated audits and makes it trivial to verify coverage with `comm -23`. The 4 renames (plus fixing relative import paths) took ~2 minutes.

**Consequences:** Updated 4 files, fixed relative `importOriginal` paths (added one `../` level). Full suite passes.

## 2. Fix Enrollment Status Query to Use Valid CHECK Values
**Type:** Implementation
**Topics:** d1

**Trigger:** Writing tests for `courses/[id]/sessions.ts` revealed the endpoint queried `status = 'active'` which is not a valid enrollment status per the schema CHECK constraint.

**Options Considered:**
1. Add `'active'` to the CHECK constraint
2. Change query to use valid statuses ← Chosen

**Decision:** Changed the enrollment verification query from `status = 'active'` to `status IN ('enrolled', 'in_progress', 'completed')` to match valid CHECK constraint values and allow students at any active enrollment stage to view their sessions.

**Rationale:** `'active'` was never a valid enrollment status — the CHECK constraint has always been `enrolled | in_progress | completed | cancelled | disputed`. Adding `'active'` would be inventing a new status. The intent was to find students with valid, non-cancelled enrollments.

**Consequences:** Endpoint now works correctly (was silently returning 403 for all requests). Test verifies the fix.
