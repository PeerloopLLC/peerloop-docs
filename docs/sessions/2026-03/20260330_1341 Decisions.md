# Conv Decisions - 2026-03-30

## 1. Analytics Callback URL Is Self-Configuring — No Vendor Contact Needed
**Type:** Implementation
**Topics:** bbb, analytics, blindside

**Trigger:** Fred Dixon's email confirming JWT shared secret

**Options Considered:**
1. Send production URL to Blindside Networks
2. Rely on self-configuring `meta_analytics-callback-url` ← Chosen

**Decision:** The analytics callback URL does not need to be provided to Blindside — it's passed per-meeting via the `create` API call, built dynamically from `request.url.origin`.

**Rationale:** Already wired in Conv 037 (`join.ts:135` → `bbb.ts:306`). Each environment (prod, staging) auto-configures.

**Consequences:** Removed "send URL to Blindside" from RFC checklist. Only remaining step is end-to-end verification (deferred to DEV-WEBHOOKS).

## 2. Discovery Cards Capped at 1 Per Feed
**Type:** Implementation
**Topics:** smart-feed, discovery, diversity

**Trigger:** 3 identical "Intro to n8n" discovery cards in Sarah's feed

**Options Considered:**
1. Keep 3 per feed (current behavior)
2. Cap at 1 per feed ← Chosen

**Decision:** Discovery cards represent "this feed is active, check it out" — showing 1 card per discovered feed, not 3 individual posts.

**Rationale:** Discovery cards are feed-level suggestions, not post-level content. Multiple cards from the same feed are redundant.

**Consequences:** Modified `applyDiversityCap()` to use `effectiveCap = 1` for discovery items.

## 3. Include Completed Enrollments in Teacher/Creator Scoring
**Type:** Implementation
**Topics:** smart-feed, scoring, enrollments

**Trigger:** "From Teachers" tab empty for Sarah despite Guy being her teacher

**Options Considered:**
1. Keep current scope (enrolled/in_progress only) — teacher relationship ends at completion
2. Include completed enrollments ← Chosen

**Decision:** Teacher/creator relationship persists after course completion for Smart Feed scoring and filtering.

**Rationale:** Students maintain meaningful relationships with their teachers. The "From Teachers" filter showing nothing after completion is poor UX.

**Consequences:** Modified `loadScoringContext()` teacher and creator queries to include `'completed'` status.
