# Conv Decisions - 2026-03-17

## 1. Retire w-* Skills With r-* Equivalents
**Type:** Strategic
**Topics:** cc-workflow, docs-infra

**Trigger:** RESUME-STATE.md cleanup task asked to review w-* skills for Skills 2.0 opportunities. Found 6 w-* skills that were direct duplicates of r-* Conv lifecycle skills.

**Options Considered:**
1. Update w-* skills to match r-* patterns — keep both sets
2. Retire w-* skills that have r-* equivalents — remove duplicates ← Chosen
3. Leave w-* as-is — legacy but not actively harmful

**Decision:** Retired 6 w-* skills (w-eos, w-learn-decide, w-dump, w-update-plan, w-commit, w-save-state) that had direct r-* equivalents. Kept 10 w-* skills with no r-* equivalent (timecard, codecheck, sync-docs, etc.).

**Rationale:** Maintaining duplicate skills creates confusion about which to use, doubles the terminology-update burden, and wastes context window on stale content. The r-* skills are the active Conv lifecycle and should be the only versions.

**Consequences:** Updated all dangling references across settings.json, settings.local.json, CLAUDE.md, PLAYBOOK.md, PLAN.md, skills-system.md, DEVELOPMENT-GUIDE.md, w-post-fix, w-git-history, and w-sync-docs.

## 2. Merge w-docs Into r-docs as Single Canonical Docs Skill
**Type:** Strategic
**Topics:** cc-workflow, docs-infra

**Trigger:** After retiring other w-* duplicates, w-docs and r-docs remained as two versions of the same skill. r-docs was a slimmed-down copy that symlinked to w-docs/scripts.

**Options Considered:**
1. Keep both — r-docs as slim version, w-docs as full version
2. Merge w-docs content into r-docs, delete w-docs ← Chosen
3. Make r-docs a thin wrapper pointing to w-docs

**Decision:** Merged w-docs (the fuller version with config.json support, progress tracking, CLI detail files table, extra matrix rows) into r-docs. Moved scripts from w-docs to r-docs (replacing symlink with real directory). Deleted w-docs entirely.

**Rationale:** The r-* skills live in the project folder and are project-specific. If skills are copied to another project, they'll be adapted there. Having a single canonical implementation eliminates the symlink dependency and the confusion of two docs skills.

**Consequences:** r-docs now owns its scripts directory directly. All w-docs references updated across the repo. Permission in settings.local.json changed from `Skill(w-docs)` to `Skill(r-docs)`.
