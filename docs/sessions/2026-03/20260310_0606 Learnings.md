# Session Learnings - 2026-03-10

## 1. Skills 2 Directory Structure Solves the Global/Local Split
**Topics:** cc-workflow, docs-infra

**Context:** The old command system required two separate files (global `q-docs.md` + local `q-docs-local.md`) with explicit handoff logic between them. The global file had to say "now run /q-docs-local" and the local file had to say "extends /q-docs."

**Learning:** Skills 2's directory structure (`SKILL.md` + supporting files) naturally accommodates both global logic and project config in one unit. The old two-file pattern maps cleanly to a single directory. Even better, the files can be merged into a single `SKILL.md` since the file boundary was solving a deployment problem (global vs project repo) that no longer exists when both are colocated.

## 2. `!` Backtick Injection Shifts Claude from Detective to Editor
**Topics:** cc-workflow

**Context:** The old `/q-docs` command was 625 lines of instructions telling Claude to "go find what changed" — run `find`, read `package.json`, compare against docs, build topic indexes, etc. Each step required tool calls (reasoning + execution + parsing).

**Learning:** Skills 2's `!` backtick syntax runs shell commands *before* the prompt is assembled. Helper scripts (`detect-changes.sh`, `sync-gaps.sh`, `tech-doc-sweep.sh`) do all the mechanical work and inject results directly. Claude sees actual data (changed files, gaps, flagged docs) instead of instructions to compute that data. Instructions shrank from 629 → 200 lines (68% reduction), though total prompt grew because it carries pre-computed data. The net effect is fewer tool calls, faster execution, and more deterministic behavior.

**Pattern:** Place scripts in `${CLAUDE_SKILL_DIR}/scripts/` and reference via `!` backtick:
```markdown
### What Changed
!`${CLAUDE_SKILL_DIR}/scripts/detect-changes.sh 2>/dev/null || echo "(unavailable)"`
```

## 3. macOS Bash 3.x Doesn't Support Associative Arrays
**Topics:** cc-workflow

**Context:** Wrote a `tech-doc-sweep.sh` script using `declare -A` for associative arrays to map code patterns to doc topics.

**Learning:** macOS ships with bash 3.x which doesn't support `declare -A` (added in bash 4.0). Scripts targeting macOS must use indexed arrays with parallel name/value arrays, or simple string-based approaches. No need to install bash 4+ — the workaround is simple and portable.

## 4. Skills 2 Scoping Handles Per-Project Migration Naturally
**Topics:** cc-workflow

**Context:** Needed to migrate `/q-docs` to Skills 2 for Peerloop without breaking other projects that still use the old command format.

**Learning:** Skills 2 has a precedence hierarchy: project `.claude/skills/` overrides personal `~/.claude/commands/` of the same name. So placing a skill at `.claude/skills/q-docs/SKILL.md` shadows the global command — but only in that project. Other projects continue using the old global command + local extension pair. No renaming needed. Migration can happen one project at a time.

## 5. Drift Between Skill Copies Is Manageable, Not Preventable
**Topics:** cc-workflow, docs-infra

**Context:** Explored symlinks to prevent drift between canonical and project skill files. Realized that projects will inevitably need project-specific changes to the "global" parts of a skill (e.g., dual-repo path handling).

**Learning:** Real copies with drift management tools (drift-report.sh, skill-patch.sh) are more practical than symlinks that enforce uniformity. The key tooling: (1) `.sync` metadata files track last reconciliation point, (2) section-level diff reporting shows *which* sections diverged, (3) bidirectional sync allows innovations to flow in any direction. The `<!-- PROJECT -->` comment convention in canonical templates marks expected customization points.
