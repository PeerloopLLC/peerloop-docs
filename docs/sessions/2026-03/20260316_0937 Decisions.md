# Session Decisions - 2026-03-16

## 1. First-Booking Teacher Assignment (Backfill on NULL)
**Type:** Implementation
**Topics:** d1, auth

**Trigger:** "Teacher does not match your enrollment" error on staging when booking with any teacher after enrolling without a teacher pre-selected.

**Options Considered:**
1. Restrict UI to only show assigned teacher (breaks enrollments with no teacher)
2. Remove strict check entirely, allow booking with any teacher always
3. Allow first booking to assign teacher, enforce match on subsequent bookings ← Chosen

**Decision:** When `enrollment.assigned_teacher_id` is NULL, the session booking API accepts the selected teacher and backfills both `assigned_teacher_id` and `teacher_certification_id` on the enrollment. When the field is already set and doesn't match, the existing rejection stands.

**Rationale:** Option 3 preserves the 1-on-1 teacher relationship model (students can't silently switch teachers mid-course) while supporting the legitimate flow where a student enrolls without choosing a teacher upfront. Option 2 would break the teacher continuity guarantee. Option 1 would block all enrollments that didn't pre-select a teacher.

**Consequences:** `POST /api/sessions` now does an `UPDATE enrollments` when `assigned_teacher_id` is NULL. The teacher certification query was expanded to return `id` for the backfill. No schema changes needed.
