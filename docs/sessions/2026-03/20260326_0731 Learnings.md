# Conv Learnings - 2026-03-26

## 1. Child component dependencies surface during test mocking
**Topics:** testing, react, currentUser

**Context:** After adding useCurrentUser() to TeacherDashboard, tests crashed in MyFeeds child component which independently calls useCurrentUser().getFeeds()

**Learning:** When mocking a global hook like useCurrentUser(), the mock must satisfy ALL consumers in the component tree — not just the component under test. MyFeeds also needed getFeeds() and its /api/me/feed-badges fetch to be mocked.

**Pattern:** Include method stubs (getFeeds, etc.) in mock CurrentUser objects, and add catch-all fetch mocks for child component API calls.

## 2. API response trimming is safe when client already has the data
**Topics:** api-design, currentUser, architecture

**Context:** Teacher-dashboard and creator-dashboard APIs returned user.name/handle that the client already had from CurrentUser

**Learning:** Removing identity fields from API responses is a clean optimization when the client has a global cache (CurrentUser). The key distinction is identity (cacheable) vs transactional (must be fresh). is_available is transactional teacher state, not identity — it belongs in the API response.
