# Conv Learnings - 2026-03-27

## 1. CurrentUser Already Has All Role Data Client-Side
**Topics:** currentUser, architecture, data-strategy

**Context:** Investigating whether new API endpoints were needed for the unified course pages

**Learning:** The `MeFullResponse` from `/api/me/full` already includes enrollments, teacher certifications, created courses, and `communityModeratedCourseIds` — all loaded into the CurrentUser singleton with O(1) Map lookups. This means the listing page can annotate every course with role badges entirely on the client with zero new API endpoints.

**Pattern:** For any feature that needs per-course role detection, use `currentUser.isStudentFor(id)`, `.isTeacherFor(id)`, `.isCreatorFor(id)`, `.canModerateFor(id)` — no fetch needed.

## 2. Wrapping vs Copying Large Components
**Topics:** architecture, react, component-design

**Context:** CourseTabs is 1392 lines. The plan originally called for copying it to extend with role tabs.

**Learning:** Instead of copying, `ExploreCourseTabs` wraps the existing `CourseTabs` component and adds a role-specific section below it. This avoids duplicating 1392 lines of universal tab logic while still adding the role section divider. Trade-off: role tabs appear in a separate section rather than inline in the tab bar, but this actually matches the "Approach 1 — role section divider" design naturally.

## 3. CourseCard Stretched-Link Pattern Prevents External Overlays
**Topics:** react, component-design, css

**Context:** Considered wrapping existing `CourseCard` to overlay role badges externally

**Learning:** `CourseCard` uses an `after:absolute after:inset-0` stretched-link pattern on the title anchor. This makes the entire card clickable but also means any externally-positioned overlays (like role badges) would be behind the link layer. The solution was to copy the 198-line component as `ExploreCard` with a `roleBadges` prop and `relative z-10` on the badge container.

## 4. DB-GUIDE.md Table Count Drift Pattern
**Topics:** documentation, maintenance

**Context:** Heading said "47 tables" but rows listed 67 — a drift of 20 tables over many sessions

**Learning:** The table listing was maintained (new tables added as rows) but the heading total was never incremented. This is a systematic documentation drift pattern where summary statistics diverge from the detailed content they summarize. The DOC-SYNC-STRATEGY block (pending in PLAN.md) would catch this class of error.
