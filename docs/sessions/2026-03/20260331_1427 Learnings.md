# Conv Learnings - 2026-03-31

## 1. $persona resolves from current action's actor only
**Topics:** plato, testing

**Context:** follow-user run needed creator's handle but action was performed by student actor

**Learning:** `$persona.X` always resolves from `this.currentActor`'s persona. For cross-actor data, either use `$context` from a prior discovery action, `$actor.X.userId` for session fields, or `$runtime.X` for shared values.

**Pattern:** Runner now auto-propagates `courseTitle` to `$runtime` after `flattenCourseData`, making it accessible to all actors.

---

## 2. findBy pattern for multi-entity discovery
**Topics:** plato, testing

**Context:** enroll-student used `items[0]` which breaks with multiple published courses

**Learning:** Use `findBy(field,$runtime.value)` in provides paths to match specific items in arrays. Already implemented in extractPath(). Works with `$runtime`, `$persona`, `$context` value references.

**Pattern:** `provides: { courseId: 'items.findBy(title,$runtime.courseTitle).id' }`

---

## 3. Stripe mock must return unique account IDs
**Topics:** plato, testing, stripe

**Context:** Two creators self-certifying caused `UNIQUE constraint failed: users.stripe_account_id`

**Learning:** The `stripe-connect` mock setup returned hardcoded `acct_test_123`. With multiple creators, this violates the unique constraint. Mock must use a counter or timestamp for unique IDs.

---

## 4. certify-teacher eligible_students[0] is non-deterministic
**Topics:** plato, testing

**Context:** Sarah and Marcus both completed AI Tools. eligible_students[0] could be either.

**Learning:** When multiple students complete the same course, `eligible_students[0].user_id` is non-deterministic. Use `$actor.student.userId` directly in the certify body instead of discovering from the list.

---

## 5. Handle validation rejects hyphens
**Topics:** auth, validation

**Context:** Seed personas used SQL seed handles like `guy-rymberg` which contain hyphens

**Learning:** The register endpoint validates handles with `/^[a-zA-Z0-9_]+$/` — only letters, numbers, underscores. Hyphens are rejected. SQL seed bypasses validation by inserting directly.
