# Session Learnings - 2026-03-10

## 1. Skills 2 Has No Include or Inheritance Mechanism
**Topics:** cc-workflow

**Context:** Evaluating whether multiple skills sharing common git-extraction logic could reference a shared prompt fragment.

**Learning:** Skills 2 provides no `!include`, template inheritance, base skills, or cross-skill invocation. Each SKILL.md is fully standalone. The only sharing mechanism is `!` shell injection via `${CLAUDE_SKILL_DIR}` — a shared script can output text that becomes part of the prompt, but shared *instructions* (formatting rules, routing logic) must still be duplicated in each SKILL.md.

## 2. PLAN.md Archival Can Silently Drop Future Items
**Topics:** cc-workflow, docs-infra

**Context:** The SKILLS-MIGRATE block in PLAN.md had two "Future:" items appended to the end of a table cell. When the block was archived to COMPLETED_PLAN.md, the summary condensed completed work only — the future items were not carried forward to a new block or the deferred list.

**Learning:** "Future" items embedded inline in a status field (rather than as proper sub-blocks with checkboxes) are vulnerable to being lost during archival. Items marked "Future" are by definition *not done* and must be explicitly handled during block completion — either promoted to a new block, added to the deferred list, or explicitly dropped with rationale.

## 3. Agent Subprocesses Cannot Access Parent Conversation History
**Topics:** cc-workflow

**Context:** Evaluating whether `/q-eos` skills (q-dump, q-learn-decide, q-update-plan) could run as parallel Agent subprocesses to reduce token cost.

**Learning:** Agent subprocesses start with a fresh context — they see only their launch prompt, not the parent conversation's history. Since 3 of 4 EOS skills need to scan the full session transcript (for different purposes: transcript extraction, insight identification, progress summary), they cannot be meaningfully parallelized via the Agent tool. The same limitation blocks checkpoint/restore — there's no programmatic `/compact` tool to reset context between sequential skill invocations.
