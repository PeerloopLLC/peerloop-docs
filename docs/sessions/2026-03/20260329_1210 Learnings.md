# Conv Learnings - 2026-03-29

## 1. NetworkState.authStatus Already Existed
**Topics:** current-user, react-hooks, auth

**Context:** Investigating how to distinguish "loading" from "not authenticated" in useCurrentUser

**Learning:** The `NetworkState.authStatus` infrastructure already tracked 5 states (`loading | authenticated | visitor | session_expired | error`) with `getNetworkState()`/`setNetworkState()` — it just wasn't wired up to React's rendering lifecycle. The fix was ~30 lines: add a listener set + notification in setNetworkState + a hook mirroring useCurrentUser's pattern.

---

## 2. Vitality Gate Creates Cold-Start Problem
**Topics:** smart-feed, feed-discovery, platform-launch

**Context:** Sarah's /discover/feeds showed empty despite having matching tags

**Learning:** Using `vitality > 0` as an inclusion gate (not just ranking signal) means feeds with zero posts are invisible to all users. On a fresh platform, this makes the entire discover surface appear empty. The fix: keep `ORDER BY vitality DESC` for ranking, remove the post-query `.filter(f => f.vitality > 0)`.

---

## 3. Dev Seed Timestamps Are All Stale
**Topics:** seed-data, testing, dev-environment

**Context:** Investigating why Smart Feed showed no content

**Learning:** All dev seed timestamps are hardcoded to 2024 (15+ months old). Any recency-aware query (vitality's 14-day window, smart feed cursors, last_login) finds nothing current. Need relative timestamps (`strftime('now', '-N days')`) for seed data to stay fresh across dev setups.

---

## 4. SSR/Hydration Mismatch with Expired Sessions
**Topics:** astro, react-islands, auth, ssr

**Context:** Visitor seeing authenticated layout with infinite skeletons on /discover/feeds

**Learning:** Astro SSR renders conditionally based on `isAuthenticated` at request time. If a session expires after SSR (without page reload), the static HTML shows the authenticated layout, but React islands (`client:load`/`client:only`) can't load user data. Components treating `!currentUser` as "loading" show infinite skeletons. Fix: expose auth status reactively so components can distinguish loading from not-authenticated.
