### CourseTabs Refactoring (Plan Mode)

**Claude:** Entered Plan mode. Launched 2 Explore agents in parallel to map all CourseTabs references, tests, explore detail-tab components, DB types overlap, and page usage. Then launched Plan agent to design the implementation.

Key design decisions:
1. `extraTabs` + `basePath` props — CourseTabs stays role-agnostic
2. Each tab becomes its own component
3. Parent keeps session fetch (shared data for Teachers + Sessions tabs)
4. ExploreCourseTabs becomes thin mapper (~40 lines)

User approved the plan.

### New Patterns
- **extraTabs prop pattern**: CourseTabs accepts additional tabs via `ExtraTabConfig[]` — id, label, icon, roleColor, groupLabel, content. Enables any caller to inject tabs without modifying CourseTabs internals.
