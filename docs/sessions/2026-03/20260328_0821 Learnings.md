# Conv Learnings - 2026-03-28

## 1. Catch-All Route for Tab Sub-Routes
**Topics:** astro, routing, architecture

**Context:** Needed bookmarkable tab URLs for `/discover/course/[slug]/teachers` etc. The original `/course/[slug]/` used 8 separate .astro files, each duplicating ~330 lines.

**Learning:** Astro's `[...rest]` catch-all parameter handles all tab sub-routes in a single file. Tab ID validation via `Set` + redirect for invalid segments gives clean UX. The tradeoff is SSR code duplication between index.astro and [...tab].astro, but it's far better than 8+ duplicate files.

**Pattern:** `VALID_TABS.has(tab) ? render(initialTab=tab) : redirect(basePath)`

---

## 2. Component Directory vs Route Directory Separation
**Topics:** astro, architecture

**Context:** After moving pages from `/explore/` to `/discover/`, tests still passed because component imports use `@components/explore/...` which is independent of page routes.

**Learning:** In Astro, component organization (`src/components/`) and URL structure (`src/pages/`) are completely decoupled. Moving pages doesn't affect component imports or tests. This separation means route changes are low-risk operations.

---

## 3. Compact Badge Pattern for Cross-Role Indicators
**Topics:** react, ui, accessibility

**Context:** Role tabs showed verbose "other role" badges (e.g., "Teaching · Active" on the Student tab) creating visual noise on role-specific tabs.

**Learning:** Single-letter colored circles (S/T/C/M) with `title`/`aria-label` tooltips provide the same information at ~15% of the visual footprint. Native HTML `title` avoids tooltip library dependencies while maintaining accessibility.
