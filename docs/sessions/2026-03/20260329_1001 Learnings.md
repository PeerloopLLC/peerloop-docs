# Conv Learnings - 2026-03-29

## 1. Bulk Text Replacement Creates SQL Comma Artifacts
**Topics:** testing, tag-taxonomy, migrations

**Context:** TAG-TAXONOMY Phase 6 (Conv 050) removed `category_id` references from 157 test files using bulk text replacement. The `?` placeholders were removed but adjacent commas merged into `,,`.

**Learning:** When bulk-removing a column from SQL INSERT statements, the replacement must handle delimiter cleanup — removing the column name from the column list AND the `?` from VALUES AND the corresponding `.bind()` parameter AND the commas on both sides. A simple find-replace of the column/value leaves orphaned commas.

**Pattern:** After any bulk rename touching SQL, run the full test suite before declaring the phase complete. Grep for `,,` in SQL strings as a sanity check.

## 2. Vitest Unhandled Errors Poison Parallel Workers
**Topics:** testing, vitest

**Context:** 24 "topics is not iterable" errors from OnboardingProfile.test.tsx caused 143 test files to be marked as failed — a 6x multiplier from the actual failures.

**Learning:** Vitest's parallel worker pool propagates unhandled exceptions across all files running in the same pool. A single crashing component test can mark dozens of unrelated test files as "failed". Always check the "Unhandled Errors" section at the bottom of Vitest output first — it reveals the true root cause count vs the inflated file count.

## 3. Two Public Endpoints for Taxonomy Data
**Topics:** api, tag-taxonomy

**Context:** Components needed either raw topic list (for dropdowns) or topic+tag groups (for TopicPicker). Two endpoints serve different shapes.

**Learning:** `GET /api/topics` returns flat topic list `{ topics: [{id, name, slug, ...}] }`. `GET /api/tags` returns tags grouped by topic `{ topics: [{id, name, tags: [{id, name}]}] }`. Components using TopicPicker must use `/api/tags`. Components needing just topic names for dropdowns use `/api/topics`.
