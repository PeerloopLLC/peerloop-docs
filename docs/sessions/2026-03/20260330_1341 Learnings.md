# Conv Learnings - 2026-03-30

## 1. Smart Feed D1 Fallback Pattern
**Topics:** smart-feed, enrichment, resilience

**Context:** Seed `feed_activities` have fake `stream_activity_id` values that Stream.io can't resolve. The enrichment pipeline returned empty placeholder activities.

**Learning:** When Stream doesn't return data for an activity, the enrichment pipeline should fall back to D1 for user names and feed names. This is also production-safe — protects against Stream outages.

**Pattern:** Three parallel batch queries (`fetchUserNames`, `fetchCommunityNames`, `fetchCourseNames`) only fire for candidates Stream didn't return. Zero overhead in normal path.

---

## 2. React Hooks Must Be Called Before Early Returns
**Topics:** react, hooks, ExploreFeeds

**Context:** `ExploreFeeds.tsx` had `useMemo`/`useEffect` after `if (!currentUser) return`. On re-render when user loads, hook count changed.

**Learning:** All hooks must be above any conditional returns. Use optional chaining (`currentUser?.getFeeds() ?? []`) to make hooks safe when data is null.

---

## 3. Filter by Actor Role, Not Surface Reason
**Topics:** smart-feed, scoring, filtering

**Context:** "From Teachers" filter checked `surfaceReason === 'teacher_post'`, but recency weight (0.30) > relationship weight (0.25) means teacher posts almost always surface as `'recent'`.

**Learning:** Client-side role filters should use boolean flags (`isTeacherPost`, `isCreatorPost`) computed during scoring, not the `surfaceReason` which reflects which algorithm signal was dominant.

---

## 4. Append-Only UPDATE Sweep for Seed Data Freshness
**Topics:** seed-data, timestamps, strftime

**Context:** 240 hardcoded 2024 timestamps in seed file, but only feeds and booking data were functionally broken.

**Learning:** Keep original INSERTs as narrative documentation. Append a `TIMESTAMP FRESHNESS` section with `strftime()`-relative UPDATEs that shift only the time-sensitive records. Self-adjusting on every `db:setup:local:dev` run.

---

## 5. Completed Enrollments Should Include Teacher Relationships
**Topics:** smart-feed, scoring, enrollments

**Context:** Teacher/creator lookup query filtered `status IN ('enrolled', 'in_progress')`, excluding completed enrollments. Sarah's completed enrollment with Guy meant he wasn't recognized as her teacher.

**Learning:** Students maintain teacher relationships after course completion. The scoring context should include `'completed'` enrollments for teacher/creator identification.
