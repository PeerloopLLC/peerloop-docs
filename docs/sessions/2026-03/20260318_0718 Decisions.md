# Conv Decisions - 2026-03-18

## 1. Rename DB Setup Combo Scripts to Additive Pattern
**Type:** Implementation
**Topics:** d1, workflow

**Trigger:** User found the existing combo script names confusing — ambiguous defaults, inconsistent suffixes, and missing combinations.

**Options Considered:**
1. Keep existing names, just add missing combos
2. Rename to additive `db:setup:{target}:{level}` pattern ← Chosen

**Decision:** Renamed all DB setup combo scripts to follow `db:setup:{target}:{level}` where each level chains through the previous:
- `db:setup:local` — reset + migrate (base, production-like)
- `db:setup:local:dev` — + dev seed
- `db:setup:local:stripe` — + dev + stripe
- `db:setup:local:booking` — + dev + stripe + booking
- Same pattern for `:staging` variants

**Rationale:** Base should be minimal (production-like), suffixes add layers. Each name tells you exactly what you get. The chain mirrors the natural data dependency order.

**Consequences:**
- Removed: `db:setup:local:clean`, `db:fullsetup:local`, `db:setup:booking` (and staging counterparts)
- Updated: CLAUDE.md, CLI-QUICKREF.md, CLI-REFERENCE.md, SCRIPTS.md, TEST-E2E.md, migrations.md, env-vars-secrets.md
- `db:setup:local:booking` now includes Stripe seed (old `db:setup:booking` did not)
