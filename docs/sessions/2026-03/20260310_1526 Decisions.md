# Session Decisions - 2026-03-10

## 1. Defer Skills Git-History Extraction Commonality
**Type:** Strategic
**Topics:** cc-workflow

**Trigger:** Analysis revealed q-timecard, q-timecard-dual, and q-git-history share near-identical git log extraction, exclusion rules, and tense conversion logic — a drift risk.

**Options Considered:**
1. Shared shell script (`_shared/git-extract.sh`) called by all three skills via `${CLAUDE_SKILL_DIR}` ← Considered
2. Merge q-timecard and q-git-history into one skill with output-format flag
3. Wait for a potential agentic approach (background commit watcher) ← Chosen

**Decision:** Defer — don't extract shared infrastructure now. The likely future direction (an agentic commit watcher monitoring both repos) would obsolete any shared script built today.

**Rationale:** The three skills work, change rarely, and drift risk is manageable. Premature extraction into shared scripts would add complexity for a pattern that may be replaced entirely.

**Consequences:** The three skills retain duplicated git extraction logic. Acceptable until an agentic approach materializes.

## 2. Drop Parallel Agents and Checkpoint/Restore EOS Sub-Blocks
**Type:** Strategic
**Topics:** cc-workflow

**Trigger:** Two "Future" sub-blocks from SKILLS-MIGRATE proposed optimizing `/q-eos` token cost: (a) run session skills as parallel Agent subprocesses, (b) automated context reset between sequential skill invocations.

**Options Considered:**
1. Parallel agents with pre-extracted session content
2. Sequential with automated compact/checkpoint between skills
3. Drop both — not viable with current Claude Code capabilities ← Chosen

**Decision:** Drop both. Neither approach is viable until Agent subprocesses can access the parent conversation history.

**Rationale:** 3 of 4 EOS skills scan the full session conversation (for transcript, insights, and progress). Agents can't see it. Pre-extraction would do most of the intellectual work sequentially anyway, negating the parallelism benefit. Checkpoint/restore requires a programmatic `/compact` that doesn't exist.

**Consequences:** `/q-eos` continues running 4 skills sequentially with full context accumulation. Token cost remains high but functional.
