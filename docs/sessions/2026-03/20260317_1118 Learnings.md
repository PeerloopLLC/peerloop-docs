# Conv Learnings - 2026-03-17

## 1. Backtick Pre-computation Grep Pitfall
**Topics:** cc-workflow

**Context:** Auditing w-* and r-* skills for Skills 2.0 compliance required counting `!` backtick pre-computation lines. Ripgrep's regex engine couldn't reliably match the pattern.

**Learning:** Shell `grep -c '!\`'` is more reliable than ripgrep for matching backtick-containing patterns. When auditing skills for `!` backtick usage, use a bash loop with `grep -c` rather than the Grep tool.

## 2. Skill Retirement Requires Systematic Reference Sweep
**Topics:** cc-workflow, docs-infra

**Context:** Retiring 7 w-* skills (w-eos, w-learn-decide, w-dump, w-update-plan, w-commit, w-save-state, w-docs) created dangling references across many files.

**Learning:** Skill names propagate into settings.json (permissions), settings.local.json (permissions), CLAUDE.md (skill tables), PLAYBOOK.md (decision records), PLAN.md (open questions), skills-system.md (inventory), other skills (cross-references), and reference docs. After deleting a skill, always grep the entire repo for its name — both as `/w-skillname` (user-facing) and `w-skillname` (path/reference form). Historical session logs are exempt.

## 3. Symlinked Skill Scripts Create Hidden Dependencies
**Topics:** cc-workflow, dual-repo

**Context:** r-docs had a symlink `scripts -> w-docs/scripts/`. When consolidating r-docs and w-docs, deleting w-docs first would have broken r-docs silently.

**Learning:** Before deleting a skill directory, check if other skills symlink to it. Use `ls -la` on the `scripts/` subdirectory of dependent skills. When consolidating, move the real directory first, then delete the source.
