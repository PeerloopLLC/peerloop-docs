### Creator Studio Edit Button Bug

**User:** Reported that clicking "Edit" on a course at `/creating/studio` navigates back to the same route instead of showing the editor.

**Claude:** Found the root cause: `handleEditCourse` accepted `courseId` but ignored it, just doing `window.location.href = '/creating/studio'`. Same issue with `handleCourseCreated`. The `CourseEditor` component existed but was never imported or rendered.

### Fix Attempt 1: React State Approach

**Claude:** Implemented a state-based fix: added `editingCourseId` state, imported `CourseEditor`, conditionally rendered it when a course was selected. Both handlers set state instead of navigating.

### Review and Switch to URL Param Approach

**User:** Asked to review the approach for a better alternative.

**Claude:** Identified problems with the state approach: URL doesn't change (no refresh safety, no browser back, not bookmarkable), asymmetric enter/exit navigation. Recommended URL query params (`?course=<id>`) instead, consistent with existing patterns in `Messages.tsx` and `StripeConnectSettings.tsx`.

**User:** Acknowledged the state approach worked nicely from a UX perspective but agreed to try the URL param method.

### Fix Attempt 2: URL Param Approach

**Claude:** Switched handlers to use `window.location.href = /creating/studio?course=${courseId}`. Read the param via `new URLSearchParams(window.location.search)` directly in the component body.

### SSR Error: `window is not defined`

**User:** Reported the `window is not defined` error — Astro SSR has no `window` object.

**Claude:** Moved URL reading into a `useEffect` (client-only). Combined the URL check and course fetch into a single mount effect to avoid a race condition where two separate effects would both fire on the same tick. Also moved the `editingCourseId` render guard before the `loading` guard to prevent a stuck spinner.

### Breadcrumb Update for Editor Mode

**User:** Asked to update breadcrumbs when in editor mode.

**Claude:** Updated `studio.astro` to read `Astro.url.searchParams.has('course')` server-side. When editing: breadcrumbs show "Creating > Studio > Edit Course" (with Studio as a link), page title changes to "Edit Course | PeerLoop", and the static header is hidden (CourseEditor provides its own).
