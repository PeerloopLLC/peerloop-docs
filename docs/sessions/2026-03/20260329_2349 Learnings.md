# Conv Learnings - 2026-03-29

## 1. Communities Table Has No deleted_at Column
**Topics:** d1, schema

**Context:** Created batch community intel endpoint mirroring the courses batch pattern, which filters with `AND deleted_at IS NULL`

**Learning:** Unlike courses, users, and enrollments, the `communities` table does not have a `deleted_at` column for soft deletes. The batch community endpoint SQL had to omit this filter. Always verify schema columns before copying patterns from other entity endpoints.

---

## 2. AdminBadge Tooltip Grammar Bug
**Topics:** testing, components

**Context:** Writing comprehensive AdminBadge unit tests revealed a grammar issue in the tooltip

**Learning:** The tooltip used `${count} item${count !== 1 ? 's' : ''} need attention` which produces "1 item need attention" (incorrect). Fixed to also conjugate the verb: `need${count === 1 ? 's' : ''}`. Writing tests for display text catches English grammar bugs that code review misses.

---

## 3. Batch Intel Fetch Keyed on Paginated IDs
**Topics:** performance, react

**Context:** Wiring batch intel into ExploreAllTab and CommunityAllTab listing pages

**Learning:** The `useEffect` dependency is `paginatedItems.map(c => c.id).join(',')` — this re-fetches only when the visible page changes (max 12 items), not on every filter/sort. Well within the 50-ID API cap. This avoids fetching badge data for off-screen items.
