# Conv Learnings - 2026-03-29

## 1. Prefix vs Exact-Match Route Classification
**Topics:** astro, middleware, routing

**Context:** `/community` (index) is member-only but `/community/[slug]` is public. A prefix match on `/community` incorrectly blocks public community pages.

**Learning:** When a URL convention uses the same base path for both member-only (bare path) and public (with slug), split route matching into two sets: prefix-based for pure member trees (e.g., `/admin/*`) and exact-match for bare-only guards (e.g., `/community` but not `/community/[slug]`).

**Pattern:** `PROTECTED_PREFIXES` (match path + sub-paths) + `PROTECTED_EXACT` (match only the exact path)

## 2. Client-Side State Tampering Is Cosmetic, Not Security
**Topics:** security, auth, browser

**Context:** User asked about protecting `window.__peerloop.currentUser` from console tampering via hashing.

**Learning:** Any defense that runs in the browser is bypassable by the browser. Client-side state tampering only changes what the UI shows — every API call validates auth via httpOnly JWT cookies that JS cannot access. The security perimeter is the server, not the client. Hashing client state provides no real security because the attacker can overwrite the verification function.

---

## 3. Middleware = Authentication, Page/API = Authorization
**Topics:** security, middleware, auth

**Context:** Designing the auth guard architecture for Astro pages.

**Learning:** Clean separation: middleware handles "are you logged in?" (authentication via JWT check), while individual pages and API endpoints handle "are you allowed?" (authorization — role checks, enrollment verification, participant access). This avoids middleware needing DB queries for authorization while keeping the auth perimeter tight.

---

## 4. Onboarding Is UX, Not Security
**Topics:** onboarding, auth, ux

**Context:** Debated whether to enforce onboarding completion via middleware gate.

**Learning:** Onboarding completion is a UX concern handled at the component level (nudge banners, CTAs), not a security concern enforced by middleware. Users who skip onboarding simply get a degraded experience (no personalized recommendations). The OAuth callback redirect to `/onboarding` is a good first-touch nudge, but shouldn't trap the user.
