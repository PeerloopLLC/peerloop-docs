# Conv Decisions - 2026-03-28

## 1. Replace Regex-Based href Extraction with Balanced Expression Parser
**Type:** Implementation
**Topics:** route-matrix, tooling

**Trigger:** Route-matrix scanner missed ternary links in CourseHero and Leaderboard components.

**Options Considered:**
1. Add specific regex patterns for ternary variants (quick but fragile)
2. Use balanced brace parser to extract full expression, then regex strings from it ← Chosen
3. Integrate a full AST parser like acorn (heavyweight)

**Decision:** Balanced brace parser — handles all expression types without requiring a full AST dependency.

**Rationale:** Covers ternaries, logical operators, and any future expression patterns. The parser is ~35 lines and handles string/template literal escaping correctly.

**Consequences:** `scripts/route-matrix.mjs` gained `extractBalancedExpr()` function; two simple JSX patterns replaced with general extractor.

## 2. Post-Processing Param Resolution Over Smarter Normalization
**Type:** Implementation
**Topics:** route-matrix, tooling

**Trigger:** `normalizeDynamic` produced generic `[id]` when actual routes use specific params like `[courseId]`, `[handle]`.

**Options Considered:**
1. Add hardcoded path→param mappings in `normalizeDynamic` (fragile, needs updating per route)
2. Post-process targets against known routes using structural matching ← Chosen
3. Parse Astro file names to build a param dictionary (complex, still needs matching)

**Decision:** Structural matching via `resolveTargetParams()` — compare segment patterns (`[*]` wildcards), adopt actual route's param names when structure matches.

**Rationale:** Zero maintenance — automatically works for any new routes without code changes.

**Consequences:** Broken targets dropped from 3 to 1 (the genuinely missing `/course/[slug]/certificate`). No more false positives from param name mismatches.

## 3. Rename components/explore → components/discover
**Type:** Strategic
**Topics:** project-structure, naming-conventions

**Trigger:** User noticed the naming mismatch during route-matrix work.

**Options Considered:**
1. Leave as-is (functional, no runtime impact)
2. Rename folder to match route namespace ← Chosen

**Decision:** Rename to eliminate cognitive dissonance between component folder and route namespace.

**Rationale:** Consistency prevents drift as the project grows. The rename is mechanical now but gets harder over time.

**Consequences:** 37 component files, 8 test files moved; 12 page imports, 19 test imports, 5 doc files updated. Session logs left untouched (historical).
