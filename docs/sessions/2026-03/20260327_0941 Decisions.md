# Conv Decisions - 2026-03-27

## 1. Post-/r-end Fix Pattern: /r-start Without /clear
**Type:** Strategic
**Topics:** cc-workflow

**Trigger:** User frequently needed to make fixes after seeing /r-end output, leading to awkward backfill commits.

**Options Considered:**
1. Gate inside /r-end before commit — adds complexity
2. Split /r-end into /r-end + /r-close — two commands to remember
3. Gate + inline topup skill — new skill to maintain
4. No changes — /r-start without /clear ← Chosen

**Decision:** Make zero changes. When fixes are needed post-/r-end, run /r-start (without /clear) to open a new conv with full context, fix, then /r-end.

**Rationale:** Simplest approach. Each fix gets its own conv number with proper session docs. No new skills, no gates, no conditional branching.

**Consequences:** No code changes. Pattern documented in feedback memory.
