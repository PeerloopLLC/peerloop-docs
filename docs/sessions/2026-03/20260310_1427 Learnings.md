# Session Learnings - 2026-03-10

## 1. Compact CLI Argument Syntax via Letter-Number Pairs
**Topics:** cc-workflow

**Context:** Designing the argument syntax for `/q-timecard-dual` which needs two independent counts (code commits and docs commits).

**Learning:** Single-letter prefixes followed by digits (`c2d3`) create an unambiguous, order-independent compact syntax. The regex `c(\d+)` and `d(\d+)` can extract both values from any ordering, and the full string validates against `^[cd\d]+$`. This avoids the verbosity of `code=2 docs=3` while remaining parseable. The pattern works whenever you need multiple named numeric arguments — the key constraint is that the prefix letters must be unique and not digits themselves.

**Pattern:** Same principle as `chmod 755` — positional meaning from letter+number pairs.

## 2. Fixed Temp Filenames for Disposable Output
**Topics:** cc-workflow

**Context:** Both `/q-timecard` and `/q-timecard-dual` were writing to `/tmp/timecard-session-N.md`, accumulating files in `/tmp/` across sessions (11 found during cleanup).

**Learning:** When output is copy-once-and-discard (like timecards copied to Obsidian), use a fixed filename (`/tmp/timecard.md`) instead of session-numbered names. Each invocation overwrites the last — no accumulation, no cleanup needed. This also eliminates the need to pre-compute the session number via backtick injection, saving a shell call at skill load time. The same pattern already worked for `/q-git-history` with its fixed `/tmp/git-history.md`.

## 3. Marker-Based Tracking Suitability Depends on Failure Mode Asymmetry
**Topics:** cc-workflow

**Context:** Evaluated adding a `latest` argument to `/q-timecard-dual` that would track the last timecarded commit and auto-detect new ones (similar to `.last-qdocs-run` marker in `/q-docs`).

**Learning:** Marker-based tracking works well when a stale/premature marker has low-cost consequences (e.g., detect-changes.sh showing extra files — harmless). It fails when the consequence is *missing data* (e.g., missing billable work from a timecard). The asymmetry in failure modes determines whether the pattern fits. For timecards, the marker would advance at generation time but the user might not actually copy the output — those commits would be invisible to future `latest` runs. The explicit `c2d3` syntax costs 5 seconds of counting but eliminates an entire class of state-tracking bugs.

## 4. Template-as-Truth for Skill Design
**Topics:** cc-workflow

**Context:** During the optimization review of `/q-timecard-dual`, found that the template example showed a title format without commit count, while a prose rule said to add `(N commits)` when total > 1.

**Learning:** When a skill has both a template example and prose rules describing the same formatting, they can drift apart. Making the template show the *most common case* (with realistic values, not placeholders) eliminates ambiguity — Claude treats the template as the primary reference and prose rules as edge-case modifiers. Always update the template first when changing formatting rules.
