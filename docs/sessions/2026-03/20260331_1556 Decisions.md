# Conv Decisions - 2026-03-31

## 1. Enrollment Lookup Strategy: JOINs Over CTEs
**Type:** Implementation
**Topics:** plato, d1, testing

**Trigger:** CTE-based enrollment lookups returning NULL in INSERT context

**Options Considered:**
1. Debug the CTE cross-reference issue in better-sqlite3
2. Use explicit JOINs on base tables for enrollment lookups <- Chosen
3. Create a temporary table for ID lookups

**Decision:** Use explicit JOIN pattern for all enrollment-dependent SqlTopUp steps

**Rationale:** JOINs are proven to work, self-contained per step, and don't depend on CTE cross-referencing behavior. The CTE limitation may be a SQLite/better-sqlite3 quirk not worth debugging for seed data.

**Consequences:** Enrollment-dependent steps use verbose but reliable SQL. Non-enrollment steps still use the shared CTE (`_u`, `_c`, `_comm`).

> **Insight:** When working with SQLite CTEs in INSERT context, prefer explicit JOINs over CTE cross-references for reliability. The verbosity cost is minor compared to the debugging cost of silent NULL propagation.
