# Conv Learnings - 2026-03-20

## 1. sync-gaps.sh Had 3 Compounding Bugs Causing 93% False Positive Rate
**Topics:** docs-infra, cc-workflow

**Context:** Conv 021 flagged 62 undocumented API routes. Conv 022 audited them and found the sync-gaps script itself was the primary problem, not missing documentation.

**Learning:** Three independent bugs compounded to inflate the gap count from ~15 real gaps to 63 reported:

1. **`index.ts` literal matching** — `basename "index.ts" .ts` → `index`, then `grep -qi "index"` in the doc file. Since docs use full route paths like `/api/admin/analytics` rather than the word "index", the match always failed. Fix: search for the parent route path instead (e.g., `admin/analytics`).

2. **Regex bracket interpretation** — `grep -qi "[id]"` interprets `[id]` as a character class matching `i` or `d`, not the literal string `[id]`. Fix: use `grep -qiF` (fixed-string) instead.

3. **Missing route mappings** — 12 route prefixes (conversations, contact, faq, team, stories, debug, moderator-invites, certificates, resources, reviews, stream) had no entry in `route-mapping.txt`, falling through to `API-REFERENCE.md` as default.

**Pattern:** When a detection script reports a large number of issues, verify the script's accuracy on a sample before acting on the full list. False positive rates compound when multiple matching strategies each have independent failure modes.

## 2. API Doc Param Notation Is Inconsistent Across Files
**Topics:** docs-infra

**Context:** After fixing the first two bugs, `[param].ts` files still showed as undocumented because the script converted `[id]` → `:id` but some docs use `[id]` notation (API-SESSIONS.md, API-COURSES.md) while others use `:id` (API-ADMIN.md).

**Learning:** The API docs have no single convention for path parameter notation. The fix was to search for both notations (`/api/sessions/:id` AND `/api/sessions/[id]`). A future cleanup could standardize on one notation across all API docs.

## 3. Single-Level Route Prefix Mapping Is Insufficient for `/me/*` Routes
**Topics:** docs-infra

**Context:** The `me` prefix maps to API-ENROLLMENTS.md, but `/me/*` routes are spread across 6 different doc files (API-USERS.md for profile/settings/notifications, API-MESSAGES.md for messages, API-COMMUNITY.md for communities, API-SESSIONS.md for availability/teacher, API-PAYMENTS.md for payouts, API-ENROLLMENTS.md for courses/enrollments).

**Learning:** Added two-level sub-route mapping support to sync-gaps.sh — `me/profile|API-USERS.md`, `me/messages|API-MESSAGES.md`, etc. The script now checks `me/{sub}` first, then falls back to `me` → API-ENROLLMENTS.md. This pattern would apply to any other route prefix that spans multiple doc files.
