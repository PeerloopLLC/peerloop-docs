# Conv Learnings - 2026-03-28

## 1. Route-Matrix Parameter Name Drift
**Topics:** docs, routing, maintenance

**Context:** Investigating 3 "broken link targets" in route-matrix docs

**Learning:** 2 of 3 "broken" routes were actually parameter naming mismatches — the docs used generic `[id]` while the actual Astro files used semantic names (`[handle]`, `[courseId]`). Route documentation drifts when param names are changed in code but not updated in architecture docs.

**Pattern:** When renaming route parameters, grep route doc files (ROUTE-GRID-MAP.tsv, ROUTE-GRID-REFERENCE.tsv, page-connections.md) in addition to code references.

## 2. HAVING-without-GROUP-BY False Alarm
**Topics:** sqlite, testing, d1

**Context:** TodoWrite item from Conv 044 flagged `getDiscoveryCandidates()` for HAVING clause incompatibility between D1 and better-sqlite3

**Learning:** The function had no HAVING clause — it uses CTEs, subqueries, and app-code filtering. The issue likely existed in an earlier code state and was already resolved by the time it was flagged. RESUME-STATE items should be verified before spending time on fixes.
