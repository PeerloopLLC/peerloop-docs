# Session Learnings - 2026-03-11

## 1. Skills 2 Naming Collisions Cause Confusing UX
**Topics:** cc-workflow

**Context:** When a project skill (`.claude/skills/q-eos/`) and a global command (`~/.claude/commands/q-eos.md`) share the same name, the Claude Code UI shows both as `/q-eos (project)` and `/q-eos (user)`. Selecting the wrong one silently runs the wrong skill.

**Learning:** Skills 2 scoping rules mean project skills shadow global commands of the same name — but only when invoked by exact name. The UI autocomplete shows both, creating a selection hazard. Using a different prefix for project skills (`w-*`) vs global commands (`q-*`) eliminates the ambiguity entirely. The user always knows: `w-*` is project-local, `q-*` is global.

## 2. Bulk Rename Requires Careful Boundary Awareness
**Topics:** cc-workflow, docs-infra

**Context:** Renaming 11 skill directories from `q-*` to `w-*` touched skill files, command files, CLAUDE.md, PLAYBOOK.md, skills-system.md, DEVELOPMENT-GUIDE.md, and shell scripts. A blanket sed replacement would incorrectly change references to global commands (`~/.claude/commands/q-*.md`), historical session logs, and canonical repo paths (`~/skills-canon/q-docs/`).

**Learning:** When doing cross-file renames, categorize references into: (1) active references that should change, (2) global/external references that must stay, (3) historical references left as-is. Directory path patterns (`.claude/skills/q-docs/`) are safer to bulk-replace than slash-command patterns (`/q-docs`) because paths are unambiguous about which copy they reference. Slash-command names require per-occurrence judgment about whether they mean the local or global version.

## 3. Orphaned Local Extension Commands After Skills 2 Migration
**Topics:** cc-workflow

**Context:** Three `*-local` command files (`q-end-session-local.md`, `q-force-end-local.md`, `q-resume-session-local.md`) remained in `.claude/commands/` after the Skills 2 migration. They extended global commands that were no longer the primary workflow — replaced by `w-eos` and similar project skills.

**Learning:** When migrating to Skills 2, audit the old `*-local` command files. If the parent global command is no longer the primary entry point (replaced by a project skill), the local extension is orphaned. The global commands still reference them (`/q-force-end-local if it exists`), but if nobody invokes the global command, the local file is dead code.
