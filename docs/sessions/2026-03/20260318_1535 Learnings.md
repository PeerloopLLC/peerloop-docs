# Conv Learnings - 2026-03-18

## 1. Systematic Documentation Drift Detection via Git Diff
**Topics:** docs-infra, dual-repo

**Context:** Conv 008 (ENROLL-AVAIL) made significant changes — 4 new files, 11 modified files — but the end-of-conv `/r-docs` pass only caught 6 of 10 docs that needed updating. 4 docs were left stale.

**Learning:** Comparing `git diff --name-status` of code changes against the list of docs updated in the same conv reliably surfaces documentation gaps. The approach: (1) list all code files changed, (2) list all docs updated, (3) for each code change, identify which docs should reference it, (4) flag any doc not in the "updated" list.

**Pattern:** The docs most commonly missed by `/r-docs` are specification-level docs (`COMPONENTS.md`, `DB-API.md`) and pattern docs (`DEVELOPMENT-GUIDE.md`), because `/r-docs` focuses on reference docs (`CLI-*.md`, `API-*.md`, `TEST-COVERAGE.md`). Architecture and specification docs require manual review.

## 2. Four Documentation Layers for New Features
**Topics:** docs-infra

**Context:** The ENROLL-AVAIL feature touched 4 distinct documentation layers, each serving different readers and purposes.

**Learning:** A significant new feature typically needs updates across 4 layers: (1) **Component spec** (`COMPONENTS.md`) — props, states, display; (2) **API spec** (`DB-API.md`) — endpoint contract, request/response; (3) **Dev patterns** (`DEVELOPMENT-GUIDE.md`) — reusable patterns for future developers; (4) **Index/reference** (`API-REFERENCE.md`) — quick lookup tables. Missing any layer creates a gap for a different audience.

## 3. Custom Icon System — Three Layers, No External Library
**Topics:** astro

**Context:** Quick question about the icon library surfaced that all icons are custom-built with no npm dependency.

**Learning:** The three-layer icon system (React JSX components in `icons.tsx` ~96 exports, brand icons in `brand-icons.tsx`, Astro path registry in `icon-paths.ts` ~35 entries wrapped by `Icon.astro`) means new icons require adding SVG path data manually rather than importing from a library. This is a deliberate choice for zero-JS Astro pages and full control over the icon set.
