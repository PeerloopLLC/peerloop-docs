# Conv Learnings - 2026-03-26

## 1. Extract files serve as intermediate representations in collect-then-dispatch pipelines
**Topics:** cc-workflow, r-end2

**Context:** Reviewing r-end2 Extract files revealed ~70% content duplication with companion Learnings.md, Decisions.md, and Dev.md files

**Learning:** The Extract is an intermediate artifact — agents consume it, then it should be pruned to its unique value (narrative + uncategorized). The pruning should happen at the synchronization point (after all agents complete) using a manifest of consumed line numbers.

**Pattern:** Immutable source + append-only manifest + reverse-order deletion = safe parallel pruning

---

## 2. Append-only shared files are safe for parallel agent coordination
**Topics:** cc-workflow, r-end2, concurrency

**Context:** Needed a way for parallel agents to communicate which Extract lines they consumed without race conditions

**Learning:** `echo "..." >> file` is atomic on macOS/Linux for writes under PIPE_BUF (4096 bytes). Multiple parallel agents can safely append to a shared manifest file. The key constraints: (1) append-only, never read-modify-write, (2) small writes, (3) no agent modifies the source file.

---

## 3. Descending-order line deletion prevents index cascade
**Topics:** cc-workflow, text-processing

**Context:** Need to delete specific lines from the Extract based on a manifest of consumed line numbers

**Learning:** When deleting multiple lines from a file by line number, sort in descending order (highest first). This ensures earlier line numbers remain valid as later lines are removed. Classic array-removal technique applicable to file editing.
