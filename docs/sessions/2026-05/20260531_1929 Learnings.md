# Conv Learnings - 2026-05-31

## 1. The Conv-226 "garble" is an upstream Claude Code bug cluster, not ours
**Topics:** cc-workflow, tooling

**Context:** Conv 226 reported intermittent blank/partial tool outputs + one fabricated-failure narration.

**Learning:** It maps to a known OPEN upstream cluster — harness parallel-tool cascade-cancel (#22264 → #63966/#63859/#63797: empty-in-UI then late/out-of-order delivery) stacked with model confabulation (#63538: invents un-received output). Strongly correlated with Opus 4.8 + parallel batches + any one sibling failing (runtime error, non-zero tail, flaky MCP, OR a PreToolUse-hook block — we run `guard-dangerous-bash.sh`). Not platform-specific. Unfixed as of 2.1.159.

**Pattern:**
On a suspicious-empty result, verify out-of-band with a cheap *different* probe (`wc -c` / `git status` / `ls`); never re-spam the identical call; never narrate output not actually received.

## 2. A pre-built primitive can sit unadopted by design — Rule-of-Three deferral
**Topics:** matt, cc-workflow

**Context:** `RoleTabBar.tsx` existed (in the dev showcase) but all 3 discover islands rolled their own tabs.

**Learning:** This was intentional Tier-1 shipping with a written IOU — the islands' header comments named [DISC-RTB-RECONCILE] as the owner of convergence. The 3rd instance (Feeds) was the Rule-of-Three trigger; the reconcile is the deferred consolidation pass.

## 3. A behavior-preserving refactor shows as zero test-delta
**Topics:** testing, matt

**Context:** Converging 3 islands onto RoleTabBar.

**Learning:** Suite went 6458→6458 with no test edits because the islands kept their exact state/event/role logic and only swapped render output. The one new behavior (role-colored active tab) is pure CSS/inline-style — invisible to the unit suite, which is why a browser look was the right final check (same blind spot as the Conv 174 spacing regression).
