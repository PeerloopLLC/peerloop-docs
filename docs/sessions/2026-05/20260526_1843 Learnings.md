# Conv Learnings - 2026-05-26

## 1. The wall-time lever is the maintenance *contract*, not the doc count
**Topics:** docs-infra, cc-workflow

**Context:** User wanted less time on doc updates; instinct says "delete docs."

**Learning:** Doc cost has three parts — creation (one-time), maintenance (per-conv churn), and a read-tax ("check the doc first"). Deleting files cuts creation only. The recurring wall-time comes from the *obligation to keep docs current*, enforced by the SessionStart drift hook + /r-end docs agent + the CLAUDE.md "update the doc" mandate. Removing docs from those machines' scope (category) reclaims the time even if zero files are deleted.

## 2. Check for an existing system before building one
**Topics:** docs-infra, cc-workflow

**Context:** Was about to design a `Doc-Tier:` header convention.

**Learning:** A full `docsRegistry` (4 categories = maintenance tiers) already existed from Convs 132–137. The work collapsed from "build tiers" to "re-tune categories + narrow two consumers." Reading the actual machinery first (per ESOT) prevented a parallel reinvention.

## 3. Separate "what relates to a doc" from "do we maintain it"
**Topics:** docs-infra

**Context:** B1 could have been done by deleting vendor keyword rules.

**Learning:** The category-filter design keeps the rules (a useful code→vendor knowledge map) and gates *output* on the registry category. Policy then lives in ONE place (the category field); every consumer inherits it. Also preserved most of the test suite (only vendor positive-controls flipped to negatives).

**Pattern:**
Add a `doc-category` resolver to the registry reader; gate the sweep on `== driftCheck`; default unmatched paths to `manual`.

## 4. Placeholder chars in string-replace must be text tokens, not control chars
**Topics:** cc-workflow, testing

**Context:** globToRegex used a placeholder for `**`→`.*` two-pass replacement.

**Learning:** The placeholder landed as a literal NULL byte (`\x00`) in the .mjs file. Node ran fine and all tests passed — but `git` flagged the file binary. A passing test suite does NOT catch a non-text-byte hazard. Use a visible token like `@@GS@@`. `file <path>` / a NULL-byte grep is the check.

## 5. A thin wrapper hook inherits policy from the script it wraps
**Topics:** docs-infra, cc-workflow

**Context:** User asked why no hook file was edited.

**Learning:** `tech-doc-drift.sh` has no selection logic — it runs `tech-doc-sweep.sh` and presents the output. Changing the sweep + registry changed the hook's behavior with zero edits to the hook. Behavior ≠ file changed.

## 6. spt-docs and Peerloop have diverged capture architectures
**Topics:** cc-workflow, dual-repo

**Context:** User referenced live-capture behavior they see in spt-docs.

**Learning:** spt-docs runs a live event-capture system (`PostToolUse:TaskUpdate` + `UserPromptSubmit` → `.conv-events.jsonl`, Conv 213), read by `event-log-reader.js` / `r-update-tasks`. It RECORDS prompts + task-completions; it does not create tasks. Peerloop has none of this (SessionStart hooks only; Extract built by /r-end conversation scan). Same-named-skill divergence ([[reference_spt_dual_repo]]) extends to the whole capture layer.
