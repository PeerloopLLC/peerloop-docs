# Conv Learnings - 2026-05-30

## 1. conv-tasks.md is the disk-resident crash restore source
**Topics:** cc-workflow, docs-infra

**Context:** Conv 219's crashed predecessor left TodoWrite empty because `/r-start` Step 7 transfers RESUME-STATE → TodoWrite then *deletes* RESUME-STATE; an in-memory TodoWrite doesn't survive a process restart.

**Learning:** `.scratch/conv-tasks.md` is never deleted mid-conv, so it is the surviving on-disk task list after a crash. On a resume-without-`/r-start`, the first move should be to rehydrate TodoWrite from it. RESUME-STATE may also be recoverable via `git show HEAD:RESUME-STATE.md` — but only if its delete was uncommitted.

**Pattern:** resume-without-r-start → restore TodoWrite from `.scratch/conv-tasks.md`; never let a populated conv-tasks.md be overwritten with fewer/zero tasks.

## 2. The pre-primitive tier
**Topics:** matt, cc-workflow

**Context:** ErrorRetryCard — a recurring error+retry card extracted from the settings islands.

**Learning:** A "pre-primitive" composes vetted primitives (FormBanner + Button) but isn't itself registered/vetted. It carries no `data-prov` stamp (its vetted children stamp themselves); §12's 3-value enum has no slot for it, and stamping `legacy` would be a semantic lie. Leaving it unstamped is correct.

---

## 3. Astro primitives can't be composed by React pre-primitives
**Topics:** astro, matt

**Context:** ErrorRetryCard.tsx wanted to use `<Card>`.

**Learning:** `Card` is `.astro`; a React island/component can't render it. React pre-primitives must inline the Matt-tokenized card chrome instead. This is the real reuse boundary between the Astro-primitive layer and React islands.

---

## 4. "Good candidate" needs a primitive-existence check the sensor can't do
**Topics:** matt, testing

**Context:** Grading TopicPicker's 5 candidates.

**Learning:** The sensor nominates structurally; whether a nomination is a *good* matt-inspired candidate depends on whether a matching primitive already exists (clean swap), needs extending (variant gap), or doesn't exist (new-primitive risk). That existence+fit judgment is exactly what `[PRIM-MATCH-INDEX]` would encode.

---

## 5. The conversion crawl was self-inflicted by doing primitive-design inline
**Topics:** matt, cc-workflow

**Context:** User's struggle surfacing the page-conversion slowdown.

**Learning:** Trying to make every page conversion *also* a primitive-design exercise (guessing reuse from n=1) caused the crawl. Splitting into Tier-1 (compose obvious existing primitives now) and Tier-2 (extract/extend later from Rule-of-Three evidence) ends it; the sensor + scratch reports are the deferral's safety net.
