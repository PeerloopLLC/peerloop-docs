## 2. r-quiet-mode: the log file IS the state
**Type:** Strategic
**Topics:** cc-workflow

**Trigger:** Need quiet mode to persist across turns + integrate with r-start/r-end.

**Options Considered:**
1. Context-only + separate marker
2. Hook-reinforced (per-turn injection)
3. Log file as the sole state ← Chosen

**Decision:** Quiet mode is ON iff `.scratch/quiet-mode-log.md` exists; off deletes it; r-end blocks while it exists; r-start surfaces a leftover. Single fixed filename (only ever 0 or 1).

**Rationale:** Eliminates a separate state mechanism; self-consistent lifecycle; no per-turn hook cost.

**Consequences:** New skill + r-end Step 0 guard + r-start Step 5.8. OFF augmented to a mandatory issue-raising pause.
