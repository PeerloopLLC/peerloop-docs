# Conv Learnings - 2026-05-29

## 1. CLAUDE.md-prune vs MEMORY.md-prune are different files with different cap mechanics
**Topics:** cc-workflow, docs-infra

**Context:** The /r-start MEMORY.md-cap alert recommended `/r-prune-claude`, which only prunes CLAUDE.md.

**Learning:** `CLAUDE.md` (repo-root project instructions) has a *soft* always-in-context performance cost, pruned into `CLAUDE-OFFLOAD.md` by `/r-prune-claude`. `MEMORY.md` (auto-memory index under `~/.claude/projects/<slug>/memory/`) has a *hard* SessionStart truncation at 200 lines / 25 KB and is pruned into sibling `memory/*.md` sub-files by the new `/r-prune-memory`. The naming collision ("prune X") masked that they target different files — line-count tools are useless against a byte-bound index.

## 2. MEMORY.md byte bloat is dominated by removable redundancy, not content
**Topics:** cc-workflow, docs-infra

**Context:** MEMORY.md tripped 80% bytes; content re-flattening hit a floor at 72%.

**Learning:** Two byte sinks: (1) index pointer-lines whose hooks grew into paragraphs (re-flatten, but bounded by a ~90B per-line floor → diminishing returns), and (2) the display-label `[filename.md]` echoing the `(filename.md)` target — pure redundancy worth ~9% of the cap, removable by one sed with zero risk because tooling keys on the target, not the label. Attack mechanical redundancy (label normalization) before semantic trimming.

## 3. Skill `!`-backtick executor expands `$0`/`$N` even inside single quotes
**Topics:** cc-workflow

**Context:** r-prune-memory's awk precompute worked in the Bash tool but failed when run as a skill bang-backtick.

**Learning:** The skill bang-backtick execution path performs `$`-token substitution that consumes `$0`/`$1`… even inside single-quoted awk programs (a real shell protects them). Write `$`-free awk: bare `length` ≡ `length($0)`, and a bare `/regex/` pattern ≡ `$0 ~ /regex/`. Named vars (`$MEM`) are the *intended* expansions and survive. Always dogfood a new skill's precompute via actual invocation, not just the Bash tool.

**Pattern:**
```
awk 'length>250 && /\]\([a-z0-9_-]+\.md\)/ { print length"\t"NR }'
```

## 4. Memory-filename regex must allow digits + hyphens
**Topics:** cc-workflow

**Context:** `[a-z_]+\.md` misclassified `e2e-testing-patterns.md` and `project_route_404_honesty_standin.md` as inline-only.

**Learning:** Memory filenames include digits (e2e, 404) and hyphens; the correct character class is `[a-z0-9_-]+`. A too-narrow class silently misroutes the very files that need handling.
