# Conv Decisions - 2026-05-19

## 1. Skill convention: tilde-literal everywhere, no $CLAUDE_PROJECT_DIR or $HOME in skill-issued Bash
**Type:** Strategic
**Topics:** cc-workflow, dual-repo

**Trigger:** Recurring `simple_expansion` permission prompts on every `git -C $CLAUDE_PROJECT_DIR ...` invocation interrupted user workflow. User asked whether the variable could be substituted before the command is issued.

**Options Considered:**
1. Run `/fewer-permission-prompts` to add per-path Bash allowlist entries.
2. Add specific patterns like `Bash(git -C /Users/jamesfraser/...:*)` to `.claude/settings.local.json` by hand.
3. ← Chosen: Sweep all skill SKILL.md files to use tilde-literal `~/projects/peerloop-docs` (outside quotes) and unquoted-tilde assignments (`VAR=~/...`). Mirror spt-docs's converged solution (its Conv 164 `w-test-tilde` empirical confirmation).

**Decision:** Adopt spt-docs's tilde-everywhere convention. Forbid `$CLAUDE_PROJECT_DIR` and `$HOME` in skill-issued Bash command strings. Use `$(echo ~/projects/peerloop-docs | tr / -)` for slug derivations instead of `${CLAUDE_PROJECT_DIR//\//-}`. Applies across all skill SKILL.md files; local-scope script variables (`$SLUG`, `$LIVE`, etc.) are unaffected because the gate only flags external env-var references.

**Rationale:** Substitution-side fix is durable — doesn't require touching settings, and new skills inherit the convention by example. spt-docs proves the pattern works at scale. Settings allowlist (options 1 & 2) would still leak prompts whenever a new path or variant appeared. Tilde is functionally equivalent to `$HOME` and resolves per-user at runtime, so cross-machine portability (M4Pro `jamesfraser` ↔ M4 `livingroom`) is preserved without hardcoding.

**Consequences:** Single-line `git -C ~/projects/peerloop-docs ...` commands now run prompt-free. Multi-line memory-sync forensics blocks also run prompt-free (verified empirically via `HOME=/Users/livingroom bash -c '...'` simulation). 10 skill files swept: `r-start`, `r-commit`, `r-end`, `r-end/refs/fmt-docs.md`, `r-timecard-day`, `w-post-fix`, `w-review-resume-state`, `w-sync-skills`. CLAUDE.md §Path Conventions and memory `feedback_git_dash_c_enforcement.md` updated to document the rule. `persist-project-dir.sh` SessionStart hook flagged as historical (only `w-test-env` diagnostic skill still references `$CLAUDE_PROJECT_DIR`).

> **Insight:** A permission-gate friction loop has two structural fixes: shape the command strings so the gate doesn't fire (substitution-side), or extend the allowlist so the gate accepts them (settings-side). Substitution-side is durable because it's self-propagating — every new skill written by example inherits it. Allowlist-side is leaky because each new path variant requires a new entry. When `~` is functionally equivalent to `$HOME` and the gate treats them differently at parse time, the substitution-side fix is essentially free.

## 2. Mirror student "Recording" affordance verbatim on teacher's My Sessions tab
**Type:** Implementation
**Topics:** ui, video

**Trigger:** User-reported gap — student saw a "Recording" button on `/discover/course/{slug}/sessions`, but teacher saw nothing on `/discover/course/{slug}/my-sessions` for the same recorded session. User asked for UI parity in both places.

**Options Considered:**
1. Use the icon-only + tooltip pattern that Conv 161 [TCV-REC] used for TeacherCourseView.
2. ← Chosen: Mirror student-view `SessionsTabContent`'s bordered text "Recording" button exactly (lines 277-286 pattern).
3. Defer to the open [REC-LABEL] standardisation task.

**Decision:** Verbatim mirror of `SessionsTabContent`'s recording affordance (bordered, text "Recording", `target=_blank`, `rel=noopener`) into `TeacherTabContent.tsx`. Two client-side edits: add `recording_url: string | null` to `SessionRow` interface; add conditional bordered text button in `SessionRowView`. API endpoint `src/pages/api/teaching/courses/[courseId].ts` already returned `recording_url` — gap was client-side JSX only.

**Rationale:** User explicitly asked for UI parity ("I would like the UI to be similar in both places"). Standardisation across all user-facing surfaces is a separate task [REC-LABEL]. Matching one known good surface is lower risk than inventing a new pattern. Same root-cause shape as Conv 161 [TCV-REC] bug (type declared but JSX missing).

**Consequences:** Deployed to staging (Version ID `36c761e7-1dd1-4760-b56b-4d53fa6d30a0`); verified live (Sarah Miller row shows the Recording button; Marcus Thompson row stays plain since no recording_url). Discovery confirms [BR-TRACE]'s "7 user-facing surfaces" map from Conv 161 was incomplete — TeacherTabContent's My Sessions is the 8th. [REC-LABEL] inventory needs updating from 7 to 8 surfaces when tackled.
