# Conv Learnings - 2026-05-27

## 1. The layout already encoded the subnav-ownership answer
**Topics:** astro, cc-workflow

**Context:** User asked whether the subnav should be sent into the page or owned by the page.

**Learning:** `AppLayout` renders `<slot name="sub-nav" />`; an unfilled slot collapses the SubNav strip entirely (no empty rail). So "page owns its subnav (opt-in)" is structural, not a new decision — three states fall out free: page fills it, page omits it, page imports a shared list. The stub duplication was a copy-paste artifact, not the intended design (SubNav is page-section nav).

## 2. "Matt-inspired" has no third provenance marker — unmarked=ours by design
**Topics:** docs-infra, cc-workflow

**Context:** Asked whether Matt-style extrapolations get a dedicated home.

**Learning:** `matt-provenance.md` §6a: authorship is binary (`@matt-source` vs unmarked=ours); Phase-6 extrapolations are unmarked BY DESIGN — their unmarked-but-registered (`prov-candidates.ts`) state is what makes them collision-reconcilable if Matt later draws his own. A dedicated marker would HIDE them from `prov:sweep`.

## 3. prov-sweep's two consumers had diverged (heuristic-calibration catch)
**Topics:** testing, cc-workflow

**Context:** New Phase-6 primitives registered in `PHASE6_EXTRAPOLATION_CANDIDATES` still flagged UNTRACKED.

**Learning:** `prov-sweep` line 201 (collision matcher) read all 3 candidate arrays; line 152 (4c untracked-note check) read only `COMPONENT_CANDIDATES`. The new primitives were the first PHASE6 entries to exercise that path → first to expose the gap. Fix: union COMPONENT + PHASE6 in the registered-path set.

**Pattern:** When a registry has a dedicated array for a class, every consumer must read it.

## 4. @stand-in is a self-erasing to-do written into code
**Topics:** cc-workflow, docs-infra

**Context:** Decided not to formalize `@stand-in` in `matt-provenance.md`/`prov-sweep`.

**Learning:** Retrofitting a page to Matt-style is the act that removes its `@stand-in` marker, so the marker's lifecycle is bounded by the [STANDIN-MATT] task. Unlike `@matt-source`/unmarked (permanent classes `prov-sweep` must police forever), `@stand-in` is transient → `grep -rl '@stand-in' src/pages` is literally the remaining-work counter. Transient markers don't warrant heavy machinery formalization.
