# Conv Learnings - 2026-05-31

## 1. Whole-file `diff -rq` can't see content-level drift inside a shared file
**Topics:** cc-workflow, dual-repo

**Context:** /r-start Step 5.7's data-loss guard only fires on `Only in $LIVE` whole files. A live memory file edited *after* the prior r-end commit (newer content inside a file present on both sides) is invisible to that guard — `mirror→live --delete` would silently revert it.

**Learning:** On a Step 5.7 "1 modified" diff, check direction before applying: if the live mtime is newer than the mirror's last commit, the safe move is live→mirror, not the default mirror→live. Inspecting the textual diff + git log of the mirror copy disambiguates.

---

## 2. The real constraint when splitting a heavily-referenced doc is the write-path, not the read layout
**Topics:** docs-infra

**Context:** DECISIONS.md had 189 inbound refs, and 4 skills + config.json treat it as an *append* target ("update DECISIONS.md inline").

**Learning:** Splitting the file is easy; the risk is future appends landing in a now-pointer root. Keep the root a valid landing page, document the write-path in the index, and file a tracked task to retarget the writers rather than silently breaking them.

---

## 3. Pass derived data INTO a hook, not a derivation callback, to dodge stale-deps
**Topics:** astro, testing

**Context:** Extracting useRoleTabs from two adapters whose visibleTabs derivation differs.

**Learning:** Having the hook own the mechanics and *receive* the caller's `useMemo`-derived `visibleTabs` keeps each component's memo deps natural and lint-clean, and means the hook needn't call useCurrentUser. Passing a derivation callback would have reintroduced the pass-a-function-to-a-hook memo problem.
