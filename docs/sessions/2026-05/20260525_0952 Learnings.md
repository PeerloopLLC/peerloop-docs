# Conv Learnings - 2026-05-25

## 1. Browser verification catches duplicated-source bugs that static checks cannot
**Topics:** astro, testing

**Context:** [SNV-ICONS] gates were green but the bare About page rendered no SubNav icons.

**Learning:** A second copy of `courseTabs` lived in a separate route file (`index.astro`); the edit only touched `[...tab].astro`. tsc/astro/lint can't see "you edited one of two duplicates." Only rendering the actual page surfaced it. The duplication itself was the latent risk — fixed first by extracting one shared source (`_course-tabs.ts`), then by consolidating the routes entirely.

## 2. "Pre-MCP holdover" rot lives in the assembly/shell, not the MCP-extracted primitives
**Topics:** astro, cc-workflow

**Context:** User said the sidebar didn't match Matt and "might be better to start over on this component."

**Learning:** The `NavItem`/`MainNav` primitives (Conv 183 MCP extraction) were already correct; the mismatches (emoji nav, bottom single-arrow collapse, white-on-white shell, flat active state) were all in the *assembly* (`Sidebar.tsx`) and the *shell* (`AppLayout`). "Start over" became "rewrite 2 assembly files + shell, keep the verified primitives." When a Matt-faithful component looks wrong, suspect the wiring above the MCP-extracted leaves before the leaves themselves.

## 3. Figma letterSpacing is a percentage, not pixels
**Topics:** matt-design-system, astro

**Context:** User flagged crammed "Entrepreneur" text in `text-body-medium-bold`.

**Learning:** Figma's `letterSpacing: -2.2` means **-2.2%** = `-0.022em` (= -0.352px at 16px — exactly what Figma's own code export emitted as `tracking-[-0.352px]`). It had been baked as `-2.2px` (~6× too tight) across all four "Body larger sizes" tokens. Units-interpretation bugs at the token layer have design-system-wide blast radius. Use `em` (scales per size: 16px→-0.352px, 20px→-0.44px) rather than a fixed px.

## 4. `position: sticky` on a flex child needs `self-start`
**Topics:** astro

**Context:** SubNav sticky didn't pin until `self-start` was added.

**Learning:** A flex child defaults to `align-items: stretch`, so it fills the row's full height and `position: sticky` has no slack to travel within. `self-start` collapses it to content height, giving sticky somewhere to move. (Same root cause as the sidebar `<aside>` stretching to full body height and pushing its aux cluster to page-bottom instead of viewport-bottom — fixed with `lg:sticky lg:top-0 lg:h-screen lg:overflow-y-auto`.)

## 5. dev-login is email-only impersonation; real logins keep the password with the user
**Topics:** auth, cc-workflow

**Context:** Asked to log in as Brian to test the Profile row.

**Learning:** `/api/auth/dev-login` authenticates by email (no password, dev-gated) — built for PLATO walkthroughs. But the user's standing preference is to drive real password entry themselves via the browser; Claude sets up the login page and waits. (Reinforces `feedback_no_paste_tokens_in_chat`.)
