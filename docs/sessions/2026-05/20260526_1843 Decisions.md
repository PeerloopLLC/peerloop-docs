# Conv Decisions - 2026-05-26

## 1. Invert the default: docs are unmaintained unless opted into driftCheck
**Type:** Strategic
**Topics:** docs-infra, cc-workflow

**Trigger:** Reducing per-conv doc-maintenance wall-time.

**Options Considered:**
1. Tag every doc with an explicit tier header (~90 edits)
2. Default-to-manual; only `driftCheck` is opt-in; unmatched paths resolve to manual ← Chosen

**Decision:** Unclassified/new docs default to `manual` (unmaintained); the matcher returns `manual` for any path not in a driftCheck group.

**Rationale:** Smaller diff, and the *default* now serves the wall-time goal — new docs cost nothing unless deliberately opted in.

**Consequences:** Future docs are free by default; the drift machinery only ever touches an explicit small set.

## 2. Category-filter the sweep instead of deleting vendor keyword rules
**Type:** Implementation
**Topics:** docs-infra

**Trigger:** B1 (narrow the drift hook); test suite asserted on the rule set.

**Options Considered:**
1. Delete ~8 vendor rules + rewrite the count/positive-control tests
2. Keep rules; gate sweep output on `category==driftCheck`; flip vendor-docs→manual ← Chosen

**Decision:** Added a `doc-category` matcher; the sweep skips non-driftCheck docs.

**Rationale:** Keeps rules as a knowledge map, centralizes policy in the category field, and preserves most of the test suite.

**Consequences:** Vendor docs never flag; architecture docs still do; only 3 vendor positive-controls flipped to negatives.

## 4. Vendor docs → manual, preserved (not deleted/trimmed)
**Type:** Strategic
**Topics:** docs-infra

**Trigger:** "Confirm nothing is lost."

**Decision:** Reclassify vendor docs to manual + add snapshot banners; do NOT mass-trim accurate ones; archive (not delete) frozen docs.

**Rationale:** Once unmaintained, trimming accurate content is low-value busywork that risks losing Peerloop-specific gotchas. Git preserves everything regardless.

**Consequences:** Content intact; only `reactjs.md` (actively misleading) was refreshed.
