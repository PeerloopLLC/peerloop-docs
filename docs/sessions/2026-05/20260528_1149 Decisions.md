## 2. Move asterisk OUT of FormField's <label> element
**Type:** Implementation
**Topics:** a11y, testing, astro

**Trigger:** 35 test failures from `getByLabelText(/^password$/i)` not matching "Password*" (asterisk inside label textContent).

**Options Considered:**
1. Loosen test regexes (write-only fix)
2. Move asterisk outside <label> as visual sibling ← Chosen
3. Use CSS ::after to render asterisk (would not affect DOM textContent but adds complexity)

**Decision:** Restructure FormField so the asterisk is a sibling of `<label>`, not a child. The label's accessible name now is exactly the label text. Tests revert to standard `getByLabelText(/^password$/)` patterns.

**Rationale:** Durable fix — every future form built with FormField gets clean accessible labels. The canonical "required" a11y signal is the `required` attribute on the inner `<input>` (which FormField forwards), not visual asterisks. No regression risk.

**Consequences:** All 35 test failures resolved without test-content changes (only 3 small drift edits in OnboardingProfile.test.tsx for unrelated token/text drift).
