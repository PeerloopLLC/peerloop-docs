# Conv Learnings - 2026-05-07

## 1. Sub-Agent Cold-Start Penalty for Large Context Skills
**Topics:** cc-workflow, timecard, sub-agent, model-testing

**Context:** Tested Sonnet and Haiku as sub-agents dispatched via Agent tool to run /r-timecard-day. Wall-clock times: Opus 15s (main context), Sonnet 60-345s (sub-agent), Haiku 3× failures (hallucinated permission asks).

**Learning:** Sub-agent re-reads skill SKILL.md from filesystem, re-derives context, and loses prompt-cache benefits of the main-context model. For skills with large pre-computed context (100+ KB), sub-agent overhead can be 4-20× wall-clock penalty. Block-summary adoption glide path means this is temporary — as blocks mature and adopt the `Block-summary:` field, the LLM step disappears entirely, making sub-agent dispatch unnecessary.

**Pattern:** When evaluating sub-agent suitability for a skill, measure wall-clock impact empirically. A 15s task becoming 60-300s under sub-agent dispatch is not worth the marginal benefit unless the task is genuinely blocked on foreground model capacity.

## 2. Permission Boundaries in Sub-Agent Dispatch
**Topics:** cc-workflow, sub-agent, permissions, file-write

**Context:** Attempted to write timecard outputs to Obsidian vault (`~/Obsidian Vaults/...`) from sub-agent context. Sonnet Write permission denied (real boundary). Haiku fabricated permission asks (hallucination).

**Learning:** Sub-agents have unfettered Write access to project-internal paths (`$CLAUDE_PROJECT_DIR/...`) but trigger denial-roulette for writes outside the project tree. Haiku under sub-agent context hallucinated permission asks rather than handling real denials, suggesting weaker models may not be suitable for permission-boundary detection. Switching the target to project-internal `.timecards/` folder eliminated the issue cleanly.

**Pattern:** When sub-agent dispatch requires file write, prefer project-internal targets over user-local paths. Architect the skill to accept an output-path parameter so the caller can route write targets appropriately.

## 3. Permission Hallucination Rate Differs by Model Tier
**Topics:** cc-workflow, sub-agent, model-testing

**Context:** Haiku sub-agent fabricated permission asks when attempting to write to vault path, rather than returning a real Write permission error like Sonnet did. The hallucinations were sufficiently detailed (asking for Bash permission) that they appeared legitimate, but repeatedly failed on retry.

**Learning:** Weaker models may not be suitable for permission-boundary detection or may exhibit higher hallucination rates under sandbox constraints. Treat hallucinated permission asks as a tier-specific failure mode when evaluating sub-agent dispatch with Haiku — don't conflate them with real Write boundary denials from a stronger model.

## 4. Timecard-Day Skill Scope Mismatches Sub-Agent Architecture
**Topics:** cc-workflow, timecard, skill-architecture

**Context:** /r-timecard-day carries large pre-computed context (100+ KB SKILL.md, JSON script output, config fields). Sub-agents re-read all of this from the filesystem, losing prompt-cache benefits of the main-context model.

**Learning:** The cold-start penalty (4-20× wall-clock) is not justified for a deterministic script-based task that the skill is designed to run in main context. Future optimization could split deterministic vs. LLM portions (only LLM synthesis varies; everything else is mechanical), making sub-agent dispatch viable for the LLM portion alone.
