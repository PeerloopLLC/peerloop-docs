# Conv Learnings - 2026-05-29

## 1. Adjacency-based git safety rules systematically miss the project's mandated `git -C` form
**Topics:** cc-workflow, dual-repo

**Context:** [GUARD-VERIFY] exercised `guard-dangerous-bash.sh`. The Conv 213 commit-message-strip used `\bgit[[:space:]]+commit\b`, which requires `git` adjacent to `commit`. The project MANDATES `git -C <dir> commit` for all git (CLAUDE.md; `feedback_git_dash_c_enforcement.md`), so the strip never ran on real commits — a message quoting `wrangler --remote` / `DROP TABLE` self-escalated. The same blind spot lived in the force-push rule (`\bgit[[:space:]]+push\b`) AND in the settings.json `deny` prefix `Bash(git push --force:*)` — neither can see `git -C <dir> push --force` (command starts with `git -C …`, path varies). Net: the safety rules covered the bare form the project never uses and missed the `-C` form it always uses.

**Learning:** When authoring ANY git-related guard regex or deny prefix, tolerate git global options (`-C <path>`, `-c k=v`, `--opt[=val]`, `-x` flags) between `git` and the verb. Prefix-based `deny` is structurally incapable of covering the `-C` form (path varies) — the PreToolUse hook is the correct layer for those.

**Pattern:**
Shared ERE fragment: `GITOPTS='([[:space:]]+(-C|-c)[[:space:]]+[^[:space:]]+|[[:space:]]+--[A-Za-z][A-Za-z-]*(=[^[:space:]]+)?|[[:space:]]+-[A-Za-z])*'` then `\bgit\b${GITOPTS}[[:space:]]+<sub>\b`. Stops at the first non-option word so it can't span a `;`/`&&` boundary.

## 2. The Conv 213 guard fix was a textbook heuristic-calibration miss
**Topics:** cc-workflow

**Context:** Conv 213 verified the option-C fix with "11 test cases" but only against bare `git commit` — never the canonical `git -C` form. `feedback_heuristic_calibration.md` already mandates running a new gate against the canonical case before commit.

**Learning:** The "canonical case" for a git-command guard in this project is the `git -C <dir>` form, not bare `git`. A gate that passes its tests but never fires on the form the project actually uses is mis-calibrated even if every listed test is green.
