# Conv Learnings - 2026-05-24

## 1. Probe the real data before deciding the design-to-code mapping
**Topics:** astro, d1, cc-workflow

**Context:** Four course tabs were built this conv, each with a different data verdict: CreatorTab (no backing → static), Reviews (clean `course_reviews` table → real via loader query), Modules (clean 1:1 curriculum → real), Feed (Stream-backed via existing API → real client island).

**Learning:** The right data strategy per tab is only knowable after probing the actual schema/seed/API — not from the Figma frame alone. Defaulting all tabs to one approach (all static, or all "needs schema") would have been wrong in 3 of 4 cases.

**Pattern:** For each design-to-code surface: (1) probe the frame, (2) read the loader/schema/existing-API, (3) classify each section real vs unbacked, (4) only then choose render strategy.

## 2. A "no SSR backing" conclusion can be wrong if an API already exists
**Topics:** stream, astro, cc-workflow

**Context:** I concluded the course feed needed a big Stream integration because `feed_activities` is metadata-only. The user knew there was already a working legacy feed.

**Learning:** Before declaring data "unreachable", grep the legacy app for an existing route/component/API serving the same data. The legacy `CourseFeed.tsx` already had `GET /api/feeds/course/[slug]`. Reusing it made the Matt feed real-data with only presentation code.

**Pattern:** When a feature exists in the legacy UI, find its data path first — the API is usually already built.

## 3. `--body-default-line-height: 1` makes wrapped body copy cramped
**Topics:** astro, testing

**Context:** Qualifications descriptions had touching lines.

**Learning:** The typography token `--body-default`/`--body-small` set line-height 1 (from Figma `lineHeight:100`), which is fine single-line but cramps any wrapped paragraph. `--body-medium*` correctly uses 1.5. This is the visible symptom behind [LH-VERIFY].

**Pattern:** Add `leading-normal` to wrapping body text locally; the global token fix needs an app-wide sweep ([LH-VERIFY]).
