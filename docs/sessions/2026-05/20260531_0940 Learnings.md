# Conv Learnings - 2026-05-31

## 1. `appearance: none` does not remove a forms-plugin select chevron
**Topics:** astro, testing, cc-workflow

**Context:** A "double caret" on the Matt `Select` primitive. Two guesses (native-arrow leak → `appearance-none` + `-webkit-appearance` reinforcement) both failed; `getComputedStyle().appearance` even read `none`.

**Learning:** `@tailwindcss/forms` (`@plugin '@tailwindcss/forms'` in `global.css`) paints EVERY `<select>` with a chevron `background-image` AND a border. The `Select` primitive adds its own caret SVG on top → two chevrons + a nested box. `appearance: none` governs the *native* control rendering, not a `background-image`. The DOM tell was `backgroundImage: url(data:image/svg+xml,…)`.

**Pattern:**
Any custom-styled `<select>` wrapper under the forms plugin must set `bg-none border-0` (not just `appearance-none`) to suppress the plugin chrome. The fix is app-wide — every Select under the plugin was affected; no audit done of visible count.

## 2. DOM-truth before shipping a visual fix
**Topics:** cc-workflow, testing

**Context:** Shipped two wrong caret fixes from code-reading + a stale screenshot before inspecting the live DOM.

**Learning:** Per `feedback_dom_truth_over_screenshots`, inspect the rendered DOM/computed style FIRST for precise visual bugs. The first browser inspection that mattered immediately surfaced the `background-image` chevron the code-reading lens had missed twice. The trigger should fire on the FIRST visual-bug report, not after failed guesses.

## 3. Member directory role = entity property, not viewer relationship
**Topics:** astro, project

**Context:** Deciding whether `/members` should adopt the `/courses`+`/communities` per-role dispatcher.

**Learning:** On /courses the role tabs mean "courses where I am X" (viewer relationship); on /members the roles mean "users who HAVE role X" (entity property), are multi-selectable, and the data is server-paginated. The dispatcher's defining concept has no referent — so the faithful port keeps the server-driven directory pattern and only conforms the Matt visual grammar.
