# Conv Decisions - 2026-05-31

## 1. /members keeps the directory pattern, conforms only the Matt visual grammar
**Type:** Implementation
**Topics:** astro, project

**Trigger:** User asked to conform /members to /courses+/communities "unless unique aspects we cannot easily accommodate."

**Options Considered:**
1. Force the 3-island per-role dispatcher onto /members
2. Keep the server-driven directory; conform only the Matt visual grammar (search/sort/count/card grid via Matt primitives), roles as multi-select filter pills ← Chosen

**Decision:** Build a single server-driven `MembersDirectory` island with multi-select role pills + sort + Load-More; reuse Matt primitives + card grid; no single-select role-tab bar.

**Rationale:** Entity-role semantics + multi-select + server pagination are structural mismatches with the dispatcher; forcing it would drop legacy behavior (a failed port).

**Consequences:** Role-tab bar + role-color reconciliation stay deferred to #6/#13.

## 2. Moderator source of truth → community_moderators (not can_moderate_courses)
**Type:** Implementation
**Topics:** d1, testing

**Trigger:** Moderator filter on /members returned nobody real.

**Options Considered:**
1. Grant `can_moderate_courses=1` to community moderators in seed
2. Change `/api/members` to derive moderator from `community_moderators` (filter + sort + badge) ← Chosen
3. Both

**Decision:** Query `community_moderators` (EXISTS, is_active=1) as the moderator signal; dropped the dead `can_moderate_courses` column from the query.

**Rationale:** `can_moderate_courses` is a capability flag the seed only grants to admins; real moderators live in `community_moderators`. Matches the moderation lens used elsewhere.

**Consequences:** Real moderators (Sarah, Marcus) now surface; +2 regression tests lock it in. Admin privacy (`privacy_public=0`) left unchanged per user (option C).
