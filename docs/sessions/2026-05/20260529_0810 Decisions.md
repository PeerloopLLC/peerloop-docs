## 2. MEMORY.md pointer display label = constant `[link]`; do NOT rename sub-files
**Type:** Strategic
**Topics:** cc-workflow, docs-infra

**Trigger:** ~9% of the cap was wasted on filename-echo display titles; user weighed renaming files.

**Options Considered:**
1. Rename files short (max savings) — but ~127 cross-refs incl. 37 frozen archival docs; risk of unpredictable names
2. Shorten display titles to short labels, keep filenames
3. Constant `[link]` label, keep filenames ← Chosen (user)
4. Leave as-is

**Decision:** Sweep every pointer's display label to the constant `[link]`, preserve filenames and targets. Never rename sub-files.

**Rationale:** Zero risk (tooling keys on the `](file.md)` target; filename still visible there), biggest single byte win (~9%), no cross-reference blast radius, no risk of unpredictable filenames. The scannable meaning lives in the `##` header + hook, not the label.

**Consequences:** 69 labels swept → MEMORY.md 80%→65% across the conv; convention codified in the skill (operation c) + `feedback_memory_index_load_bearing.md`.

## 3. SETTINGS-GUARD option-C: exclude git-commit message bodies from the danger scan
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** Conv-212 false positive — commit message documenting "wrangler --remote" escalated the commit.

**Options Considered:**
1. Whitelist specific commit shapes
2. Strip `-m`/`--message` body from a scan copy only for `git commit` commands ← Chosen
3. Disable the wrangler-remote rule

**Decision:** Strip the single-arg `-m "..."`/`-m '...'`/`--message=...` body from a scan copy when the command contains `git commit`; scan everything else and everything outside the quotes.

**Rationale:** Preserves all real detection (chained `&& wrangler --remote` survives outside the quotes) while ignoring inert message prose. Minimal, verified against 11 cases.

**Consequences:** Hook updated; #29 closed, #30 updated; memory note added.
