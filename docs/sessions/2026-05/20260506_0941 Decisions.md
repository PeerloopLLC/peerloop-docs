# Conv Decisions - 2026-05-06

## 1. useCallback-wrap (not inline) as the uniform Cat 3 fix pattern
**Type:** Implementation
**Topics:** react, hooks, refactor

**Trigger:** Phase 2 fix strategy choice — PLAN.md offered two options ("useCallback-wrap fn OR inline into effect"). 14 files; mix of single-callsite and multi-callsite fetch functions.

**Options Considered:**
1. Inline each fetch function into the useEffect that uses it (simpler when single-callsite, but breaks when fn is also called from event handlers).
2. useCallback-wrap each fetch function uniformly ← Chosen
3. Per-file judgment: inline if single-callsite, useCallback if multi-callsite (mixed result, harder to scan)

**Decision:** Apply useCallback-wrap uniformly to all 14 files.

**Rationale:** ~5 of 14 files use the fetch function from event handlers (`onClick={fetchX}`) or other effects in addition to the original useEffect — inlining would require also rebuilding the call site as a useCallback handler anyway. Uniform pattern is mechanical, predictable, easy to review. Per `feedback_no_simplest_fix.md` and CLAUDE.md "default durable", wrapping costs a small amount of boilerplate but doesn't lose any behavior — and survives future scope changes (adding a new callsite from a handler doesn't require revisiting).

**Consequences:** All 14 files now have stable function references via useCallback. 12 files gained `useCallback` import. Effect deps replaced with `[fetchX, ...other-vars-read-by-effect-body]`. Behavior identical for single-callsite cases; multi-callsite cases benefit from stable identity.
