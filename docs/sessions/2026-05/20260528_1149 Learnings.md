# Conv Learnings - 2026-05-28

## 1. Asterisk-outside-label preserves accessible name
**Topics:** a11y, primitives, testing, astro

**Context:** Conv 207 test fix for 35 form-test failures across LoginForm/SignupForm/OnboardingProfile.

**Learning:** When a `<label>` element wraps both the visible label text AND a styling decoration (e.g., a required `*`), testing-library's `getByLabelText(/^pattern$/)` queries fail because the label's textContent includes the decoration. Two valid mitigations: (a) loosen the regex in every test, (b) restructure the primitive so the decoration sits OUTSIDE `<label>` while remaining visually adjacent. Option (b) is durable — it fixes the issue for every future form built with the primitive, and matches a11y best practice (the canonical "required" signal is the `required` attribute on `<input>`, not visual asterisks).

**Pattern:** In `FormField.tsx`, wrap label+asterisk in a flex inner div as siblings: `<div><label>{label}</label>{required && <span aria-hidden>*</span>}</div>{labelAccessory}`. Label's accessible name = exact label text.

## 2. /w-codecheck trigger is a decision point, not a routine
**Topics:** cc-workflow, testing, deployment

**Context:** Conv 207 [issue 1] — manual `tsc + astro check + lint` at quiet-mode off skipped tests and missed 35 form-component test regressions for hours.

**Learning:** Running stock `tsc / astro check / lint` inline as a "lightweight pass" silently bypasses both the 5 Peerloop-specific bug-class checks AND the decision-point for tests/build/prov-sweep. The `/w-codecheck` trigger should be treated as the deliberate "I want to verify this is clean" moment — and at that moment, the right question is "what's the most thorough check this change deserves?" not "what's the fastest check that won't fail?"

**Pattern:** Memory `feedback_codecheck_moment_includes_tests_and_build.md` (Conv 207).

## 3. Page-provenance is its own marker class, distinct from component provenance
**Topics:** workflow, astro, docs-infra

**Context:** User asked what marker a Matt-style-retrofitted page should carry once @stand-in is removed.

**Learning:** Existing `@matt-source <nodeId>` convention applies only to leaf components/tokens with a 1:1 Figma frame mapping. Pages compose components and may have no source frame at all. They deserve their own marker class: `@matt-inspired` for "built with Matt tokens + primitives + design language, no source Figma frame". Per the new 3-marker convention adopted this conv, every non-legacy page carries exactly ONE of `@stand-in` / `@matt-source <nodeId>` / `@matt-inspired`. Unmarked = legacy.

**Pattern:** Doc-comment block at top of .astro/.tsx page:
```
/*
 * @matt-inspired — Provenance: MATT-INSPIRED (Conv N). NOT a 1:1 port from a
 * Matt-designed Figma frame (none exists for this surface), but built with Matt
 * tokens, primitives, and design language. Promote to @matt-source if/when a
 * Figma frame lands and is re-validated against this page.
 */
```

## 4. Pre-emptive infra without a consumer is liable to be deleted
**Topics:** workflow, astro

**Context:** `/teachers/[handle].astro` built Conv 193 [MATT-SUBNAV-STUBS] "to close the dangling deep-link in StudentTeacherAnchor.tsx". Conv 207 audit found StudentTeacherAnchor has zero production callers — the file existed solely to make a pre-emptive default href resolve.

**Learning:** Pre-emptive infrastructure (a route to satisfy a default href in an unused primitive) accumulates as dead code. The 404-honesty pattern (Conv 203 `project_route_404_honesty_standin.md`) is the better default: let dangling defaults 404 honestly, and the failure surfaces the absent consumer so it can be re-justified or removed.

## 5. Retrofit pass = primitive-extraction opportunity
**Topics:** astro, workflow

**Context:** Login retrofit revealed 3 form primitives; signup composed them with zero new ones; onboarding revealed 6 more (Select, SelectableCard, FormBanner, FormSection, SkeletonCard, ErrorState); /courses revealed 2 more (SearchInput, SegmentedPills) — 11 net new primitives in one conv, all reusable.

**Learning:** Translating a legacy page line-by-line into Matt tokens (the obvious move) ports the look without extending the design system. Scanning the legacy page FIRST for repeating/composable UI elements that should become primitives — BEFORE writing inline JSX — turns every retrofit into a design-system growth event. Each subsequent retrofit gets faster because the primitive surface grows.

**Pattern:** Memory `feedback_scan_for_primitive_candidates_on_retrofit.md` (Conv 207).
