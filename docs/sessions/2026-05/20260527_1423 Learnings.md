# Conv Learnings - 2026-05-27

## 1. Matt's design system has no catalog/browse card — only feed-embed rows
**Topics:** astro, matt-design

**Context:** "Retrofit the catalog grid" — needed a card for an all-courses grid.

**Learning:** Every Matt course card (CourseEmbedCard 480:10833, CourseAnchor 317:10557, CourseInFeed) is a horizontal anchor/embed ROW designed for feeds + review cards; none tiles into a browse grid. `matt-design-system.md` has no courses-index layout. A catalog/browse card is therefore "ours" to compose — same as a thumbnail card (Matt's cards use the course icon, not images).

## 2. Course image source is `thumbnail_url` via Matt's CourseHeader backdrop pattern
**Topics:** astro, matt-design

**Context:** User wanted images on cards.

**Learning:** `course.thumbnail_url` (on CourseListItem; live API serves picsum placeholders) is rendered by Matt's `CourseHeader` (517:8934) as a CSS `background-image` with a dark gradient scrim (`rgba(0,0,0,0.7→0.4)`) for white-text legibility. Reused that scrim pattern for the `overlay` card variant. Fallback color `#1f2937`.

## 3. Stretched-link card pattern keeps whole-card-click a11y-clean
**Topics:** astro, accessibility

**Context:** Dropped per-card CTA; whole card should be the link.

**Learning:** One real `<a>` on the title with `after:absolute after:inset-0` makes the entire card clickable while keeping exactly one anchor per card (accessible name = title). Verified via `elementFromPoint` at card center resolving to the anchor.

**Pattern:**
```
<a href={...} class="after:absolute after:inset-0">Title</a>
```
NOTE: only stays clean while a card has ONE action — a second per-card action (e.g. enrolled "Continue") can't nest under the stretch and forces explicit buttons back.
