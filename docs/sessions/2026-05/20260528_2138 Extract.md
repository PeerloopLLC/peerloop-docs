## Prompts & Actions

### Thread A — npm: should we upgrade npm? (continuation from pre-compaction)

**User:** "should we upgrade npm itself on this computer? Any downsides?"

**Claude:** Recommended **no** — npm 10.9.3 is current and `engines` allows it; the principal downside is cross-machine version skew. M4 + M4Pro stay aligned by construction (`.nvmrc` pins node 22.19.0, which bundles npm 10.9.3), so a standalone `npm i -g npm@11` on one machine would split the toolchain and risk author-side lockfile-normalization churn — the exact problem the `npm ci` switch eliminated. If upgrading is ever wanted, do it as a coordinated **node** bump (`.nvmrc` + `engines.node`) on both machines, not a per-machine global npm upgrade. Noted nvm-global npm upgrades are non-durable + can break npm's symlink.
