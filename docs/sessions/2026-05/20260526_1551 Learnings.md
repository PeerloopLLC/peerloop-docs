# Conv Learnings - 2026-05-26

## 1. Doc reconciliation strategy is a function of doc TYPE, not the stale string
**Topics:** docs-infra, cc-workflow

**Context:** All four docs had "stale `/matt/` refs" post-flip, but a uniform find-replace would have been wrong for two of them.

**Learning:** The right treatment follows from what kind of doc it is: (a) **architecture/design doc with timeless content** (url-routing.md) → keep canonical content, confine churn to a status banner (Option B); (b) **living how-to guide** (DEVELOPMENT-GUIDE.md) → straight path swaps, paths must point at real files; (c) **design spec written during the build era** (matt-design-system/) → banner to contextualize historical narrative, fix only concrete paths + leave "the work" prose; (d) **machine-checkable lookup** (matt-frames-ready-for-dev.md) → bulk `replace_all`, every cell is a real path.

**Pattern:** Before mass-editing stale refs, classify the doc: timeless-design / living-guide / historical-spec / machine-lookup — each gets a different ratio of churn-vs-contextualize.

## 2. Historical narrative needs path correction in a *guide* but only contextualization in a *spec*
**Topics:** docs-infra

**Context:** Both DEVELOPMENT-GUIDE.md (RTCONS, Conv 190) and matt-design-system/03 (Conv 188) had historical paragraphs with `src/pages/matt/...` paths.

**Learning:** A historical paragraph in a *living guide* still misleads a developer who reads "See: src/components/matt/X" today, so its paths must be updated (conv attribution kept). A historical paragraph in a *spec* whose whole point is recording the design conversation can keep its narrative and just be contextualized by a top-of-doc banner. Same content type, opposite treatment, driven by reader intent.

## 3. Path-reconciliation sweeps surface unrelated stale cross-refs
**Topics:** docs-infra

**Context:** While fixing matt-paths in 05-color-and-tokens.md, found its cross-ref to a DEVELOPMENT-GUIDE section heading that had been renamed in Conv 188 (broken independent of the flip). Also found 03 vs DEVGUIDE contradicting each other on whether the course `index.astro` exists (Conv 188 "untouched" vs Conv 190 "deleted").

**Learning:** A line-by-line stale-ref pass is a natural moment to catch *other* drift (renamed section anchors, doc-internal contradictions) because you are reading every ref in context. Fix the adjacent drift while there rather than deferring.
