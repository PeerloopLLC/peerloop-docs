# Conv Learnings - 2026-05-28

## 1. Reuse the tab-family idiom rather than inventing routing
**Topics:** astro, cc-workflow

**Context:** Retrofitting `/profile` into a multi-destination page like `/course/[slug]`.

**Learning:** The codebase's "tab family" idiom — a `[...tab].astro` catch-all + a `_`-prefixed config module (`buildXTabs()`) + `<SubNav currentPath>` + a `tab` switch — is reusable verbatim. Reusing it gave `/profile` the exact same active-matching (most-specific-href-wins), breadcrumb shape, and unknown-segment redirect with zero novel routing decisions.

**Pattern:** New tabbed page = copy `course/[slug]/{[...tab].astro,_course-tabs.ts}` structure; only the tab list + per-tab bodies differ.

## 2. Middleware EXACT→PREFIX is the load-bearing change for sub-routes
**Topics:** auth, astro, testing

**Context:** `/profile` was in `PROTECTED_EXACT`; the retrofit added `/profile/<tab>` sub-routes.

**Learning:** Leaving `/profile` in `PROTECTED_EXACT` would have left every `/profile/edit|payments|security` **publicly reachable** — a string-route hole `tsc`/`astro check`/`build` cannot see. Only a middleware unit test catches it. Moved it to `PROTECTED_PREFIXES` and added 4 sub-route assertions.

**Pattern:** Any new `/x/<sub>` family under an existing exact-protected `/x` must move `/x` to the prefix list AND add a sub-route protection test.

## 3. Verify a harvested icon actually resolves in the glob it's dropped into
**Topics:** matt, cc-workflow

**Context:** Added `lock.svg` for the Security tab; MEMORY.md cited the registry path as `src/components/matt/icons/`.

**Learning:** MEMORY.md's registry path was stale — the real path is `src/components/icons/MattIcon.tsx` with `import.meta.glob('./svg/*.svg')`. A wrong-dir drop would have silently shown a dashed placeholder (no build error). Confirmed `lock.svg` sits in the scanned dir AND browser-verified it renders with a `<path>` (not dashed).

**Pattern:** After harvesting an icon: confirm the glob path, then DOM-verify the rendered `<svg>` has real geometry, not the placeholder.
