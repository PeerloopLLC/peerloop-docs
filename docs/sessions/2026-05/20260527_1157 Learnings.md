# Conv Learnings - 2026-05-27

## 1. history.replaceState fires no hashchange — sibling islands need an explicit event
**Topics:** astro, frontend

**Context:** CoursesRoleTabs wrote the active tab to the URL hash via `replaceState`; CoursesCatalog listened for `hashchange` and never updated.

**Learning:** `history.replaceState`/`pushState` do NOT emit `hashchange` or `popstate`. Sibling islands sharing "hash state" must communicate via an explicit custom event (`window.dispatchEvent(new CustomEvent(...))`).

**Pattern:** Decoupled island↔island comms — publisher dispatches `courses:tabchange`/`courses:filterchange`; subscribers `addEventListener`. Keeps slots independent.

## 2. Current-user singleton init lived only in legacy navbars
**Topics:** auth, astro

**Context:** Role tabs rendered nothing on Matt `/courses`; `useCurrentUser()` returned null.

**Learning:** `initializeCurrentUser()` (fetch `/api/me/full` → populate `window.__peerloop.currentUser`) was called only by legacy AppNavbar/AdminNavbar. Matt AppLayout uses the Sidebar, so no Matt page initialized the singleton. Fixed shell-wide with a headless `CurrentUserInit` island in AppLayout.

**Pattern:** Shell-level headless init island (`client:load`, renders null) for cross-page client state.

## 3. /courses primary topic = authoritative column, not tag-derived
**Topics:** d1, astro

**Context:** Topic filter returned 0 (loader hardcoded `primary_topic_id: null`); a tag-derived subquery (legacy) gave wrong counts.

**Learning:** `courses.primary_topic_id` is a denormalized creator choice (schema 0001 line 346-347, re-added Conv 108) — "not derivable from tags". Use the column, not a `course_tags`→`tags`→`topics` subquery.

---

## 4. React controlled inputs ignore raw .value sets (browser-test gotcha)
**Topics:** testing, frontend

**Context:** A search test set `input.value` + `dispatch('input')` and falsely showed no filtering.

**Learning:** React overrides the input value setter; to trigger `onChange` from automation use `Object.getOwnPropertyDescriptor(HTMLInputElement.prototype, 'value').set`.
