# Conv Learnings - 2026-05-29

## 1. The `{...rest}` spread on a wrapping primitive does NOT land where you assume
**Topics:** astro, testing

**Context:** Subagents stamped `PasswordInput`/`SearchInput` by putting `data-prov-name` on the `<Input>` JSX element, assuming the spread would carry it to Input's root div and override.

**Learning:** `Input` spreads `{...rest}` onto its inner `<input>`, NOT its root `<div>`. So props passed to `<Input>` that aren't destructured land on the child input — the DOM **outermost** element kept `data-prov-name="Input"`. A composing primitive can't passively inherit the root identity; the wrapped primitive must explicitly forward the override to its own root.

**Pattern:**
Made `Input` destructure `data-prov`/`data-prov-name` out of rest and spread them onto its root div AFTER its own literal default (`<div data-prov="matt-inspired" data-prov-name="Input" {...provOverride}>`) — later spread wins at the DOM root, literal default stays grep-able for the static sweep.

## 2. Build the gate before the bulk edit — it becomes the checklist + the subagent safety net
**Topics:** testing

**Context:** 59 mechanical stamp edits delegated to 5 parallel subagents.

**Learning:** Wiring the §12c conformity assertions into `prov-sweep` FIRST (it reported 153 issues pre-stamp) gave a deterministic verifier that caught every UNSTAMPED/UNTRACKED/wrong-node before any of the subagent edits could silently drift. The static gate checks stamp PRESENCE; tsc/astro-check/build cover JSX correctness; the DOM page-report covers "did it reach the DOM + is it the outermost element." The three layers together make parallel mechanical delegation safe.

---

## 3. A static source gate and a DOM report answer different questions
**Topics:** testing

**Context:** §12c specifies both a sweep (source) and a page report (DOM).

**Learning:** The source sweep can prove registry⟺marker⟺stamp agree, but it CANNOT see composed children (a primitive stamps its own root; `CourseAnchor.tsx` contains `<EntityPill/>`, not EntityPill's stamp — that only appears at runtime). The page marker (§11) also doesn't propagate to embedded components. So the DOM report is the only thing that can find "a legacy island inside a @matt-inspired page" — the exact gap that motivated the whole initiative. `/dev/primitives` (81 sourced + 152 inspired, 0 uncovered) is a great all-primitives DOM smoke test.
