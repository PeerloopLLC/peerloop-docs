## 1. `*/` inside a block comment detonates the whole file
**Topics:** astro, testing

**Context:** A docstring line `legacy secondary-*/primary-* palette` inside a `/* */` block comment in `FeedDirectoryCard.tsx`.

**Learning:** The `*/` in Tailwind-class shorthand prematurely terminates the comment, mis-tokenizing everything after it. One mangled comment produced 137 astro-check errors plus tsc/lint/build failures — a terrifying count from a one-line cause. Sibling `MemberCard` dodged it by writing `secondary-*` without the slash.

**Pattern:** Never write `*/` (or `-*/...`) inside a `/* */` doc-comment; use `secondary/primary` or space the glob.

## 2. Feeds maps to a third island architecture
**Topics:** astro

**Context:** `/communities` uses SSR-fetch islands; `/members` is an API-driven single island.

**Learning:** Feeds are entirely client-side from `useCurrentUser().getFeeds()` (no SSR) plus `/api/feeds/discover`, so `feeds.astro` is a thin page mounting client islands, and `FeedsDirectory` is ONE self-contained island (role-tab bar is a controlled child) — no cross-island event bus like `/communities`' three-island shell.

## 3. DOM-truth beats narration (again)
**Topics:** cc-workflow, testing

**Context:** A `javascript_tool` browser probe threw on top-level `await`; the failed read was narrated as "discovery grid empty".

**Learning:** A clean re-probe showed live discovery cards — contradicting the narration. Trusting DOM ground-truth over the assumption caught a false conclusion. Also: `javascript_tool` runs in a non-async scope, so top-level `await` throws.

## 4. Verify tool side-effects under render garble
**Topics:** cc-workflow

**Context:** Heavy terminal-render garble dropped/duplicated many tool results; the pre-computed `/r-end` repo status was even stale on one file.

**Learning:** When outputs are unreliable, re-establish ground truth with a single clean `git status`/`grep` before any destructive step (commit). This caught both that the registry edit HAD landed (status was stale) and that stray `.bak` files needed removing before `git add .`.
