# Conv Learnings - 2026-05-24

## 1. Figma MCP asset URLs return SVG for vector sources
**Topics:** figma, design-system, mcp

**Context:** Probing Brand component (Logo + LogoMark) via `get_design_context` returned `<img src="https://www.figma.com/api/mcp/asset/...">` markup. Initially assumed PNG raster (would have been blocking for a brand primitive that needs to be scalable). curl'd the URL to verify format — got raw SVG markup with `fill="var(--fill-0, #hex)"` Variable-fallback pattern.

**Learning:** Figma's MCP asset endpoint preserves the source format. Vector layers come back as SVG (not raster PNG); raster sources come back as PNG. This makes the `<img>` → `<svg dangerouslySetInnerHTML>` conversion natural — same pattern as MattIcon's Vite `?raw` glob (with optional outer-`<svg>` strip + viewBox-per-variant if multiple sizes). `get_screenshot` continues to return PNG regardless.

**Pattern:** `curl -sSL -o file.svg "https://www.figma.com/api/mcp/asset/<uuid>"` then `perl -pi -e 's/fill="var\(--fill-0, #[0-9A-Fa-f]+\)"/fill="currentColor"/g'` to normalize for Tailwind theming.

## 2. Conv 178 reconnaissance over-engineered with `entity` prop on non-entity primitives
**Topics:** design-system, conv-178-drift

**Context:** C178-REVAL probed Module + ToDoItem and found both had been built with 4-color `entity` props (course/student/creator/default) when Matt's Figma only has 2 variants each (Default/Current and Default/Checked respectively). The "active" state in both uses a single hardcoded color (`--primary/light` for Module, `--pastel-blue` for ToDoItem — same #F1F9FF underlying value), NOT entity-cascade.

**Learning:** When a component has variants in Matt's Figma, mirror those variant labels exactly via `property1` enum — don't extrapolate to entity-collection unless Matt explicitly drew 4 entity colors for that primitive. Over-engineering with `entity` props adds API surface that has to be supported but isn't in the source design. Strict-B mirror is the safer default.

**Pattern:** `property1?: 'Default' | 'Current'` matching Matt's exact variant labels; reserve `entity` cascade for primitives Matt drew as multi-entity (UserIcon, EntityPill, EntityLink, Button which has 6 variants including 3 entity ones).

## 3. Pre-mirrored asset SVGs vs CSS-transform mirroring
**Topics:** figma, design-system

**Context:** ChatBubble has Default tail bottom-left and Us tail bottom-right. The Figma export rendered Default as `rotate-180 -scale-y-100` CSS transform; both variants returned different asset URLs with different MD5s.

**Learning:** Matt drew each variant as a separate, pre-mirrored SVG file rather than relying on CSS transforms. Trust the export: download both SVG files and use them as-is. Trying to "optimize" to one SVG + CSS transform discards Matt's intent and adds runtime complexity for zero size benefit (each SVG is ~3KB).

## 4. SectionTitle name collision — Matt's Figma vs our code
**Topics:** design-system, naming, conv-178-drift

**Context:** Re-probing Matt's `Section Title` Figma frame (722:14801) revealed it's a 1280×96px orange/green/red **dev-status banner** (WIP / Dev Ready / Archived) using TT Norms Pro Mono font — a Figma-internal document-organization marker, NOT a product component. Our code's `SectionTitle.astro` is a generic content heading using Inter that wraps an h-tag.

**Learning:** Same name, totally different purposes. Matt's SectionTitle is Figma-design-system meta, not a product primitive. Our code's usage as a generic heading is legitimate and unrelated. Document the collision in matt-design-system.md but don't try to "mirror" Matt's banner — it would never go in the actual product.

**Pattern:** Before assuming a Figma frame maps to a code component, check if the frame is a product element or a design-system meta element (status badges, version stamps, designer notes). Visual-canvas furniture isn't always a primitive.
