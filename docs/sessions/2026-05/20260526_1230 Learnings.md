# Conv Learnings - 2026-05-26

## 1. Cross-file plans hide coherence bugs that single-file review can't catch
**Topics:** cc-workflow, docs-infra

**Context:** MATT-CUTOVER design was documented across matt-provenance.md (Conv 195), url-routing.md §8 (Conv 193), and matt-pre-plan.md. Each file was locally coherent, but the legacy-prefix name diverged (`/old` vs `/fraser`) between them.

**Learning:** Comprehensiveness bugs hide *within* a file; coherence bugs hide *between* files. A plan spread across many files needs an explicit cross-file pass — the user's "review as it touches many files" instinct was exactly right.

**Pattern:** When a decision is re-made in a later conv, grep all docs for the prior naming and reconcile, don't just write the new doc.

## 2. Verification steps must be able to fail loudly
**Topics:** cc-workflow

**Context:** Plan's verification was "grep for `fraser` → expect zero in as-designed/." It came back non-zero, surfacing two `matt-pre-plan.md` refs the 3 Explore agents had all missed.

**Learning:** A verification that can only pass is theatre. This grep was designed to fail if the contradiction lingered anywhere — so it found residue the agents didn't. Don't trust the plan's own claim about which file is the sole offender.

## 3. Domain and authorship are orthogonal; only authorship causes collisions
**Topics:** matt-design, architecture

**Context:** Provenance discussion repeatedly conflated "is this design-system or legacy?" with "did Matt draw this or Claude?"

**Learning:** A folder/route prefix (`/old`, `matt/`) cheaply encodes the *domain* axis but cannot encode *authorship* — authorship is mixed *within* the design-system folder by definition (Matt won't draw everything). That's the irreducible reason a marker (`@matt-source`) must exist. Mark the minority (Matt's), leave the majority (ours) unmarked.

---

## 4. Mark provenance before a namespace-dissolving move
**Topics:** matt-design, cc-workflow

**Context:** ROUTE-FLIP dissolves the `matt/` folder.

**Learning:** The `matt/` folder is a free audit aid (everything under it is in-domain) — marking while it exists reduces the question to "which of these did Matt draw?" After the flip, markers are the only way to recover provenance. Sequence: [PROV] before [ROUTE-FLIP].
