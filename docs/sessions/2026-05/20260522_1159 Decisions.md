# Conv Decisions - 2026-05-22

## 1. Include `--spacing-N` in Tailwind bridge (accept global utility-scale break)
**Type:** Strategic
**Topics:** matt-design-system, astro

**Trigger:** Pre-plan §5 bridge file template included spacing tokens in @theme. Before authoring, audit revealed 572 `p-4` callsites in existing app + many more `m-N`/`gap-N`/etc. Adding `--spacing-N: 0.25rem` would override `p-4` from 1rem (default) to 0.25rem (Matt's 4px), violating Decision 4=B "coexist" intent.

**Options Considered:**
1. Omit spacing from bridge — `/matt/*` consumes via `var(--space-N)` or `p-[var(--space-N)]`. Existing utilities unchanged.
2. Include per pre-plan §5 ← Chosen — global `p-4` shrinks 4×; 572 callsites visually break; tightest semantics.
3. Matt-namespaced `--spacing-m4` prefix — new `p-m4`/`m-m4` utilities, zero collision, ugly names.

**Decision:** Include `--spacing-N` per pre-plan §5. Accept global utility scale break across non-`/matt/*` pages.

**Rationale:** User explicitly chose B. Treats this branch (`jfg-dev-13-matt`) as the future-default identity destination; the existing-app utility values are transitional anyway pending the flip block.

**Consequences:** All `p-N`/`m-N`/`gap-N`/`px-N`/`py-N` callsites on `jfg-dev-13-matt` where N ∈ {4,8,12,16,20,24,32,40,48,64} now resolve to Matt's pixel-named values. Effective scale becomes mixed (p-1=4 multiplicative; p-4=4 overridden; p-5=20 multiplicative; p-8=8 overridden; etc.). `jfg-dev-12` and earlier branches unaffected — change is scoped to `jfg-dev-13-matt` until flip.
