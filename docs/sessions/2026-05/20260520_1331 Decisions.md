## 3. Defer [AAP] Astro absolute-path fix (vs middleware HTML-rewrite vs patch-package)
**Type:** Implementation
**Topics:** astro, dual-repo

**Trigger:** Surfaced 🟠 finding that Astro dev emits absolute filesystem paths into `<script src>`. User asked "we should fix that".

**Options Considered:**
1. A — defer + file diagnosis as a follow-up task + verify on each Astro upgrade ← **Chosen**
2. B — dev middleware that rewrites every HTML response (read+rewrite+rebuild cost per request)
3. C — `patch-package` monkey-patch to `node_modules/astro/dist/vite-plugin-astro/compile.js:50`

**Decision:** Defer per user. Task description makes clear it's waiting on an Astro upstream fix post-6.3.6.

**Rationale:** Functionally a no-op (Vite serves 200 either way). The two workarounds add maintenance burden for a purely cosmetic issue. Latest stable Astro (6.3.6) has the identical buggy line, so checking each release for the fix is cheap; meanwhile the diagnosis is preserved so future-CC can act on it.

**Consequences:** New task [AAP] with full root-cause diagnosis (file:line, version proof, verification probe). No code changes this conv.
