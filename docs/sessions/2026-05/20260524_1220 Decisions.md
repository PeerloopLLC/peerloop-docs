## 3. Side-effect Material-icon discovery via drift-check workflow
**Type:** Implementation
**Topics:** figma, matt-design

**Trigger:** User asked how we'd handle finding more Material icons over time.

**Options Considered:**
1. Separate umbrella task for "incremental Material harvest" (the original `[CMP-EXT-ICN]`)
2. Discovery as side-effect of any `get_design_context` probe ← Chosen
3. Periodic manual sweep

**Decision:** Every deep frame probe scans `<img data-name="...">` instances and auto-creates `[<ICON>-ICN]` TaskCreate for unharvested icons. Codified into `[MFRD-LOOKUP]`'s workflow section 5.

**Rationale:** Material icons enter the system one frame at a time; discovery should follow the same trigger as drift detection (i.e., whenever we deep-probe a frame). Umbrella tasks vague and easy to forget; side-effect mechanism is automatic.

**Consequences:** Deleted `[CMP-EXT-ICN]` (#10). Five specific icon tasks remain (#5, #6, #25, #26, #27) until they're harvested.
