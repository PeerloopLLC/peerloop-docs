# Conv Learnings - 2026-05-22

## 1. Astro Fragment-forwarding suppresses child slot fallback
**Topics:** astro, matt-design

**Context:** Conv 175 [MSH-VIZ] hit an empty-pill bug at tablet portrait — HeaderBar's brand fallback never rendered because AppLayout forwarded the slot via `<Fragment slot="header-center"><slot name="header-bar-mobile-center" /></Fragment>`.

**Learning:** Astro counts a Fragment-wrapped forwarded slot as "filled" even when the forwarded inner slot is empty. The child component's `<slot name="x">FALLBACK</slot>` fallback is suppressed. `Astro.slots.has + &&` short-circuit did NOT restore the fallback in our test — root cause unconfirmed, but the dual-layer fallback pattern is too fragile to rely on regardless.

**Pattern:** Put defaults at the layout/consumer level via ternary inside *unconditional* Fragments:
```astro
<HeaderBar>
  <Fragment slot="header-center">
    {Astro.slots.has('header-bar-mobile-center')
      ? <slot name="header-bar-mobile-center" />
      : <span data-matt="brand-mark">∞ PeerLoop</span>}
  </Fragment>
</HeaderBar>
```
Tracked in `memory/reference_astro_slot_forwarding.md`.

## 2. Visual-fidelity validation must use Figma references early, not late
**Topics:** matt-design, workflow

**Context:** Built Phase 3 page + Phase 4 primitives from pre-plan's structural prose (sectioned About body, prose lists, generic spacing) without ever opening Matt's `Course.svg`. User correctly pointed out the visual was far off. Pre-plan §9 specifically calls this gate out: "If no, retreat to Phase 1/2 and tune tokens before continuing."

**Learning:** Structural completeness is not a visual-fidelity proxy. When the pre-plan §9 says "visual diff vs Figma," that has to mean opening the SVG before/during the build, not after. The mid-conv pivot to symlink SVGs into `public/_matt-ref/` worked for diff-driven iteration but should have happened in Step 1 of the page build, not Step 7.

**Pattern:** For matt/* page builds going forward: symlink the relevant Figma export(s) into a dev-only `public/_matt-ref/` path FIRST, then build. Visual-diff iteration becomes the loop, not the afterthought.

## 3. Tailwind 4 arbitrary-value `min-h-[NN]px` may interact poorly with `--spacing-*` overrides
**Topics:** astro, tailwind, matt-design

**Context:** Set `min-h-[480px]` on the CourseHeader element. Class appeared in rendered HTML but Tailwind didn't generate the CSS rule for it. Workaround: inline `style="min-height: 480px"` in the bgStyle string.

**Learning:** The Conv 174 spacing override (`--spacing-4`/`-8`/`-12`/... in `tokens-tailwind-bridge.css @theme`) may affect Tailwind 4's arbitrary-value parsing for height-axis utilities. Not confirmed via root cause; logged as `[TSV]` follow-up. For now, when a min-height arbitrary value won't take effect, prefer inline style or use Tailwind's fixed scale (`min-h-48` etc.).
