# Conv Learnings - 2026-05-24

## 1. Tailwind 4 `@theme` flattens cascade-driven tokens; use `@theme inline`
**Topics:** astro, matt-design-system, css

**Context:** A role-color "corner dot" reading `var(--Entity-Primary)` rendered gray; live DOM showed the var was correct at the element but the computed color was the `:root` default.

**Learning:** In Tailwind 4, `@theme { --color-x: var(--Y) }` resolves the inner `var(--Y)` once at `:root`, so utilities (`bg-x` / `text-x`) ignore the use-site cascade. **Static design tokens belong in plain `@theme`; cascade-driven tokens that change per-subtree (set by `.entity-*` classes) MUST be in `@theme inline`**, which emits the variable reference directly so it re-evaluates at the element. This bug had silently broken EntityPill / EntityLink / UserIcon-initials role colors app-wide (all rendered the gray default).

**Pattern:**
```css
@theme inline {
  --color-entity-primary: var(--Entity-Primary);
  --color-entity-background: var(--Entity-Background);
}
```
Keep static role colors in the plain `@theme` block.

## 2. "Most-specific-match-wins" for section-index nav active-state
**Topics:** astro, matt-design-system

**Context:** SubNav active-matching used `path === href || path.startsWith(href + '/')`. A base/index item (About) whose href is a prefix of all sibling tab hrefs would stay Selected on every child route.

**Learning:** For a nav where one item's href is a prefix of its siblings, compute the single longest matching href across all items+subItems and mark only that one active. Exact matches win naturally (an exact href equals the full path). One rule handles base + nested routes without per-item flags.

## 3. The Ready-for-Dev lookup's "expected primitives" are inferences, not probes
**Topics:** matt-design-system, figma

**Context:** Lookup said Modules=ModuleAnchor, Reviews=ReviewAnchor, etc. Probing the actual frames showed the tabs are bespoke page-level card composites that *reuse* leaf primitives; the named "…Anchor" primitives are social-post embeds (wrong layer).

**Learning:** Designer-frame tabs are happy-path instance snapshots of one demo course in a specific state (Resources = empty state; Modules = 2-session demo). Faithful building = reproduce the drawn state; populated/other states are Phase-6 extrapolation. Always probe the frame before building; "expected to suffice" notes are guesses.
