## 3. prov-sweep.ts extended with page-level classifier, line-anchored regex
**Type:** Implementation
**Topics:** matt-provenance, prov-sweep, tooling

**Trigger:** [PROV-SWEEP-MI] — teach the sweep about @matt-inspired and the 3-marker page convention.

**Options Considered:**
1. Re-use existing component `MARKER_RE` — failed on prose pollution
2. Line-anchored regex with allowed comment leaders (` * `, `// `, `/* `, indent) ← Chosen
3. Restrict scan to first comment block — brittle on .astro frontmatter shape

**Decision:** Three new regexes (`PAGE_SRC_RE`, `PAGE_INSPIRED_RE`, `PAGE_STANDIN_RE`), each line-anchored with `m` flag and `^[\s/*]*` prefix accepting common comment leaders.

**Rationale:** Pages legitimately cite child markers in prose; component-side `MARKER_RE` has no such issue. Per `feedback_heuristic_calibration.md`, re-derive the accept-condition for the new context. Verified against all 8 canonical pages before commit.

**Consequences:** Sweep reports `@matt-source` / `@matt-inspired` / `@stand-in` page counts + flags unmarked pages as drift. Stand-in backlog visibly listed each run.
