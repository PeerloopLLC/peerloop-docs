### New Patterns

- **Hybrid plan-file mode** — PLAN.md ACTIVE-table rows for migrated blocks end with `→ [plan/<slug>/README.md](plan/<slug>/README.md)`. The `/r-start` skill Step 8 now crawls `plan/*/README.md` (head -40 per file) and includes the output in pre-computed context. Non-migrated blocks read inline from PLAN.md as today.
- **Per-block file structure (matt-family precedent)** — `plan/<family>/README.md` (overview + phase table + decisions + conv timeline) + `plan/<family>/<phase-or-sub-block>.md` (current state + per-conv history + decisions + open items). Family-level grouping (`plan/matt/`) when a block spawns sibling blocks (MATT-DESIGN-PUSH + MATT-CUTOVER + STANDIN-MATT).
- **Migration moment as the cleanup moment** — long PLAN.md status cells (12KB narratives) only get rewritten when their block migrates to `plan/<slug>/`. Don't sweep the broader pattern in one conv; let migration trigger the rewrite per block.
