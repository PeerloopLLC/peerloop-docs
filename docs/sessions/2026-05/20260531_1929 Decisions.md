# Conv Decisions - 2026-05-31

## 1. Matt's §5 role palette is canonical for the role-tab bar
**Type:** Implementation
**Topics:** matt

**Trigger:** Reconciling 3 discover role-tab islands onto the shared RoleTabBar surfaced a palette conflict (islands teacher=green/moderator=amber + uniform text-primary active; primitive teacher=blue/moderator=neutral + role-colored active).

**Options Considered:**
1. Adopt RoleTabBar's Matt-§5 styling as canonical (role-colored active, teacher→blue, moderator→neutral) ← Chosen
2. Keep islands' look (uniform active + dot, teacher→green, moderator→amber); conform primitive down
3. Render both, pick visually

**Decision:** Adopt the primitive's Matt-§5 palette; all 3 destinations converge on it; active tab renders in its role's own color.

**Rationale:** It's our documented §5 spec and the whole point of having the shared primitive; user directive "favour Matt's colours."

**Consequences:** Visible restyle of /courses, /communities, /feeds role tabs (browser-confirmed). Legacy `ROLE_COLORS` green-teacher/amber-moderator retired for these tabs.

## 2. RoleTabBar is the single canonical role-tab strip; per-destination components become stateful adapters
**Type:** Implementation
**Topics:** matt

**Trigger:** Three near-identical tab-strip implementations + one shared primitive.

**Options Considered:**
1. RoleTabBar = presentational primitive; islands keep state, delegate rendering; delete the pure-presentational FeedsRoleTabs ← Chosen
2. Keep all 3 as thin wrappers

**Decision:** Generalize RoleTabBar (id decoupled from role; null=neutral All). CoursesRoleTabs + CommunitiesRoleTabs keep state/event/role logic and delegate rendering; FeedsRoleTabs deleted, FeedsDirectory renders RoleTabBar directly. Deregister the 3 from the prov registry (now adapters, not primitives); RoleTabBar is the registered primitive.

**Rationale:** Single source of truth for tab markup + palette; honors the §12 primitive-definition rules (a component that just composes a primitive isn't itself a primitive).

**Consequences:** 3 registry entries removed, RoleTabBar note updated; ~222 deletions / 120 insertions net. Follow-up [RTB-HOOK] tracks consolidating the still-duplicated visible-tab derivation logic.
