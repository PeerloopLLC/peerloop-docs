## 1. Apply isHydrated flag pattern from AppNavbar to fix BR-NAVBAR-HYDRATE
**Type:** Implementation
**Topics:** auth, frontend

**Trigger:** Hydration mismatch root-caused at AdminNavbar.tsx:90; three viable fix shapes.

**Options Considered:**
1. Mirror AppNavbar's `isHydrated` flag pattern — `useState(null)` + useEffect + render guard ← Chosen
2. Always render the avatar slot (blank when no admin) — eliminates conditional but flickers an empty row pre-hydration
3. `suppressHydrationWarning` — masks the warning but mismatch still happens; React still regenerates the tree

**Decision:** Option 1 — mirror AppNavbar's `isHydrated` pattern.

**Rationale:** Already the established codebase convention (AppNavbar.tsx:144-158). Minimum surface area (2 small edits to one file). Zero UX flicker because the chip simply isn't rendered until after hydration. Most durable per CLAUDE.md §Solution Quality.

**Consequences:** AdminNavbar's admin chip now appears milliseconds-later than before (post-hydration), matching AppNavbar's behavior elsewhere. Two SSR-safe navbars in the same codebase now share the same pattern — easier to remember the convention next time a similar component is written.
