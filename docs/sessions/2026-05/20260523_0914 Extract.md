### More web research

**User:** "First , we need to do web research as this CAN'T be our problem alone"

**Claude:** 4 parallel WebSearches with the new symptom signature. Found the cold-start pattern is **industry-wide and well-documented**:
- remix-run/remix #10156 — "Vite dev server crashes Remix/React when reloading after finding new dependencies to optimize"
- TanStack/router #4264 — "Cannot Read Properties of null After Vite Dep Optimization"
- storybookjs/storybook #32049 — "Vitest tests fail on first run"
- vitejs/vite #17979 + #17986 — exact error string match

Used gh CLI to fetch issue comments (WebFetch couldn't pull comment sections). Found the canonical fix from @luisdlopez on Remix #10156:
```js
optimizeDeps: {
  entries: ["src/**/*.tsx", "src/**/*.ts"],
}
```

Forces Vite to scan all source at startup → nothing discovered mid-session → no reload → no crash.
