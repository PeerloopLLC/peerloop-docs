# Conv Decisions - 2026-05-25

## 1. Consolidate the two course-page routes into one catch-all
**Type:** Implementation
**Topics:** astro

**Trigger:** Duplicated `courseTabs` in `index.astro` + `[...tab].astro` caused the SubNav-icon drift; user asked why two routes exist and approved consolidation.

**Options Considered:**
1. Keep both files, share only the data via `_course-tabs.ts`
2. Fold About into `[...tab].astro` (empty segment → `'about'` view), delete `index.astro` ← Chosen

**Decision:** Single catch-all owns the whole family; empty segment renders About instead of redirecting.

**Rationale:** Removes the duplicated shell (loader, CourseHeader, SubNav wiring) — the whole class of "edited one route, forgot the other" drift — and the redirect-loop lock-in.

**Consequences:** `index.astro` deleted; `route-map.generated.ts` regenerated; conditional title/breadcrumb added. Reverses the Conv 188 [MATT-EXEC-PG2] "Option A — shell duplicated, index.astro untouched" structure decision.

## 2. Scope B — rewrite Sidebar AND shell (not sidebar only)
**Type:** Implementation
**Topics:** matt-design-system, astro

**Trigger:** Sidebar didn't match Matt's Layout Desktop; the "blue background on selected" only works against a grey page.

**Options Considered:**
1. Sidebar internals only (active pill would be white-on-white, subtle)
2. Sidebar + shell — adopt `#f8fafc` page bg + white rounded content card ← Chosen

**Decision:** Adopt the full shell so the active pill pops as designed.

**Rationale:** The pill contrast is enabled by the grey page; sidebar-only would not match.

**Consequences:** `AppLayout` body bg + content-card treatment changed for every `/matt/*` page; sidebar made transparent. Logo dropped Medium→Small; MainNav gap 16→24; collapse `«` double-chevron harvested as `chevrons-left.svg` (43rd Matt icon).

## 3. Role-display logic + hierarchy for the Profile row
**Type:** Implementation
**Topics:** auth

**Trigger:** User spec — users hold multiple roles; need a compact reusable label.

**Options Considered:**
1. Single derived role (is_admin?Admin:…:Student)
2. Multi-role `describeRoles()`: 1→role, 2→"A, B" (higher first), 3+→"A + N more"; Visitor when not logged in ← Chosen

**Decision:** New `src/lib/roles.ts` (`userRoles` + `describeRoles`) on hierarchy Admin>Creator>Teacher>Moderator>Student (sourced from `UserProfileHeader.tsx`; Student base-only).

**Rationale:** Reusable, matches the established badge convention, handles multi-role users.

**Consequences:** `AppLayout` selects all 5 capability flags; Sidebar shows avatar+name+role (logged in) or filled-circle+"Visitor" (links `/login`).
