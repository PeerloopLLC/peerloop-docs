# Conv Learnings - 2026-05-26

## 1. Post-flip namespace dissolution removes the domain delimiter for provenance detection
**Topics:** astro, cc-workflow

**Context:** Building the PROV-SWEEP collision detector after the Conv-197 route flip.

**Learning:** Before the flip, `src/components/matt/` was a free domain delimiter — "unmarked design-system primitive" = "unmarked file under matt/". Post-flip the namespace dissolved into `src/components/*`, co-mingling design-system components with original legacy app components; only *pages* got the `/old/*` split, not components. So the collision-candidate set cannot be inferred from folder structure — it must be an explicit, hand-maintained enumeration. "Derive what's marked (grep), declare what isn't (registry)."

## 2. Tighten the accept-condition, don't chase reject-conditions
**Topics:** cc-workflow

**Context:** The marker scanner counted prose mentions of `@matt-source` in `icon-provenance.ts`'s own header as markers (39 vs expected 38).

**Learning:** The §9 grep-pollution gotcha (Provenance notes containing the token) was handled by exclusion rules, but a file that *documents* the marker convention in prose is a new variant. The robust fix is making the positive match stricter — a real marker is `@matt-source` + a node-shaped ref (`\d+:\d+`), never an English word. This also faithfully implements §4 "the node ref is load-bearing."

**Pattern:**
```
const MARKER_RE = /@matt-source\s+(\S*\d+:\d+\S*)/;
```

## 3. Name-match is necessary but not sufficient for collision detection — filter on node type
**Topics:** astro, cc-workflow

**Context:** PROV-MATCH found two exact name-matches against Matt's Figma, both false positives.

**Learning:** "Section Title" matched Matt's design-process readiness-banner component (WIP/Dev Ready/Archived variants); "Sub Nav" matched Figma canvas *sections*. A purely mechanical name-matcher would have flagged both as "reconcile now" and triggered needless re-translation. The disambiguator is node *type* (`<section>` vs `<symbol>`/component) and variant semantics. The eventual automated matcher must filter section-type and status-component nodes out of the candidate pool.

---

## 4. Triage terse task codes before acting — grab-bag closeout can silently drop big work
**Topics:** cc-workflow

**Context:** "Keep going closing similar work" against a grab-bag of terse RESUME-STATE codes.

**Learning:** Recovering each terse code's actual scope from docs *before* touching it separated the genuinely-closeable five from two that needed re-scoping: `[ICN-NS]` was a 204-file migration (not cleanup) and `[HOWTOREG-ICN]` was blocked on an unlocatable source. Force-closing either would have dropped real work or burned the conv hunting an unknown Figma file. Verify-before-closing is the same discipline that caught the NAV-APP-A grep false-positive and the sweep's prose-marker false-positive.
