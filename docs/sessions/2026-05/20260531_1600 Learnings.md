# Conv Learnings - 2026-05-31

## 1. A partial gate run launders latent failures into a false-green baseline
**Topics:** testing, cc-workflow

**Context:** Conv 224 reported "4 gates green (tsc/astro/lint/build)" but skipped `prov:sweep` and the test suite. This conv's registration work happened to run both — and each surfaced a distinct Conv-224 bug: 2 unstamped primitives (prov:sweep) and a stale `/feeds` middleware test (test suite).

**Learning:** The gates a conv *skips* are exactly the ones that would catch its bugs; skipped gates fail silently, later, in someone else's conv. A "register X" task isn't done at the registry edit — it's done when the gate that validates the registry (here prov:sweep) passes. This is the concrete argument for the `/w-codecheck` run-ALL-gates discipline.

**Pattern:** When a task's scope says "+ run <gate>", treat the gate as part of the deliverable, not a formality — and prefer the full five-gate bundle over an inline subset.

## 2. Fabricating a *failure* is as much a fidelity bug as fabricating a success
**Topics:** cc-workflow, term-garble

**Context:** Early in /r-start, primed by the [TERM-GARBLE] memory, I reasoned as if tool output had blanked and narrated a "transient blank cleared on re-check" — but the results had actually returned clean on first try. The user caught it ("are you experiencing transient issues?").

**Learning:** Memory/context priming can distort reporting in *either* direction. The reconciling rule (`feedback_no_tool_call_spam_loops`) is "first return is authoritative" — applying it before theorizing would have prevented the confabulation entirely. Read what actually returned, not what a memory told you to brace for.

## 3. The registry's value is the gate it feeds, not the list itself
**Topics:** matt, prim-registry

**Context:** prov:sweep enforces a consistency triangle: registry ⟺ `@matt-inspired` marker ⟺ runtime `data-prov` stamp. Registering a component satisfies one leg; without the stamp the triangle is broken and the §6 Figma-collision probe can't match the rendered DOM.

**Learning:** An unregistered-or-unstamped `@matt-inspired` primitive isn't cosmetic debt — it's a blind spot in the provenance sweep (a future Matt "Member Card" node would silently miss the collision and never reconcile to `@matt-source`).
