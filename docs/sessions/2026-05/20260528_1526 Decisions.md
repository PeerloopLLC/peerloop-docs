## 7. plan/COMPLETED.md stays terse; migrated-block "complete" status lives in the README
**Type:** Strategic
**Topics:** docs-infra, cc-workflow

**Trigger:** User asked: "How do we avoid the same growing size of COMPLETED_PLAN.md?"

**Options Considered:**
1. Strict terseness rule (one line per block, name + summary + conv range) + for migrated blocks, mark the per-block README "✅ COMPLETE" in place ← Chosen
2. Multi-paragraph entries per block (current de-facto practice in some entries)
3. Year-rollover only when needed (`COMPLETED-001-200.md`, …)

**Decision:** Stay terse per `fmt-update-plan.md`. When a migrated block completes, mark its `plan/<slug>/README.md` "✅ COMPLETE" rather than moving the content into COMPLETED.md. Year-rollover is the fallback if growth ever happens.

**Rationale:** Growth comes from drift on the terseness rule, not necessity. COMPLETED.md isn't auto-loaded by Claude (unlike MEMORY.md/CLAUDE.md), so its size isn't acute. The per-block README staying in place keeps the substantive history accessible at the per-phase granularity.

**Consequences:** No structural change this conv. Documented in the conv extract for future reference; not codified in `fmt-update-plan.md` (the terse rule is already there).
