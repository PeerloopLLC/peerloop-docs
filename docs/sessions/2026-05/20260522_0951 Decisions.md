**Type:** Strategic
**Topics:** matt-design, design-tokens, tailwind

## 3. Matt-composes-pages-from-components → we mirror with parameterized React/Astro
**Type:** Strategic
**Topics:** matt-design, component-architecture, react, astro

**Trigger:** User: "I think Matt is using UI components: he assembled components and then places the components in pages (possibly with some overrides). I think this is confirmed."

**Options Considered:** (no alternatives explicitly considered — user-stated principle directly mapped)

**Decision:** Every Matt component becomes a React or Astro component with parameters. Defaults come from component-level definition; overrides at call-site via props. **Rationale:** Matt's Button (6-mode multi-variant) is direct evidence of his Figma-component practice. Mirroring it in code preserves design coherence and avoids re-deriving the pattern for each primitive.

**Consequences:** Added §2 "Matt composes pages from reusable components (Conv 172 — confirmed)" sub-section with 5 implementation rules + resolution-hint paragraph linking to earlier ⚠ items (Header Bar slot, Sub Nav drawer, Role Tab Bar) showing how the composition principle answers each.

## 4. Preserve cascade chains in CSS — never flatten downstream semantics to primitives
**Type:** Implementation
**Topics:** css-variables, design-tokens

**Trigger:** Color Semantics extraction revealed 2-layer indirection (Student/Primary → Primary/Default → americana-blue); Entity adds 3rd layer.

**Options Considered:**
1. Flatten to primitives: `--Student-Primary: var(--americana-blue);` (simpler CSS, less indirection)
2. Preserve cascade: `--Student-Primary: var(--Primary-Default);` (matches Matt's Figma exactly) ← Chosen

**Decision:** Author CSS variables to preserve Matt's full indirection chain. Downstream semantics reference upstream semantics, NOT the primitives they resolve to.

**Rationale:** The cascade IS the design system's resilience. Changing the brand-primary value at `Primary/Default` should propagate automatically to all downstream semantics without manual updates. Flattening destroys this propagation.

**Consequences:** Documented as implementation rule in §5 Color Semantics + Entity sections. Will guide `tokens-semantics.css` authoring during [MATT-PRE-PLAN].
