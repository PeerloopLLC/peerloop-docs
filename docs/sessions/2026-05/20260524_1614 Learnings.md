# Conv Learnings - 2026-05-24

## 1. All Figma read tools are selection-free with an explicit nodeId
**Topics:** figma, cc-workflow

**Context:** Conv 181 saw `get_variable_defs`/`get_design_context` fail with "select a layer first"; Conv 186 disproved that for `get_design_context`; `get_variable_defs` was left "likely, not re-tested".

**Learning:** `get_variable_defs(519:9096)` with an explicit nodeId and NO desktop selection returned the full Variable map. Combined with Conv 186, ALL Figma read tools (`get_metadata`, `get_design_context`, `get_variable_defs`, `get_screenshot`, `get_libraries`, `search_design_system`) are selection-free given a nodeId. The "selection-required" tool class is retired. This is what makes the URL-driven drift-lookup workflow viable without the designer present.

## 2. Master/instance is the recurring shape for Matt's hero frames
**Topics:** figma, matt-design-system

**Context:** Both Course In Feed (502:12911) and Course Header (517:8934) on the Layout page turned out to be component SETS (sections wrapping `<symbol>` variants), while the α1 Happy Path nodes are instances.

**Learning:** When a Layout-page "Hero X" frame and an α1-Happy-Path "X" frame look like duplicates, they're almost certainly master/instance. The master reveals variants the instance hides (Course In Feed has a Mobile variant; Course Header has Default/Enrolled/Scheduled). Probe `get_metadata` on the section first to enumerate variants before building.

## 3. Addressability, not page-count, is the load-bearing routing decision
**Topics:** astro, matt-design-system, cc-workflow

**Context:** Presented route-shapes as "one page vs many files"; user redirected.

**Learning:** The architecturally-meaningful, implementation-independent question is whether a screen needs a jump-to/deep-link/redirect URL. An addressable route can still be ONE state-driven page (`/session/[id]` renders Prepare/During/After from status). "Needs a URL" and "how many files" are orthogonal — decide the first now, defer the second to build. Captured as `feedback_routing_addressability_first.md`.

---

## 4. Re-render-and-verify catches drift that building-from-spec misses
**Topics:** matt-design-system

**Context:** MMP-PH4's whole purpose was the empirical re-render gate; it surfaced both the 20-vs-24 icon-grid registry assumption AND (when checking the sibling course page) the CourseHeader trio drift.

**Learning:** Visually re-rendering against the source frame exposes assumptions that look fine in code. The Conv 182 "all icons 24×24" convention held only by luck until a 20dp Material icon arrived; the Conv 184 creator-trio drifted from Matt's actual frame. Both only surfaced under live re-render.
