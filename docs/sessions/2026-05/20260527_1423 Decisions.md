# Conv Decisions - 2026-05-27

## 1. Compose the Matt catalog card from primitives (not Figma-probe, not reuse-row)
**Type:** Implementation
**Topics:** astro, matt-design

**Trigger:** Matt has no catalog-card design; "retrofit the catalog grid" needs a card.

**Options Considered:**
1. Probe Matt's Figma for a courses-index design
2. Compose from existing Matt primitives ← Chosen
3. Reuse an existing Matt course row in a grid

**Decision:** New `CourseCatalogCard.tsx` composed from MattIcon + IconLabelChip + Button + tokens (user: "B. Matt has no card grid yet.").

**Rationale:** No Figma source exists to match; a row card doesn't tile; composing from primitives stays design-system-consistent.

**Consequences:** New UNMARKED (ours) component. Two layout variants (stacked grid / overlay band).

---

## 2. Drop the per-card CTA — whole card is a stretched-link
**Type:** Implementation
**Topics:** astro, accessibility, matt-design

**Trigger:** User: a "View Course" button on every card in an all-courses grid is redundant; clicking the card is natural.

**Options Considered:**
1. Keep the explicit "View Course" CTA
2. Drop CTA, whole-card stretched-link + hover/focus affordance ← Chosen

**Decision:** Removed Button/CTA; title `<a>` + `after:inset-0` overlay; hover lift-shadow (stacked) / white ring (overlay); price kept.

**Rationale:** Card is the nav unit in a catalog (unlike feed-embedded anchors which need their own CTA); less repetition, bigger hit target.

**Consequences:** Single anchor per card. Caveat: future per-card secondary actions will need explicit buttons reintroduced for those variants.

---

## 3. Drop the leading course icon on catalog cards
**Type:** Implementation
**Topics:** astro, matt-design

**Trigger:** User: the course icon repeats on every card in a course list and carries no info, especially with images present.

**Decision:** Removed the `MattIcon name="course"` glyph from the catalog card; the thumbnail is the identifier.

**Rationale:** Redundant in an all-courses context.

**Consequences:** Cards rely on the image (with `#1f2937` fallback when no thumbnail).
