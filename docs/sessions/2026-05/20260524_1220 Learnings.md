**Topics:** figma, mcp

**Context:** Components page has 12 status banners; only Icons (row 1) was probed initially. Needed banner IDs for the other 11.

**Learning:** All product-section banners on the Components page sit at y=785 directly above their sections at y=981+. The one off-axis case (Brand at y=-266 above Brand at y=-70) is the only deviation. Probed all 11 derived mappings in parallel — 11/11 matched expected sections. **Pattern:** `find each section's banner = same x, smaller y; verify with one probe before bulk-adding`

## 2. Matt's "Last Touched" field is the single drift signal
**Topics:** figma, mcp

**Context:** Designing drift detection. Initial schema said "Last Changed" — field name mismatch caught on first probe.

**Learning:** Matt's banner text is `Last Touched`, not `Last Changed`. Three text nodes per banner (title, status, date) all come back from a single `get_design_context` call — no need to drill into sub-nodes. Date format `May 20, '26` is unstable for diff (Matt could type `May 20, 2026` or `5/20/26`); always store both raw + ISO for stable comparison.

## 3. The `Home /` vs `Page /` prefix convention
**Topics:** figma, matt-design

**Context:** Row 16 (Home / Course Completed) was the first frame outside the `Page / *` namespace since row 2 (Home / Feed).

**Learning:** Matt's frame-banner titles use a deliberate prefix convention: `Home /` denotes a state or variant of the home page (Feed default, Course Completed celebration), `Page /` denotes a standalone destination screen. This affects routing: Home variants could be states/overlays of an existing route; Page-prefixed screens need their own files.

## 5. Matt's review passes are batched, not incremental
**Topics:** matt-design, drift-detection

**Context:** All 32 rows added this conv share `Last Touched May 20, '26`.

**Learning:** Matt promotes many frames to Ready For Dev in single review sessions, not incrementally. Future drift checks should expect bulk-update patterns — when a new Last Touched date appears on N>5 banners simultaneously, that's one Matt session, treat as one response unit. Don't plan 32 separate reviews when you can plan one batch review.

## 6. Figma MCP `get_design_context` works without selection on remote MCP
**Topics:** figma, mcp

**Context:** Per `reference_figma_mcp_behavior.md`, `get_design_context` was classified as "selection-REQUIRED" (needs user-clicked layer in Figma desktop).

**Learning:** With the remote MCP server + user-supplied node-IDs, `get_design_context` works without any active Figma desktop selection. 11+ parallel probes against arbitrary nodes returned full JSX without selection setup. The "selection-required" memory entry needs an update — selection-free probing works as long as you have the node-id.

**Pattern:** Update `reference_figma_mcp_behavior.md` Conv 186 finding: selection is required only when the user wants the *currently-selected* node; explicit node-IDs bypass the requirement.

## 8. Name-collision drift catches: section-name vs banner-title
**Topics:** figma, source-of-truth

**Context:** Two cases this conv: section frame `Button Primary` (`40:482`) has banner title `Buttons`; user-supplied label `Page Template` maps to Matt's banner `Layout Desktop`.

**Learning:** Section frame names (set by Matt when creating the section) and banner titles (set by Matt when stamping Ready For Dev) can drift. Per `feedback_external_source_of_truth_first.md` designer-supplied-catalogues rule, the banner is the authoritative product label. Preserve both in lookup: banner as canonical title, frame name as cross-reference, user label as semantic alias when supplied.
