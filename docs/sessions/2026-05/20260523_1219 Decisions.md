## 1. Figma MCP path: Remote (hosted) over Local (Dev Mode)
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** User pointed at `developers.figma.com/docs/figma-mcp-server/` after I drafted the local-MCP walkthrough from training-data recollection.

**Options Considered:**
1. Official local MCP (Figma desktop + Dev seat + manual settings.json) — original v1 path
2. Official remote MCP (`https://mcp.figma.com/mcp` + OAuth) — Figma's documented recommendation ← Chosen
3. Community `figma-developer-mcp` (npx + personal access token) — fallback

**Decision:** Use Figma's hosted Remote MCP server. Registration via `claude mcp add --transport http figma https://mcp.figma.com/mcp`, auth via OAuth in browser, no Figma desktop or Dev seat required.

**Rationale:** (a) Figma's own docs recommend it; (b) one-command setup vs multi-step install + admin + UI toggle; (c) OAuth means account-level identity matches whatever files the logged-in account can already see — likely sidesteps the Dev seat blocker that killed Conv 169's first MCP attempt; (d) works on any machine — just re-run `claude mcp add` and re-auth on M4Pro.

**Consequences:** v1 walkthrough doc rewritten as v2. The previously-mentioned message to Brian (Dev seat assignment) is now likely unnecessary for the primary path but harmless as fallback intel.
