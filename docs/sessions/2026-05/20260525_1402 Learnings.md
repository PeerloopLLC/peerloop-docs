# Conv Learnings - 2026-05-25

## 1. Spacing-bridge hijack scope: 3,894 utilities / 354 legacy files vs 11 matt files
**Topics:** astro, tailwind, matt-design

**Context:** "Run the legacy spacing audit" — quantified the Conv 174 `--spacing-*` override blast radius.

**Learning:** The Conv 174 bridge override hijacks every utility reading the spacing scale (padding, margin, gap, AND width/height/inset) at steps {4,8,12,16,20,24,32,40,48,64}, each rendering at exactly ¼ size. 3,894 occurrences across 354 legacy files depend on the OLD (Tailwind) meaning; only 11 `matt/` files depend on the NEW (Matt) meaning. That asymmetry inverts the original "don't revert" logic — but the user chose to leave it and migrate legacy→Matt incrementally instead. Per-component remediation = convert hijacked-step utilities to arbitrary `[Npx]`.

**Pattern:** Spacing-hijack remediation, per-component: `h-12 w-12`→`h-[48px] w-[48px]`, `py-12`→`py-[48px]`, `gap-4`→`gap-[16px]`, `mb-8`→`mb-[32px]`, etc. Steps NOT in the hijacked set (2, 2.5, 3, 6) are unaffected and need no conversion.

## 2. Discover course-link destinations live in shared components, not the page
**Topics:** astro, routing

**Context:** Evaluating the "copy /discover/courses → /matt/courses and change destinations" idea.

**Learning:** `/discover/courses.astro` is thin; the `/discover/course/${slug}` hrefs are hardcoded several layers down in shared `ExploreCard.tsx:83`, `ExploreCourseTabs.tsx:73`, `ExploreTeachingTab.tsx`, `ExploreAllTab.tsx:320`, all reused by the real discover page. "Change the card destinations" is not a page edit — it requires forking or prop-threading the shared tree. This is why building a fresh Matt-native index with cards that own their own hrefs (approach B) sidesteps the problem entirely.

## 3. Byte-lossless doc split via sed line-ranges + reconstruction diff
**Topics:** docs-infra

**Context:** Splitting matt-design-system.md (1,717 lines) into a folder.

**Learning:** Slicing a large doc by verified line-range boundaries (`sed -n 'A,Bp'`) preserves content exactly. Confirm losslessness by concatenating the split bodies (stripping added breadcrumbs/titles) and `diff` against the original line range — proves zero content drift before the stub overwrites the source.
