# Conv Decisions - 2026-05-07

## 1. /r-timecard-day Skill Remains Unchanged — Foreground Model Testing via Fresh Session
**Type:** Implementation
**Topics:** cc-workflow, timecard, skill-architecture, model-testing

**Trigger:** User disappointment with sub-agent dispatch (Sonnet inconsistency, 2-3 minute runs vs. 15-second main-context baseline). User suggested testing with Sonnet/Haiku in foreground (main context) instead of sub-agent.

**Options Considered:**
1. Modify /r-timecard-day to conditionally dispatch to sub-agent based on model tier
2. Leave /r-timecard-day unchanged; user tests via fresh foreground session with model override ← Chosen
3. Refactor /r-timecard-day to reduce complexity so sub-agent overhead is negligible

**Decision:** Option 2 — no skill changes. The skill is designed to run in main context and already supports all models equally.

**Rationale:** Sub-agent dispatch introduced unexpected costs (cold-start, hallucination) without corresponding benefits. Foreground testing is simpler: user runs `claude --model claude-sonnet-4-6 --add-dir ../Peerloop` in fresh terminal, then invokes /r-timecard-day normally. This matches the skill's original design and avoids skill-modification debt.

**Consequences:** User can now test smaller models without sub-agent overhead. The experiment is deferred pending foreground test results. Block-summary adoption glide path remains the long-term solution (LLM step becomes unnecessary as commits mature).
