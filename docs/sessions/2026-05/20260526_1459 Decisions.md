**Type:** Implementation
**Topics:** astro, docs-infra

**Options Considered:**
1. Markdown `_INDEX.md` only

## 2. Semantic-token provenance via consuming-node re-probe
**Type:** Implementation
**Topics:** astro, cc-workflow

**Trigger:** Semantic tokens are Matt-extracted but `tokens-semantic.css` recorded no node; "unmarked = ours" would falsely tag them.

**Options Considered:**
1. Re-probe Figma for a consuming node ← Chosen (user)
2. Collection-name marker `@matt-source collection:"…"`
3. Defer / flag as gap

**Decision:** Probe a frame that consumes the semantic Variables, cite that node (`40:484` entity/color, `40:482` button) — same convention as primitives/typography.

**Rationale:** Most consistent + fully verifiable (re-probe returns the semantics); Figma access was live.

**Consequences:** Semantic blocks now carry verifiable markers; Speculative `--Alert-*` stay unmarked.

## 4. `matt-embedded` provenance class
**Type:** Strategic
**Topics:** astro, docs-infra

**Trigger:** `stars_2`/`accessibility_new` are Material glyphs Matt placed in his Social Post but never catalogued.

**Options Considered:**
1. Treat as ours (Material origin)
2. New `matt-embedded` class — Matt-authoritative by curation ← Chosen

**Decision:** Authorship is decided by curation, not who drew the pixels. Material glyphs Matt placed in his Figma = Matt's (cite his frame); Material glyphs *we* picked = ours.

**Rationale:** Consistent with §3 dec 1 (the marker, not pixel-origin, is the authorship signal). Recorded as the registry `source` field.

**Consequences:** `matt-provenance.md` §9 documents the third class; icon registry encodes it.
