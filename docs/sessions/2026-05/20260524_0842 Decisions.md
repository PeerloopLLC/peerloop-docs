## 2. ChatBubble width drifts from Matt's 159px canvas placeholder
**Type:** Implementation
**Topics:** chat, design-system, strict-b

**Trigger:** ChatBubble's Figma frame is 159px wide with Matt's "Text" placeholder. Real chat messages vary widely in length.

**Options Considered:**
1. Strict-B mirror: hardcode `w-[159px]` (matches Matt exactly but breaks UX in production)
2. ← Chosen: `inline-flex max-w-[280px]` content-sized with optional className override
3. Take a `width` prop (over-engineered API)

**Decision:** Drift from strict-B for this single dimension. Use `inline-flex max-w-[280px]` as default, allow className override (e.g., `!w-[159px]` for strict mirror or `!w-full` for column stretch).

**Rationale:** Matt's 159px is a Figma drawing placeholder, not a UX rule. Chat bubbles MUST size to content; 280px is a reasonable industry-standard cap for narrow chat columns. Documented in component docstring so the drift is explicit, not silent.

**Consequences:** ChatBubble works correctly in real chat threads. Strict-B is preserved as an optional override.
