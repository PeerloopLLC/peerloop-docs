**Type:** Strategic
**Topics:** astro, deployment, cc-workflow

**Trigger:** Recreating Home from `/old` required deciding which shell legacy bodies land in; governs ~80 more pages.

## 2. Page owns its SubNav (slot-based, opt-in)
**Type:** Implementation
**Topics:** astro

**Trigger:** Subnav-ownership question.

**Options Considered:**
1. Layout owns a default subnav, page overrides
2. **Page fills `<slot name="sub-nav">` (current pattern)** ← Chosen

**Decision:** Keep page-owned. Home + subnav-less pages simply omit the slot.

**Rationale:** SubNav is page-section nav with unique per-page content; a shared default isn't semantically shared (Sidebar is the app-wide nav).

**Consequences:** No layout change; new Home is subnav-less.
