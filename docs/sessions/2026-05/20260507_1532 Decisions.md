## 3. /r-timecard-day writes to Obsidian vault, replacing .timecard.md
**Type:** Strategic
**Topics:** infra, obsidian, dual-repo, cc-workflow

**Trigger:** User asked to write timecard to a uniquely-named file in their Obsidian vault.

**Options Considered:** Across 4 sub-decisions:
1. Vault path: inside repo / outside repo / docs/sessions/ → Outside repo
2. Filename format: bullets+comma kept / dashes / ISO-sortable → Bullets+comma kept (matches H3 literally)
3. Write target: replace .timecard.md / both → Replace
4. Conflict: overwrite / halt-and-ask / -v2 suffix → Halt-and-ask

**Decision:** Vault file is the only output. Path stored as tilde-prefix in committed config (`.claude/config.json → rTimecardDay.vaultPath = "~/Obsidian Vaults/main2025/_projects/Peerloop/timecards"`). Filename mirrors actual H3 emission with `:`-stripped start time.

**Rationale:** Tilde-prefix + Obsidian Sync = full cross-machine portability. Replace eliminates dual-write artifacts. Halt-and-ask defends against accidental overwrites of manually-edited timecards.

**Consequences:** New script fields (vaultPath, vaultFilename, vaultTargetPath, vaultDirExists, vaultTargetExists). SKILL.md Step 5 rewritten with three-branch flow. Stale `.timecard.md` deleted from repo root. M4 folder created (Obsidian Sync replicated to M4Pro).

> **Insight:** Cross-machine portability is two-layered: the **path** (tilde + `$HOME`) and the **content at the path** (Obsidian Sync, iCloud, git, etc.). Tilde-prefix in committed config solves layer 1; the chosen file-sync mechanism solves layer 2. Together: zero per-machine bootstrap. Skills designed against this primitive don't need machine-detection branches.

## 4. Filename matches H3 literally — "May 6, 2026" not zero-padded "May 06"
**Type:** Implementation
**Topics:** infra

**Trigger:** Discovered preview text said "May 06, 2026" but actual H3 emits "May 6, 2026" via `dt.getDate()`.

**Options Considered:**
1. Filename mirrors actual H3 ("May 6, 2026") ← Chosen
2. Zero-pad both H3 and filename ("May 06, 2026")

**Decision:** A.

**Rationale:** Preserves the "non-icon H3" interpretation literally. Zero-padding would change `renderedMarkdown` byte-for-byte for every prior timecard — a v2-of-the-format moment not warranted by this design.

**Consequences:** Filename today is `Peerloop Timecard • Coding • May 6, 2026 • 0910.md`. Future timecards inherit same format.

## 5. Don't auto-mkdir the vault folder on first run; error and instruct
**Type:** Implementation
**Topics:** infra, obsidian

**Trigger:** Vault target derivation surfaced `vaultDirExists: false` on M4 (first run).

**Options Considered:**
1. Auto-mkdir -p on first run
2. Error with mkdir command in the message; user creates manually ← Chosen

**Decision:** B.

**Rationale:** Auto-mkdir masks vaultPath typos (config off-by-one silently spawns a fresh folder, files vanish into wrong place). Catching the missing folder once per machine is cheap; catching a typo months later is expensive.

**Consequences:** Step 5 first branch is the dir-missing STOP path with a copy-pasteable mkdir command. User retains the safety net.
