### New Patterns
- **API scope parameter for role-aware endpoints:** When an endpoint serves multiple scopes that map to different UI surfaces, accept an explicit `scope=...` query param so the caller (UI tab) declares intent rather than relying on server-side role precedence. Server validates: 403 if caller lacks the role for the requested scope. Default-without-scope keeps highest-privilege precedence for backwards compat. (Conv 165 [CRT-2.5])
- **Primitive role-flag props for Astro→React composition:** When a React island needs to render role-specific child content based on data from Astro frontmatter, do NOT pass `ReactNode` instances through `client:load` — they get JSON-serialized and React rejects them. Instead, pass primitive booleans/strings and let the island import the child components directly and build the content internally. (Conv 165 [CRT-3])
