## 2. Tier 1 + Tier 3-git tightening (path-scope interpreters, add 15-entry git deny block)
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** CC's danger-audit identified residual Tier 1 (arbitrary code execution) gaps after `bash:*` removal — `node:*`/`python3:*`/`npx tsx:*`/`xargs:*` were equivalent escape hatches.

**Options Considered:**
1. Tighten Tier 1 only — scope `node`/`python3`/`tsx` to known dirs; drop `xargs:*`
2. Tighten Tier 1 + 3-git — also add deny block for destructive git shapes ← Chosen
3. Full lockdown — Tier 1 + git split + wrangler scoped to `--local`
4. Don't change anything

**Decision:** Replaced `node:*`/`python3:*`/`npx tsx:*` with path-scoped variants (`node .claude/scripts/*`, `node ../Peerloop/scripts/*`, `node scripts/*`, `node /tmp/*` and parallel for tsx/python3). Dropped `xargs:*`. Kept broad `git:*` in allow; added 15-entry deny block targeting `git push --force*`, `git push -f*`, `git reset --hard*`, `git clean -fd*`, `git checkout -- *`, `git branch -D *`.

**Rationale:** Path-scoping preserves legitimate use while removing `-e '<arbitrary>'` escape. Deny block lets normal git workflow run unattended while gating destructive shapes.

**Consequences:** 77 allow + 15 deny entries. Known limitation: `git -C <path> push --force` form not covered by deny patterns (CC matcher is prefix-based; see Learnings §1).
