### Timestamp Precision Debugging

**Claude:** Hit a recurring test issue where tests passed individually but failed when run together. Root cause: SQLite's `datetime('now')` returns `YYYY-MM-DD HH:MM:SS` format while JavaScript's `new Date().toISOString()` returns ISO 8601 (`YYYY-MM-DDTHH:MM:SS.sssZ`). The `T` character sorts differently from space in string comparison, breaking `created_at > last_read_at` comparisons. Additionally, sub-millisecond test execution meant mark-read and send-message could produce identical timestamps.

Solved by restructuring tests to use NULL `last_read_at` (which counts all messages from others, immune to precision issues) and explicit state cleanup between test sections (DELETE messages + reset last_read_at). All 92 messaging + notification tests pass together across 8 files.
