# Session Decisions - 2026-03-05 (Session 336)

## 1. BBB Connectivity as Integration Test (Not Standalone Script)
**Type:** Implementation
**Topics:** testing, video

**Trigger:** User wanted routine verification that BBB is working. Initial approach was a standalone `scripts/test-bbb.mjs`.

**Options Considered:**
1. Standalone script in `scripts/` — run manually when needed
2. Integration test in `tests/integration/` with skip helpers ← Chosen
3. Separate npm script (e.g., `npm run test:bbb`)

**Decision:** BBB connectivity is tested as a vitest integration test (`tests/integration/bbb-connectivity.test.ts`) using `describeWithBBB` for conditional execution. Runs as part of `npm test` when credentials are available, skips otherwise.

**Rationale:** Integrating into the existing test suite means it runs routinely without extra steps. The skip helper pattern (`canUseBBB` / `describeWithBBB`) matches the established `canUseTestDB` / `describeWithTestDB` convention. A standalone script would be forgotten.

**Consequences:** Added `tests/helpers/bbb.ts`, extended `tests/helpers/skip.ts` and `tests/helpers/index.ts`. Deleted `scripts/test-bbb.mjs`.

## 2. BBB Test Client Independent of Source Adapter
**Type:** Implementation
**Topics:** testing, video

**Trigger:** Needed to call BBB API from tests. Could either import `BBBProvider` from `src/lib/video/bbb.ts` or reimplement the minimal API client.

**Options Considered:**
1. Import and use `BBBProvider` from src ← Tests our adapter, not the service
2. Standalone test client using Node `crypto` ← Chosen

**Decision:** The test helper (`tests/helpers/bbb.ts`) reimplements BBB checksum auth and API calls using Node's `crypto` module, independent of `src/lib/video/bbb.ts`.

**Rationale:** The purpose is to test that the external BBB service is reachable and functional — not to test our adapter code. If the adapter has a bug, this test still catches BBB outages. Unit tests for the adapter itself would import from `src/`.
