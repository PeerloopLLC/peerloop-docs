# Conv Learnings - 2026-06-15

## 1. Generated route-migration README went stale fast — reconcile against the live tree before planning off it
**Topics:** route-migration, planning, docs-infra

**Context:** The RTMIG-4 backlog README was generated Conv 249 with "53 remaining"; by Conv 290 the tree had moved substantially (admin rehosted, role workspaces built, profiles still unported).

**Learning:** A point-in-time generated planning doc must be re-derived from `src/pages` (file list + provenance markers) before it's trusted for decomposition — the genuinely-remaining work was a fraction of the stale count, and most `/old/*` files were either stale ported-copies or owned by other blocks (ROLE-STUDIOS, PUBLIC-PAGES).

**Pattern:**
```
find src/pages -not -path 'old/*'
# + grep '@stand-in|@matt-source|@matt-inspired' per file
# → reconcile against the doc's cluster table
```

## 2. Tier-1/Tier-2 is not a per-cluster bucket
**Topics:** route-migration, matt-design

**Context:** Discussion framed "Tier-1 vs Tier-2 handling needs" per cluster.

**Learning:** Per `docs/decisions/11-new-routing.md:445`, every remaining cluster is Tier-1 (Matt shell + SubNav + tokens + existing-primitive swaps); Tier-2 is a cross-cutting deferred new-primitive-extraction pass (Rule-of-Three), not a phase any single cluster "is in." The per-cluster decision is really rehost-then-port sequencing + how much existing-primitive reuse is available.
