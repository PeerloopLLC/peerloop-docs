# Conv Learnings - 2026-06-27

## 1. `snapshot: true` is write-only-on-pass against the in-memory test DB
**Topics:** testing, plato

**Context:** Needed to extend the snapshot pattern to 3 instances without risking the green baseline or touching the user's dev D1.

**Learning:** `plato-scenarios.api.test.ts:400` gates the snapshot write on `if (file.snapshot && result.status === 'pass')`, and `runInstanceFile` serializes `getRawTestDB()` (the in-memory better-sqlite3 test DB) — never the Cloudflare dev D1. So adding the flag can only *add* a write side-effect on a passing run; it cannot regress the baseline, and snapshot generation never mutates the dev server/DB. Generation is driven by the dynamic `PLATO_INSTANCE` env + `--testNamePattern="run <name>"`.

## 2. "API GREEN" did not cover the instanceFile path — a class of latent verify drift
**Topics:** testing, plato

**Context:** `activities` passed as a `Scenario:` block in `npm test` but FAILED the moment it ran as an instanceFile (for snapshot generation).

**Learning:** Standard `npm test` runs activities/ecosystem/member-directory only as `Scenario:` blocks; the static `Instance:` blocks cover only new-user-pair/flywheel/session-invite. So the **file-level `verify` arrays** of those 3 were never executed — `activities` carried a stale `expected: 7` availability assertion that contradicted both the step that produces the data (5 weekday slots) and that step's own verify. Lesson: a green API suite can still hide instanceFile-level drift; gating the instanceFile path (static blocks or a `PLATO_INSTANCE` CI loop) would catch it.

## 3. String drift vs. semantic drift
**Topics:** plato, route-migration

**Context:** Bucket-2 prose had two very different kinds of staleness.

**Learning:** `/discover/courses`→`/courses` is *string drift* — the route moved, the fix is mechanical and unambiguous. `member-directory`'s "2 cards on default load" is *semantic drift* — the page's behavior changed (now defaults to the Creator filter), so the walkthrough's assertions are logically wrong, and refreshing them correctly depends on runtime behavior the source only partially reveals. Semantic drift pairs naturally with a live re-walk; only string drift is safe to blind-edit.

## 4. The route-flip flattened navigation depth
**Topics:** plato, navigation

**Context:** Fixing the 17 nav waypoints.

**Learning:** The fixes weren't URL swaps — they collapsed multi-click paths. The old `[DiscoverSlidePanel]` was 2 clicks (open panel → pick item); the Matt Sidebar exposes Courses/Members as direct top-level items (1 click). The old `[UserAccountDropdown]` dropdown actions (sign-out, onboarding) became first-class `/profile` settings tabs (Account / Interests). So each waypoint's *shape* changed, not just its destination.
