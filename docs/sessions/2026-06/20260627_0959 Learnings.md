# Conv Learnings - 2026-06-27

## 1. Subagent CLOSE recommendations need adversarial verification
**Topics:** cc-workflow, agents

**Context:** A 3-agent task-review fan-out returned ~13 dispositions; 2 CLOSE calls (#18 OLD-DOCS-RECON, #5 VITE-DEDUP) were wrong in the *harmful* direction — they would have buried real drift / closed a still-broken task.

**Learning:** Delegated "is this done?" judgments have a measurable error rate, and the dangerous failure is a false-CLOSE. Verify any agent recommendation to close/delete before acting — especially on drift-tracking or dead-code tasks. The agent's plausible prose ("already reconciled", "fix in place") is not evidence; the file/commit is.

## 2. Doc-reconciliation is a verb-tense test, not a keyword scan
**Topics:** docs-infra, drift

**Context:** A subagent declared `url-routing.md` clean because it read `/old/*` mentions as "historical context." But lines like "`/old/learning*` stays live as the island source" are present-tense assertions that went silently false when Conv 339 deleted those pages.

**Learning:** When reconciling a doc against deleted entities, the test is "is this claim still true *now*?" — not "does the doc mention the entity?" Dated changelog entries ("Previously: Conv 255 did X") are history and stay; present-tense current-state claims are the drift to fix. The same principle let me correctly preserve url-routing.md's `Previously:` block while rewriting its status banner.

## 3. A command error in tool output means re-run, not trust
**Topics:** cc-workflow, shell

**Context:** `grep -rl ... --include=*.tsx` under zsh glob-expanded `*.tsx` and errored (`no matches found`), so the importer-detection loop returned "orphan" for *every* `unified/` file. Trusting it would have deleted PriorityHeader / NeedsAttention / CollapsibleSection / types and broken TriageStrip on the live Home page.

**Learning:** Tool output is authoritative only when the command actually ran. An error line in the output (`no matches found: --include=*.tsx`) invalidates the result — re-run with a fixed command (quote/drop the `--include` glob). This is distinct from the no-spam rule: re-running a *malformed* command is correct, not buffer-flushing.

**Pattern:**
For "is X imported anywhere?" use `grep -rlE "import[^=]*\bX\b" src` (no `--include` glob under zsh) + a separate `export .*from` barrel check.

## 4. Delete-orphan tasks must verify shared-component reuse first
**Topics:** testing, refactor

**Context:** [UNIFIED-DASH-RM] read as "delete UnifiedDashboard + its `unified/` subtree." But 4 of 13 subtree files are imported from *outside* `unified/` (TriageStrip uses PriorityHeader/NeedsAttention/types; MyFeeds uses CollapsibleSection).

**Learning:** "Orphaned component" ≠ "orphaned directory." Before deleting a subtree, check each file for importers *outside* the subtree; keep the shared ones. The deletion set was 9, not 13. All 5 gates confirmed green afterward.

**Topics:** astro, state-management

**Context:** Reconciling state-management.md required finding what replaced the deleted AppNavbar's CurrentUser-global init.

**Learning:** The Matt-shell APP side extracted global init *out of the navbar* into a dedicated headless `CurrentUserInit` island (`AppLayout`, `client:load`, `return null`); the visible nav is `Sidebar` (SSR `sidebarUser` prop, persists as `matt-sidebar`).
