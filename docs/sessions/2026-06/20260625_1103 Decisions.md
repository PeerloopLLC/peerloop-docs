## 1. Stat-card hues mapped by lifecycle meaning, not original colour
**Type:** Implementation
**Topics:** rg-admin, conformance, stat-cards

**Trigger:** moderators `StatCard` used tinted `bg-{hue}-50` cards (yellow/blue/green/gray); the white-card admin pattern needs a semantic value hue.

**Options Considered:**
1. Map by original colour (yellowŌåÆwarning, blueŌåÆinfo, ŌĆ”) ŌĆö preserves designer intent but yields odd semantics (Active Moderators = warning reads as "something wrong")
2. Map by lifecycle meaning (ActiveŌåÆneutral headline, PendingŌåÆwarning, AcceptedŌåÆsuccess, DeclinedŌåÆerror) ŌåÉ Chosen

**Decision:** Map stat *values* to semantic tokens by their lifecycle meaning, mirroring the certificates Total/Pending/Issued/Revoked = neutral/warning/success/error pattern.

**Rationale:** Consistency across the admin surface + honest semantics beat preserving an arbitrary original palette.

**Consequences:** moderators + moderation stat cards now share the certificates hue convention.

## 2. Suspend footer button uses the CC-owned `suspend` variant
**Type:** Implementation
**Topics:** rg-admin, rg-mod, button-variants

**Trigger:** `/admin/moderation` footer Suspend button (was `bg-red-700`, more severe than Remove's `red-600`).

**Options Considered:**
1. `danger` (clearly correct for a severe action, matches the FormModal submit)
2. `suspend` (CC-owned graded-orange variant minted in RG-MOD Conv 313) ŌåÉ Chosen

**Decision:** Suspend ŌåÆ `variant="suspend"`, mirroring RG-MOD's ModeratorQueue action vocabulary (Warn=warning, Suspend=suspend).

**Rationale:** `/admin/moderation` and `/mod` are sibling moderation surfaces with the same actions; matching RG-MOD's variant choices keeps them visually identical. The `suspend` variant exists precisely for this graded tier.

**Consequences:** All four moderation footer actions (Dismiss/Remove/Warn/Suspend) now use the same Button-variant vocabulary across both surfaces.
