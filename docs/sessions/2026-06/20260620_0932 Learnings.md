# Conv Learnings - 2026-06-20

## 1. Screenshot beats getComputedStyle for painted colour under a CSS transition
**Topics:** testing, browser-verify, astro

**Context:** While verifying the new StarRating, `getComputedStyle(button).color` reported `border-default` (rgb 234,239,245) on stars whose class was correctly `text-star` and whose `--color-star` var resolved to `#F5A623` — and an identical fresh button rendered gold. Chased it as a CSS-override bug through ancestor scope, sibling selectors, and stylesheet enumeration before a zoomed screenshot showed the paint was correct (4 gold + 1 grey).

**Learning:** The project's "DOM-truth over screenshots" rule (`feedback_dom_truth_over_screenshots`) is about **geometry** (position/size). For **painted colour while a `transition-property: color` is mid-flight / under programmatic interaction**, `getComputedStyle().color` can return an interpolated or stale value, whereas the rasterized screenshot shows what the user actually sees. The two tools are authoritative for *different* questions — geometry → DOM, final paint under animation → pixels.

**Pattern:** When a star/colour widget has `transition` and a JS-read disagrees with expectation, zoom-screenshot it before assuming a cascade bug.

## 2. prov:sweep is a bidirectional contract — enrolling demands a stampable DOM root
**Topics:** testing, cc-workflow, matt-provenance

**Context:** Resolving the prov:sweep "UNTRACKED" debt by adding registry entries worked for 5 components but flipped FeedPost into 2 *new* "UNSTAMPED" errors.

**Learning:** The sweep enforces both directions: (a) a file that stamps `data-prov-name` must have a registry entry; (b) a file IN the registry must carry a matching `data-prov` stamp. So before enrolling a flagged file, ask "does it render its own DOM element?" A pure pass-through adapter (returns another component directly — like `FeedPost` → `<SocialPost>`) has no identity to stamp, so it must NOT be registered; resolve its `Provenance: UNMARKED` drift by reclassifying the note (drop the keyword) instead. The drift detector greps the literal `/Provenance:\s*UNMARKED/`.

**Pattern:** Registry entry ⟺ stampable DOM root. Adapters/wrappers that return another component → reclassify at source, don't enroll.

## 3. Time-gated UI states are reachable by a throwaway seed mutation
**Topics:** testing, d1

**Context:** The early/joinable SessionPrepare composer only renders when `now < scheduledStart − 15min`, and all seed sessions were past.

**Learning:** To browser-verify a time-gated state, temporarily UPDATE an existing scheduled session's `scheduled_start`/`_end` into the future in local D1 (capture originals first), verify, then restore — cleaner than INSERTing a new row (no FK/NOT-NULL juggling) and fully reversible.
