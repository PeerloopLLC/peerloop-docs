# Conv Learnings - 2026-06-21

## 1. SegmentedPills single-select ‚Č† MembersFilters multi-select
**Topics:** route-migration, primitives, tier-2

**Context:** The Tier-2 ledger (Conv 309) named MembersFilters a "latent 4th adopter" of `SegmentedPills` based on identical *look*.

**Learning:** `SegmentedPills` is single-select by construction (`value: T`, `active = value === opt.value`); MembersFilters is multi-select (`Set<string>` toggle + an "All = none-selected" pill). Adoption would require adding a `multiple`/`values[]` mode to the primitive (touching its 3 single-select consumers). The Conv-309 "identical look" note missed the behavior mismatch. Kept hand-rolled and logged as un-ripe.

**Pattern:** Classify a candidate by BEHAVIOR (selection model, state shape), not just visual look, before declaring it a drop-in adopter.

## 2. An [Opus]-tagged blocking task can be mostly-already-done
**Topics:** role-semantics, task-scoping

**Context:** ROLE-SEMANTICS (#4, [Opus]) read like an open architectural question and was treated as a hard blocker for RG-PUBPROF.

**Learning:** The model was decided Conv 252 and incrementally implemented ‚ÄĒ nearly every API endpoint + SSR loader already funnels through canonical `isCreatorSubquery`/`isTeacherSubquery`; UserProfileHeader badges were already identity-correct. The actual remaining work was one stubbed field (`primary_topic_id: null`) + an optional DRY refactor. "Blocked by ROLE-SEMANTICS" carried far more weight than the real residue.

**Pattern:** When a task is tagged hard/[Opus] and blocking, inventory what's *already built* before scoping the work ‚ÄĒ the decision may be old and the implementation may already exist.

## 3. The /old‚Üíroot map is reshape-distorted
**Topics:** route-migration, old-cleanup

**Context:** User asked which /old/* routes are no longer needed (pre-RG-PUBPROF).

**Learning:** A naive "strip /old/ ‚Üí check root path" map under-reports porting because the migration *reshaped* routes (e.g. `/old/discover/members`‚Üí`/members`; 7-file `/old/course/[slug]/*`‚Üíone `[...tab]` hub; `/old/profile`+`/old/settings/*`‚Üí`/profile/[...tab]`). "Root missing at stripped path" ‚Č† "unported." Of 77 /old routes: 33 keep, 44 candidate-deletable.

**Pattern:** Classify /old routes by the *swept root route their function reshaped into*, not by literal path equivalence. And "swept root exists" ‚Č† "safe to delete /old copy" ‚ÄĒ needs a per-route inbound-ref check (OLD-PORTED-CLEANUP).

## 4. The inverted Matt spacing scale ‚ÄĒ numbers outside {4,8,‚Ä¶,64} fall back to default Tailwind
**Topics:** matt-design, spacing, conformance

**Context:** Deciding how to snap `/members` badge `py-[2px]` + `mt-2` during the conformance sweep.

**Learning:** `tokens-tailwind-bridge.css` overrides only the set `--spacing-{4,8,12,16,20,24,32,40,48,64}` to literal px. Numbers OUTSIDE that set (e.g. `2`) fall back to default Tailwind v4 (`mt-2`=8px), so `mt-2`(8px) is LARGER than `mt-4`(4px) ‚ÄĒ the inverted-naming trap. `mt-2`‚Üí`mt-8` is a zero-visual-change canonicalization; explicit arbitrary `py-[2px]`(2px) snaps up to `py-4`(4px) per the Conv-305 ties-up policy (and the AnalyticCount `px-[9px]/py-[5px]`‚Üí`px-8/py-4` precedent).

**Pattern:** Verify the override set before snapping bare numeric spacing classes; only {4,8,‚Ä¶,64} are literal-px, the rest are default-Tailwind and can be larger than they look.
