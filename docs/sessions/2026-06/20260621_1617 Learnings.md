# Conv Learnings - 2026-06-21

## 1. The sweep's browser-verify step catches latent bugs, not just conformance
**Topics:** route-migration, testing, astro

**Context:** Both RG-PUBPROF routes swept this conv surfaced non-styling defects during the DOM-truth browser-verify step (step 6), not during code review.

**Learning:** Browser-verify is the highest-yield step of the per-route sweep for *correctness*, not only visual conformance. On the hub it surfaced two dead components (one edited by the prior conv); on /teacher it surfaced a hydration mismatch. Treat browser-verify as a real test pass, not a formality.

## 2. `new Date(s).toLocaleDateString()` with no `timeZone` hydration-mismatches on month-boundary dates
**Topics:** timezone, astro, testing

**Context:** /teacher's `formatDate` (certified/issued dates) rendered "Dec 2023" server-side (UTC) and "Jan 2024" client-side (local TZ) → React hydration error, tree regenerated on client.

**Learning:** Any SSR-hydrated `toLocaleDateString`/`toLocaleString` without an explicit `timeZone` will diverge between the UTC server and a non-UTC browser for boundary values. Fix: add `timeZone: 'UTC'` (preserves the exact output) or use the project's `formatDateUTC`. This is a [TZ-AUDIT]-class issue and likely recurs in other ported components.

**Pattern:**
```js
new Date(d).toLocaleDateString('en-US', { month: 'short', year: 'numeric', timeZone: 'UTC' })
```

## 3. Shared legacy components can hide on already-"Swept" routes
**Topics:** route-migration, cc-workflow

**Context:** `CourseCard` (18 consumers) is untracked, legacy, and renders on already-swept `/courses` + `/course/[slug]` — yet was never caught by those sweeps.

**Learning:** A per-route sweep can mark a route Swept while a shared sub-component it renders stays legacy, if the browser-verify states didn't exercise the section that mounts it (e.g. related-courses grid, review list). Cross-cutting shared components need their own tracking (ledger row) the first time *any* route touches them; the Conv-304 "every rendered surface conforms" rule only holds if verify actually exercises every surface.

## 4. zsh does not word-split unquoted `$VAR` — multi-file greps silently match nothing
**Topics:** cc-workflow, dual-repo

**Context:** A residual-token scan used `FILES="a b c"; grep ... $FILES`. With zsh's default `SH_WORD_SPLIT` off, `$FILES` expands as ONE argument → grep treats it as a single (nonexistent) filename → empty output read as "clean."

**Learning:** In zsh, pass multiple files to a command as explicit quoted args (or a real array `${(f)...}` / `"${arr[@]}"`), never an unquoted space-joined string. A grep that "found nothing" across several files is suspect — verify the files were actually scanned (the companion keeps-grep returning empty was the tell).

## 5. [BRIDGE-MEM] own-profile detection needs a warm client cache
**Topics:** plato, cc-workflow

**Context:** Guy viewing his own /teacher page first rendered the *non-own* branch (Book a Session) instead of Edit/Teaching/Availability; a second navigate (cache warm) resolved it.

**Learning:** Components that set `isOwnProfile` from `getCurrentUser()` in a mount-only `useEffect` render the wrong branch until the client user-cache (`peerloop_user_cache`) is populated. When DOM-verifying own-profile views via dev-login, navigate once to warm the cache, then navigate again before asserting. Matches the documented [NUDGE-CACHE-FLASH] stale-cache caveat.
