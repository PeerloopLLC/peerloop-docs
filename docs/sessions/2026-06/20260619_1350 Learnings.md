# Conv Learnings - 2026-06-19

## 1. Generalizing the spacing-snap policy exposes stale "clean" spacing on already-swept components
**Topics:** conformance, spacing, astro

**Context:** TeachersTab's ledger row said Spacing ☑ ("clean, only a name-tracking drop pending"), but it actually carried `gap-[15px]` (off-scale) + arbitraries. That ☑ predated the Conv-305 §170 snap carve-out.

**Learning:** Any component marked Spacing ☑ *before* the snap policy may still hold off-scale/arbitrary spacing that the new rule would snap. The "clean" was true under the old "keep as exception" regime, not the new one. Sweeping forward must treat pre-policy ☑ spacing as suspect — this is exactly what `[SWEEP-SPACING-GREP]` #31 / `[SPACING-4PX-SWEEP]` #30 exist to catch across RG-HOME/COURSES/PROFILE. When a conformance rule tightens, every prior green checkmark for that axis becomes a hypothesis, not a fact.

## 2. Re-skinned components (Conv 242 [CALENDAR2]) did real conformance groundwork
**Topics:** conformance, spacing, astro

**Context:** SessionBooking (903-line booking wizard) had only 2 arbitrary-spacing signals (both `px-[2px]` sub-scale) despite its size — it already used scale classes + role tokens from its earlier `@stand-in`→`@matt-inspired` graduation.

**Learning:** A prior Matt re-skin leaves a component largely conformance-ready; the residue is usually just bare `font-*` tokenization + keep-exceptions, not a full pass. Gauge effort with a signal-count grep before assuming a large file = large work — line count is a poor proxy for conformance debt on re-skinned components.

## 3. Bare `font-medium` (no size class) tokenizes by inherited context
**Topics:** conformance, type, tokens, astro

**Context:** SessionBooking confirmation `<dd className="font-medium text-text-default">` rows had no explicit size; the parent `<div className="...text-body-default">` sets 14px.

**Learning:** A weight-only utility on an element with no size class inherits the nearest ancestor's size — verify that ancestor before mapping to a sized token. Here the `<dl>` rows were `text-body-default` (14px) → `text-body-default-medium` was the zero-change tokenization. Don't guess the size; read the wrapper that sets it.
