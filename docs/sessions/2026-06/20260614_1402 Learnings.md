# Conv Learnings - 2026-06-14

## 1. `townhall` is simultaneously stale branding AND a live infrastructure key
**Topics:** stream, terminology, refactoring

**Context:** A "rename all townhall→system" sweep would have broken the live system feed.

**Learning:** Three layers wear the same word: the D1 `feed_type` enum (renamed `townhall`→`system` in Conv 280), the **Stream.io feed group** (`SYSTEM_STREAM_GROUP='townhall'` — must stay, Stream groups are dashboard-declared not write-creatable, pointing code at a renamed group 500s with `FeedConfigException`), and the UI label. The correct rule for any cosmetic rename here is "rename the *branding*, preserve the *value*." Keepers: `system-feed.ts:29`, `seed-feeds.mjs` `feedGroup:'townhall'`, `tests/lib/promotion.test.ts:137` `streamGroup:'townhall'`, and `townhall:main` Stream addresses in docs.

## 2. Editing a comment in an applied D1 migration is not risky here
**Topics:** d1, deployment, migrations

**Context:** Skipped migration comment edits citing "risk to applied DDL"; user challenged it.

**Learning:** Wrangler D1 tracks applied migrations by **filename** in `d1_migrations` (no content hash); comments execute nothing; this project's `migrations/` is plain `.sql` + README with no drizzle `_journal.json`/snapshot. So a comment edit never triggers re-apply or a "migration modified" error. The legitimate concern was only low-value churn — which is not "risk." Mislabeling churn as risk discouraged a correct, in-scope edit (and would have left a factual auto-join error in the seed comment).

## 3. A carry-over task's premise can be stale — verify against disk before trusting scope
**Topics:** cc-workflow, audit

**Context:** The #31 carry-over said "~26 files" and named specific doc lines (ROLES.md:680, SCRIPTS.md:412, sentry.md:407).

**Learning:** Actual scope was 159 occurrences and the named line numbers didn't match disk. An audit-style sweep should re-derive the hit-list from a fresh grep, not execute against a remembered count. The user's instinct to widen scope (migrations, then the full audit) repeatedly surfaced genuine drift the original scope would have missed — 3 factual auto-join errors total across ROLES/DB-API/google-oauth + the seed comment.
