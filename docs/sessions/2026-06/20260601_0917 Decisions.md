## 9. Route scanner: getSession-without-redirect = "Public (adapts)" (not "Auth required")
**Type:** Implementation
**Topics:** testing, astro, docs-infra

**Trigger:** Decision 8 added `getSession` to index.astro; the route scanner then mislabeled `/` as "Auth required" (#33 [ROUTE-AUTH-HEURISTIC]) — same pre-existing misclassification already affecting `/feeds` and `/communities`, all of which welcome visitors.

**Options Considered:**
1. Special-case `/` in the scanner (narrow)
2. Fix the root cause: classify getSession pages by whether they actually gate access ← Chosen
3. Leave it (generated-doc inaccuracy)

**Decision:** In `route-matrix.mjs` `detectAuth`, a page that reads the session is "Auth required" only if it also `Astro.redirect`s the unauthenticated visitor away; if it reads the session but never redirects (just branches/adapts), or carries adapt markers (`isEnrolled|canPost|userId ?`), it is "Public (adapts)".

**Rationale:** The real auth signal is the redirect, not the mere presence of getSession. Root-cause fix beats special-casing.

**Consequences:** `scripts/route-matrix.mjs` updated; regenerated route docs reclassify `/`, `/communities`, `/feeds` → "Public (adapts)" (genuinely auth-optional); login/signup stay "Auth required". A latent dual-source dedup risk in r-end Step 2 was also found during this test and deferred → #34 [REND-DEDUP-GUARD].
