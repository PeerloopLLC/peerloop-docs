## 2. Give the `generated` doc category an executable regen binding + deterministic r-end gate
**Type:** Strategic
**Topics:** docs-infra, cc-workflow

**Trigger:** User asked whether a routine mechanism already solves stale crucial docs.

**Options Considered:**
1. Full build — `regen` binding + gate + reclassify + #25 fix ← Chosen
2. Minimal close-the-loop (fix #25 + reclassify + gate for existing docs only)
3. Capture as a task, build later

**Decision:** Full durable build, binding expressed by **extending the registry group** with `regen:{cwd,commands,inputs,alsoWrites}` (option A over a separate manifest).

**Rationale:** The registry already varies per-group config and owns doc→category; single source of truth. The gate (`regen-generated-docs.mjs`) runs at r-end Step 5c, inputs-gated on `src/pages|components|lib`, idempotent (so day-stamped docs don't churn).

**Consequences:** "Regenerate stale route docs" is now a deterministic step, not a recurring task. Future generated docs need only a new group — no engine change.

## 3. route-stories.md stays driftCheck (NOT generated)
**Type:** Implementation
**Topics:** docs-infra

**Trigger:** Building the `generated` group's doc list.

**Options Considered:**
1. Include all route-named docs as generated
2. Include only script-produced docs; verify by provenance header ← Chosen

**Decision:** Generated set = `route-api-map.md`, `page-connections.md`, 3 ROUTE-*.tsv (+ code `route-map.generated.ts`). `route-stories.md` is hand-written (canonical story↔route map) → stays driftCheck.

**Rationale:** Classifying a hand-authored doc as generated would let the gate clobber it. The file's provenance header is the only reliable discriminator.

**Consequences:** Gate never overwrites the hand-written story map.
