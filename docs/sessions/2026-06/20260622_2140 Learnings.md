# Conv Learnings - 2026-06-22

## 1. Before conforming a shared component, check whether the design system already absorbed its variants
**Topics:** matt-design, components, conformance

**Context:** Both conformance tasks this conv (ConfirmModal, CourseFeed) looked like judgment calls but collapsed into mechanical alignment once the codebase was checked.

**Learning:** ConfirmModal's danger/warning/default buttons mapped 1:1 onto `<Button>` variants that were *purpose-built* (Conv 306/313) to "dedup the hand-rolled red buttons." CourseFeed's local `RoleBadgeInline` was a stale duplicate of an already-conformed `RoleBadge` in its sibling FeedActivityCard. In both cases the conformant answer already existed nearby — "conform" became "align to the existing precedent," not "invent a mapping." Check for an existing primitive/precedent before treating a categorical-colour or component conform as a novel decision.

**Pattern:** grep the primitive's variant enum / sibling components for an existing conformed map before authoring inline classes.

## 2. A DOM back-glance whose strings don't match your edit = you're looking at a different component
**Topics:** dom-verify, plato, routing

**Context:** After conforming `community/CourseFeed`, the live `/course/<slug>/feed` rendered a feed whose composer placeholder ("Ask a question…") and disabled text ("hasn't enabled discussions") did NOT match the conformed source.

**Learning:** The mismatch was the tell that the route renders a *different* component — `MattCourseFeed` (the @matt-source student-page feed), not the one edited. The DOM back-glance is what surfaced a two-feeds split that a code-only review would have missed. Probe live DOM for your own edited strings to confirm you verified the component you actually changed.

---

## 3. The spacing bridge silently shrinks legacy bare-numeric utilities ~4×
**Topics:** matt-design, spacing, tailwind

**Context:** ConfirmModal's `p-6` and CourseFeed's `p-4`/`py-12` render at literal px under the tokens-tailwind bridge (`p-6`→6px, not 24px).

**Learning:** Conforming spacing to literal-px (`p-6`→`p-24`, ×4) is not merely stylistic — it *restores* the intended visual size that the bridge had been shrinking. Treat spacing conformance on legacy components as a latent-bug fix, not cosmetic polish.
