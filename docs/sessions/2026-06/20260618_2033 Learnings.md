# Conv Learnings - 2026-06-18

## 1. Spacing audits must include legacy numeric scale classes, not just arbitrary `[Npx]`
**Topics:** route-sweep, tailwind, conformance, palette-fdn

**Context:** Sizing RG-PROFILE's spacing axis, an initial grep for arbitrary `[Npx]` returned 0 across all files → wrote "Spacing clean group-wide." Reading NotificationSettings + DOM-checking revealed legacy numeric scale classes (`py-4`, `px-6`, `space-y-6`, `mt-0.5`) the grep never matched.

**Learning:** The NAV-RETROFIT bridge aliases the Matt set {4,8,12,16,20,24,32,40,48,64} to literal px, so legacy `py-4` renders at **4px** (collapsed from intended 16px) while off-set numbers like `px-6` fall through to default Tailwind (24px). A `[Npx]`-only grep is blind to both failure modes. A spacing-axis sizing pass MUST grep legacy numeric utilities too, and DOM-confirm a sample, before claiming clean.

**Pattern:**
```
grep -nE "\b(p|px|py|pt|pb|pl|pr|m|mx|my|mt|mb|gap|space-x|space-y)-([0-9]|0\.5)\b"
```
Run this alongside the `[Npx]` grep.

## 2. Several Matt tokens already encode weight/tracking — dropping the redundant utility is zero-change
**Topics:** typography, conformance, matt-design-system

**Context:** /courses backfill. `text-hN` utilities and `text-body-medium-bold` carried extra `font-medium` / `tracking-[-0.352px]` classes.

**Learning:** `text-hN` already applies Matt's Medium weight (500) per `tokens-typography.css`; `text-body-medium-bold` already encodes `-0.022em` (= -0.352px @16px). So `text-hN font-medium` and `text-body-medium-bold tracking-[-0.352px]` are redundant — removing the extra utility is provably zero-change (DOM-confirmed: title letter-spacing stayed -0.352px after dropping the class). Same precedent as Conv-300 SocialPost.

**Pattern:** When tokenizing, check the token's own definition before adding/keeping a weight or tracking utility — the token may already provide it.

## 3. 16px-medium (16/500) has no exact §09 token
**Topics:** typography, matt-design-system, foundation-gap

**Context:** NotificationSettings toggle labels use `font-medium` with no size → computed 16px/500.

**Learning:** §09 offers `text-body-medium` (16/400) and `text-body-medium-bold` (16/600) but **no 16/500**. Legacy forms that used `font-medium` on 16px text have no exact token — each needs a per-case call (mint a 16/500, drop to 14/500 `text-body-default-medium`, or bump to 16/600). A real foundation gap the settings-form sweep will hit repeatedly.

## 4. Ported route shells can hide fully-unswept legacy-Tailwind islands
**Topics:** route-sweep, palette-fdn, legacy-migration

**Context:** RG-PROFILE's route shell is `@matt-inspired` and structurally ported (✅), but its 5 settings islands (`ProfileSettings`, `StripeConnectSettings`, `NotificationSettings`, `SecuritySettings`) are full legacy-Tailwind (`secondary-*`/`primary-*` ramps, `text-sm`, `font-semibold`, `bg-white`, `rounded-xl`).

**Learning:** PALETTE-FDN only colour-swept Matt-surface feed/course routes; the settings cluster was never touched. A "✅ ported" route-table status does NOT imply its island content is conformant. Each settings tab is a heavy 3-axis legacy→Matt restyle. Budget route sweeps accordingly — the island content, not the shell, is the cost.

## 5. Grep heuristics need a DOM spot-check before any "clean" claim
**Topics:** route-sweep, conformance, calibration

**Context:** Made and self-corrected an assessment error this conv (the RG-PROFILE "spacing clean group-wide" claim). The correction came from DOM-truth, not re-reading the grep.

**Learning:** Reinforces `feedback_dom_truth_over_screenshots` — grep heuristics are sizing aids, not proof. A "clean" conformance claim requires a DOM spot-check, never a grep alone. The `[Npx]`-only spacing grep is a recurring blind spot that would mislead any future route-sweep sizing.
