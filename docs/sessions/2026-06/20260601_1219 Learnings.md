# Conv Learnings - 2026-06-01

## 1. Matt's green "Get Started Now" pill IS `Button variant="course"`
**Topics:** matt-design-system, astro, primitives

**Context:** The precheckout CTA is a green pill (#e8f4df bg / #327d00 text, rounded-39px + chevron). Looked like it needed custom styling.

**Learning:** Those are exactly the `course-background` / `course-primary` design tokens — `Button variant="course"` renders the pill 1:1. No bespoke styling needed; the durable CTA path was "render EnrollButton's state machine through the Button primitive," not a new button.

---

## 2. The course `[...tab].astro` shell renders a persistent hero → mini-hero would duplicate
**Topics:** astro, matt-design-system

**Context:** Matt's precheckout frame has its own centered mini course-hero, but the tab shell already renders the full-width `CourseHeader` above every tab.

**Learning:** Hosting one component in two shells needs a `showHero` prop — `true` standalone (Matt 1:1), `false` in-tab (the shell's CourseHeader supplies identity). This is the general pattern for any "standalone + in-shell tab" component pair.

---

## 3. A named route file beats the `[...tab]` catch-all in Astro
**Topics:** astro, routing

**Context:** `/precheckout` (named file) and `/benefits` (tab segment) live in the same folder as `[...tab].astro`.

**Learning:** Astro routes a sibling `precheckout.astro` ahead of the rest-param `[...tab].astro`, so a standalone page and a catch-all tab coexist cleanly — `/precheckout` → the file, `/benefits` → the catch-all.

---

## 4. MattIcon (React) needs `className`, not `class`, even in Astro
**Topics:** astro, react, matt-design-system

**Context:** `astro check` flagged 5 `ts(2322)` errors on `<MattIcon ... class=...>`.

**Learning:** MattIcon is a `.tsx` React component; Astro's `class` shorthand only applies to native elements. Imported React components take `className` even when used in `.astro`. (tsc alone missed this — astro check caught it; reinforces the five-gate baseline.)

---

## 5. Implementation drift can be a legitimate decision input
**Topics:** routing-addressability, matt-design-system

**Context:** Conv 187 classified precheckout as non-addressable. The Conv 232 audit confirmed all three deep-link candidates still say "No."

**Learning:** The decision still flipped to addressable — because the *implemented* `CourseHeader` CTA had already been coded as an `<a href="/checkout">`, the frame is a full standalone view, and its funnel siblings are addressable. The strict addressability test was a Conv-187 tiebreaker for a page presumed transient; once the page was built as a real destination with a real link, that tiebreaker no longer governed.
