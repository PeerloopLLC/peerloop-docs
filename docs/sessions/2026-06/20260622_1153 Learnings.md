# Conv Learnings - 2026-06-22

## 1. Literal `*/` inside a JSDoc string closes the block comment
**Topics:** astro, testing

**Context:** TeacherCourseView's new doc-comment described the change as `secondary-*/primary-*`. The `*/` sequence terminated the `/** … */` block early; everything after parsed as broken code → a cascade of TS1109/TS1005 errors at the top of the file.

**Learning:** When a doc-comment needs to mention a glob/path containing an asterisk-then-slash, never write the bare `*/` sequence. Reword (`secondary/primary tokens`) or space it (`* /`). The `tsc` gate caught it immediately; a leak-grep would not have.

## 2. Thin-shell dashboards: conform the shell + only the not-yet-conformed children
**Topics:** astro, cc-workflow

**Context:** CreatorDashboard (and TeacherDashboard before it) is a thin composition shell over many sub-components. 4 of CreatorDashboard's children (DashboardStatCard, EarningsOverview, QuickActionButton, MyFeeds) were already conformed by earlier /teaching + /learning sweeps.

**Learning:** Before conforming a dashboard-style island, enumerate its children and check which are already swept (grep for `dark:` / `secondary-` in each). The unit = shell + the role-specific children only. Saves re-touching shared, already-clean components and avoids accidental churn.

## 3. Role-gated routes need a role-appropriate dev-login account
**Topics:** testing, cc-workflow

**Context:** `/creating` is gated by `useCreatorGate` (users.`can_create_courses`). Marcus (a teacher) can't see it. Found seeded creators via D1 (`can_create_courses=1`): gabriel-rymberg@example.com etc.

**Learning:** For DOM-verify of a gated surface, resolve the gate's data source first (here a users column, not a role string or a separate table) and dev-login as a matching seeded account. Query D1 for the flag rather than guessing which seeded user qualifies.
