# Conv Learnings - 2026-06-16

## 1. Intelligent terse summaries can't be delegated to a hook
**Topics:** cc-workflow, docs-infra

**Context:** Designing the `conv-turns.md` re-orientation log; the user flagged a hook might already do "some of this."

**Learning:** A shell hook can deterministically capture the user's raw prompt, but it cannot produce an *intelligent* terse summary of CC's reply, extract the selected `AskUserQuestion` choice value, or distinguish a meaningful turn from noise (slash-command plumbing, bare yes/no). The valuable part of the request was exactly the part only CC can do — so a CC-maintained live-sync file (mirroring the trusted `conv-tasks.md` pattern) was the right architecture, not a hook.

**Pattern:** When deciding hook-vs-CC for a per-turn artifact, ask "does the value depend on judgment/summarization?" If yes → CC-maintained with a CLAUDE.md per-turn discipline; if it's pure deterministic capture → hook.

## 2. "Terse" scoped to the reply only
**Topics:** cc-workflow

**Context:** First draft of `conv-turns.md` made both the question and the reply terse.

**Learning:** The user's spec was "access these last questions and a terse form of your reply" — terseness applies only to the reply; the question must be captured in full (verbatim) so it's faithfully recall-able. Re-read the literal wording rather than symmetrizing the two halves.
