## 1. Reframe RTMIG-4 from a porting backlog into a 14-group visual-presentation route sweep
**Type:** Strategic
**Topics:** matt-design, cc-workflow

**Trigger:** User found no Tier-2 home for ported pages + no per-page Tier visibility.

**Options Considered:**
1. Add a single cross-cutting Tier-2 task
2. Per-page Tier-2 tasks
3. Group by route; routes = sweep units; every route swept (even ported); Tier assessed per-route ← Chosen

**Decision:** RTMIG-4 is now a full visual-presentation sweep of every route, organized into 14 `RG-*` route-group tasks. The unit is a route's `.astro` page but scope = the whole rendered page + every subcomponent. Porting is just one kind of per-route work.

**Rationale:** Matches the user's intent (route as the lens for visual assessment); exhaustive coverage; "done" routes still get swept.

**Consequences:** README rewritten as the full-sweep checklist; 27→26 tasks restructured into 14 groups + cross-cutting tasks; relaxes the Conv-219 "Tier-2 = one deferred cross-cutting pass" framing.

## 2. Canonical 8-step per-route sweep process (exhaustive, not hurried)
**Type:** Strategic
**Topics:** cc-workflow, matt-design

**Trigger:** User: document the process so a new conv can resume the multi-conv task; value comprehensive assessment.

**Decision:** Per route: (1) assess Tier-1, (2) assess Tier-2 via `/w-prim-candidates`→ledger (log ALL candidates), (3) surface, (4) PAUSE (no code till cleared), (5) do work (ripe extractions), (6) browser-verify, (7) user out-of-scope review → `[<ROUTE>-FIXES]` task, (8) mark Swept. Swept = Tier-1 done + Tier-2 fully assessed + browser-verified + out-of-scope captured; un-ripe ledger candidates don't block Swept.

**Rationale:** Multi-conv-resumable; user-gated; nothing skipped, only deferred-with-a-record.

**Consequences:** Documented in `plan/route-migration/README.md` § Working protocol + `feedback_route_sweep_pause_protocol.md`.
