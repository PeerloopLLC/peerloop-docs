# Conv Learnings - 2026-06-14

## 1. CD-039 RFC was written against the pre-migration component map
**Topics:** route-migration, rfc, refactoring

**Context:** Phase 4 named `FeedAllTab`/`FeedRoleTab`; Phases 5/6 named `Explore*Tab`/`Community*Tab`. All turned out to be legacy `/old/discover/*` components.

**Learning:** When an RFC names specific components, verify they're still **live** before converting — the root migration may have superseded them. The live equivalents here were `FeedsDirectory` (Phase 4) and the catalog components (Phases 5/6 = already done). Provenance comments ("legacy X parity") confirm supersession.

**Pattern:**
Grep to distinguish live from legacy:
```
grep -rln <Component> src/pages | grep -v /old
```
Per-phase disposition table (RFC target / live status / disposition) surfaced before acting let the user ratify the "subsumed" call rather than CC editing `/old` or silently skipping.

## 2. /feeds has no standalone filter island → placeholder branch, not event-bus split
**Topics:** astro, design-translation, refactoring

**Context:** LIST-1COL learning #5 says monolithic-filter surfaces → event-bus split (the /members precedent). /feeds looked monolithic (FeedsDirectory holds tabs+search internally).

**Learning:** Role tabs are **navigation coupled to counts**, not relocatable filters. The /communities precedent kept role tabs inline and moved only the standalone search/sort island. A surface with no standalone filter island uses the "else light-blue placeholder" right-panel branch — do NOT force an event-bus split just to populate the panel. Encoded as PLAN learning #6.

## 3. Phase-1's "cards already use aspect-video" prediction held for the whole sweep
**Topics:** design-translation, testing

**Context:** Phase 7 was scoped as an image-frame standardization sweep across all card components.

**Learning:** The catalog cards' `stacked` variant already used `aspect-video` + `object-cover`; members use avatars, feeds use icon tiles. The only real Phase-7 deliverable was upload-guidance copy. A "sweep" phase can collapse to near-nothing when an earlier phase established the convention — verify before assuming work remains.

---

## 4. `.astro` components are unit-tested by source assertion, not rendering
**Topics:** testing, astro

**Context:** Phase 8 called for a `ListingShell` unit test; the repo has no AstroContainer rendering pattern.

**Learning:** The established precedent (onboarding.test.ts) reads the `.astro` file with `readFileSync` and asserts on source content. For `ListingShell` this is genuinely useful — it locks in the exact mobile-contract class combos (filled panel `order-1 lg:order-2` not `hidden`; placeholder `hidden lg:block`) whose violation caused the Phase-1 bug. Following the precedent kept it non-novel (no new test infra).
