# Conv Learnings - 2026-06-21

## 1. DOM-truth requires selecting the RIGHT node, not just reading the DOM
**Topics:** plato, testing, cc-workflow

**Context:** A `/reset-password` probe read `border-radius:0px` on `input[type="email"]`; this was escalated to a 🟠 flag AND written into the conformance ledger as a "future form/Input conformance pass" — implying a real defect that didn't exist.

**Learning:** The `Input` primitive is **wrapper-styled**: the rounded border + padding live on the outer `<div data-prov-name="Input">`, and the inner `<input>` is deliberately `bg-transparent` with no radius. Probing the inner element reads `0px` while the field renders rounded (12px DOM-confirmed on /login: wrapper `border-radius:12px`, border-default, padding 12px/16px). DOM-truth (`feedback_dom_truth_over_screenshots`) is about trusting the DOM over screenshots — but it still demands measuring the *correct* element. For wrapper-styled primitives the codebase hands you the right handle: `[data-prov-name="X"]`.

**Pattern:**
Before escalating a computed-style anomaly to a flag/ledger entry, probe the primitive's provenance-stamped root (`.closest('[data-prov-name="…"]')`), not the leaf — and confirm the inference with one extra measurement rather than reasoning "swept routes use it so it must be fine."

## 2. `.scratch/conv-tasks.md` is machine-local — a cross-machine switch makes it look like data loss
**Topics:** cc-workflow, dual-repo

**Context:** r-start's Step 7.5 no-shrink reconciliation tripped: the existing companion was Conv 309 (30 tasks) vs the new 27.

**Learning:** `.scratch/` is gitignored + machine-local, so on a machine that sat idle for several convs the companion is frozen at its last local conv while the *other* machine advanced the backlog. The "shrink" isn't loss — it's staleness across a machine switch. The reconciliation correctly required every dropped code to be explained; here all 7 mapped to completed/converged work across Convs 306–313 (RG-MOD/STALE-TESTS Conv 313, RG-COMMS Conv 311, MEM-CONSOLIDATE/MEM-PRUNE/SEGPILL Conv 309, RIPE Conv 306), verified via `plan/COMPLETED.md` + git log, so it reconciled-and-proceeded.

**Pattern:**
When the companion's header conv is several behind the current conv, expect a large shrink and verify dropped codes against git history / COMPLETED.md rather than halting.

## 3. Bridge-shrunk spacing is a recurring port hazard
**Topics:** astro, cc-workflow

**Context:** PasswordResetForm's success-icon `h-16 w-16`/`h-8 w-8` rendered 16/8px; /verify `p-8`/`h-16 w-16`/`mt-2`; /404 `mt-4`/`mt-2`/`mt-8`/`gap-4`.

**Learning:** Legacy pages written in standard-Tailwind intent (where `h-16`=64px, `p-8`=32px) silently shrink under the `@theme --spacing-*` bridge (which aliases the numeric scale to Matt literal-px: `h-16`=16px). Every unported/legacy surface carries this latent shrink; the port must restore intent (`size-[64px]`, `p-32`, `mt-16`) — same class of finding as RG-MOD Tranche A.
