# Conv Learnings - 2026-06-21

## 1. Astro `[param]` route dirs collide with git pathspec glob syntax
**Topics:** astro, dual-repo, cc-workflow

**Context:** Committing the /creator sweep, `git add 'src/pages/creator/[handle]/index.astro'` (quoted against the shell) silently failed to match.

**Learning:** Git interprets `[handle]` in a pathspec as a glob character-class (one char from {h,a,n,d,l,e}), which never matches the literal `[handle]` directory. An unmatched pathspec makes `git add` exit non-zero **without staging any** of its arguments — so the commit captured only the pre-staged `git mv` rename, none of the content edits. The first commit (`c6dffa1b`) was a rename-only no-op; caught via `--stat` and fixed by re-staging the parent dir + `--amend`.

**Pattern:**
Stage bracketed Astro route paths via the parent dir (`git add src/pages/creator/`) or `:(literal)` pathspec magic; always `--stat`-verify a commit touching `[param]` paths.

## 2. Carry-over claims are hypotheses — verify counts/footprints before acting
**Topics:** cc-workflow, testing

**Context:** The carry-over described CourseCard as an "18-consumer" component "rendering legacy on already-swept /courses + /course/[slug]."

**Learning:** Reality: the shared `courses/CourseCard.tsx` has **5 consumers / 6 usages** (the other ~13 `*CourseCard` hits are distinct components), and it renders **live only on /creator** — /courses uses `CourseCatalogCard`, and /course/[slug]'s related-section was empty. An in-code breadcrumb (`RecommendedCourses.tsx:9` "used by 5 other surfaces") corroborated the smaller count. DOM-truth (`rounded12Count: 0` on /courses) instantly falsified "renders there."

**Pattern:**
Treat a hand-off number as a hypothesis; confirm with grep + DOM before scoping conformance to it.

## 3. Grep finds strings; only reading distinguishes a value from a type
**Topics:** testing, cc-workflow

**Context:** Confirming `CourseDetail.tsx` was dead while `<CourseDetail` appeared in production `src`.

**Learning:** The `<CourseDetail` hits in CoursesAdmin/CourseEditor were `useState<CourseDetail | null>` — the **type** `CourseDetail` imported from `./CourseDetailContent` (name collision), not the dead default-exported component. A default-exported React component can't be used as a type, so the collision is unambiguous once read.

**Pattern:**
When a grep "use" decides a delete, open the reference and confirm value-vs-type before concluding live.

## 4. The RG-WORKSPACES ⛔ protects components the new page is made of
**Topics:** astro, cc-workflow

**Context:** /learning's StudentDashboard renders EnrollmentCard/CertificatesSection/MyFeeds, all of which the protected UnifiedDashboard (/old/dashboard) also renders (via Merged* + direct).

**Learning:** Unlike CourseCard (whose blast radius shrank on inspection), here the comparison target and the sweep target are the *same components* — the workspaces and the frozen composite were built from a common card/section vocabulary. So "sweep the islands" can't cleanly proceed while the comparison is active; the islands' content *is* the comparison. (UnifiedDashboard's own Merged* wrappers are separate, which is why the composite shell itself stays safe.)

**Pattern:**
Before sweeping a workspace island, check whether it (or its children) renders on a comparison-frozen route.

## 5. Chrome bridge auto-redacts JS-result keys containing "Token"
**Topics:** cc-workflow, testing

**Context:** A DOM-assertion result key named `forbiddenLegacyTokens` returned `[BLOCKED: Sensitive key]`.

**Learning:** The bridge's secret-redaction matches the substring "Token" in result keys and blanks the value. Renaming to `legacyClassHits` returned the data.

**Pattern:**
Avoid "token"/"key"/"secret"/"password" in `javascript_tool` return-object key names when the value is non-sensitive.
