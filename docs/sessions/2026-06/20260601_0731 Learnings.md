# Conv Learnings - 2026-06-01

## 1. A half-migrated structure rots from the writers, not the readers
**Topics:** cc-workflow, docs-infra

**Context:** Conv 228 split DECISIONS.md → docs/decisions/ but left the decision-authoring skills pointing at the old file.

**Learning:** When a structural migration moves the *readers* (the file becomes a pointer) but not the *writers* (skills still append to it), the next write silently lands in the dead location and desyncs the new structure. The fix is to make the correct write-path the skill's default, not to rely on a manual-rerouting note.

## 2. Read the config *consumer* before editing config that "names" a file
**Topics:** cc-workflow

**Context:** The DEC-SKILL-SYNC task named `.claude/config.json` as needing changes, but its `DECISIONS.md` refs were timecard/registry recognition, not write-targets.

**Learning:** Reading `timecard-day.js` revealed `routineStrip.pathPatterns` (substring match) + `docPathsExclude` (extraction filter) as the right keys — and that `docNameWhitelist` only matches ALL-CAPS stems, so lowercase chunk filenames don't belong there. Pattern-matching the config without reading the consumer would have produced cargo-cult additions.

## 3. INDEX anchor bug: 0 orphaned decisions, 22 broken links
**Topics:** docs-infra

**Context:** User asked whether the Conv-228 split orphaned decisions in the index.

**Learning:** Distinguish orphaned *decision* (title with no content = data loss) from malformed *anchor* (content present, link misfires). The Conv-228 INDEX-generator stripped `_`/`.`/special chars from anchors, but GitHub's rendered anchors keep underscores → 22 misfiring links, 0 lost decisions. One-shot generator artifact, no recurrence risk (decisions are now hand-authored).

## 4. TERM-GARBLE leaves no recoverable transcript trace — detection is architecturally external
**Topics:** cc-workflow

**Context:** Attempted a deterministic `/w-garble-check` over the JSONL transcript.

**Learning:** Calibration against the known-garble Conv 226 transcript failed on BOTH a payload scan and a narration scan — the JSONL persists the reconciled (clean) exchange. Worse, batch metrics anti-discriminate (Conv 226 scored "cleaner" than Conv 229). The symptom (live-render blanks + #63538 confabulation) is structurally invisible from inside the model: the component that would self-report "I was garbled" is the one that got garbled. Only external/ground-truth verification works. **Don't re-attempt a transcript scanner.**

## 5. The no-shrink backstop fires every r-start by design
**Topics:** cc-workflow

**Context:** /r-start Step 7.5 flagged conv-tasks.md (35 rows) > TaskList (33).

**Learning:** Because completed rows stay in conv-tasks.md with a `*DONE*` prefix while TaskList carries only pending tasks, the shrink is the steady state. Verify `oldRows − #*DONE* == TaskList`; only halt on mismatch. Don't pause-and-ask on the benign case.
