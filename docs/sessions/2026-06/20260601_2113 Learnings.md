# Conv Learnings - 2026-06-01

## 1. Astro template expressions can't contain bare `<`/`<=` — hoist comparisons to frontmatter
**Topics:** astro, frontend

**Context:** ESLint error 1002 "Unable to assign attributes when using <> Fragment shorthand syntax" on the `canJoin = now <= end` comparison inside a `.map()` in the MySessionsTab template.

**Learning:** Inside an Astro `{...}` template expression, a bare `<` (comparison, or TS generic) is ambiguous with a JSX tag-open and the compiler mis-parses it. Keep template expressions trivial (`.map(row => <jsx>)`); push comparisons/generics/date-math up into the `---` frontmatter (precompute `upcomingRows` with the `canJoin` flag).

**Pattern:**
`const rows = items.map(s => ({ s, flag: <comparison> }))` in frontmatter; template iterates `rows.map(({s,flag}) => ...)`.

## 2. Tailwind alpha modifiers on token colors are fragile — use solid tokens
**Topics:** astro, frontend

**Context:** Used `bg-course-primary/10` and `bg-student-background/40` in MySessionsTab.

**Learning:** Alpha-opacity modifiers on CSS-var-backed theme colors can silently fail to apply. The codebase's established soft-circle background is the solid `bg-student-background` token (ModulesTab, success page) — prefer it over alpha-modified brand colors.

## 3. The `npm test` runner is vitest, not jest
**Topics:** testing

**Context:** `npx jest tests/...` produced a babel parse error + 0 tests (fetched a fresh jest without the project transform).

**Learning:** Run tests via `npm test -- <path>` (= `vitest run <path>`); `npx jest` bypasses the project config. Test DB = better-sqlite3 (3 migrations). Full suite = 371 files / 6460 tests / ~3min.

---

## 4. Browser verification catches what gates can't — assert pathname inside DOM probes
**Topics:** testing, browser, cc-workflow

**Context:** The two review bugs both lived behind an authed enrolled session (no gate exercises that path). During verification a navigation race returned a false "NO MySessionsTab" against the wrong page.

**Learning:** Authed render verification is load-bearing for enrolled/role-gated paths. Inside browser DOM probes, assert `location.pathname` first so a navigation-race / stale-page artifact is distinguishable from a real failure (dom-truth-over-screenshots discipline).
