# Conv Decisions - 2026-06-18

## 1. Home-only RIGHT panel, bespoke in index.astro, accept the 1248px cap
**Type:** Implementation
**Topics:** astro, layout, matt-design

**Trigger:** Client markup wants a right-side "coming soon" panel pushing the feed left.

**Options Considered:**
1. Enable ListingShell panel as-is (renders LEFT)
2. Home-only RIGHT placement ← Chosen
3. Flip the global panel side to RIGHT for all listing pages

**Decision:** Home-only right panel, built bespoke in `index.astro` (not extending the shared shell); accept that panel growth caps at AppLayout's 1248px content-card.

**Rationale:** Diverging Home keeps the 4 filter pages on the Conv-289 LEFT decision untouched; bespoke keeps the shared shell single-purpose (its flex model is the inverse of what's wanted); the 1248 cap matches the client's working width.

**Consequences:** `index.astro` now has a bespoke two-column layout; documented as a Home-only divergence.

## 2. Mint a 14px paragraph type token
**Type:** Strategic
**Topics:** matt-design, typography

**Trigger:** App body text is 14px but Matt's 14px token is a caption (lh 1.0); paragraphs read cramped.

**Options Considered:**
1. Mint `text-body-default-prose` (14px/lh-1.5) ← Chosen
2. Migrate all paragraphs to Matt's `text-body-medium` (16px) — upsizes app body text
3. Change `text-body-default` line-height — breaks ~67 legit caption/label uses

**Decision:** Mint the 14px prose token as a CC-ownership scale extension.

**Rationale:** Preserves the app's 14px density, fixes leading, keeps Matt's caption regime intact; fills a genuine gap Matt's frames never covered.

**Consequences:** New token + bridge; §09-typography documents the role discipline; ~99 `<p>` misuses migrate onto it over time.

## 3. TYPO-FDN ledger: component-level SoT, route completion derived
**Type:** Strategic
**Topics:** cc-workflow, docs-infra

**Trigger:** User's concern that deferred shared-component work has no completion mechanism.

**Options Considered:**
1. Route/page-level checklist — misses shared components
2. Component-level rows, route completion derived ← Chosen

**Decision:** `plan/typo-fdn/migration-ledger.md` with one row per component, routes cross-referenced, per-axis checkboxes; a route is swept only when all its components are ☑.

**Rationale:** Shared components tracked once; no route can claim done while a shared piece is ☐; the audit becomes a standing committed obligation.

**Consequences:** 23-row ledger; the model retroactively applies to colour (`[FAC-RECOLOR]` now a red row).

## 4. Unified feed-card spec + no-arbitrary-px-for-spacing rule
**Type:** Implementation
**Topics:** matt-design, typography

**Trigger:** Home cards of several component origins present inconsistently; user wants unified font/size/lineheight/padding/margins and "drop all px" for spacing.

**Decision:** Canonical card = `p-16` + `rounded-12` + per-slot type tokens (title `text-body-medium-bold`, body `text-body-default-prose`, label/meta `text-body-small(-medium)`, rhythm `mt-4`/`mt-8`/`gap-12`). Margin/padding/gap use scale classes only — no arbitrary `[Npx]`; off-scale values snap or are flagged as sanctioned optical exceptions. Locked into §9.4a.

**Rationale:** One contract all card components conform to; ledger checks each against it.

**Consequences:** Off-scale px (gap-[10/6px], mt-[2px], m-[76px]) become explicit snap-or-flag decisions per component.
