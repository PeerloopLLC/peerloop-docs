# Conv Learnings - 2026-06-19

## 1. Verify-before-counting catches the `primary-*` homonym trap
**Topics:** matt-design, conformance, route-sweep

**Context:** Both messages and notifications surfaced ~12 apparent legacy `primary-*` / `primary-light` utilities that looked like sweep violations.

**Learning:** Two unrelated things share the spelling "primary": Matt's semantic `--Primary-Default` (americana-blue `#0777B6`, the Student/Text-Primary role — a *valid* role token, ledger-blessed) and legacy Tailwind's numeric `primary-600/50` sky scale (what the settings islands carried, migrated to `brand-*`). A grep can't distinguish them; only resolving the token through `tokens-semantic.css`→`tokens-tailwind-bridge.css` does. Counting first would have produced a dozen phantom findings and a destructive recolor.

**Pattern:**
Resolve every flagged token to its semantic backing before counting it as a violation — `grep -nE '\-\-Primary-Default' src/styles/`.

## 2. `<Button>` adoption is colour-neutral for americana-blue primaries
**Topics:** matt-design, primitives

**Context:** Worried adopting `<Button>` for the hand-rolled primary buttons would flip americana-blue → brand-purple.

**Learning:** Button's `primary` variant is `bg-text-primary` → `--Text-Primary` → `--Primary-Default` → americana-blue `#0777B6` — the *same* colour the hand-rolled `bg-primary-default` buttons used. So the primitive swap is colour-neutral (DOM-confirmed r39px / `rgb(7,119,182)`). The app's standard primary-action colour IS americana-blue; brand-purple `#584DF4` is the separate `course` accent.

## 3. Honest-orphan content colours stay on Tailwind defaults
**Topics:** matt-design, conformance

**Context:** NotificationCenter's per-type icon tints (`text-blue-500 bg-blue-50` … ×18 types).

**Learning:** When Matt's palette has no role scale for a content-differentiation feature (notification type), the honest-orphan principle keeps that feature on the default Tailwind palette while the shell uses Matt tokens. The file documents the intent; a sweep leaves it untouched (C-keep). Same shape as the `primary-*` blessing — "leave it alone" came from recorded intent, not guesswork.

## 4. Cross-route Tier-2 accumulation ripens a primitive
**Topics:** matt-design, primitives, route-sweep

**Context:** The All/Unread filter row is the same hand-rolled `PILL_BASE/PILL_ON/PILL_OFF` grammar on SmartFeed, messages, and notifications.

**Learning:** A single-route sweep can't see a cross-cutting primitive's Rule-of-Three; the Tier-2 ledger carries the per-route count across routes until it ripens. Notifications was the 3rd inline sighting → RoT met → SegmentedPills converge is now ripe ([SEGPILL] #33). A cross-cutting converge that touches already-swept routes is tracked as its own task (backward-pointer re-glance), not folded into the single-route commit.

## 5. Chrome bridge REPL needs an async IIFE
**Topics:** plato, browser-testing

**Context:** `javascript_tool` errored on top-level `await` despite the tool description claiming it works.

**Learning:** Wrap async DOM-truth probes in `(async () => { … })()` returning the result — top-level await is not actually supported in this REPL context. Settle-then-read (~1.5s sleep inside the IIFE) per [BRIDGE-MEM].
