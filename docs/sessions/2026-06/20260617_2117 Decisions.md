## 2. Remove /precheckout; keep the catch-all redirect (not 404)
**Type:** Implementation
**Topics:** astro

**Trigger:** User: /precheckout is a subnavbar-experiment remnant; /benefits is the home.

**Options Considered:**
1. Delete route + repoint CTA; let unknown tabs **redirect** to course root (existing behavior) ← Chosen
2. Delete + force a hard 404 for unknown tab segments

**Decision:** Delete `precheckout.astro`, repoint CourseHeader CTA → /benefits, leave the `[...tab].astro` catch-all's 302→course-root behavior untouched.

**Rationale:** The course is real, only the tab is invalid — a soft landing (URL rewrites to `/course/[slug]`) matches the user's stated requirement and is consistent for all unknown course-tab segments; a hard 404 would need a special-case for worse UX. Distinct from route-honesty (which targets masking *unconverted* routes, not normalizing a malformed sub-segment of an existing route).

**Consequences:** RG-COURSES dropped 5→4 routes; `/precheckout` → 302 → course root.
