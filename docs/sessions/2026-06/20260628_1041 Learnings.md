# Conv Learnings - 2026-06-28

## 1. "Backend-ready, UI-only" must be verified per-actor, including the enumeration path
**Topics:** api, auth, homework, planning

**Context:** Conv 345's carry-over framed `[HW-GRADE-UI]` as a pure-frontend task ("no React UI consumes the reviewer endpoints"). But the two reviewer endpoints (`GET …/submissions`, `PATCH …/submissions/[subId]`) are **per-assignment** — they take an assignment id. The only endpoint that *enumerates* a course's assignments (`GET /api/courses/[id]/homework`) is hard-gated to enrolled students (`courses/[id]/homework.ts:62-70`), so a teacher (not enrolled) had no way to discover assignments to grade.

**Learning:** A "backend-ready" claim covers the *action* endpoints but not necessarily the *navigation/enumeration* path the UI needs for the acting role. Before accepting a UI-only scope, trace the full read path the actor's role must traverse — list → detail → action — and confirm each hop is reachable under that role's auth predicate.

## 2. Chrome-bridge: coordinate clicks no-op on hydrated islands; native `.click()` works
**Topics:** plato, browser-testing, dom-truth

**Context:** During live DOM verification on `:4321`, `computer left_click` (both by ref and by computed center coordinate) on the `TeacherCourseView` tab buttons silently failed to switch tabs — the panel stayed on Overview. A native `element.click()` dispatched via `javascript_tool` switched the tab immediately.

**Learning:** Under the persistent dev server's parallel island hydration, synthetic coordinate clicks can land before/outside the React handler, while a native `.click()` bubbles to React's delegated root listener and fires reliably. For island interaction verification, prefer `javascript_tool` `element.click()` + DOM-read over coordinate clicks. Complements `[BRIDGE-MEM]` (which covers stale-cache / dev-login settle) with a distinct click-method gotcha.

**Pattern:**
```js
(async()=>{ const el=[...document.querySelectorAll('button')].find(b=>b.textContent.trim()==='Homework'); el.click(); await new Promise(r=>setTimeout(r,1200)); return document.querySelector('main').innerText.slice(-500); })()
```

## 3. `homework_submissions` has UNIQUE(assignment_id, student_id)
**Topics:** database, testing, homework

**Context:** The new endpoint test seeded two submissions for one assignment from the same student to exercise `submission_count=2 / ungraded_count=1`, and hit `UNIQUE constraint failed: homework_submissions.assignment_id, homework_submissions.student_id`.

**Learning:** One submission per (assignment, student). Tests covering multiple submissions on a single assignment must use distinct students (each with its own enrollment row). Resubmission overwrites the existing row rather than inserting.
