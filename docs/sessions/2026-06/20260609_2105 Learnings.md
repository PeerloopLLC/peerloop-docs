# Conv Learnings - 2026-06-09

## 1. Self-gating island makes placement nearly free
**Topics:** astro, role-studios, components

**Context:** Phase 4 needed nudges on 5 different surfaces, some (Home, CourseDetail) with no server-side viewer context.

**Learning:** Making `ProgressionNudge` read the client `useCurrentUser()` store itself and return `null` when ineligible meant each placement was a 2-edit drop-in (one import + one tag) with zero prop-threading or SSR auth plumbing. Pulling placement ① forward cost exactly two edits because the eligibility logic was already wholly inside the component.

**Pattern:** Self-gating client island (mirrors TriageStrip / OnboardingNudgeBanner): `client:only="react"` or `client:load`; component computes eligibility from the store and renders nothing for ineligible viewers.

## 2. A "locked" design can rest on a falsified premise
**Topics:** role-studios, process

**Context:** Decision C ("simplest two: ② + ③") assumed `CourseDetail.tsx` was the live course view.

**Learning:** It was orphaned dead code (last touched Conv 050; the live view is `course/[slug]/[...tab].astro`). The "easy" placement ② and the "deferred-hard" placement ① were secretly the same surface. Verifying mount-status of a design's named target before building catches this; the diagnosis cost more than the real placement.

## 3. The /chrome bridge live-DOM can diverge from served HTML
**Topics:** plato, testing, cc-workflow

**Context:** Verifying client-gated nudge renders in-browser.

**Learning:** The bridge's `document.body` view did not match the authenticated `fetch()` HTML for the same URL (0 astro-islands live vs 17 in the fetch) — likely View-Transition / hydration timing. For verifying client-gated islands, D1 user-classification + manual login (or curl of served HTML) is the reliable method; trust the bridge DOM with suspicion here. Complements `feedback_dom_truth_over_screenshots` (which trusts DOM over screenshots) — but the *bridge's* DOM snapshot itself can be stale.

## 4. Inferring "visitor" from absence-of-feature is a trap
**Topics:** testing, process

**Context:** Saw no nudge on Home, labeled the tab "visitor."

**Learning:** The tab was logged in as Sarah Miller (a teacher); Home correctly hid the S→T nudge because she's already a teacher. "No feature rendered" has many causes — confirm the actual auth state before naming it. The user caught the mislabel.
