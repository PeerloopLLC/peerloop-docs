## 2. The smart-feed integration test is the high-value guard, not the lib unit tests
**Topics:** testing, smart-feed

**Context:** Announcements are pinned atop the feed via 3 orchestrator edits (parallel fetch, union widen, prepend).

**Learning:** The pin contract ("pinned, first-page-only, never in the cursor") lives in the orchestrator wiring, not the lib functions. The integration test that asserts `activities[0]` is the announcement, `nextCursor` keys off a sample-post, and page-2 (with a `before` cursor) has no announcement is what locks that contract. Lib unit tests alone wouldn't catch an orchestrator regression.

## 3. Chrome innerText reflects CSS text-transform
**Topics:** testing, cc-workflow

**Context:** A DOM check for the literal "Announcement" label returned false even though the card rendered.

**Learning:** Chrome's `element.innerText` is layout-aware and returns the CSS-`uppercase`-transformed text ("ANNOUNCEMENT"), so a case-sensitive regex on a `uppercase`-styled label fails. Use case-insensitive matching for label-text assertions, or assert on the source via `textContent`.
