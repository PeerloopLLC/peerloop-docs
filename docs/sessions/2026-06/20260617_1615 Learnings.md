# Conv Learnings - 2026-06-17

## 1. Provenance-scoped colour sweep reveals scale of the gap
**Topics:** matt-design, tokens, colour

**Context:** PALETTE-FDN reframed to sweep only `@matt-source`/`@matt-inspired` surfaces.

**Learning:** The Matt surfaces carry **3,708 Tailwind-default colour-util occurrences across 198 files**, full 50→900 ramps across ~10 hue families — vs Matt's 15 disjointed primitives with no numeric scales and no status hue. The "disjointed gradation system" is real and large; the colour foundation is a multi-conv effort whose per-file migration rides the RT-MIG sweeps.

---

## 2. Anchor-via-var() makes mint-vs-map legible in source
**Topics:** tokens, css

**Context:** Wiring the derived scales into the 3-layer cascade.

**Learning:** Authoring each ramp step as either `var(--matt-primitive)` (map) or a literal hex (mint), with an inline `(M)`/`mint` marker, encodes the mint-vs-map decision directly in `tokens-primitives.css` — and keeps the cascade intact so tuning a Matt primitive propagates.

**Pattern:**
```
--neutral-100: var(--gray-100); /* (M) */   vs   --neutral-50: #FBFBFB; /* mint */
```

## 3. Tailwind v4 JIT tree-shakes unused theme color vars
**Topics:** tailwind, tokens, verification

**Context:** DOM-probing `bg-neutral-300`/`bg-brand-100` returned transparent / empty `--color-*`.

**Learning:** v4 only emits `--color-{role}-{step}` to `:root` (and generates the utility) when a *source file* uses that exact class. Bare classes no component uses won't resolve — not a wiring bug. Verify via classes actually in source, or read the underlying `--{role}-{step}` primitive.

---

## 4. Legacy-survivor greps must match numeric ramps, not Matt semantics
**Topics:** sweep, tooling, matt-design

**Context:** A `[class*="primary-"]` probe flagged 15 "legacy survivors" on `/`.

**Learning:** Matt's semantic tokens `primary-default`/`primary-light` contain the substring `primary-` and false-positive against a legacy-ramp search. The legacy ramp is `(primary|secondary)-[0-9]+` (numeric); exclude `primary-(default|light)`. ~11 of the 15 were false positives.
