# Conv Learnings - 2026-06-12

## 1. Salience failures need a channel change, not more prose or a detector
**Topics:** cc-workflow, docs-infra

**Context:** The inline A)/B) question rule lived in always-loaded CLAUDE.md + MEMORY.md yet failed ~once/conv for ~10 convs; Conv 272 added a QLINT Stop-hook to detect it.

**Learning:** A rule that's *known but not retrieved at the authoring moment* is a salience failure. Proactive prose decays; a post-hoc Stop-hook is only a partial detector (no hook point exists at question-authoring time). The robust lever is changing the **channel** — route decisions through `AskUserQuestion` so the malformed shape is unrepresentable (the user selects rather than types). Residual risk = remembering to invoke the tool (same salience dependency), accepted as the tradeoff.

## 2. Premise-check claims must be verified, not asserted from a narrow grep
**Topics:** testing, cc-workflow

**Context:** Claimed "no rating system in schema" from a grep scoped to `level|member_count|icon` during the FEED-U2 premise-check gate; user pushed back.

**Learning:** A wider grep found `courses.rating`/`rating_count` + `course_reviews` + an established `X.X (N reviews)` label-format pattern used across course surfaces. A premise-check assertion that gates a design decision (hide-vs-show) must be grounded in a real search, not a partial one. The user's "research why we couldn't use the course rating" caught a fidelity bug before it shipped.

## 3. The Edit/Write path mangles `\u` escapes into literal control bytes
**Topics:** cc-workflow

**Context:** A regex `/[^ -]/` written via Write landed as a broken regex with embedded NUL/DEL bytes (Read rendered them as spaces; `awk` revealed the break).

**Learning:** Avoid unicode escapes in tool-authored source; use an escape-free equivalent (e.g. `charCodeAt(i) > 127`). Verify suspicious content out-of-band (`awk`/`od`) rather than trusting the normalized Read render.

**Pattern:**
```js
// instead of a /[^ -~]/ style regex written via Write,
// test code points directly:
if (str.charCodeAt(i) > 127) { /* non-ASCII */ }
```
