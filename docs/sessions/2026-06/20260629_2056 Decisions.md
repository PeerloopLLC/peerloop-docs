# Conv Decisions - 2026-06-29

## 1. Adopt the spt CURRENT-TASKS.md task-persistence model (decision B)
**Type:** Strategic
**Topics:** cc-workflow, docs-infra

**Trigger:** conv-tasks.md ballooned from a throwaway current-conv reading aid into a semi-persistent backlog mirror, and its machine-local nature caused stale-on-return friction.

**Options Considered:**
1. A — re-separate: shrink conv-tasks.md back to ephemeral; persistence stays with RESUME-STATE + PLAN
2. B — adopt the spt CURRENT-TASKS.md model ← Chosen
3. C — patch the seams: keep the split, fix the machine-switch reconciliation only

**Decision:** Replace the split (RESUME-STATE-Remaining + machine-local conv-tasks.md) with one git-tracked, hand-editable, persistent `CURRENT-TASKS.md`; demote RESUME-STATE.md to narrative-only; retire conv-tasks.md. (DEC-350-1)

**Rationale:** One git-tracked file is the same on both machines, ends the machine-switch staleness + the no-shrink guard, and gives a persistent readable "what I'm working on" surface the machine-local file could never provide cross-machine.

**Consequences:** Multi-conv block `[CURTASKS]`. Phase 1 (design + seed) done Conv 350; the skill cutover is Phases 2–5.

## 2. Active-only TodoWrite hydration
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** Deciding how `/r-start` should populate TodoWrite under the new model.

**Options Considered:**
1. Active-only — r-start leaves TodoWrite empty; TaskCreate (reusing the [CODE]) when an item is started ← Chosen
2. Hydrate-all — load all pending CURRENT-TASKS.md items into TodoWrite (today's behavior)

**Decision:** Active-only (matches spt). CURRENT-TASKS.md is the backlog; TodoWrite shows only in-flight work. (DEC-350-2)

**Rationale:** Cleaner spinner / less parked-task noise; keeps the mid-conv TodoWrite delta small (which also minimizes the checkpoint-refresh crash-loss window).

**Consequences:** `/r-start` Phase-2 rewire drops the RESUME-STATE→TodoWrite transfer.

---

## 3. Checkpoint-refresh, not live-sync
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** conv-tasks.md was live-synced (crash mirror); CURRENT-TASKS.md needs a refresh cadence.

**Options Considered:**
1. Checkpoint-refresh at /r-commit, /r-end, /r-update-tasks ← Chosen
2. Live-sync (preserve the crash mirror)

**Decision:** Checkpoint-refresh (matches spt). (DEC-350-3)

**Rationale:** Simpler; `/r-checkpoint` covers explicit mid-conv saves; active-only hydration keeps the at-risk delta small. Flagged as revisitable at cutover.

**Consequences:** A mid-conv crash loses the in-conv task delta (acceptable trade).
