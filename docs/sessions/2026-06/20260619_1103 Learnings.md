# Conv Learnings - 2026-06-19

## 1. `text-body-default leading-normal` → `text-body-default-prose` (not a leading-drop)
**Topics:** typo-fdn, conformance, astro

**Context:** Hub paragraphs used `text-body-default leading-normal`. The first instinct was "drop the redundant `leading-normal`."

**Learning:** `text-body-default` alone is the **dense** caption token (line-height 1.0); the `leading-normal` (=1.5) was faking prose. Dropping it would *tighten* prose to 1.0. The conformant fix is the prose token `text-body-default-prose` (14/1.5) — exact-equivalent, zero visual change. Applies across the hub.

## 2. Early-swept routes carry a conformance-backfill debt
**Topics:** rtmig-4, route-sweep

**Context:** User expected `/course/[slug]` to be unswept; it had actually been swept Conv 295.

**Learning:** Routes swept before Conv 299 were done to the **legacy-port** standard; the formal 4-gate (the TYPO-FDN ledger) post-dates them and was back-applied only to `/` + `/courses`. A "done" stamp reflects the standard *at sweep time* — so an early route can carry unpaid conformance debt without it being "deferred work." A swept route is not automatically a conformed route.

## 3. Exception taxonomy — 3 kinds, only one is a judgment call
**Topics:** conformance, exceptions

**Context:** User demanded the full exception inventory plus the reasoning per item.

**Learning:** "Exception" had been over-applied. Three distinct kinds exist:
1. **Out-of-scope** — the gate doesn't govern it (radii, sizes, sub-4px nudges, image scrims/fallbacks, dynamic icon names).
2. **No-token holds** — could be tokenized but no matching token exists, and inventing one is unjustified (`text-white`/`bg-white`, emoji glyphs, person-name `#000` per SocialPost precedent).
3. **Judgment calls** — the only genuinely discretionary ones (the About-heading `text-black` and the 💬 emoji).

Naming the kind sharpens what "consciously allowed" actually means per item.

## 4. MattIcon static names ARE registered — the "unregistered icons" are emoji
**Topics:** icons, matticon

**Context:** A first grep diff falsely reported all 17 hub MattIcon names as missing from the registry.

**Learning:** Scripting bug (multiline `$registered` var + `grep -qx`). The registry (`components/icons/svg/*.svg`, 59 names, globbed by `MattIcon.tsx`) contains all 17. The genuinely-"unregistered icons" the user meant are the **emoji-as-icons** (👍 💕 have no glyph; 💬 has `chat`/`video-comment` but stays emoji for reaction-bar consistency). Sanity-check an "everything is broken" grep against the known-good browser-verified state before alarming.

## 5. AnalyticCount `#f6f6f6`/rgba is a documented @matt-source signature
**Topics:** conformance, primitives, palette-fdn

**Context:** ReviewsTab had hand-rolled the AnalyticCount Default chrome with raw hexes.

**Learning:** `bg-[#f6f6f6] border-[rgba(88,77,244,0.14)]` lives *inside* AnalyticCount as Matt's deliberate muted-state signature (no exact neutral token; `#f6f6f6` sits between neutral-50 `#F8F8F8` and neutral-100 `#F1F1F1`). The correct fix is to **adopt the primitive** — the signature is a recorded exception owned once at the primitive for all consumers — rather than hand-tokenize each call site.
