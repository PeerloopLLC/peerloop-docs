# Conv Learnings - 2026-06-06

## 1. A prose spec can lag the actual Figma frame — probe the node first
**Topics:** matt-provenance, figma

**Context:** The plan called `[SUCCESS-COMMUNITY]` a "composer-only mode of MattCourseFeed." Probing node `729:15940` showed a *different* shape: milestone meta row + course embed inside the composer + distinct copy, no feed list, no "Post Something" h2.

**Learning:** When a plan describes a Matt feature by analogy to an existing component, the analogy was written before (or without) seeing the frame. Probe the node before committing to "extend X" — the divergence drove the new-focused-component decision over extending MattCourseFeed.

---

## 2. Generated registries self-heal pre-existing drift
**Topics:** matt-provenance, tooling

**Context:** Adding `MilestoneComposer`'s `data-prov-name` stamp made prov:sweep go 5→6 (UNTRACKED). Fix was `npm run gen:registries` (the registry is generated from `@matt-source` markers, not hand-edited).

**Learning:** Regenerating dropped the count to 4, not back to 5 — it swept in `PrecheckoutContent`, which had a marker but a stale registry (generator hadn't been re-run since it was added). Running the generator is strictly better than hand-editing: it fixes your change AND any latent drift hand-editing would miss.

## 3. `.entity-student` cascade is the Student palette (#0777b6 / #f1f9ff)
**Topics:** matt-design-system, tokens

**Context:** The design UserIcon is primary-blue on light-blue; needed to know which cascade class yields that rather than guessing.

**Learning:** `tokens-semantic.css` comment confirms `.entity-student` / `.entity-student-teacher` → `--Student-Primary #0777B6` / `--Student-Background #F1F9FF`. MattCourseFeed already wraps its composer in `entity-student` for the same reason — reused the pattern.

## 4. Enrolled student can post to the course feed → real composer viable on /success
**Topics:** feeds, permissions

**Context:** Deciding real-functional vs decorative for the milestone composer.

**Learning:** `api/feeds/course/[slug].ts:339` returns `canPost: true` for an enrolled student (status 'completed' → graduate, else student). The success page self-heals the enrollment row server-side before the client island calls the API, so a just-enrolled buyer qualifies — no new endpoint, real posting works.
