# Conv Learnings - 2026-06-07

## 1. driftCheck docs split into two lifecycles — projection vs prose
**Topics:** docs-infra, cc-workflow

**Context:** #22 (route docs) kept reappearing as a task despite a docs-drift system existing.

**Learning:** A doc that's a pure projection of code (route maps) and a hand-written doc about code (API param tables) both landed in `driftCheck`, but only the former is mechanically regenerable. Conflating them is why projections leak out as human tasks. The fix is a distinct `generated` category with an executable regen binding — projections are regenerated, never TaskCreate'd.

---

## 2. The `generated` category existed but was toothless
**Topics:** docs-infra

**Context:** Expected to invent a new category; found `generated` already declared (`config.json:358`, "Produced entirely by a tool, on-source-change") with one passive user (`skill-generated`).

**Learning:** It was a passive marker — r-end Agent 3 treats `generated` as out-of-scope ("do not edit"), and there was no regen binding. Reclassifying route docs to `generated` as-is would make them rot *harder*. The missing piece was an executable binding + a gate, not a new category.

## 3. Ambient-auth suppression must match resolved basename, not import string
**Topics:** testing, cc-workflow

**Context:** Phase-1 scanner fix to trace `@lib` fetch; first attempt suppressed `current-user` by checking the import string `@lib/current-user`.

**Learning:** `current-user.ts` is also reached *transitively* via other libs' relative imports (`./current-user`), which bypass the `@lib/` guard. Suppressing by the **resolved file basename** catches every entry path. Verified: `/api/me/full` from current-user dropped to 0 attributions (the 2 remaining are a genuine direct call in `/old/index.astro:106`), while `useCanMessage`→`/api/me/can-message` correctly surfaced (14×).

## 4. Structural consumers absorb generated-fixture changes; snapshot consumers don't
**Topics:** testing

**Context:** Regenerating `route-map.generated.ts` with new @lib-traced API attributions; worried about the test suite.

**Learning:** Test count held at exactly 6458 because the sole consumer (`tests/plato/lib/navigation-helper.ts`) reads `routeMap`/`apiToRoutes` structurally (BFS), not via snapshot assertions. A snapshot-based consumer would have broken on the count change.
