## 2. RG-MOD category badges — hybrid colour policy
**Type:** Implementation
**Topics:** matt-design, testing

**Trigger:** The 3 badge helpers (`getReason/Priority/ContentTypeBadgeClass`) map 9 hues, several with no Matt role token (orange/purple/indigo/cyan/pink).

**Options Considered:**
1. Hybrid: priority→status tokens; reason/content-type map clean hues + keep orphan hues ← Chosen
2. Honest-orphan keep all 9 (zero mapping, max distinguishability)
3. Collapse all to role/status tokens (loses category distinguishability)

**Decision:** Priority → status tokens (urgent=error/high=warning/normal=info/low=neutral). Reason + content-type: map clean hues (red→error, yellow→warning, gray→neutral), keep orphan hues (orange/purple/indigo/cyan/pink) as documented honest-orphans (the `/notifications` per-type-tint precedent).

**Rationale:** Priority is genuinely status-like and maps cleanly; reason/content-type are category cues where forcing role tokens would collapse distinct categories to the same colour.

**Consequences:** Mapped hues use role tokens; 5 orphan hues remain raw + commented as honest-orphans.
