# Conv Learnings - 2026-06-22

## 1. Legacy "sky" collapses two Peerloop blues — conformance must split by role
**Topics:** matt-design, conformance, color-tokens

**Context:** The /learning plan said "sky `primary-*`→`brand-*`" but ledger precedents diverged (/profile mapped sky→brand-purple; auth-modal mapped sky→`text-primary-default`).

**Learning:** Peerloop has TWO blues that legacy `primary-*` (sky) conflated: **americana-blue** (`primary-default` — the interactive/link/CTA colour) and **brand-purple** (`brand-*` #584DF4 — the accent scale). Conformance must classify every sky usage by role: interactive → americana (`<Button>` for buttons, `text-primary-default` for inline links), decorative → brand (the only tokenized fill option, but it shifts the hue purple). The decorative case has no americana background scale, so brand is effectively forced for tinted fills.

**Pattern:**
Per sky occurrence, ask: is it a button/link (→ americana) or a tint/fill/badge (→ brand)?

## 2. The sparse neutral/brand scales force a re-grade, not a literal map
**Topics:** conformance, color-tokens

**Context:** A literal `secondary-200`→`neutral-200` map would emit broken classes — those scale steps aren't registered.

**Learning:** The registered scales are sparse — neutral {50,100,300,500,700,900}, brand {100,300,500}. A literal same-number map breaks for 200/400/600/800. The pinned re-grade (from DOM-verified precedent): `secondary-{200,300,400}`→`neutral-300`, `600`→`neutral-500`, `700`→`neutral-700`, `900`→`neutral-900`; decorative `primary` tint-bg→`brand-100`, solid-fill→`brand-300` (#584DF4), text/icon→`brand-500`.

## 3. Spacing conformance is the uniform transform X-N → X-(N×4)
**Topics:** conformance, spacing, matt-design

**Context:** The Conv-174 bridge aliased only the Tailwind keys {4,8,12,16,20,24,32,40,48,64} to literal-px (so `--spacing-4`=4px), shrinking those ~4×; out-of-set keys still render N×4 px normally.

**Learning:** Every Tailwind spacing utility `X-N` *intends* N×4 px regardless of the bridge. So the single conformance transform is `X-N`→`X-(N×4)` using the literal-Matt class (number = px): it restores collapsed in-set utilities (`p-4`→`p-16`) AND zero-change-normalizes the rest (`gap-6`→`gap-24`). Fractional keys (`mt-0.5`) stay as sub-scale optical keeps.

## 4. DOM-truth verifies token RESOLUTION — gates can't
**Topics:** conformance, dom-truth, chrome-bridge

**Context:** All static gates passed, but a non-existent token computes to `rgba(0,0,0,0)` and still passes tsc/lint/astro-check (the Conv-291 `bg-primary` trap).

**Learning:** Reading `getComputedStyle` for `bg-brand-100`→`#ECEBFE` proves the token is wired. Once one brand step resolves, the same-mechanism siblings (brand-300/500, neutral-*) resolve too. This is the one thing browser-verify adds over the gates, and why step 6 catches CSS bugs the gates miss.

## 5. A "tab" can be a full unit — scope the composition tree before bulk-editing
**Topics:** decomposition, route-sweep, planning

**Context:** /teaching overview looked like a 291-line file, but TeacherDashboard is a thin shell over 8 sub-components (~914 ln, one shared across 10 consumers).

**Learning:** Composition-shell components hide the real surface area in their children. Scoping (line counts + legacy density + consumer counts per sub-component) BEFORE editing reveals the true unit size and the cross-cutting back-glance obligations — preventing a "small tab" from silently ballooning mid-edit.
