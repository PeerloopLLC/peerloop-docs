# Conv Learnings - 2026-06-30

## 1. MEMORY.md auto-load cap: bytes bind before lines, and there is no import lever
**Topics:** cc-workflow, docs-infra, memory

**Context:** [MEM-CAP-ARCH] diagnosis, cross-checked against `code.claude.com/docs/en/memory.md`.

**Learning:** The SessionStart auto-load cap is "first 200 lines OR first 25 KB, whichever comes first" — for this index, bytes bind (86%) far before lines (62%), because the index is written as marker-rich mini-summaries (median 286 B/line). Content past the threshold is silently not loaded. Critically: **only MEMORY.md auto-loads; the 82 sub-files load on-demand**, and `@path` imports are a CLAUDE.md-only feature that load at launch without reducing context — so there is no mechanism to auto-load more than 25 KB of index. The only durable lever is making the 25 KB window carry more value per byte.

## 2. The "lean index → on-demand detail" structure is not unique to auto-memory
**Topics:** cc-workflow, docs-infra

**Context:** User pointed out the index could live in CLAUDE.md without loading the sub-files.

**Learning:** Plain markdown links `[link](file.md)` are inert — only `@path` syntax triggers auto-import. So the index could equally live in CLAUDE.md (uncapped) with sub-files still loading on-demand; net startup context would be unchanged (~22 KB either way). The real reasons to keep the index in MEMORY.md are not technical-capacity but: (a) framing — MEMORY.md is delivered as "background context", CLAUDE.md as "instructions", and a situational-recall index is genuinely background; (b) avoiding adherence-dilution of CLAUDE.md's actual behavioral rules; (c) auto-write integration / drift. Decision landed on keeping it in MEMORY.md for the framing reason.

## 3. Measure a sample before projecting byte budgets
**Topics:** cc-workflow

**Context:** Projected ~13.5 KB for the rewrite; delivered 18 KB (71%).

**Learning:** The projection underweighted that HOT entries are the *longest* lines by design (the always-on rules carry the most detail) and overweighted how terse COLD lines could be while still preserving the load-bearing marker. A sample rewrite of ~5 representative HOT and COLD lines, measured, would have caught the ~35% gap before the user approved against a wrong number. Surface byte estimates as ranges, or measure first.
