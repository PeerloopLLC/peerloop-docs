# Conv Learnings - 2026-06-24

## 1. Route audits must check what RENDERS, not just inbound links
**Topics:** routing, route-migration, astro

**Context:** The user was repeatedly confused about `/feed` vs `/feeds` vs course/community feeds; the first answers described entry links, which didn't resolve it.

**Learning:** Clarity came from mapping what each route *renders* and its data source. `/feed` renders the *same* `<SmartFeed>` island as Home (`/api/feeds/smart`) — a literal duplicate, confirmed by the middleware already redirecting `/feed`→`/` for visitors. The code itself carried a "Distinct from the other two feed surfaces (recurring confusion)" comment. When a route's value is unclear, enumerate {what renders, data source, who else renders it} before judging.

## 2. Legacy-surface links don't prove a route earns its place
**Topics:** routing, navigation-architecture

**Context:** Argued `/feeds` was a "primary nav destination" partly because AppNavbar + DiscoverSlidePanel link it.

**Learning:** Under [OLD-RETIRE-DEFAULT], AppNavbar (and the AppNavbar-hosted DiscoverSlidePanel) are legacy — the canonical Sidebar deliberately dropped `/feed`+`/feeds` (Conv 258). Verifying *which shell* a linker belongs to (canonical AppLayout+Sidebar vs legacy AppNavbar/old) is essential before crediting a link as "value." DiscoverSlidePanel is only *referenced in a comment* by the canonical Sidebar, not rendered by it.

## 3. Verify out-of-band on contradictory grep results (ugrep quirk)
**Topics:** tooling, testing

**Context:** A combined-category legacy-token grep returned 0 for admin, while a per-file probe with the same pattern returned 170.

**Learning:** The environment aliases `grep`→`ugrep`, which parsed the combined pattern differently and silently returned 0. A direct per-file probe confirmed the real debt (admin is heavily legacy, ~1470 hits). On contradictory grep output, re-probe directly rather than trusting either run.

## 5. `find -name "*.astro"` matches directories
**Topics:** tooling, cruft

**Context:** Flagged `src/pages/api/communities/.astro` + `.../node_modules/.astro` as "phantom route files."

**Learning:** They were *directories* (`.astro` build-cache + a stray nested `node_modules`), not files — `find -name` matches dirs too. Both were gitignored local cruft (no git/build/deploy impact). Investigate before alarming on a "phantom route" finding.
