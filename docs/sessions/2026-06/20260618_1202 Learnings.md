# Conv Learnings - 2026-06-18

## 1. "Route Swept" ≠ style-guide conformant — they are separate gates
**Topics:** route-migration, design-system, cc-workflow

**Context:** User asked whether Home + /courses were "caught up" on Tier-1/2 + colour + typography/spacing. The route sweep marks routes ☑ Swept; TYPO-FDN tracks per-component conformance — easy to conflate.

**Learning:** The route sweep's "Swept" = *Tier-1 done + ripe Tier-2 extracted* — a deliberately bounded pass that **defers** fine-grained type/spacing/colour conformance. RG-HOME (Conv 291) and RG-COURSES (4/4, Conv 297) are Swept yet most of their components are non-conformant (TYPO-FDN 3/23). The fix this conv: make conformance a **4th "Swept" gate** so the two notions converge — conformance now "rides the sweep" exactly as PALETTE-FDN's per-route colour migration already did.

## 2. VS Code "lose all but N tabs" is default preview-editors, not an editor limit
**Topics:** cc-workflow, dev-env

**Context:** User lost all but ~3 editor tabs when opening Markdown Preview; suspected a "max 3 files" setting.

**Learning:** `workbench.editor.limit` was unset everywhere (user/profile/workspace/.code-workspace). The cause is the default `workbench.editor.enablePreview:true` — files opened from links/Quick-Open are *transient* preview tabs (italic) that get replaced; the survivors are the ones promoted to permanent (double-click/edit). Fix: `enablePreview:false` + `enablePreviewFromQuickOpen:false`. For deep (non-root) files, **Cmd+P fuzzy filename** beats tree-hunting (depth-independent, searches all workspace folders).

## 3. The Conv-174 bridge collapse hides as wrong-but-plausible spacing across many utilities
**Topics:** design-system, route-migration

**Context:** Migrating Home components surfaced multiple collapsed utilities.

**Learning:** The `tokens-tailwind-bridge` `@theme --spacing-*` remap of the Matt set `{4,8,12,…,64}` to literal px makes Tailwind-numbered classes render at *literal* px: `p-4`→4px (not 16px), `space-y-4`→4px, `h-4 w-4`→4px, **and width too — `w-20`→20px (not 80px)**. Conformance migration must both convert to scale classes AND restore the intended value, often with coupled changes (e.g. card `p-4`→`p-16` forces the image-strip bleed `-my-4`→`-my-16`). Classes *outside* the set (`gap-6`, `space-y-6`) fall back to the Tailwind multiplier (24px), so they're not collapsed — the trap is set-membership-specific.

## 4. Coarse accent scales need a deterministic nearest-step tie-break
**Topics:** design-system, palette

**Context:** FeedActivityCard's colours span full Tailwind ramps; PALETTE-FDN accents are only `{100,300,500}` (neutral is fine: `{50,100,300,500,700,900}`).

**Learning:** Map each tone to the nearest available step, **round-half-up** on ties (200→300, 400→500, 600→700) — backgrounds keep lightness, saturated/badges go to 500. Saturated buttons hit a ceiling (no step darker than 500) → use `hover:opacity-90` instead of a darker step. This makes the mapping reproducible rather than per-util guesswork.

**Pattern:**
Round-half-up nearest-step mapping for coarse accent scales; saturated buttons that need a darker hover use `hover:opacity-90` since no step exists below 500.

## 5. text-body-default-bold is not a real token (latent broken class)
**Topics:** design-system

**Context:** ProgressionNudge's inline variant used `text-body-default-bold`.

**Learning:** That token isn't defined in `tokens-typography.css` / the bridge (defined: default, default-medium, default-prose, large, large-medium, medium, medium-bold, small, small-medium). The class silently no-ops. Fixed → `text-body-default-medium`. Worth a grep sweep for other phantom `text-body-*` variants ([TYPO-PHANTOM]).
