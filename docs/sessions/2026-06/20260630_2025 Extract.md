### Page responsiveness / sub-466px overflow

**User:** "Something odd going on the the <nav> CSS and its container. The Hero too … the text in the hero is not bound in its container. We may need to reflow this page … overflows spilling out below about 466px width" → picked "Shrink inline" for the hero image → "It works well below 375."

**Claude:** Bridge-investigated (constrained content to 343px): confirmed real overflow (`main` → 691px). Root cause = the **community header**: a fixed 224px cover image (`w-56`) + a text column with `min-width: auto` (stuck at 160px min-content) can't fit below ~466px. Reflow: `min-w-0` on the text column + image-group, image `w-[96px] min-[480px]:w-56`, `flex-wrap` on the name/meta rows; plus `min-w-0` on the nav grid items. **Learned the constrain-main test is misleading for responsive elements** (media queries key off viewport, stuck at 558/1163 on the bridge) — verified instead by *forcing the mobile state* (96px image + grid + min-w-0): hero 0 overflow, nav 0 overflow. Then applied the analogous (but different) fix to `CourseHeader` — its overflow is the two-column `justify-between` row + `px-64` padding, not an image: `px-24 py-32 sm:px-64 sm:py-40` + content row `flex-col gap-24 sm:flex-row`. Forced-state verified 0 overflow. User confirmed the community page works below 375.
