# Conv Decisions - 2026-06-30

## 1. Global toggle for SubNav placement
**Type:** Implementation
**Topics:** navigation, astro

**Trigger:** User wants to try top-at-all-sizes but "always go back" to the desktop rail.

**Options Considered:**
1. Global toggle (shared constant) ← Chosen
2. Hard flip (delete rail)
3. Breakpoint shift
4. Runtime UI toggle

**Decision:** A single build-time `SUBNAV_LAYOUT` constant imported by AppLayout + SubNav; flip to `'left'` restores the rail everywhere in one line.

**Rationale:** Cheapest reversibility; both code paths kept; no per-page prop threading.

**Consequences:** `src/lib/subnav-layout.ts` is the single source of truth; the zoned journey is excluded via a `!zoned` guard + per-page `subNavLayout="left"` pins.

**Type:** Strategic
**Topics:** navigation

**Options Considered:**
1. A. Exclude (keep rail)

**Consequences:** 4 course pages carry `subNavLayout="left"` until Phase 2; [SNAV-TOP] Phase 2 queued in CURRENT-TASKS.

## 3. Lift persistent entity headers into the `entity-header` slot
**Type:** Implementation
**Topics:** navigation, layout

**Trigger:** In top mode the persistent header wedged between the tabs and the per-tab content.

**Decision:** Move the community header Card and `CourseHeader` into AppLayout's unused `entity-header` slot (above `sub-nav`). Per-tab title headers stay in content.

**Rationale:** That slot exists for exactly this ("Entity Header (Course/Teacher/…)"); gives `[hero][tabs][content]` in both modes.

**Consequences:** Applies to community + course now; a general pattern for persistent entity headers.

## 4. Mobile tabs: equal-width 2-col grid fill (not wrap, not scroll, not dropdown)
**Type:** Implementation
**Topics:** navigation, css

**Trigger:** 5 tabs overflow a phone row; wrap looked ragged/tall; scroll hides tabs.

**Decision:** Mobile `grid grid-cols-2` with tabs filling their cells (`w-full`), collapsing to the content-width `md:flex` row on desktop.

**Rationale:** Fills the bar (no dead space), all tabs visible, tidy aligned; desktop untouched.

**Consequences:** Still 3 rows for 5 tabs at 375px (acceptable to user); depth-vs-labels tradeoff noted.
