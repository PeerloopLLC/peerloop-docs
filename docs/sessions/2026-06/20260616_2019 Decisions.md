## 1. conv-turns.md maintenance mechanism + ordering
**Type:** Implementation
**Topics:** cc-workflow, docs-infra

**Trigger:** User requested a `.scratch/conv-turns.md` re-orientation log of recent turns (question + terse reply / selected choice).

**Options Considered:**
1. CC-maintained live-sync (like conv-tasks.md) ← Chosen — intelligent summaries + clean choice values; best-effort reliability via CLAUDE.md discipline
2. Hybrid hook + CC — UserPromptSubmit hook captures the raw question deterministically, CC adds the reply; never misses a question but logs noise + new hook infra
3. Deterministic transcript-parse — fully automatic, but mechanical truncation summaries

**Decision:** CC-maintained, **newest-first** ordering (latest entry at the top for instant re-orientation).

**Rationale:** Only CC can deliver the intelligent terse reply + selected-choice value the user asked for; CC-maintained follows the established conv-tasks.md precedent (not novel). Newest-first avoids scrolling to find the latest.

**Consequences:** Created `.scratch/conv-turns.md`; added CLAUDE.md "Conversation Turn Log" per-turn discipline; wired `/r-start` Step 7.7 (seed fresh each conv) and `/r-end` Step 2 (read as COLLECT cross-check).

## 2. vernacular.md format — 3-column living glossary
**Type:** Implementation
**Topics:** cc-workflow, docs-infra

**Trigger:** User asked for a `.scratch/vernacular.md` acronym lookup; later asked to expand it and reformat.

**Decision:** Single alphabetical markdown table, **3 columns** (Acronym · Literal meaning · Context/comments), terse 1–2 lines each, append-as-we-go, **not** conv-scoped (persists/accumulates across convs, unlike conv-turns.md).

**Rationale:** Single alphabetical table is Ctrl-F friendly; the 3rd column carries the project-specific context (e.g. SOP = the RTMIG-4 route-sweep process) that distinguishes our usage from the generic expansion.

**Consequences:** Created `.scratch/vernacular.md` (~38 terms); added CLAUDE.md "Vernacular Glossary" pointer.
