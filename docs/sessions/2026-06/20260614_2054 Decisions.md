# Conv Decisions - 2026-06-14

## 1. /feeds right-panel = placeholder branch (role tabs stay inline)
**Type:** Implementation
**Topics:** astro, design-translation

**Trigger:** Phase 4 — deciding what goes in `/feeds`'s ListingShell right panel.

**Options Considered:**
1. Empty light-blue placeholder; role tabs + search inline ← Chosen
2. Event-bus split to relocate role tabs/search into the panel
3. Convert grids only, no ListingShell

**Decision:** Use the "else placeholder" branch; keep role tabs inline (no `right-panel` slot).

**Rationale:** Matches the /communities precedent (role tabs never moved; only standalone filter islands relocate). /feeds has no standalone filter island. An event-bus split of data-coupled role-tabs-with-counts would be large and fragile for no design payoff. Follows established pattern → not a novel decision.

**Consequences:** `/feeds` shows the light-blue placeholder panel on desktop; role tabs + per-tab search remain in the column.

## 2. Phases 5 & 6 marked subsumed, not implemented
**Type:** Strategic
**Topics:** route-migration, rfc

**Trigger:** Phase 5/6 targets (`Explore*Tab`/`Community*Tab`) found to be legacy-only.

**Options Considered:**
1. Mark subsumed → Phase 7 (record in PLAN + check off RFC, legacy excluded by route-honesty) ← Chosen
2. Convert legacy `/old` components too
3. Stop for user to verify

**Decision:** Record 5 & 6 as done-by-subsumption; live equivalents (`CoursesCatalog`/`CommunitiesCatalog`) were converted in Phases 2/1.

**Rationale:** Editing `/old/*` violates route-honesty and would put new design into pages kept for old-vs-new comparison. The live discover-courses/communities surfaces are root `/courses` and `/communities`, already single-column.

**Consequences:** LIST-1COL effectively jumped Phase 4 → Phase 7. RFC grid items checked off with subsumption notes. User ratified.
