## 2. Promotion password-gate policy (4 clarifications)
**Type:** Strategic
**Topics:** promote-pipeline, auth

**Trigger:** PROMOTE-PIPELINE #31 had 4 OPEN password clarifications blocking the build.

**Options Considered (each resolved to its recommendation):**
1. Gate scope: every escalation step ← Chosen (vs first step only)
2. Secret count: one global password ← Chosen (vs per-feed/per-role)
3. Entry timing: per-promotion ← Chosen (vs session state)
4. Storage: bcrypt hash in `platform_stats` (`promotion_gate_password_hash`) ← Chosen

**Decision:** One global password · entered per-promotion · bcrypt hash in `platform_stats` · every escalation step gated.

**Rationale:** Without payment the password is the sole anti-spam control (gate every step); one global secret matches "distribute to trusted promoters"; per-promotion avoids session state; reuse bcrypt (`src/lib/auth/password.ts`) + `platform_stats` (no new infra).

**Consequences:** #31 password-gate unblocked + built. Recorded RESOLVED in spec README + client-meeting doc.
