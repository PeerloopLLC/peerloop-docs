# Conv Learnings - 2026-06-27

## 1. Regenerating a deterministic generator IS the verification
**Topics:** testing, astro, docs-infra

**Context:** ROUTEGEN-NAV swapped the hardcoded nav model in two route-doc generators (`scripts/route-matrix.mjs`, `scripts/route-api-map.mjs`). Rather than predict every downstream nav-path change by hand, the generators were re-run and the output diffed.

**Learning:** For deterministic generators, the regen-diff is ground truth — it surfaced exactly one behavioral change beyond the rename (`/creating/apply` → honest "no discovered path" orphan) and many shortened paths (adding `/` as a direct `[Sidebar]` node lets the BFS reach `/`-rooted routes in one hop). Predicting these by hand would have been error-prone; diffing is exact.

**Pattern:**
Edit the generator → run it → `git diff` the generated artifacts → assert the change set matches intent. Never hand-edit generated output.

## 2. PLATO is two systems under one name, with very different health
**Topics:** testing, plato

**Context:** "Is PLATO working after 100+ dormant convs?" was ambiguous until the two layers were separated.

**Learning:** **API mode** (scenarios = chains of steps, run by `tests/plato/api/plato-scenarios.api.test.ts`) is part of `npm test` and was green (10/10) — it proves the data model, personas, endpoints, and SQL `verify` queries still hold. **Browser mode** (instances' `walkthrough[]` BrowserIntents, walked manually via the Chrome bridge) is the only part that rotted, and only at its entry-nav waypoints. Separating the layers dissolved the question and produced a "refresh, not rebuild" verdict.

**Pattern:**
When assessing a dormant test system, first find which layer runs in the automated suite — that layer's health is already known; focus assessment on the manual layer.

## 3. A live browser-run reveals page-behavior drift that a nav-string audit cannot
**Topics:** testing, plato, cc-workflow

**Context:** The PLATO instance audit found stale *nav waypoints*. The live member-directory run found more.

**Learning:** Walking `/members` live (DOM-probed, not screenshotted) surfaced drift the static audit missed: the page now defaults to the Creator role filter (not "show all"), and role chips are additive multi-select (OR), not single-select. These are `expect:`/`pageAction:` drift, distinct from nav drift — a third bucket only a live run exposes.

**Pattern:**
Audit instance nav statically to scope the mechanical fixes, but walk at least one instance live to catch destination-page behavior changes.
