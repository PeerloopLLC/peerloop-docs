# Conv Decisions - 2026-06-08

## 1. MOVE-not-copy `/old` policy — reverses the Conv-221 keep-/old-live rule
**Type:** Strategic
**Topics:** cc-workflow, dual-repo, astro

**Trigger:** Matt's phase-out (Conv 239) caused files to yo-yo between `/old/*` and root. The user questioned why `/old/*` copies are still kept once the pre-flip worktree (`peerloop-ref`) and git history both exist.

**Options Considered:**
1. Relax → **move** legacy pages to the target route (delete the `/old` copy) ← Chosen
2. Keep the Conv-221 rule (copy, leave `/old` live as rollback)
3. Decide per-cluster

**Decision:** When porting/rehosting a legacy page, `git mv` it from `/old/*` to its target route rather than copying. The `/old` copy is not retained. Recorded in `memory/project_old_pages_no_delete_until_vetted.md` (+ MEMORY.md index line) and `plan/route-migration/README.md` (§ Migration policy). `[PREFLIP-WT]` re-scoped to "keep until client-vet."

**Rationale:** `/old` served two jobs and both are now covered better elsewhere — behavioral reference by the preflip worktree + git history, and live production rollback was already moot (nothing links to `/old`). Git history preserves the files regardless, so a parallel `/old` copy is throwaway weight. For *code* archaeology the live-repo copy beats the worktree (current imports, zero setup).

**Consequences:** Spawned `[OLD-PORTED-CLEANUP]` (#21) to delete the 44 already-ported `/old/*` copies now stale under the new policy. Future ports edit the moved file in place; the move commit becomes the behavior-diff baseline.

## 2. Admin cluster: rehost-now / Matt-port-late via move → `@stand-in`
**Type:** Strategic
**Topics:** astro, cc-workflow

**Trigger:** User needs the admin pages usable at `/admin/*` now (Matt redesign deferred to late in the migration plan) and asked what hurdles that poses vs leaving them at `/old/admin/*` or redirecting.

**Options Considered:**
1. Copy `/old/admin/*` → `/admin/*` (leave `/old` copies)
2. Move `/old/admin/*` → `/admin/*` as `@stand-in` ← Chosen
3. Rewrite/redirect layer

**Decision:** `git mv` the 14 `/old/admin/*` pages to `/admin/*` and mark each `@stand-in`; Matt-inspired retrofit happens later, in place.

**Rationale:** The app was ALREADY wired to `/admin/*` (middleware `PROTECTED_PREFIXES`, AdminNavbar 14 links, admin-links.ts 6, context-actions 3, AdminDashboardCard 5; **zero** `/old/admin` refs) — so the console was BROKEN at `/old/admin/*` (every nav/deep-link 404'd). "Leave in /old" was the *most* work (~30 throwaway link rewrites); redirect flips the URL + bounces; rewrite risks catch-all shadowing. Move-as-`@stand-in` fixes the links for free since the Matt shell is already mounted.

**Consequences:** Console restored; admin is cluster 1 of 8 tiered in `plan/route-migration/README.md`. Established the move→`@stand-in` rehost pattern (git mv + marker + commit-as-baseline, port later in place).

## 3. Admin-role gate belongs in middleware, not the layout
**Type:** Implementation
**Topics:** auth, astro

**Trigger:** `[ADMIN-REDIRECT-BLANK]` — a non-admin on `/admin/*` got a blank 200 because the `AdminLayout` role-redirect ran inside a nested component.

**Options Considered:**
1. Keep the role-redirect in `AdminLayout` (Conv-083 pattern)
2. Move the admin-role gate into `middleware.ts` ← Chosen

**Decision:** Move the admin-role check + redirect into `middleware.ts` after the session check; remove the dead auth block + unused imports from `AdminLayout`.

**Rationale:** A `return Astro.redirect()` inside a layout component halts that component's render into empty output rather than emitting the redirect Response — only middleware or page frontmatter can return the HTTP-level Response. Middleware already classified `/admin/*` as a protected prefix for the session check, so the role gate is the natural sibling there (clean 302 vs blank 200).

**Consequences:** Partially reverses Conv 083 "Admin Auth Guard in Layout, Not Pages" — the role-based redirect now lives in middleware; `middleware.test.ts` gained an admin-gate suite (dropped /admin from the student-pass list).

## 4. Sidebar selection = fixed-height white pill, no layout shift
**Type:** Implementation
**Topics:** matt-design, astro

**Trigger:** Sidebar lower menu items showed no selection state, and the active highlight (a `p-[20px]` card variant) resized the row and shifted everything below it.

**Options Considered:**
1. Fixed-size pill within the existing row box ← Chosen ("but keep the rows the same height")
2. Blue-text active indicator
3. Background tint

**Decision:** `NavItem` 'Selected' renders a white pill inside the existing row box with rounding + border reserved in all states and `border-box`, so the active row keeps the non-selected row height (no shift). `MainNav` routes childless-active items → 'Selected', exports `matchHref`; Sidebar bottom items compute Selected the same way.

**Rationale:** The shift came from the active `'Main'` variant being a padded card vs flat rows; reserving the pill geometry in every state removes the resize. A pill (vs blue-text / bg-tint) matches the existing upper-item selection language.

**Consequences:** Upper and lower sidebar items now share one selection treatment; later given `py-[6px]` reserved padding + gap tightened 24→18.

## 5. Inline the island-unique gap for VT-safety
**Type:** Implementation
**Topics:** astro, matt-design

**Trigger:** On `/profile`, all sidebar items reverted to old spacing until a click — the documented View-Transition island-unique-arbitrary-utility drop.

**Options Considered:**
1. Inline the value as `style={{ gap: '18px' }}` ← Chosen
2. Leave it as the `gap-[18px]` Tailwind class

**Decision:** Inline the 18px gap as an inline style on both MainNav and Sidebar containers.

**Rationale:** `gap-[18px]` lives ONLY in MainNav/Sidebar (unlike `py-[6px]` in Button or app-wide `px-[20px]`), and the sidebar is a `transition:persist` island, so its unique utility dropped from the destination route's CSS chunk on the VT. An inline style rides with the persisted DOM node and is immune to CSS-chunk drop.

**Consequences:** New pattern recorded — inline island-unique arbitrary utilities on `transition:persist` components.
