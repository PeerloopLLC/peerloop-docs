# Conv Decisions - 2026-06-06

## 1. Real-functional MilestoneComposer as a new focused component
**Type:** Implementation
**Topics:** matt-design-system, feeds

**Trigger:** `[SUCCESS-COMMUNITY]` — the plan's "composer-only mode of MattCourseFeed" wording vs the divergent Figma frame, plus the "decide function first" phase-out mandate.

**Options Considered:**
1. Real-functional, new focused `MilestoneComposer` reusing primitives + shared helper ← Chosen
2. Real-functional, extend `MattCourseFeed` with a `mode="milestone"` prop (plan's literal wording)
3. Decorative only (Matt happy-path, no real posting)

**Decision:** Option A — new `MilestoneComposer` island, posting to the existing course-feed endpoint via a shared `postCourseFeed()` helper.

**Rationale:** The frame diverges enough that a `mode` prop would bloat the shared Feed-tab component with milestone-only branching. A focused component + shared post helper keeps both lean. Real posting honors "never fake function"; the enrolled student qualifies (`canPost`), and no legacy behavior is lost (`/old` had only a static bullet).

**Consequences:** New `src/lib/feeds.ts` (`postCourseFeed`), adopted by MattCourseFeed too; new `MilestoneComposer.tsx`; `CourseEmbedCard` gained `showCta`.

## 2. `CourseEmbedCard` `showCta` extension (default true)
**Type:** Implementation
**Topics:** matt-design-system

**Trigger:** The milestone frame's embedded course card has no "Enroll Now" CTA (nonsensical post-enrollment); `CourseEmbedCard` always rendered one.

**Options Considered:**
1. Add `showCta?: boolean` (default true) ← Chosen
2. Inline a reduced embed in MilestoneComposer (duplicates the primitive)

**Decision:** Extend `CourseEmbedCard` with `showCta`, default true.

**Rationale:** Reuse over duplication (feedback_reuse_existing_components); default true leaves every existing call site untouched; semantically correct (you just enrolled).

**Consequences:** One extended prop; MilestoneComposer passes `showCta={false}`.
