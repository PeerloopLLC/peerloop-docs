# Conv Learnings - 2026-06-20

## 1. Pre-bridge legacy components have silently bridge-shrunk spacing
**Topics:** matt-design, spacing, conformance, route-sweep

**Context:** The 4 `Admin*` primitives predate the `@theme` spacing bridge (Conv 174). That bridge aliases the numeric Tailwind steps in `{4,8,12,16,20,24,32,40,48,64}` to literal px (`--spacing-16` = 16px), so a legacy `p-4` (written expecting Tailwind's 16px) renders as **4px**, `h-12`→12px, `h-16`→16px (skeleton bars/rows ~4× too small).

**Learning:** When sweeping a component authored before the bridge, the spacing isn't just "off-scale" — it's actively mis-rendering. The conformance fix is to convert every spacing/size utility to the **intended literal-px Matt class** (`p-4`→`p-16`, `h-12`→`h-48`, `h-16`→`h-64`, icon `w-4 h-4`→`size-16`). Numbers NOT in the aliased set (`gap-1`, `px-2.5`, `h-5`, `h-10`) render at Tailwind defaults — a mixed state, so check each utility's number against the aliased set.

**Pattern:**
Before converting, grep the `@theme --spacing-*` block to confirm the aliased set, and compare against a known-conformed sibling (here `CommunityMembersTab` uses `px-16`/`gap-12`).

## 2. A role-token *name* can hide a legacy survivor (verify-before-counting, both directions)
**Topics:** matt-design, colour, palette

**Context:** `ModeratorQueue` used `bg-primary-100`/`text-primary-600`. The Messages/Notifs precedent was that `primary-default`/`primary-light` ARE valid americana-blue role tokens. But `global.css` defines `--color-primary-100..600` as a separate **legacy sky scale** (`#e0f2fe`/`#0284c7`); only `--color-primary-default/light` are the role tokens.

**Learning:** Verify-before-counting cuts both ways: a token can be a false *positive* (looks legacy, is a valid role token) OR a false *negative* (named like a role token, is actually legacy). The numeric `primary-N` here are legacy sky → map to `brand-*` (sky→brand, the ProfileSettings Conv-303 precedent). Always resolve the token to its CSS definition, not its name.

## 3. Directory ≠ blast radius — check actual consumers
**Topics:** route-sweep, conformance-scope, tier-2

**Context:** `AdminDataTable`/`AdminFilterBar`/`AdminPagination`/`AdminDetailPanel` live in `src/components/admin/`, suggesting they're shared across the 16 conf-OUT RG-ADMIN routes (which would make conforming them a large cross-cutting decision).

**Learning:** A `grep -rl "admin/<Primitive>'"` showed **only `ModeratorQueue`** imports them — they're misnamed (mod-only, in the admin folder). So conforming them for the conf-IN `/mod` route has **zero RG-ADMIN blast radius**. Always confirm the real consumer set before treating a "shared primitive" as app-wide scope.
