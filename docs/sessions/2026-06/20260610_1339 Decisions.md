# Conv Decisions - 2026-06-10

## 1. POST-MATT scope = Home aggregated feed, display-only
**Type:** Strategic
**Topics:** feeds, product-model

**Trigger:** User questioned whether an isolated/aggregated post needs live likes/comments.

**Options Considered:**
1. Restyle FeedActivityCard in place (keep interactivity)
2. Build interactive FeedPost replacing FeedActivityCard across 6 consumers
3. Make SocialPost interactive
4. Display-only aggregated post; native feeds untouched ← Chosen

**Decision:** POST-MATT delivers a DISPLAY-ONLY post for the Home aggregated feed. Reaction/comment pills are non-interactive social proof. Native feeds (course/community/system) keep full interactivity on legacy `FeedActivityCard` and are out of scope.

**Rationale:** Better product (drives into communities/courses = flywheel), sidesteps mixed-source reaction-API/optimistic/visitor problems, and matches what `SocialPost` (pure-render) + `FeedActivityCard`'s existing `showFeedLink` mode already pointed at.

**Consequences:** The big interactivity-preservation risk evaporates for this task. FeedPost is a thin adapter, not a behavioral port.

## 2. Two non-colliding click targets; card is not a single link
**Type:** Implementation
**Topics:** feeds, ux

**Trigger:** User: embedded PostAnchor content must stay clickable to its promo.

**Decision:** Embedded anchor → its own promo destination (its CTA). A discrete "in {feed}" header link → the post's home feed. No whole-card link.

**Rationale:** A whole-card link would swallow the anchor's own CTA.

**Consequences:** `SocialPost.feedLink` optional prop added (ours-extension; Matt's isolated frame has no feed-context link). FeedPost derives it from the post's home feed.
