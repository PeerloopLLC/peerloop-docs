# Conv Learnings - 2026-06-17

## 1. Cross-machine stale-branch hazard is invisible without a gate
**Topics:** dual-repo, cc-workflow

**Context:** M4 sat on `jfg-dev-13-matt` (Conv 291) while all of Conv 292/295/296 — including the PALETTE-FDN foundation — was on `jfg-dev-14`. `git pull --ff-only` only fast-forwards the *current* branch; it never switches or surfaces a sibling-branch divergence. The SessionStart hook printed the branch name but nothing compared it to RESUME-STATE's recorded branch.

**Learning:** The first symptom was a colour-token lookup silently failing mid-sweep — a confusing, downstream failure for an upstream cause. A deterministic gate comparing checked-out-branch vs RESUME-STATE's `Branch: code:` (now [RSTART-DIFFGATE] Step 5.6) converts this into an explicit, early, offer-to-fix prompt.

## 2. map-or-flag on a coarse 3-step accent scale = nearest-tone, not 1:1
**Topics:** matt, astro

**Context:** /book's SessionBooking used off-scale Tailwind steps (red-700, red-50, green-600, amber-600/700/800, border-amber-200) but PALETTE-FDN accents only expose {100,300,500} (only `neutral` is fine-grained).

**Learning:** Each off-scale tone maps to the *nearest* available accent step, deliberately collapsing tonal variation (amber-600/700/800 all → warning-500). The high-value catch was reading each site in context: `text-amber-500 ★` is a rating star → `text-star`, NOT a warning — a blind hue-swap would have mis-tokenized it.

## 3. Primitive adoption surfaces colour bugs a swap never would
**Topics:** matt, testing

**Context:** ExpectationsForm's retrofit adopted FormField/Select/Input-family primitives. Those primitives use `--Alert-Default` for invalid/required.

**Learning:** Rendering the *real* primitives in context exposed that `--Alert-Default` was still the neon `#FF0038` PALETTE-FDN had tuned `error` away from — an app-wide divergence (23 utils / 14 files). A pure per-component colour swap would never have surfaced it. Adopting primitives also makes error semantics come "for free" and stay consistent (only the one form-level error banner needed hand-styling).

## 4. The full test suite is the only honest baseline; import-graph proves pre-existing
**Topics:** testing, cc-workflow

**Context:** Conv-297 codecheck ran the full suite and found 5 failures (RecommendedCourses/Communities + OnboardingNudgeBanner dismiss tests + ListingShell mobile) — latent since Conv 292's ephemeral-dismiss change, hidden because the suite hadn't been run (the carried-forward-baseline trap).

**Learning:** To prove the failures were *not* my regression without re-running, checked the import graph: none of the failing components import any Conv-297-changed file → they cannot be my cause. This cleanly separated "commit-blocking" from "pre-existing debt" ([STALE-TESTS] #32), letting the commit proceed honestly.

## 5. gen:registries is matt-sourced only; matt-inspired is hand-maintained
**Topics:** matt

**Context:** Registering the new `@matt-inspired` Textarea primitive, `npm run gen:registries` regenerated `matt-sourced-registry.generated.ts` (wrong registry) and didn't resolve the untracked flag.

**Learning:** `matt-sourced-registry.generated.ts` is generated; `matt-inspired-registry.ts` (COMPONENT_CANDIDATES) is hand-maintained. New matt-inspired primitives need a manual entry there. (gen:registries also opportunistically added a stale-missing Avatar sourced entry — reverted, as it was out of scope.)
