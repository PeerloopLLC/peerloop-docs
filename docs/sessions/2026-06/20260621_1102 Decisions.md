# Conv Decisions - 2026-06-21

## 1. OAuth buttons adopt the blue Matt `<Button variant="outlined">` pill
**Type:** Implementation
**Topics:** astro, cc-workflow

**Trigger:** `OAuthButtons` used the legacy `btn-outline` global @utility (neutral grey-bordered, 8px radius). Conforming could either preserve the neutral look with role tokens or adopt the Button primitive (blue text+border, 39px pill) — a visible recolor.

**Options Considered:**
1. Keep neutral outlined (role tokens, appearance unchanged)
2. Adopt `<Button variant="outlined">` — blue Matt pill ← Chosen (by user)

**Decision:** Swap both OAuth links to `<Button variant="outlined" href=… className="w-full">`.

**Rationale:** User chose maximal primitive adoption over the conventional-neutral OAuth look. CC deliberately did NOT make this design call solo (Critical Rule — visible recolor).

**Consequences:** OAuth buttons render blue 39px pills (DOM-verified r39, `rgb(7,119,182)` text+border, white bg). Legacy `btn-outline` @utility no longer used on these surfaces.

## 2. `/verify` cert-type + course-title use `text-h3-bold` (preserve 600 weight)
**Type:** Implementation
**Topics:** astro

**Trigger:** Legacy `text-lg font-semibold` (18/600) has no exact Matt token; `text-body-large-*` only offers 400/500 (weight drop), `text-h3-bold` is 20/600 (weight match, +2px).

**Options Considered:**
1. `text-body-large-medium` (20/500 — semantically body prose, weight drops)
2. `text-h3-bold` (20/600 — preserves weight, matches user-name heading) ← Chosen (by user)

**Decision:** Map cert-type + course-title to `text-h3-bold`.

**Rationale:** User preferred preserving the original semibold weight over the body-prose semantics.

**Consequences:** Cert-type, course-title, and user-name all render 20/600 on /verify (DOM-verified).
