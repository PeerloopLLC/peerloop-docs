# Conv Decisions - 2026-06-14

## 1. Split the two feed seeds across two files by FK dependency
**Type:** Implementation
**Topics:** d1, stream, testing

**Trigger:** Both announcements and post_promotions needed seed; where should each live?

**Options Considered:**
1. Both in a new `migrations-dev/0004_seed_feed_features.sql` (needs new npm wiring; promotions would FK-fail — feed_activities not yet seeded)
2. Both in seed-feeds.mjs (announcements don't belong in a Stream-seed script; SQL is cleaner for pure-D1 rows)
3. Announcements → append 0001_seed_dev.sql; promotions → seed-feeds.mjs ← Chosen

**Decision:** Each seed lives where its FK dependencies are already satisfied — announcements (FK users) in the SQL seed that runs first; promotions (FK feed_activities) in the runtime feed-seed after feed_activities are written.

**Rationale:** Principled by data dependency; reaches the default `db:setup:local:dev` chain without new npm wiring; keeps each seed co-located with its prerequisites.

**Consequences:** +3 announcements in 0001_seed_dev.sql, +5 promotions in seed-feeds.mjs via inline promoteTo.

## 2. Full staging chain (db:setup:staging:feeds), preserving Stripe/booking data
**Type:** Strategic
**Topics:** deployment, d1, cloudflare

**Trigger:** "reseed staging and dev to be in sync" — but a reset wipes everything; staging has Stripe/booking integration data.

**Options Considered:**
1. Full staging chain (reset+migrate+dev+stripe+booking+feeds) — superset of dev, keeps integration data ← Chosen (with a dev re-run first)
2. Mirror dev exactly (dev+feeds only) — removes Stripe/booking test data from staging
3. Re-run dev too first, then the full staging chain (user pick)

**Decision:** Re-ran `db:setup:local:dev` for a clean pair, then `db:setup:staging:feeds` for the full staging chain.

**Rationale:** Staging is the integration target — wiping its Stripe/booking test data would regress those test surfaces. The new feed data lands either way (in 0001 + seed-feeds.mjs).

**Consequences:** Staging fully reseeded + redeployed; D1 matches dev on all feed tables.
