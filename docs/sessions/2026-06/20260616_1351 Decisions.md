## 2. Select gains a `clearable` mode (primitive-level fix)
**Type:** Implementation
**Topics:** forms, astro

**Trigger:** Filter dropdowns couldn't return to "All" (placeholder was a disabled empty option).

**Options Considered:**
1. Per-filter explicit `{value:'', label:'All X'}` option in each call site
2. Add a `clearable` prop to the Select primitive ← Chosen

**Decision:** `Select` placeholder renders `disabled={!clearable}`; Level/Length/Topic opt in; Sort stays required.

**Rationale:** One primitive change fixes all current + future filter-selects; non-disabled empty option is re-selectable.

**Consequences:** Reusable; other filter-selects app-wide can adopt `clearable`.

## 3. Dismissibles always reappear on reload in dev/staging
**Type:** Implementation
**Topics:** astro, cc-workflow

**Trigger:** Dismissed nudges/recs stay hidden across reloads, so their UI is unaccounted for while building pages; client tests on staging.

**Options Considered:**
1. Manually clear localStorage each time
2. Hostname/env-gated `readDismissed()` helper ← Chosen
3. Build-time `PUBLIC_` env var (more robust; deferred)

**Decision:** New `src/lib/ephemeral-dismiss.ts`; `readDismissed()` returns false in dev (`import.meta.env.DEV`), localhost, or `*staging*` hostname; production persists.

**Rationale:** Self-contained, no config wiring, covers the known staging URL immediately.

**Consequences:** Onboarding nudge + both recs bands reappear on refresh in dev/staging. Caveat: keys on hostname containing "staging" — recorded in memory `project_ephemeral_dismiss_dev_staging`; upgrade path = PUBLIC_ env var.
