# Conv Learnings - 2026-06-15

## 1. Live Peerloop pages never reach document_idle → Chrome bridge read_page/screenshot fail; javascript_tool + curl are the workaround
**Topics:** plato, cc-workflow, testing

**Context:** AppLayout shell holds an open feeds connection, so `read_page` and `computer screenshot` both 45s-timeout on `executeScript` waiting for `document_idle`, even though the page rendered (title set).

**Learning:** For populated/authed app pages, extract DOM via `mcp__claude-in-chrome__javascript_tool` (works at `readyState=complete`, doesn't gate on idle) and verify load out-of-band with `curl -s -o /dev/null -w "%{http_code}"`. Reserve screenshots for static/Figma assets.

**Pattern:** `javascript_tool` returning a JSON.stringify of computed layout (getComputedStyle/children/class) is a reliable layout-extraction path when the visual tools are blocked.
