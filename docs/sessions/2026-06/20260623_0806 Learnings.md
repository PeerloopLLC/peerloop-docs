# Conv Learnings - 2026-06-23

## 1. The "two leading regimes" model had no branch for non-prose glyph sizing
**Topics:** matt-design-system, typography, conformance

**Context:** §09's §9.4 "tokenize or flag" SOP has exactly two branches — tokenize raw type onto a role token, or flag a novel need and mint. Avatar initials, the 404 numeral, sub-12 badges, and icon glyphs fell into neither and were tolerated as scattered per-component "display-exceptions" cross-referencing each other ("404-numeral precedent", "sub-12 glyph").

**Learning:** A glyph sized by its *container/display role* (not a text role) has no §9.3 token that fits, and `text-[Npx]` is also forbidden — so neither existing branch applies. The fix is a documented **third regime** (display/glyph, §9.2c) with its own governed tokens; future glyph cases land in it by rule, not by precedent-citation. This is consistency-by-construction vs consistency-by-exception.

---

## 2. A non-conformant SHARED primitive's fix conforms every consumer at once — with zero consumer churn
**Topics:** conformance, shared-primitives, route-migration

**Context:** UserAvatar's only violations lived in its internal `sizeClasses` map; its 15 consumers pass `size="xs"`, not a class. PromoteButton/form/Modal similarly own their styling internally.

**Learning:** Fixing a shared primitive's *internal* classes conforms all consuming routes simultaneously without touching (or re-Sweeping) any consumer file. This is why shared-primitives-first ordering (Phase 3 before Phase 4) is the high-leverage sequence — and why a non-conformant shared primitive silently invalidates the Swept claim on every route in its tree.

## 3. An upstream summary's framing assumption can hide real debt
**Topics:** conformance, deep-verify, charts

**Context:** The deep-verify's chart-primitive note said the violations were "chart.js config hex (sanctioned, verify-only)." A per-file read found `FunnelChart.tsx` had genuine `className`-level debt (`text-sm/xs`, `font-medium`, `text-secondary-*`) — not config hex at all.

**Learning:** When an audit summary classifies a whole group under one assumption ("config hex only"), per-file verification still beats trusting it — the assumption can be true for most files and wrong for one. Same lesson the bridge-shrink icon bugs taught: read the actual file.

---

## 4. The NAV-RETROFIT bridge-shrink bug is still live in unmigrated components
**Topics:** conformance, tailwind-v4, bridge

**Context:** PromoteButton's icons used `w-4 h-4`; with `--spacing-4` aliased to `--space-4`=4px (NAV-RETROFIT override), those icons rendered at **4px**, not 16px.

**Learning:** Any not-yet-conformed component using `w-4 h-4`/`h-4` for icons is silently rendering them at 4px. Converting to `size-16` both conforms (scale class) AND fixes the shrink. Worth grep-checking `w-4 h-4` on every Phase-4 file.
