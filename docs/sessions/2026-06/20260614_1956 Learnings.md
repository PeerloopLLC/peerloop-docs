# Conv Learnings - 2026-06-14

## 1. Flex max-width column needs `flex-1`, not `w-full`
**Topics:** astro, css, matt

**Context:** ListingShell column rendered 367px instead of 640 under `justify-center`.

**Learning:** A `w-full` (width:100%) child is still a non-growing flex item; `justify-center` shrinks it to content width. Use `min-w-0 flex-1` + a `max-width` cap so it grows to fill then caps.

**Pattern:**
```html
<div class="min-w-0 lg:flex-1" style="max-width:640px">
```

## 2. ListingShell mobile contract — reflow, don't hide
**Topics:** astro, css, matt

**Context:** `hidden lg:block` on a filter panel silently hid the filters below `lg` (they were trapped in a `display:none` aside) — broke filtering on mobile.

**Learning:** A FILLED (filter) panel must REFLOW to the top of the column on mobile (`order-1 lg:order-2`), not hide. Only an EMPTY placeholder panel is desktop-only (`hidden lg:block`). The filter island works at any DOM position because its events are global.

## 3. Grid→single-column recipe
**Topics:** matt, css

**Context:** Converting catalog grids across surfaces.

**Learning:** Swap `grid grid-cols-1 … sm:grid-cols-2 xl:grid-cols-3` → `flex flex-col gap-16`; change `<li className="flex">` wrappers to plain block `<li>` so the block-level (display:flex) card fills the column width. A `flex` li keeps the card at content width.

## 4. Monolithic-filter surfaces → event-bus split
**Topics:** matt, testing

**Context:** `/members` had filters as internal island state, unlike communities/courses (already two-island event-bus).

**Learning:** To relocate internal filters into the shell's panel slot, split the island into a filters island + a list island coordinating via global window events. Both seed initial state from the SAME source (`?roles=` + defaults) so the first fetch matches — no mount race. Skip the filter island's mount-time dispatch (the list does its own initial fetch) to avoid a double fetch.

## 5. Horizontal filter bars need a vertical pass for the 320px rail
**Topics:** matt, css

**Context:** CoursesFilters overflowed the rail by 44px (Select label-beside-control + min-w floor).

**Learning:** Filter bars built as wide horizontal rows must be restructured to vertical stacks (label-above-control, drop min-w floors) when moved into the 320px right rail. Phases 5–6 (Discover tabs reusing these components) will need the same.
