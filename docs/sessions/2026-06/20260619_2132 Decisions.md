# Conv Decisions - 2026-06-19

## 1. `text-[10px]` unread badge kept as C-keep exception
**Type:** Implementation
**Topics:** matt-design, conformance

**Trigger:** Messages unread count badge uses `text-[10px]`, below Matt's smallest token (body-small = 12).

**Options Considered:**
1. Bump to `text-body-small-medium` (12px)
2. Keep `text-[10px]` as a recorded C-keep exception ← Chosen

**Decision:** Keep as C-keep.

**Rationale:** Matches the ledger's sub-12px-glyph precedent (`text-[7px]` avatar initials, `text-[20px]` emoji); bumping a 2-char "9+" up 20% inside a 20px circle is the riskier choice for zero conformance gain.

**Consequences:** Recorded as exception in both ledgers; no code change to the badge.

## 2. `<Button>` adopted only for text buttons, not icon/neutral buttons
**Type:** Implementation
**Topics:** matt-design, primitives

**Trigger:** Several hand-rolled buttons across messages/notifications.

**Options Considered:**
1. Adopt `<Button>` everywhere
2. Adopt only where a variant matches; leave the rest ← Chosen

**Decision:** Adopt `<Button>` for the primary text buttons (Try Again / New Message / Start Conversation / notifications Try again); leave icon-only buttons (send/back/new-message/delete) and the neutral-outlined Load-more hand-rolled.

**Rationale:** Icon-only buttons map to an as-yet-unbuilt IconButton (un-ripe); the neutral-outlined Load-more has NO matching Button variant (outlined = brand-bordered, default = filled-gray) — forcing it would recolor it.

**Consequences:** Icon/neutral candidates logged un-ripe in the Tier-2 ledger.
