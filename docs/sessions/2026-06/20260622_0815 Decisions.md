# Conv Decisions - 2026-06-22

## 1. EarningsOverview "Request Payout" button → americana `<Button>`
**Type:** Implementation
**Topics:** matt-design, conformance

**Trigger:** TCH-OVERVIEW conform — the payout button was green (`bg-green-600`); ambiguous whether green is an intentional money-CTA signal or should conform to the standard americana primary CTA.

**Options Considered:**
1. Keep green as a deliberate positive/money signal (honest-orphan)
2. Americana `<Button variant="primary">` ← Chosen

**Decision:** Convert to `<Button variant="primary" property1="Small" disabled={!canRequestPayout}>` (americana), losing the green.

**Rationale:** Consistency with the other 7 overview CTAs and the rest of the sweep; the Button primitive is the canonical primary action.

**Consequences:** Applied identically to the same Request-Payout button in EarningsDetail (#28) for cross-tab consistency.

## 2. AvailabilityCalendar — keep availability-blue as a semantic status hue (not brand)
**Type:** Strategic (playbook refinement)
**Topics:** matt-design, conformance, color-tokens

**Trigger:** The calendar uses sky `primary-*` for ALL "available" states (recurring/override/selected cells, slot pills, today badge, recurring dot). The decorative-sky→brand rule would recolor the entire calendar purple.

**Options Considered:**
1. Brand-purple per the decorative-sky rule (whole calendar purple)
2. Keep availability-blue as a semantic status hue ← Chosen

**Decision:** Keep the sky-blue "available" tints as a semantic status hue (parallel to red=blocked/amber=series-end/green=saved); migrate `primary-*`→Tailwind `sky-*` (identical hex, off the deprecated ramp). Only interactive-sky→americana.

**Rationale:** Blue=available is a semantic status, not decoration; Matt has no "available" token, so an honest-orphan Tailwind hue is correct (same reason green-saved is kept). Recoloring purple would destroy a coherent semantic colour system.

**Consequences:** Established the general status-token rule (Learning #1), applied again to EarningsDetail's transaction-status palette (#28). Recorded in the route-migration ledger.
