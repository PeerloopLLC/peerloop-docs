# Conv Learnings - 2026-06-26

## 1. "Mentions the symbol" vs "imports the symbol" — anchor importer greps
**Topics:** refactoring, dead-code-detection, grep

**Context:** Auditing `/old` references, an importer grep `from .*AppNavbar` falsely flagged canonical pages (`course/[...tab].astro`, `Sidebar`, `404`) as live consumers — the word "from" matched prose like "re-implemented *from* the legacy AppNavbar."

**Learning:** When deciding whether a symbol is dead, a substring/keyword grep conflates prose mentions with real dependencies. Anchor on import syntax: `^\s*import\b[^=]*\bName\b` (and check `export … from` re-exports separately). The difference between "mentions" and "imports" is the entire answer to "is this dead?"

**Pattern:**
```bash
grep -rnE "^\s*import\b[^=]*\bComponent\b" src/        # real importers
grep -rnE "export\b[^=]*from '\./Component'" src/      # barrel re-exports (import-anchored regex misses these — see Learning 3)
```

## 2. A loose classifier that says "delete everything" is the dangerous failure
**Topics:** migration, verification, tooling

**Context:** A bash classifier meant to split `/old` pages into ported/only-copy used a `compgen -G "…/[[]...*.astro"` glob that returned false positives, marking `dashboard`, all `discover/*`, and `feed`/`feeds` as "PORTED-tabfamily" when no such root destination existed. It also contradicted an earlier check on `feed.astro`.

**Learning:** When a heuristic classifies a destructive set, treat a too-clean result ("everything is safe to delete, zero REVIEW") as a red flag, not a green light. Cross-check against ground truth (here: `find src/pages -not -path old` + direct `[ -f ]` on each suspected destination). A contradiction with a prior result is the tell that the tool is lying.

## 3. The five-gate rule earns its keep: `build` passed, `tsc`/`astro check` caught it
**Topics:** testing, baseline, astro

**Context:** After deleting `FeaturedCourses.tsx`, the `marketing/welcome/index.ts` barrel still re-exported it. `npm run build` (vite/astro) succeeded (tree-shaken), but `tsc --noEmit` and `astro check` both errored TS2307.

**Learning:** A green build is NOT a green type-check. Astro/Vite builds don't full-type-check by default, so dangling re-exports survive `build` but fail `tsc`/`astro check`. This is exactly why the project's baseline requires all five gates — `tsc` alone misses `.astro`, and `build` alone misses type errors.

## 4. Deletion order as a correctness tool
**Topics:** migration, safety

**Context:** Retiring 60 `/old` pages + ~9 coupled components.

**Learning:** `/old` pages are route endpoints with zero importers, so deleting them can't break a build — it only *manufactures* downstream orphans. Sequencing as Phase A (pages, risk-free) → Phase B (re-detect orphans against the *post-deletion* graph) means component-orphan status is verified against reality, not predicted. Each layer assumes the previous might be wrong, which is why the buggy classifier + 3 stale-ledger claims never reached the actual `rm`.

## 5. One import decision gated an entire cascade
**Topics:** architecture, dead-code

**Context:** The legacy app shell (`AppNavbar`/`DiscoverSlidePanel`/`layouts/old/AppLayout`) became deletable only because the 14 kept marketing pages turned out to use `LandingLayout`.

**Learning:** Before declaring a shared shell deletable, check what the *kept* survivors import. Had even one retained page used `layouts/old/AppLayout.astro`, the whole `AppNavbar` subtree would have to stay. The keep-set's import choices, not the delete-set's, decided the cascade.

---

## 6. Soft vs hard test references determine cleanup cost
**Topics:** testing, dead-code

**Context:** Feared the legacy-shell deletion would break integration + plato tests (AppNavbar appeared in 5, DiscoverSlidePanel in 4).

**Learning:** Most were soft — `describe()` labels, comments, plato nav-model strings, the generated route-map, and a coincidentally-named mock-data helper (`getTestFeaturedCourses`). Plato `.instance.ts`/`.generated.ts` files aren't matched by the vitest `*.test.*` glob, so they don't gate the unit suite. Only `CourseBrowse.test.tsx` was a real dedicated test. Always check the *nature* of a test reference (import-and-render vs string-mention) before scoping test cleanup.
