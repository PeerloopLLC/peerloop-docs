# Conv Learnings - 2026-06-08

## 1. `Astro.redirect()` from a layout component yields blank 200, not 302 (recap)
**Topics:** astro, auth

**Context:** Cluster-0 review surfaced the already-fixed ADMIN-REDIRECT-BLANK; reaffirmed.

**Learning:** Route-level authz must live in `middleware.ts`, not a layout component return. A nested `Astro.redirect()` inside a shared layout renders a blank 200 instead of issuing a clean 302.

---

## 2. A "summary layer" cannot absorb "deep functional surfaces"
**Topics:** routing, architecture

**Context:** Deciding whether /dashboard porting could replace clusters 2/3/4.

**Learning:** DASH-GAP split the role-route differentiators into (a) hub-level (dashboard-shaped: badges, toggles, CTAs) and (b) deep functional surfaces (editors, analytics, full lists). A command-center can hold (a) but never (b). If (b) has no dedicated root home, "fold into dashboard" silently loses it once /old retires — hence the need for Creator/Teacher Studios.

---

## 3. Profile loaders dead because pages query D1 inline
**Topics:** astro, testing

**Context:** #22 dead-loader puzzle on cluster-5 profile routes.

**Learning:** `/old/{teacher,creator}/[handle]` do D1 queries inline in frontmatter; the faithful `fetch*ProfileData` loaders were written but never wired. Teacher loader is identical (drop-in); creator loader diverged (predicate + dropped `primary_topic_id`) — never exercised, so the drift went unnoticed. Written-but-unwired code accumulates silent divergence.

---

## 4. One field, three meanings is a latent correctness hazard
**Topics:** auth, architecture

**Context:** ROLE-SEMANTICS review of role flags site-wide.

**Learning:** `is_creator` is populated by `canCreateCourses` (permission) in some paths, `hasCreatedCourses()`/`EXISTS course` (behavioral) in others, and `can||has` (hybrid) in others. The same flag name means different things depending on code path — a class of bug that type-checking can't catch. There is no canonical source (`/api/me/*` recompute inline); scale ~93/68/55 refs.

---

## 5. A generated-from-audit doc can carry a silent conflation error
**Topics:** docs-infra, routing

**Context:** The README "Ported (44): index/dashboard→`/`" line conflated two distinct legacy pages.

**Learning:** Only `/old/index.astro` (legacy home) actually became root `/`; `/old/dashboard.astro` (the UnifiedDashboard cross-role command center) was never ported, leaving `/dashboard` auth-gated→404 with a broken `AppNavbar.tsx:97` link. A single audit-summary line collapsed two pages into one "ported" claim — worth a future pass to verify the other 43 "ported" claims aren't similarly conflated.
