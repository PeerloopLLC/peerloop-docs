# Conv Learnings - 2026-06-07

## 1. Matt Select primitive is `w-full` — constrain via wrapper, not className
**Topics:** matt-design, primitives, tailwind

**Context:** Swapping inline `text-sm` raw selects into the `@matt-inspired` Select primitive, which hardcodes `w-full` first in its class array.

**Learning:** Passing `w-48`/`w-auto` via `className` won't reliably override the primitive's `w-full` (equal-specificity width utilities resolve by stylesheet source order, not class-string order). The established idiom (`CoursesFilters.tsx`) is to wrap `<Select>` in a `<div className="min-w-[Npx]">` — the `w-full` select then fills the sized wrapper.

**Pattern:**
```tsx
<div className="min-w-[200px]"><Select options={...} value={...} onChange={...} /></div>
```

## 2. Select primitive `placeholder` prop renders a *disabled* option — not for valid empty choices
**Topics:** matt-design, primitives, forms

**Context:** Both TownHall selects have a first option with value="" that is a legitimate selectable choice ("None (general post)", "All posts").

**Learning:** `Select`'s `placeholder` prop renders `<option value="" disabled>` — correct for "no selection yet" prompts, wrong when value="" is itself a valid choice. Keep such first options as real `options` entries instead.
