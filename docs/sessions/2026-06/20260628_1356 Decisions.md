# Conv Decisions - 2026-06-28

## 1. Converge the staging DB via destructive feeds-level reseed
**Type:** Strategic
**Topics:** deployment, d1

**Trigger:** Staging D1 schema 7 commits stale; deploying jfg-dev-14 would 500 on new-schema code paths.

**Options Considered:**
1. Code-only deploy, skip DB (new-schema paths error)
2. Reset + reseed at stripe / booking level
3. Reset + reseed at **feeds** level ← Chosen

**Decision:** Run `db:setup:staging:feeds` (destructive — wipes staging data, re-migrates the updated schema, seeds dev+stripe+booking+feeds).

**Rationale:** Migrations-apply can't pick up edits to an already-tracked `0001_schema.sql`; only a reset converges. Feeds level gives the most complete demo data. Staging data is reproducible seed data, so the wipe is acceptable.

**Consequences:** Staging data replaced; all new tables (homework, promote-pipeline, system feed, announcements) present; homework sample included.
