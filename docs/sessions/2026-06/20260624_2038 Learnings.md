# Conv Learnings - 2026-06-24

## 1. UserAvatar was bridge-shrunk 4× app-wide since Conv 174 — adoption surfaced it
**Topics:** astro, matt-design, tooling

**Context:** Adopting the shared `UserAvatar` primitive in the admin user table, the DOM-verify showed the avatar rendering at 8px (`h-8`→8px). Its `sizeClasses` used `h-8`/`h-12`/`h-16`/`h-24` — all in the Conv-174 token-bridge shrink-set {4,8,12,16,20,24,32,40,48,64} — so `sm`/`md`/`lg`/`xl` rendered at 8/12/16/24px instead of the intended 32/48/64/96px on **every consumer** since Conv 174 (~15 live). `size="xs"` (`h-6`, off-set) was correct by accident.

**Learning:** A shared primitive authored with **in-set numeric** sizing utilities is silently bridge-shrunk app-wide; the bug hides because consumers were browser-verified at the shrunk sizes and "looked fine." Always `getComputedStyle`-check a primitive's rendered px when adopting it into a new surface — the tsc/lint/astro gate cannot catch it.

**Pattern:**
Restore intended px via bridge-safe utilities — in-set N→`N`px (e.g. 32px=`h-32`), off-set values as arbitrary `h-[96px]`.

## 2. The bridge shrink-set trap recurs throughout legacy admin components
**Topics:** matt-design, astro

**Context:** This conv fixed bridge-shrunk sizes in 5 places across 3 routes: toggle knob `w-4 h-4` (→4px), course thumbnail `w-12 h-8` / `h-8 w-12` (→12×8px), rating star `w-4 h-4` (→4px), list progress wrapper `w-32` (→32px), module-row icons `w-4 h-4` (→4px).

**Learning:** Any pre-bridge component's numeric `w-N`/`h-N`/`translate-x-N`/`size-N` in the shrink-set renders ~4× small. When sweeping a legacy admin component, audit **every** numeric sizing utility, not just colour/type tokens.

## 3. The `text-red-600` link bug recurs across admin detail components
**Topics:** matt-design

**Context:** Three more "View X →" links carried `text-red-600` (error-red used as a link colour): UserDetailContent (View profile), CourseDetailContent (View course + View creator). Same class was fixed on route #1 (PayoutsAdmin).

**Learning:** The legacy admin detail components systematically used error-red for member/course deep-links. Canonical fix = `text-info-500 hover:text-info-300` (the route-#1 precedent).
