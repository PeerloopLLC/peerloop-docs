## 1. Defer COMM-TAG-FILTER to post-production (retire as port-artifact)
**Type:** Strategic
**Topics:** stream, docs-infra

**Trigger:** User asked "What is the need for channels? Do admins even need them?" after correcting the feed naming model.

**Options Considered:**
1. Build channels (community feeds only)
2. Build channels (Commons/system only)
3. Defer post-production ← Chosen
4. Pivot to naming cleanup instead (also chosen as the active work)

**Decision:** Defer COMM-TAG-FILTER to post-production; do not build for MVP.

**Rationale:** Channels were a port-artifact of the legacy Commons' decorative `?tag=` chips. Their value is a function of feed volume + modal variety; the system feed is now admin-only/low-volume, so the member-town-hall premise that justified them is gone. No user/admin need at genesis-cohort scale (60–80 students).

**Consequences:** Task #1 marked deferred; `plan/comm-tag-filter/README.md` retained as a "revisit-if-volume" note; REASSEMBLY-CONV271 parked section corrected.

## 2. Full pre-production rename of the system feed: the-commons/townhall → system
**Type:** Strategic
**Topics:** stream, testing, docs-infra

**Trigger:** User: "Townhall is no longer in use. Nor is The Commons... we have the System feed." Dead names leak into discussions from code identifiers.

**Options Considered:**
1. Display-string only
2. Constant-ize but keep `the-commons`/`townhall` values
3. Full value rename to `system` ← Chosen (substantive+structural now, cosmetic deferred)

**Decision:** Rename slug `the-commons`→`system`, id `comm-the-commons`→`comm-system`, name `The Commons`→`System`, route `/api/feeds/townhall`→`/api/feeds/system`, component `TownHallFeed`→`SystemFeed`. New `src/lib/system-feed.ts` constants.

**Rationale:** Pre-production (no users, no real data, full seed control) is the one window to rename values at the root cheaply; post-launch the slug/Stream group calcify. Constant-izing-but-keeping-values leaves a permanent naming scar.

**Consequences:** ~50 files changed + re-seed; 5 gates green (test 6713); browser-verified. Cosmetic var-name/comment residue (~26 files) deferred to [SYS-RENAME-COSMETIC].

## 3. Stream feed group stays `townhall` (documented constant)
**Type:** Implementation
**Topics:** stream

**Trigger:** GET /api/feeds/system 500'd — "system feed group does not exist" (Stream groups are dashboard-declared).

**Options Considered:**
1. Create a `system` feed group in the Stream dashboard, then flip
2. Keep `townhall` behind a documented constant ← Chosen

**Decision:** `SYSTEM_STREAM_GROUP = 'townhall'` in `src/lib/system-feed.ts`, with a comment explaining Stream groups are app-config, not write-creatable.

**Rationale:** The Stream group is internal infra never shown to users; renaming requires a coordinated Stream-dashboard change for zero user-facing benefit. User confirmed "Leave it 'townhall'".

**Consequences:** The only place `townhall` survives as a functional value is this one constant (+ seed `feedGroup`). Fully eliminating it would need a dashboard-created `system` group + constant flip.
