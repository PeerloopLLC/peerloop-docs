# Conv Learnings - 2026-06-03

## 1. The `[...tab].astro` + `_*-tabs.ts` + SubNav triad is now a standardized 3-family pattern
**Topics:** astro, routing, matt-design

**Context:** Building `/community/[slug]` by mirroring `/course/[slug]`.

**Learning:** `/course` (dual-zone) and `/profile` (flat, `@matt-inspired`, no Figma source) already share the catch-all + tab-config + `SubNav` triad. `/profile` is the key precedent: it proves the pattern works `@matt-inspired` with no Matt frame — exactly community-detail's situation. Community is the 3rd family; `/feed/[slug]` will be the 4th.

**Pattern:**
New tabbed detail family = `pages/X/[slug]/_X-tabs.ts` (`buildXTabs(slug, opts)`) + `[...tab].astro` (empty segment = overview default, `VALID_TABS` guard → redirect unknown to bare, per-tab data load, `SubNav slot="sub-nav"`).

---

## 2. SubNav holds destinations only — actions and filters live elsewhere
**Topics:** matt-design, routing

**Context:** Deciding which legacy community items become SubNav tabs.

**Learning:** A SubNav rail is for sub-route destinations. Membership actions (Manage/Leave) belong in the header (like CourseHeader's Enroll CTA); in-content filters (The Commons `?tag=` chips) belong in the tab body; back-nav is the breadcrumb. `/profile`'s doc-comment states this rule outright ("Help/Dark-mode/Sign-out are actions… not tabs").

---

## 3. Three distinct "feed" surfaces — a recurring confusion to keep straight
**Topics:** feed, routing

**Context:** Isolating The Commons / TownHall before verifying.

**Learning:** `/feed` (singular) = personal smart/aggregated timeline — NO route file yet, 404. `/feeds` (plural) = discovery directory — built. `/community/the-commons/feed` = the townhall (The Commons' feed). The Commons ≡ TownHall: `/api/feeds/townhall` is hardwired to `the-commons`.

---

## 4. A legacy "control" may be decorative — porting it faithfully can mean dropping it
**Topics:** porting, testing

**Context:** The Commons `?tag=` filter chips.

**Learning:** The legacy tag chips updated the URL but `TownHallFeed` never consumed `tag` — dead UI. Faithful port = remove the dead control + flag it ([COMM-TAG-FILTER]), not replicate a non-functional widget. Same for the legacy "Leave" button (no handler).

---

## 5. Browser-verify catches route-string correctness tsc can't (and a query-string guard gotcha)
**Topics:** testing, cc-workflow

**Context:** DOM-probing the new route on :4322.

**Learning:** All 5 static gates passed but the meaningful confirmation (tabs resolve, islands hydrate, gated affordances, home card fulfilled) came from the browser. Note: the chrome `javascript_tool` blocks results that return hrefs containing query strings ("Cookie/query string data") — return counts/booleans, not raw URLs.
