# Conv Learnings - 2026-06-17

## 1. Static prim-treewalk sensor cannot see prop short-circuits — trace the render flow
**Topics:** route-sweep, primitives, matt-design

**Context:** The prim-treewalk sensor flagged EnrollButton's legacy-styled button cluster as a STRONG primitive candidate on `/course/[slug]/[...tab]`.

**Learning:** EnrollButton has a `variant="matt"` path (Button primitive) and a legacy path; the Benefits tab passes `variant="matt"`, so the legacy styling never renders on this route — it's reachable only via `/old` (CourseHero). The static import-graph walk can't know which branch actually renders. Always trace the real render/prop flow before treating a sensor STRONG as in-scope.

**Pattern:** sensor STRONG ≠ in-scope. Confirm the prop branch that renders on *this* route before scoping a candidate.

## 2. Classify a hard-hex before swapping — primitive-signature vs convention vs stray
**Topics:** route-sweep, design-tokens, matt-design

**Context:** 4 hard-hex "ripe quick fixes" on route 2 (`/course/[slug]/[...tab]`) were applied, gate-passed, DOM-verified, then reverted.

**Learning:** `#f6f6f6`+`rgba(88,77,244,.14)` is the documented `AnalyticCount` primitive Default signature; `#1f2937` is the documented shared no-thumbnail fallback (CourseHeader + CourseCatalogCard + CommunityCatalogCard). They are intentional Matt values, not legacy strays. Nearest-token mapping (gray-100 `#F1F1F1` ≠ `#F6F6F6`; gray-base ≠ `#1f2937`) plus Tailwind v4 `/14` opacity resolving as oklab (not sRGB rgba) silently diverged them.

**Pattern:** Before swapping a hard-hex, classify it — primitive-signature → adopt the primitive; convention/recurring → tokenize app-wide once; only true strays get a per-route swap. Run the remaining-site grep + registry read DURING assessment, not after editing.

## 3. Matt's palette is anchor points, not ramps — and has no status hues
**Topics:** design-tokens, matt-design, colour

**Context:** `/book` SessionBooking + EnrollButton + route-2 pills all surfaced Tailwind-default colours with no Matt token to map to.

**Learning:** Matt defines 3 grays, 3 greens, several blues/purple — anchor values, not 50–950 ramps. He has NO warm hue (amber/yellow/orange) and only a retracted-speculative red. The app uses full Tailwind ramps across 20–115 files. Adopting Matt's colours means deriving tint/shade scales from his anchors; only warning + error are genuine new hues. The Conv-181 "tokenize only Matt-formalized variables" rule is superseded for these blank categories by Conv-289 CC-ownership.
