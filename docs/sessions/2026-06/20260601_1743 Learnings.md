## 1. A Figma frame's editorial alias can lie — verify the layer name
**Topics:** figma-mcp, matt-provenance

**Context:** The frames-doc labeled `667:12040` "Page / Choose Teacher", implying a booking screen.

**Learning:** `get_metadata` returned the true layer name "Content / Happy / Course Teachers Enrolled" — it's the Teachers tab (enrolled state), not a booking page. Always confirm a frame's actual layer name before treating a catalogue alias as its identity. The same probe revealed `622:17884` = Session Prepare (post-booking pre-join view, session-lifecycle family) and `622:15671` = Teacher Schedule (the real date-picker, creator-side, not Ready-for-Dev).

## 3. The Matt rewrite silently dropped the enrolled-operational tabs
**Topics:** matt-design-system, port-fidelity, enroll-nav

**Context:** Grounding the ENROLL-NAV design in the legacy course tabs.

**Learning:** Legacy `CourseTabs` had Sessions (session list + booking entry) and Learn (modules + progress + homework) as **enrollment-gated** tabs, plus role tabs (My Teaching / All Sessions). The new Matt rail (`_course-tabs.ts`) kept only public/marketing tabs + added Benefits/Reviews/Meet-the-Creator — dropping the post-enroll operational surfaces. ENROLL-NAV re-homes them. Same port-fidelity class as Conv 222.

## 4. SubNav active-matching prefix-matches the section-index href
**Topics:** subnav, navigation, astro

**Context:** Adding the course rail to `/book`.

**Learning:** `SubNav.matchHref` uses `path.startsWith(href + '/')`, so a non-tab child route like `/book` prefix-matches the About tab (href = bare course path) and would wrongly highlight it.

**Pattern:** For a page with no corresponding tab, pass `currentPath=""` → nothing matches → no false-active. Proper fix is a real SubNav item (ENROLL-NAV).
