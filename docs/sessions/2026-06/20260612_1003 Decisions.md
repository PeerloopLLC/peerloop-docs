# Conv Decisions - 2026-06-12

## 1. QLINT exemption model — labels-only, not 👉-or-labels
**Type:** Implementation
**Topics:** cc-workflow

**Trigger:** User's correction that `👉 **X, or Y?**` (no labels) should still fire.

**Options Considered:**
1. Lenient (spec v1): either 👉 OR labels exempts
2. Strict/typeability: only A)/B) labels exempt; bare 👉 does not ← Chosen

**Decision:** Only labeled options exempt a soliciting message from the unlabeled-choice check; add a "yes or no" carve-out + parenthetical-slash-list detection.

**Rationale:** A 👉 marks "I want a response" and rides on every question; it doesn't make a choice typeable. The labels do.

**Consequences:** Catches the compound-or-with-pointer case; `(yes/no)` and plain yes/no stay silent.
