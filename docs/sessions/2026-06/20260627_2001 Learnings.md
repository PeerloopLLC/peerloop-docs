# Conv Learnings - 2026-06-27

## 1. A PLATO `expect` is a frozen functional spec of the legacy page
**Topics:** plato, testing, matt-migration

**Context:** An audit found ~7 UI elements PLATO expects that don't exist on the current Matt-ported pages. The first instinct was to "fix" the test prose to match the current pages.

**Learning:** PLATO browser-mode `expect`/`pageAction` was written against the *original* pages, so it encodes legacy behavior. A "missing element" finding is ambiguous between REDESIGN (renamed/relocated, intact), REGRESSION (dropped in the legacy→Matt port — a failed port), and NEVER-EXISTED (prose over-claimed UI never built; the automated step hits the API directly). Editing the test to match a possibly-regressed page would silently hide the regression — PLATO is the canary. Always triage against the legacy source (preflip `608346a2`) before editing.

**Pattern:** For any "test expects UI that isn't there" on a ported page: read BOTH sides (current + preflip), classify REDESIGN/REGRESSION/NEVER-EXISTED, then decide test-edit vs page-restore vs gap-log. Saved as memory `feedback_plato_expect_is_legacy_spec`.

## 2. Source-of-truth beats observation for deterministic queries
**Topics:** plato, verification

**Context:** Two `// LIVE-CONFIRM` markers asked whether a role-less user lists under "no filter" and the exact count — originally flagged for a live browser walk.

**Learning:** For a deterministic query, the frontend→API→SQL chain IS the contract; a live walk only observes one render of one dataset and can't be more authoritative than the code. Tracing `MembersDirectory.tsx` → `/api/members` resolved both markers definitively without a dev-D1-mutating snapshot restore.

## 3. The Matt Tier-1/Tier-2 port preserved behavior (0 regressions)
**Topics:** matt-migration, port-fidelity

**Context:** 7-element legacy-vs-current triage.

**Learning:** Several components are byte-identical between `608346a2` and current (only re-skinned with Matt tokens). The port's fidelity is high — the "missing UI" findings were all redesign relabels or prose over-claims, not dropped behavior. This is a stronger guarantee than the original task scoped.
