## 2. RS-SQL-SWEEP — reuse COUNT(*)>0 fragments for EXISTS filters, not new EXISTS-form variants
**Type:** Implementation
**Topics:** role-semantics, sql

**Trigger:** The DEFER tail's EXISTS-in-WHERE identity filters (16 sites).

**Options Considered:**
1. Add `isCreatorExistsSubquery`/`isTeacherExistsSubquery` EXISTS-form fragments (preserves short-circuit)
2. Reuse existing COUNT(*)>0 fragments ← Chosen

**Decision:** Reuse the existing fragments; `AND (SELECT COUNT(*)>0 …)` ≡ `AND EXISTS(…)` in SQLite. `NOT EXISTS` → `NOT ${fragment}`.

**Rationale:** One fragment form across all 40 sites beats doubling the API for a short-circuit perf delta that's moot at 2-user scale.

**Consequences:** 16 EXISTS filters converted; 40 total inline identity-SQL instances now route through `roles.ts` fragments.
