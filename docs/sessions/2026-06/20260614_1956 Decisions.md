# Conv Decisions - 2026-06-14

## 1. CD-039 image handling = standardized 16:9 frame, not intrinsic height
**Type:** Strategic
**Topics:** matt, docs

**Trigger:** Client "retain aspect ratio at any size" was ambiguous (cover/contain/intrinsic all "retain ratio").

**Options Considered:**
1. Intrinsic/variable card heights (no crop, no bars)
2. object-contain letterbox
3. Standardized responsive 16:9 frame + object-cover ← Chosen (client clarified)

**Decision:** One shared responsive 16:9 frame (matches existing `aspect-video`), `object-cover`, with creator upload guidance.

**Rationale:** Client wants to standardize the screen-portion images occupy; cards already use aspect-video so least churn.

**Consequences:** Phase-1 needed no image-primitive work for the piloted card; real work is the Phase-7 sweep.

## 2. ListingShell = page-level primitive, not an AppLayout slot
**Type:** Strategic
**Topics:** astro, matt

**Trigger:** Where the centered-column + right-panel shell lives (sets the pattern for all 6 surfaces).

**Options Considered:**
1. Page-level `ListingShell.astro` inside AppLayout's default slot ← Chosen
2. New `aside-right` slot baked into AppLayout

**Decision:** Page-level `src/components/layout/ListingShell.astro`; listing pages opt in.

**Rationale:** Lowest blast radius — AppLayout + ~80 other pages untouched; mirrors the existing "pages compose primitives in the default slot" pattern.

**Consequences:** Each listing page wraps its content + passes filters into `right-panel`.

## 3. /members filters → event-bus refactor (Option B)
**Type:** Implementation
**Topics:** matt, testing

**Trigger:** /members filters are internal island state, not separable like communities/courses.

**Options Considered:**
1. A — inline filters + placeholder panel (low-risk, client's "unfilled panel" wording)
2. B — refactor to event-bus two-island pattern ← Chosen (user directive)

**Decision:** Split MembersDirectory into MembersFilters + list islands via `members:*` events.

**Rationale:** User wanted full consistency (all panels filled with filters); brings /members into the same architecture as the others.

**Consequences:** New `MembersFilters.tsx`; MembersDirectory becomes list-only; `members:filterchange`/`members:clearfilters` contract.
