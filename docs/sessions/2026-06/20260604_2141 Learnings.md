# Conv Learnings - 2026-06-04

## 1. prov-sweep's NODE_RE only matches literal `data-prov-node="…"`
**Topics:** provenance, tooling

**Context:** CourseHeader's NODE-MISMATCH (one of the 6 PROV-SWEEP-DEBT errors).

**Learning:** `scripts/prov-sweep.ts` extracts stamped nodes via `NODE_RE = /data-prov-node=["']([^"']+)["']/g` — a dynamic `data-prov-node={ternary}` matches NOTHING, so the marker's expected node is "missing" → NODE-MISMATCH. Fix: stamp the component-SET node literally (`data-prov-node="517:8934"`); per-variant instance nodes live in the `@matt-source` marker + comment, not the DOM stamp. Verified empirically (6→5).

## 2. Gates-green ≠ behavior-preserved — CALENDAR2's booking tests were silently dead
**Topics:** testing, timezone

**Context:** Rewriting `SessionBooking.test.tsx` during the re-skin.

**Learning:** The mock slots used non-ISO times (`'09:00'`) → `new Date('09:00')` = Invalid Date → every time-pill text mismatched `queryByText('09:00 - 10:00')` → the `if (timeSlot)` blocks (confirm/booking/success) NEVER ran. The day-click also no-op'd silently when runner TZ ≠ UTC (mock keys built from `toISOString()` UTC vs component's local date keys). The suite looked like it covered booking end-to-end; it covered almost none of it. Rewrote with ISO local-date-key mocks + robust pill/Continue matchers (31/31 real).

## 3. Server-UTC vs client-local date-bucketing (the [TZ-AUDIT] root pattern)
**Topics:** timezone, ssr

**Context:** The test bug pointed at a production class.

**Learning:** `availability-utils.ts` builds `slot.date` from `getFullYear/getMonth/getDate` — "local" on a Cloudflare Worker = **UTC**. The browser builds calendar cells from browser-local parts. Server-UTC day-bucket matched against client-local day-bucket → off-by-one for far-TZ users.

**Pattern:** Keep ISO UTC in the data, format the day+time **client-side** in the viewer's TZ (`<time datetime>` + `astro:page-load` upgrade).

## 4. Astro type-checks `<script>` blocks as TypeScript
**Topics:** astro

**Context:** The CH-VARIANTS client enhancement.

**Learning:** `npm run check` (astro check) flags `ts(7006) implicit any` on un-annotated params inside an Astro `<script>`. Annotate them (`iso: string`, `x: Date`).
