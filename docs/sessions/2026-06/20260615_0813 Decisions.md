## 1. v2 progression-gap semantics = A (path-capstone)
**Type:** Implementation
**Topics:** testing, role-studios

**Trigger:** NUDGE-TC-V2 augmented to build the deferred T→C progression-gap.

**Options Considered:**
1. A — path-capstone: `progression_position === course_count AND badge='learning_path'` ← Chosen
2. B — any last course (incl. standalones) — too broad, barely a gap
3. C — prerequisite-gap (leaf in prereq graph) — sparse data
4. Drop v2, test v1 only

**Decision:** Gate the T→C nudge's per-course placement (⑤) on the taught course being a learning-path capstone.

**Rationale:** Most precisely matches "walked the path to its end → author the follow-on"; B waters it down; C depends on sparsely-populated `course_prerequisites`.

**Consequences:** New `lib/progression/capstone.ts`, `ProgressionNudge` `capstone?` prop (overview ④ stays v1, course ⑤ gains the gap), host computes server-side and passes it.
