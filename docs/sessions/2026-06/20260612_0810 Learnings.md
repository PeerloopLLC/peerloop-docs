# Conv Learnings - 2026-06-12

## 1. A pure-SQL seed can never mint working external-service IDs
**Topics:** stream, testing, d1

**Context:** `[SEED-STREAM-FIDELITY]` was framed as "use real-shaped Stream activity ids in the SQL seed."

**Learning:** Stream activity IDs are minted by Stream when an activity is created; a D1 SQL row can only ever store a dangling pointer. The only fix that produces working content is to actually write to Stream (`scripts/seed-feeds.mjs`) and dual-write the returned IDs. A "real-shaped string" fixes nothing — the missing Stream activities ARE the problem (enrichment fallback `buildPlaceholderActivity` renders author but no `.text` → "empty" posts).

**Pattern:** When a seed/test depends on an external service's identifiers, the seed must call the service, not fabricate IDs.

## 2. Decompose by cohesion (vertical slices), never pseudo-isolated fragments
**Topics:** cc-workflow, planning

**Context:** The feed/promotion/discovery group repeatedly "missed fundamentals"; the user diagnosed the decomposition itself as the bug — fragments kept needing pieces of sibling fragments.

**Learning:** A fragment that needs a *piece* of a sibling is mis-cut. Correct isolation = each unit owns every layer it touches (pipeline/schema/component/render/seed/tests), depends only on WHOLE prior units, unidirectionally, and has a standalone done-test. Rule-4 (done-test) is the tell: a fragment with no self-contained done-test was cut through the middle of a cohesive thing. Saved to `memory/feedback_decompose_by_cohesion_not_pseudo_isolation.md`.

## 3. The premise-check gate catches under-scoping before it's committed
**Topics:** cc-workflow, testing

**Context:** Used a "trace the real code path before building" gate on each FEED-U1 piece.

**Learning:** It paid off twice in one unit: (a) exposed a dormant `feed_type='townhall'` CHECK-constraint failure the moment the optional script became canonical (the system-feed posts had never been writing to D1); (b) caught that "display-strings-only" SYS-NAMING was actually a cross-cutting rename (~10 test fixtures + FeedPost.tsx), preventing an inconsistent half-rename commit.

**Pattern:** Making an optional/untested path canonical surfaces bugs that "worked in isolation" only because nobody exercised them.

## 4. Passive rules that keep slipping need harness enforcement, not more prose
**Topics:** cc-workflow, docs-infra

**Context:** Claude twice violated the `X, or Y?` + missing-👉 rules despite them being the FIRST CLAUDE.md section §Recurring-Failures + 2 MEMORY.md entries.

**Learning:** When a rule is maximally documented and still fails at send-time, the gap is in-the-moment application — which a passive rule can't fix. The fix is deterministic enforcement (a Stop hook that lints the final message and blocks malformed pointing questions), not additional documentation. → `[QLINT]` #46.
