# Conv Decisions - 2026-06-25

## 1. Sessions 6-stat-card hue mapping — semantic over visual variety
**Type:** Implementation
**Topics:** rg-admin, conformance, design-tokens

**Trigger:** SessionsAdmin has 6 stat cards (Total/Today/This-Week/Completed/Cancelled/With-Recording) using 6 decorative hues (gray/blue/indigo/green/red/purple), but the admin semantic palette has fewer roles.

**Options Considered:**
1. Keep the 6 hues as a categorical "honest-orphan" quartet+ (DashboardStatCard precedent)
2. Map all to semantic tokens, accepting repetition ← Chosen

**Decision:** Total→neutral, Today→info, This-Week→info (same as Today), Completed→success, Cancelled→error, With-Recording→brand (purple→brand per AdminDashboard precedent).

**Rationale:** ADMIN-CONF-POLICY drops decorative hue variety; AdminDashboard already mapped its stat hues to semantic tokens rather than keeping a quartet. Semantic correctness > visual distinctness. User approved via "Apply all as proposed."

**Consequences:** Today + This-Week render identically (both info-blue); accepted.

## 2. Footer Button variant mappings follow the locked reversibility rule
**Type:** Implementation
**Topics:** rg-admin, conformance

**Trigger:** Teachers + sessions detail-footer action buttons needed Button-variant assignment.

**Decision:** Reversible toggles → `warning` (teachers Deactivate); irreversible destructive → `danger` (sessions Delete-Recording); graded resolution → `warning` (sessions Resolve-Issue); positive/restore → `primary` (teachers Activate — no success variant); neutral → `default` (sessions Edit-Notes); external/nav href → `outlined` (teachers View-Public-Profile).

**Rationale:** Direct application of the Conv-332 locked sub-pattern (a). No novel decisions.

**Consequences:** Consistent with Payouts/Users/Enrollments footers.

## 3. Date-range filters inline-conformed, not adopting form/Input
**Type:** Implementation
**Topics:** rg-admin, conformance

**Trigger:** SessionsAdmin has two `type="date"` inputs + a "Clear dates" text button in a dense filter row.

**Decision:** Inline-conform (neutral-300/rounded-8/body-small) under relaxation C rather than adopting the `form/Input` primitive.

**Rationale:** Relaxation C sanctions lightweight inline controls in dense rows (precedent: EnrollmentsAdmin reassign `<select>`). `form/Input` would be heavier than warranted for a `type="date"` filter.

**Consequences:** Date inputs match the dense console aesthetic without primitive overhead.
