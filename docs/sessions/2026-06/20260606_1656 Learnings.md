# Conv Learnings - 2026-06-06

## 1. The Matt Sidebar silently 404s on Notifications; Messages is absent entirely
**Topics:** route-migration, matt-design, navigation

**Context:** Investigating whether Notifications was ported.

**Learning:** `Sidebar.tsx` (canonical Matt shell) links `/notifications` which has no root route → live 404 for every logged-in user (same href in AppNavbar/AppHeader/ControlBar). `/messages` has no root route AND no Sidebar entry. Both islands (`NotificationsList`, `Messages`+3) are unmarked legacy using `@components/ui/icons`. This is the `/messages` incident pattern (Conv 203) recurring.

**Pattern:** When auditing a Matt-shell nav item, check (a) does the href resolve to a root route, and (b) does the target carry a provenance marker — a MattIcon nav link is not evidence the destination is ported.

## 2. Messages is REST-polling, not Stream.io
**Topics:** messages, architecture

**Context:** Scoping the /messages port; assumed Stream.io real-time.

**Learning:** `Messages.tsx` uses plain React state + a 10s polling interval against `/api/*`, not Stream.io. De-risks the port — no websocket/real-time complexity. (Stream.io is feeds-only in Peerloop per the stack table.)

## 3. prov:sweep enforces icon-provenance entries — dropping an SVG in is not enough
**Topics:** icons, provenance, testing

**Context:** PROV-SWEEP-DEBT root cause (av-timer/verified).

**Learning:** Adding an SVG to `svg/` auto-registers it for `MattIcon` rendering but `prov:sweep` fails until a matching `icon-provenance.ts` entry exists. This bit Conv 233's icon additions. Documented in the new HOWTOREG-ICN how-to.

## 4. /r-prune-memory re-flatten floor is higher than naive estimate
**Topics:** memory, cc-workflow

**Context:** MEM-CAP landed at 77% not the projected 69%.

**Learning:** A re-flattened pointer keeps a fixed prefix (`- [link](file.md) — `) PLUS retained distinctive markers (~80–180B), so realistic trimmed lines are ~150–230B, not hook-only. Net savings ≈ original − ~190B. The big wins are 500B+ lines; below ~350B returns diminish. 8 trims cleared the 80% warn threshold (the actual goal); a 70% target would have needed ~15 low-yield trims.
