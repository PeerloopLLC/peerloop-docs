# Conv Learnings - 2026-06-15

## 1. e2e login bug had two homes — shared helper (fixed Conv 286) AND inline copies
**Topics:** testing, e2e

**Context:** Conv 286 fixed the `getByLabel('Password')` strict-mode violation in `e2e/helpers.ts`, but 3 specs (`admin-overview`, `auth-dashboard`, `browse-enroll`) carried their *own* inline login blocks that never used the helper, so they kept failing.

**Learning:** A fix to a shared helper only protects callers that actually import it. When auditing a class of failure, grep for the buggy pattern (`getByLabel('Password')`) across the whole suite, not just the helper — inline duplicates are invisible to the helper fix.

**Pattern:**
`grep -rln "getByLabel('Password')" e2e/` surfaced all inline copies in one shot.

## 2. "N failures" ≠ "N bugs" — bucket by root cause before fixing
**Topics:** testing, e2e, process

**Context:** 55 e2e failures looked like a multi-day slog. Bucketing by spec-file + error-signature collapsed them into: 1 login-helper class (~9-13), 1 large UI-structure-drift class (~45), 1 parallel flake.

**Learning:** Two cheap groupings — `failures per spec file` and `error-type frequency` — turn a scary raw count into an effort estimate and reveal which fixes are mechanical vs. investigative. Do this before touching any spec.

---

## 3. Don't update test expectations to match in-flight UI
**Topics:** testing, e2e, role-studios

**Context:** ~25 of the residual failures are dashboard-heading assertions (`'My Learning'` etc.) that will change when ROLE-STUDIOS ships — a block currently BLOCKED BY CLIENT.

**Learning:** Test expectations should track *settled* behavior. Updating them to today's interim UI creates false-green that silently re-breaks when the in-flight feature lands, and the re-break diff reads like a regression. Defer the coupled bucket; fix only already-shipped drift.
