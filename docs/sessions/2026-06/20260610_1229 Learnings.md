# Conv Learnings - 2026-06-10

## 1. The smart feed's "townhall" enum and Stream feed group are decoupled identifiers
**Topics:** stream, d1, refactoring

**Context:** SYS-RENAME needed to rename the `feed_type` token without breaking Stream continuity.

**Learning:** D1 `feed_type` (values townhall/community/course, with `feed_id='the-commons'`) is independent of the Stream feed group `('townhall','main')`. They share the name but address different things — the D1 enum can be renamed to `'system'` while the Stream group stays `'townhall'`, and reads/writes remain internally consistent as long as all D1 sites agree. This let boundary C be a clean enum-only rename, deferring the Stream/route rename to the cosmetic follow-up ([SYS-NAMING]).

## 2. Admin flag leaks a 'moderator' role onto course feeds in getFeeds()
**Topics:** auth, testing

**Context:** Making `getFeeds` tests admin (to keep the System feed visible) broke `['student']` course-role assertions → got `['student','moderator']`.

**Learning:** `CurrentUser.isAdmin` makes `canModerateFor()` return true for every course, so `getFeeds` adds `'moderator'` to all course feeds for admins. A test that wants default course roles must NOT be an admin user. Fix: keep the system-presence tests admin and the course-role tests non-admin.
