# Conv Learnings - 2026-06-29

## 1. Machine-local `.scratch/` files are stale-on-return by the count of convs run elsewhere
**Topics:** cc-workflow, docs-infra

**Context:** Conv 350's r-start hit a `.scratch/conv-tasks.md` that was 3 convs stale (Conv 346 vs the Conv 349 RESUME-STATE). Initially mis-framed as a guard failure.

**Learning:** The user works **several convs on one machine, then switches** (346 = M4; 347–349 = M4Pro; 350 = M4). Because `.scratch/` is gitignored/machine-local, any machine-local companion (conv-tasks.md, conv-turns.md) is naturally stale by exactly the number of convs run on the other machine since this one was last used. A multi-conv "shrink" on return is the **expected** state, not an anomaly — reconcile (via git history if the latest RESUME-STATE can't explain it), don't halt.

**Pattern:** When a machine-local task file's header conv differs from `current_conv − 1`, widen the reconciliation evidence window to git history / COMPLETED.md across the elapsed range rather than trusting only the latest RESUME-STATE ledger.

## 2. The no-shrink reconciliation guard has a machine-switch blind spot
**Topics:** cc-workflow, skills

**Context:** The Step 7.5 guard verifies "missing" task codes against only the immediately-preceding RESUME-STATE's Completed/Dropped sections.

**Learning:** That assumes conv-tasks.md is regenerated every conv. On a machine-switch return the local file lags N convs, so the single latest RESUME-STATE cannot enumerate N convs of attrition → systematic false-halt. The git-history cross-check is the correct escape hatch (used manually this conv). This whole class of friction disappears once `[CURTASKS]` retires the machine-local file in favor of a git-tracked `CURRENT-TASKS.md`.

## 3. The spt CURRENT-TASKS.md model splits roles, it doesn't merge them
**Topics:** cc-workflow, docs-infra

**Context:** "B" was framed as "consolidate RESUME-STATE + conv-tasks into one file." The survey of spt showed otherwise.

**Learning:** spt keeps **three** roles distinct: `CURRENT-TASKS.md` = durable git-tracked task store (Ordered / Unordered+Parked / Completed-this-conv, `[TAG]`-keyed, hand-editable, never deleted); `RESUME-STATE.md` = narrative-only per-conv handoff (still deleted+rebuilt, keeps the Branch line); no machine-local conv-tasks.md at all. The refresh is a Claude-executed **preserve-then-overlay** (full Write reconstructed from the old file, never Edit/delete/reorder), checkpoint-refreshed (not live-synced), with an **active-only TodoWrite** model (r-start does not hydrate). Demoting rather than deleting RESUME-STATE is what keeps `conv-branch-check.sh` + `[RSTART-DIFFGATE]` working untouched.
