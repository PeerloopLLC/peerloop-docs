# Conv Learnings - 2026-06-25

## 1. Per-route effort tracks architecture, not line count
**Topics:** rg-admin, route-migration, conformance

**Context:** RG-ADMIN routes #11–#14 ranged 14 → 1257 lines, but effort didn't track length.

**Learning:** The Conv-334 "zero backward-pointer" predictor held all conv: effort is driven by (a) whether the route first-renders an *unconformed* shared primitive, and (b) whether it carries a hand-rolled `fixed inset-0` modal needing a structural FormModal migration. Certificates (604 ln) was *more* work than moderators (647 ln) purely because certificates had a custom modal; moderation (1257 ln across 4 files) was high-volume but purely mechanical because everything composed already-conformed primitives. Line count is a poor proxy — modal-presence + primitive-conformance state is the real predictor.

## 2. Cross-surface consistency by reference, not by reinvention
**Topics:** rg-admin, rg-mod, badge-policy

**Context:** `/admin/moderation`'s `ModerationDetailContent` had 3 category-badge helpers (reason/priority/content-type) with arbitrary orphan hues.

**Learning:** `/admin/moderation` and `/mod` show the *same flag data with the same badges*, so they must look identical. Rather than invent a hue→token mapping, read RG-MOD's already-conformed `ModeratorQueue` helpers and mirror them verbatim (priority→status tokens; reason mapped where valence is clear — harassment→error, spam→warning; content-type orphans kept honest — indigo/cyan/pink). The 9 test failures were a *good* signal — they confirmed the badges genuinely changed, and updating them to the values `ModeratorQueue.test` already asserts proves the two surfaces now share one vocabulary.

**Pattern:** When conforming a route that has a sibling already-conformed surface, grep the sibling's helpers and mirror them; treat resulting test-assertion failures as confirmation, and update them to the sibling's asserted values.

## 3. `grep -rl` false-positives from provenance breadcrumbs
**Topics:** provenance, tooling, marker-tally

**Context:** After flipping route #11, the admin marker tally read 10 `@stand-in` + 11 `@matt-inspired` = 21 across only 16 files.

**Learning:** Flipped pages carry a `@matt-inspired — … Was @stand-in` breadcrumb, so `grep -rl '@stand-in'` matches the explanatory suffix and a converted page looks unconverted. A robust tally reads only the *first* marker token on the doc-comment line — but cross-checking the count against an independently-derived "Remaining N" list resolved it without a fancier grep. Verifying the anomaly beat propagating a wrong README count.

## 4. The RG-ADMIN locked cycle has no per-route user pause
**Topics:** route-sweep, process, locked-playbook

**Context:** The general RTMIG-4 route-sweep protocol is "assess → surface → await refinements → do work → mark Swept", but RG-ADMIN's playbook was LOCKED (Conv 332) into "assess → conform → gate → DOM-verify → record".

**Learning:** Because the conformance rules (token vocabulary, 3 sub-patterns, hue mappings) are all pre-decided, per-route execution under the locked playbook is mechanical follow-through with zero novel decisions — so no per-route surface-and-pause is required (matches the Conv-334 rhythm). Even a 4-component / ~98-hue route warrants no check-in when every disposition maps to a locked rule; §Critical Rule "size is not the criterion" applies. The user picking a route IS the authorization.

## 5. High-volume mechanical conform: color replace_all + explicit size/weight
**Topics:** conformance, editing-strategy

**Context:** ModerationAdmin/DetailContent/SystemPromotions had ~98 scattered hue/size utilities.

**Learning:** Pure COLOR tokens (`text-gray-900`→`text-text-default`, etc.) map 1:1 and are safe to `replace_all`. SIZE+WEIGHT combos (`text-sm font-medium`→`text-body-small-medium`) are NOT safe for blanket `replace_all` (the weight-on-token rule), so handle those explicitly. Do avatar/button block-swaps and the error-card FIRST (they remove their own special colors), then color `replace_all` the rest, then explicit size edits.

**Pattern:** Re-read the file region after `replace_all`s before targeting size edits — the strings have changed, so stale `old_string` matches will fail.
