# Conv Decisions - 2026-06-28

## 1. Add 4 columns to `homework_submissions` vs repurpose `file_url`
**Type:** Implementation
**Topics:** d1

**Trigger:** `homework_submissions` had only `file_url` (external URL); R2 uploads need a key + metadata.

**Options Considered:**
1. Repurpose `file_url` to hold the R2 key
2. Add `r2_key` + `file_name` + `mime_type` + `file_size` columns ← Chosen

**Decision:** Add the four nullable columns, mirroring `session_resources`; keep `file_url` for external links (backward compatible).

**Rationale:** Follows the established resources pattern; preserves the external-link path; gives the download route everything it needs (key, content-type, filename) without a second fetch.

**Consequences:** Schema lands in `0001_schema.sql` (pre-launch); the dev DB needed an `ALTER` to catch up (see Learning 1).

## 2. Download Cache-Control — `no-store` for private submissions
**Type:** Implementation
**Topics:** auth, cloudflare

**Trigger:** Live DOM-verification reproduced a cross-user cache hit on the download URL.

**Options Considered:**
1. Keep `private, max-age=3600` (copied from the resources route)
2. `private, no-store` ← Chosen

**Decision:** `private, no-store` on the submission download route.

**Rationale:** Submissions are per-user private; resources are enrollment-gated (same bytes for all enrolled users), so `max-age` is tolerable there but not here. `no-store` forces the auth check on every request.

**Consequences:** Submission files are never browser-cached; locked with a test assertion.

## 3. Backend-complete cut; reviewer grading UI deferred
**Type:** Strategic
**Topics:** cc-workflow

**Trigger:** No React UI consumes the reviewer homework endpoints (list + grade are backend-only).

**Decision:** Ship the full backend + student UI now; surface `download_url` from the list endpoint so a future grading UI has it; track the reviewer grading UI as a separate task (#15 [HW-GRADE-UI]) rather than expand this conv.

**Rationale:** The access policy already covers creator/teacher, so the reviewer-download half is backend-ready; building the grading surface is a distinct piece of work.

**Consequences:** #15 created; the reviewer download is unsurfaced until that UI exists.
