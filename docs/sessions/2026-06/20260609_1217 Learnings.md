# Conv Learnings - 2026-06-09

## 1. The /profile scaffold precedent resolves the "shell now vs. full restyle" tension for SubNav-workspace ports
**Topics:** astro, matt-design, route-migration

**Context:** ROLE-STUDIOS ports must honor TWO co-equal obligations (faithful function + full Matt styling), but fully Matt-restyling a 200-line React island plus its shared components is multi-conv work on its own.

**Learning:** `/profile/[...tab].astro` (Conv 212) established the accepted intermediate landing: Matt-shell the workspace (AppLayout + SubNav + breadcrumb + Matt-token header) and embed the EXISTING islands, deferring faithful internal restyle to a separately-tracked task ([PROF-TAB-REDESIGN]). This is an *established pattern*, so following it is "proceed", not a novel decision — provided behavior is preserved (no drop) and the restyle is tracked. `/learning` followed it with [LEARN-ISLAND-RESTYLE].

**Pattern:** Reuse existing role islands inside a Matt shell; the DASH-GAP (a) differentiators that "must be preserved" are often already inside the island, so reuse preserves them for free.

## 2. node-fetch against the dev server catches the RTMIG route-wiring bug class without a browser
**Topics:** testing, astro, route-migration

**Context:** RTMIG history: string-valued route refs are invisible to tsc and "only the browser caught them" (404s). Wanted authed verification of a new SubNav workspace.

**Learning:** Astro dev SSR runs middleware + page redirect logic + island mount server-side, so an authed `fetch(..., {redirect:'manual'})` with a minted session cookie reproduces exactly the SSR pipeline that produces those 404/redirect bugs — and lets you grep the rendered HTML for shell markers (SubNav presence, correct per-tab island mount, active `aria-current`, sidebar entry). Hydration (client-only behavior) is the only residue a browser adds; for unchanged reused islands that's low-risk.

**Pattern:** Mint session via POST /api/auth/login → capture set-cookie → fetch protected routes with `{redirect:'manual', headers:{cookie}}` → assert status/location + grep body for shell markers.
