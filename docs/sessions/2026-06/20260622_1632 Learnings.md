# Conv Learnings - 2026-06-22

## 1. Carried line-count estimates can silently conflate sibling components
**Topics:** route-migration, matt-conformance, scoping

**Context:** [CR-COMMUNITIES] was filed as "~1,029 ln". The actual communities **tab** is ~453 ln (3 files: CreatorCommunities 206 + CommunityCard 86 + CreateCommunityModal 161). The "~1,029" had summed the whole `creators/communities/` directory, which also contains the 4 **CR-COMMUNITY-MGMT** files (CommunityManagement/CommunitySettings/CreateProgressionModal/ProgressionCard ≈ 1,044 ln) belonging to the sibling `/creating/communities/[slug]` route.

**Learning:** Before sizing a sweep unit, resolve the actual import graph / render site — don't trust a carried directory-level line count. A "monster" estimate may be two adjacent units bundled.

## 2. Substring grep gives false cross-cut signals in import-graph checks
**Topics:** tooling, scoping, grep

**Context:** `grep -rln "CommunityCard"` flagged `ExploreCommunityCard` + several discover tabs as importers, implying `CommunityCard` was shared (cross-cut risk). It is not — those were substring matches on the *name* `ExploreCommunityCard`.

**Learning:** To confirm a component is exclusive (no ripple), grep the precise import path (`from './CommunityCard'` / `from '...creators/communities/CommunityCard'`), not the bare name. The bare-name grep is a discovery aid, not an exclusivity proof.

**Pattern:**
```
grep -rlnE "from ['\"](\.\./)*(@components/)?creators/communities/CommunityCard['\"]|from ['\"]\./CommunityCard['\"]" src
```

## 3. DOM-verifying role-gated UI states needs the matching account
**Topics:** dom-verify, chrome-bridge, role-gating

**Context:** The creator-application form (`/creating/apply`) renders the **form** only for a NON-creator; a creator sees the "already-creator" state. Gabriel (creator) could verify already-creator but not the form.

**Learning:** For multi-state role-gated components, enumerate which account each state needs and dev-login the right one per state (used `newuser@example.com` / Alex Chen for the form). Restore the session to the original user afterward — `dev-login` swaps the shared `:4321` session cookie for ALL same-origin tabs the user has open.

## 4. (reaffirmed) Tailwind-v4 bridge spacing-collapse — translation direction
**Topics:** matt-conformance, spacing, tailwind-v4

**Context:** Legacy components written in standard Tailwind use `p-4`(=16px), `gap-6`(=24px), icon `w-4`(=16px). Under the bridge, in-set numbers {4,8,12,16,20,24,32,40,48,64} are LITERAL px, so `p-4`→4px (a 4× collapse); out-of-set numbers fall back to Tailwind's 0.25rem base and render at their standard px.

**Learning:** Translate each legacy `N` to the in-set number equal to its intended px (`p-4`→`p-16`, `gap-6`→`gap-24`, `w-4`→`w-16`, `h-8`→`h-32`, `h-12`→`h-48`); when the target px ∉ set use arbitrary `[Npx]` (`py-2.5`→`py-[10px]`, `w-11`→`w-[44px]`, `w-3.5`→`w-[14px]`).
