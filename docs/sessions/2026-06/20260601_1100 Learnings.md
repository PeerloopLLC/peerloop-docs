# Conv Learnings - 2026-06-01

## 1. MCP server registered mid-session is invisible to in-session `/mcp` until restart
**Topics:** cc-workflow, mcp

**Context:** Registered figma via `claude mcp add` mid-conv on M4Pro; `claude mcp list` (CLI) showed it immediately but the user's `/mcp` panel did not.

**Learning:** The in-session `/mcp` panel and the MCP tool list are both populated **once at session launch**. A server added mid-session is written to `~/.claude.json` (so the CLI sees it) but does not appear for OAuth or tool use until CC is relaunched. Figma MCP fundamentally cannot come online within the same conv it was registered in — both the OAuth step and tool availability need a fresh session. The doc's "register + auth in one conv, use the tools next" understates it: even the auth step needs the restart.

**Pattern:**
Register (`claude mcp add`) → relaunch CC → `/mcp` authenticate → tools live that session.

## 2. Figma MCP permissions travel via git; only registration + OAuth are machine-local
**Topics:** cc-workflow, mcp, dual-repo

**Context:** Setting up Figma MCP on M4Pro when M4 already had it.

**Learning:** The committed `.claude/settings.json` already allowlists the `mcp__figma__*` tools, so a new machine inherits the permissions through git. What's missing per-machine is the server registration (`claude mcp add` → `~/.claude.json`, project-scoped) and the OAuth token — both machine-local and outside the repo. This is why M4 had Figma working and M4Pro didn't despite a clean pull.

## 3. `/r-start` pre-computed snapshot is captured pre-pull — re-read resume context on a far-behind machine
**Topics:** cc-workflow, dual-repo

**Context:** The /r-start pre-computed snapshot was 13 convs stale (217 vs 230) because M4 had run Convs 218–230 unpulled; the Step 2 pull brought them all in.

**Learning:** The skill's pre-computed `!`-backtick context is captured **before** the Step 2 pull. On a machine that is far behind, the resume context shown at session start reflects the pre-pull state and must be re-read fresh post-pull to reflect actual current state.
