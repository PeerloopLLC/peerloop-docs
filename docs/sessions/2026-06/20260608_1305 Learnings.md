# Conv Learnings - 2026-06-08

## 1. AdminLayout role-redirect yielded a blank 200 — nested-component returns don't propagate as the HTTP response
**Topics:** astro, auth

**Context:** `[ADMIN-REDIRECT-BLANK]` — an authenticated non-admin hitting `/admin/*` got a blank 15-byte 200 instead of a 302.

**Learning:** A `return Astro.redirect()` inside a *layout* component (not the page top-level) halts that component's render into empty output rather than emitting the redirect Response. Only middleware or the page frontmatter can return a Response that becomes the HTTP response. Route-level authorization that must redirect belongs in `middleware.ts`, not a shared layout.

**Pattern:** Admin-role gate added to middleware after the session check:
```ts
if ((pathname === '/admin' || pathname.startsWith('/admin/')) && !session.roles.includes('admin'))
  return context.redirect('/')
```

## 2. View Transitions drop island-unique arbitrary utilities — confirmed again with `gap-[18px]` on /profile
**Topics:** astro, matt-design

**Context:** A newly added `gap-[18px]` reverted on `/profile` until a click; `py-[6px]` / `px-[20px]` did not.

**Learning:** An arbitrary Tailwind utility used ONLY by a `transition:persist` island can be missing from a destination route's CSS after a View Transition, silently reverting style until a re-render. Utilities also used by other components (Button's `py-[6px]`, app-wide `px-[20px]`) survive because they're in every route's CSS. The tell: a *container* property (gap) reverting → all children reposition at once.

**Pattern:** Inline the island-unique value (`style={{ gap: '18px' }}`) so it rides with the persisted DOM node. (Same class as the Conv 191 slideout-offset fix; see `memory/feedback_dom_truth_over_screenshots`.)

## 3. Moved `@stand-in` flips the porting topology from empty-destination to in-place
**Topics:** matt-design, cc-workflow

**Context:** User asked whether porting an existing `@stand-in` (post-move) is still tractable vs the old "build new page at an empty route" flow.

**Learning:** Under move-to-`@stand-in`, the legacy implementation lives at the target path, so the port edits it in place — the file is its own functional spec and names the island holding behavior. The "did I drop behavior?" diff source becomes the move commit (`git diff` vs the `@stand-in` baseline), not a sibling `/old` file. Lower drop-risk than re-authoring from scratch.

## 4. /old's reference value is largely superseded once a frozen pre-flip ref + git history exist
**Topics:** cc-workflow, dual-repo

**Context:** Matt phase-out + the preflip worktree prompted reconsidering the keep-/old-live rule.

**Learning:** `/old` served two jobs — behavioral reference (now better covered by the preflip worktree + git history) and live production rollback (already moot since nothing links to /old). For *code* archaeology the live-repo copy beats the worktree (current imports, zero setup); the worktree's value is runtime observation only.
