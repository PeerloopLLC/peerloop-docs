# Conv Decisions - 2026-06-20

## 1. Do not kill the orphan-looking Brave; reserve Chrome for the bridge
**Type:** Strategic
**Topics:** plato, cc-workflow

**Trigger:** Bridge listed a browser the user couldn't see; user said "destroy it."

**Options Considered:**
1. Force-kill the connected process
2. Investigate first, then act ← Chosen
3. Ignore

**Decision:** Investigated, found it was the user's active Brave (not a zombie); did not kill. Established the Brave-personal / Chrome-bridge-only reservation on M4Pro.

**Rationale:** The "invisible zombie" premise was falsified (40+ renderers, tabs from today). Killing would have lost the user's live session. Surfaced and corrected rather than executing the now-obsolete instruction.

**Consequences:** No kill. Reservation saved to memory. User re-logged the extension in Chrome to bring the bridge up.

## 3. Systemic radius fix — move scale into `@theme`
**Type:** Implementation
**Topics:** astro, matt-design

**Trigger:** ~32 bare rounded-N no-ops app-wide (incl. Card.astro).

**Options Considered:**
1. Move scale into @theme ← Chosen
2. var()-indirection rename mirroring spacing
3. Keep primitives, duplicate in @theme

**Decision:** Moved `--radius-{2,4,6,8,12,16,24}` from primitives `:root` into the `@theme` bridge block in `tokens-tailwind-bridge.css` (literal px); left a breadcrumb. Omitted `--radius-full`/`--radius-0` (v4 built-ins).

**Rationale:** Idiomatic Tailwind v4 single-source; the primitives were dead (0 var refs) so no migration risk.

**Consequences:** All bare rounded-N resolve app-wide; Card.astro residual cleared; RG-COMMS fully clean.
