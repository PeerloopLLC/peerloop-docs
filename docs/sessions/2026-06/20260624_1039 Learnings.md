# Conv Learnings - 2026-06-24

## 1. The display/glyph type regime now has 3 minted families (A avatar, C micro)
**Topics:** matt-design, typography, tokens

**Context:** AvailabilityCalendar had six sub-12px badges (`text-[9px]`/`text-[10px]`) below Matt's 12px scale floor — the case the §9.2c "Family C (micro-badge)" placeholder was minted to cover.

**Learning:** Family C (`text-display-micro` 10/400 + `-medium` 500, lh 1.0, ls 0) joins Family A (avatar initials). The "one token, snap 9→10" ruling follows the consistency-first precedent (no lower tier to snap to → a single member absorbs both sizes). Type does NOT borrow the spacing-axis "ties round up" rule — but here there is no competing tier, so a single mint is unambiguous.

**Pattern:**
A new display-regime member is defined in `tokens-typography.css` + a bridge utility in `tokens-tailwind-bridge.css` + documented in `09-typography.md` §9.2c. Verify the utility actually emits via `grep '.text-display-micro{' dist/**/*.css` after a build.

## 2. "keep only what has no token home" disambiguates status-colour conformance
**Topics:** matt-design, colour-tokens

**Context:** AvailabilityCalendar mixed sky (available), red (blocked), amber (series-end) status hues; the MyFeeds-style "verified-keep" precedent made it ambiguous whether to keep them all raw.

**Learning:** sky stays raw because the sparse semantic ramp {error, warning, success} has NO info/available token (Conv 319). red→error / amber→warning DO have homes, so they map. The same principle (keep iff no token home) that justified keeping sky also justifies mapping red/amber — they are not in tension. Bonus facts: `warning-500` == Tailwind `amber-700` exactly; `error-*` is the deliberately-tuned house carmine (Conv 296).

## 3. font-weight-on-token ≡ the token's weight variant (zero visual change)
**Topics:** matt-design, typography, conformance

**Context:** Phase-6 of RTMIG-RECON found ~55 `text-body-small font-medium`-style spots; the migration ledger's own rule forbids raw font-weight.

**Learning:** Because every body token is lh 1.0, `text-body-small font-medium` is pixel-identical to `text-body-small-medium`, and `text-body-default font-semibold` ≡ `text-body-default-bold`. So enforcing the rule is a pure code-hygiene normalization — the visual is unchanged, only the form differs. The "debatable" part was never the visual; it was whether to reopen "done" groups (answer: yes, enforce).

## 4. A grep "importer" can be a comment — verify render before classifying live/dead
**Topics:** conformance, audit-method

**Context:** SHARED-COLOUR audit. UserCardCompact's only "importer" was UserAvatar (architecturally backwards); Breadcrumbs' only non-`/old` reference was in `creating/communities/[slug].astro`.

**Learning:** Both turned out to be **comments**, not renders. A `grep -l ComponentName` match is a *mention*, not proof of use. For liveness, grep for the JSX usage (`<Component`) or confirm the match is not a doc-comment. This collapsed a 6-file "colour residual" finding down to 1 genuine live fix (ProgressBar).
