# Conv Decisions - 2026-06-22

## 1. Retired Course-Status Badge → neutral (not warning)
**Type:** Implementation
**Topics:** matt-conformance, ui-ux

**Trigger:** CreatorStudio + CourseEditor render a Published/Draft/Retired status badge; Published→success and Draft→warning are settled, but Retired (legacy gray) had no obvious map.

**Options Considered:**
1. neutral — grey, treats Retired as inactive/terminal ← Chosen (user-confirmed)
2. warning — amber, per the playbook's literal "archived→warning"
3. error — red, "removed/dead"

**Decision:** Retired → `bg-neutral-100 text-neutral-700`.

**Rationale:** Keeps Retired visually distinct from Draft (which is amber); retired reads as inactive, not attention-needed. Deviates from the literal "archived→warning" rule, hence surfaced for confirmation.

**Consequences:** Applied in CreatorStudio (Unit A) + CourseEditor header (Unit E); recorded in the SoT route-migration README.

## 3. "About PeerLoop" Info Box → brand (not info)
**Type:** Implementation
**Topics:** matt-conformance, ui-ux

**Trigger:** PeerLoopFeaturesTab has an indigo informational callout (InfoCircleFilledIcon + "About the PeerLoop Model").

**Options Considered:**
1. `info` ramp (blue) — generic system-info semantic
2. `brand` ramp (purple) — consistent with decorative-indigo→brand, and it's brand-identity content ← Chosen

## 4. Prerequisite-Type + "Required" Badges → semantic error/warning/success
**Type:** Implementation
**Topics:** matt-conformance, ui-ux

**Trigger:** HomeworkEditor "Required" badge (legacy red) and CourseEditor prerequisite chips (required/nice-to-have/not-required = red/yellow/green) had no Matt token.

**Decision:** Required→`error`; prereq required→`error`, nice_to_have→`warning`, not_required→`success`; pending→`warning`; publish-checklist tick→`success`/incomplete→neutral.

**Rationale:** These red/amber/green hues are semantic-emphasis with existing Matt tokens; map rather than keep raw-legacy. DOM-confirmed (Required→error-100/error-500, prereq chip→error-100/error-500, checklist tick→success-500).
