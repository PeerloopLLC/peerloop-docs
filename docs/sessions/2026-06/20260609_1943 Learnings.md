# Conv Learnings - 2026-06-09

## 1. client:only islands silently render nothing on error — SSR 200 doesn't prove paint
**Topics:** astro, testing

**Context:** TriageStrip is `client:only="react"`, so a curl of `/` returns 200 with only an island placeholder — a runtime error inside the island is indistinguishable from "caught up" (both render nothing).

**Learning:** For `client:only` islands, shell-level SSR verification (200 + island-wired) is necessary but not sufficient; the actual paint needs a browser. State this caveat explicitly rather than claiming "verified."

---

## 2. Self-collapsing child components make "harvest into a new island" trivial
**Topics:** astro, react

**Context:** PriorityHeader (`if cards.length===0 return null`) and NeedsAttention (`if items.length===0 return null`) both guard empty internally.

**Learning:** Because the harvested components self-collapse, the new TriageStrip orchestrator needs no empty-state logic or conditional mounting — render both unconditionally and trust them to disappear. The orchestrator owns only the data-load.

**Pattern:** TriageStrip orchestrator — a thin island that replicates only the data subset its harvested children need, relying on the children's self-collapse for empty states.

---

## 3. The route-honesty rule pre-stages the workspace builds
**Topics:** astro, cc-workflow

**Context:** Creator/Teacher islands already hardcode `/creating/*` and `/teaching/*` root links (404'd until the routes existed); AppLayout already SELECTed `is_creator`/`is_teacher`.

**Learning:** Because islands were written against the *intended* root URLs and the data was pre-fetched, standing up the real route retroactively fixes a cluster of dead links and needs only one-field sidebar wiring — building the page IS the fix.

---

## 4. The sandbox breaks `for` loops in inline Bash (broken-PATH eval)
**Topics:** cc-workflow, testing

**Context:** Verification curls inside a `for` loop all failed `command not found` (curl/grep/wc), while the same commands worked as flat statements. The loop body runs via a restricted eval with no PATH.

**Learning:** Move loops into a script file and run `bash script.sh` — the tool's command string then has no inline loop and bash executes normally.

**Pattern:** `.scratch/verify-*.sh` reusable SSR-verification scripts (login → cookie → fetch routes → assert) to dodge the inline-`for`-loop sandbox bug.
