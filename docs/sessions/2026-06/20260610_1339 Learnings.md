# Conv Learnings - 2026-06-10

## 1. The "post redesign" was a reuse-and-adapt job, not a build-new-primitives job
**Topics:** matt-design, feeds, reuse

**Context:** The Conv-258 POST-MATT spec hedged "new primitives likely needed: ReactionPill, comment-pill, the post shell itself."

**Learning:** All of them already existed from the Conv-184 Matt component extraction — `SocialPost` (shell), `AnalyticCount` (reaction/comment pill, incl. string-label comment summary), `CourseAnchor` (embedded entity card + 11 siblings), `UserIcon`, `IconLabelChip`. Always read the existing Matt primitive set before trusting a spec's "new primitive" list. The actual work collapsed to one guarded prop + a thin data-adapter.

---

## 2. Display-only vs interactive splits cleanly on aggregated-vs-native context
**Topics:** feeds, product-model, architecture

**Context:** `SocialPost` is pure-render; `FeedActivityCard` (6 consumers) is fully interactive. A naive swap would drop behavior (a failed port).

**Learning:** A post in the *aggregated* Home feed is a teaser shown out of its home → display-only + click-through-to-source is correct (and is the flywheel-aligned product choice). Interactivity belongs in the *native* feed where the post lives. This dissolves the "preserve 502 lines of interactivity" problem for POST-MATT — that work simply stays in `FeedActivityCard`, out of scope.

## 3. Peerloop's Course entity color IS green
**Topics:** matt-design, tokens

**Context:** Matt's complex-post frame draws a green "Learn More" CTA (#327D00 text / #E8F4DF bg); the spec asked whether Button needs a new green variant.

**Learning:** No. `--Course-Primary` = `--dark-green` (#327D00) and `--Course-Background` = `--pastel-green` (#E8F4DF). Button `variant="course"` already maps to exactly those, so CourseAnchor's default CTA renders Matt's green. The Figma `get_variable_defs` "Color/Background/Border" bound values matched these tokens. No new variant/token — confirmed live in the browser DOM.
