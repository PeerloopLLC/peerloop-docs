# Conv Learnings - 2026-06-25

## 1. Per-route sweep effort is dominated by shared-primitive conformance, not own-chrome size
**Topics:** route-migration, rg-admin, conformance

**Context:** Three RG-ADMIN routes swept this conv ranged from 171 ln (recordings) to 979 ln (sessions), yet all moved at similar speed.

**Learning:** The expensive part of a route sweep is conforming any *new* shared primitive it first-renders (Conv 333 paid that for FormModal + UserAvatar). All three Conv-334 routes were "zero backward-pointer" — every shared dep (`Admin*` primitives, `ConfirmModal`, `FormModal`, `RecordingLink`) was already conformed by earlier convs — so even the 979-line sessions route was just mechanical token swaps. **Predictor of per-route effort = whether it mounts an unconformed shared primitive, NOT line count.** The one remaining route to watch is `/admin/analytics` (mounts no `admin/*` island — different shape).

## 2. The admin "View X →" red-link bug is a copy-paste lineage, not coincidence
**Topics:** rg-admin, bug, conformance

**Context:** `text-red-600` deep-links surfaced again in TeacherDetailContent (×2) and SessionDetailContent (×1), after earlier appearances in Payouts/Users/Courses (×2).

**Learning:** These admin detail-panel components were cloned from one ancestor that used `text-red-600` for "View X →" links (red was the pre-Matt admin link colour). Per-route sweeps catch each copy individually, but a single `grep -rn 'text-red-600' src/components/admin` would flush the whole family at once. Logged as a candidate for [XCUT-BACKREF].

Canonical fix: `text-red-600 hover:text-red-700` → `text-info-500 hover:text-info-300`.

## 3. The bridge-shrink trap is set-membership-gated
**Topics:** conformance, spacing, tailwind-bridge

**Context:** Auditing spacing on RecordingsAdmin — needed to know which numeric utilities actually render shrunk.

**Learning:** The `@theme` bridge only re-aliases `--spacing-N` for N ∈ {4,8,12,16,20,24,32,40,48,64}. So `p-4`/`gap-4`/`px-4`/`p-8` render at literal 4/4/4/8px (shrunk ~4× from author intent), but `space-y-6`/`mt-1`/`px-2`/`py-3` are NOT in the set and render at normal Tailwind values. Conformance still normalizes ALL of them to literal-px Matt naming for consistency — but only the set-members are actual *visual* bugs today. A grep regex flagging any `-4/-5/-6` over-matches: `gap-4`/`mt-4`/`mb-8` are legitimate 4px-scale values post-conform.

## 4. conv-tasks.md machine-local staleness is the expected case on a skipped-conv machine
**Topics:** memory-discipline, dual-repo, skills

**Context:** /r-start Step 7.5 found conv-tasks.md was from Conv 328 (29 tasks) not the immediately-prior Conv 333 (26 tasks) — a 29→26 "shrink".

**Learning:** Because `.scratch/` is gitignored + machine-local, a machine that skipped several convs (M4 skipped 329–333, which ran on M4Pro) lags by the whole gap. The reconciliation guard assumes conv-tasks.md is from the *immediately* prior conv (so RESUME-STATE's one-conv ledger explains any shrink) — but cross-machine gaps make a "legitimate but unexplained-by-this-ledger" shrink the *expected* case. The safety net: RESUME-STATE (which DOES sync via commit) is authoritative for what's pending; conv-tasks.md is a regenerable view.

## 5. Carry-forward effort notes can mislead — verify before trusting "trivial"
**Topics:** route-migration, process

**Context:** RESUME-STATE described RecordingsAdmin as "already 0/0/0 — likely trivial."

**Learning:** "0/0/0" was the gate counts (tsc/check/lint clean), not forbidden-token counts — the file actually carried the full secondary/green/amber/red/primary/raw-type/raw-radii/bridge-spacing load. A clean gate says nothing about conformance debt. Read the file before trusting a carried "trivial" label.
