# Conv Learnings - 2026-06-24

## 1. Button's `primary` variant is already the admin accent (americana-blue #0777B6)
**Topics:** rg-admin, matt-design, button-primitive

**Context:** The `[ADMIN-CONF-POLICY]` wants admin CTAs in info-blue, NOT brand-purple. The legacy admin code used `bg-primary-600` / `bg-indigo-600` (both purple-ish). The concern was needing to mint a dedicated admin/info Button variant (the way RG-MOD minted warning/suspend).

**Learning:** Button's default `variant="primary"` renders `bg-text-primary` = `#0777B6` = the americana-blue = exactly `info-500`. The Matt "primary" in this design system is **blue, not purple** — the `primary-*` Tailwind ramp is a *separate* (purple) namespace. So admin action buttons adopt `<Button>` with NO new variant: `primary`=blue CTA, `default`=neutral, `warning`=amber, `danger`=red, `outlined`=external link. Confirmed by the `HOVER_GRADIENT_STYLE` comment (`rgb(7,119,182)`) and DOM (`rgb(7,119,182)` r39 on every conformed admin button).

## 2. The neutral ramp is sparse — {50,100,300,500,700,900} only
**Topics:** rg-admin, palette-fdn, tokens

**Context:** Restyling the dark admin sidebar needed hover/border/text greys on a charcoal background.

**Learning:** The PALETTE-FDN bridge (`tokens-tailwind-bridge.css`) exports `neutral-*` at only **50/100/300/500/700/900** — there is NO `neutral-200/400/600/800`. Using `bg-neutral-800` or `text-neutral-400` would be off-token (no utility generated). The dark-sidebar palette must use 900 (bg #1F1F1F) / 700 (#414141 hover+border) / 500 (#767676 muted) / 300 (#DADADA inactive text). The same constraint applies to brand/info/success/error/warning (all {100,300,500} only).

## 3. Shared `Admin*` primitives can surface unconformed mid-sweep
**Topics:** rg-admin, rg-mod, shared-primitives

**Context:** `/admin/topics` rendered 16 row-action buttons with `gray-` / `indigo-` classes despite the source island being clean.

**Learning:** RG-MOD (Conv 313) conformed AdminFilterBar/Pagination/DataTable/DetailPanel — but NOT `AdminActionMenu`, because ModeratorQueue doesn't use it. So it surfaced unconformed when the first admin *table* route (topics) was swept. Unlike the app-wide Footer (#26, out of RG-ADMIN scope), `AdminActionMenu` lives in `components/admin/` → it IS RG-ADMIN scope → conform in-place. Watch for more such primitives as routes start using them. It also carried 2 latent bridge-shrink bugs (`w-48` dropdown→48px, `w-4 h-4` icons→4px) — the `@theme` spacing-override shrink hits shared primitives that predate it.

## 4. DOM-truth via `getComputedStyle` beats screenshots for conformance verify
**Topics:** rg-admin, browser-verify, dom-truth

**Context:** The Chrome bridge screenshot capped at 952px wide and kept rendering the `<lg` mobile header even after `resize_window(1440)`, so the desktop charcoal sidebar never appeared in screenshots.

**Learning:** Don't rabbit-hole on screenshot sizing. `getComputedStyle` reads (sidebar bg, active-pill bg, token resolution, font sizes, primitive presence via `[data-prov-name]`) are authoritative and unaffected by viewport-capture quirks. Every conformance signal was confirmed via DOM reads; screenshots were supplementary. The `[data-prov-name="Input"]` selector is a reliable way to count adopted form primitives.
