## 1. /old-retirement recovery = git history (commit 608346a2), not an archive folder
**Type:** Strategic
**Topics:** route-migration, cc-workflow

**Trigger:** User asked whether a git commit is sufficient recovery or whether to package retired files into a folder before deleting.

**Options Considered:**
1. Git history + existing preflip snapshot (commit 608346a2) ← Chosen
2. Archive folder (move out of src/, commit, then dead code)
3. Both (belt + suspenders)

**Decision:** Recover from git history; the permanent anchor is commit `608346a2` (also live as the preflip worktree :4331). No `/_archive` folder.

**Rationale:** Git already preserves everything losslessly; an archive folder duplicates git, fights tsc/lint/build (excludes needed), rots, and diverges from the Conv-250 convention. Discoverability is solved once by a tombstone commit + a docs ledger, not by carrying dead code.

**Consequences:** Codified in `plan/route-migration/README.md § OLD-PORTED-CLEANUP`; deletions land as atomic tombstone commits with restore commands.
