# Conv Learnings - 2026-06-15

## 1. Test driftCheck docs drift in roll-ups, not detail
**Topics:** testing, docs-infra

**Context:** TEST-COMPONENTS / TEST-COVERAGE / TEST-PAGES reconciliation.

**Learning:** The per-file/per-subsection detail tables stay accurate (maintained incrementally), but the roll-up totals + section-header counts drift. TEST-COVERAGE's API subsections summed correctly to 237 while the header said 233 and summary 234. Reconcile the roll-ups against the detail + an authoritative `vitest run`, not every leaf.

**Pattern:**
`find tests/<cat> -name '*.test.ts*' | wc -l` per category; cross-check the runner's "Test Files N" line (it includes `src/**` co-located tests the docs never counted).

## 2. vitest counts src/ AND tests/
**Topics:** testing

**Context:** "Vitest Total" never reconciled to `find tests/`.

**Learning:** `vitest.config.ts` include = `['src/**/*.{test,spec}.*', 'tests/**/*.{test,spec}.*']`. 2 `src/__tests__/` files were run but never recorded in the docs. The runner's file count (401) is the source of truth, not a `tests/`-only `find` (399).

## 3. The townhall→system rename had three `#`-namespaces colliding
**Topics:** testing, cc-workflow

**Context:** PLAN-RENUM + townhall test cleanup.

**Learning:** `#NN` syntax is shared by volatile TodoWrite task-ids, GitHub PR numbers (`PR #26`), and upstream GitHub issue ids (`#22264`, `#63538`). A blind strip conflates them. Discriminate by digit-width (≤3 = task, 5 = issue) + `[CODE]`-adjacency. Likewise `townhall` in tests = cosmetic branding OR the load-bearing Stream group (`SYSTEM_STREAM_GROUP='townhall'`, dashboard-declared) — never blanket-replace.

**Topics:** testing, e2e

**Context:** Verifying townhall e2e edits.
