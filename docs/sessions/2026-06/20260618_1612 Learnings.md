# Conv Learnings - 2026-06-18

## 1. Obsidian hides a symlink whose real target is a dotfolder
**Topics:** obsidian, dual-repo, cc-workflow

**Context:** `_scratch → .scratch` symlink failed to appear in Obsidian even though its name has no dot.

**Learning:** Obsidian canonicalizes a symlink to its real path and then applies the dotfile-hiding rule. A non-dot symlink pointing at a dotfolder is still hidden. To surface a dotfolder in Obsidian, the REAL directory must be non-dot; use a dot-named compat symlink for back-references. Verified on a vault with no Obsidian Sync.

**Pattern:** real `_scratch/` dir + `.scratch → _scratch` symlink; both gitignored; machine-local (re-create the flip per machine).

## 2. @matt-source primitives carry the feed's visible type — conformance must reach into them
**Topics:** typography, matt-design, cc-workflow

**Context:** RG-HOME's ledger marked FeedPost/CourseAnchor "Type ☑", but DOM showed raw `text-[Npx]`/`font-*`.

**Learning:** A wrapper component's "Type ☑" can be hollow — the visible type lives in the `@matt-source` sub-primitives it composes (`SocialPost`, `EntityLink`, `EntityPill`, `IconLabelChip`). Conformance verification must walk to where the text is actually styled, not stop at the wrapper. Led to the @matt-source conformance policy (Decision 2).

## 3. Tailwind v4 `space-y-*` applies margin-bottom, not margin-top
**Topics:** astro, testing, cc-workflow

**Context:** `space-y-16` container showed `margin-top: 0px` on every child — looked like a spacing-collapse bug.

**Learning:** Tailwind v4 implements `space-y-N` via `margin-bottom` on `:not(:last-child)`, not `margin-top`. Read the *rendered gap* (`getBoundingClientRect` deltas), not `getComputedStyle().marginTop`, to verify vertical rhythm. Rendered gaps were a clean 16px.

## 4. `text-body-medium-bold` encodes the Figma tracking exactly
**Topics:** typography, matt-design

**Learning:** `--body-medium-bold-letter-spacing: -0.022em` = `-0.352px` at 16px — the exact value `SocialPost` carried as `tracking-[-0.352px]`. Tokenizing the author name to `text-body-medium-bold` absorbed size + weight + tracking with zero visual change. The token IS the Figma spec.

## 5. ripgrep `-h` is `--help`, not `--no-filename`
**Topics:** cc-workflow

**Learning:** `rg -ohN` printed ripgrep's help instead of searching, because `-h` = help in rg (no-filename is `-I`). When `-o` output looks like a man page, check for a `-h` flag collision. Re-ran with `-oIN`.
