# Conv Decisions - 2026-06-24

## 1. Adopt `<Button>` for admin action buttons (the route-by-route button pattern)
**Type:** Implementation
**Topics:** rg-admin, button-primitive, relaxation-c

**Trigger:** The first route-by-route route (`/admin/payouts`) had 6 inline `bg-indigo-600` / `bg-gray-600` / etc. action buttons; relaxation C says "adopt Button for primary actions."

**Options Considered:**
1. Recolor the inline `<button>`s indigo→americana-blue (lighter touch)
2. Adopt `<Button>` with existing variants (primary=blue, default, warning, outlined) ← Chosen

**Decision:** Adopt `<Button>`. Maps: money-actions→`primary`, cancel→`default`, retry→`warning`, external deep-link→`outlined`. No new variant needed (primary IS the americana-blue #0777B6).

**Rationale:** Durable, design-system-consistent, matches the RG-MOD precedent; the americana-blue=primary discovery made it zero-cost. Sets the pattern for all 16 admin routes.

**Consequences:** LOCKED sub-pattern (a). Applied to all 4 routes done this conv.

## 2. Admin forms adopt `form/Input` / `form/Textarea` / `form/Select`
**Type:** Implementation
**Topics:** rg-admin, form-primitives, relaxation-c

**Trigger:** `/admin/announcements` is the first form-heavy admin route (inline authoring form with title/body/CTA/datetime/checkbox).

**Options Considered:**
1. Conform the shared inline `inputClass` / `labelClass` tokens (preserves layout, stays admin-tight 12px)
2. Adopt `form/Input` + `form/Textarea` primitives ← Chosen

**Decision:** Adopt the form primitives. `datetime-local` forwards through the primitive's `...rest` spread (DOM-verified rendering inside the Input wrapper); the checkbox stays inline-conformed.

**Rationale:** Matches every other app form, dedups hand-rolled input classes, inherits future Input improvements. Inputs render at 14px (the primitive default) — correct for form fields, not a density violation.

**Consequences:** LOCKED sub-pattern (b). Applied to announcements + the topics modal.

## 3. Admin modals adopt `ui/Modal`
**Type:** Implementation
**Topics:** rg-admin, modal-primitive

**Trigger:** `/admin/topics` has a hand-rolled `TopicModal` (bespoke backdrop/card/header) with custom auto-slug logic.

**Options Considered:**
1. Conform the inline modal tokens, keep the hand-rolled chrome
2. Adopt `ui/Modal` for the chrome + `form/Input` for fields + Button footer ← Chosen

**Decision:** Adopt `ui/Modal` (clean `isOpen` / `onClose` / `title` / `maxWidth` API; provides conformed backdrop + header; children area unpadded so body/footer add their own padding). The custom slug-gen logic is preserved.

**Rationale:** Consistent with the prior two "adopt primitive" choices + §Solution-Quality durable default; `ui/Modal` already handles Escape/backdrop/scroll-lock.

**Consequences:** LOCKED sub-pattern (c). FormModal was NOT used (its declarative fields don't fit the custom slug auto-gen).
