# Conv Learnings - 2026-06-20

## 1. Reserved-Chrome-for-bridge workflow (M4Pro)
**Topics:** plato, cc-workflow, dual-repo

**Context:** The `/chrome` bridge listed a connected "Browser 1" the user couldn't see; a long investigation nearly treated it as a kill target.

**Learning:** On M4Pro the user runs **Brave for all personal browsing** (hands off — never drive/kill it) and **reserves the Google Chrome app exclusively for `/chrome` bridge testing**. "If you see Chrome running it is serving a Claude Code session." Brave (`/Applications/Brave Browser.app`) and Chrome (`/Applications/Google Chrome.app`) are distinct at the pid level — a `Helper`-filtered process read can falsely look like a zombie. Saved to `reference_chrome_bridge_island_stale_cache`.

## 2. Chrome-bridge dead transport is fixed by re-login in the extension
**Topics:** plato

**Context:** Conv 310 saw the bridge transport go dead after a CC re-login; the suspected (untested) fix was restarting Claude Code.

**Learning:** This conv the transport was alive but Chrome wasn't registering (`list_connected_browsers` returned `[]` even after Chrome started). The actual fix was **re-logging into the Claude extension inside Chrome** — not a CC restart. After re-login, the Chrome entry appeared and the bridge worked.

## 3. The `--radius-*` scale was dead — radius must live in `@theme`, not `:root`
**Topics:** astro, matt-design

**Context:** Every bare `rounded-8/12/16` in the app computed 0px (square `<Card>`s app-wide), including `ui/Card.astro:40`.

**Learning:** Tailwind v4 generates `rounded-<key>` utilities from `--radius-<key>` **declared in an `@theme` block**. The scale was defined in `tokens-primitives.css :root` (not `@theme`) and `var(--radius-N)` had 0 references — so the tokens were dead and no utilities were generated. Mirrors the Conv-174 spacing-bridge mechanism (`@theme --spacing-N: var(--space-N)`).

**Pattern:** Move the numeric scale into the `@theme` block in `tokens-tailwind-bridge.css`; omit `--radius-full`/`--radius-0` (v4 built-ins).

## 4. `javascript_tool` blocks objects whose values trip a sensitive-key filter
**Topics:** plato

**Context:** Returning a forbidden-token tally object from `javascript_tool` came back as `"[BLOCKED: Sensitive key]"` (also hit when returning a `--color-*` token map).

**Learning:** The Chrome-extension result serializer redacts some objects with dynamically-built keys. DOM-truth probes that aggregate should **stringify** results.

**Pattern:** Return a joined string instead of a raw object, e.g. `Object.entries(o).map(([k,v])=>k+'×'+v).join(', ')`.

## 5. The system community feed is admin-only
**Topics:** matt-design

**Context:** `/community/system/feed` 404'd ("Community not found") for member guy-rymberg.

**Learning:** `community/[slug]/[...tab].astro:106` gates `isSystem && !isAdmin → notFound`. SystemFeed renders there only for admins; its reach to members is via other surfaces. To DOM-verify SystemFeed in `/community`, dev-login as an admin (`admin@peerloop.com`).
