# CD-039: Single-Column "Twitter-style" Listings

**Date:** 2026-06-14
**Source:** Client directive (paraphrased; see [original.txt](./original.txt))
**Type:** UI/Layout change + new feature (right panel) + image-handling clarification

---

## Purpose

The client wants listing surfaces to abandon the multi-column card **grid** in favor of a
**single, centered column** — one card per row, even at desktop widths — explicitly modeled
on Twitter/X. Capping card width creates right-side whitespace, which the client wants filled
by a **right panel** (placeholder for now, light-blue background). Separately, the client wants
images to **retain their aspect ratio at any size**.

---

## Decision 1: Single centered column for listings

> "Move away from a grid layout for listings and instead use a mobile-like presentation:
> 1 card per row, even at desktop sizes… like how Twitter does this. Cards are always in
> the center going down the screen."

- Replace `grid sm:grid-cols-2 xl:grid-cols-3` (and siblings) with a single centered column
  at a capped max-width.
- The personal feed surfaces (`HomeFeed.tsx` on `/feed` and `/`) **already** use this pattern
  (`space-y-4`, single column, infinite scroll) — they are the in-repo reference implementation.

**Affected grid surfaces (from Conv 284 inventory):**

| Surface (route) | Component | Current grid |
|---|---|---|
| `/communities` | `CommunitiesCatalog.tsx` | `grid-cols-1 sm:grid-cols-2 xl:grid-cols-3` |
| `/courses` | `CoursesCatalog` (+ `CoursesFilters` left rail) | `grid sm:grid-cols-2 xl:grid-cols-3` |
| `/members` | `MemberDirectory.tsx` | `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` |
| `/feeds` | `FeedsDiscoveryGrid.tsx`, `FeedAllTab.tsx`, `FeedRoleTab.tsx` | `grid sm:grid-cols-2` |
| Discover course tabs | `Explore{All,Teaching,Student,Created,Moderation}Tab.tsx` | `grid sm:grid-cols-2 xl:grid-cols-3` |
| Discover community tabs | `Community{All,Created,Member,Teaching,Moderation}Tab.tsx` | `grid sm:grid-cols-2` |
| `/feed`, `/` (reference) | `HomeFeed.tsx` | already single column ✅ |

## Decision 2: Right panel to absorb whitespace

> "What to do with the extra whitespace on the right, because cards must not get too wide.
> The solution he proposes is a right panel, currently unfilled, with a light blue background."

- No persistent right-rail component exists today. The app has **left** filter sidebars
  (`w-64 shrink-0`) and **right-edge slide-in modals** (`AdminDetailPanel`) — neither is a
  Twitter-style persistent right column.
- Net-new: a right panel, light-blue background, empty placeholder for now.
- **Resolved (2026-06-14):** the panel is **sticky** and **hosts the page's filters when present**
  (relocating `CoursesFilters` + the Explore-tab filter rails off the left into the right panel);
  on pages without filters it shows the light-blue placeholder. One shell, content varies per page.

## Decision 3: Images retain aspect ratio at any size

> "He wants any images to retain aspect ratio at any size."

- **Resolved (2026-06-14):** client clarified the intent is a **standardized responsive frame**,
  not intrinsic/variable heights. Source images vary in height; the card image area is one shared
  proportion that scales with screen size. Chosen ratio: **16:9** (matches existing `aspect-video`),
  rendered with `object-cover` (scales proportionally, no distortion). Creators get upload guidance
  to supply ~16:9 images ("they may require a certain size image") so cropping is negligible.

---

## Implied Goals

- A more focused, readable, mobile-first reading experience for discovery (parity with the
  feed's single-column timeline).
- A consistent page shell across feed + discovery surfaces.
- A future home for contextual/secondary content (the right panel).

## Implied User Stories

- As a user browsing communities/courses/members/feeds, I see one card per row in a centered
  column so the layout matches the familiar feed timeline.
- As a user on a wide screen, I see a right panel instead of stretched cards or dead whitespace.
- As a user viewing card images, images are shown without distortion at any viewport size.

## Technical Implementation Notes

- Likely a **new shared layout primitive** (a centered-column + right-rail shell), `@matt-inspired`,
  reused across the grid surfaces — rather than editing ~15 components ad hoc.
- The single-column conversion itself is largely mechanical (swap grid utilities for a capped
  centered container + vertical spacing like `HomeFeed`'s `space-y-4`).
- Right panel hidden below a breakpoint so mobile is a pure single column.
- Decide interaction with existing **left** filter rails on `/courses` + Explore tabs
  (2-column center+right vs 3-column left+center+right).

## Comments

**2026-06-14 (Conv 284)** — All 6 open questions resolved live with client:
1. Scope = **all six surface families**.
2. Right panel = **filters when present, else placeholder**; sticky.
3. Column max-width = **~640px**.

5. Left filter rails = **relocate into the right panel**.
6. Mobile = **collapse side panels below `lg`** → pure single column.
