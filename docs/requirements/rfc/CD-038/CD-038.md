## Thread Summary

### Question 1: Download recordings in video format

**Answer (Fred Dixon, 2026-03-26):**

- Use `getRecordings` API to find recordings. The response includes a video playback format.
- Format: `.m4v` (MPEG-4 Video)
- Two-step download process (cookie-based auth):
  1. Open the "capture" URL to obtain a session cookie:
     `https://recordings.rna1.blindsidenetworks.com/bn/{record_id}/capture/`
  2. Use that cookie to download the actual file:
     `https://recordings.rna1.blindsidenetworks.com/bn/{record_id}/capture/capture-0.m4v`

**Key detail:** You must fetch the first URL to get the cookie, then pass that cookie when fetching the second URL. Direct download of the `.m4v` without the cookie will fail.

### Question 2: Session statistics and call data

**Answer (Fred Dixon, 2026-03-26):**

> "We can do better -- we can give you a callback of all that data after every session (just create an endpoint that we can send the data to)."

- Blindside Networks provides a **Hosting Analytics Callback** system
- Pass `meta_analytics-callback-url` as a meta parameter on the `create` API call
- After a meeting ends, BBB processes all events and POSTs a JSON analytics payload to that URL
- Authentication: JWT Bearer token (HS512, signed with BBB shared secret)
- See attached PDF for full specification

**Per-user data available:**
- Chat count, talk count, emoji count, raise-hand count
- Poll responses
- Full name, external user ID, moderator status
- Join/leave timestamps (supports multiple join/leave cycles)
- Total time in session
- Talk time (seconds)

**Session-level data:**
- Meeting duration, start/finish timestamps
- Files shared, polls conducted
- Full metadata echo-back

### Question 4: Account settings for stored data

**Answer (Fred Dixon, 2026-03-26):**
