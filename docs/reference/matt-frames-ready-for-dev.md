## How Matt's Status Frame works (probed Conv 186 via node `725:15176`)

Each frame on the Components page (and other pages, presumably) has a sibling **Section Title** instance acting as a status banner above it. The banner is 1280×96px, TT Norms Pro Mono font, and contains three editable text nodes:

| Sub-node ID (within banner) | Field | Notes |
|---|---|---|
| `722:14806` | Title (section name) | e.g., "Icons" |
| `722:14807` | Status label | One of: `WIP`, `Ready For Dev`, `Archived` |
| `723:14868` | Last Touched (date) | Matt's raw format, e.g., `May 20, '26` |

Banner color encodes status:
- Green (`#5db54f` / `#138801` border) = **Ready For Dev**
- (WIP / Archived colors to be verified on first such probe)

## How to add a frame (input format)

User pastes per frame:

```
Frame:  <URL of the actual content frame>
Status: <URL of the status banner above it>
```

Optional third line `Name: <preferred display name>` if the display label should differ from Matt's banner title. Multiple frames in one message — separate with blank lines. Everything else (node IDs, section name, status, Last Touched date, etc.) is derived from the probe.

## Drift-check workflow

When user says "check Matt frames" or similar:

1. For each row below, call `mcp__figma__get_design_context(fileKey, statusBannerNode)`.
2. Parse `title`, `status`, `date` from the returned JSX.
3. Normalize `date` to ISO (e.g., `May 20, '26` → `2026-05-20`).
4. Compare normalized date to stored `Last Touched (ISO)`:
   - Same → silent (or one-line "checked, unchanged" summary at end).
   - Newer → 🔴 surface drift; probe the actual frame; A/B/C decide whether to update app components.
   - Status changed (WIP/Ready/Archived transition) → 🟠 also surface even if date is same.
5. **Side-effect:** Every deep probe scans `<img data-name="...">` instances → any unharvested Material icon → auto-create `[<ICON>-ICN]` TaskCreate.
6. Append to "Drift Check History" below.

Decision axis is **addressability** (does a screen need a jump-to / deep-link / redirect URL), NOT page-count (one state-driven page vs separate files = deferrable build detail).
