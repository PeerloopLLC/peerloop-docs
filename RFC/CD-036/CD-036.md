# CD-036: Communities, Progressions & Feeds

**Source:** Session 169 verbal notes
**Date:** 2026-02-03
**Priority:** High (foundational data model change)

---

## Summary

Introduces a new content hierarchy: **Community → Progression → Course**. Progressions become first-class entities for discovery, displayed as "Learning Paths" (multi-course) or "Stand-alone Courses" (single course). Clarifies feed scoping across four contexts.

---

## Content Hierarchy

```
Creator
  └── Community (topic-focused, e.g., "n8n Automation")
        └── Progression (learning journey or standalone)
              └── Course (1-10 BBB sessions)
```

**Key Rules:**
- Every course belongs to exactly ONE progression
- Every progression belongs to exactly ONE community
- Progressions are the primary discovery unit (not individual courses)
- Users purchase individual courses, but browse progressions

---

## Key Concepts

### 1. Communities

**Definition:** Topic-focused groupings with a common feed.

- Creators can create **multiple** communities
- Each community has its own feed
- Contains progressions (which contain courses)
- Examples: "n8n Automation", "Python for Data Science", "No-Code Tools"
- Platform courses belong to system "Peerloop" community

### 2. Progressions

**Definition:** A learning journey containing one or more courses.

| Type | Course Count | Display | Badge |
|------|--------------|---------|-------|
| Learning Path | 2+ | Shows sequence of courses | `Learning Path` |
| Stand-alone | 1 | Shows single course | `Stand-alone` |

**Progression order is advisory, not enforced:**
- Users can start at any course in the progression
- UI shows "Recommended: Complete X first" but doesn't block
- Experienced users may skip introductory courses

**To reuse course content in multiple progressions:**
- Copy the course (new ID, shared resources)
- Each copy belongs to its own progression

### 3. Courses

**Definition:** A series of 1-10 BBB (BigBlueButton) sessions in a set sequence.

- Not a self-paced video library
- Structured, scheduled live sessions
- Has its own feed for enrolled students
- Position within progression is for display order

### 4. Feeds

Feeds are filtered Stream.io feeds scoped to one of four contexts:

| Feed Type | Scope | Who Can Post | Who Can View |
|-----------|-------|--------------|--------------|
| **Personal** | One person | That person | TBD (followers?) |
| **Community** | One community | Members only | TBD (design for flexibility) |
| **Course** | One course | Enrolled + Creator/S-T | Enrolled students |
| **The Commons** | Everyone | Anyone | Everyone (including visitors) |

**Community membership:** Derived from enrollments. If you've purchased any course in a community, you're a member.

## Data Model

### New Entities

```sql
communities
├── id (TEXT PRIMARY KEY)
├── creator_id (FK → users, NOT NULL)
├── name (TEXT NOT NULL)
├── slug (TEXT UNIQUE NOT NULL)
├── description (TEXT)
├── cover_image_url (TEXT)
├── is_public (BOOLEAN DEFAULT true) -- for feed viewing (TBD)
├── is_system (BOOLEAN DEFAULT false) -- true for "Peerloop" community
├── is_archived (BOOLEAN DEFAULT false)
├── created_at (TEXT NOT NULL)
└── updated_at (TEXT NOT NULL)

progressions
├── id (TEXT PRIMARY KEY)
├── community_id (FK → communities, NOT NULL)
├── creator_id (FK → users, NOT NULL)
├── name (TEXT NOT NULL)
├── slug (TEXT NOT NULL) -- unique within community
├── description (TEXT)
├── tagline (TEXT) -- e.g., "Master automation in 3 steps"
├── cover_image_url (TEXT)
├── difficulty_level (TEXT) -- beginner/intermediate/advanced
├── estimated_duration (TEXT) -- e.g., "6 weeks"
├── is_archived (BOOLEAN DEFAULT false)
├── created_at (TEXT NOT NULL)
└── updated_at (TEXT NOT NULL)
```

### Modified Entities

```sql
courses
├── id (TEXT PRIMARY KEY)
├── progression_id (FK → progressions, NOT NULL) -- required!
├── progression_position (INTEGER NOT NULL) -- 1, 2, 3...
├── source_course_id (FK → courses, nullable) -- if copied
├── name (TEXT NOT NULL)
├── slug (TEXT NOT NULL)
├── description (TEXT)
├── session_count (INTEGER NOT NULL) -- 1-10 BBB sessions
├── price (INTEGER) -- in cents
├── is_archived (BOOLEAN DEFAULT false) -- soft delete
├── created_at (TEXT NOT NULL)
└── updated_at (TEXT NOT NULL)

-- Note: community_id removed from courses
-- Community is derived via: course → progression → community
```

### Course Copying

When a course is copied:
- New course ID created
- `source_course_id` points to original
- Resources (BBB recordings, materials) are **shared by reference** initially
- Creator can update resources independently later
- Copy belongs to its own progression

### Derived (No Table Needed)

```sql
-- Community members = users with enrollment in any course of that community
SELECT DISTINCT e.user_id
FROM enrollments e
JOIN courses c ON e.course_id = c.id
JOIN progressions p ON c.progression_id = p.id
WHERE p.community_id = ?
```

## Display Logic

### Badge Assignment (UI Layer)

```typescript
function getBadge(progression: Progression): string {
  const courseCount = progression.courses.length;
  return courseCount === 1 ? 'Stand-alone' : 'Learning Path';
}
```

## Feed Architecture

Stream.io feed types:

| Feed | Stream Feed ID Pattern | Purpose |
|------|------------------------|---------|
| Personal | `user:{user_id}` | User's posts |
| Community | `community:{community_id}` | Community feed |
| Course | `course:{course_id}` | Course announcements |
| Commons | `commons:global` | Platform-wide |

## Comments

### 2026-02-04 - Session 181 Clarifications

**Community/Feed Simplification:**

| Decision | Detail |
|----------|--------|
| 1:1 Community:Feed | Each community has exactly 1 feed. The community page IS the feed. |
| Tags replace feeds | Announcements, Help & Support are post tags, not separate feeds |
| The Commons | PeerLoop community's feed, with posts tagged as general/announcements/help |

**Feed Hierarchy (Finalized):**

```
Creator
├── Creator Feed (1) ← IFED page
└── Communities (many)
    ├── Community Feed (1) ← posts tagged: help, announcements, general
    └── Courses (many)
        └── Course Feed (1 per course)
```

**Routing Updates (per tech-021 singular convention):**

| Old (CD-036) | New (Session 181) | Rationale |
|--------------|-------------------|-----------|
| `/communities/[slug]` | `/community/[slug]` | Singular for resources |
| `/communities/[slug]/feed` | `/community/[slug]` | Community IS the feed |
| `/communities` | `/community` | My Communities hub (like /discover hub) |

**My Communities Hub (`/community`):**
- Mirrors the My Communities SlideOut Panel
- The Commons at top (always)
- Separator
- Other joined communities below
- Same pattern as `/discover` hub mirroring Discovery SlideOut

**My Feeds Section in AppNavbar:**
- My Communities → slideout → `/community` hub
- Creator Feeds → creators I follow → `/creator/[handle]/feed`
- Course Feeds → from enrollments → `/course/[slug]/feed`
